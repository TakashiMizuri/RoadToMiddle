# LINQ Where, Select, OrderBy, GroupBy — краткая выжимка

> `1.3.7` · ~8 мин на повторение

## Главная идея

**LINQ method syntax** — composable pipeline над **`IEnumerable<T>`**: **`Where`** (filter), **`Select`** (project/map), **`OrderBy`/`ThenBy`** (sort), **`GroupBy`** (partition как SQL **GROUP BY**). Каждый шаг возвращает новую lazy sequence; execution откладывается до **`foreach`/`ToList()`** (`1.3.9`–`1.3.10`).

## Ключевые пункты

- Operators — **extension methods** в **`System.Linq.Enumerable`**; lambdas (`1.3.6`) → **`Func<T, bool>`**, **`Func<T, TResult>`**.
- **Порядок operators меняет semantics**: `.Take(10).GroupBy()` ≠ `.GroupBy().Take(10)`.
- **Filter early**: **`Where`** до **`Select`**, если filter на source fields.
- **`OrderBy`** → **`IOrderedEnumerable<T>`**; secondary sort только через **`ThenBy`**, не второй **`OrderBy`**.
- **`GroupBy`** → **`IEnumerable<IGrouping<TKey, TElement>>`**, не dictionary; lazy, может re-run.
- **`OrderBy`/`GroupBy`** internally buffer — не «бесплатные» iterators как **`Where`**.

## Самые важные примеры

```csharp
var dtos = products
    .Where(p => p.IsActive)
    .Select(p => new ProductDto(p.Id, p.Name))
    .OrderBy(p => p.Name)
    .ToList();
```

```csharp
var byRegion = sales
    .Where(s => s.Year == 2026)
    .GroupBy(s => s.Region, (region, items) => new { region, Total = items.Sum(x => x.Amount) })
    .OrderByDescending(x => x.Total);
```

## «Под капотом» в одном абзаце

Operators оборачивают source в iterator classes (**`yield return`**). Pipeline хранит delegates с possible **closures** — captured variables читаются при enumeration, не при построении. Compiler infer generic types на каждом **`Select`**. Те же method names на **`IQueryable<T>`** строят **expression trees** для EF (`1.3.11`–`1.3.12`) вместо in-memory loops.

## Частые ошибки

- **`Select` then `Where`** на поля, уже спроецированные away.
- Side effects в **`Select`**/`Where`.
- **`OrderBy` + OrderBy** вместо **`ThenBy`**.
- Повторный **`foreach`** на **`GroupBy`** без materialize.

## Мнемоника

> **W**here фильтрует, **S**elect меняет shape, **O**rder сортирует, **G**roup собирает buckets — порядок: **W → G → S → O → page**.
