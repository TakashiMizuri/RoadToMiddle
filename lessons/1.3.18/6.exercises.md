# Exercises: Index and Range

> Complete after passing `4.test-yourself.md`.

## Easy

### E1. Last and first
**Goal:** Given `int[]`, print first (`[0]`) and last (`[^1]`) without `Length - 1`.
**Success criteria:** Works on length 1 array.

### E2. Head and tail
**Goal:** Split array into `head = a[..2]` and `tail = a[2..]` for demo array.
**Success criteria:** Concatenate conceptually recovers original.

### E3. String ends
**Goal:** `"filename.txt"` → name `[..^4]`, extension `[^3..]` (assume fixed `.txt`).
**Success criteria:** Comment on length assumptions.

## Medium

### M1. Drop first and last
**Goal:** `int[] data` → inner `[1..^1]`; handle length < 3 safely.
**Success criteria:** No throw on small arrays (guard before slice).

### M2. Span version
**Goal:** Same slice on `ReadOnlySpan<char>` from string — no substring allocation until print.
**Success criteria:** Use `AsSpan()` + range.

### M3. Last N lines
**Goal:** Split text by `\n`, take last 3 lines via index/range (materialize lines array first).
**Success criteria:** Uses `^3..` or similar on line array.

## Hard (stretch)

### H1. Parse last path segment
**Goal:** Extract `"42"` from `"/api/users/42"` using `LastIndexOf` + range only (no Split).
**Success criteria:** Single segment after final `/`.

### H2. Benchmark (optional)
**Goal:** Compare `Substring` vs `AsSpan()[range].ToString()` for 1M iterations — note when Span wins.
**Success criteria:** One paragraph result.

## Submission checklist

- [ ] Used `^1` at least once
- [ ] Used open-ended range `..` or `[n..]`
- [ ] One exercise documents exclusive end
