# BackgroundTaskQueue — краткая выжимка

> `1.4.34` · ~10 мин на повторение

## Главная идея

**`Channel` + `BackgroundService`** — безопасная очередь фоновой работы в ASP.NET: bounded backpressure, **новый DI scope** на job, **await** с логированием ошибок.

## Ключевые пункты

- `IBackgroundTaskQueue` — enqueue `Func<IServiceProvider, CancellationToken, ValueTask>`.
- **`BoundedChannelFullMode.Wait`** — producer ждёт при переполнении.
- Consumer: **`CreateAsyncScope()`** → resolve → **await work**.
- В closure — **примитивы/ID**, не scoped services.
- Singleton queue + `AddHostedService`.
- In-process; crash = потеря очереди → outbox/broker для critical.

## Самые важные примеры

```csharp
await _queue.QueueBackgroundWorkItemAsync(async (sp, ct) =>
{
    await sp.GetRequiredService<IMailer>().SendAsync(id, ct);
});
```

## «Под капотом» в одном абзаце

HTTP handler пишет delegate в channel и возвращает ответ. Hosted loop читает, создаёт scope (fresh DbContext), await'ит delegate. Exception ловится в consumer — не unobserved. Bounded channel блокирует enqueue при перегрузке — backpressure вместо OOM от unbounded fire-and-forget.

## Частые ошибки

- Захват `DbContext` из request в closure.
- Unbounded channel.
- Один consumer + долгие задачи без parallel workers.

## Мнемоника

**«Queue the ID, scope the work, await the task»**
