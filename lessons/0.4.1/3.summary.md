# Модель OSI vs TCP/IP — краткая выжимка

> `0.4.1` · ~8 мин на повторение

## Главная идея

**Layered models** делят сеть на уровни с чёткими контрактами: HTTP (`0.3`) — **application layer**; ему нужны **TCP** (transport) и **IP** (internet) снизу. **TCP/IP (4 уровня)** — рабочая карта разработчика; **OSI (7)** — учебное vocabulary.

## Ключевые пункты

- **Application:** HTTP, DNS, TLS — semantics для программ.
- **Transport:** TCP/UDP, **ports** — доставка между процессами на host.
- **Internet:** IP, routing — packets между сетями (best-effort).
- **Link:** Ethernet/Wi‑Fi, MAC — frames в local segment.
- **Encapsulation:** каждый уровень добавляет header при отправке вниз; при приёме снимает вверх.
- HTTPS: DNS → TCP connect → TLS → HTTP (`0.3.8`, `0.3.14`).
- `Connection refused` / timeout — часто transport/network, не HTTP status.

## Самые важные примеры

**Стек для `https://api.example.com/users`:**
```
DNS (app) → TCP :443 (transport) → TLS (app) → HTTP GET (app)
```

**Диагностика:**
| Симптом | Уровень |
|---------|---------|
| 404 | Application |
| Could not resolve host | DNS (app) |
| ECONNREFUSED | Transport |
| Certificate error | TLS |

## Частые ошибки

- Смешивать IP (host) и port (process).
- Искать баг JSON при `connection refused`.
- Зубрить OSI без связи с TCP/IP.

## Мнемоника

> **Data down, headers on; data up, headers off.** Четыре уровня TCP/IP + TLS между HTTP и TCP.
