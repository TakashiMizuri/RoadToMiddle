# RequestAborted в ASP.NET — краткая выжимка

> `1.4.29` · ~10 мин на повторение

## Главная идея

**`HttpContext.RequestAborted`** — `CancellationToken`, который срабатывает при disconnect клиента, shutdown хоста или linked timeout. Пробрасывай его во всё I/O, чтобы не тратить ресурсы на «никому не нужную» работу.

## Ключевые пункты

- ASP.NET Core **автоматически** биндит `CancellationToken` в endpoint/controller.
- Отмена **кооперативная** — без передачи в `HttpClient`/EF ничего не остановится.
- **`OperationCanceledException`** при `RequestAborted` — обычно не 500.
- **`CreateLinkedTokenSource`** + `CancelAfter` — server deadline + client abort.
- Отмена **не rollback** транзакций и не отменяет committed side effects.
- CPU-циклы: `ThrowIfCancellationRequested()`.

## Самые важные примеры

```csharp
app.MapGet("/x", async (AppDbContext db, CancellationToken ct) =>
    await db.Products.ToListAsync(ct));
```

```csharp
using var cts = CancellationTokenSource.CreateLinkedTokenSource(
    httpContext.RequestAborted);
cts.CancelAfter(TimeSpan.FromSeconds(30));
```

## «Под капотом» в одном абзаце

Kestrel связывает lifetime TCP/stream с токеном запроса. При RST или graceful shutdown токен переходит в cancelled; EF/Npgsql и `HttpClient` прерывают ожидание I/O. Исключение всплывает в async state machine endpoint'а. Это не preemptive kill потока — только прерывание cooperative awaits.

## Частые ошибки

- Принять `ct`, но не передать в `ToListAsync` / `SendAsync`.
- Логировать каждый disconnect как Error.
- Использовать `RequestAborted` для фоновой работы после отправки ответа.
