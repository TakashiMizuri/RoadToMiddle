# Git Essentials: init, clone, status, add, commit, log, diff — краткая выжимка

> `0.1.3` · ~8 мин на повторение

## Главная идея

Семь базовых команд покрывают ежедневный local Git workflow: **создать/скопировать repo** (`init`, `clone`), **увидеть изменения** (`status`, `diff`), **записать snapshot** (`add`, `commit`), **изучить history** (`log`). Цикл: **edit → status → diff → add → diff --staged → commit → log**.

## Ключевые пункты

- **`git init`** — новый repo с `.git/`; **`git clone`** — полная копия remote + `origin`.
- **`git status`**: long (для людей), `-s` short, `--porcelain` (для scripts). Коды: ` M`, `M `, `MM`, `??`.
- **`git add`**: файл, dir, `.` (subtree), `-A` (весь repo), `-u` (только tracked), **`-p`** (hunks).
- **`git commit -m "..."`**: только staged; **`-a`** auto-stage tracked modified/deleted (не новые файлы).
- **Commit message:** imperative, короткий subject, body = why; один logical change.
- **`git log --oneline --graph --decorate`** — daily view; `-p` — diff в history; `-n N` — limit.
- **`git diff`**: unstaged (work↔index); **`--staged`** (index↔HEAD); **`A..B`** между commits.

## Самые важные примеры

**Новый проект:**
```bash
git init -b main
git add README.md
git commit -m "Initial commit: add README"
git log --oneline
```

**Перед commit — review:**
```bash
git add src/OrderService.cs
git diff --staged          # что попадёт в commit
git commit -m "Fix order total calculation"
```

**Diff targets:**
```bash
git diff                   # unstaged changes
git diff --staged            # staged vs last commit
git diff HEAD                # all local changes vs HEAD
```

## «Под капотом» в одном абзаце

`add` пишет blobs в objects и обновляет index. `commit` создаёт tree + commit object и двигает branch ref. `log` обходит graph от HEAD. `diff` сравнивает content между working tree, index и commit snapshots — не «магия текста», а сравнение stored blobs.

## Частые ошибки

- Commit без `add` для **новых** untracked файлов.
- Думать, что **`commit -a`** добавляет новые файлы.
- **`git add .`** из subdirectory — stage'ит не всё repo (нужен `-A`).
- Не смотреть **`git diff --staged`** → debug/secrets в history.
- Путать **`git diff`** (unstaged) и **`git diff --staged`**.

## Мнемоника

> **Status → Diff → Add → Diff staged → Commit → Log** — «SDADCL»: See, Diff, Add, Double-check, Lock in, Look back.
