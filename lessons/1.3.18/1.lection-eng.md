# Index and Range (^1, ..) Syntax

> Roadmap: `1.3.18` · Node: `1.3` — C# Advanced Topics · Depth: practice

## Learning Objectives

After this lesson you will be able to:

- Use **`Index`** with **`^n`** (from end) and **`0`** (from start) indexing.
- Use **`Range`** with **`start..end`** slice syntax on arrays, spans, strings.
- Predict **inclusive/exclusive** bounds and **`^0`** invalidity.
- Apply slicing without manual `Length - 1` arithmetic.
- Know which types support **`System.Index`** / **`System.Range`** via pattern.
- Avoid allocations when using **`Span<T>`** / **`ReadOnlySpan<char>`** slices.

---

## Why This Matters

Before C# 8, «last element» meant `arr[arr.Length - 1]` and «all but last» meant `Substring(0, s.Length - 1)` — easy off-by-one bugs, noisy code in parsers and log processors. **`^1`** reads as «last item»; **`..`** reads as «slice from A to B» — closer to how humans describe sequences.

You see this in **stack traces parsing**, **URL trimming**, **taking last N log lines**, **unit tests** asserting tail of arrays. Combined with **Span**, slicing can be **allocation-free** on hot paths.

Middle developers should reach for index/range syntax instead of manual length math — but must know **`Range` end is exclusive** and **`^0` is illegal**.

## Core Concepts

### Index: from start or end

**`Index`** struct represents an offset from start or end:

```csharp
int[] nums = { 10, 20, 30, 40 };
nums[0];    // 10 — first
nums[^1];   // 40 — last
nums[^2];   // 30 — second from end
```

**`^n`** means «n elements from the end» where **`^1` is last**, **`^2` is second-to-last**. **`^0` is invalid** — throws `ArgumentOutOfRangeException` (there is no «one past last» from end like some Python intuitions).

`Index` can be constructed: `new Index(2, fromEnd: true)` ≡ `^2`.

### Range: slices

**`Range`** describes start (inclusive) and end ( **exclusive** ):

```csharp
var slice = nums[1..3];   // elements at index 1, 2 → { 20, 30 }
var tail = nums[2..];     // from index 2 to end → { 30, 40 }
var head = nums[..2];     // from start to before index 2 → { 10, 20 }
var all = nums[..];       // whole array copy
var lastTwo = nums[^2..]; // from index Length-2 to end
```

Syntax **`start..end`** — if either omitted, defaults to start or end of sequence.

**End is exclusive:** `[0..3]` includes indices 0,1,2 — not 3. Same half-open interval as `Substring(start, length)` mental model with different second parameter semantics (end index vs length).

### Supported types

Built-in support (C# 8+):

- **Arrays** `T[]`
- **String** — returns **new string** (immutable copy)
- **Span<T>**, **ReadOnlySpan<T>**
- Types with accessible **`Length`/`Count`** + **`get_Item(int)`** or **`Slice`** extension pattern

**`List<T>`** uses **`list[index]`** with Index but range on list returns **`List<T>` copy** via internal slice — not Span. For zero-copy prefer **`CollectionsMarshal.AsSpan(list)`** (advanced, mutable list only) or array.

### String examples

```csharp
string path = "/api/users/42";
var id = path[(path.LastIndexOf('/') + 1)..]; // or split helpers
string file = "report.pdf";
var ext = file[^3..]; // "pdf" if length >= 3
var name = file[..^4]; // "report" if ".pdf" 4 chars — watch lengths
```

Always validate length before `^n` slices shorter than n.

### Range and Index as variables

```csharp
Index last = ^1;
Range middle = 1..^1; // exclude first and last
var part = nums[middle];
```

Useful for configurable parsing windows.

### LINQ vs Range

**`Take`/`Skip`** — lazy IEnumerable, may allocate iterators. **Range on array** — copy segment to new array (allocation). **`Span` slice** — no heap allocation for view. Choose by mutability and lifetime needs.

## Under the Hood

`Index.GetOffset(length)` computes `offset from start` for given sequence length — validates bounds.

`Range` → `start.Offset(length)` and `end.Offset(length)`; slice length = end - start; negative or zero length yields empty.

Compiler rewrites `arr[^1]` to `Index` indexer helper; runtime type must provide **`ElementAt`** or intrinsic array indexer.

For **`string`**, range creates new string copying char buffer segment — O(n) time and allocation.

**`ReadOnlySpan<char>`** from string via `AsSpan()` supports range without string allocation until `ToString()`.

---

## Syntax / Commands / API

```csharp
arr[^1]           // last
arr[..3]           // [0..3)
arr[2..]           // [2..Length)
arr[^3..^1]        // third-from-end up to (not including) last
Range.All          // equivalent ..
Index.Start
Index.End          // conceptual; use ^1 for last element
```

## Examples

### Last line of text

```csharp
ReadOnlySpan<char> line = text.AsSpan();
int nl = line.LastIndexOf('\n');
ReadOnlySpan<char> last = nl >= 0 ? line[(nl + 1)..] : line;
```

### Drop first and last in array

```csharp
int[] inner = data[1..^1]; // excludes first and last elements
```

### Parse IPv4 last octet (illustrative)

```csharp
string ip = "192.168.0.1";
int lastDot = ip.LastIndexOf('.');
string lastOctet = ip[(lastDot + 1)..];
```

## Common Mistakes & Anti-patterns

**Using `^0`** — always throws.

**Assuming end inclusive** — `0..3` has length 3, not 4.

**Short sequence** — `arr[^5]` on length-3 array throws.

**Confusing `..` with length** — `arr[0..3]` is not «3 elements from 0» in the sense of length parameter confusion — it *is* 3 elements (0,1,2).

**Range on IEnumerable** — not supported directly; materialize first.

**Negative indices** — only `^n` from end; `-1` is not Python-style index.

## Production & Real-World Notes

Parsers: combine **`Span`**, **`IndexOf`**, ranges for HTTP headers, CSV fields — reduce GC in high-throughput services.

Unit tests: `Assert.Equal(expected, actual[^1])` for last event in pipeline.

Logging: `lines[^100..]` pattern for tail — watch allocation on large strings; prefer line reader.

API design: expose **`ReadOnlySpan<char>`** overloads when slicing strings frequently (advanced libraries).

## Comparison / Trade-offs

| API | Allocation | Lazy |
|-----|------------|------|
| `arr[i..j]` on array | New array copy | No |
| `s[i..j]` string | New string | No |
| `span[i..j]` | None (view) | No |
| `Skip`/`Take` LINQ | Iterator object | Yes |

## Quick Reference

```
^1 = last, ^2 = second from last, ^0 INVALID
start..end  — end EXCLUSIVE
..end  | start..  | ..
Span for zero-copy string/array views
Validate length before ^n
```

---

## Key Takeaways

- **`^n`** indexes from end; **`^1`** is last; **`^0`** invalid.
- **`start..end`** — end **exclusive**; omitted bound = start/end of sequence.
- Works on **array, string, Span** natively.
- **String range allocates**; **Span slice** does not.
- Replace manual `Length - 1` with **`^1`** for clarity.
- Validate sequence length for small buffers before slicing.
