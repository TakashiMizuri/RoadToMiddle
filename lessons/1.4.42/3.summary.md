# Async all-the-way checklist — краткая выжимка

> `1.4.42` · ~8 мин на повторение · **финал узла 1.4**

## Главная идея

**Async all-the-way** в ASP.NET: от endpoint до EF/HTTP **нет sync blocking на I/O**. Один **checklist** + **CI grep** + load test — release gate против sync-over-async regressions.

## Ключевые пункты (8 категорий)

1. **Endpoints** — `Task`, `CancellationToken`, no `.Result`, no sync EF
2. **Services** — `*Async`, propagate `ct`, no property `.Result`
3. **EF Core** — `ToListAsync`, `SaveChangesAsync`, scoped context
4. **HttpClient** — factory, await `SendAsync`
5. **Middleware/Filters** — `await next`
6. **Background** — scope factory, no blind fire-and-forget
7. **DI** — no async ctor, no singleton `DbContext`
8. **CI** — grep `.Result`/`.Wait(`, analyzers, load test

## Audit workflow

Grep → trace 3 endpoints → load test → PR checkbox → allowlist legacy

```bash
rg "\.(Result|Wait\()" src/ --glob "*.cs"
```

## Частые ошибки

- «Маленький» `.Result` в merge
- `async void` в controllers
- Checkbox audit без автоматизации

## Мнемоника

> **Async to the wire** — async до сокета и БД, без block посередине.
