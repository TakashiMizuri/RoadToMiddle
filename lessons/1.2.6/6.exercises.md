# Exercises: Inheritance — base, override, sealed, new

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Predict the output
**Goal:** Internalize static vs virtual dispatch.

**Tasks:**
For each snippet, write output and one-line reason:
```csharp
// Snippet A
class Base { public virtual void Go() => Console.Write(1); }
class Der : Base { public override void Go() => Console.Write(2); }
Base b = new Der(); b.Go();

// Snippet B — same but Der uses `new void Go()` instead of override
```

**Success criteria:** Correct output and dispatch type named (`runtime` vs `compile-time`).

---

### E2. Constructor chain
**Goal:** Practice `: base(...)`.

**Tasks:**
Implement **`Product(int id, string name)`** deriving from **`Entity(Guid id)`** where **`Entity`** has no parameterless constructor. Assign **`Name`** in derived ctor.

**Success criteria:** Compiles; **`Id`** set only via base.

---

## Medium

### M1. Extend, don't duplicate
**Goal:** Use **`base`** in override.

**Tasks:**
**`AuditLogger`** derives **`FileLogger`**. Override **`Log(string msg)`** to prepend UTC timestamp, then call base implementation.

**Success criteria:** Single place for file write logic in base; derived only wraps.

---

### M2. Seal the hot path
**Goal:** Apply **`sealed override`**.

**Tasks:**
Three-level hierarchy: **`Shape` → `Rectangle` → `Square`**. **`Area()`** virtual in **`Shape`**, overridden in **`Rectangle`**, **`sealed override`** in **`Square`**. Attempt fourth override — document compile error.

**Success criteria:** CS0239 on illegal override; explain why sealing helps JIT/readers.

---

### M3. When `new` is wrong
**Goal:** Refactor hiding to polymorphism.

**Tasks:**
Given **`new`-hidden** **`Send()`** on **`EmailNotifier : Notifier`**, refactor so **`Notifier n = new EmailNotifier(); n.Send()`** calls email logic. Two steps: base change + derived change.

**Success criteria:** Behavior fixed with **`virtual`/`override`**, not rename-only workaround.

---

## Hard (stretch)

### H1. Fragile base constructor
**Goal:** Reproduce and fix partial construction bug.

**Tasks:**
Base ctor calls **`virtual Init()`**. Derived overrides **`Init()`** and uses derived field not yet assigned. Show bug; fix by removing virtual call from ctor or using **`Init()`** after construction.

**Success criteria:** Written explanation of initialization order.
