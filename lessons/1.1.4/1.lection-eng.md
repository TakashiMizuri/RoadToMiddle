## Why This Matters

Strings appear in every layer: HTTP headers, JSON payloads, SQL parameters, log messages, UI text. **`string` in C# is not a mutable character array** — it is an immutable sequence of UTF-16 code units on the heap. Code like `s[0] = 'x'` does not compile; `s += "!"` creates a **new** object. Misunderstanding immutability causes unnecessary allocations in hot loops and wrong assumptions about thread safety ("I'll mutate shared string").

Backend developers concatenate paths, build queries, and format errors daily. **`$"User {id} not found"`** is readable but compiles to machinery similar to `string.Format`. Verbatim strings simplify regex and JSON snippets. **Interning** deduplicates literal strings in memory — relevant for memory profiling and occasionally for reference equality (`ReferenceEquals`) surprises in interviews and debugging.

Depth here connects to security (never trust interpolated strings in SQL), culture (`ToString` formatting), GC pressure (concat in loops), and EF/log parameterization.

## Core Concepts

### `string` Is a Reference Type, Immutable

```csharp
string name = "Ada";
// name[0] = 'B'; // compile error — indexer returns char, not assignable
```

**`string`** is **`System.String`** — a **sealed class** (reference type) wrapping UTF-16 characters. Length fixed at creation. Any API that "changes" a string returns a **new** instance:

```csharp
string s = "hello";
string upper = s.ToUpperInvariant(); // new string "HELLO", s unchanged
```

**Thread safety:** immutability means shared read-only strings need no locks. Multiple threads can read the same instance safely.

Variables hold **references** (pointers to heap objects). Assignment copies the reference:

```csharp
string a = "hi";
string b = a;
// a and b refer to same object if same content interned — see interning
```

### UTF-16 and `char`

Each **`char`** is 16 bits — a **UTF-16 code unit**. Most characters are one unit; supplementary characters (many emoji) use **surrogate pairs** (two `char` values). **`string.Length`** counts **code units**, not visual graphemes:

```csharp
string emoji = "😀"; // may be Length 2 (surrogate pair)
```

For grapheme-aware operations, .NET 5+ offers **`StringInfo`**, **`Rune`**. For typical ASCII/ Cyrillic text, length ≈ character count.

Internal layout (conceptual): object header + length field + char buffer.

### Literal Forms

**Regular:**

```csharp
string path = "C:\\Users\\data"; // backslash escaped
string quote = "She said \"Hi\"";
```

**Verbatim** — `@` disables escape processing except `""` for quote:

```csharp
string path = @"C:\Users\data";
string json = @"{ ""name"": ""Ada"" }";
```

**Raw string literals (C# 11+)** — triple quotes, newlines preserved:

```csharp
string html = """
    <div class="x">
        Hello
    </div>
    """;
```

Indentation trimming follows opening `"""` column rules.

### Interpolation and Formatting

**Interpolated string (C# 6+):**

```csharp
int id = 42;
decimal total = 19.99m;
string msg = $"Order {id} total {total:C}";
```

**`:C`** is format specifier (currency, culture-dependent). Constant interpolation may compile to **`string.Format`** or **`DefaultInterpolatedStringHandler`** (.NET 6+) for fewer allocations.

**Composite formatting:**

```csharp
string msg = string.Format("Order {0} total {1:C}", id, total);
```

**Culture:** use **`CultureInfo.InvariantCulture`** for logs and wire formats; **`CurrentCulture`** for UI.

```csharp
$"{total.ToString(CultureInfo.InvariantCulture)}"
```

### Concatenation

**`+` and `string.Concat`** create new strings:

```csharp
string full = first + " " + last;
```

Compiler may optimize **`"a" + "b" + "c"`** at compile time to single literal. Runtime loop **`s += part`** in **`for`** — O(n²) allocations — anti-pattern; use **`StringBuilder`** (`1.1.16`) or **`string.Join`**.

```csharp
string joined = string.Join(", ", items);
```

### String Interning

The **intern pool** stores unique string contents per **AppDomain** ( .NET Core / .NET 5+ — per process effectively for literal semantics). **Compile-time literals** with same content may share one instance:

```csharp
string a = "hello";
string b = "hello";
ReferenceEquals(a, b); // often true for literals
```

**Runtime constructed** strings (`new string(...)`, **`Substring`**, concatenation) are **not** interned by default:

```csharp
string x = string.Concat("hel", "lo");
ReferenceEquals(a, x); // typically false
```

**`string.Intern(s)`** returns canonical pool instance; **`string.IsInterned(s)`** returns reference or `null`. Rare in app code — used for memory dedup or legacy interop. Over-interning large dynamic strings **increases** long-lived memory.

### Null and Empty

**`string?`** with nullable reference types — `null` means absence. **`string.Empty`** vs **`""`** — same interned empty instance. Prefer **`IsNullOrEmpty`** / **`IsNullOrWhiteSpace`** for validation.

## Under the Hood

### Object Layout on Heap

A **`string`** object contains length and immutable char array (implementation details may vary; JIT intrinsics optimize common ops). Because content never changes, hash code can be **cached** after first **`GetHashCode()`** — important for dictionary keys.

### What `s += "x"` Does

1. Allocate new char buffer length `s.Length + 1`.
2. Copy existing chars + append.
3. Assign new reference to `s`.
4. Old string becomes garbage unless other references exist.

No in-place mutation — unlike **`StringBuilder`** internal buffer.

### Interpolation Lowering

Roslyn transforms **`$"A {x} B"`** into:

- **`string.Format("A {0} B", x)`** (older), or
- Calls to **`String.Concat`**, or
- **`DefaultInterpolatedStringHandler.AppendFormatted`** building on stack buffer then final string (.NET 6+).

For **`const`** interpolation, whole thing may become compile-time literal.

### Intern Pool Implementation (Concept)

CLR maintains hash table of interned strings. Literal loading may call **`Intern`**. **`Assembly`** metadata stores user strings in **#Strings heap** — loaded with dedup. Dynamic intern is explicit opt-in.

**Security note:** intern pool never releases — **do not intern** unbounded user input (memory leak).

## Syntax / Commands / API

```csharp
// Length, access (read-only)
int len = s.Length;

// Common methods
bool blank = string.IsNullOrWhiteSpace(s);
bool eq = string.Equals(a, b, StringComparison.OrdinalIgnoreCase);
int idx = s.IndexOf('x');

string trimmed = s.Trim();

// Interpolation
var line = $"{name} ({id:D4})";

// Verbatim + interpolation (C# 8+)
var dir = $@"C:\Users\{name}\Documents";

// Join / split
var csv = string.Join(';', parts);
var tokens = s.Split(' ', StringSplitOptions.RemoveEmptyEntries);

// Interning (careful)
var canon = string.Intern(userInput); // avoid for unbounded input
```

## Examples

### Example 1: Immutability in Practice

```csharp
string greeting = "Hello";
greeting = greeting + ", World";
Console.WriteLine(greeting); // "Hello, World"
// Original "Hello" object orphaned if no other refs
```

### Example 2: Verbatim Regex

```csharp
string pattern = @"^\d{4}-\d{2}-\d{2}$";
string path = @"\\server\share\file.txt";
```

Backslashes literal — essential for regex and Windows paths.

### Example 3: Culture-Safe Log Line

```csharp
using System.Globalization;

decimal amount = 1234.5m;
string log = string.Create(CultureInfo.InvariantCulture, $"Amount={amount}");
Console.WriteLine(log); // Amount=1234.5
```

## Common Mistakes & Anti-patterns

**Mutating assumption** — treating strings like char arrays.

**Concat in tight loop** — quadratic allocations; use **`StringBuilder`**.

**`+` for SQL/command** — injection risk; use parameters.

**`ReferenceEquals` for value equality** — use **`==`** or **`Equals`** with correct **`StringComparison`**.

**Interning user content** — unbounded pool growth.

**Ignoring culture** in **`ToString()`** for API responses — use invariant or explicit format.

## Production & Real-World Notes

Logging frameworks defer formatting; structured logging passes objects, not pre-interpolated strings when lazy evaluation matters.

JSON serializers emit UTF-8 bytes — strings round-trip as .NET **`string`** (UTF-16 internal).

PostgreSQL **`text`** maps cleanly; collation affects sort in DB, not in-memory **`string.Compare`**.

Performance: **`string.Create`** and **`Span<char>`**-based APIs reduce intermediate allocations in high-throughput parsers.

## Comparison / Trade-offs

| Approach | When |
|----------|------|
| `$"..."` | Readable composite messages |
| `string.Join` | Known separator + sequence |
| `StringBuilder` | Many append steps in loop |
| Verbatim `@""` | Paths, regex, embedded quotes |
| Raw `"""` | Multiline templates (SQL, HTML snippets) |
| Ordinal vs culture compare | Logs/IDs vs user-visible sort |

## Key Takeaways

- **`string`** — immutable **reference type**; "changes" allocate new objects.
- Internal encoding **UTF-16**; **`Length`** = code units.
- **`$""`** interpolation; **`@""`** verbatim; **`"""`** raw multiline (C# 11+).
- **`+` in loops** — costly; prefer **`Join`** / **`StringBuilder`**.
- **Intern pool** deduplicates literals; runtime strings usually not interned; don't intern unbounded input.
- Equality: **`==`** compares content; **`ReferenceEquals`** only same instance.
- Use **InvariantCulture** for wire/logs; culture-aware for UI.
