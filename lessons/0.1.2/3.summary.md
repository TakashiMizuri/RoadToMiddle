# Repository, working tree, staging area, commit — краткая выжимка

> `0.1.2` · ~7 мин на повторение

## Главная идея

**Git repository** — это **working tree** (файлы, которые ты редактируешь) плюс скрытая папка **`.git/`** (база history). Каждый tracked файл проходит три состояния: **modified → staged → committed**. Между правками и history стоит **staging area (index)** — осознанный preview следующего snapshot.

## Ключевые пункты

- **Working tree** — видимые файлы проекта; **`.git/`** — objects, commits, refs, config.
- **Modified** — изменён на диске, не в index. **Staged** — после `git add`, ждёт `commit`. **Committed** — в history.
- **Three trees:** working directory → staging area (index) → repository (`.git/`).
- **Staging нужен** для atomic commits, partial commits и review через `git diff --staged`.
- **Commit** = snapshot всех файлов + metadata (author, date, message, parent). Immutable.
- **Untracked** файлы вне трёх состояний — Git их ещё не version'ит.

## Самые важные примеры

**Lifecycle:**
```
edit file → modified
git add   → staged
git commit → committed (clean working tree для этих изменений)
```

**Partial commit:**
```bash
git add AuthService.cs      # только auth в staging
git commit -m "Add JWT validation"
# appsettings.json остаётся modified, unstaged
```

**Диаграмма (three trees):**
```
Working Directory  --git add-->  Staging Area  --git commit-->  .git/
```

## «Под капотом» в одном абзаце

`git add` хеширует content в **blob** и обновляет **index** (`.git/index`). `git commit` строит **tree object** из index, создаёт **commit object** с parent и metadata, двигает **branch ref** (например `main`). До commit history не меняется — меняются только working tree и index относительно `HEAD`.

## Частые ошибки

- `git add .` без review → смешанные unrelated changes в одном commit.
- Думать, что commit = save в редакторе (commit — intentional history entry).
- Путать **local repo** (`.git`) с **GitHub** (remote для sync).
- Игнорировать, что один файл может быть одновременно staged и modified.

## Мнемоника

> **Edit → Add → Commit** — три шага, три дерева. Staging = «что кладём в коробку перед отправкой».
