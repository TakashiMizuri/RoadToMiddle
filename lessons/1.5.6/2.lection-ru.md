# GC roots, reachability и mark phase

> Roadmap: `1.5.6` · Depth: **глубоко**

## Learning Objectives

- Определить **GC roots**.
- Объяснить **reachability** через reference chains.
- Описать **mark phase** mark-sweep-compact.
- Перечислить roots: stacks, statics, handles, finalizer queue.
- Объяснить сбор **cycles** без external roots.

---

## Why This Matters

Object «мертв» не когда перестали использовать локально, а когда **нет path от root**. Event handlers, static caches — **leaks** через accidental roots.

**Mark phase** определяет liveness — основа pauses и finalizer delays.

---

## Core Concepts

### Roots

| Source | Example |
|--------|---------|
| Thread stacks | Locals, params (`1.5.4`) |
| Static fields | `static Cache` |
| GCHandle | Normal/Pinned |
| Finalizer queue | Pending Finalize |

### Reachability

Unmarked после mark from roots → garbage (for collected scope). **Cycles** without roots — collected (tracing GC vs ref counting).

`local = null` removes one link; other refs keep alive.

### Mark Phase

1. Suspend/coordinate threads.
2. Scan roots → mark objects.
3. Trace reference fields (MethodTable GC desc `1.5.5`).
4. Sweep/compact unmarked.

**Write barriers** для generational GC (`1.5.7`).

### WeakReference

Not strong root — target collectable if only weak refs remain.

---

## Key Takeaways

- Live = reachable from roots.
- Leaks = hidden strong roots.
- Mark traces graph from roots.
- Cycles OK if unreachable externally.
