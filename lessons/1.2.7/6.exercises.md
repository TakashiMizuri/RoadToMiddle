# Exercises: Polymorphism — virtual/override runtime dispatch

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Virtual zoo
**Goal:** Minimal polymorphic hierarchy.

**Tasks:**
Create **`Animal`** with **`virtual Speak()`**, **`Dog`** and **`Cat`** overrides. **`List<Animal>`** with mixed instances; loop and **`Speak()`**.

**Success criteria:** No type switches; correct output per runtime type.

---

### E2. Abstract shape
**Goal:** **`abstract`** + **`override`**.

**Tasks:**
**`abstract class Shape`** with **`abstract double Area()`**. **`Rectangle`**, **`Circle`** concrete. Call **`Area()`** via **`Shape`** reference.

**Success criteria:** Cannot instantiate **`Shape`**; each area correct.

---

## Medium

### M1. Template method exporter
**Goal:** Non-virtual orchestration + virtual hooks.

**Tasks:**
**`CsvExporter : DataExporter`** — override **`LoadData()`** only; **`Export()`** inherited. Verify **`Export()`** on base reference uses derived data.

**Success criteria:** Explain which methods dispatch virtually when **`Export()`** runs.

---

### M2. LSP smell
**Goal:** Recognize bad subtype.

**Tasks:**
Sketch **`Rectangle`** with setters and **`Square : Rectangle`**. Explain why **`Square`** substituted as **`Rectangle`** breaks expectations. Propose composition or separate types.

**Success criteria:** One paragraph LSP violation; alternative design named.

---

### M3. Interface strategy
**Goal:** Polymorphism without inheritance.

**Tasks:**
**`IPaymentMethod`** with **`Pay(decimal)`**; **`CardPayment`**, **`WalletPayment`**. **`Checkout`** accepts **`IPaymentMethod`**.

**Success criteria:** Same extensibility as virtual class; no shared base class required.

---

## Hard (stretch)

### H1. Dispatch trace
**Goal:** Connect source to mechanism.

**Tasks:**
Build E1, inspect IL for **`Speak()`** call in loop — find **`callvirt`**. Write which type's method table is used.

**Success criteria:** IL snippet or description of slot lookup.
