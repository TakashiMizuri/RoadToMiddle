# Boxing и unboxing — когда это происходит

## Why This Matters

В `1.1.6` `int` local хранит значение inline, `string` указывает на heap object. Модель ломается на:

```csharp
object o = 42;
```

Примитив из `1.1.3` становится **heap object**: allocation, копия `int` в payload, учёт GC. **Boxing** — частый скрытый источник allocations.

Boxing связывает **value types** с **reference-based API** (`object`, non-generic `IList`). До generics каждая коллекция boxing'ила integers. Middle-разработчик всё ещё попадает на boxing через **`object` parameters**, **interface dispatch на struct**, **`enum` → `object`**.

Понимание boxing превращает «странный GC» в инженерную задачу: профиль, `box` в IL, замена на generic API.

## Core Concepts

### Что такое boxing

**Boxing** превращает **value type** в **reference type**, создавая object на **managed heap** с **копией** значения.

```csharp
int n = 42;
object boxed = n;   // boxing
```

Шаги CLR:

1. Allocate object под header + payload.
2. **Скопировать** `int` в payload box'а.
3. `object` variable держит **reference** на box.

Исходный local `n` не меняется; box — **отдельная копия** на heap.

Boxing **implicit** при conversion к `object` или **interface**, который struct/primitive реализует.

### Unboxing

```csharp
object boxed = 42;
int m = (int)boxed;   // unboxing
```

Правила:

- Box должен содержать **тот же** value type.
- Cast **точный**: `(int)`, не `(long)` для int box → **`InvalidCastException`**.
- Unboxing **копирует** bits в local; box object остаётся до GC.

### Зачем boxing существовал

До generics API нужен был общий **`System.Object`**. Value types boxing'ились для `ArrayList`, `Hashtable`, logging `object value`.

**`List<T>`** для value type `T` хранит элементы **inline в array** — **без boxing на элемент** (JIT specialization).

### Когда boxing происходит

**1. К `object`:**

```csharp
object o = 1;
object[] arr = { 1, 2 };
```

**2. Non-generic collections:**

```csharp
new ArrayList().Add(42);
```

**3. Interface на primitive/struct:**

```csharp
IComparable c = 10;
```

**4. Enum → object:**

```csharp
object o = Day.Monday;
```

**5. `string.Format("{0}", 42)`** — может box argument (современные API часто оптимизируют).

**6. `GetType()` на struct** — boxing.

**7. Nullable:**

```csharp
int? a = 5;  object o = a;     // boxes 5
int? b = null; object p = b;  // p == null, box нет
```

### Когда boxing не происходит

- `List<int>.Add(42)`
- Assignment между одинаковыми value types
- Generics без cast к `object`
- **`ref struct`** — boxing запрещён

### Pitfalls unboxing

```csharp
long x = (long)(object)42;   // InvalidCastException
```

## Under the Hood

### IL

Boxing:

```il
ldc.i4.s 42
box [System.Runtime]System.Int32
stloc.1
```

Unboxing:

```il
ldloc.1
unbox.any [System.Runtime]System.Int32
ldind.i4
stloc.2
```

**`box`** — allocate + copy value → push reference.  
**`unbox` / `unbox.any`** — typed pointer / load; wrong token → exception.

Проверяй через ILSpy / dotPeek после `dotnet build`.

### Layout box'а

Boxed `int`: method table + sync header + **4 bytes payload**. Boxing **всегда копирует**.

### Performance

Каждый **`box`** — обычно **Gen0 allocation**. Частый boxing → allocation rate → GC pauses. **Unboxing** без allocation, но type check + copy.

### Связь с 1.1.6

Boxing — value type на heap **внутри reference object**. После boxing — reference semantics и GC tracking.

## Examples

**Hidden boxing:**

```csharp
void Print(object value) => Console.WriteLine(value);
Print(100);   // boxes
```

Fix: overload `Print(int)` или generic.

**Safe unbox:**

```csharp
if (unknown is int i) Use(i);
```

## Common Mistakes & Anti-patterns

`ArrayList` в hot path. DTO с `object`. Wrong unbox cast. Boxing больших struct.

---

## Production & Real-World Notes

`Console.WriteLine(object)` boxes `int` — в loop используй typed overload. Structured logging — typed templates. Analyzers flag лишние `object` overloads.

---

## Comparison / Trade-offs

| `ArrayList` | boxing | heterogeneous |
| `List<int>` | no storage box | typed |
| `object` API | often boxes | one surface |
| Generic `T` | avoids for struct T | type-safe |

## Quick Reference

```
Value → object/interface = box
Exact cast back          = unbox
int? null → object       = null, no box
IL: box, unbox.any
```

---

## Key Takeaways

- Boxing = heap object + **copy** value type.
- Unboxing = **exact** cast + copy out.
- Generics **`List<T>`** избегают storage boxing.
- IL явно показывает **`box`**.
- Лишний boxing → **GC pressure**.
