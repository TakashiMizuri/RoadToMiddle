# struct vs class — performance, mutability

## Why This Matters

Каждый тип C# — **value type** или **reference type**. От этого зависят allocation, GC pressure, equality, threading и API design. `int`, `DateTime`, `Guid` — struct; `string`, `List<T>`, domain services — class. Неверный выбор стоит денег: копирование struct 256 байт на каждой итерации; boxing в `object`; mutable struct — «изменения пропали».

Из `1.2.1` ты знаешь, что объекты — ссылки в heap. Эта лекция завершает картину **stack-allocated value types** и случаев, когда runtime всё равно кладёт данные в heap. Middle-разработчик обоснует `readonly struct Vector3` в hot path и объяснит, почему EF entities — **classes**.

## Core Concepts

### Два семейства типов

**Value types** (`struct`, primitives, enums) по умолчанию **copy-by-value**: присваивание дублирует bits (field-wise copy). **Reference types** (`class`, `interface`, `delegate`, `record class`) копируют **указатель**; две переменные могут alias один объект.

```csharp
var a = new Point(1, 2);   // struct
var b = a;                 // полная копия
b.X = 99;
Console.WriteLine(a.X);    // 1

var c = new Customer();    // class
var d = c;                 // копия ссылки
d.Name = "X";
Console.WriteLine(c.Name);   // "X"
```

**Identity:** у reference types стабильная **object identity** (`ReferenceEquals`). У struct нет отдельной identity — equality по значению (пока не boxed).

### Объявление struct и defaults

```csharp
public struct Point
{
    public int X;
    public int Y;
}
```

`new Point()` zero-initializes. Нельзя наследовать struct от class (кроме implicit `System.ValueType`). Можно implement interfaces — но вызов через interface может **box**.

**Size guideline:** исторически **< 16 bytes** как ориентир для struct; больше — overhead копирования. Профилируй hot paths.

### Copy semantics в методах

Struct передаётся **by value** — копия целиком, если не **`ref`**, **`in`**, **`out`** (preview `1.2.15`):

```csharp
void Move(Point p) { p.X++; }           // copy — caller не меняется
void MoveRef(ref Point p) { p.X++; }     // alias
```

Mutating struct methods часто лучше как immutable (return new struct) или `ref` API.

### class: ссылка в heap

```csharp
public class Customer
{
    public int Id { get; set; }
    public string Name { get; set; } = "";
}
```

`new Customer()` — allocation в **managed heap**; GC при unreachable. Reference fields внутри struct всё ещё указывают в heap — копия struct дублирует pointers, не deep graph.

### Ловушки mutability

**foreach** копирует:

```csharp
foreach (var p in points)
    p.X++;   // меняет копию — list не обновится
```

Компилятор вставляет defensive copies для readonly struct в properties. Prefer **immutable structs** (`1.2.15`).

### Когда prefer struct

- Маленькое **logical value** без identity (координаты, пара amount+currency).
- **High churn** locals, где heap allocation мешает (после профилирования).
- **Immutable** data, часто передаваемая by value.
- **`record struct`** (`1.2.13`) для synthesized equality.

### Когда prefer class

- **Identity** (customer id 42 — одна сущность).
- **Большое** или **растущее** состояние.
- **Polymorphism** через inheritance.
- **Null** как sentinel.
- **EF Core entities**, shared mutable graphs.

### Boxing и unboxing

Присваивание struct в `object` **boxes** — heap object с копией struct. **Unboxing** копирует обратно.

```csharp
object o = 42;        // box
int x = (int)o;       // unbox
```

Generics избегают boxing. Связь с **`1.1.7` boxing** — GC pressure.

### string — особый reference type

Immutable reference type с value-like equality — исключение из правила identity для `==`.

## Under the Hood

### Memory layout

**Stack:** stack frames для locals/parameters. Маленькие value types часто в stack slots — не гарантировано: JIT может enregister или положить в heap при escape (closures, `async`, `ref` to heap).

**Heap:** reference types — object header + fields. Struct **внутри** class object лежит inline в payload.

```
class Container { public Point P; }  →  [header][Point.X][Point.Y] на heap

struct Point на stack frame:  [X][Y]
```

### Copy cost

Присваивание struct размера N копирует N bytes. Для больших N — **`in` parameter**:

```csharp
void Print(in LargeStruct s) { ... }
```

### GC interaction

Structs не tracked GC, пока не **boxed** или нет reference fields. Struct с `string` — копия struct копирует reference на string.

**Array of structs** — contiguous values, cache-friendly vs array of class references.

### Interface на struct

Вызов через interface type **boxes** (если JIT не devirtualizes generic):

```csharp
IComparable c = myStruct;  // box
```

### ref struct preview

`Span<T>` — **ref struct** (`1.2.15`): stack-only, no boxing.

## Examples

### foreach copy

```csharp
foreach (var p in list)
    p.X = 10;
Console.WriteLine(list[0].X);  // 1
```

### struct с reference field

```csharp
var b = a;
b.Text = "bye";
Console.WriteLine(a.Text);  // "bye" — shared reference
```

### Array struct vs class

`Point[1_000_000]` — contiguous structs vs `Customer[]` — million heap objects.

## Common Mistakes & Anti-patterns

**Large mutable struct** — дорогие silent copies.

**foreach mutation** на struct в collection.

**struct для OOP hierarchy** — невозможно.

**Boxing в hot loops**.

**Каждый DTO как struct** — большие копии при binding.

**«struct всегда на stack»** — escape legal.

## Production & Real-World Notes

Профилируй allocation. Public APIs редко expose mutable struct. BCL: `DateTime`, `Guid`, `TimeSpan`. EF — classes.

## Comparison / Trade-offs

| Критерий | struct | class |
|----------|--------|-------|
| Assignment | Copy fields | Copy reference |
| Null | `T?` | Nullable reference |
| Inheritance | Нет | Да |
| Identity | Нет | `ReferenceEquals` |
| GC | Ниже без boxing | Heap per instance |

## Quick Reference

| **struct** | **class** |
|------------|-----------|
| Маленький immutable value | Identity, ORM |
| Нет inheritance | Polymorphism |
| `record struct` | `null`, большие графы |

## Key Takeaways

- **struct** = copy-by-value; **class** = shared reference.
- Stack vs heap — решение JIT; boxing и escape в heap.
- **Mutable struct** + foreach = баги.
- **Boxing** — GC cost; generics помогают.
- **Большие struct** — частые копии; `in`/`ref` или class.
- **EF / identity** → class; **малые values** → struct.
