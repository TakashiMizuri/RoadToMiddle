# HTTP/1.1 vs HTTP/2 vs HTTP/3 — краткая выжимка

> `0.3.13` · ~7 мин на повторение

## Главная идея

Версии HTTP меняют **transport efficiency**, не semantics REST. **HTTP/1.1** — text и parallel TCP; **HTTP/2** — multiplex на одном TCP (HPACK); **HTTP/3** — HTTP поверх **QUIC/UDP**, меньше cross-stream blocking при packet loss.

## Ключевые пункты

- **Keep-alive** в HTTP/1.1 переиспользует TCP; ~6 connections/host для parallelism.
- **HTTP/2:** binary frames, many streams, negotiation через **TLS ALPN** (`h2`).
- **TCP packet loss** блокирует все HTTP/2 streams — причина HTTP/3.
- **HTTP/3:** QUIC, **Alt-Svc**, fallback chain h3→h2→h1.
- Methods, status, JSON — **одинаковы** для всех версий.

## Самые важные примеры

```bash
curl -v https://example.com   # смотри ALPN и HTTP/2
```

DevTools → Network → column **Protocol**.

## Частые ошибки

- «HTTP/2 = быстрый API» без fix N+1 и caching.
- Block UDP 443 → HTTP/3 never works, silent fallback.
- Путать HTTP/2 multiplex с WebSockets.

## Мнемоника

> **h1: много conn. h2: один TCP, много streams. h3: QUIC, streams переживают loss лучше.**
