# Answers: SemaphoreSlim for throttling

## Scoring

≥80% pass → `1.4.19`

---

## Answers

### 1. [Recall]
**Answer:** **Await'ит** (async wait), пока другой caller не вызовет **`Release()`** — не busy-spin на thread.

---

### 2. [Recall]
**Answer:** Чтобы **вернуть permit** даже при exception в `try` — иначе **permit leak**.

---

### 3. [Understanding]
**Answer:** Semaphore limits **concurrent execution**; channel buffers **queued work** и decouples producer/consumer speed. Different problems; often combined.

---

### 4. [Understanding]
**Answer:** Thread **удерживает lock** во время **await** → другие не войдут → **deadlock/starvation** thread pool.

---

### 5. [Application]
**Answer:** Singleton **`SemaphoreSlim(3)`**; для каждого URL task: **`WaitAsync`**, **`try { GetAsync }`**, **`finally { Release }`**, **`WhenAll`**.

---

### 6. [Application]
**Answer:** **Нет** — если **`WaitAsync`** canceled/failed before acquire, permit **не взят**.

---

### 7. [Analysis]
**Answer:** **Permit leak** — missing **`Release`** on some code paths (exception without finally).

---

### 8. [Analysis]
**Answer:** **1000 concurrent DB** calls при pool **100** → timeouts, pool exhaustion; **`N`** must match downstream.

---

### 9. [Recall]
**Answer:** **`SemaphoreFullException`**.

---

### 10. [Understanding]
**Answer:** Custom pipelines, **mixed work types**, dynamic **`N`**, compose with other gates — manual semaphore гибче.

---

### 11. [Application]
**Answer:** Любая reasonable async signature returning **`Task`** aggregating all jobs with throttling.

---

### 12. [Understanding]
**Answer:** **Нет** — только **concurrency**; RPS → **`RateLimiter`** / token bucket.

---

### 13. [Analysis]
**Answer:** **Нет** — каждый action свой limit 10 ≠ global 10; нужен **shared singleton**.

---

### 14. [Application]
**Answer:** При **`new SemaphoreSlim(N,N)`** в steady state **`CurrentCount`** should fluctuate; если **stuck at 0** forever while idle → leak suspicion; log on wait time.

---

### 15. [Understanding]
**Answer:** **`lock`** проще для **short sync** critical section; **`SemaphoreSlim(1,1)`** + **`WaitAsync`** when need **async** gate — not re-entrant like lock.
