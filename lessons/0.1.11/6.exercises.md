# Exercises: Detached HEAD

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Observe HEAD file
**Goal:** See attached vs detached in `.git/HEAD`.

**Steps:**
1. Init repo, one commit on `main`.
2. Record `cat .git/HEAD` (attached).
3. `git switch --detach HEAD`
4. Record `cat .git/HEAD` again (detached).
5. `git switch main` and verify attached restored.

**Success criteria:** You explain difference in file contents.

---

### E2. Safe read-only detach
**Goal:** Detach without losing anything.

**Steps:**
1. Create 3 commits on `main` with different content in `version.txt`.
2. `git switch --detach HEAD~2`
3. Read `version.txt` content.
4. `git switch main` — confirm branch unchanged, no extra commits.

**Success criteria:** `git log main` identical before and after.

---

## Medium

### M1. Lose and recover a commit
**Goal:** Practice reflog rescue (controlled environment).

**Steps:**
1. `git switch --detach HEAD~1`
2. Create `fix.txt`, commit `"detached work"`
3. Note SHA with `git log -1`
4. `git switch main` — confirm commit not on `main` log
5. `git reflog` — find SHA
6. `git switch -c rescued SHA`
7. Verify file exists on `rescued` branch

**Success criteria:** Documented reflog line used for recovery.

---

### M2. Correct hotfix from tag
**Goal:** Production workflow.

**Steps:**
1. Create commits, tag `v0.1.0` on `main`.
2. Add more commits on `main` (simulate continued dev).
3. Start hotfix **from tag**: `git switch -c hotfix/0.1.1 v0.1.0`
4. Commit fix, verify `main` tip does not include hotfix until merge.

**Success criteria:** Hotfix branch parent is tag commit, not `main` tip.

---

## Hard (stretch)

### H1. Explain to a junior
**Goal:** Feynman technique.

Write a 150–200 word explanation (Russian or English) for a junior who says: *"Detached HEAD popped up and I'm scared to touch anything."* Include: what it is, when it's OK, one dangerous pattern, one recovery command.

**Success criteria:** Mentions branch ref, `-c`, reflog; no "repo is broken" myth.

---

### H2. bisect awareness
**Goal:** Connect bisect to detached HEAD.

**Steps:**
1. Read `git bisect --help` intro (or lection).
2. In a repo with good/bad commits, start `git bisect` (optional full run).
3. Note `git status` during bisect — detached?
4. Write when **not** to commit during bisect.

**Success criteria:** 3 sentences on bisect + detached relationship.
