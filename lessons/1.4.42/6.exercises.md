# Exercises: Async all-the-way checklist

## Easy

### E1. Grep audit
**Goal:** Run checklist grep on sample project.
**Steps:** Create small API with intentional `.Result`; run `rg`; fix to pass.
**Success criteria:** Zero matches in `src/`.

### E2. PR template
**Goal:** Add async checkbox to PR template.
**Steps:** Markdown 3 bullets from checklist categories.
**Success criteria:** Reviewable in repo `docs/` or `.github/`.

## Medium

### M1. Full category pass
**Goal:** Audit provided or your API against all 8 categories.
**Steps:** Spreadsheet Pass/Fail per category; fix one Fail.
**Success criteria:** Documented audit artifact.

### M2. Load test gate
**Goal:** 100 concurrent GET; thread pool counter stable.
**Steps:** `wrk`/`bombardier` + `dotnet-counters` threadpool-queue-length.
**Success criteria:** No unbounded queue growth vs baseline.

## Hard (stretch)

### H1. CI rule
**Goal:** Fail build on `.Result` in `src/` except `legacy-allowlist.txt`.
**Steps:** GitHub Action or script step with allowlist.
**Success criteria:** PR with new `.Result` fails CI.

### H2. Close node 1.4
**Goal:** Oral/written 2-min explanation HTTP→DB async + checklist top 5.
**Success criteria:** Mentor/AI review pass; update PROGRESS.md.

## Submission checklist

- [ ] Grep clean or allowlisted
- [ ] 8 categories reviewed
- [ ] Ready for `1.5.1` CLR
