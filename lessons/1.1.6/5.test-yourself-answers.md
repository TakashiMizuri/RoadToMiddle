## Answers

### 1. [Recall]
**Answer:** Value type assignment **копирует данные** (независимые копии). Reference type assignment **копирует reference** — обе переменные указывают на **тот же object**.

### 2. [Recall]
**Answer:** `int`, `decimal`, `bool` — **value types**. `string` — **reference type** (immutable class).

### 3. [Recall]
**Answer:** **Managed heap** — `List<int>` object; local variable хранит **reference** в stack frame.

### 4. [Understanding]
**Answer:** Value types на heap как **fields reference type class**; как **elements** `int[]`/array of struct; при **boxing**; в **closure** display class. Stack — не единственное место.

### 5. [Understanding]
**Answer:** **`a`** тоже изменится — `"hi!"`. **`b`** копирует reference на **тот же** mutable object; `Append` мутирует shared heap object.

### 6. [Understanding]
**Answer:** Plain `int` всегда holds valid int (default 0). **`int?`** — `Nullable<int>`, value type с `HasValue` flag, может represent «отсутствует значение» без null reference semantics для самого `int` storage.

### 7. [Understanding]
**Answer:** Frame: `n` = 5 inline; `s` = ref → heap String; `list` = ref → heap List object → internal array ref → heap int[].

### 8. [Application]
**Answer:** **`1`**. Struct copied by value; `p2` — independent copy.

### 9. [Application]
**Answer:** **1 элемент (42)**. Reference copied; callee mutates **same** List object.

### 10. [Application]
**Answer:** **Нет**, `mine` unchanged (empty or prior contents). Reassignment **`items = new ...`** меняет только **local parameter slot**, not caller's reference.

### 11. [Analysis]
**Answer:** **D** неверно. **`int` copy** — inline bits в stack/frame, **не** отдельный heap object.

### 12. [Analysis]
**Answer:** Local **`int`** живёт в stack frame; **GC не «собирает int»** как object. GC reclaim **heap objects** unreachable by references. Frame pop удаляет stack slots без GC для value types.

### 13. [Understanding]
**Answer:** `M(int x)` — копия **4/8 bytes value**. `M(Customer c)` — копия **reference** (pointer), object shared.

### 14. [Application]
**Answer:** `string Name` → **`null`**. `int Count` → **`0`**.

### 15. [Analysis]
**Answer:** **Не противоречит.** Две variables могут **alias** один heap object; intern pool (`1.1.4`) — оптимизация, где несколько refs делят один String object.

### 16. [Understanding]
**Answer:** Struct pass-by-value **копирует все bytes** (256) каждый вызов — CPU/memcopy. Reference pass — **pointer-sized copy**, object один на heap.
