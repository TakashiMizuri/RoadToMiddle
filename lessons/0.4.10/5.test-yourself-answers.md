## Answers

### 1. [Recall]
**Answer:** **Scheme + host + port** (e.g. `https`, `app.example.com`, `443`).

---

### 2. [Recall]
**Answer:** **Same origin** — path differs, origin tuple same.

---

### 3. [Recall]
**Answer:** **Different origin** — different **ports** (5173 vs 5000).

---

### 4. [Understanding]
**Answer:** CORS enforced by **browser** for JS security. curl/Postman **не выполняют** same-origin policy on scripts.

---

### 5. [Understanding]
**Answer:** Server processed request; response ** lacked** `Access-Control-Allow-Origin` (or wrong origin). Browser **hides body** from JS despite HTTP 200.

---

### 6. [Understanding]
**Answer:** **Non-simple** cross-origin request: e.g. PUT/DELETE, custom headers (`Authorization`), `application/json` content-type.

---

### 7. [Application]
**Answer:** **Yes** — non-simple → **OPTIONS preflight** before GET/POST.

---

### 8. [Recall]
**Answer:** Declares which **origin** may read response in browser.

---

### 9. [Analysis]
**Answer:** CORS controls **browser read access**, not blocking **non-browser clients**. Auth, authorization, validation still required. Attacker curl bypasses CORS.

---

### 10. [Application]
**Answer:** **API Kestrel** (or gateway in front) — must emit CORS headers. CDN static host не помогает для API responses.

---

### 11. [Understanding]
**Answer:** Proxy makes API appear **same origin** (`localhost:5173`) — browser sees one origin, SOP satisfied.

---

### 12. [Recall]
**Answer:** **No** — browser rejects `*` with credentials; need **specific** origin.

---

### 13. [Application]
**Answer:** `curl -H "Origin: https://app.example.com" -I https://api.example.com/...` and inspect headers. Or OPTIONS request manually.

---

### 14. [Understanding]
**Answer:** Different **hosts** (`app` vs `api`) = **different origins** — cross-origin fetch requires CORS.

---

### 15. [Analysis]
**Answer:** **No CORS** for `/api` if same scheme/host/port as SPA — **same origin**. (Path doesn't split origin.)
