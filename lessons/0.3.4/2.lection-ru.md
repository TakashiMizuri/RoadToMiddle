# HTTP Methods: GET, POST, PUT, PATCH, DELETE, HEAD, OPTIONS

> Roadmap: `0.3.4` · Node: `0.3` — HTTP и Web · Depth: **глубоко**

## Learning Objectives

После этой лекции ты сможешь:

- Сформулировать назначение **GET, POST, PUT, PATCH, DELETE, HEAD, OPTIONS** в HTTP semantics.
- Сопоставить methods с **CRUD** и REST без worship method as единственного design tool.
- Объяснить, почему **POST** — catch-all для non-idempotent actions, а **GET** не должен менять server state.
- Выбирать **PUT vs PATCH** для full vs partial update с правильными idempotency expectations.
- Использовать **HEAD** для metadata-only и **OPTIONS** для capabilities (включая CORS preflight).
- Настраивать ASP.NET Core routing для restriction methods per endpoint.
- Избегать anti-patterns: GET delete, POST everything, игнор **405 Method Not Allowed**.

---

## Why This Matters

**Method** — verb в первой строке request (`0.3.3`). Он говорит server'у, *какую операцию* client намеревается выполнить на **request-target** (`0.3.2`). `GET /api/orders/42` читает; `DELETE /api/orders/42` удаляет. Frameworks route по method + path — `[HttpGet]`, `MapPost`.

Команды, делающие всё через POST с `{ "action": "delete" }`, теряют semantics HTTP, cache, intermediary safety. Destructive GET — **CSRF** и cache poisoning. Middle dev выбирает methods осознанно; **safe** и **idempotent** (`0.3.5`) drive retry, CDN, browser prefetch.

Methods связаны со status codes (`0.3.6`), headers `Allow`/`Location`, REST (`0.3.10`).

## Core Concepts

### Method as intent, not enforcement

Methods выражают **intent**. Server может вернуть **405** + `Allow`. HTTP не запрещает delete на GET кодом — но ломает caches, proxies, clients.

### GET — Retrieve

**Safe**, **idempotent** (`0.3.5`). Fetch representation. Body в GET — undefined; не отправляй.

Cacheable при правильных headers (`0.3.7`). **Никогда** GET для mutation (logout link, delete в href).

```csharp
app.MapGet("/api/products/{id}", (int id) => GetProduct(id));
```

### POST — Process / Create

**Not safe**, обычно **not idempotent** — повтор может создать дубликаты.

Create: `POST /api/orders` → **201** + `Location`. Commands: login, search с complex body.

```csharp
app.MapPost("/api/orders", (CreateOrderDto dto) => CreateOrder(dto));
```

**Idempotency-Key** — application-layer retry для POST.

### PUT — Replace at URI

**Idempotent**, **not safe**. Full replace at URI. `PUT /api/users/42` — весь resource JSON.

PUT create: client задаёт URI; POST create — server assigns id.

### PATCH — Partial update

**Not safe**; idempotency зависит от patch format. Partial fields или JSON Patch.

Null vs missing в partial DTO — document merge rules.

### DELETE — Remove

**Idempotent**, **not safe**. Повторный DELETE — 204/404, не multiply effect.

Не `?delete=true` на GET.

### HEAD — GET without body

Как GET, без body. **Safe**, **idempotent**. Metadata: ETag, Content-Length.

### OPTIONS — Discover methods

**Safe**. `Allow: GET, POST, OPTIONS`. **CORS preflight** — browser OPTIONS перед cross-origin POST с custom headers.

### Summary table

| Method | Safe | Idempotent (typical) | Body request |
|--------|------|---------------------|--------------|
| GET/HEAD | Yes | Yes | No |
| POST | No | No* | Often |
| PUT/DELETE | No | Yes | Often/Rare |
| PATCH | No | Sometimes | Often |
| OPTIONS | Yes | Yes | No |

## Under the Hood

### Routing

ASP.NET Core match **method + template**. Wrong method on path → **405** (if configured). `HttpMethodMetadata`, `[AcceptVerbs]`.

### GET caching

CDN/proxy cache **method + URL**. GET mutation → stale wrong data in cache.

### CORS preflight

Cross-origin JSON + custom headers → **OPTIONS**. `Access-Control-Allow-Methods` must include real method.

### Method override legacy

`_method=PUT` — avoid в new JSON APIs.

### REST state machine

GET read, PUT replace, PATCH merge, DELETE remove, POST create/action. `POST .../cancel` OK для side-effect command.

```bash
curl -X DELETE https://api.example.com/users/1
curl -X HEAD -I https://api.example.com/users/1
```

```csharp
[HttpPatch("{id}")]
public IActionResult Patch(int id, [FromBody] PatchDto dto) => Ok(...);
```

## Common Mistakes & Anti-patterns

POST для всего; GET с side effects; PUT partial без rules; 404 вместо 405; DELETE с huge 200 body вместо 204.

---

## Production & Real-World Notes

OpenAPI documents methods. Rate limit POST login. AuthZ на DELETE/PATCH. GraphQL — POST only — другая paradigm.

---

## Comparison / Trade-offs

PUT vs POST create — who owns URI. PATCH vs PUT — bandwidth vs simplicity. RPC POST vs REST verbs.

## Quick Reference

GET read · HEAD headers · POST create/command · PUT replace · PATCH partial · DELETE remove · OPTIONS capabilities.

---

## Key Takeaways

- Method = intent на URL в start line.
- GET/HEAD safe; PUT/DELETE idempotent по standard semantics.
- POST — duplicates без idempotency keys.
- OPTIONS + CORS preflight.
- 405 + Allow когда method unsupported.
- ASP.NET Core: method + path routing.
