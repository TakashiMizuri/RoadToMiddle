# Exercises: IP Addresses, Ports, Sockets

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. URL decomposition
**Goal:** Link `0.3.2` to `0.4.2`.

For each URL, list **host**, **port** (explicit or default), and **layer** where each is used:

1. `https://github.com/octocat`
2. `http://10.0.0.5:8080/health`
3. `postgresql://db.local:5432/myapp`

**Success criteria:** Correct default ports; host vs path separated.

---

### E2. Private or public?
**Goal:** Recognize RFC 1918 space.

Classify each IPv4 as **private**, **public**, or **special** (loopback/link-local):

- `127.0.0.1`
- `8.8.8.8`
- `172.17.0.2`
- `169.254.1.1`

**Success criteria:** Four labels with brief reason.

---

## Medium

### M1. netstat inspection
**Goal:** See listening sockets on your machine.

Run `netstat -ano | findstr LISTENING` (or `ss -tlnp` on Linux). Pick two lines; identify **local IP:port**, guess process (PID → Task Manager), and protocol (TCP/UDP).

**Success criteria:** Two endpoints documented; explain why `0.0.0.0:443` vs `127.0.0.1:5000` differ for remote access.

---

### M2. Docker publish mapping
**Goal:** Port NAT mental model.

Write what listens where when you run:

```bash
docker run -p 5000:8080 -p 5001:8080 myapi:latest
```

Can both mappings work simultaneously? Why or why not?

**Success criteria:** Identifies host port uniqueness constraint.

---

## Hard (stretch)

### H1. Design connection limits
**Goal:** Reason about ephemeral ports.

One client IP opens 30,000 outbound HTTPS connections to the same `(server IP, 443)`. What resource on the **client** might exhaust first? What on the **server**?

**Success criteria:** Mentions ephemeral port range (~28k usable), server memory/file descriptors, possible NAT table limits.
