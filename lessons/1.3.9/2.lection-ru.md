# Deferred execution — lazy IEnumerable

> Roadmap: `1.3.9` · Node: `1.3` — C#: продвинутые темы · Depth: **глубоко**

## Learning Objectives

После этой лекции ты сможешь:

- Определить **deferred execution** (lazy evaluation) для LINQ на **`IEnumerable<T>`**.
- Различать **query construction** и **query execution** (enumeration).
- Назвать triggers execution: **`foreach`**, **`ToList`**, **`Count`**, terminal operators (`1.3.10`).
- Предсказать поведение при **multiple enumeration**.
- Найти bugs от **captured mutable state** и **changing data sources**.
- Разделить operators, которые **stream**, vs **buffer** sequence.
- Связать deferral in-memory с **`IQueryable`** до SQL (`1.3.11`).

## Why This Matters

В `1.3.7`–`1.3.8` ты писал pipelines вроде `.Where(...).Select(...)`. Типичная ошибка новичка: «Я вызвал **Where**, значит список уже отфильтрован». Нет — **ещё нет**. LINQ to Objects строит **recipe** (цепочку iterators) и ждёт, пока consumer **pull** элементы через эту цепочку.

Deferred execution даёт силу: compose filters без копирования data; собирать query условно и выполнить один раз. И опасность: enumerate twice → run twice; держать query пока database меняется → inconsistent results; side effects в lambdas → непредсказуемое число вызовов. Production incidents от «materialized too early» или «enumerated thrice without snapshot» — routine.

Entity Framework Core опирается на ту же идею: **`IQueryable`** остаётся deferred до **`ToListAsync`**, **`CountAsync`** и т.д. — тогда один SQL round-trip. Понимание **`IEnumerable`** deferral сначала делает поведение EF predictable, а не mystical.

## Core Concepts

### Construction vs execution

```csharp
var query = numbers.Where(n => n > 0).Select(n => n * 2);  // construction
foreach (var x in query)  // execution
    Console.WriteLine(x);
```

**Construction** аллоцирует iterator objects и хранит delegates. **Execution** — когда consumer вызывает **`MoveNext()`** на enumerator: через **`foreach`**, terminal LINQ methods или manual enumeration.

Присвоение **`var q = ...`** never означает work над elements. Это отличается от eager APIs (**`List.FindAll`**, **`Array.Sort`**) — они compute immediately.

### Immediate operators в LINQ (не deferred result)

**`Count`**, **`ToList`**, **`First`**, **`Sum`**, **`Max`**, **`Aggregate`** — execute on call (`1.3.10`). Всё **до** terminal в chain — deferred до pull.

### Multiple enumeration

```csharp
var evens = numbers.Where(n => n % 2 == 0);
var count = evens.Count();   // pass 1
var first = evens.First();   // pass 2
```

Каждый enumeration **re-executes** pipeline. **Fix:** **`.ToList()`** когда нужен reuse.

### Mutating source

Deferred query читает source **на момент execution**, не construction. Modify collection during **`foreach`** → **`InvalidOperationException`**.

### Captured variables

```csharp
int min = 10;
var q = values.Where(v => v > min);
min = 20;
Console.WriteLine(q.Count());  // min = 20
```

Copy to local в loops при dynamic query building.

### Streaming vs buffering

**Stream:** **`Where`**, **`Select`**, **`Take`**, **`Skip`**.  
**Buffer:** **`OrderBy`**, **`GroupBy`**, **`Distinct`**, **`Join`** — internal collections, often full scan.

### Side effects в lambdas

**`calls`** в **`Where`** increment per **element per enumeration** — может быть 0, N, или 2N.

### Conditional query building

```csharp
IEnumerable<Order> query = dbOrders;
if (filterByDate) query = query.Where(o => o.Date >= start);
var page = query.Skip(...).Take(...).ToList();  // one execution
```

EF: один SQL с composed **`WHERE`** — payoff deferral в API.

## Under the Hood

### Iterator blocks и `yield return`

Большинство deferred operators — **iterator blocks** (`1.3.20`). Compiler генерирует state machine class с **`MoveNext`**, **`Current`**, **`Dispose`**. **`Where`** conceptually:

```csharp
foreach (var item in source)
    if (predicate(item))
        yield return item;
```

**`foreach`** over **`query`** drives nested **`MoveNext`** — matryoshka enumerators, по одному на operator в chain.

### No magic caching

LINQ to Objects **не cache** results **`Where`/`Select`**. **`ICollection<T>`** source может дать **`Count()`** fast path, но **`Where`** wrapper всё равно re-filter при enum.

### Relationship to IQueryable

| | IEnumerable deferral | IQueryable deferral |
|--|---------------------|---------------------|
| Хранит | Delegates | Expression trees |
| Executes | CLR loops | Provider (SQL) |
| Re-enum | Re-run loops | Re-send query (unless materialized) |

Lesson **`1.3.11`–`1.3.12`**. Mental habit один: build first, execute once intentionally.

## Examples

### Example 1 — Double enumeration cost

```csharp
IEnumerable<int> expensive = ReadNumbersFromNetwork()
    .Where(n => SlowIsPrime(n));
expensive.Any();   // compute
expensive.Any();   // compute again
```

Materialize **`ToList()`** после expensive **`Where`**, если reuse нужен.

### Example 2 — Any then foreach

```csharp
var q = logs.Where(l => l.Level == LogLevel.Error);
if (!q.Any()) return;
foreach (var log in q) Process(log);  // second scan
```

Лучше: **`var list = q.ToList()`** once.

## Common Mistakes & Anti-patterns

- Думать, что **`var q = ...`** уже отфильтровал
- **`Count()`** + **`foreach`** = two passes
- Side effects в **`Select`**
- **`ToList()`** too early (RAM) / too late (duplicate SQL)
- Modify collection during **`foreach`** on deferred query
- EF: enumerate after **`DbContext`** disposed

---

## Production & Real-World Notes

Profile with intention: **`ToList()`** too early — millions rows в RAM; too late — duplicate SQL. Middle rule: **materialize at boundary** — API response, view model, end of request — once semantics fixed.

---

## Comparison / Trade-offs

| Deferred | Eager (materialize) |
|----------|---------------------|
| Compose filters cheaply | Stable snapshot |
| Single pass if enum once | Safe multiple iteration |
| Reads live source | Predictable при mutations |

## Quick Reference

- **Deferred:** most **`IEnumerable`** LINQ operators
- **Execute on:** **`foreach`**, **`ToList`**, **`Count`**, **`First`**
- **Re-enumerate** = re-run pipeline
- **Buffering:** **`OrderBy`**, **`GroupBy`**, **`Distinct`**
- **Fix reuse:** **`.ToList()`** once

---

## Key Takeaways

- LINQ **описывает**; enumeration **выполняет**.
- Re-enumeration = re-run from source.
- **`OrderBy`/`GroupBy`** buffer.
- Materialize **once** at API boundary when semantics fixed.
