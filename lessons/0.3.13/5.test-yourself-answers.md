## Answers

### 1. [Recall]
**Answer:** **Несколько parallel TCP connections** на один host (historically ~6).
**Why:** Одна connection HTTP/1.1 — один request in flight (без working pipelining).

### 2. [Recall]
**Answer:** **TLS ALPN** — client предлагает `h2`, server принимает during handshake.
**Why:** HTTP/2 over TLS de facto через ALPN.

### 3. [Recall]
**Answer:** **QUIC over UDP** (HTTP semantics поверх QUIC).
**Why:** HTTP/3 = mapping HTTP to QUIC transport.

### 4. [Understanding]
**Answer:** Потеря **одного TCP packet** останавливает delivery **всех streams** на connection, пока TCP не восстановит порядок bytes.
**Why:** HTTP/2 multiplexes на одном TCP byte stream.

### 5. [Understanding]
**Answer:** QUIC изолирует **per-stream** loss — потеря в одном stream не блокирует другие так же, как TCP блокирует весь connection.
**Why:** Multiplexing ниже, на transport designed for independent streams.

### 6. [Understanding]
**Answer:** **Нет** — semantics (methods, codes, headers) сохраняются.
**Why:** Versions меняют framing/transport, не REST model.

### 7. [Application]
**Answer:** **HTTP/2**
**Why:** ALPN `h2` = HTTP/2 over TLS.

### 8. [Application]
**Answer:** Подключиться по **HTTP/3** на port 443 (если client поддерживает).
**Why:** Alt-Svc advertises alternative service.

### 9. [Analysis]
**Answer:** **Legacy clients** без HTTP/2 не смогут работать — need HTTP/1.1 fallback.
**Why:** Not all embedded/old stacks speak h2.

### 10. [Analysis]
**Answer:** **Нет** — bottleneck database, не wire protocol.
**Why:** Protocol optimizes connection efficiency, not query time.

### 11. [Recall]
**Answer:** **Header compression** (static + dynamic tables).
**Why:** Repeated cookies/auth headers shrink on wire.

### 12. [Understanding]
**Answer:** Меньше **отдельных requests** — обход limit parallel connections и HoL на HTTP/1.1.
**Why:** One big file vs many small on few conns.

### 13. [Analysis]
**Answer:** **Нет** для корректности REST; HTTP/1.1 достаточен. HTTP/2 — opportunistic performance.
**Why:** gRPC mandates h2; typical JSON REST does not.

### 14. [Application]
**Answer:** **HTTP/2 or HTTP/1.1 over TCP** (HTTPS on TCP 443).
**Why:** HTTP/3 needs UDP; blocked → fallback.
