## Answers

### 1. [Recall]
**Answer:** Declarative metadata object (subclass of `Attribute`); данные в **assembly metadata** / IL `.custom`, не в теле метода.
**Why:** Compiler embeds; runtime/framework reads.

### 2. [Recall]
**Answer:** `GetCustomAttribute<T>()` (extension on `MemberInfo` / `ParameterInfo` etc.).
**Why:** Standard BCL API.

### 3. [Understanding]
**Answer:** ASP.NET Core at **startup** reflects controller types, находит routing HTTP attributes, регистрирует endpoints в route table — без твоего явного кода.
**Why:** Framework consumer of metadata.

### 4. [Understanding]
**Answer:** Generator emits **concrete code** at compile time — no runtime scan; быстрее, AOT-friendly. Runtime reflection создаёт/читает attributes when app runs — гибче, дороже, проблемы с trimming.
**Why:** Trade-off flexibility vs performance.

### 5. [Application]
**Answer:** **Compile error** — `Mark` на field, но `AttributeTargets.Method` only.
**Why:** AttributeUsage enforcement.

### 6. [Application]
**Answer:**
```csharp
typeof(Order).GetMethod("Save")!.GetCustomAttribute<DescriptionAttribute>();
```
**Why:** Nullable `GetMethod` may need `!` if sure exists.

### 7. [Understanding]
**Answer:** Late binding, boxing args, permission checks, no inlining — indirect dispatch vs direct call/jump.
**Why:** Reflection overhead.

### 8. [Analysis]
**Answer:** **Неверно.** Attributes — **data**; CLR не «выполняет» их автоматически. Consumer (framework, твой код) читает metadata.
**Common mistake:** Confusion with aspect-oriented «weaving» (not default C#).

### 9. [Application]
**Answer:** Refactor-safe rename; `"Name"` typo → **null** at runtime silently or fail later.
**Why:** Compile-time check on symbol.

### 10. [Understanding]
**Answer:** Не искать attributes на **base** members при lookup на derived type — только declared level (depending on overload + flags).
**Why:** Inheritance control for metadata.

### 11. [Recall]
**Answer:** Любые два: `[Route]`, `[HttpGet]`, `[Authorize]`, `[ApiController]`, `[FromBody]`, etc.
**Why:** —

### 12. [Analysis]
**Answer:** Trimmer удаляет «неиспользуемые» types/methods; reflection discovery needs **preserved** metadata — без `[DynamicallyAccessedMembers]` / RD.xml endpoints не найдутся.
**Why:** AOT static analysis vs dynamic reflection.
