# Events: Add/Remove Accessors and Memory Leak Patterns

> Roadmap: `1.3.5` · Node: `1.3` — C# advanced topics · Depth: **глубоко**

## Learning Objectives

After this lesson you will be able to:

- Distinguish **`event`** from **`delegate` field** and explain **add/remove accessors**.
- Implement **custom event accessors** with **`EventHandlerList`** or backing delegate field.
- Explain the **publisher-subscriber memory leak**: long-lived publisher holds subscriber references.
- Apply fixes: **unsubscribe (`-=`)**, **weak event pattern** (concept), **short-lived publisher**, **`WeakReference`** trade-offs.
- Recognize **anonymous/lambda** and **capturing** leaks in UI and domain services.
- Follow **thread-safe event** raising patterns and **`?.Invoke`** null checks.

---

## Why This Matters

```csharp
public class Button
{
    public event EventHandler? Click;
}
```

Looks harmless. If **`Form`** subscribes **`button.Click += Form.HandleClick`** and **`Form`** is disposed but **`Button`** lives longer (static toolbar, singleton event bus), **`Form` stays reachable** through the delegate's target reference — **memory leak** + **zombie callbacks** (ObjectDisposedException or corrupt state).

ASP.NET Core uses **`IHostedService`** cancellation tokens more than classic events, but **domain events**, **Reactive Extensions**, **UI** (even Blazor **`EventCallback`** has different model), and **legacy integrations** still use **`event`**. Middle developers debug «memory grows overnight» and «handler called after dispose» — root cause is often **missing `-=`** or **static event** holding entire graph.

Understanding **custom add/remove** matters for **thread-safe collections**, **weak references**, and **Source Generators** for observability — and for reading framework source (`PropertyChanged`, `CollectionChanged`).

## Core Concepts

### `event` Keyword — Encapsulation

Compiler transforms:

```csharp
public event EventHandler? SomethingHappened;
```

Into **private backing field** + **public add/remove** only (no external **`=`**):

```csharp
private EventHandler? SomethingHappened;
public event EventHandler? SomethingHappened
{
    add => SomethingHappened += value;
    remove => SomethingHappened -= value;
}
```

Outside code: **`+=`** and **`-=`** only. Cannot **`obj.SomethingHappened = null`** from consumer — protects invocation list.

Raising from inside class: invoke backing field **`SomethingHappened?.Invoke(this, EventArgs.Empty);`**

### Standard Signatures

**`EventHandler`** → **`void(object? sender, EventArgs e)`**

**`EventHandler<TEventArgs>`** where **`TEventArgs : EventArgs`**

Custom **`EventArgs`** derivatives carry payload (**`PropertyChangedEventArgs`**, domain event data).

### Custom Add/Remove Accessors

```csharp
private EventHandler? _click;
private readonly object _lock = new();

public event EventHandler Click
{
    add
    {
        lock (_lock) { _click += value; }
    }
    remove
    {
        lock (_lock) { _click -= value; }
    }
}
```

Use when you need **logging subscriptions**, **weak references**, or **synchronized** list mutation. Default compiler accessors are sufficient for most code.

### Memory Leak Pattern

**Publisher** (long lifetime) holds **multicast delegate** → each **subscriber** (target object) **reachable from GC root** through publisher.

```
Static EventBus ──► delegate ──► Form instance (should be collected)
                      ▲
                 still subscribed
```

**Subscriber shorter than publisher** must **` -= ` on dispose/unload**.

**Capturing lambda** makes **display class** the target:

```csharp
button.Click += (s, e) => this.DoWork(); // targets closure holding `this`
```

Even instance method **`button.Click += HandleClick`** holds **`this`** reference from publisher side.

### Static Events — Global Leaks

```csharp
public static event EventHandler? GlobalRefresh;
```

Every subscriber stays alive until **unsubscribed** or **AppDomain** unload — classic WPF/WinForms footgun. Prefer **instance events** or **mediator** with scoped lifetime.

### Fix Strategies

1. **Always unsubscribe** in **`Dispose`**, **`IAsyncDisposable`**, **`IDisposable`**, component **`OnUnmounted`** (Blazor), **`finally`** block.

```csharp
public void Dispose()
{
    _publisher.DataChanged -= OnDataChanged;
}
```

2. **Weak event pattern** — store **`WeakReference`** to subscriber; on raise, prune dead refs. Used in WPF **`WeakEventManager`**. Complexity cost; not in BCL for general events.

3. **Short-lived publisher** — scope event source to same lifetime as subscribers.

4. **Avoid lambdas** when you need clean unsubscribe — use named method matching **`-=`** identity (`1.3.3`).

5. **CancellationToken** instead of event for one-shot async workflows (modern ASP.NET).

### Raising Events Safely

```csharp
protected virtual void OnSomethingHappened(EventArgs e)
{
    SomethingHappened?.Invoke(this, e);
}
```

**Snapshot pattern** for thread safety (optional advanced):

```csharp
var handlers = SomethingHappened;
handlers?.Invoke(this, e);
```

Copy local prevents race if **`remove`** on another thread between null check and invoke — still not full guarantee for all scenarios; **`lock`** on add/remove/raise if required.

**Re-entrancy**: handler subscribes/unsubscribes during raise — iterate **`GetInvocationList()`** if needed.

## Under the Hood

### Compiler-Generated Accessors

**`event`** field-like declaration stores delegate in private field; metadata marks **`event`** with **`.addon`** / **`.removeon`**. Reflect **`GetEvent("Click").GetAddMethod()`**.

### Delegate Holds Strong Reference

**`Target`** property on **`Delegate`** — strong ref to subscriber. No GC of subscriber until removed from invocation list and publisher collectable.

### `EventHandlerList` (WinForms legacy)

**`System.ComponentModel.EventHandlerList`** — keyed listeners for multiple events on one object — used in **Windows Forms** designer-generated code; pattern for custom storage.

### Events vs IObservable

**Rx** **`IObservable<T>`** / **`Subscribe`** returns **disposable** subscription — **`using`** unsubscribes automatically — better lifetime story for streams.

## Examples

### Leak Demo (Conceptual)

```csharp
public sealed class Cache
{
    public event Action? Invalidated;

    public void Invalidate() => Invalidated?.Invoke();
}

// Short-lived listener
var listener = new Listener(cache);
cache.Invalidated += listener.OnInvalidated;
// listener out of scope but NOT collected if still subscribed
```

**Fix:** `cache.Invalidated -= listener.OnInvalidated` in `Listener.Dispose`.

### Named Handler for Unsubscribe

```csharp
private void OnClick(object? sender, EventArgs e) { /* ... */ }

void Subscribe() => _button.Click += OnClick;
void Unsubscribe() => _button.Click -= OnClick;
```

### Custom Accessor with Count (Debug)

```csharp
private EventHandler? _handlers;
public event EventHandler Handlers
{
    add { _handlers += value; LogCount(); }
    remove { _handlers -= value; LogCount(); }
}
```

## Common Mistakes & Anti-patterns

**Subscribe in constructor, never unsubscribe** — leak when publisher outlives subscriber.

**Lambda subscription in long-lived object** — cannot `-=` without stored reference.

**Raising event from wrong thread** without UI **`SynchronizationContext`** — cross-thread exception in UI apps.

**Public raise** — external code invokes your event; use **`protected virtual OnXxx`** pattern.

**Swallowing all exceptions** in one handler — others may not run; log and policy.

**Assuming `event` is weak** — default is **strong** references always.

## Production & Real-World Notes

**IDisposable** pattern: unsubscribe in **`Dispose(bool disposing)`**.

**Unit tests**: mock with **`Mock<EventHandler>`** or count invocations; reset between tests.

**Domain events** in EF/CQRS: MediatR **`INotification`** avoids manual event lifetime if scoped per request.

**Blazor**: prefer **`EventCallback<T>`** — different semantics, async-friendly.

Memory profiler (dotMemory, VS Diagnostic Tools): look for **delegate** paths to disposed objects.

## Comparison / Trade-offs

| Approach | Lifetime control | Complexity |
|---|---|---|
| **event + `-=`** | Manual | Low |
| **Weak event** | Automatic prune | High |
| **IObservable + IDisposable** | `using` subscription | Medium |
| **CancellationToken** | Token link | Low for async |

---

## Quick Reference

| Pattern | Leak risk |
|---|---|
| Instance event, unsub in Dispose | Low |
| Static event | High |
| Lambda += without -= | High |
| Publisher singleton, subscriber transient | High |

**Rule:** subscriber shorter life than publisher → **must unsubscribe**.

---

## Key Takeaways

- **`event`** restricts external assignment; uses **add/remove** accessors.
- Delegates hold **strong references** to subscribers — GC leak if not removed.
- **Static events** pin subscribers globally.
- Use **named methods** or **stored lambdas** for reliable **`-=`**.
- Raise with **`?.Invoke`**; consider **snapshot** or **lock** for threading.
- Modern async code often prefers **CancellationToken** / **IObservable** over classic events.
