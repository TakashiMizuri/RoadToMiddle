# Postman / Bruno / curl — краткая выжимка

> `0.3.12` · ~8 мин на повторение

## Главная идея

**Manual HTTP clients** (curl, Postman, Bruno) изолируют protocol layer для быстрого debug и reproducible requests. curl — everywhere и scripts; Postman — GUI и team workspaces; Bruno — **collections в Git**.

## Ключевые пункты

- **curl:** `-X`, `-H`, `-d`, `-F`, `-i` (headers), `-v` (verbose/TLS).
- **Postman:** collections + environments `{{baseUrl}}`, test scripts для token.
- **Bruno:** `.bru` files в repo, diff в PR.
- Копируй failing request из **DevTools Network** — убери browser-only headers.
- Secrets — environment variables, не commit.
- Windows: **`curl.exe`**, не PowerShell alias.

## Самые важные примеры

```bash
curl -X POST "$BASE/api/items" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"test"}' -i
```

## «Под капотом» в одном абзаце

curl использует libcurl (DNS → TCP → TLS → HTTP). Postman/Bruno сериализуют multipart и могут генерировать curl/fetch из working request.

## Частые ошибки

- Смотреть только JSON body, игнорируя **429 + Retry-After**.
- Тестировать только Swagger UI без auth flow.
- Shared collection с live production tokens.

## Мнемоника

> **Reproduce → isolate header/body → fix → save to collection.**
