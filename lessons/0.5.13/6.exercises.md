# Exercises: SEO Metadata in HTML

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Fix the Head Block
**Goal:** Correct a broken `<head>` SEO setup.
**Steps:**
1. Create `pricing.html` with intentional errors: duplicate `<title>`, missing charset, relative canonical, `name="og:title"`.
2. Fix all issues per lesson rules.
3. Validate mentally: one title, absolute canonical, `property` on OG tags.
**Success criteria:** Valid minimal head with unique title, description, canonical, and three OG tags.

### E2. Write Snippets for Three Pages
**Goal:** Practice unique metadata per route.
**Steps:** For a fictional docs site, write `<title>` + meta description for: Home, API Reference, Changelog. Keep titles under ~60 chars; descriptions ~150 chars each.
**Success criteria:** All six strings unique, human-readable, no keyword stuffing.

## Medium

### M1. Duplicate URL Scenario
**Goal:** Apply canonical correctly.
**Steps:** Document four URL variants of one blog post (http/https, www, trailing slash, UTM). Pick canonical. Write the `<link rel="canonical">` and explain in one paragraph why others stay 200 OK.
**Success criteria:** Single absolute canonical; explanation mentions link equity and tracking params.

### M2. Social Preview Audit
**Goal:** Design OG tags for an article.
**Steps:** Pick any public tech article URL (or invent one). Draft full OG block including 1200×630 image URL pattern. List two checks before deploy (image reachable, absolute URLs).
**Success criteria:** Complete OG set; checklist mentions HTTP 200 on image and no auth wall.

## Hard (stretch)

### H1. SPA vs SSR Metadata Plan
**Goal:** Connect HTML lesson to future React work.
**Steps:** Sketch architecture for a marketing site + authenticated app shell on same domain. Which routes need SSR/prerender for SEO/OG? Which can be client-only? One paragraph trade-off.
**Success criteria:** Public pages identified for server metadata; `/app/dashboard` justified as noindex or client-only.
