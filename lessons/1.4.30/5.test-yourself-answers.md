# Answers: Fire-and-Forget Task

## Scoring

- 90–100%: отлично · 80–89%: pass · <80%: повтори lection + summary

## Answers

### 1. [Recall]
**Answer:** Запуск `Task` без ожидания завершения в текущем потоке управления.
**Why:** Work continues after caller moves on.

### 2. [Recall]
**Answer:** Нельзя await; исключения — unhandled; нет нормального observation path.
**Why:** `async void` designed for UI event handlers only.

### 3. [Understanding]
**Answer:** Unobserved exceptions; disposed scoped DI; wrong cancellation token / no backpressure.
**Why:** Core production traps from lesson.

### 4. [Understanding]
**Answer:** Request scope ends; `DbContext` disposed while task still uses it.
**Why:** Scoped lifetime tied to HTTP request.

### 5. [Application]
**Answer:** Нет — клиент уже получил успешный ответ; exception unobserved или в глобальном handler.
**Why:** No await in request pipeline.

### 6. [Application]
**Answer:** `IServiceScopeFactory.CreateScope()` в worker, `GetRequiredService<T>()` внутри scope.
**Why:** Fresh scope per background unit.

### 7. [Analysis]
**Answer:** `_db` scoped и disposed; fire-and-forget; `Task.Run` для I/O без причины; `async void`-like fire in sync `Post`.
**Why:** Multiple anti-patterns stacked.

### 8. [Understanding]
**Answer:** CPU-bound сегмент, который нельзя await на I/O thread и который короткий/ограничен — всё равно лучше очередь при масштабе.
**Why:** Offload CPU, not I/O.

### 9. [Analysis]
**Answer:** Логирует fault, но не решает disposed scope, durability, retry, backpressure.
**Why:** Observation ≠ correct hosting.

### 10. [Application]
**Answer:** Обычно нет — нужен отдельный token или none; иначе disconnect отменит email.
**Why:** Post-response work outlives client connection.

### 11. [Understanding]
**Answer:** Durability в БД, retry, survives crash, transactional with order.
**Why:** Outbox is persistent queue.

### 12. [Recall]
**Answer:** CS4014 — call not awaited.
**Why:** Compiler warns on unobserved async calls.

### 13. [Analysis]
**Answer:** Thread pool / memory (queued work); possible unobserved exceptions storm.
**Why:** Unbounded detached tasks.

### 14. [Understanding]
**Answer:** Centralized await loop, scopes, shutdown token, bounded queue.
**Why:** Hosted lifecycle.

### 15. [Application]
**Answer:** POST handler awaits SaveChanges → enqueue work item with order id → return 202; BackgroundService dequeues, creates scope, sends notification, logs errors.
**Why:** Safe separation of HTTP and background work.
