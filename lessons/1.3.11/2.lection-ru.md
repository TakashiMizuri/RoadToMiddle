# IEnumerable vs IQueryable — фундаментальная разница

> Roadmap: `1.3.11` · Node: `1.3` — C#: продвинутые темы · Depth: **глубоко**

## Learning Objectives

После этой лекции ты сможешь:

- Contrast **`IEnumerable<T>`** (in-memory) и **`IQueryable<T>`** (expression tree + provider).
- Объяснить **`AsEnumerable()`** / **`AsQueryable()`** и **where** code executes.
- Описать **`IQueryProvider`** и **`Expression`**.
- Различить **client evaluation** vs **server translation** (EF preview).
- Выбирать signatures repository/API.
- Различать **`Func<>`** vs **`Expression<Func<>>`** overloads.
- Применять deferral model (`1.3.9`–`1.3.10`) к remote queries.

---

## Why This Matters

`.Where().Select()` на lists — **`IEnumerable`**, CLR loops. **`DbSet<T>`** — **`IQueryable`**, PostgreSQL. **Identical syntax**, different engine.

Confusion → fetch all rows + filter in C#; **`ToList()`** too early; **`IQueryable`** outside **`DbContext`**. Middle backend must feel boundary before EF chapters.

---

## Core Concepts

### Два interface, один syntax

```csharp
IEnumerable<User> e = usersList.Where(u => u.Age > 18);  // Func — in process
IQueryable<User> q = db.Users.Where(u => u.Age > 18);      // Expression — SQL
```

| | `IEnumerable` | `IQueryable` |
|--|---------------|--------------|
| Storage | Delegates | **Expression tree** |
| Execution | LINQ to Objects | **IQueryProvider** (EF) |
| Source | List, array | **DbSet** |

**`IQueryable`** extends **`IEnumerable`** — not every **`IEnumerable`** is queryable remotely.

### IEnumerable — LINQ to Objects

**`Func<T,bool>`** — compiled delegate, local **`foreach`**. Любой C# code в **`Where`** OK.

### IQueryable — remote description

Properties: **`Expression`**, **`Provider`**, **`ElementType`**.  
**`Where`** append nodes to tree; **`Execute`** → SQL (EF).

Lambdas — **`Expression<Func<...>>`** (`1.3.12`).

### AsEnumerable() — client boundary

```csharp
db.Orders.Where(...).AsEnumerable().Where(o => ComplexLocalLogic(o));
```

После **`AsEnumerable()`** — CLR на уже loaded data. Early **`AsEnumerable()`** — anti-pattern (full table).

### AsQueryable() на List

**`EnumerableQuery`** — **`IQueryable`** facade, still **local** execution. Not SQL.

### Func vs Expression — overload resolution

Same lambda syntax; compiler picks by static type of source (**`IEnumerable`** vs **`IQueryable`**).

### Repository signatures

- **`IQueryable`** — compose in same **`DbContext`** scope (leaky abstraction).
- **`List<T>`/`Task<List<T>>`** — complete query, hide EF.
- **Never** **`IQueryable`** from controller.

### EF client vs server (preview)

Non-translatable **`Where`** → exception (EF Core 3+) or client eval.  
Debug: **`ToQueryString()`**.

---

## Under the Hood

**`Queryable`** static class — parallel to **`Enumerable`**.  
**IQueryable path:** terminal → **`provider.Execute(tree)`** → SQL → rows → **`List`**.

**`EnumerableQuery`:** test double for provider model without DB.

---

## Examples

```csharp
// Good
await db.Products.Where(p => p.IsActive).Select(p => new Dto(...)).ToListAsync();

// Bad
(await db.Products.ToListAsync()).Where(p => p.IsActive);
```

---

## Key Takeaways

- Same syntax; **execution location** differs.
- **`IQueryable`** = expression trees, not local filters.
- **`AsEnumerable()`** — осознанная граница client/server.
- Materialize at scope edge; compose filters on **`IQueryable`** before **`ToListAsync`**.
