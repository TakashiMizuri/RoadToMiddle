# Answers: Task.WhenAll / WhenAny

## Answers

### 1. [Recall]
**Answer:** **`Task`** or **`Task<TResult[]>`** that completes when **all** inputs complete successfully.
**Why:** Composite task semantics.

### 2. [Understanding]
**Answer:** Sequential ~600ms; WhenAll ~200ms (approx max).
**Why:** Overlapped I/O wait.

### 3. [Application]
**Answer:** **OK after WhenAll** — tasks completed; prefer `await t1` for style.
**Why:** `.Result` before complete blocks — dangerous.

### 4. [Application]
**Answer:** **`await` throws** that exception (first observed).
**Why:** Faulted WhenAll.

### 5. [Application]
**Answer:** **Cancel or ignore** via linked CTS — avoid orphan work.
**Why:** Loser may continue otherwise.

### 6. [Analysis]
**Answer:** **Parallel I/O** if `ProcessAsync` truly async I/O — good pattern.
**Why:** All tasks started in Select before WhenAll.

### 7. [Analysis]
**Answer:** **Thread pool starvation** (`1.4.10`).
**Why:** Unbounded CPU on pool.

### 8. [Understanding]
**Answer:** **Нет** для I/O wait — threads released; continuations on pool.
**Why:** `1.4.2`, `1.4.7`.

### 9. [Application]
**Answer:** **Продолжают** без token — must pass token to each child.
**Why:** `1.4.11`.

### 10. [Application]
**Answer:** **Throws** winner's exception.
**Why:** WhenAny returns Task, not result.

### 11. [Analysis]
**Answer:**
```csharp
var aTask = GetAAsync();
var bTask = GetBAsync();
await Task.WhenAll(aTask, bTask);
var result = (await aTask, await bTask);
```
**Why:** Start both before awaiting.

### 12. [Practical]
**Answer:** Three tasks + `CreateLinkedTokenSource(user, timeout)` + `WhenAll` + DTO map.
**Why:** Standard aggregation pattern.
