# async void — краткая выжимка

> `1.4.13` · ~8 мин

## Главная идея

**`async void`** — только event handlers. Нет **`Task`** → exceptions **не observable** caller'ом; unhandled / crash. Та же state machine (`1.4.7`), **`AsyncVoidMethodBuilder`**.

## Ключевые пункты

- `async Task` — await + catch.
- `async void` — top-level try/catch в handler.
- Не в services, tests, ASP.NET actions.
- Fire-and-forget → `async Task` + log (`1.4.30`).
