## Answers

### 1. [Recall]
**Answer:** Target object (or null for static) + method reference (MethodInfo / function pointer).
**Why:** Enables type-safe callback invocation.

### 2. [Recall]
**Answer:** Returns array clone of all handlers in multicast chain.
**Why:** Safe iteration / individual invoke without modifying during broadcast.

### 3. [Recall]
**Answer:** **3** ‚Äî only last handler's return value.
**Why:** Multicast `Invoke` overwrites result each iteration.

### 4. [Understanding]
**Answer:** Each lambda is separate compiler-generated method unless same delegate instance reused; `-=` matches target+method identity.
**Why:** No structural equality for lambdas.

### 5. [Understanding]
**Answer:** `=` **replaces** entire list with one handler; `+=` **appends** to chain (new combined delegate).
**Why:** Immutability semantics.

### 6. [Understanding]
**Answer:** Delegates immutable ‚Äî Combine allocates new MulticastDelegate + possibly new invocation array.
**Why:** Thread-safety snapshot semantics (old subscribers keep old delegate ref).

### 7. [Application]
**Answer:** **`12`** ‚Äî first lambda remains; second `-=` did not remove first lambda (different method).
**Why:** Classic lambda unsubscribe failure.

### 8. [Application]
**Answer:**
```csharp
Action handler = () => DoWork();
combined += handler;
// later:
combined -= handler;
```

### 9. [Analysis]
**Answer:** **handler3 likely not invoked** ‚Äî exception propagates from default `Invoke`.
**Why:** No per-handler isolation unless custom loop.

### 10. [Analysis]
**Answer:** External code can `Click = null` or replace all subscribers; `event` restricts to `+=`/`-=` only.
**Why:** Encapsulation of invocation list.

### 11. [Application]
**Answer:**
```csharp
int sum = 0;
foreach (Func<int> h in f.GetInvocationList().Cast<Func<int>>())
    sum += h();
```

### 12. [Understanding]
**Answer:** Invoked **twice**; one `-=` removes **one** occurrence.
**Why:** List contains duplicate entries.

### 13. [Recall]
**Answer:** **`MulticastDelegate`** (inherits `Delegate`).

### 14. [What happens]
**Answer:** **`NullReferenceException`** on invoke (not on delegate creation).
**Why:** Null delegate invoke throws.

### 15. [Understanding]
**Answer:** Isolate exceptions, logging per subscriber, avoid one failure aborting others, optional timeout per handler.
**Why:** Production resilience.
