## Answers

### 1. [Recall]
**Answer:** Tree — **data structure** describing code; delegate — **compiled callable** instructions executed directly.
**Why:** Provider inspects trees; CLR runs delegates.

### 2. [Recall]
**Answer:** **`Expression<Func<T, bool>>`** (for `Where` on `T`).
**Why:** `Queryable.Where` overload signature.

### 3. [Understanding]
**Answer:** **`LocalValidate`** — arbitrary method; EF cannot map to SQL node → translation exception (EF Core 3+ default).
**Why:** Provider only understands translatable expression subset.

### 4. [Understanding]
**Answer:** **`CreateQuery`** — returns new **`IQueryable`** with extended **`Expression`** (still deferred). **`Execute`** — runs query now (SQL + materialize/scalar).
**Why:** Core provider contract.

### 5. [Application]
**Answer:** **`List`**: **`Func<User,bool>`** delegate. **`db.Users`**: **`Expression<Func<User,bool>>`** tree.
**Why:** Extension method overload resolution by receiver type.

### 6. [Analysis]
**Answer:** **`Compile()`** produces **`Func`** — **`Where`** binds to **`Enumerable`**/`Func` overload or opaque delegate EF cannot parse into SQL.
**Why:** Tree replaced by runtime delegate — not translatable.

### 7. [Understanding]
**Answer:** **`ParameterExpression`** (`u`), **`MemberExpression`** (`u.Age`), **`BinaryExpression`** (`GreaterThan`) + **`ConstantExpression`** (`18`) — accept Member/Binary/Constant/Parameter trio.
**Why:** Tree anatomy lesson.

### 8. [Application]
**Answer:** **`ToQueryString()`** on **`IQueryable`** (before or instead of execute in dev).
**Why:** Shows generated SQL from current expression.

### 9. [Analysis]
**Answer:** **`equals`** in join clause ensures both sides are **expression tree nodes** combined as equi-join keys; **`==`** not valid in join grammar; important for provider translation of join keys.
**Why:** Spec + `1.3.8` join desugaring.

### 10. [Understanding]
**Answer:** **Нет** — **`EnumerableQuery`** compiles expression to delegates, runs **in memory**.
**Why:** Local test provider, not EF/SQL.

### 11. [Application]
**Answer:** Conditional **`return q.Where(u => u.IsActive == value)`** on **`IQueryable`** — adds **`MethodCallExpression`**; or build **`Expression.Lambda`** manually / specification object.
**Why:** Stay in translatable expression land.

### 12. [Understanding]
**Answer:** Each operator **appends** to tree without executing; **`ToListAsync`** triggers **`Execute`** — same defer-until-terminal as `IEnumerable`, different payload (tree vs iterators).
**Why:** Unified deferral model (`1.3.9`).
