# Анатомия URL — краткая выжимка

> `0.3.2` · ~8 мин на повторение

## Главная идея

URL — структурированный адрес ресурса: **scheme** + **host** (+ **port**) + **path** + **query** + **fragment**. В HTTP request на server уходят **path и query** (+ **Host** header); **fragment (`#...`)** остаётся только у client.

## Ключевые пункты

- Default ports: **http 80**, **https 443** — обычно не пишут в URL.
- **Relative URL** резолвятся от base; `/api` vs `api` — разные absolute paths.
- **Percent-encoding** для пробелов и `&`, `=` в query values: `encodeURIComponent`.
- Секреты (token, password) — **не в query** (logs, Referer); headers или body.
- Path — идентичность ресурса; query — фильтры, pagination, sort.
- **Double encoding** (`%2520`) — encode один раз при сборке URL.

## Самые важные примеры

```
https://host:443/path?a=1&b=2#section
         └ authority ┘
```

Request line:
```
GET /path?a=1&b=2 HTTP/1.1
Host: host
```

**JS URL API:**
```javascript
new URL('https://x.com/p?q=1#h').searchParams.get('q'); // '1'
```

## «Под капотом» в одном абзаце

Browser парсит строку по WHATWG algorithm; Kestrel выставляет PathString/QueryString для routing. DNS и TLS используют scheme+host+port до первого HTTP byte. Fragment не входит в request target по RFC/WHATWG для origin HTTP.

## Частые ошибки

- Ожидать `#hash` на backend.
- `fetch('api/x')` без leading slash с nested page path.
- JWT в query string.
- Путать trailing slash policies между `/users` и `/users/`.

## Мнемоника

> **# hash stays home** — fragment не летит на server; **? query asks options**, **path points** to the resource.
