# LINQ query syntax — краткая выжимка

> `1.3.8` · ~6 мин на повторение

## Главная идея

**Query syntax** (`from`/`where`/`select`) — syntactic sugar над теми же **`Enumerable`/`Queryable`** methods, что и **method syntax** (`1.3.7`). Compiler desugar один в один; выбор — readability, не performance.

## Ключевые пункты

- **`from x in src`**, **`where`**, **`select`**, **`orderby`**, **`group by`**, **`join ... on ... equals`**.
- **`join ... into`** → **`GroupJoin`** + **`DefaultIfEmpty`** (LEFT JOIN pattern).
- **`let`** — промежуточная variable; desugar в nested **`Select`**.
- **`Take`/`Skip`/`Distinct`** — только method syntax (или dot после query в скобках).
- Multiple **`from`** = **`SelectMany`** (flatten nested collections).
- В **`join`** только **`equals`**, не **`==`**.

## Самые важные примеры

```csharp
from u in users
where u.IsActive
orderby u.Email
select u.Email
```

≡ `users.Where(...).OrderBy(...).Select(...)`

```csharp
from o in orders
join c in customers on o.CustomerId equals c.Id
select new { o.Id, c.Name }
```

## Частые ошибки

- **`==` в join**
- Забыть **`into`** после **`group`** при projection per group
- Query syntax для тривиального `.Where()` — шум

## Мнемоника

> **Query = SQL-shaped reads**; **method = pipeline-shaped writes**. Mix: `(from ... select ...)`.Take(5)`.
