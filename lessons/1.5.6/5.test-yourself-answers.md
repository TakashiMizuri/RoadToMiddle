# Answers: GC Roots, Reachability, Mark Phase

## Answers

### 1. [Recall]
**Answer:** Starting point for liveness — runtime scans these references as unconditionally live at GC start.
**Why:** Defines reachability graph entry.

### 2. [Recall]
**Answer:** Thread stacks, static fields, GC handles, finalizer queue (also interop refs).
**Why:** Common root sets.

### 3. [Understanding]
**Answer:** Live if reachable via reference chain from any root; dead if unreachable.
**Why:** Tracing GC definition.

### 4. [Understanding]
**Answer:** (1) Coordinate/suspend (2) scan roots mark objects (3) trace fields mark more (4) unmarked reclaimed/compact.
**Why:** Mark-sweep-compact core.

### 5. [Understanding]
**Answer:** **Collected** — no path from roots; cycle alone doesn't keep alive.
**Why:** Tracing not ref-count.

### 6. [Application]
**Answer:** **No** while in static list — static is root.
**Why:** Strong root until removed.

### 7. [Application]
**Answer:** Publisher holds delegate → target subscriber — subscriber kept alive while publisher alive.
**Why:** Delegates are strong refs to target.

### 8. [Application]
**Answer:** **Not collected** — `b` still roots object.
**Why:** One strong ref enough.

### 9. [Analysis]
**Answer:** Scope removes **local** root only; static, fields elsewhere, events may still root.
**Why:** Multiple root paths.

### 10. [Analysis]
**Answer:** Weak — optional cache, object collectable when memory pressure; strong — prevents GC until removed.
**Why:** Cache semantics.

### 11. [Application]
**Answer:** **Pinned handle** roots object AND prevents movement during GC.
**Why:** Interop stability.

### 12. [Understanding]
**Answer:** Unreachable finalizable object queued — **rooted until Finalize runs** — delayed reclaim.
**Why:** Finalization path.

### 13. [Application]
**Answer:** Shows **why object still alive** — path from root to instance.
**Why:** Leak diagnosis.

### 14. [Analysis]
**Answer:** Tracing collects cycles without roots; naive ref-count leaks cycles.
**Why:** .NET uses tracing.

### 15. [Understanding]
**Answer:** Old gen objects pointing to young must be scanned — card table records cross-gen stores.
**Why:** Partial Gen0 collection correctness.
