## Answers

### 1. [Recall]
**Answer:** **Untracked** files (and with `-d`, untracked **directories**) in the working tree.

**Why:** Clean only targets paths Git is not tracking.

---

### 2. [Recall]
**Answer:** **`-n`** or **`--dry-run`**.

---

### 3. [Recall]
**Answer:** Deletion is **irreversible** — Git requires explicit **force** to prevent accidental data loss.

---

### 4. [Understanding]
**Answer:** `Program.cs` is **tracked** — clean never removes tracked files, only untracked paths. Modified tracked content requires **`git restore`**.

---

### 5. [Understanding]
**Answer:** **`-fd`** respects `.gitignore` — ignored files/dirs **skipped**. **`-fdx`** also removes **ignored** files (e.g. `bin/`, `obj/`).

**Why:** `-x` disables ignore protection.

---

### 6. [Understanding]
**Answer:** Preview shows **exactly** what will be deleted — prevents accidental loss of important untracked work, `.env`, etc.

**Why:** No undo in Git for clean.

---

### 7. [Application]
**Answer:** **`-fd`** — `-d` includes untracked directories, `-f` forces deletion.

Example: `git clean -fd` (after `git clean -nd` preview).

---

### 8. [Application]
**Answer:**
```bash
git clean -ndx
git clean -fdx
```

**Why:** `bin/` and `obj/` are ignored — need `-x` to remove them.

---

### 9. [Application]
**Answer:** Use **`git stash push -u`** — preserves untracked files. **Do not** `git clean` if you need the files later.

---

### 10. [Analysis]
**Answer:** **B) `git restore file.txt`**

**Why:** A doesn't work on tracked modifications; C is overkill and wrong target.

---

### 11. [Analysis]
**Answer:** **`-d`** — without it, untracked **directories** are not removed (only loose untracked files at listed levels).

---

### 12. [Analysis]
**Answer:** **False.** Clean removes files from the **working directory (disk)** only. **Git history** is unchanged — untracked files were never in history.

---

### 13. [Understanding]
**Answer:** **`stash -u`** **saves** untracked files into stash stack (recoverable). **`clean -fd`** **deletes** untracked files permanently from disk.

---

### 14. [Application]
**Answer:**
1. `git status`
2. `git clean -nd` (preview)
3. Read output carefully
4. `git clean -fd` (or `-fdx` if intentional)

Accept 3-step if status implied: preview → read → execute.

---

### 15. [Analysis]
**Answer:** **`.env` NOT deleted** (ignored, no `-x`). **`scratch.log` IS deleted** (untracked, not ignored).

**Why:** Default clean skips ignored paths.

## Model answers for practical sections

### P1. Flag matching
| Purpose | Flag |
|---------|------|
| Preview only | **`-n`** |
| Untracked directories | **`-d`** |
| Include ignored files | **`-x`** |
| Actually delete | **`-f`** |

---

### P2. Choose the tool
1. Delete untracked `temp.sql` → **clean** (`git clean -fd`)
2. Undo unstaged edits to tracked `App.cs` → **restore**
3. Save untracked WIP before branch switch → **stash -u**
4. Remove tracked obsolete file from repo → **rm**
