# SemaphoreSlim для throttling — краткая выжимка

> `1.4.18` · ~6 мин на повторение

## Главная идея

**`SemaphoreSlim`** ограничивает **число одновременных** async операций (permits). **`WaitAsync`** не блокирует thread; **`Release`** в **`finally`** — обязателен.

## Ключевые пункты

- **`new SemaphoreSlim(N, N)`** — max N concurrent.
- Pattern: **`await WaitAsync(ct); try { work } finally { Release(); }`**
- **`Task.WhenAll` + semaphore** — many tasks, limited parallel execution.
- **`lock` + await** — anti-pattern; semaphore или redesign.
- Throttling (parallel in-flight) ≠ **rate limiting** (RPS) — **`RateLimiter`** .NET 7+.
- Singleton gate; **`N`** ≤ DB **`Max Pool Size`** / API limits.

## Пример

```csharp
await _gate.WaitAsync(ct);
try { await _http.GetAsync(url, ct); }
finally { _gate.Release(); }
```

## Частые ошибки

- Забытый **`Release`** → все ждут forever.
- Double **`Release`** → **`SemaphoreFullException`**.
