## Practical

### P1. Three-tree scenario
Начальное состояние: HEAD на commit C3; в `app.ts` есть **staged** изменение (+1 строка) и **дополнительное unstaged** изменение (+ещё 1 строка в том же файле).

Для каждой команды укажи: HEAD (C2/C3/другое), staged changes (есть/нет), unstaged в `app.ts` (есть/нет/потеряны):

1. `git restore app.ts`
2. `git restore --staged app.ts`
3. `git reset --soft HEAD~1` (до команды HEAD был на C3, parent = C2)
4. `git reset --hard HEAD~1`

---

### P2. Choose the tool
Для каждого сценария — **одна** лучшая команда: `restore`, `restore --staged`, `reset --soft`, `reset --mixed`, `reset --hard`, `revert`, `reflog`:

1. Discard unstaged edits in one file.
2. Undo last commit on unpushed branch; keep edits unstaged.
3. Undo published bad commit on `develop`.
4. Recover commit after accidental hard reset.
