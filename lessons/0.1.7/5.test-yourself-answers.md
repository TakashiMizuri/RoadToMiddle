## Answers

### 1. [Recall]
**Answer:** Metadata commit (author, date, message) + **patch** — unified diff изменений относительно parent commit.

**Why:** Default `git show` для commit object = commit info + diff.

---

### 2. [Recall]
**Answer:** Содержимое файла `src/app.ts` таким, каким оно было в commit `HEAD~3` (три commits назад от HEAD).

**Why:** Синтаксис `<rev>:<path>` — tree lookup без checkout.

---

### 3. [Understanding]
**Answer:** Для **последнего** commit результат очень похож (patch между parent и commit). `git show <hash>` работает для **любого** commit и включает metadata; `git diff A B` — только diff между двумя revisions без обязательного вывода message.

**Why:** `git show` = object inspector; `git diff` = comparison tool.

---

### 4. [Understanding]
**Answer:** Минимум: **commit hash**, **author name**, **timestamp**, **line number**, **line content**.

**Why:** Blame = annotate each line with last-modifying revision metadata.

---

### 5. [Understanding]
**Answer:** Ограничивает blame только строками 100–120 — быстрее на больших файлах, фокус на подозрительном блоке.

---

### 6. [Understanding]
**Answer:** `-C` и `-M` (часто вместе: `git blame -C -M`).

**Why:** Detect copy/move from other files or within file across history.

**Common mistake:** Только `-M` без `-C` при cross-file copy.

---

### 7. [Application]
**Answer:**
```bash
git blame -L 42,42 -- PaymentProcessor.cs
git show <hash-from-output>
```

---

### 8. [Application]
**Answer:** `git show --stat f4e8b21` или `git show --name-only f4e8b21`

---

### 9. [Application]
**Answer:**
```bash
git log -S "TODO: remove hack" --oneline -- src/
```
С patch: добавить `-p`.

---

### 10. [Analysis]
**Answer:**
1. Blame = **last modifier**, не necessarily тот, кто ввёл логическую ошибку (строка могла быть корректной, баг от другого изменения).
2. Строка могла быть **скопирована/перемещена** — нужны `-C -M` для истинного источника.

---

### 11. [Analysis]
**Answer:** Добавить path: `-- src/` или конкретный файл; сузить по времени: `--since=2026-01-01`; использовать более уникальную строку.

---

### 12. [Recall]
**Answer:** Игнорирует revisions, где изменился **только whitespace** — blame «проваливается» к содержательному изменению.

---

### 13. [Application]
**Answer:** `git show v1.0.0:README.md`

---

### 14. [Understanding]
**Answer:** **Не меняют.** Read-only queries к object database. Безопасны на любом clone.

---

### 15. [Analysis]
**Answer:** `git blame -w` (ignore whitespace commits) или blame на commit **до** formatting: `git blame <format-commit>^ -- file`.
