# Generations Gen0/1/2 и promotion

> Roadmap: `1.5.7` · Depth: **глубоко**

## Learning Objectives

- Объяснить **generational hypothesis**.
- Описать **Gen0**, **Gen1**, **Gen2** и **promotion**.
- Предсказать Gen0 vs full GC и pauses.
- Различить **Workstation** vs **Server GC** для ASP.NET Core.
- Связать allocation rate (`1.5.5`) с частотой Gen0.

---

## Why This Matters

Allocations из `1.5.5` попадают в **Gen0** (кроме LOH `1.5.8`). Request DTOs, JSON buffers умирают после request — generational GC собирает young cheaply.

Gen2/full GC — дороже. Static caches держат objects в Gen2 — pressure и pauses. Server apps default **Server GC** — heap per CPU.

---

## Core Concepts

### Generational Hypothesis

Большинство objects die young; old tend to live. SOH split:

| Gen | Contents | Collection |
|-----|----------|------------|
| 0 | New | Very often |
| 1 | Survivors | Less |
| 2 | Long-lived | Rare, expensive |

Survivors **promote** upward.

### Workstation vs Server GC

**Server GC** — default ASP.NET Core: per-CPU heaps, background GC, throughput. **Workstation** — client/interactive.

### Write Barriers

Old gen → young ref tracked via **card table** for partial Gen0 collections.

---

## Under the Hood

**Ephemeral segments** for Gen0/1. **Card table** + barriers. Finalizers may premature promotion.

---

## Key Takeaways

- Gen0 frequent, Gen2 expensive.
- Promotion Gen0→1→2.
- Server GC for web throughput.
- Long-lived roots defeat generational benefit.
