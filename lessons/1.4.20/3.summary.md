# Async all the way — краткая выжимка

> `1.4.20` · ~7 мин на повторение

## Главная идея

**Sync-over-async** (`.Result`, `.Wait()`, `GetAwaiter().GetResult()`) **блокирует thread pool thread** в ASP.NET Core → **starvation** под нагрузкой. **Async all the way** — `async Task` + `await` от Kestrel до DB/HTTP.

## Ключевые пункты

- ASP.NET Core: **нет** classic UI deadlock, но **pool exhaustion** реален.
- **Async-over-sync** (`Task.Run` для disk I/O) — code smell; native `*Async`.
- Refactor **cascade** — interface + все callers вместе.
- **`Task.Run`** не лечит I/O sync-over-async.
- Legacy sync SDK → **queue/background**, не `.Result` в action.
- Middleware: **`InvokeAsync`**, не `task.Wait()` в `Invoke`.

## Anti-pattern → fix

```csharp
// BAD
var x = GetAsync().Result;
// GOOD
var x = await GetAsync();
```

## Частые ошибки

- Async controller + sync EF `ToList()`.
- `Task.Run` как «универсальный async».
- `.GetAwaiter().GetResult()` «вместо Result».
