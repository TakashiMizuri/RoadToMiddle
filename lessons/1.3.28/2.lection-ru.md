# LINQ OrderBy, ThenBy — Stable Sort

> Roadmap: `1.3.28` · Node: `1.3` — C#: Advanced Topics · Depth: practice

## Learning Objectives

После урока ты сможешь:

- Сортировать через **`OrderBy`**, **`OrderByDescending`**, **`ThenBy`**, **`ThenByDescending`**.
- Объяснить **stable sort** и tie-breaking для pagination и UI.
- Использовать **`IComparer<T>`** и culture-aware comparers.
- Sort на **`IQueryable`** → SQL **`ORDER BY`**.
- Избегать client-side sort через **`AsEnumerable()`** trap.

---

## Why This Matters

Users ожидают consistent lists. **`OrderBy`** — primary key; **`ThenBy`** — ties («Department, then Name»). Без **`ThenBy`** equal keys → undefined order в SQL и unstable pages.

LINQ sort **deferred** на **`IEnumerable`**; на **`IQueryable`** — **`ORDER BY`** на server. **Stable sort** сохраняет input order among equal keys — важно для FIFO внутри same priority.

---

## Core Concepts

### OrderBy и ThenBy

```csharp
employees
    .OrderBy(e => e.Department)
    .ThenBy(e => e.LastName)
    .ThenByDescending(e => e.HireDate);
```

Returns **`IOrderedEnumerable<T>`**. **Wrong:** второй **`OrderBy`** **заменяет** первый — только **`ThenBy`**.

### Stable Sort

LINQ to Objects — **stable** (merge sort based). Equal keys сохраняют relative order из input.

Explicit tie-break: **`ThenBy(e => e.Id)`** или original index.

### Custom Comparer

```csharp
.OrderBy(s => s, StringComparer.OrdinalIgnoreCase)
```

### EF Core

```csharp
db.Products.OrderBy(p => p.CategoryId).ThenBy(p => p.Name);
```

Sort **before** **`Skip/Take`**. **`AsEnumerable().OrderBy(...)`** — client pull all rows.

---

## Examples

### Leaderboard + tie

```csharp
scores.OrderByDescending(s => s.Points).ThenBy(s => s.SubmittedAt).Take(10);
```

### Stable pagination

```csharp
db.Items.OrderBy(i => i.SortOrder).ThenBy(i => i.Id).Skip(...).Take(...);
```

---

## Common Mistakes

Second **`OrderBy`** вместо **`ThenBy`**. Pagination без unique tie-break. Early **`AsEnumerable()`**. Nullable sort order surprises.

---

## Quick Reference

| Operator | Role |
|----------|------|
| `OrderBy` | primary ↑ |
| `ThenBy` | secondary ↑ |
| `*Descending` | ↓ на этом level |

Stable + **`ThenBy(Id)`** для deterministic paging.

---

## Key Takeaways

- **`ThenBy`** extends; **`OrderBy`** replaces.
- Stable sort among equal keys.
- Tie-break для pagination.
- EF: server-side ORDER BY.
