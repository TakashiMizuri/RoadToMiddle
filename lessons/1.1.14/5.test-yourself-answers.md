# Answers: Массивы

## Answers

### 1. [Recall]
**Answer:** **Reference type** — переменная хранит ссылку на объект массива в heap.
**Why:** Массивы — классы (`System.Array`).

### 2. [Recall]
**Answer:** **12** (3 × 4 = total element count).
**Why:** `Length` на multidimensional — произведение размеров.

### 3. [Recall]
**Answer:** **`grid.GetLength(1)`** (dimension 1 = columns при convention row=0, col=1).
**Why:** `Length` alone не даёт одно измерение.

### 4. [Understanding]
**Answer:** `int[,]` — **один** contiguous block; `int[][]` — outer array references + **отдельные** row arrays.
**Why:** Jagged не гарантирует прямоугольник в памяти.

### 5. [Understanding]
**Answer:** **`j[0]`** ещё **null** — не выделен inner array → **NullReferenceException**.
**Why:** `new int[5][]` только outer shell.

### 6. [Understanding]
**Answer:** **Нет** (без helper/`GetEnumerator` hack) — `int[,]` не implements простой single foreach over all cells in older patterns; нужны nested loops.
**Why:** Multidimensional enumerator exists but not same as 1D foreach ergonomics.

### 7. [Application]
**Answer:**
```csharp
int[][] triangle = {
    new[] { 1 },
    new[] { 2, 3 },
    new[] { 4, 5, 6 }
};
```
**Why:** Each row is separate array object.

### 8. [Application]
**Answer:**
```csharp
for (int r = 0; r < visited.GetLength(0); r++)
    for (int c = 0; c < visited.GetLength(1); c++)
        visited[r, c] = false;
```
**Why:** Must use comma indexer and per-dimension length.

### 9. [Analysis]
**Answer:** **Compile error** — `int[,]` не поддерживает `[][]` indexer.
**Common mistake:** Confusing jagged vs rectangular syntax.

### 10. [Analysis]
**Answer:** **Array covariance** — compile-time `object[]` ok; runtime slot is `string`, `42` invalid → **InvalidCastException**.
**Why:** Writes must match actual element type.

### 11. [Application]
**Answer:** Когда **число «столбцов» (tokens) в каждой строке разное** — rectangular потребовал бы padding или wasted cells.
**Why:** Jagged моделирует ragged rows напрямую.

### 12. [Understanding]
**Answer:** Singleton **immutable empty array** shared across app — avoids allocating new empty arrays.
**Why:** Micro-optimization + reference equality for empty.

### 13. [Analysis]
**Answer:** **`IndexOutOfRangeException`** when `i == arr.Length` (valid indices `0..Length-1`).
**Common mistake:** `<=` instead of `<`.

### 14. [Application]
**Answer:** **`Array.Copy(source, dest, source.Length)`** or `source.CopyTo(dest, 0)` (with length checks).
**Why:** BCL bulk copy.

### 15. [Understanding]
**Answer:** First index **0**, last **`n - 1`**.
**Why:** Zero-based C# convention.
