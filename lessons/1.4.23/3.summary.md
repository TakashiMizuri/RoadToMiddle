# Awaitable pattern — краткая выжимка

> `1.4.23` · ~10 мин на повторение

## Главная идея

`await` — не магия `Task`, а **протокол**: `GetAwaiter()` → `IsCompleted` → `GetResult()` / `OnCompleted`. Компилятор генерирует state machine с этими вызовами.

## Ключевые пункты

- **GetAwaiter()** — extension или instance; возвращает awaiter.
- **IsCompleted** — fast path без continuation.
- **GetResult()** — результат или exception (await marshaling).
- **INotifyCompletion.OnCompleted** — с ExecutionContext.
- **ICriticalNotifyCompletion.UnsafeOnCompleted** — без context, быстрее.
- **ConfigureAwait(false)** — не возвращаться на captured SyncContext.
- **ValueTask** — тот же pattern; осторожно с double-await.
- Legacy API → **TaskCompletionSource** bridge.

## Самые важные примеры

```csharp
var awaiter = task.GetAwaiter();
if (!awaiter.IsCompleted)
    awaiter.OnCompleted(() => { /* MoveNext */ });
var result = awaiter.GetResult();
```

```csharp
await task.ConfigureAwait(false); // library intent
```

## «Под капотом» в одном абзаце

Roslyn создаёт `IAsyncStateMachine` с полями awaiter и `__state`. Race между incomplete check и completion решается внутри awaiter (exactly-once continuation). Faulted exceptions через `ExceptionDispatchInfo` в `GetResult()`.

## Частые ошибки

- «Awaitable = только Task».
- `GetResult()` без проверки completion.
- Double-await на one-shot **ValueTask**.

## Мнемоника

**GIGO: GetAwaiter → IsCompleted → GetResult / OnCompleted**
