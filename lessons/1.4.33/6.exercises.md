# Exercises: SemaphoreSlim vs lock

> Complete after passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Compile-Time Errors
**Goal:** List 3 illegal patterns mixing lock/Monitor and await.
**Success criteria:** Includes CS1996 example.

### E2. Mutex Template
**Goal:** Write `AsyncLock` wrapper class with `WaitAsync`/`Release` IDisposable pattern.
**Success criteria:** `finally` or `await using` releases.

## Medium

### M1. Async Cache
**Goal:** Implement `GetOrLoadAsync(key)` with global `SemaphoreSlim(1,1)` or per-key — justify choice.
**Success criteria:** Double-check after acquire; passes `CancellationToken`.

### M2. Throttle HTTP
**Goal:** Wrap 20 parallel `HttpClient` calls with `SemaphoreSlim(5,5)`.
**Success criteria:** Max 5 concurrent; all complete.

## Hard (stretch)

### H1. Per-Key Semaphores
**Goal:** `ConcurrentDictionary<string, SemaphoreSlim>` with lazy create and cleanup policy sketch.
**Success criteria:** Document leak if never removed.
