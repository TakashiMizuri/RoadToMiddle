# Exercises: HTML Lists

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Navigation list
**Goal:** Correct nav + ul + li.

In `site.html` (minimal skeleton), add `<header>` with `<nav aria-label="Primary">` containing links: Home (`/`), Blog (`/blog`), About (`/about`).

**Success criteria:** ul > li > a structure; no bare anchors under ul.

---

### E2. Shopping list
**Goal:** Simple unordered list.

Inside `<main>`, add `<h1>Groceries</h1>` and `<ul>` with five items (any food names).

**Success criteria:** Five li elements; no ol.

---

## Medium

### M1. Deployment steps
**Goal:** Ordered list with nesting.

Create `deploy.html` with `<main>`, h1 "Deploy Checklist", and `<ol>` with:

1. Run tests
2. Build artifacts (nested `<ul>` with two sub-tasks)
3. Publish

**Success criteria:** Nested ul inside li #2 only; valid nesting.

---

### M2. Glossary page
**Goal:** Description list.

Create `glossary.html` with at least four term/definition pairs about HTML (e.g. DOCTYPE, charset, main, nav). Use `<dl>`.

**Success criteria:** Each term has dt + dd; no ul/ol for pairs.

---

### M3. Fix broken nesting
**Goal:** Analysis + repair.

Broken:

```html
<ul>
  <li>Backend</li>
  <ul>
    <li>ASP.NET Core</li>
    <li>PostgreSQL</li>
  </ul>
  <li>Frontend</li>
</ul>
```

Fix and explain the rule in one sentence.

**Success criteria:** Inner ul inside first li; valid tree.

---

## Hard (stretch)

### H1. Choose list types
**Goal:** Semantic judgment.

For each scenario, write **ul**, **ol**, **dl**, or **none**:

1. Recipe instructions
2. Site footer legal links
3. API field docs: "id — unique identifier"
4. Single sentence paragraph

**Success criteria:** 1 ol, 2 ul, 3 dl, 4 none with brief why.
