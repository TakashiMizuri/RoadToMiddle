## Answers

### 1. [Recall]
**Answer:** Address treated locally by host stack; **127.0.0.0/8** reserved for loopback — packets loop internally, not routed to LAN/internet.
**Why:** Special internet-layer range (RFC 1122).

### 2. [Recall]
**Answer:** **No** — `0.0.0.0` means «any interface» for **bind**, not valid unicast destination for connect.
**Why:** OS routing/connect semantics.

### 3. [Understanding]
**Answer:** **127.0.0.1** — only same-machine clients. **0.0.0.0** — listen on **all IPv4 interfaces** (LAN, loopback, etc.).
**Why:** Interface selection on listen socket.

### 4. [Understanding]
**Answer:** **`localhost`** hostname resolves (hosts/DNS) to **127.0.0.1** and/or **::1** — ping targets loopback.
**Why:** RFC 6761 special name.

### 5. [Understanding]
**Answer:** **`bind()`** — which local IP:port accepts inbound; **`connect()`** — which remote IP:port to reach.
**Why:** Server vs client socket roles (`0.4.2`).

### 6. [Application]
**Answer:** (1) Configure Kestrel **`ListenAnyIP(5000)`** or bind LAN interface. (2) Allow **firewall inbound** on 5000 if blocked.
**Why:** localhost bind doesn't listen on 192.168.x interface.

### 7. [Application]
**Answer:** **`http://localhost:8080`** or `http://127.0.0.1:8080` (same machine).
**Why:** 0.0.0.0 is listen wildcard, not browse target.

### 8. [Analysis]
**Answer:** **C** неверно — `localhost` may resolve to **::1** (IPv6) as well as 127.0.0.1.
**Why:** A, B, D correct.

### 9. [Analysis]
**Answer:** **nginx** public on **443 all interfaces**; **Kestrel** only local behind proxy — reduces direct exposure, only reverse proxy talks to app.
**Why:** Defense in depth on same host.

### 10. [Understanding]
**Answer:** Resolver returns **::1 first**; server bound **IPv4-only 127.0.0.1** → connection to ::1 fails. Fix dual-stack bind or use 127.0.0.1 explicitly.
**Why:** Happy Eyeballs / resolution order.

### 11. [Recall]
**Answer:** **`::1`**
**Why:** IPv6 loopback.

### 12. [Application]
**Answer:** **Different origins** — scheme/host/port; ports 5173 vs 5000 differ → CORS may apply (`0.4.10`).
**Why:** Same host localhost but different port.

### 13. [Understanding]
**Answer:** **Overrides** public DNS for `api.local` at **stub/hosts** level — static mapping before recursive resolver (`0.4.5`).
**Why:** Local dev DNS bypass.
