# Exercises: .NET SDK and Project Structure

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Install verification
**Goal:** Confirm your toolchain is ready.

Run `dotnet --info` and record:
1. Highest SDK version listed
2. At least one runtime version
3. Your OS in the output

**Success criteria:** All three recorded; SDK ≥ 8.0 (or team-agreed version).

---

### E2. Scaffold and inspect
**Goal:** Map CLI output to on-disk structure.

```bash
dotnet new console -n SdkLab -o SdkLab
```

Without running the app yet, list every **file** (not `bin/`/`obj/`) in `SdkLab/`. Open `SdkLab.csproj` and identify: `OutputType`, `TargetFramework`, `ImplicitUsings`, `Nullable`.

**Success criteria:** Four properties named with correct values from your template.

---

## Medium

### M1. Args and Release build
**Goal:** Wire `Main` args and configurations.

In `SdkLab/Program.cs`, print: `Args count: {args.Length}` and each argument on its own line.

Run:
```bash
dotnet run --project SdkLab -- alpha beta
dotnet build SdkLab -c Release
```

Confirm Release DLL exists under `SdkLab/bin/Release/<TFM>/`.

**Success criteria:** Output shows count 2 and lines `alpha`, `beta`; Release folder contains `.dll`.

---

### M2. Add a package
**Goal:** Practice NuGet via CLI.

```bash
dotnet add SdkLab package System.Text.Json
```

Rebuild. In `Program.cs`, serialize a small anonymous object `{ "status": "ok" }` to JSON string and print it (use `System.Text.Json.JsonSerializer`).

**Success criteria:** Build succeeds; console prints valid JSON; `.csproj` contains `PackageReference` for `System.Text.Json`.

---

## Hard (stretch)

### H1. global.json experiment
**Goal:** Understand SDK pinning.

In a **throwaway copy** of `SdkLab`, from parent folder:

```bash
dotnet new globaljson --sdk-version <your exact SDK patch version>
dotnet --version
```

Document whether `dotnet build` still works. Then change `rollForward` to `"disable"` in `global.json` with a **non-installed** patch version — observe the error message.

**Success criteria:** Short write-up (5–8 sentences): what `global.json` enforced and what error appears when SDK is missing.
