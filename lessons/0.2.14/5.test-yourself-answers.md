## Answers

### 1. [Recall]
**Answer:** LF — один byte `\n` (0x0A); CRLF — два bytes `\r\n` (0x0D 0x0A).
**Why:** Git diff сравнивает raw bytes.

### 2. [Recall]
**Answer:** Файлы `.png` treated as **binary** — Git **не** конвертирует line endings и не портит content.
**Why:** `binary` отключает text normalization.

### 3. [Understanding]
**Answer:** Если endings разлились (CRLF vs LF), **каждая строка** другая на byte level.
**Why:** Не logical change, но valid diff.

### 4. [Understanding]
**Answer:** `.gitattributes` **versioned** — все clones одинаковые rules; per-user autocrlf различается Windows/Linux.
**Why:** Repo-level source of truth.

### 5. [Understanding]
**Answer:** **Normalized LF** line endings в blob (canonical storage).
**Why:** Clean filter on commit.

### 6. [Application]
**Answer:** `git add --renormalize .` (или path).
**Why:** Re-run filters per current attributes.

### 7. [Application]
**Answer:** `* text=auto eol=lf` (часто + explicit binary patterns отдельно).
**Why:** Auto-detect text, LF canonical.

### 8. [Application]
**Answer:** `*.sh text eol=lf` (или `* text eol=lf` если global LF).
**Why:** Force LF для scripts на Windows checkout.

### 9. [Analysis]
**Answer:** Windows commits CRLF blobs; Linux devs/CI видят inconsistent endings, noisy diffs, script failures.
**Why:** No shared repo policy.

### 10. [Analysis]
**Answer:** **Не баг** — ожидаемо при первой normalization; один commit fixes forward.
**Why:** Index blobs обновляются к LF.

### 11. [Understanding]
**Answer:** EditorConfig — **editor on save**; `.gitattributes` — **Git add/checkout**. Оба нужны для alignment.
**Why:** Git не читает EditorConfig сам.

### 12. [Application]
**Answer:** `git ls-files --eol`.
**Why:** Shows index/working tree EOL attributes.

### 13. [Recall]
**Answer:** **`false`** (documented for team; sometimes `input` on Windows).
**Why:** Avoid double conversion with `eol=lf`.

### 14. [Analysis]
**Answer:** DLL — binary; EOL conversion **corrupts** or misclassifies; diff useless.
**Why:** Not text.

### 15. [Understanding]
**Answer:** **Clean** on **add/commit** (WT→index/LF); **smudge** on **checkout** (blob→WT per eol).
**Why:** Filter pipeline under the hood.
