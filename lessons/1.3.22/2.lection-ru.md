# LINQ SelectMany — flatten nested collections

> Roadmap: `1.3.22` · Node: `1.3` — C#: продвинутые темы · Depth: **глубоко**

## Learning Objectives

- Объяснить **`SelectMany`** как **map + flatten** (monadic bind).
- Использовать overloads с one/two sequences.
- Завершать **left join**: **`GroupJoin` + SelectMany + DefaultIfEmpty`**.
- Отличать **`Select`** (nested **`IEnumerable`**) от **`SelectMany`** (flat stream).
- Понимать translate EF (**`CROSS APPLY`**) и риск cartesian explosion.

---

## Why This Matters

**`Order.Lines`**, **`Post.Tags`**, paginated batches — нужен flat stream для Sum/Where/export. **`SelectMany`** заменяет nested **`foreach`**. Вторая половина **left join** после **`GroupJoin`**. Путаница с **`Select`** → **`IEnumerable<IEnumerable<T>>`** и double **`foreach`**.

---

## Core Concepts

**`Select`**: `orders.Select(o => o.Lines)` → nested.

**`SelectMany`**: `orders.SelectMany(o => o.Lines)` → flat **`Line`**.

Two-arg overload:

```csharp
orders.SelectMany(o => o.Lines, (o, line) => new { o.Id, line.Qty });
```

Left join flatten:

```csharp
.GroupJoin(..., (c,g) => new { c, g })
.SelectMany(x => x.g.DefaultIfEmpty(), (x, o) => new { x.c, o });
```

**`null`** inner sequence → **`ArgumentNullException`** — **`?? Enumerable.Empty<T>()`**.

Query: **`from o in orders from line in o.Lines`** desugars to **`SelectMany`**.

---

## Examples

### Все line items по orders

```csharp
var allLines = orders.SelectMany(o => o.Lines);
var totalQty = allLines.Sum(l => l.Qty);
```

Pipeline lazy до **`Sum`** — lines не собираются в mega-list, если source deferred.

### Tags с post id

```csharp
var tagRows = posts.SelectMany(
    p => p.Tags,
    (p, tag) => new TagRow(p.Id, tag));
```

Two-arg overload избегает промежуточного nested **`Select`**.

### Paginated API

```csharp
IEnumerable<Item> AllItems(IEnumerable<Page<Item>> pages) =>
    pages.SelectMany(p => p.Items);
```

Если **`pages`** сам **`yield`**'ится — items stream без materialize всех pages.

## Under the Hood

**`Enumerable.SelectMany`** — outer foreach, inner foreach, yield each element. Время O(число output elements), память O(1) сверх enumerators.

Overload **`SelectMany(a => bs, (a,b)=>...)`** pair'ит **каждый** outer **a** с **каждым** **b** из одной inner sequence **bs** — O(n×m) cross product, **не** key-based **`Join`**. Ошибка intent — частый источник row explosion.

EF translate **`SelectMany`** over navigation в **`CROSS APPLY`** или join expansion; цепочки без projection review → SQL cartesian.

---

## Common Mistakes & Anti-patterns

**`Select` вместо flatten** — API возвращает **`IEnumerable<IEnumerable<T>>`**, UI делает double **`foreach`**.

**Null inner sequence** — **`ArgumentNullException`**; guard **`?? Enumerable.Empty<T>()`**.

**Cross product вместо `Join`** — когда нужен key match, не **`SelectMany(a => allBs)`**.

**Multiple `SelectMany` на EF graph** без SQL review — runaway row count.

---

## Production & Real-World Notes

Report: **`SelectMany`** to lines + **`Where`/`Sum`** — держи **`IQueryable`** пока translatable. Optional 0..1 navigation — **`SelectMany`** с empty fallback или explicit left join, что читабельнее для команды.

Tests: flat count = sum inner counts; cases empty outer, empty inner, null guard.

---

## Key Takeaways

- **`SelectMany`** = concat projected sequences.
- **`from-from`** = **`SelectMany`**.
- Left join flat — с **`DefaultIfEmpty`**.
- Guard null; **`Join`** для key pairing.
