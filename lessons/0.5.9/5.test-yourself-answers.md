## Answers

### 1. [Recall]
**Answer:** `<thead>`, `<tbody>`, `<tfoot>` (любые три из этих секций строк).
**Why:** Секции группируют header rows, data rows и footer/summary rows для браузера и AT.

### 2. [Recall]
**Answer:** `<caption>`.
**Why:** Caption — programmatic name таблицы в accessibility tree.

### 3. [Recall]
**Answer:** `<th>` — header cell (column/row header); `<td>` — data cell.
**Why:** AT map headers к data cells через scope/headers; `<td>` не объявляется как заголовок.

### 4. [Understanding]
**Answer:** AT объявляет табличную структуру без реальных двумерных данных; навигация по «ячейкам» бессмысленна; responsive хуже.
**Why:** Семантика table promises matrix relationships.
**Common mistake:** «Выглядит как колонки — значит table».

### 5. [Understanding]
**Answer:** `scope="col"` на заголовках в header row (названия столбцов); `scope="row"` на заголовках первого столбца (метки строк).
**Why:** Scope указывает направление, в котором header label applies.

### 6. [Understanding]
**Answer:** Caption привязан к **table** в AT; heading — отдельный landmark. При входе в таблицу пользователь слышит caption как контекст сетки, не перечитывая весь документ.
**Why:** WCAG 1.3.1 — relationships inside table; caption is table-specific label.

### 7. [Application]
**Answer:** Первая строка в `<thead>`: `<th scope="col">Month</th>`, `<th scope="col">Revenue</th>`. January — `<th scope="row">January</th>` (если row label) или `<td>` (если просто data в первом столбце без row header semantics — для Month column лучше row header для месяца).
**Why:** Header row = col scope; first column month names often row headers.

### 8. [Application]
**Answer:** `<th scope="row">Total</th>` в первой ячейке строки; сумма в `<td colspan="3">...</td>`.
**Why:** Total labels the row; colspan merges data cells only.

### 9. [Analysis]
**Answer:** Нет `<thead>`/`<tbody>` (менее критично, но плохо); заголовки в `<td>` с `<b>`, не `<th>`; нет `<caption>`; нет `scope`.
**Why:** Bold td не создаёт columnheader role.

### 10. [Analysis]
**Answer:** При многоуровневых заголовках, colspan/rowspan, когда одна ячейка данных относится к нескольким headers не по простой оси col/row.
**Why:** headers/id — explicit association list.

### 11. [Understanding]
**Answer:** display:table строит layout box, но не table accessibility roles и header/data mapping из `<th>`.
**Why:** Native table elements feed platform a11y APIs.

### 12. [Application]
**Answer:** **Grid** (или Flexbox) — нет двумерной связи «строка × столбец», только список карточек.
**Why:** Table semantics без matrix misleads AT.

### 13. [Recall]
**Answer:** Нет, ровно один `<caption>` на `<table>`.
**Why:** HTML spec — one caption element.

### 14. [Understanding]
**Answer:** Объявляет связанные headers (February, возможно row header) перед или вместе со значением 135 — не голое число.
**Why:** scope lets AT compute header chain for cell.
