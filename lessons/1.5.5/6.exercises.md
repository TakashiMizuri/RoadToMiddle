# Exercises: Heap Allocation

## Easy
### E1. Minimum object size
**Goal:** Observe overhead.
**Steps:** Allocate `new object()` in loop 1M times (careful in debug); note memory vs 1M × expected zero payload — conceptual or use profiler.
**Success criteria:** Mention header overhead.

## Medium
### M1. Boxing hunt
**Goal:** Find boxing in code review.
**Steps:** Given snippet with `ArrayList` storing ints or `object sum = 0; sum += i` — list allocations.
**Success criteria:** Identify box per add.

## Medium
### M2. Field reorder
**Goal:** Reduce padding.
**Steps:** Design struct/class fields `byte, long, byte` vs reorder `long, byte, byte` — estimate size difference.
**Success criteria:** Explain alignment padding.
