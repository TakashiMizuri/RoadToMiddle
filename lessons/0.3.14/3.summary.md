# Keep-Alive и connection pooling — краткая выжимка

> `0.3.14` · ~7 мин на повторение

## Главная идея

**Keep-alive** переиспользует TCP между HTTP requests; **connection pooling** держит готовые sockets на client (и upstream у proxy). Без reuse платишь DNS + TCP + TLS на каждый call — под нагрузкой latency и **socket exhaustion**.

## Ключевые пункты

- HTTP/1.1: persistent connections **by default**; ~6 parallel conn/host.
- HTTP/2: many streams на **одном TCP** (`0.3.13`).
- **`.NET`:** `IHttpClientFactory` + `SocketsHttpHandler`; не `new HttpClient()` в цикле.
- **`PooledConnectionLifetime`** — rotation для DNS после deploy.
- **`MaxConnectionsPerServer`** — cap на host.
- Server: Kestrel `KeepAliveTimeout`, nginx `keepalive` к upstream.
- **Stale conn:** NAT/LB idle → reset; retry на fresh socket.

## Самые важные примеры

```csharp
builder.Services.AddHttpClient("external-api");
// DI inject IHttpClientFactory.CreateClient("external-api")
```

## Частые ошибки

- Новый `HttpClient` на каждый request.
- Client pool без proxy upstream keepalive.
- Путать HTTP pool wait с PostgreSQL pool wait.

## Мнемоника

> **Connect once, request many — pool is a parking lot for sockets.**
