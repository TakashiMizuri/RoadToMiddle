# Exercises: Form validation attributes

> Complete after `0.5.7` markup lesson and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Pick the attribute
**Goal:** Match constraint to attribute.

| Requirement | Attribute(s) |
|---------------|--------------|
| ZIP code exactly 5 digits | ? |
| Meeting not before 2026-07-04 | ? |
| Comment optional, max 280 chars | ? |
| Must accept terms checkbox | ? |

**Success criteria:** Four correct attribute choices; no wrong min vs minlength.

---

### E2. Fix broken validation
**Goal:** Analysis + correction.

```html
<form action="/signup" method="post">
  <label for="u">Username (3-16 chars)</label>
  <input id="u" name="username" type="text" min="3" max="16" required />
  <label for="e">Email</label>
  <input id="e" name="email" type="text" required />
  <button type="submit">Sign up</button>
</form>
```

Fix validation attributes and type where wrong.

**Success criteria:** minlength/maxlength on username; `type="email"` on email.

## Medium

### M1. Event registration form
**Goal:** Combined constraints in semantic page.

In `<main>`, build form POST `/events/register`:

- Full name: required, minlength 2
- Email: required, type email
- Tickets: number 1–10
- Event date: date with reasonable min (document choice in comment)
- T-shirt size: select required (S/M/L)
- Code of conduct: checkbox required

**Success criteria:** All labels from `0.5.7`; constraints match field types; submit button typed.

---

### M2. Pattern with title
**Goal:** Regex UX.

Add product SKU field: format `ABC-1234` (three uppercase letters, hyphen, four digits). Use `pattern` and helpful `title`. Required.

Test mentally: would `ABC-12345` pass? `abc-1234`?

**Success criteria:** Correct regex; title explains format; short written answers for two test values.

## Hard (stretch)

### H1. Draft vs publish
**Goal:** novalidate vs formnovalidate.

Extend M1 form with two submit buttons: **Publish registration** (full validation) and **Save draft** (skip required checks). Explain difference between putting `novalidate` on entire form vs `formnovalidate` on one button.

**Success criteria:** Only draft skips validation; publish still validates; 3–4 sentence comparison.

---

### H2. Server mirror sketch
**Goal:** Connect HTML to ASP.NET.

For M1 fields, list matching C# DataAnnotations attributes (names only: `[Required]`, `[Range]`, etc.) for each field. One paragraph: why both HTML and server layers stay.

**Success criteria:** Annotations align with HTML rules; security argument present.
