## Answers

### 1. [Recall]
**Answer:** Нет — `async` constructors не существуют в C#.
**Why:** Конструктор не возвращает `Task`.

### 2. [Recall]
**Answer:** Нет — compile error, async property accessors запрещены.
**Why:** `get` должен синхронно вернуть `T`.

### 3. [Understanding]
**Answer:** После `new` вызывающий код ожидает полностью созданный объект; `newobj`+ctor выполняются атомарно на calling thread; async прервал бы инвариант готовности.
**Why:** Фундамент object construction в CLR.

### 4. [Understanding]
**Answer:** Ctor возвращается до завершения warm; объект half-initialized; исключения unobserved; гонки между callers.
**Why:** Fire-and-forget ломает lifecycle.

### 5. [Understanding]
**Answer:** Каждый resolve блокирует thread pool thread на I/O; под нагрузкой starvation; плюс риск sync-over-async deadlock в библиотеках.
**Why:** Scoped/transient multiply effect.

### 6. [Application]
**Answer:** `static CreateAsync` factory + `await` при регистрации singleton; или `IHostedService` загружает config в cache, ctor только принимает `IConfigCache`.
**Why:** Async на composition root, sync ctor.

### 7. [Application]
**Answer:** `Task<string> GetUserNameAsync()` или preload в init + sync `string UserName { get; }`.
**Why:** Явный async boundary.

### 8. [Analysis]
**Answer:** (1) Sync-over-async block (2) Deadlock risk (3) Скрытый I/O в property (4) плохо в циклах/N+1 blocking.
**Why:** Тройная interview/classic anti-pattern.

### 9. [Analysis]
**Answer:** Flaky null/empty/stale data; один видит cache, другой нет; intermittent 500.
**Why:** Race до завершения WarmCacheAsync.

### 10. [Understanding]
**Answer:** Composition root при startup — после `Build()`/`app` создан, до `Run`/`MapEndpoints`, явный loop или hosted service orchestration.
**Why:** DI сам не вызывает InitializeAsync.

### 11. [Application]
**Answer:** Да, при build host для singleton с дорогой one-time async setup; не для scoped per-request.
**Why:** Единственный await на root допустим.

### 12. [Analysis]
**Answer:** `readonly` присваивается только в ctor; continuation после await — уже после ctor; без init-only/backing field pattern нельзя легально.
**Why:** Language rule on readonly.
