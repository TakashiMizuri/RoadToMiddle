# Headings, p, strong, em, mark — краткая выжимка

> `0.5.3` · ~8 мин на повторение

## Главная идея

**h1–h6** задают outline документа для людей и AT — не используй уровни ради размера шрифта. **p** — абзац; **strong** — важность; **em** — ударение; **mark** — подсветка релевантного (поиск, контекст).

## Ключевые пункты

- Один явный **h1** на view в main/article; h2 → h3 без пропусков.
- **`<p>`** вместо `<br><br>`; не вкладывать block в p.
- **strong** ≠ bold; **em** ≠ italic — смысл, не декор.
- **mark** — reference highlight, не общий дизайн.
- AT navigates headings по DOM order, не CSS order.
- `<b>`/`<i>` — только stylistic offset без semantics.

## Самые важные примеры

```html
<h1>FAQ</h1>
<h2>Shipping</h2>
<p>We <em>must</em> ship with <strong>tracking</strong>.</p>
<p>Configure <mark>UTF-8</mark> in headers.</p>
```

Плохо: `h1` → `h4`. Хорошо: `h1` → `h2` + CSS.

## Частые ошибки

- Заголовки для font-size.
- Пустой `<p>` для отступа.
- strong на каждом ключевом слове для SEO.

## Мнемоника

> **strong = serious, em = emphasis in voice, mark = match highlight.**
