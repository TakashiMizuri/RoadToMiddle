# Перегрузка операторов — краткая выжимка

> `1.2.17` · ~7 мин на повторение

## Главная идея

Перегрузка операторов — **public static** методы с именами `operator +`, `operator ==` и т.д.; компилятор подставляет их в выражения. Это sugar для **очевидных** операций над value types; бизнес-логику лучше выражать **именованными методами**.

## Ключевые пункты

- Перегружаются не все операторы: **нельзя** `=`, `.`, `?:`, произвольные имена.
- Частые: `+`, `-`, `==`, `!=`; conversion — `implicit`/`explicit`.
- **`==`/`!=` на class** — согласуй с `Equals`/`GetHashCode` (`1.2.16`).
- Value objects: операторы возвращают **новый** экземпляр (immutable).
- **Records** генерируют `==`/`!=` по свойствам.
- Плохо: `db + migration`; хорошо: `db.Apply(migration)`.
- Runtime = обычный static call; выбор по читаемости, не по speed.

## Самые важные примеры

```csharp
public static Vector2 operator +(Vector2 a, Vector2 b) =>
    new(a.X + b.X, a.Y + b.Y);

public static bool operator ==(Money? l, Money? r) =>
    l is null ? r is null : l.Equals(r);
```

## Частые ошибки

- `==` без пары `Equals` + `GetHashCode`.
- Мутация операндов внутри operator.
- Перегрузка для «красивого» DSL с неочевидным смыслом.
- Implicit conversion, теряющий валюту/контекст.

## Мнемоника

**Numbers and vectors → operators. Services and workflows → methods. Equality → same contract as 1.2.16.**
