# Exercises: OSI vs TCP/IP Model

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Layer assignment
**Goal:** Map symptoms to TCP/IP layers.

For each scenario, write the **most likely layer** (Application, Transport, Internet, Link):

1. Browser: `Could not resolve host: api.contoso.com`
2. `curl`: `Connection timed out` after 30s (no HTTP headers received)
3. HTTP `200 OK` with wrong JSON shape from your API
4. Laptop cannot reach any site; Wi‑Fi icon shows connected but `ping 8.8.8.8` fails from café (others OK)

**Success criteria:** Four layers named with one-sentence justification each.

## Medium

### M1. HTTPS stack trace on paper
**Goal:** Connect node `0.3` to node `0.4`.

Draw a vertical stack for `GET https://localhost:7150/weather` (ASP.NET dev cert). Label at least five steps from DNS/hosts file through TLS to HTTP. Note which steps are skipped or special for localhost (`0.4.6` preview).

**Success criteria:** Ordered steps; TLS after TCP; HTTP after TLS.

---

### M2. Tool output classification
**Goal:** Read layered clues in CLI output.

Run (or imagine output of) `curl -v http://example.com` and `tracert example.com`. List **three lines** from each command and label their layer.

**Success criteria:** Six lines labeled; no HTTP status from tracert classified as application HTTP.

## Hard (stretch)

### H1. L4 vs L7 balancer decision
**Goal:** Production trade-off thinking.

Your team deploys a new microservice `/reports` alongside `/api`. Compare routing with:

- AWS NLB (Layer 4) + separate target groups per port
- ALB (Layer 7) path-based routing on one listener :443

Write 5 bullets: latency, TLS termination point, canary by path, operational complexity, what breaks if Host header wrong.

**Success criteria:** At least one pro/con per approach grounded in layer behavior.
