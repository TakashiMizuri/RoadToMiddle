# Static members, static classes, static constructors

> Roadmap: `1.2.12` · Node: `1.2` — C# ООП · Depth: понимание

## Learning Objectives

После этой лекции ты сможешь:

- Объяснить, что означает **`static`** на уровне **типа** vs **экземпляра**.
- Корректно использовать **static fields**, **static properties**, **static methods** — и знать, когда они вредят тестируемости.
- Описать **static classes** (`static class`) и их ограничения.
- Понимать **static constructors** (type initializers): когда выполняются, thread safety, правила порядка.
- Различать **static** utility code и **singleton services** в ASP.NET Core DI.
- Избегать типичных ловушек: mutable static state, скрытые зависимости, static в generic types.

---

## Why This Matters

После `1.2.1` ты знаешь, что объекты живут в heap с reference semantics; после `1.2.11` композируешь instance services через DI. **`static`** — другая ось: члены принадлежат **самому типу**, а не конкретному объекту. Ровно один слот хранения для static field на **app domain** (на загруженный тип), сколько бы экземпляров ты ни создал.

`static` везде: `Math.Max`, `string.IsNullOrEmpty`, `File.ReadAllText`, `DateTime.UtcNow`, `Program.Main`. Неправильное использование — mutable static caches без синхронизации, `static` service locators с `DbContext` — даёт баги только под нагрузкой и ускользает от unit tests. Middle-разработчик знает, когда `static` — идиоматичная utility, когда нужен **instance + interface** в DI, и когда CLR запускает static constructors незаметно.

## Core Concepts

### Instance vs type members

**Instance member** работает с конкретным объектом: `order.Total` читает total этого order. **Static member** принадлежит объявляющему типу: `decimal.Max(a, b)` не требует экземпляра `Math` — per-call state на объекте `Math` нет; создать `Math` нельзя.

```csharp
public class Counter
{
    public int InstanceCount { get; private set; }

    public void Increment() => InstanceCount++;

    public static int GlobalCount;
    public static void IncrementGlobal() => GlobalCount++;
}
```

Два экземпляра `Counter` имеют разные `InstanceCount`. `GlobalCount` общий для **всех** экземпляров и доступен без объекта: `Counter.IncrementGlobal()`.

Static methods **не** virtual в смысле полиморфизма из `1.2.7` — нет указателя `this`. Они не могут обращаться к instance members без явно переданного экземпляра. Поэтому static methods не реализуют instance methods интерфейса напрямую (кроме static abstract members в C# 11+ — продвинуто; стандартный паттерн — instance methods на implementations).

### Static classes

**`static class`** нельзя инстанцировать; только static members. Компилятор помечает его `abstract sealed` — наследование запрещено.

```csharp
public static class StringValidation
{
    public static bool IsValidEmail(string email)
        => email.Contains('@') && email.Contains('.');
}
```

Используй static class, когда тип — **чистый namespace для utilities** без instance identity, как `System.Math`. Если нужны полиморфизм или state на consumer — обычный class + interface и регистрация в DI.

**Ограничения:** нет instance constructors (только static constructor), нет instance fields/methods, нельзя implement interfaces как instance types (static interface members C# 11+ — отдельная тема).

### Static fields и properties

**Static field** аллоцируется один раз при загрузке типа. Порядок инициализации важен: static fields до static constructor; explicit field initializers — в порядке объявления.

```csharp
public class AppConfig
{
    public static string EnvironmentName { get; set; } = "Development";
    private static readonly Lazy<IReadOnlyList<string>> _features = new(LoadFeatures);

    public static IReadOnlyList<string> Features => _features.Value;

    private static IReadOnlyList<string> LoadFeatures()
        => File.ReadAllLines("features.txt");
}
```

**Mutable static state** — глобальная переменная в disguise. В ASP.NET Core избегай `public static DbContext` или `public static IServiceProvider` кроме жёстко контролируемой инфраструктуры. Concurrent requests мутируют общий static field → race conditions.

**Readonly static** (`static readonly`) безопаснее для вычисления один раз. **`const`** — compile-time constant для primitives/strings — ещё строже, inlining компилятором.

### Static constructors (type initializers)

**Static constructor** (**.cctor** в IL) выполняется **один раз на тип** до первого доступа к любому static или instance member этого типа, в **CLR-guaranteed thread-safe** режиме.

```csharp
public class DatabaseBootstrap
{
    public static string ConnectionString { get; }

    static DatabaseBootstrap()
    {
        ConnectionString = Environment.GetEnvironmentVariable("DB_CONN")
            ?? throw new InvalidOperationException("DB_CONN missing");
    }
}
```

Правила:

- Без access modifiers (`static DatabaseBootstrap()` — implicitly `private`).
- Без параметров; нельзя вызвать явно.
- Не более одного раза на app domain на тип.
- Если static constructor бросает исключение, тип **faulted** — последующие использования → `TypeInitializationException`.

**Порядок:** CLR инициализирует base type static fields/constructors перед derived. Между несвязанными типами порядок **не** гарантирован, пока не создашь явную зависимость.

### Static в generics

Каждый **closed constructed type** получает свои static fields:

```csharp
public class TypeCache<T>
{
    public static int AccessCount;
}
```

`TypeCache<int>.AccessCount` и `TypeCache<string>.AccessCount` — **разное** хранилище. Так фреймворки держат per-`T` metadata без словарей. Удивляет тех, кто ожидает один global counter для всех `T`.

### Static local functions и static lambdas

C# поддерживает `static` на **local functions** и lambdas (без capture) против accidental closure allocations:

```csharp
public int Sum(int[] values)
{
    static int Add(int a, int b) => a + b;
    return values.Aggregate(0, Add);
}
```

Это про **capture semantics**, не type-level static — но keyword связывает с «нет скрытого instance state».

## Under the Hood

### Где живут static data

Instance fields внутри каждого объекта в heap. **Static fields** принадлежат **type object** (method table / type handle) загруженного типа в app domain. JIT резолвит доступ к static field через фиксированный offset — быстро, но globally shared.

Одна копия — GC считает static fields **GC roots**; они держат объекты живыми всё время процесса, пока не обнулишь.

### Когда CLR запускает `.cctor`

Runtime триггерит инициализацию типа при первом касании: `new Derived()`, доступ к `Derived.StaticField`, или `typeof(Derived)` в некоторых случаях. CLR использует lock per type — два потока не выполнят `.cctor` дважды.

**BeforeFieldInit** (default для C# classes): CLR может чуть отложить field initializers в рамках правил — редко кусает application code, но объясняет subtle ordering в сложных static graphs.

### Static vs singleton instance

**Singleton** (один экземпляр класса) всё ещё имеет **instance methods** и может implement interfaces — testable через замену в DI. **Static class** прибит к типу — mock без wrappers невозможен.

```csharp
// Сложно тестировать — прямой static call
public static class Clock
{
    public static DateTime Now => DateTime.UtcNow;
}

// Тестируемо — inject IClock
public interface IClock { DateTime UtcNow { get; } }
public sealed class SystemClock : IClock { public DateTime UtcNow => DateTime.UtcNow; }
```

Production ASP.NET регистрирует `IClock` как singleton **instance** — поведенчески один объект, но replaceable в tests.

## Syntax / API

| Конструкция | Синтаксис | Заметки |
|-------------|-----------|---------|
| Static field | `public static int Count;` | Shared; thread-safety — твоя задача |
| Static readonly | `private static readonly Regex Rx = new(...);` | Init один раз |
| Static property | `public static string Name { get; set; }` | Избегай mutable в web apps |
| Static method | `public static int Parse(string s)` | Нет `this` |
| Static class | `public static class Util { }` | Нет instance members |
| Static constructor | `static MyClass() { }` | Без параметров, private |
| const | `public const int Max = 100;` | Compile-time |

**Нельзя:** override static methods (`new` hiding возможен — избегай).

## Examples

### Пример 1: Идиоматичная utility

```csharp
public static class Guard
{
    public static void AgainstNull(object? value, string paramName)
    {
        if (value is null)
            throw new ArgumentNullException(paramName);
    }
}
```

Pure functions, без mutable static state — ок везде.

### Пример 2: Lazy static initialization

```csharp
public sealed class FeatureFlags
{
    private static readonly Lazy<HashSet<string>> _enabled = new(Parse);

    public static bool IsEnabled(string flag) => _enabled.Value.Contains(flag);

    private static HashSet<string> Parse()
        => File.ReadAllLines("flags.txt").ToHashSet(StringComparer.OrdinalIgnoreCase);
}
```

Чтение файла один раз, thread-safe через `Lazy<T>`.

### Пример 3: Anti-pattern в web app

```csharp
public class UserContext
{
    public static int CurrentUserId; // BAD в ASP.NET Core
}
```

Два concurrent HTTP request перезаписывают user id друг друга. **Fix:** `AsyncLocal<int>`, `HttpContext.User`, или scoped service.

## Common Mistakes & Anti-patterns

**Mutable static в multi-threaded apps** — race без `lock` или concurrent collections.

**Static service locator** — `MyApp.Services.GetRequiredService<T>()` прячет dependencies; constructor injection из `1.2.11`.

**Static `DbContext` или connections** — lifetime и thread-safety нарушают дизайн EF Core.

**Предполагать порядок static ctor между типами** — хрупкий bootstrapping; prefer `IHostedService` в ASP.NET.

**Тестировать static methods с IO** — tests бьют в реальный filesystem без wrapper + interface.

## Production & Real-World Notes

Framework code использует static широко (`EntityFrameworkQueryableExtensions`, `Enumerable`). Application code склоняется к **instance + DI** для всего с side effects или external IO.

**Configuration:** prefer `IOptions<T>` над `public static AppSettings` — reloadable, testable.

**Analyzers:** CA1810 — performance static constructors; Sonar флагает singletons как static mutable state.

**Source generators** иногда emit `static partial` helpers — всё ещё stateless utilities.

## Comparison / Trade-offs

| Подход | Когда | Риск |
|--------|-------|------|
| `static` utility | Pure helpers, без IO или one-time init | Низкий если immutable |
| `static` mutable | Legacy globals | Высокий на web/server |
| Singleton instance (DI) | Shared service, testable | Lifetime должен совпадать со scope |
| `const` | Истинные константы | Recompile для изменения |

## Quick Reference

| Нужно | Используй |
|-------|-----------|
| Math/string helpers | `static` methods на static class |
| One-time type setup | `static constructor` + `static readonly` |
| Shared per-request state | **Не** static — scoped service |
| Per-generic-type cache | `static` field на `Cache<T>` |
| Testable time/API | `IClock` instance, не `DateTime.UtcNow` в business logic |

## Key Takeaways

- **`static`** = один member на **тип**, не на объект.
- **Static classes** — sealed utility holders; нет instances, нет inheritance.
- **Static constructors** — один раз, thread-safe, до первого использования типа.
- **Generic static fields** — per closed type `Foo<int>` vs `Foo<string>`.
- **Mutable static** в ASP.NET Core почти всегда неверен — DI scopes.
- **Singleton service** ≠ static class: singleton — replaceable instance с одним lifetime.
