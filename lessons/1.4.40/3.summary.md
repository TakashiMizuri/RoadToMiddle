# HTTP → service → DB async — краткая выжимка

> `1.4.40` · ~8 мин на повторение

## Главная идея

HTTP GET проходит цепочку **async delegates**: Kestrel → middleware (`await next`) → controller → service → **EF `*Async`** → PostgreSQL. На SQL I/O **thread pool thread не блокируется** — scalability async.

## Ключевые пункты

- Kestrel: non-blocking sockets, work на thread pool.
- Middleware: `RequestDelegate` возвращает `Task`.
- Controller/service: `await` + `CancellationToken` (`RequestAborted`).
- **Scoped `DbContext`** — один на request, dispose в конце.
- **Connection pool** (Npgsql) отделён от thread pool.
- Continuation после SQL может быть на **другом** thread pool thread.
- ASP.NET Core: **нет** `SynchronizationContext` → `ConfigureAwait(false)` не критичен в app code.

## Самые важные примеры

```csharp
var user = await _db.Users.AsNoTracking()
    .FirstOrDefaultAsync(u => u.Id == id, ct);
```

## Частые ошибки

- `Task.Run` + sync EF вместо `*Async`.
- Parallel queries на одном `DbContext`.
- `.Result` где должен быть `await` — ломает thread model диаграммы.

## Мнемоника

> **Await at I/O, release the thread** — await на I/O, thread свободен.
