# break, continue, return — управление потоком

> Roadmap: `1.1.11` · Node: `1.1` — C#: основы языка · Depth: practice

## Learning Objectives

После этого урока ты сможешь:

- Использовать **`break`** для выхода из loops и `switch`.
- Использовать **`continue`**, чтобы пропустить остаток текущей итерации loop.
- Использовать **`return`** для немедленного выхода из method с value или без.
- Применять **early return (guard clauses)** для flatten nested logic.
- Объяснить, почему в C# **нет labeled `break`/`continue`**, и что делать вместо них.

---

## Why This Matters

Loops из `1.1.10` повторяются, пока condition не изменится — но real programs нужно **выйти раньше** (Exit, item found, fatal validation) или **пропустить элемент** (blank line, invalid row). Три keyword — **`break`**, **`continue`**, **`return`** — управляют *куда* идёт execution без дублирования conditions.

Junior часто использует flags (`bool found`) там, где хватит **`break`**. Злоупотребление **`return`** во вложенных loops даёт "arrow code". Middle балансирует **clarity** и **structure**: `break`/`continue` в одном loop; **`return`** на границах method для guards; extract method при nesting > 2 уровней. Эти tools — в каждом console app, service, parser до async и exceptions.

## Core Concepts

### `break`: выход из innermost construct

**`break`** завершает ближайший **`switch`**, **`while`**, **`do`** или **`for`** — не весь method (кроме tiny `Main`).

```csharp
for (int i = 0; i < 100; i++)
{
    if (i == 5)
        break;

    Console.WriteLine(i);
}
// 0..4
```

В **`switch`** `break` блокирует fall-through. В loops — **"итерации достаточно"**: search found, user quit.

**Nested loops:** `break` выходит **только из inner loop**. Для обоих — **`goto`** (редко), **flag**, лучше — **extract method** и `return` (`1.1.12`).

```csharp
bool found = false;

for (int i = 0; i < rows && !found; i++)
{
    for (int j = 0; j < cols; j++)
    {
        if (grid[i, j] == target)
        {
            found = true;
            break; // только inner for
        }
    }
}
```

Когда inner `break` мало — **`return`** из helper method яснее.

### `continue`: следующая итерация

**`continue`** переходит к **next iteration** innermost loop, пропуская rest body:

```csharp
for (int i = 0; i < 10; i++)
{
    if (i % 2 == 0)
        continue;

    Console.WriteLine(i); // 1,3,5,7,9
}
```

**`continue`** для **edge cases в начале** body — invalid parse, empty string — без extra nesting. С **`TryParse`** из `1.1.9`:

```csharp
while (true)
{
    Console.Write("Number (0 to stop): ");
    if (!int.TryParse(Console.ReadLine(), out int n))
    {
        Console.WriteLine("Try again.");
        continue;
    }

    if (n == 0)
        break;

    Process(n);
}
```

Не разбрасывай `continue`, если теряется happy path. >2 continues в одном loop — restructure.

### `return`: выход из method

**`return`** завершает **current method**. **`void`**: bare **`return;`**. С value: **`return expression;`**:

```csharp
static int Absolute(int x)
{
    if (x >= 0)
        return x;

    return -x;
}
```

После `return` на этом path statements method не выполняются. Caller продолжает после call site.

В **top-level** / **`Main`**, **`return`** завершает entry point (status 0). Idiomatic для **guard clauses** при invalid input:

```csharp
if (!int.TryParse(args[0], out int n))
{
    Console.Error.WriteLine("Usage: app <number>");
    return;
}
```

### Early return и guard clauses

**Guard clauses** проверяют preconditions сверху и **`return`** (или throw) early — без pyramid `if`:

```csharp
static string DescribeUser(string? name, int age)
{
    if (string.IsNullOrWhiteSpace(name))
        return "Anonymous";

    if (age < 0)
        return "Invalid age";

    return $"{name} ({age})";
}
```

Guards читаются как checklist. Лучше, чем один trailing `result` под nested braces.

### `return` vs `break` в loops

| Goal | Use |
|------|-----|
| Только current loop | `break` |
| Skip one iteration | `continue` |
| Весь method и все loops | `return` |

Loop в method + find one item → **`return`** часто яснее `break` + flag после loop.

### Нет labeled break/continue в C#

Java: `break outerLoop;`. **C# — нет**. Alternatives:

1. **`return`** из extracted method с loops.
2. **Boolean flag** в outer condition.
3. **`goto`** (редко в app code).

Знай это, чтобы не искать Java syntax в C#.

## Under the Hood

`break`, `continue`, `return` → **branch instructions** к IL labels — exit loop, increment step, method epilogue. **`return`** может запустить **`finally`** в `try/finally` — order важен для disposal.

Compiler rejects **`break`/`continue` outside loops**. Unreachable code after `return` — **warning** CS0162.

## Examples

### Пример 1: search с break

```csharp
int[] ids = { 4, 7, 12, 19 };
int target = 12;
int index = -1;

for (int i = 0; i < ids.Length; i++)
{
    if (ids[i] == target)
    {
        index = i;
        break;
    }
}

Console.WriteLine(index >= 0 ? $"Found at {index}" : "Not found");
```

### Пример 2: filter с continue

```csharp
string[] lines = { "  ok ", "", "data", "   ", "end" };

foreach (string line in lines)
{
    string trimmed = line.Trim();
    if (trimmed.Length == 0)
        continue;

    Console.WriteLine(trimmed);
}
```

### Пример 3: early returns

```csharp
static decimal ApplyDiscount(decimal total, string? coupon)
{
    if (total <= 0)
        return 0m;

    if (string.IsNullOrEmpty(coupon))
        return total;

    if (coupon == "SAVE10")
        return total * 0.9m;

    return total;
}
```

## Common Mistakes & Anti-patterns

**`break` из nested loops** — только inner; logic bug.

**`continue` перед forgotten update в `while`** — update в consistent place или `for`.

**Много `return` без structure** — extract method past ~3–4.

**`return` в `Main` vs `break` в menu loop** — `return` ends app.

---

## Production & Real-World Notes

Service methods: guard **`return`** до DB. **`continue`** в batch processors для bad records + logging.

Style guides по-разному к **`continue`**. Следуй team; default — readability.

В **`async`**, **`return`** completes task — same keyword, другая machinery (позже).

## Comparison / Trade-offs

| Technique | When |
|-----------|------|
| `break` | Target found; exit switch |
| `continue` | Skip bad item |
| `return` | Method-level final |
| Extract + return | Nested loops |

Flags vs early return: flags плохо в nested blocks; **`return`** — intent на scope method.

## Quick Reference

```
break     → switch / innermost loop
continue  → next iteration
return    → exit method
C#: no labeled break → extract method
Guards: validate early, return early
```

---

## Key Takeaways

- **`break`** — innermost loop/switch, не arbitrary nesting.
- **`continue`** — skip iteration; invalid rows.
- **`return`** — end method; guard clauses flatten nesting.
- `return` over flag+break, когда нужен exit всего method.
- C# без labeled break — extract methods.
- Связка с loops (`1.1.10`) и methods (`1.1.12`).
