# Primitive Types: `int`, `long`, `float`, `double`, `decimal`, `bool`, `char`

> Roadmap: `1.1.3` · Node: `1.1` — C#: language basics · Depth: **глубоко**

## Learning Objectives

After this lesson you will be able to:

- Map C# **primitive keywords** to **CLR types** (`int` → `System.Int32`, etc.).
- Explain **value type** semantics: stack allocation (conceptually), copying by value, no `null` (except `Nullable<T>`).
- Choose among **integer**, **floating-point**, and **decimal** types for a given numeric domain.
- Read and write **literals** with correct suffixes (`L`, `F`, `D`, `M`) and bases (`0x`, binary).
- Predict **overflow** behavior with checked vs unchecked contexts.
- Describe why **`decimal`** suits money and **`double`** suits science — at the bit level (intro).

---

## Why This Matters

Every variable declaration in C# picks a type — and that choice encodes range, precision, memory footprint, and API contracts. Backend code processes order quantities (`int`), timestamps as ticks (`long`), tax rates (`decimal`), sensor readings (`double`), and feature flags (`bool`). Using `float` for currency or `int` for billions of micro-transactions causes production bugs that unit tests may miss until scale hits.

Primitive types are **aliases** for CLR value types defined in `System`. They behave identically to their struct forms but read naturally in code. Because they are **value types**, assignment copies bits — mutating a copy does not affect the original. That differs sharply from **reference types** like `string` (next lesson) where variables share heap objects.

Middle developers must articulate trade-offs in code review: "Why `decimal` here?" "Will this `int` overflow at peak load?" "Is `double` comparison with `==` safe?" Depth on primitives connects directly to GC pressure (boxing), API design (DTO field types), and PostgreSQL column types (`numeric` vs `double precision`).

## Core Concepts

### Keywords and CLR Types

C# provides keywords that map 1:1 to BCL struct types:

| C# keyword | CLR type | Size (typical) | Signed | Approx range |
|------------|----------|----------------|--------|--------------|
| `bool` | `System.Boolean` | 1 byte (stored) | — | `true` / `false` |
| `byte` | `System.Byte` | 1 byte | No | 0 … 255 |
| `sbyte` | `System.SByte` | 1 byte | Yes | −128 … 127 |
| `short` | `System.Int16` | 2 bytes | Yes | −32 768 … 32 767 |
| `ushort` | `System.UInt16` | 2 bytes | No | 0 … 65 535 |
| `int` | `System.Int32` | 4 bytes | Yes | ±2.1×10⁹ |
| `uint` | `System.UInt32` | 4 bytes | No | 0 … 4×10⁹ |
| `long` | `System.Int64` | 8 bytes | Yes | ±9×10¹⁸ |
| `ulong` | `System.UInt64` | 8 bytes | No | 0 … 1.8×10¹⁹ |
| `char` | `System.Char` | 2 bytes | — | UTF-16 code unit |
| `float` | `System.Single` | 4 bytes | Yes | IEEE-754 binary32 |
| `double` | `System.Double` | 8 bytes | Yes | IEEE-754 binary64 |
| `decimal` | `System.Decimal` | 16 bytes | Yes | ±7.9×10²⁸, 28–29 sig digits |

Roadmap focus: **`int`, `long`, `float`, `double`, `decimal`, `bool`, `char`**. Others appear in interop and low-level code.

`int` is the default integer — loop counters, collection counts, IDs in many apps. **`long`** holds larger ranges (Unix milliseconds, snowflake IDs). **`uint`/`ulong`** appear in hashing, bit masks, protocols.

### Value Type Semantics

Primitives are **structs** (value types). Local variables conceptually live on the **stack** (JIT may enregister them). Assignment **copies value**:

```csharp
int a = 10;
int b = a;
b = 20;
// a still 10
```

Method parameters pass **by value** unless modified with `ref`/`out`/`in` (`1.1.13`). Returning `int` copies the bits out — cheap for small types.

**No null** for bare `int` — compiler error. Optional integers use **`int?`** (`Nullable<int>`) — later in `1.1.18`.

### Integer Literals and Bases

```csharp
int dec = 42;
int hex = 0x2A;           // 42
int binary = 0b101010;    // 42
long big = 1_000_000_000L; // suffix L or l
uint u = 100U;
ulong ul = 100UL;
```

Underscores `_` improve readability — ignored by compiler. Default integer literal is **`int`**; if too large, compile error — append **`L`**.

### Floating-Point: `float` and `double`

Based on **IEEE 754**. **`double`** is default for floating literals:

```csharp
double pi = 3.141592653589793;
float approx = 3.14F;   // suffix F required (or 3.14f)
double e = 2.718D;      // D optional for double
```

**Binary floating cannot represent all decimals exactly** — `0.1 + 0.2 != 0.3` in `double`. Never use `==` for money or critical comparisons; use epsilon or **`decimal`**.

**`float`** — 32-bit, less precision, useful for graphics/interop when memory bandwidth matters. **`double`** — default for scientific/engineering math in .NET.

Special values: `double.NaN`, `Infinity`, `-Infinity` from invalid ops (`0.0 / 0.0`).

### `decimal` for Base-10 Precision

**`decimal`** is a 128-bit **decimal floating-point** type (not IEEE binary). Microsoft designed it for **financial and accounting** calculations:

```csharp
decimal price = 19.99m;
decimal tax = price * 0.08m;
```

Suffix **`m` or `M`** required on literals. Slower than `double`, larger memory — acceptable for business logic layers handling money. Maps well to PostgreSQL **`numeric`/`decimal`**.

Range and scale limits exist (28–29 significant digits) — not infinite precision.

### `bool` and `char`

**`bool`** — only `true` or `false`. No implicit conversion to/from numbers (unlike C). Logical operators: `&&`, `||`, `!` (short-circuit).

**`char`** — single **UTF-16 code unit** (16 bits), not full Unicode scalar by itself:

```csharp
char letter = 'A';
char newline = '\n';
char unicode = '\u0061'; // 'a'
```

Supplementary characters (emoji) often need two `char` (**surrogate pair**) or use **`string`** / **`Rune`** (.NET 5+). Roadmap defers full Unicode to string lesson.

### Conversions and Overflow

**Implicit widening** — safe direction (e.g. `int` → `long`, `int` → `double`):

```csharp
long extended = intValue;
```

**Explicit cast** — narrowing may lose data:

```csharp
int truncated = (int)3.99; // 3
```

**Overflow** on integers: default **unchecked** — high bits discarded, silent wrap (two's complement). **`checked` context** throws **`OverflowException`**:

```csharp
checked
{
    int max = int.MaxValue;
    int oops = max + 1; // throws
}
```

Use **`checked`** for user input parsing and financial counters where wrap is unacceptable.

## Under the Hood

### CLR Storage and Metadata

Each primitive struct exists in **`System`** assembly. IL uses **`load`/`store` instructions** specialized by size (`ldc.i4`, `conv.i8`, etc.). JIT maps locals to CPU registers or stack slots.

**`bool`** stored as 1 byte (0/1) — not a single CPU bit in memory. **`decimal`** implemented as struct with scale, sign, and 96-bit integer mantissa components — arithmetic in software, not FPU binary ops.

### IEEE 754 Binary Layout (Concept)

**`float`**: 1 sign bit, 8 exponent bits, 23 mantissa bits. **`double`**: 11 exponent, 52 mantissa. Rounding errors arise when decimal fractions have no exact binary expansion — fundamental limitation, not a .NET bug.

### Boxing

Assigning value type to **`object`** or interface it implements causes **boxing** — heap allocation + copy. Hot paths avoiding boxing use generics (`List<int>` vs legacy `ArrayList`). Primitives in APIs — prefer **`T`** generic constraints over **`object`**.

```csharp
object boxed = 42; // boxes int
int unboxed = (int)boxed; // unbox copy
```

```csharp

int count = 0;
```

## Examples

### Example 1: Money with `decimal`

```csharp
decimal subtotal = 100.00m;
decimal discountRate = 0.15m;
decimal discount = subtotal * discountRate;
decimal total = subtotal - discount;
Console.WriteLine(total); // 85.00
```

Same with `double` might print `84.999999999999`.

### Example 2: Overflow Awareness

```csharp
int items = 2_000_000_000;
int weight = 2;
// Unchecked: wraps negative
int uncheckedProduct = items * weight;

try
{
    checked
    {
        int checkedProduct = items * weight;
    }
}
catch (OverflowException)
{
    Console.WriteLine("Use long for product");
}

long safe = (long)items * weight;
```

## Common Mistakes & Anti-patterns

**`double` for currency** — rounding surprises at tax lines.

**Comparing floats with `==`** — use tolerance or integer cents.

**Forgetting `m` suffix** — `19.99` is `double`, not `decimal`; may require cast.

**Assuming `char` is a letter** — it's a UTF-16 code unit; surrogate pairs need two.

**Ignoring overflow** on `int` multiplication before assignment to `long` — multiply in `long` **before** product exceeds `int.MaxValue`.

## Production & Real-World Notes

DTOs for APIs: JSON numbers map to `double` by default in System.Text.Json — configure **`JsonNumberHandling`** or use **`decimal`** / string for money fields.

Database: **`bigint`** ↔ `long`, **`integer`** ↔ `int`, **`numeric`** ↔ `decimal`. ORMs preserve types when mapped correctly.

Performance: tight loops over millions of floats — `float` may help cache; financial ledger — always `decimal` in domain layer.

| Need | Prefer |

## Key Takeaways

- Primitive keywords alias **CLR struct types**; all are **value types** copied on assignment.
- **`int`** default integer; **`long`** for large ranges; **`decimal`** for base-10 money math.
- **`float`/`double`** are IEEE binary — precision limits; never `==` for decimals as doubles.
- Literals need suffixes: **`L`**, **`F`**, **`M`**, **`U`**, **`UL`** as appropriate.
- **`checked`** throws on integer overflow; default unchecked wraps.
- **Boxing** value types to `object` allocates on heap — avoid in hot paths.
