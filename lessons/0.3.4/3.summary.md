# HTTP Methods — краткая выжимка

> `0.3.4` · ~10 мин на повторение

## Главная идея

**Method** в start line выражает **intent** операции на URL: GET читает (safe), POST создаёт/командует (not idempotent), PUT заменяет целиком (idempotent), PATCH — частично, DELETE удаляет (idempotent), HEAD — metadata без body, OPTIONS — capabilities и CORS preflight.

## Ключевые пункты

- **GET/HEAD** — safe (no intended state change); не использовать для mutation/CSRF-prone links.
- **POST** — create/command; повтор может дублировать; **Idempotency-Key** на application layer.
- **PUT** — full replace at URI, idempotent; **PATCH** — partial, merge rules важны.
- **DELETE** — idempotent remove; prefer **204** без лишнего body.
- **405 Method Not Allowed** + header **Allow** — когда path есть, method нет.
- ASP.NET Core: `[HttpGet]`, `MapPost`, routing = method + path.

## Самые важные примеры

```bash
curl -X PATCH https://api.example.com/users/1 \
  -H "Content-Type: application/json" \
  -d '{"email":"new@example.com"}'
```

CRUD mapping: `GET/POST /users`, `GET/PUT/PATCH/DELETE /users/{id}`.

## «Под капотом» в одном абзаце

Router match HTTP method metadata + template; GET responses cacheable по URL+method — поэтому GET mutation ломает CDN/browser cache. CORS preflight — OPTIONS с `Access-Control-Request-Method` перед cross-origin POST/PATCH с JSON.

## Частые ошибки

- DELETE через GET query param.
- POST для всех операций « потому что проще ».
- PUT с partial DTO без документированных null semantics.
- 404 вместо 405 при unsupported method.

## Мнемоника

> **GRUD + extras**: **G**et read, **P**ost create, **P**ut replace, **Patch** partial, **Delete** remove; **Head** peek, **Options** ask.
