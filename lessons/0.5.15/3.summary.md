# data-* и custom elements — краткая выжимка

> `0.5.15` · ~8 мин на повторение

## Главная идея

**`data-*`** — легальный способ повесить custom metadata на HTML; в JS — **`element.dataset`** (camelCase, values = strings). **Custom Elements** — register своих tags (`x-name`) с lifecycle; часть **Web Components** вместе с Shadow DOM и `<template>`.

## Ключевые пункты

- HTML: `data-product-id`; JS: `dataset.productId`.
- Values всегда **strings** — парси числа/boolean явно.
- Invalid arbitrary attrs (`foo="bar"`) — fail W3C validation.
- **Event delegation:** `[data-action]` + `closest`.
- Custom element name **must contain hyphen** (`user-card`).
- **`connectedCallback`** — setup при insert в document.
- Shadow DOM — style/DOM encapsulation для portable widgets.
- React: `data-testid`; CE consume в design systems — не обязательно писать с нуля.

## Самые важные примеры

```html
<button data-action="save" data-doc-id="99">Save</button>
```

```javascript
btn.dataset.docId; // "99"
customElements.define('formatted-date', FormattedDate);
```

## Частые ошибки

camelCase в HTML (`data-productId`). `"false"` string truthy в `if (dataset.flag)`. `<div data-click>` вместо `<button>`.

## Мнемоника

> **data-** = «private labels для JS/CSS»; **custom element** = «новый tag с hyphen и lifecycle».
