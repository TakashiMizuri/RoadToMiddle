# Exercises: Nullable Reference Types

> Complete after passing `4.test-yourself.md`.

## Easy

### E1. Enable NRT
**Goal:** New console project with `<Nullable>enable</Nullable>`; fix all warnings in 20-line sample.
**Success criteria:** Zero nullable warnings without `!` except one documented case.

### E2. Guard clause
**Goal:** Method `void SendEmail(string to, string? cc)` — validate `to` with `ThrowIfNull`; no warning on `to`.
**Success criteria:** CS8602-free body.

### E3. Honest optional
**Goal:** DTO with `required string Id` and `string? MiddleName`.
**Success criteria:** No CS8618.

## Medium

### M1. TryGet pattern
**Goal:** `bool TryGetUser(string id, [NotNullWhen(true)] out User? user)` — caller uses without `!` after true.
**Success criteria:** Analyzer happy at call site.

### M2. JSON boundary
**Goal:** Deserialize JSON missing optional field into record with NRT; add validation for required field.
**Success criteria:** Explain runtime null vs compile intent in comment.

### M3. Remove `!` refactor
**Goal:** Given code with 5× `!`, refactor to guards/patterns.
**Success criteria:** Same behavior, fewer suppressions.

## Hard (stretch)

### H1. Custom attribute
**Goal:** Method `[MemberNotNull(nameof(_cache))] void Init()` — field used after Init without `!`.
**Success criteria:** Compiler tracks `_cache` non-null.

## Submission checklist

- [ ] Project has Nullable enable
- [ ] At least one `string?` and one `required` member
- [ ] No unjustified `!`
