# HttpClient pitfalls — DNS и socket exhaustion

> Roadmap: `1.4.26` · Node: `1.4` — Async + ASP.NET preview · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Объяснить **socket exhaustion** от **`new HttpClient()` per request** и **TIME_WAIT**.
- Описать **DNS caching** в **SocketsHttpHandler** и stale endpoints после failover.
- Выбрать правильный **HttpClient lifetime** в ASP.NET Core.
- Распознать симптомы: **SocketException**, **AddressAlreadyInUse**, **502** под load.
- Настроить **PooledConnectionLifetime**, **IdleTimeout**, **MaxConnectionsPerServer**.

---

## Why This Matters

Классический footgun: **`using var client = new HttpClient()`** на каждый request. IDisposable вводит в заблуждение — для HttpClient нужен **reuse**. Каждый instance → connections в **TIME_WAIT** → исчерпание ephemeral ports.

Второй pitfall: **DNS** кешируется на lifetime handler. Singleton без rotation → старый IP после deploy.

Production microservices → **IHttpClientFactory**, не raw **new**.

---

## Core Concepts

### Lifetime misconception

**HttpClient** — facade над **HttpMessageHandler** (pool, DNS, TLS). **Dispose** каждый call → нет reuse → TIME_WAIT avalanche.

Anti-pattern:

```csharp
using var client = new HttpClient();
return await client.GetStringAsync(url);
```

ASP.NET: **IHttpClientFactory**.

### Socket exhaustion

Ephemeral ports + быстрый connect/disconnect. Симптомы при load / **ForEachAsync** без factory.

### DNS stale

**PooledConnectionLifetime** / factory **HandlerLifetime** (~2 min default) — periodic refresh.

```csharp
new SocketsHttpHandler
{
    PooledConnectionLifetime = TimeSpan.FromMinutes(5),
    MaxConnectionsPerServer = 20
};
```

### Dispose factory clients

**CreateClient()** — не dispose как short-lived per anti-pattern; factory owns handler.

## Common Mistakes & Anti-patterns

new per request. Singleton без DNS rotation. Dispose factory client wrong. Unbounded connections.

## Production & Real-World Notes

Official guidance: reuse. Incidents: deploy + DNS, load spike, new parallel code.

---

## Key Takeaways

- Per-request **new HttpClient** → socket exhaustion.
- Shared **handler pool** + **PooledConnectionLifetime**.
- **IHttpClientFactory** — решение в ASP.NET (`1.4.27`).
- Async + bounded parallelism, не больше clients.
