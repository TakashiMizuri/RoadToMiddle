# Parallel.ForEachAsync — краткая выжимка

> `1.4.25` · ~8 мин на повторение

## Главная идея

**Parallel.ForEachAsync** ограничивает число **одновременных async операций** при обходе коллекции — защита от unbounded **Task.WhenAll** при mass I/O (HTTP fan-out).

## Ключевые пункты

- .NET 6+; delegate `async (item, ct) => ...`.
- **MaxDegreeOfParallelism** — ceiling (default ProcessorCount — переопределяй для I/O).
- **Parallel.ForEach** (sync) — CPU; для I/O — **ForEachAsync**.
- **CancellationToken** в options + `ct` в delegate.
- Thread-safe results: **ConcurrentBag**, не `List` без lock.
- ASP.NET: **IHttpClientFactory**, token из request.
- Per-item try/catch vs fail-fast — явный выбор.

## Самые важные примеры

```csharp
await Parallel.ForEachAsync(ids, new ParallelOptions
{
    MaxDegreeOfParallelism = 10,
    CancellationToken = ct
}, async (id, ct) =>
{
    await client.GetAsync($"/api/{id}", ct);
});
```

## Частые ошибки

- `Task.WhenAll` на 500 элементов без limit.
- `Parallel.ForEach` + `.Result` на HTTP.
- Новый HttpClient на iteration.

## Мнемоника

**ForEachAsync = async + cap; WhenAll = no cap until you regret it.**
