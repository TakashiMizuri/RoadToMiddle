# Формы: `form`, `label`, типы `input`, `textarea`, `select`

> Roadmap: `0.5.7` · Node: `0.5` — HTML · Depth: practice

## Learning Objectives

После этой лекции ты сможешь:

- Строить ввод данных через **`<form>`**, **`action`**, **`method`**, **`enctype`**.
- Связывать **`<label for>`** с controls (explicit и implicit).
- Выбирать **`input type`**: text, email, password, number, date, checkbox, radio, file, hidden.
- Использовать **`<textarea>`**, **`<select>`**, **`<option>`**, **`<optgroup>`**.
- Группировать поля через **`<fieldset>`** и **`<legend>`** в semantic layout из `0.5.2`.

---

## Why This Matters

Login, checkout, фильтры, тикеты — всё это forms. HTML form — native contract browser ↔ server. Даже React-приложения часто начинают с `<form>` для progressive enhancement, autofill и a11y. Form без **labels** проваливает WCAG. Неверные **`method`** или **`enctype`** ломают upload или светят credentials в query string.

`0.5.8` добавит validation attributes, но разметка полей сейчас определяет, к чему привяжутся ошибки, что услышит screen reader и какие `name` получит ASP.NET Core model binder.

## Core Concepts

### `<form>`

- **`action`** — URL приёма данных.
- **`method`** — **`get`** (query string) или **`post`** (body). Мutations и passwords — **`post`**.
- **`enctype`** — для post: default urlencoded; файлы — **`multipart/form-data`**.

```html
<form action="/api/contact" method="post">
  <button type="submit">Send message</button>
</form>
```

Enter в single-line input submit’ит form — ожидаемое keyboard behavior. **`get`** для поиска (`/products?category=shoes`) — bookmarkable URL state. Никогда **`get`** для паролей.

Forms — в **`<main>`** (`0.5.2`); один логический **`h1`** (`0.5.3`).

### Labels

**Explicit:**

```html
<label for="email">Work email</label>
<input id="email" name="email" type="email" autocomplete="email" />
```

**Implicit** — label оборачивает control.

**`for`** = **`id`**; **`name`** — ключ на server. Placeholder **не** label.

### Типы `input`

**text**, **email**, **tel**, **password**, **number**, **date** / **time** / **datetime-local**, **checkbox**, **radio** (общий **`name`**), **file** (+ multipart form), **hidden** (CSRF — verify server-side), **submit** / prefer **`<button type="submit">`**.

Radio group:

```html
<fieldset>
  <legend>Shipping speed</legend>
  <label><input type="radio" name="shipping" value="standard" checked /> Standard</label>
  <label><input type="radio" name="shipping" value="express" /> Express</label>
</fieldset>
```

Unchecked checkbox не попадает в submit payload.

### `<textarea>`

```html
<label for="message">Message</label>
<textarea id="message" name="message" rows="5" maxlength="2000"></textarea>
```

Для длинного текста; escape default content при re-render (XSS).

### `<select>`

```html
<label for="country">Country</label>
<select id="country" name="country">
  <option value="">Select a country</option>
  <optgroup label="Europe">
    <option value="DE">Germany</option>
  </optgroup>
</select>
```

**`value`** option уходит на server; label the `<select>`, не только первый option.

### `fieldset` / `legend`

Group title для AT при входе фокуса в группу — особенно radio.

## Under the Hood

Submit → constraint validation (`0.5.8`) → form data из successful controls (`name`, not disabled) → encode по method/enctype → HTTP.

ASP.NET Core model binding: **`name`** → DTO properties (case-insensitive). **`autocomplete`** — password managers.

## Common Mistakes & Anti-patterns

Placeholder-only; missing `name`; radio с разными `name`; file без multipart; fake `<div>` inputs; nested forms.

---

## Production & Real-World Notes

CSRF hidden / antiforgery headers; server-side validation обязательна; согласованные `name` с API DTO.

## Quick Reference

```html
<form action="/path" method="post">
  <label for="x">Label</label>
  <input id="x" name="x" type="email" />
  <button type="submit">Save</button>
</form>
```

---

## Key Takeaways

- `method` + `enctype` определяют transport.
- Label через `for`/`id`; `name` для server.
- Radio — один `name`; file — multipart.
- Native controls first; custom widgets — с a11y.
