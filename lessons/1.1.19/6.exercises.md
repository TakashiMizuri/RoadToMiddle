# Exercises: const vs readonly vs static readonly

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. What compiles?
**Goal:** Learn constant expression rules.

**Tasks:**
For each line, write **OK** or **ERROR** and why:
```csharp
public const int A = 1 + 2;
public const string B = "a" + "b";
public const double C = Math.PI;
public static readonly double D = Math.PI;
public readonly int E = 5; // instance field in class
```

**Success criteria:** All five explained with const vs readonly rules.

---

### E2. Choose the modifier
**Goal:** Pick correct keyword per scenario.

**Tasks:**
Match scenario to **`const`**, **`readonly`**, or **`static readonly`**:
1. `[MaxStringLength(500)]` argument value
2. Order ID generated in constructor
3. Process-wide `Stopwatch` start timestamp
4. HTTP header name `"Authorization"`

**Success criteria:** One-line justification each.

---

## Medium

### M1. Const versioning demo
**Goal:** Observe cross-assembly inlining (conceptual or two projects).

**Tasks:**
1. Class library `Lib` with `public const int Limit = 5`.
2. Console app prints `Lib.Limit`.
3. Change library to `10`, rebuild **only** library, copy DLL — observe printed value.
4. Repeat with `public static readonly int Limit = 5` changed to `10`.

**Success criteria:** Written explanation matching observed behavior; mention recompile requirement for `const`.

---

### M2. Immutable service class
**Goal:** Instance readonly in DI style.

**Tasks:**
```csharp
public sealed class InvoiceService
{
    // add readonly dependency field + ctor
    public decimal CalculateTotal(Invoice inv) { /* use _calculator */ }
}
```

Register in minimal DI container; verify field assigned once.

**Success criteria:** No public setter; dependency via constructor only.

---

### M3. Static initialization order
**Goal:** Understand `.cctor` pitfalls.

**Tasks:**
Create two types where static readonly fields cross-reference (carefully). Document initialization order or failure. Refactor to break cycle.

**Success criteria:** Explain **TypeInitializationException** risk.
