## Answers

### 1. [Recall]
**Answer:** `.gitignore` задаёт паттерны путей, которые Git **не предлагает** к tracking (скрывает из untracked в `git status`, не добавляет через обычный `git add`).

**Не влияет на:** файлы **уже в index** (staged) или **уже закоммиченные** в history.

**Why:** Ignore — фильтр при обнаружении новых файлов, не ретроактивное удаление.

---

### 2. [Recall]
**Answer:**
1. **Global excludesfile** (`core.excludesfile`, например `~/.gitignore_global`)
2. **`.git/info/exclude`** — per-repo, не коммитится
3. **`.gitignore`** в репозитории (корень и/или подпапки)

**Common mistake:** Забыть про `.git/info/exclude` как третий стандартный источник.

---

### 3. [Understanding]
**Answer:**
- `/build/` — только директория `build` в **корне** репозитория.
- `build/` — директория `build` на **любой** глубине (`src/build/`, `out/build/`).

**Why:** Leading `/` anchor'ит паттерн к корню repo.

---

### 4. [Understanding]
**Answer:** Когда родительская директория `logs/` полностью ignored, Git **не заходит** внутрь неё — negation для файла внутри не срабатывает.

**Fix:** `logs/*` + `!logs/important.log` (игнорировать содержимое, не саму папку).

---

### 5. [Understanding]
**Answer:** `.gitignore` не удаляет файл из **index** и history. `git rm --cached .env` убирает из staging/index, сохраняя файл на диске; следующий commit перестаёт track'ить `.env`.

**Why:** Ignore применяется к untracked discovery, не к уже tracked.

---

### 6. [Application]
**Answer:**
```gitignore
.env*
!.env.example
```
или:
```gitignore
.env
.env.*
!.env.example
```

**Why:** Negation `!` должно идти **после** более общего ignore-правила.

---

### 7. [Application]
**Answer:** `node_modules/` или `**/node_modules/`

**Why:** Без leading `/` паттерн match'ится на любой глубине; `**` явно подчёркивает вложенность.

---

### 8. [Application]
**Answer:** Создать placeholder, например `uploads/incoming/.gitkeep`, закоммитить. В `.gitignore` добавить `uploads/*` с исключением `!uploads/incoming/.gitkeep` (или игнорировать только файлы загрузок, не структуру).

**Why:** Git track'ит файлы, не пустые папки.

---

### 9. [Analysis]
**Answer:** Наиболее вероятно:
1. `bin/Debug/app.dll` **уже tracked** (был добавлен до `.gitignore`) — нужен `git rm --cached -r bin/`.
2. Паттерн **не match'ит** путь (опечатка, другой регистр на case-sensitive FS) — проверить `git check-ignore -v`.

**Common mistake:** Думать, что `.gitignore` мгновенно «отключает» tracked файлы.

---

### 10. [Analysis]
**Answer:** Покажет строку вида `.gitignore:5:node_modules/    node_modules/react/index.js` — **файл правила**, **номер строки**, **паттерн**, **путь**.

**Why `-v`:** Verbose — без него вывод только пути, без объяснения *почему*.

---

### 11. [Understanding]
**Answer:**
- **Global:** личный OS/IDE шум — `Thumbs.db`, `*.swp` (не засорять team `.gitignore`).
- **Repo `.gitignore`:** project-specific — `node_modules/`, `bin/`, `.env` (все в команде одинаково).

**Why:** Global не коммитится; repo `.gitignore` — shared contract.

---

### 12. [Application]
**Answer:**
```gitignore
[Bb]in/
[Oo]bj/
```
или `**/bin/`, `**/obj/` если нужна явная глубина.

---

### 13. [Analysis]
**Answer:** `debug.log` — **ignored**.

Порядок: `*.log` match'ит `debug.log` → ignored. Строка `!important.log` не относится к `debug.log`. Строка `debug.log` без `!` — снова ignore (избыточно). Итог: ignored.

**Common mistake:** Думать, что явное имя файла без `!` «перебивает» wildcard.

---

### 14. [Recall]
**Answer:** **Конвенция** (community convention). Git не знает специального типа `.gitkeep` — это обычный tracked файл-заглушка.

---

### 15. [Application]
**Answer:** `git check-ignore -v .env.local`

| Path | Result | Why |
|------|--------|-----|
| `dist/app.js` | **not ignored** | `/dist/` только корневой `dist/` |
