# TimeProvider (.NET 8+) — краткая выжимка

> `1.4.36` · ~8 мин на повторение

## Главная идея

**`TimeProvider`** (.NET 8+) — injectable абстракция для «сейчас», задержек и таймеров. **`FakeTimeProvider`** в тестах продвигает время через **`Advance`**, завершая `Delay` без реального ожидания.

## Ключевые пункты

- `DateTime.UtcNow` / `Task.Delay` — статические, ломают детерминизм тестов.
- Production: `TimeProvider.System` (регистрируется в host по умолчанию).
- Тесты: `new FakeTimeProvider()` + `Advance(delta)` / `SetUtcNow(instant)`.
- Заменяй **`Task.Delay`** на **`await timeProvider.Delay(ts, ct)`** в тестируемом коде.
- `CancelAfter` на `CancellationTokenSource` **не** привязан к `FakeTimeProvider`.
- Пакет тестов: `Microsoft.Extensions.TimeProvider.Testing`.

## Самые важные примеры

```csharp
public sealed class SessionValidator(TimeProvider time)
{
    public bool IsExpired(DateTimeOffset expiresAt) =>
        time.GetUtcNow() >= expiresAt;
}
```

```csharp
var fake = new FakeTimeProvider();
var task = service.RefreshAfterDelayAsync(TimeSpan.FromMinutes(5));
fake.Advance(TimeSpan.FromMinutes(5));
await task;
```

## Частые ошибки

- Смешение `DateTime.UtcNow` и injected `TimeProvider`.
- `Task.Delay` «один раз» в том же классе.
- Ожидание, что `CancelAfter` ускорится в fake time.
- Общий `FakeTimeProvider` между параллельными тестами.

## Мнемоника

> **Inject clock, advance test** — внедри часы, в тесте «перемотай» время.
