# Awaitable pattern — GetAwaiter и INotifyCompletion

> Roadmap: `1.4.23` · Node: `1.4` — Async + ASP.NET preview · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Описать контракт **awaitable pattern**: `GetAwaiter()`, `IsCompleted`, `GetResult()`, `OnCompleted`.
- Объяснить, как компилятор **понижает** `await` до вызовов awaiter в state machine.
- Различать **INotifyCompletion** и **ICriticalNotifyCompletion**, когда используется `UnsafeOnCompleted`.
- Реализовать **custom awaitable** для yield, throttling или legacy interop.
- Узнавать **TaskAwaiter**, **ValueTaskAwaiter**, **ConfiguredTaskAwaitable** в disassembly.
- Избегать pitfalls: двойной `GetResult()`, неправильный await, capture ExecutionContext.

## Why This Matters

Каждый `await` в C# — синтаксический сахар над **фиксированным протоколом**, а не магия только для `Task`. Компилятор вызывает `GetAwaiter()`, проверяет `IsCompleted`, либо сразу `GetResult()`, либо регистрирует continuation через `OnCompleted`. Авторы библиотек иногда экспонируют **custom awaitables** (`YieldAwaitable`, `ConfiguredTaskAwaitable`, `ValueTask`) для контроля scheduling и allocations.

Middle-разработчик с этой моделью **дебажит** порядок continuations, **оборачивает** callback API в `await`-friendly surface и **читает** Roslyn state machines. В ASP.NET Core большинство await — `Task` без `SynchronizationContext`, но pattern встречается в `SemaphoreSlim.WaitAsync`, **channels**, **IAsyncEnumerable**. `GetAwaiter()` — универсальный ключ.

## Core Concepts

### Pattern по шагам

Для `await expr` компилятор генерирует примерно:

```csharp
var awaiter = expr.GetAwaiter();
if (!awaiter.IsCompleted)
{
    awaiter.OnCompleted(stateMachine.MoveNext);
    return;
}
var result = awaiter.GetResult();
```

**GetAwaiter()** — instance или extension method. **IsCompleted** — sync fast path. **GetResult()** — результат или throw (для `Task` exception marshaled сюда). **OnCompleted** — callback при pending operation.

### INotifyCompletion и ICriticalNotifyCompletion

`OnCompleted` **захватывает ExecutionContext** (`AsyncLocal`, ambient scopes). `UnsafeOnCompleted` из **ICriticalNotifyCompletion** — **без** capture, дешевле для hot path (`Task`, `ValueTask`). Library awaiters должны документировать поведение.

### Custom awaitable: yield

```csharp
public readonly struct YieldAwaitable
{
    public YieldAwaiter GetAwaiter() => new();
    public struct YieldAwaiter : ICriticalNotifyCompletion
    {
        public bool IsCompleted => false;
        public void GetResult() { }
        public void UnsafeOnCompleted(Action c) =>
            ThreadPool.UnsafeQueueUserWorkItem(_ => c(), preferLocal: false);
    }
}
```

`await Task.Yield()` — встроенный аналог: продолжение на thread pool.

### TaskAwaiter и ConfigureAwait

`ConfigureAwait(false)` → **ConfiguredTaskAwaitable** без marshal на captured SyncContext. В ASP.NET Core SyncContext нет — часто no-op для correctness, но документирует intent в library code.

SharpLab показывает `AwaitUnsafeOnCompleted` vs `AwaitOnCompleted`.

## Under the Hood

Roslyn генерирует **IAsyncStateMachine** с полями awaiter, `__state`, hoisted locals. **ValueTask** + **IValueTaskSource** — pooling без `Task` allocation на hot paths.

**GetResult()** на faulted awaiter бросает exception через **ExceptionDispatchInfo** — чистый stack vs `.Result`.

Race между `IsCompleted == false` и completion до register — correct awaiter гарантирует **exactly once** continuation.

## Syntax / Commands / API

| Member | Роль |
|--------|------|
| `GetAwaiter()` | Точка входа |
| `IsCompleted` | Sync check |
| `GetResult()` | Result / throw |
| `OnCompleted` | Continuation + context |
| `UnsafeOnCompleted` | Continuation без context |

Legacy bridge: **TaskCompletionSource** + callback → consumer `await`s `Task`.

## Examples

**Manual expansion:** `GetAwaiter` → check `IsCompleted` → `GetResult()` — то же, что `await`.

**TCS bridge:** `SetResult`/`SetException` из callback API.

**One-shot awaiters:** двойной await может throw — читай docs типа.

---

## Common Mistakes & Anti-patterns

Думать, что awaitable только `Task`. Вызывать `GetResult()` на incomplete awaiter. Sync continuation на wrong thread. Путать с **async void**.

---

## Production & Real-World Notes

Public API — **`Task`/`ValueTask`**, не exotic awaitables. **ValueTask** — когда часто sync-complete; **не await дважды** без документации. Interop — **TaskCompletionSource** standard.

## Key Takeaways

- `await` = фиксированный протокол, не reflection.
- **UnsafeOnCompleted** без ExecutionContext capture.
- **GetResult()** — точка throw для faulted operations.
- Custom awaitables для interop; app code — в основном **Task**.
- **ValueTask** — тот же pattern, строже reuse.
