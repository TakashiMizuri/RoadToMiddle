# Git Stash — краткая выжимка

> `0.1.9` · ~7 мин на повторение

## Главная идея

**`git stash`** временно сохраняет незакоммиченные изменения (WIP), чтобы working tree стал **clean** — можно switch branch, pull или сделать hotfix, а потом вернуть работу через **`pop`** или **`apply`**.

## Ключевые пункты

- Stash — **стек** (LIFO): `stash@{0}` — newest.
- **`git stash push`** (или `git stash`) — сохранить; **`-m "msg"`** — описание; **`-u`** — включить untracked files.
- **`git stash pop`** — применить и удалить из стека; **`apply`** — применить, stash остаётся.
- **`git stash list`**, **`show -p`**, **`drop`**, **`clear`** — просмотр и удаление.
- Внутри stash — **merge commits** в ref **`refs/stash`** (три parents: HEAD, index, working tree).
- Stash **только local** — на remote не попадает.

## Самые важные примеры

**Срочный hotfix:**
```bash
git stash push -u -m "login WIP"
git switch main && git pull
# ... fix ...
git switch feature/login
git stash pop
```

**Untracked не попадут без `-u`:**
```bash
git stash push      # untracked остаются
git stash push -u   # полностью clean tree
```

## «Под капотом» в одном абзаце

Git создаёт специальный **merge commit** с тремя parents и сохраняет его в **`refs/stash`**. Цепочка stash'ей — через first parent и reflog. `git stash show -p` показывает diff working tree относительно baseline (HEAD на момент stash).

## Частые ошибки

- Stash как «архив на месяцы» — легко потерять через `clear` или переустановку.
- Забыть **`-u`** — untracked files блокируют checkout.
- **`git stash clear`** без просмотра `list` — необратимо.
- **`pop`** при сильно разошедшихся ветках — конфликты как при merge.

## Мнемоника

> **Push** — положил на полку · **Pop** — взял и полку освободил · **Apply** — скопировал, полка на месте · **`-u`** — untracked тоже упаковал.
