# LINQ Take, Skip, TakeWhile, SkipWhile

> Roadmap: `1.3.26` · Node: `1.3` — C#: Advanced Topics · Depth: practice

## Learning Objectives

После урока ты сможешь:

- Резать sequences через **`Take`**, **`Skip`**, **`TakeWhile`**, **`SkipWhile`** для pagination и windowing.
- Различать slicing **по count** и **по predicate**, предсказывать edge cases на границах.
- Составлять **`Skip` + `Take`** для offset/limit pagination и понимать, когда нужен **`OrderBy`**.
- Применять operators на **`IQueryable`**, чтобы EF переводил в **`LIMIT`/`OFFSET`**.
- Избегать **unstable pagination**, **multiple enumeration** и **`Skip(n)` при большом n**.

---

## Why This Matters

Почти каждый list UI нуждается в pagination: «20 items, page 3». **`Skip((page-1)*pageSize).Take(pageSize)`** — idiomatic LINQ. Log processing использует **`TakeWhile`** до sentinel; stream parsing — **`SkipWhile`** для headers. В отличие от **`Where`**, **`TakeWhile`/`SkipWhile`** останавливают scan, когда condition «ломается» — важно для infinite или lazy sequences.

На БД **`Skip/Take`** → **`OFFSET/FETCH`** — skip миллионов rows дорог. **`TakeWhile`** на SQL может не переводиться. Semantics operators предотвращает off-by-one bugs и объясняет, почему cursor pagination заменяет offset на scale.

## Core Concepts

### Take и Skip — fixed windows

```csharp
var nums = Enumerable.Range(1, 10);
var page = nums.Skip(3).Take(4); // 4, 5, 6, 7
```

**`Skip(n)`** отбрасывает первые **n** elements, **`Take(n)`** — максимум **n**, затем stop. Оба **deferred**.

Если **`n >= Count`**, **`Skip(n)`** — empty. **`Take(n)`** на короткой sequence — все оставшиеся, без exception.

**Pagination** (1-based page):

```csharp
var items = query
    .OrderBy(x => x.Id)
    .Skip((page - 1) * pageSize)
    .Take(pageSize);
```

Без **`OrderBy`** order undefined — pages «прыгают» между requests.

### TakeWhile — prefix пока true

```csharp
var evens = nums.TakeWhile(n => n % 2 == 0); // 2, 4, 6 — stop BEFORE 7
```

Yields **while** predicate **true**. На **first false** enumeration **stops** — tail source **не читается** (side effects после точки stop не выполняются).

Отличие от **`Where`**: **`Where`** фильтрует **весь** source; **`TakeWhile`** — **prefix** с early termination.

### SkipWhile — drop prefix

```csharp
new[] { 1, 2, 3, 1 }.SkipWhile(x => x < 3); // 3, 1
```

Skips while true; когда false — yield **этот element и все следующие**, даже если они не satisfy predicate.

### Overloads с index

```csharp
source.TakeWhile((item, index) => index < 5);
```

Positional rules без отдельного счётчика.

## Syntax / Commands / API

```csharp
source.Take(count)
source.TakeWhile(predicate)
source.Skip(count)
source.SkipWhile(predicate)
// index overloads
```

Negative count → **`ArgumentOutOfRangeException`**.

## Examples

### API Pagination

```csharp
var baseQuery = db.Orders
    .OrderByDescending(o => o.CreatedAt)
    .ThenBy(o => o.Id);

var total = await baseQuery.CountAsync(ct);
var items = await baseQuery
    .Skip((page - 1) * size)
    .Take(size)
    .ToListAsync(ct);
```

**`ThenBy(o => o.Id)`** — stable tie-break.

### Sentinel

```csharp
lines.TakeWhile(line => !string.IsNullOrWhiteSpace(line));
```

### CSV header

```csharp
ParseCsv(stream)
    .SkipWhile(row => row[0].StartsWith("#"))
    .TakeWhile(row => row[0] != "END");
```

## Common Mistakes & Anti-patterns

Pagination без **`OrderBy`**. **`Where`** вместо **`TakeWhile`** для prefix. Думать, что **`SkipWhile`** фильтрует tail. Large **`Skip`** на SQL. **Multiple enumeration** между **`Count`** и page query.

---

## Production & Real-World Notes

Keyset pagination (`WHERE id > @last`) вместо huge **OFFSET**. **`TakeWhile`** на infinite streams — OK с bounded prefix.

---

## Comparison / Trade-offs

| Need | Operator |
|------|----------|
| Page N | `Skip` + `Take` + `OrderBy` |
| First N | `Take` |
| Prefix by rule | `TakeWhile` |
| Skip header | `SkipWhile` |
| All matches | `Where` |

---

## Quick Reference

| Operator | Behavior | Early stop? |
|----------|----------|-------------|
| `Take(n)` | first n | after n |
| `Skip(n)` | drop n | — |
| `TakeWhile` | prefix while true | first false |
| `SkipWhile` | skip while true | then all rest |

---

## Key Takeaways

- **`Skip` + `Take`** = offset/limit; всегда **`OrderBy`**.
- **`TakeWhile`/`SkipWhile`** — prefix, early termination.
- **`SkipWhile`**: после boundary — **весь** tail.
- **`OFFSET`** cost растёт — keyset на scale.
