# Attributes и Reflection — краткая выжимка

> `1.3.15` · ~7 мин на повторение

## Главная идея

**Attributes** — declarative metadata в IL (не код в method body). **Reflection** читает types/members и attributes at runtime. Frameworks (ASP.NET, JSON, EF) **сканируют** attributes при startup или первом использовании.

## Ключевые пункты

- Attribute class наследует `Attribute`; **`[AttributeUsage]`** задаёт targets.
- Чтение: `member.GetCustomAttribute<T>()`, `GetCustomAttributes<T>(inherit: ...)`.
- `typeof(T)`, `GetProperty`, `GetMethod`, `Invoke`, `Activator.CreateInstance`.
- Consumers: compiler, source generators, analyzers, **runtime reflection**.
- Reflection **медленнее** direct calls — кэшируй `MemberInfo`/delegates.
- AOT/trimming может **удалить** metadata — нужны preservation attributes.

## Самые важные примеры

```csharp
[MaxLength(100)]
public string Name { get; set; }

var attr = typeof(UserDto)
    .GetProperty(nameof(UserDto.Name))!
    .GetCustomAttribute<MaxLengthAttribute>();
```

## Частые ошибки

- Думать, что attribute «выполняется» сам.
- Hot path без cache reflection.
- `GetProperty("Name")` вместо `nameof`.
- Reflection в trimmed publish без `[DynamicallyAccessedMembers]`.

## Мнемоника

> **Attribute = sticky note on metadata; something must read the note.**
