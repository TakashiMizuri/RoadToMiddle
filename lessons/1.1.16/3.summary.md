# StringBuilder vs string concatenation — краткая выжимка

> `1.1.16` · ~8 мин на повторение

## Главная идея

**`string`** immutable (`1.1.4`) — каждая concat создаёт **новый объект**. В цикле **`+=`** даёт O(n²) copying. **`StringBuilder`** — mutable buffer; финальный **`ToString()`** — одна immutable string.

## Ключевые пункты

- **`s += x`** в loop — много intermediate strings + GC pressure.
- **`StringBuilder.Append`** — amortized growth; задай **`capacity`** если размер известен.
- **`string.Join`** — часто лучше для склеивания **`IEnumerable<string>`** (one result alloc).
- **`+` / Concat / `$"..."`** — норм для **2–4 частей один раз**.
- **`StringBuilder` не thread-safe**.
- Очень большой текст → **stream/write**, не megastring в памяти (LOH).
- Modern: **`String.Create`**, **`DefaultInterpolatedStringHandler`** — awareness.

## Самые важные примеры

```csharp
// плохо в scale
string s = "";
foreach (var line in lines) s += line + "\n";

// лучше
var sb = new StringBuilder(4096);
foreach (var line in lines) sb.AppendLine(line);
string result = sb.ToString();

string joined = string.Join('\n', lines);
```

## «Под капотом» в одном абзаце

Concat: compute total length → allocate one `string` → copy. Loop `+=`: повторяется каждую итерацию. StringBuilder: chunked **`char`** buffer(s), grow on fill; **`ToString()`** copies все chunks в финальную immutable string один раз.

## Частые ошибки

- StringBuilder для пары литералов.
- `ToString()` внутри цикла сборки.
- Static shared StringBuilder без lock.
- Giant string вместо streaming HTTP response.

## Мнемоника

**Few parts once → Concat/Join. Many appends in loop → StringBuilder. Already have collection → Join.**
