# Методы: parameters, return, overloading

> Roadmap: `1.1.12` · Node: `1.1` — C#: основы языка · Depth: practice

## Learning Objectives

После этого урока ты сможешь:

- Объявлять **methods** с modifiers, return type, name и **parameter list**.
- Передавать **arguments** (value types — copy; reference types — copy ссылки на тот же object).
- Возвращать values через **`return`** и использовать **`void`**, когда результата нет.
- Определять **overloaded** methods — same name, different signatures — и объяснять compile-time resolution.
- Писать **expression-bodied** methods и структурировать console apps через static helpers.

---

## Why This Matters

До сих пор samples жили в `Main` — ok для ten lines, не для hundreds. **Methods** упаковывают behavior за именем: **`CalculateTax`**, **`ParseMenuChoice`**. Primary unit reuse в class до classes/interfaces как larger units.

Каждый ASP.NET endpoint делегирует в service methods. Unit test вызывает method и assert return. Method design — names, minimal parameters, sensible returns — путь junior → middle. **Overloading** даёт одно intuitive name (`Add`, `Write`) для разных input shapes. Misuse overloads (unrelated behaviors) путает больше, чем помогает. Урок — rules C# и taste для split vs overload.

## Core Concepts

### Анатомия method

```csharp
static int Add(int a, int b)
{
    return a + b;
}
```

- **`static`** — без instance (console `Program`, utilities).
- **`int`** — **return type** (`void` если ничего).
- **`Add`** — **identifier** (PascalCase).
- **`(int a, int b)`** — **parameters**.
- Body + **`return`**.

**Signature** для overload resolution: name + parameter types (+ ref/in/out в `1.1.13`) — **не** return type alone.

### Вызов method

```csharp
int sum = Add(3, 5);
```

Arguments вычисляются **left to right**. Value types — copy bits; reference types — copy reference, mutations видны caller (`1.1.13`).

### Parameters vs arguments

**Parameters** — в definition; **arguments** — в call site. Count/types align (optional defaults — preview; full с `ref`/`out` в `1.1.13`).

```csharp
static void Greet(string name, int times = 1)
{
    for (int i = 0; i < times; i++)
        Console.WriteLine($"Hello, {name}!");
}

Greet("Ada");
Greet("Grace", 3);
```

Optional **после** required. Omit trailing defaults.

### Return values и `void`

Value methods **must return на all paths** (**CS0161** если path без return):

```csharp
static string Status(bool ok) => ok ? "OK" : "Error";
```

**Expression-bodied** (`=>`) для single expression.

**`void`** — actions без result. **`return;`** early exit (`1.1.11`).

**`null`** для reference types только когда callers expect missing; позже **`TryX(out ...)`** лучше magic null.

### Method overloading

**Overloading** — same **name**, different **signature** (count/types parameters):

```csharp
static int Area(int side) => side * side;
static int Area(int width, int height) => width * height;
static double Area(double radius) => Math.PI * radius * radius;
```

**`Area(5)`** → square. **`Area(4, 6)`** → rectangle. **`Area(2.5)`** → circle. Compiler **best match** at compile time; ambiguous → error.

Overloads — **same conceptual operation**, different data shapes. Не unrelated duties под одним name.

**Overload by return type only** — illegal:

```csharp
// CS0111
```

### Extract из `Main`

```csharp
static void Main()
{
    int[] scores = ReadScores();
    PrintReport(scores);
}

static int[] ReadScores() { /* ... */ return Array.Empty<int>(); }
static void PrintReport(int[] scores) { /* ... */ }
```

Small methods → test + name. >3–4 parameters → record/class позже.

### Naming и single responsibility

Verbs: **`GetUser`**, **`ValidateInput`**. One method — one task. "Parse **and** Save **and** Email" → split.

**Pure** methods когда можно: same inputs → same outputs, без hidden side effects — для unit tests.

## Under the Hood

Methods → **metadata** + IL **`call`** / **`callvirt`**. Overload resolution **compile time** — zero runtime pick cost. Virtual — **`callvirt`** позже.

JIT **inlines** small methods — invisible, explains cheap tiny helpers.

**Stack frames:** call pushes frame; **`return`** pops. Deep recursion → **`StackOverflowException`**.

## Syntax / Commands / API

```csharp
[modifiers] ReturnType MethodName(ParamType name, ...) { }
ReturnType Name(params) => expression;
void NoResult() { return; }
```

Modifiers: `static`, `private`, `public` — full с classes.

## Examples

### Пример 1: overloaded Print

```csharp
static void Print(int value) => Console.WriteLine($"int: {value}");
static void Print(string value) => Console.WriteLine($"string: {value}");
static void Print(decimal value) => Console.WriteLine($"decimal: {value:F2}");
```

### Пример 2: validation с early return

```csharp
static bool IsValidAge(int age)
{
    if (age < 0) return false;
    if (age > 120) return false;
    return true;
}
```

Boolean return — caller решает UI.

### Пример 3: ReadIntPrompt

```csharp
static int ReadIntPrompt(string message)
{
    while (true)
    {
        Console.Write(message);
        if (int.TryParse(Console.ReadLine(), out int n))
            return n;
        Console.WriteLine("Invalid input.");
    }
}

static void Main()
{
    int a = ReadIntPrompt("First: ");
    int b = ReadIntPrompt("Second: ");
    Console.WriteLine($"Sum = {a + b}");
}
```

Encapsulates loop + parse из prior lessons.

## Common Mistakes & Anti-patterns

**God Main** — extract.

**Overload abuse** unrelated `Process`.

**Side effects в getters**.

**`-1` as error`** vs **`Try` pattern**.

**Too many optional parameters** — unclear call sites.

## Production & Real-World Notes

Public API stabilizes signatures. Overloads extend без breaking (`Console.Write` pattern).

Async: **`Task`/`Task<T>`**, suffix **`Async`**.

`///` XML docs на public methods — IntelliSense.

## Comparison / Trade-offs

| Choice | When |
|--------|------|
| params (`1.1.13`) | Variable arg count |
| Overloads | Fixed type alternatives, same concept |
| Optional parameters | Defaults, fewer overloads |
| Separate methods | Different behaviors |

## Key Takeaways

- Methods name behavior; signature = name + param types.
- **`void`** vs value return — по need caller.
- **Overloading** — compile-time best match.
- Optional trail required params.
- Extract helpers из `Main`.
- Overload same **concept**.
