# Exercises: Lambda Expressions

> Complete after passing `4.test-yourself.md`.

## Easy

### E1. Expression vs Statement
**Goal:** Same logic twice: `Func<string, int>` parse int — expression form (TryParse one-liner) and statement form with try/catch.
**Success criteria:** Both compile and behave identically on valid/invalid input.

### E2. Static Lambda
**Goal:** `Select` with `static item => item.Id` on collection; verify no capture needed.
**Success criteria:** Compiles with `static` modifier.

## Medium

### M1. Loop Closure Fix
**Goal:** Build `List<Func<int>>` returning 0,1,2 squared from `for` loop — demonstrate bug and fix.
**Success criteria:** Before fix all return 9; after fix return 0,1,4.

### M2. Configure Callback
**Goal:** Mock `OptionsBuilder` pattern: `Configure<MyOptions>(o => o.Name = "Test")`.
**Success criteria:** Statement or expression lambda sets property.

## Hard (stretch)

### H1. Expression Tree Inspect
**Goal:** Create `Expression<Func<User,bool>>` for `u => u.Age >= 18`; print `expr.Body` structure (console).
**Success criteria:** Shows MemberExpression/BinaryExpression nodes conceptually.
