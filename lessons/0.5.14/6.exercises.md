# Exercises: iframe Security

> Complete after reading lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Accessible Video Embed
**Goal:** Write a valid YouTube-style iframe snippet.
**Steps:** Include `title`, `width`/`height`, `loading="lazy"`, minimal `allow` for fullscreen only.
**Success criteria:** Passes accessibility checklist (named iframe); no unnecessary permissions.

### E2. Sandbox Token Match
**Goal:** Map requirements to sandbox tokens.
**Steps:** For three scenarios (static preview, form-only widget, scripted calculator), list minimum `sandbox` attribute value.
**Success criteria:** No extra tokens; scripted case does NOT add `allow-same-origin` for untrusted HTML.

## Medium

### M1. Clickjacking Headers
**Goal:** Draft defense headers for `/admin`.
**Steps:** Write CSP `frame-ancestors` and optional legacy `X-Frame-Options` for ASP.NET/nginx conceptually. One sentence why both might coexist briefly.
**Success criteria:** Admin not frameable by third parties.

### M2. postMessage Protocol Sketch
**Goal:** Design safe parent ↔ iframe messaging.
**Steps:** Define message shape `{ type, payload }`, allowed origins array, and pseudo-code for parent listener with origin check.
**Success criteria:** Rejects `event.origin === '*'` trust; ignores unknown types.

## Hard (stretch)

### H1. Threat Model: User HTML Preview
**Goal:** Compare three architectures: innerHTML in app, sandboxed iframe, separate subdomain.
**Steps:** Table or prose: XSS impact, cookie access, implementation cost.
**Success criteria:** Recommends sandbox or subdomain for untrusted HTML with justification.

## Submission checklist
