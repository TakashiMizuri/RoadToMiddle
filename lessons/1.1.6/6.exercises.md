# Exercises: Value Types vs Reference Types

> Complete after reading lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Category sort
**Goal:** Classify types from `1.1.3` and basics.

Mark **V** (value) or **R** (reference): `byte`, `double`, `string`, `char[]`, `decimal`, `object` (variable holding boxed int — discuss separately in H1).

**Success criteria:** All correct with one-line justification each.

---

### E2. Predict the output
**Goal:** Copy semantics.

Write predicted output and explain stack/heap in 2 sentences:

```csharp
int x = 10;
int y = x;
y = 20;
Console.WriteLine(x);

var sb1 = new System.Text.StringBuilder("a");
var sb2 = sb1;
sb2.Append("b");
Console.WriteLine(sb1.ToString());
```

**Success criteria:** Outputs `10` and `ab`; explanation mentions value copy vs shared reference.

## Medium

### M1. Draw the diagram
**Goal:** Visual memory model.

For this code, draw **stack frames + heap objects** (ASCII or paper):

```csharp
void Main()
{
    var order = new Order { Id = 1, Total = 99.5m };
    Process(order);
}
void Process(Order o) { o.Total = 0m; }
```

Assume `Order` is a **class** with `int Id` and `decimal Total`.

**Success criteria:** Shows `order` and `o` referencing same object; `decimal` as embedded value field in heap object.

---

### M2. struct vs class behavior
**Goal:** Contrast mutation.

Implement minimal `struct Point` and `class PointRef` with `int X`. Demonstrate assignment where struct keeps independent copy and class aliases. No full solution code required in submission — describe experiment and results.

**Success criteria:** Observable difference in `X` after mutating copy.

## Hard (stretch)

### H1. Where on the heap?
**Goal:** Nuanced placement.

List **four** situations where a **value type** occupies heap memory. For each, give a one-line C# example.

**Success criteria:** Includes field of class, array elements, boxing, closure/display class (or equivalent).
