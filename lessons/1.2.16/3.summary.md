# Object.Equals, GetHashCode, IEquatable — краткая выжимка

> `1.2.16` · ~10 мин на повторение

## Главная идея

**Reference equality** — один и тот же объект в памяти; **value equality** — одно логическое значение. Hash-коллекции (`Dictionary`, `HashSet`) требуют согласованной пары **`Equals` + `GetHashCode`**: если объекты равны, hash codes **обязаны** совпадать.

## Ключевые пункты

- Default для **class** — сравнение по **ссылке**; для **struct** — по полям (с ловушками для полей-ссылок).
- Переопределяй **`Equals` и `GetHashCode` вместе** — иначе `ContainsKey` / `Contains` врут.
- **`IEquatable<T>`** — typed `Equals(T)` без boxing; критично для struct в hot paths.
- **`HashCode.Combine(a, b, c)`** — безопасное смешивание полей в hash.
- **`==` на class** — reference equality, пока не перегрузишь; выровняй с `Equals`, если нужна value semantics.
- **Records** (`1.2.13`) генерируют structural equality автоматически.
- **Ключи Dictionary** — immutable; мутация после insert ломает lookup.
- **`IEqualityComparer<T>`** — альтернативные правила без правки типа (`StringComparer.OrdinalIgnoreCase`).

## Самые важные примеры

```csharp
public sealed class Customer : IEquatable<Customer>
{
    public Guid Id { get; }
    public override int GetHashCode() => Id.GetHashCode();
    public bool Equals(Customer? other) =>
        other is not null && Id == other.Id;
    public override bool Equals(object? obj) => Equals(obj as Customer);
}
```

```csharp
// Broken: Equals по Id, GetHashCode по identity → ContainsKey(false)
d[new BadKey(1)] = 1;
d.ContainsKey(new BadKey(1)); // false
```

## «Под капотом» в одном абзаце

`Dictionary` вычисляет bucket по `GetHashCode`, затем сравнивает ключи через `Equals` в цепочке. Virtual `Equals` даёт polymorphic dispatch. `EqualityComparer<T>.Default` предпочитает `IEquatable<T>`. Struct без `IEquatable<T>` боксируется при `Equals(object)`. JIT инлайнит маленькие sealed overrides.

## Частые ошибки

- `Equals` без `GetHashCode`.
- Mutable object как ключ словаря.
- `==` и `Equals` расходятся на class.
- Struct с `byte[]` полем — сравнение ссылок на массив, не содержимого.
- `GetHashCode()` как persistent ID — нестабильно.

## Мнемоника

**Equal objects → equal hashes. Override in pairs. Keys don't change. Structs implement IEquatable.**
