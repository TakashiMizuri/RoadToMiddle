# Exercises: break, continue, return

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. First match search
**Goal:** Use `break`.

**Tasks:**
In `int[] data`, find first value > 100 and print it; if none, print `"None"`. Use **`for`** + **`break`**.

**Success criteria:** Stops at first match; does not print later matches.

---

### E2. Skip blanks
**Goal:** Use `continue`.

**Tasks:**
`string[] lines` with empty and whitespace-only entries. Print non-empty trimmed lines with **`foreach`** + **`continue`**.

**Success criteria:** Empty lines skipped; loop completes all items.

---

## Medium

### M1. Guard clause method
**Goal:** Early `return`.

**Tasks:**
Write `static string FormatPrice(decimal? price)`:
- null → `"N/A"`
- negative → `"Invalid"`
- else → `"$" + price with 2 decimals` (use `:F2` or similar)

**Success criteria:** No deep nesting; three independent guards.

---

### M2. REPL calculator
**Goal:** `break`, `continue`, `return` together.

**Tasks:**
Loop: read `a op b` or `quit`. Invalid format → **`continue`**. `quit` → **`break`**. Implement +, -, *, /; division by zero message, no crash.

**Success criteria:** Clean exit; bad lines don't kill app.

---

### M3. Nested grid search
**Goal:** Understand inner `break` limit.

**Tasks:**
2D `int[,]` grid, find first cell equal to target. Print coordinates or `-1,-1`. Then refactor: extract **`FindFirst`** method that **`return`s** tuple when found.

**Success criteria:** Explain why inner-only `break` wasn't enough for outer exit; method version clearer.

---

## Hard (stretch)

### H1. Prime list with continue
**Goal:** Filter in loop.

**Tasks:**
Print primes 2..100. Inner trial division: if divisor found, **`continue`** outer candidate loop (use labeled logic via nested loops or helper `IsPrime` with **`return bool`**).

**Success criteria:** No false primes; prefer helper + return over messy flags.

---
