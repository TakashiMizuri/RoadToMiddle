# Exception handling в async — краткая выжимка

> `1.4.19` · ~7 мин на повторение

## Главная идея

**`await`** на faulted task бросает **inner exception** (обычно без **`AggregateException`**). **`Wait()`/`.Result`** — wrap. **`Task.WhenAll`** на **`await`** — часто **первый** fault; остальные проверяй на tasks.

## Ключевые пункты

- **`try/catch/finally`** работает через suspend points.
- **`throw;`** — сохранить stack; **`throw ex;`** — reset.
- **`async void`** — unhandled → crash, не для business logic.
- Fire-and-forget без await → **`UnobservedTaskException`**.
- ASP.NET: unhandled из **`async Task` action** → exception middleware.

## Пример WhenAll

```csharp
try { await Task.WhenAll(t1, t2); }
catch (Exception ex) { /* often only first */ }
// t2.Exception still available
```

## Частые ошибки

- Ждать **`AggregateException`** на **`await WhenAll`**.
- Swallow в **`catch`** после await.
- **`.Result`** в ASP.NET pipeline.
