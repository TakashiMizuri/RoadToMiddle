# Exercises: Implicit vs explicit interface implementation

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Implicit logger
**Goal:** Standard implicit impl.

**Tasks:**
**`FileLogger : ILogger`** with **`public void Log(string msg)`** writing to file. Call via **`FileLogger`** and **`ILogger`**.

**Success criteria:** Both call paths work without cast.

---

### E2. Explicit only
**Goal:** Hidden implementation.

**Tasks:**
**`Counter : IIncrementable`** with **`void IIncrementable.Increment()`** — no public **`Increment`**. Demo compile error on **`c.Increment()`** and success on **`((IIncrementable)c).Increment()`**.

**Success criteria:** Document CS1061 on direct call.

---

## Medium

### M1. Dual Flush
**Goal:** Collision resolution.

**Tasks:**
**`SessionStore : IPersistent, ICache`** — **`Flush`** writes disk vs clears RAM via **explicit** implementations. No public **`Flush`**.

**Success criteria:** Two interface casts, two behaviors.

---

### M2. Comparable customer
**Goal:** BCL-style explicit.

**Tasks:**
**`Customer : IComparable<Customer>`** with **`int Id`**. **`CompareTo`** only explicit. Sort **`List<Customer>`** using **`OrderBy`** or **`List.Sort()`** (requires cast/comparer awareness).

**Success criteria:** **`CompareTo`** not in public IntelliSense on **`Customer`**.

---

### M3. IEnumerable pair
**Goal:** Generic + non-generic.

**Tasks:**
**`IntBag : IEnumerable<int>`** with **`yield return`**. Implement non-generic **`IEnumerable.GetEnumerator()`** explicitly delegating to generic.

**Success criteria:** **`foreach`** works; explain which **`GetEnumerator`** is public vs explicit.

---

## Hard (stretch)

### H1. Implicit vs explicit dispatch
**Goal:** Predict output with both.

**Tasks:**
Class with **`public void M()`** and **`void IFoo.M()`**. Write table: call **`obj.M()`**, **`((IFoo)obj).M()`** on same instance.

**Success criteria:** Table matches runtime behavior.
