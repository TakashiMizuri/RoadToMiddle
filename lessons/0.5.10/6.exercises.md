# Exercises: DOM Tree

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Count node types
**Goal:** See text nodes in DevTools.

Create `dom-lab.html`:

```html
<div id="box">
  <p>Hello</p>
</div>
```

In console log `box.childNodes.length`, `box.children.length`, and `nodeType` of `box.firstChild`.

**Success criteria:** Explain difference in counts; identify TEXT vs ELEMENT nodes.

---

### E2. Safe insertion
**Goal:** Avoid XSS.

Given `const name = '<img src=x onerror=alert(1)>'`, write two snippets: one using `innerHTML`, one `textContent`. Predict DOM child structure for each **without** running alert in production — use comment or isolated iframe lab.

**Success criteria:** innerHTML creates element nodes; textContent one text node; articulates XSS difference.

---

## Medium

### M1. Build list imperatively
**Goal:** createElement workflow.

Without innerHTML, build `<ul id="tasks">` with three `<li>` from array `['Buy milk','Code','Review PR']` using `createElement`, `createTextNode` or `textContent`, and `append`.

Log `tasks.children.length` and verify in Elements panel.

**Success criteria:** Three LI element nodes; no innerHTML; correct structure.

---

### M2. Fragment batch
**Goal:** DocumentFragment pattern.

Same list as M1 but append 20 items via `DocumentFragment`, single `ul.append(frag)`.

Optional: Performance panel — qualitative note on reflows vs 20 separate appends.

**Success criteria:** Working list; explains why fragment batches inserts.

---

## Hard (stretch)

### H1. Hydration mismatch reproduction
**Goal:** Connect DOM text nodes to React SSR.

In a minimal React SSR example or pseudo-code, describe a scenario where server renders `<p>Hello</p>` and client renders `<p> Hello </p>` with extra spaces as separate text nodes. What warning appears and why?

**Success criteria:** Names text node mismatch; mentions hydration; suggests normalize whitespace or suppressHydrationWarning trade-off.
