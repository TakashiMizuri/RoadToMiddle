# LINQ OrderBy, ThenBy — Stable Sort

> Roadmap: `1.3.28` · Node: `1.3` — C#: Advanced Topics · Depth: practice

## Learning Objectives

After this lesson you will be able to:

- Sort sequences with **`OrderBy`**, **`OrderByDescending`**, **`ThenBy`**, **`ThenByDescending`**.
- Explain **stable sort** and why tie-breaking matters for pagination and UI.
- Use **`IComparer<T>`** and **`Comparer<T>.Create`** for custom ordering rules.
- Sort on **`IQueryable`** so EF translates to SQL **`ORDER BY`** with multiple columns.
- Choose between **sort in query** vs **sort in memory** (`AsEnumerable` pitfalls).
- Avoid **`OrderBy` after non-translatable operations** and redundant sorts.

---

## Why This Matters

Users expect consistent lists: same search → same order every time. **`OrderBy`** defines primary sort key; **`ThenBy`** breaks ties — "sort by Department, then by Name". Without **`ThenBy`**, equal keys have **undefined relative order** in SQL and unstable UX across pages.

LINQ sorting is **deferred** on **`IEnumerable`** until enumeration; on **`IQueryable`** it becomes **`ORDER BY`** — push sort to the server when possible. Stable sort guarantees elements with equal keys keep **input order** — important when you sorted by priority but want FIFO within same priority. Understanding stability explains why **`OrderBy(x => x).ThenBy((x,i) => i)`** patterns preserve original order as tie-breaker.

## Core Concepts

### OrderBy — Primary Key

```csharp
var sorted = employees.OrderBy(e => e.LastName);
```

Returns **`IOrderedEnumerable<T>`** — not plain **`IEnumerable`**. You **must** chain **`ThenBy`** on the **ordered** type:

```csharp
employees
    .OrderBy(e => e.Department)
    .ThenBy(e => e.LastName)
    .ThenByDescending(e => e.HireDate);
```

**Wrong:** second **`OrderBy`** replaces first sort entirely:

```csharp
// BAD — only sorts by LastName, Department sort lost:
employees.OrderBy(e => e.Department).OrderBy(e => e.LastName);
```

Use **`ThenBy`** for secondary keys.

### Stable Sort Semantics

LINQ to Objects uses a **stable** sorting algorithm (merge sort based implementation). Elements equal under the comparer **retain relative order** from the input sequence.

Example:

```csharp
var items = new[] {
    new { P = 1, Name = "A" },
    new { P = 2, Name = "B" },
    new { P = 1, Name = "C" }
};
var s = items.OrderBy(x => x.P).ToList();
// P=1: A before C (original order preserved among P=1)
```

For explicit tie-break when keys collide frequently, add **`ThenBy`** on a unique field (Id) or original index.

### Descending

**`OrderByDescending`** / **`ThenByDescending`** invert comparison for that key level only.

### Custom Comparer

```csharp
var byLength = names.OrderBy(n => n, Comparer<string>.Create((a, b) => a.Length.CompareTo(b.Length)));
```

Or pass comparer to **`OrderBy(key, comparer)`** for culture-aware strings:

```csharp
.OrderBy(s => s, StringComparer.OrdinalIgnoreCase)
```

### IQueryable and EF Core

```csharp
var q = db.Products
    .OrderBy(p => p.CategoryId)
    .ThenBy(p => p.Name);
// SQL: ORDER BY CategoryId, Name
```

Sorting before **`Skip/Take`** ensures stable pagination. **`OrderBy` on unindexed column** — table scan + sort; add indexes matching common sort keys.

Client-side sort trap:

```csharp
db.Products.AsEnumerable().OrderBy(p => p.Name.Localized()); // pulls all rows!
```

Anything after **`AsEnumerable()`** may force client evaluation.

## Syntax / Commands / API

```csharp
source.OrderBy(keySelector)
source.OrderBy(keySelector, comparer)
source.OrderByDescending(keySelector)
ordered.ThenBy(keySelector)
ordered.ThenByDescending(keySelector)

// Reverse entire sequence order (not key-based sort):
source.Reverse()
```

**`IOrderedQueryable<T>`** for queryable chains.

## Examples

### Leaderboard

```csharp
var top = scores
    .OrderByDescending(s => s.Points)
    .ThenBy(s => s.SubmittedAt)  // earlier submission wins tie
    .Take(10);
```

### API List with Stable Pages

```csharp
return await db.Items
    .Where(i => i.IsActive)
    .OrderBy(i => i.SortOrder)
    .ThenBy(i => i.Id)
    .Skip(offset)
    .Take(limit)
    .ToListAsync(ct);
```

**`ThenBy(i => i.Id)`** — deterministic last resort.

### Preserve Input Order as Secondary

```csharp
var ranked = items
    .Select((item, index) => new { item, index })
    .OrderBy(x => x.item.Priority)
    .ThenBy(x => x.index)
    .Select(x => x.item);
```

Uses stable sort + original index for FIFO within priority band.

## Common Mistakes & Anti-patterns

**Chaining `OrderBy` instead of `ThenBy`.** Replaces sort.

**Missing tie-breaker for pagination.** Duplicate sort keys → rows move between pages.

**Sorting after materialization unintentionally.** **`ToList()`** then sort OK in memory; **`AsEnumerable()`** early pulls data.

**Assuming SQL sort stability without ORDER BY uniqueness.** Always add unique tie-break column in SQL paging.

**Nullable keys** — nulls first/last depends on provider; specify explicitly if needed.

---

## Production & Real-World Notes

Index **`(CategoryId, Name)`** for common **`OrderBy` + `ThenBy`** patterns. PostgreSQL **`NULLS FIRST/LAST`** — EF Core 8+ has better null ordering control.

For dynamic UI sort columns, build **`OrderBy`** with caution (reflection/`System.Linq.Dynamic.Core`) — validate allowed columns to prevent injection.

**`Reverse()`** is O(1) lazy on lists with known count in some cases but generally materializes — read docs for your source type.

## Comparison / Trade-offs

| Need | Approach |
|------|----------|
| Single key | `OrderBy` |
| Multi-key | `OrderBy` + `ThenBy*` |
| Flip all | `Reverse()` after sort, or Descending variants |
| Server-side | Keep on `IQueryable` |
| Complex compare | `IComparer` |

## Quick Reference

| Operator | Role |
|----------|------|
| `OrderBy` | primary ascending |
| `OrderByDescending` | primary descending |
| `ThenBy` | secondary ascending |
| `ThenByDescending` | secondary descending |

**Stable sort** in LINQ to Objects — equal keys keep input order. Add **`ThenBy(Id)`** for deterministic SQL paging.

## Key Takeaways

- **`ThenBy`** extends sort; second **`OrderBy`** resets.
- Stable sort preserves order among equal keys.
- Always tie-break for pagination (`Id`, timestamp).
- EF: translate to **`ORDER BY`** — sort before **`Skip/Take`**.
- Custom comparers for culture and complex rules.
