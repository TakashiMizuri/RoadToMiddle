## Answers

### 1. [Recall]
**Answer:** Если можно использовать native HTML element с нужной semantics и behavior — не меняй его и не дублируй ARIA.
**Why:** Native controls include role, keyboard, state for free.

### 2. [Recall]
**Answer:** **Enter** и **Space** (activation).
**Why:** HTML button activation keys.

### 3. [Recall]
**Answer:** Элемент **не в sequential tab order**, но может получить focus через `.focus()`.
**Why:** tabindex -1 removes from Tab navigation only.

### 4. [Understanding]
**Answer:** Нужно самому implement Enter/Space, disabled, focus styling; легко пропустить keyboard и state; хуже consistency с OS/AT.
**Why:** First rule of ARIA — avoid redundant inferior replacements.

### 5. [Understanding]
**Answer:** **DOM order** (unless positive tabindex reshuffles); visual reorder CSS alone не меняет Tab.
**Why:** Sequential focus navigation follows DOM + tabindex rules.

### 6. [Understanding]
**Answer:** **Restore context** — пользователь keyboard знает, где оказался после overlay; иначе focus «теряется» в body.
**Why:** Focus management best practice / WCAG dialog pattern.

### 7. [Application]
**Answer:** `aria-expanded="true|false"`, `aria-controls="details"` (id panel), button type=button; panel `id="details"`.
**Why:** APG disclosure pattern links control to controlled region.

### 8. [Application]
**Answer:** **`aria-live="polite"`** на container (или `role="status"`), записать текст toast; optional `role="status"` implicit live.
**Why:** Live region announces without moving focus.

### 9. [Analysis]
**Answer:** **Positive tabindex** — custom tab order, logo before search, хрупко и confusing; anti-pattern.
**Why:** Avoid tabindex > 0.

### 10. [Analysis]
**Answer:** Focused input **hidden from AT** — serious violation; user types «в никуда» для screen reader.
**Why:** Never aria-hidden ancestor of focused element.

### 11. [Understanding]
**Answer:** **`navigation`** (landmark).
**Why:** HTML implicit role mapping for nav element.

### 12. [Understanding]
**Answer:** **`aria-label`** — string name directly on element; **`aria-labelledby`** — id другого элемента(ов), чей text becomes name.
**Why:** Name computation sources.

### 13. [Application]
**Answer:** Один tab stop на widget group; arrow keys move **focus** между tabs; siblings tabindex -1 except active 0.
**Why:** Roving tabindex APG pattern.

### 14. [Recall]
**Answer:** **2.1.1 Keyboard** (Level A).
**Why:** WCAG keyboard access criterion.

### 15. [Understanding]
**Answer:** **Срочные** ошибки/опасность — assertive/alert; polite — некритичные status updates.
**Why:** Interrupt level for AT announcements.

### 16. [Analysis]
**Answer:** **Плохо** — лучше `<a href>` для navigation; button role link confuses expected keys and semantics.
**Why:** Use correct native element for behavior.
