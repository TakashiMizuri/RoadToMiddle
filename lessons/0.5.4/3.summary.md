# Списки ul, ol, dl — краткая выжимка

> `0.5.4` · ~8 мин на повторение

## Главная идея

**ul** — набор без значения порядка; **ol** — шаги/ранг с значением порядка; **dl** — термин + описание. Пункты только в **`<li>`** (для ul/ol); вложенный список — **внутри родительского li**.

## Ключевые пункты

- Nav: `<nav><ul><li><a>…`
- Procedure: `<ol><li>Step…`
- Glossary/metadata: `<dl><dt>…<dd>…`
- Не `<a>` напрямую под `<ul>`/`<ol>`.
- `start`, `reversed`, `type` — смысл документа, не замена CSS.
- ol vs ul: перестановка items меняет meaning только у ol.

## Самые важные примеры

```html
<ol>
  <li>Install
    <ul>
      <li>npm install</li>
    </ul>
  </li>
</ol>

<dl>
  <dt>API</dt>
  <dd>Application Programming Interface</dd>
</dl>
```

## Частые ошибки

- ul sibling внутри ul без li-обёртки.
- ol для меню без semantic order.
- dl как две колонки div.

## Мнемоника

> **U**nordered = **U**l; **O**rdered = **O**l; **D**efinitions = **D**l.
