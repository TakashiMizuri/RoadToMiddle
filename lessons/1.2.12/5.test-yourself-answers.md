## Answers

### 1. [Recall]
**Answer:** Static field — **одна копия на тип**; instance field — **отдельная копия в каждом объекте**.
**Why:** Storage привязан к type handle, не к object header.

### 2. [Recall]
**Answer:** Нельзя создавать экземпляры, нельзя наследовать; только static members; неявно `abstract sealed`.
**Why:** Компилятор запрещает instance API.

### 3. [Recall]
**Answer:** **Один раз** перед первым использованием типа (static или instance member), в thread-safe режиме CLR.
**Why:** `.cctor` триггерится first touch типа.

### 4. [Understanding]
**Answer:** У static method **нет `this`** — нет конкретного объекта, чьи instance fields читать.
**Why:** Static dispatch не передаёт instance pointer.

### 5. [Understanding]
**Answer:** Concurrent requests **перезаписывают** одно общее поле — wrong user id, data leaks, race conditions.
**Why:** Request scope должен быть scoped/`HttpContext`, не global static.

### 6. [Understanding]
**Answer:** **`const`** — compile-time constant, inlined, только примитивы/string; **`static readonly`** — runtime init один раз, может быть любым типом.
**Why:** `const` требует recompile при смене значения.

### 7. [Application]
**Answer:** Одно значение **`5`** для всего типа `Counter`; экземпляры `a` и `b` не имеют своего `Total`.
**Why:** Static не дублируется per instance.

### 8. [Application]
**Answer:** **Разное** хранилище — каждый closed generic `TypeBox<int>` и `TypeBox<string>` — отдельный тип с own static fields.
**Why:** CLR создаёт per-instantiation generic type.

### 9. [Application]
**Answer:** Ввести **`IClock`** (или `TimeProvider` в .NET 8+), inject в сервисы; в тестах — fake с фиксированным временем.
**Why:** Instance dependency mockable; static `DateTime.UtcNow` — нет.

### 10. [Analysis]
**Answer:** Тип **faulted**; повторные обращения → **`TypeInitializationException`** с inner exception из `.cctor`.
**Why:** CLR не повторяет failed type init.

### 11. [Analysis]
**Answer:** **B** неверно — static methods **не** override virtual instance methods (hiding через `new` — плохая практика).
**Common mistake:** Ожидать полиморфизм у static.

### 12. [Understanding]
**Answer:** `Math` — **pure stateless functions** без identity; singleton instance добавил бы шум без выгоды.
**Why:** No per-object state → static class идиоматичен.

### 13. [Application]
**Answer:** **Нет** — static class implicitly sealed, наследование запрещено компилятором.
**Why:** `static class` = `abstract sealed`.

### 14. [Analysis]
**Answer:** **`Dictionary<>` не thread-safe** — concurrent read/write → corruption или exceptions; нужен `ConcurrentDictionary` или instance scoped cache.
**Why:** Shared mutable static без синхронизации.

### 15. [Application]
**Answer:** **Pure helpers** без IO/side effects (`Guard.AgainstNull`, string validators), когда не нужен mock и нет lifecycle в DI.
**Why:** Меньше ceremony; нет скрытого mutable state.
