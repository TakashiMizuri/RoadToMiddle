# Exercises: Polly with HttpClient

> Complete after lectures and test.

## Easy

### E1. AddStandardResilienceHandler
**Goal:** Register catalog client with default standard resilience (.NET 8+).
**Steps:** Add package; AddHttpClient + AddStandardResilienceHandler; call flaky local endpoint.
**Success criteria:** Observe retry in logs or downstream hit count > 1 on 503.

### E2. Transient vs permanent
**Goal:** Document which status codes your team retries for GET.
**Steps:** Write table: retry / no retry / case-by-case with rationale.
**Success criteria:** At least 8 status codes classified.

## Medium

### M1. Custom retry predicate
**Goal:** Retry only 503 and HttpRequestException, max 2, jitter backoff.
**Steps:** Configure StandardResilienceHandler options or custom ResilienceHandler.
**Success criteria:** 404 fails without retry (verify hit count = 1).

### M2. Breaker observation
**Goal:** Open circuit after repeated failures.
**Steps:** Local always-500 endpoint; tune SamplingDuration/FailureRatio; call until BrokenCircuitException.
**Success criteria:** Logged transition to fast-fail without HTTP call when open.

## Hard (stretch)

### H1. Parallelism × retry budget
**Goal:** Quantify load multiplier.
**Steps:** ForEachAsync 50 calls with retry 3 vs parallelism 5; compare downstream request count on failure.
**Success criteria:** Formula `parallelism × (1 + retries)` validated ± edge cases.

### H2. Total timeout vs RequestAborted
**Goal:** Cancel Polly pipeline when client disconnects.
**Steps:** Forward CancellationToken from Minimal API through typed client call.
**Success criteria:** In-flight retries stop after token cancel.
