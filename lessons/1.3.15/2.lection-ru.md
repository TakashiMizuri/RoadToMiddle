# Attributes и Reflection (GetCustomAttribute)

> Roadmap: `1.3.15` · Узел: `1.3` — C#: продвинутые темы · Глубина: понимание

## Learning Objectives

После этого урока ты сможешь:

- Объяснить, что такое **attributes** — metadata на assembly, types, members, parameters.
- Применять типичные BCL attributes (`[Obsolete]`, validation, ASP.NET routing).
- Читать attributes at runtime через **`GetCustomAttribute`** / **`GetCustomAttributes`**.
- Различать **compile-time** consumers (analyzers, source generators) и **runtime** reflection.
- Понимать **стоимость** reflection и когда кэшировать `MemberInfo`.
- Использовать basics reflection: `typeof`, `GetMethod`, `Invoke` — с оговорками по безопасности.

---

## Why This Matters

Большая часть .NET — **declarative**: аннотируешь код, frameworks интерпретируют. ASP.NET Core мапит `[HttpGet]` на routes; EF — `[Column]`; System.Text.Json — `[JsonPropertyName]`; validation — `[Required]`. Это не магия — кто-то **читает metadata** at compile time или runtime.

**Attributes** — языковая feature для такой metadata. **Reflection** — API для inspect types и чтения attributes при выполнении. Middle dev настраивает libraries через attributes ежедневно; debug «почему endpoint 404» часто = trace attribute discovery.

Тяжёлую reflection в business code пишут редко, но без понимания нельзя корректно пользоваться frameworks и легко уйти в **stringly-typed** API.

## Core Concepts

### Что такое attribute?

Attribute — instance класса, наследующего **`System.Attribute`**, в квадратных скобках **перед** target:

```csharp
[Obsolete("Use NewApi instead")]
public void OldMethod() { }
```

Targets: **assembly**, **class**, **method**, **property**, **parameter**, **return value** и др. Компилятор сохраняет данные в **assembly metadata** (IL tables), не как executable statements в теле метода.

На attribute class обычно **`[AttributeUsage(...)]`** — где можно вешать и допускается ли несколько экземпляров.

### Свой custom attribute

```csharp
[AttributeUsage(AttributeTargets.Property, AllowMultiple = false)]
public sealed class MaxLengthAttribute : Attribute
{
    public int Length { get; }
    public MaxLengthAttribute(int length) => Length = length;
}
```

Usage:

```csharp
public class UserDto
{
    [MaxLength(100)]
    public string Name { get; set; } = "";
}
```

Свойства attribute class — **простые** значения (constants, typeof, arrays) — компилятор вшивает constructor args и named properties в metadata.

### Чтение: GetCustomAttribute

Runtime: получаешь **`Type`**, **`MethodInfo`**, **`PropertyInfo`**, затем:

```csharp
var prop = typeof(UserDto).GetProperty(nameof(UserDto.Name))!;
var attr = prop.GetCustomAttribute<MaxLengthAttribute>();
int max = attr?.Length ?? int.MaxValue;
```

APIs:

- `GetCustomAttribute<T>()` — один или null.
- `GetCustomAttributes<T>()` — коллекция.
- `inherit: true/false` — обход inheritance chain.

`CustomAttributeData` — inspect без создания instance attribute (serializers, DI scanners).

### Basics reflection

**Reflection** inspect types и members at runtime:

```csharp
Type t = typeof(UserDto);
object obj = Activator.CreateInstance(t)!;
PropertyInfo? nameProp = t.GetProperty("Name");
nameProp?.SetValue(obj, "Ada");
```

`MethodInfo.Invoke` вызывает methods; `GetMethods`, `GetProperties` перечисляют members. Generics: `MakeGenericType`, `GetGenericArguments`.

Reflection **мощный и медленный** vs direct calls — JIT плохо inline `Invoke`. Frameworks кэшируют `MemberInfo` и delegates (`CreateDelegate`).

### Compile-time vs runtime consumers

| Consumer | When | Example |
|----------|------|---------|
| Compiler | Build | `[Conditional("DEBUG")]` |
| Source generator | Build | `[JsonSerializable]` |
| Analyzer | IDE/build | hints для `[Required]` |
| Runtime reflection | Execute | ASP.NET endpoint discovery |

Modern stack **предпочитает source generators** для AOT и performance (Native AOT сильно режет reflection).

### Типичные attributes

- **`[Obsolete]`** — deprecated API.
- **`[Flags]`** на enum.
- **ASP.NET:** `[ApiController]`, `[Route]`, `[Authorize]`, `[FromBody]`.
- **Validation:** `[Required]`, `[Range]`.
- **EF Core:** `[Key]`, `[Table]` — в новом коде часто Fluent API.

Понимание **кто** читает attribute предотвращает «attribute ничего не делает».

## Under the Hood

Attributes → **`.custom`** в IL на metadata tokens. Constructor args и named property values encoded отдельно.

`GetCustomAttribute<T>()` создаёт instance через constructor (если не `CustomAttributeData` only). **`[AttributeUsage(Inherited = true)]`** на attribute class управляет наследованием на derived types (тонкие правила для properties vs methods).

Reflection использует cached **`RuntimeType`** per `AppDomain`. **`BindingFlags`** — public/nonpublic, static/instance, declared-only.

Trimming/AOT: targets reflection могут быть **удалены** — `[DynamicallyAccessedMembers]`, `RequiresUnreferencedCode`.

## Examples

### Mini validation scanner

```csharp
public static List<string> Validate(object dto)
{
    var errors = new List<string>();
    foreach (var prop in dto.GetType().GetProperties())
    {
        var max = prop.GetCustomAttribute<MaxLengthAttribute>();
        if (max is null) continue;
        var val = prop.GetValue(dto) as string;
        if (val?.Length > max.Length)
            errors.Add($"{prop.Name} too long");
    }
    return errors;
}
```

Упрощённый аналог DataAnnotations.

### Route from attribute (conceptual)

Framework at startup reflects controller methods, находит `HttpGetAttribute`, строит route table.

## Common Mistakes & Anti-patterns

**Attributes сами по себе не выполняют код** — это data, пока reader не прочитает.

**Неверный `AttributeTargets`** — compile error или ignore.

**Reflection на hot path без cache**.

**Magic strings** в `GetProperty("name")` — используй `nameof`.

**Reflection в trimmed/AOT** без preservation — runtime failures.

---

## Production & Real-World Notes

Explicit registration (Minimal APIs) когда reflection overhead критичен. Attributes — для больших controller surfaces и conventions.

Testing: reflection для private fields в test assembly — OK; в production — нет.

## Comparison / Trade-offs

| Approach | Pros | Cons |
|----------|------|------|
| Attributes | Declarative, co-located | Hidden behavior, cost |
| Fluent code | Explicit | Verbose |
| Source generators | Fast, AOT | Build complexity |

## Key Takeaways

- Attributes = **metadata** в IL.
- **`GetCustomAttribute<T>()`** на `MemberInfo`.
- Frameworks consume at startup / first use.
- Reflection — cache при повторении.
- Generators и AOT снижают reliance на runtime reflection.
