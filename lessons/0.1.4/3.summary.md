# Структура `.git/` — краткая выжимка

> `0.1.4` · ~10 мин на повторение

## Главная идея

Репозиторий Git — это каталог **`.git/`**, а не файлы проекта. History хранится как **immutable objects** (blob, tree, commit, tag), адресуемые по **hash**. Ветки и **HEAD** — **mutable указатели** (refs) на commit'ы. Команды `git add`/`commit` создают objects и двигают refs.

## Ключевые пункты

- **`.git/objects/`** — content-addressable storage: hash = identity содержимого.
- **blob** — content файла; **tree** — «директория» (имя → hash); **commit** — tree + parents + metadata; **tag** — annotated метка.
- **`.git/index`** — staging area (mutable), связывает paths с blob hash'ами.
- **`refs/heads/`** — local branches; **`refs/tags/`** — tags; **`refs/remotes/`** — remote-tracking branches.
- **HEAD** — обычно `ref: refs/heads/main` (symbolic ref), не сам hash.
- **Loose objects** — по файлу на object; **packfiles** — сжатые группы после `git gc`.
- Objects неизменяемы; при новом commit создаются новые objects, ref ветки сдвигается.

## Самые важные примеры

**Цепочка одного commit:**
```
commit → tree → blobs/subtrees → bytes файлов
       → parent(s)
```

**`git add` + `git commit` (упрощённо):**
```bash
git add README.md    # blob в objects/ + запись в index
git commit -m "..."  # tree + commit object; refs/heads/main → новый hash
```

**Проверка внутренностей:**
```bash
cat .git/HEAD                    # ref: refs/heads/main
git cat-file -p HEAD             # содержимое commit object
git cat-file -t abc1234          # тип: commit | tree | blob | tag
```

## «Под капотом» в одном абзаце

При `git add` Git читает bytes файла, строит header `blob <size>\0`, считает SHA-1, пишет zlib-сжатый файл в `.git/objects/xx/yyyy...` и обновляет **index**. При `git commit` из index строятся **tree** objects (снизу вверх), затем **commit** object с полями `tree`, `parent`, author/committer, message. Git обновляет файл `refs/heads/<branch>` на новый hash. **HEAD** указывает на ref ветки, не на commit напрямую. Со временем `git gc` упаковывает loose objects в **packfiles** с delta-сжатием.

## Частые ошибки

- Редактировать `.git/objects/` вручную → corruption.
- Думать, что ветка «содержит файлы» — это 40-байтный указатель на commit.
- Путать index (staging) с object store.
- Commit'ить в **detached HEAD** без создания ветки → commit'ы потеряются при checkout.

## Мнемоника

> **Objects** = неизменяемые snapshots по hash; **Refs** = имена, которые двигаются; **HEAD** = «на какой ref я смотрю».
