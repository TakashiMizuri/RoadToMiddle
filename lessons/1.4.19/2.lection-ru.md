# Exception handling в async коде

> Roadmap: `1.4.19` · Node: `1.4` — C# async · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Объяснить propagation исключений через **`await`** и state machine.
- Использовать **`try/catch/finally`** вокруг **`await`** корректно.
- Разворачивать **`AggregateException`** от **`Wait()`**, **`.Result`**, **`WhenAll`**.
- Обрабатывать множественные fault в **`Task.WhenAll`**.
- Сопоставить **`async void`** и **`async Task`** (**`1.4.13`**).
- Применять **`throw;`**, **`ExceptionDispatchInfo`**, logging в async pipeline.

---

## Why This Matters

Async **выглядит** как sync **`try/catch`**, но failure modes другие: exception на **`Task.Exception`**, wrap в **`AggregateException`**, crash через **`async void`**. **`WhenAll`** с тремя fault — на **`await`** часто **одно** исключение.

Production: "exception проглочен после await", неверный stack после **`throw ex`**, 500 без полной картины ошибок. Middle dev должен знать **где** exception на каждом suspend point.

---

## Core Concepts

### await и исключения

Faulted task при **`await`** бросает **inner exception** (обычно **без** **`AggregateException`**):

```csharp
try { await FaultedTask(); }
catch (InvalidOperationException ex) { /* OK */ }
```

### try/catch/finally с await

**`finally`** после завершения **`await`** в **`try`**. Порядок как в sync, extended across suspensions. Empty **`catch`** — anti-pattern.

### AggregateException — когда

**`task.Wait()`**, **`.Result`** → **`AggregateException`**.

**`await Task.WhenAll(t1,t2)`** — typically **first fault only** на catch; остальные на **`t.Exception`**.

Inspect all:
```csharp
await Task.WhenAll(tasks);
foreach (var t in tasks.Where(t => t.IsFaulted)) Log(t.Exception);
```

### async void

Unhandled → process crash / context — не await'ится (**`1.4.13`**).

### Unobserved task

**`_ = FailingAsync()`** — **`UnobservedTaskException`**. Always **await** или controlled observation.

---

## Under the Hood

**`GetResult()`** awaiter unwraps. **`ExceptionDispatchInfo.Capture(ex).Throw()`** — preserve stack cross-thread. **`throw ex`** — resets stack — avoid.

---

## Examples

### WhenAll — все fault

```csharp
try { await Task.WhenAll(tasks); }
catch {
    var faults = tasks.Where(t => t.IsFaulted).ToList();
    _logger.LogError("{Count} failures", faults.Count);
    throw;
}
```

---

## Common Mistakes

WhenAll ≠ AggregateException on await. Swallow after await. **`.Result`**. **`throw ex`**. Broad **`TaskCanceledException`** catch.

---

## Key Takeaways

- **await** unwraps; **Wait/Result** → AggregateException
- **WhenAll** — first fault + manual inspect siblings
- **throw;** preserve stack
- No **async void** business logic
