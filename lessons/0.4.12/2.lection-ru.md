## Why This Matters

«API down» / «slow» — сначала network или app? **ping** и **traceroute** — первые tools: на каждой OS, без install.

Не решают всё: ICMP block common; traceroute ≠ HTTP errors; green ping ≠ port 443 open. Но traceroute с jump 30→200 ms на hop — geography/routing, не React bug.

## Core Concepts

### ping

**ICMP Echo Request/Reply**. RTT per packet = **latency** (`0.4.11`). `ping -c 4 host` (Linux/macOS), `ping -n 4 host` (Windows).

0% loss — reachable at IP (if ICMP allowed). 100% loss — down, wrong IP, ICMP blocked, broken path. DNS resolve first (`0.4.5`).

**Не** test'ит TCP 443 / ASP.NET health — после ping → `curl` / `Test-NetConnection`.

### traceroute

Path через **TTL increment**. Each router returns ICMP time exceeded → hop IP + RTT. `traceroute host` / `tracert host`.

`* * *` — hop silent (often OK). RTT jump — distance/congestion. Implementation: UDP/ICMP/TCP varies.

### ICMP blocking

Production часто **block inbound ICMP** (`0.4.7`). ping fail + curl 200 — normal. ping OK + API timeout :443 — firewall/port.

**ping** = IP (ICMP if allowed); **curl** = TCP + HTTP.

### Связь с уроками

Unknown host → DNS. ping OK, curl timeout → SG/firewall. High RTT remote office → latency. Path stops → routing.

---

## Under the Hood

TTL prevents loops; traceroute probes TTL=1,2,3… Asymmetric routing possible — hop RTT indicative.

Windows **tracert** ICMP vs Linux **traceroute** UDP/ICMP — firewalls treat differently.

## Syntax / Commands / API

```bash
ping -c 10 google.com
traceroute api.example.com
nc -zv api.example.com 443
curl -v --connect-timeout 5 https://api.example.com/health
```

```powershell
ping -n 10 google.com
tracert api.example.com
Test-NetConnection api.example.com -Port 443
```

## Common Mistakes

ping fail = down (without curl). One traceroute during glitch. ping для CORS/401 debug.
