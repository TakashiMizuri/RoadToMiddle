# Delegates: Declaration, Multicast, and Removal

> Roadmap: `1.3.3` · Node: `1.3` — C# advanced topics · Depth: **глубоко**

## Learning Objectives

After this lesson you will be able to:

- Explain **what a delegate is** at runtime (type-safe function pointer + object target).
- Declare **custom delegate types** and use **`delegate` keyword** vs built-in **`Action`/`Func`** (preview `1.3.4`).
- Combine handlers with **`+=`** into a **multicast delegate** and predict **invocation order**.
- Remove handlers with **`-=`** and understand **reference equality** for removal.
- Inspect **`GetInvocationList()`** and invoke subscribers individually for error isolation.
- Describe **Under the Hood**: `Invoke`, `BeginInvoke` legacy, allocation of combined delegates.

---

## Why This Matters

Delegates are the wiring of .NET events, LINQ (`Func<T,bool>` predicates), ASP.NET middleware (`RequestDelegate`), and callback-based APIs. **Multicast** lets multiple subscribers react to one signal — button click, domain event, `AppDomain.ProcessExit`. Misusing **`-=`** (subscriber not removed → leak, duplicate calls) or assuming **return values** combine correctly causes subtle production bugs.

Before `async`/`await` everywhere, **BeginInvoke** on delegates was common; understanding **synchronous multicast invocation** still matters for **event handlers**, **Observable** patterns, and **testing** (mock `Action` chains). Middle developers debug «why my handler fired twice» and «unsubscribe did not work» weekly.

## Core Concepts

### Delegate as Type-Safe Callback

A **delegate** is a **reference type** deriving from **`MulticastDelegate`** → **`Delegate`**. It stores:

1. **Target object** (for instance methods) or `null` (static methods).
2. **Method pointer** (`MethodInfo` + native entry point).

```csharp
public delegate void Notify(string message);

Notify log = Console.WriteLine;           // static method
Notify instance = obj.HandleNotify;         // instance method
```

Assignment replaces the invocation list with a single entry. **Type safety**: compiler verifies signature match; unlike raw function pointers in C.

### Custom Declaration vs `Action`

```csharp
public delegate int Transform(string input);

Transform t = s => s.Length;
int len = t("hello"); // 5
```

Custom delegates improve **domain naming** (`Notify` vs `Action<string>`). BCL built-ins cover most shapes (`1.3.4`).

### Multicast: `+=` Combines Handlers

Delegates are **immutable** — `+=` creates a **new** combined delegate:

```csharp
Notify combined = handler1;
combined += handler2;
combined += handler3;
combined("event"); // calls handler1, handler2, handler3 in order
```

**Order**: generally **registration order** (append). **Same target+method** added twice → appears twice in list.

Internally, **`Delegate.Combine(a, b)`** builds a new **`MulticastDelegate`** with invocation list array.

### `-=` Removes by Identity

```csharp
combined -= handler2;
```

Removes **last matching** entry where **target and method** equal the right-hand delegate. If handler2 was registered twice, one `-=` removes one occurrence.

**Pitfall**: lambdas create **new compiler-generated methods** each time unless cached:

```csharp
Notify n = msg => Console.WriteLine(msg);
combined += n;
combined -= msg => Console.WriteLine(msg); // FAILS to remove — different method
```

Store lambda in variable before `+=`/`-=`.

### Return Values in Multicast

Non-`void` delegates: **`Invoke` returns only the last handler's return value**; earlier returns discarded.

```csharp
Func<int> f = () => 1;
f += () => 2;
Console.WriteLine(f()); // 2
```

For aggregating results, iterate **`GetInvocationList()`**:

```csharp
foreach (Func<int> handler in f.GetInvocationList().Cast<Func<int>>())
    sum += handler();
```

### Exceptions During Multicast

If handler2 throws, **later handlers may not run** (default `Invoke` behavior). For events, consider per-handler try/catch when isolating failures.

### `GetInvocationList()`

Returns **`Delegate[]`** clone of subscribers — safe to iterate while planning removal without modifying during invoke.

```csharp
foreach (Notify n in combined.GetInvocationList())
    n(message);
```

## Under the Hood

### Object Layout

`MulticastDelegate` contains **`_invocationList`** (array of delegates) and **`_invocationCount`**. Single-cast delegate may optimize without full list.

### `Invoke` Implementation

Compiler generates **`Invoke`** and **`BeginInvoke`/`EndInvoke`** (legacy async) on delegate types. Call site: `del(args)` → **`del.Invoke(args)`**.

Multicast **`Invoke`** loops invocation list, calling each entry with same arguments.

### Immutability and Allocation

Each **`+=`** allocates new delegate object + possibly new array — not O(1) in place mutation. High-frequency subscribe/unsubscribe on hot path — consider **`List<Action>`** or **event** with custom accessor.

### Events vs Public Delegate Field

**`event`** keyword restricts **`=`** from outside — only **`+=`/`-=`**. Public **`Action` field** allows **`=`** wiping all subscribers — anti-pattern for events (`1.3.5`).

### Comparison to Function Pointers (C# 9+)

**`delegate*`** — unmanaged, no multicast, unsafe context. Traditional delegates — managed, GC-tracked targets, multicast.

## Syntax / Commands / API

```csharp
public delegate void Del(int x);
Del d = SomeMethod;
d += OtherMethod;
d -= SomeMethod;

Delegate[] list = d.GetInvocationList();
Delegate combined = Delegate.Combine(d1, d2);
Delegate removed = Delegate.Remove(d, d1);
bool same = d1.Equals(d2); // target + method
```

Null delegate: invoking null throws **`NullReferenceException`** — check **`?.Invoke()`** or **`if (d is not null)`**.

## Examples

### Logging Multicast

```csharp
public delegate void LogHandler(string msg);

LogHandler log = s => File.AppendAllText("app.log", s);
log += s => Console.WriteLine(s);
log("Started"); // both run
```

### Safe Unsubscribe

```csharp
void OnData(string s) => Console.WriteLine(s);

LogHandler log = null!;
log += OnData;
// later:
log -= OnData; // works — same method reference
```

### Isolated Invocation

```csharp
void InvokeAll(LogHandler log, string msg)
{
    foreach (LogHandler h in log.GetInvocationList())
    {
        try { h(msg); }
        catch (Exception ex) { /* log and continue */ }
    }
}
```

## Common Mistakes & Anti-patterns

**`-=` with new lambda** — does not remove original subscriber.

**Expecting sum of return values** from multicast `Func` — only last return wins.

**Public delegate field** instead of **`event`** — external code assigns `=` and clears handlers.

**Assuming thread safety** — `+=`/`-=` on shared delegate without lock can race; use **`event`** or synchronize.

**Duplicate registration** on every page load / constructor without guard — handler runs N times.

## Production & Real-World Notes

Domain events in DDD: **`IDomainEventDispatcher`** often replaces raw multicast for testability.

UI frameworks (WinForms/WPF) — event subscription leaks if `-=` missed on dispose.

Testing: **`Mock<Action<T>>`** or capture **`GetInvocationList()`** count.

Performance: prefer **`Action`** + **`List<Action>`** pool if subscribing thousands/sec (rare).

## Comparison / Trade-offs

| Mechanism | Multicast | Type-safe | Typical use |
|---|---|---|---|
| **Delegate** | Yes | Yes | Events, callbacks |
| **`Action`/`Func`** | Yes | Yes | LINQ, local callbacks |
| **`List<Action>`** manual | Yes (you invoke loop) | Yes | Full control, less alloc |
| **function pointer** | No | Partial | Interop, hot path |

## Quick Reference

| Operation | Effect |
|---|---|
| `d = a` | Replace list with single `a` |
| `d += a` | Append `a` (new delegate instance) |
| `d -= a` | Remove last matching `a` |
| `d()` | Invoke all; void OK |
| `Func` multicast | Last return value only |
| `GetInvocationList()` | Snapshot of handlers |

---

## Key Takeaways

- Delegate = **object + method pointer**, subtype of **`MulticastDelegate`**.
- **`+=`** builds immutable multicast chain; order preserved.
- **`-=`** uses **target + method identity** — lambdas must be stored.
- Multicast **return values**: only **last** handler counts.
- **`GetInvocationList()`** for isolation, aggregation, debugging.
- Use **`event`** to protect invocation list from external `=` assignment.
