# Async в ctor и property getters — краткая выжимка

> `1.4.38` · ~8 мин на повторение

## Главная идея

**Конструктор и property getter не могут быть async** — CLR и язык требуют синхронного завершения конструирования и доступа к property. Async setup: **`CreateAsync`**, **`InitializeAsync`** на composition root, **`IHostedService`** warmup.

## Ключевые пункты

- `async ctor` — синтаксически невозможен.
- `async property get` — compile error.
- Fire-and-forget в ctor → half-initialized object, races.
- `.Wait()`/`.Result` в ctor → deadlock, thread pool starvation.
- DI (`AddSingleton<T>()`) резолвит **синхронно** — нет await в ctor injected service.
- Альтернативы: static `CreateAsync`, `IHostedService` preload, `GetXAsync()` method.
- Sync property только на уже загруженных данных.

## Самые важные примеры

```csharp
public static async Task<ConfigService> CreateAsync(IApiClient api, CancellationToken ct)
{
    var config = await api.FetchConfigAsync(ct);
    return new ConfigService(config);
}
```

## Частые ошибки

- `InitializeAsync().Wait()` в конструкторе.
- Lazy `GetNameAsync().Result` в getter.
- Публичные методы до завершения `InitializeAsync`.

## Мнемоника

> **Construct sync, initialize async** — собери объект синхронно, инициализируй асинхронно снаружи.
