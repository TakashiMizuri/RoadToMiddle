## Why This Matters

**`throw ex;`** сбрасывает stack trace — debug nightmare. Middle dev: правильный rethrow, wrap на границе API, typed exceptions.

---

## Core Concepts

### Hierarchy

**ArgumentException**, **InvalidOperationException**, **IOException**, custom domain exceptions. **ApplicationException** — legacy, не использовать.

### throw vs throw ex

```csharp
catch (Exception ex)
{
    _logger.LogError(ex, "Failed");
    throw;        // OK — stack сохранён
}

catch (Exception ex)
{
    throw ex;     // BAD — stack с этой строки
}
```

**Wrap:** `throw new DomainException("...", ex);`

**ExceptionDispatchInfo.Capture(ex).Throw()** — async/deferred.

### catch when

```csharp
catch (SqlException ex) when (ex.Number == 1205) { /* retry */ }
```

### Layers

Repository — specific catch or propagate. Middleware — один catch **Exception** → log + 500.

---

## Key Takeaways

- **throw;** не **throw ex;**
- Wrap с InnerException на boundary.
- Не swallow empty catch.
- OperationCanceledException — не всегда error.
