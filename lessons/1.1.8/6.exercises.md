# Exercises: Operators

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Predict expressions
**Goal:** Numeric and null-coalescing fluency.

Evaluate without running (then verify):

```csharp
int a = 17 / 5;
int b = 17 % 5;
decimal c = 0.1m + 0.2m;
string? s = null;
string t = s ?? "empty";
int? n = 0;
int m = n ?? 42;
```

**Success criteria:** 6/6 correct values.

---

### E2. Fix the condition
**Goal:** Short-circuit safety.

Fix and explain:

```csharp
if (config != null || config.IsEnabled)
{
    Run(config);
}
```

**Success criteria:** No possible NRE; uses `&&` appropriately.

---

## Medium

### M1. Shipping calculator
**Goal:** Apply decimal arithmetic.

Write a method:

```csharp
decimal CalculateTotal(decimal unitPrice, int quantity, decimal? discountPercent)
```

- `total = unitPrice * quantity`
- Apply discount if `discountPercent` not null: `total *= (1 - discount/100)`
- Return `total` rounded to 2 decimal places (`Math.Round(total, 2)`)

Use **`??`** or conditional — not `if`-only if you can express default with operators.

**Success criteria:** Uses `decimal` and `m` literals; handles null discount as 0%.

---

### M2. Validation expression
**Goal:** Combine comparison + logical.

Given:

```csharp
public record User(int Age, bool IsBanned, bool EmailVerified);
```

Write one **`bool IsEligible(User u)`** expression (single return) true when: age 18–120 inclusive, not banned, email verified.

**Success criteria:** One return with `&&`; no `if` statements required.

---

## Hard (stretch)

### H1. checked vs unchecked
**Goal:** Overflow awareness.

Write two small methods `AddUnchecked` and `AddChecked` that add 1 to `int.MaxValue`. Document behavior and when you'd enable `<CheckForOverflowUnderflow>` in csproj.

**Success criteria:** Observable wrap vs exception.
