# Exercises: Methods — parameters, return, overloading

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Extract from Main
**Goal:** Structure console app.

**Tasks:**
Create `Methods.Basics` console app. Move logic to:
- `static int Add(int a, int b)`
- `static void Main()` reads two ints, prints sum via `Add`.

**Success criteria:** `Main` ≤ 10 lines; methods compile.

---

### E2. Expression-bodied helpers
**Goal:** Short method syntax.

**Tasks:**
Implement:
- `static bool IsPositive(int n) => ...`
- `static string Sign(int n) => ...` returns `"+"`, `"-"`, or `"0"`

**Success criteria:** All three paths in `Sign`; no nested Main logic.

---

## Medium

### M1. Overload Area
**Goal:** Practice overloading.

**Tasks:**
Overload **`Area`**:
- square `int side`
- rectangle `int w, int h`
- circle `double radius` (use `Math.PI`)

`Main` calls all three; print results.

**Success criteria:** Compiler resolves without casts; circle uses `double`.

---

### M2. Optional greeting
**Goal:** Default parameters.

**Tasks:**
`static void Greet(string name, string title = "Mr./Ms.", int times = 1)`

Demo: `Greet("Lovelace")`, `Greet("Hopper", "Dr.", 2)`.

**Success criteria:** Defaults work; optional params last.

---

### M3. ReadIntPrompt reuse
**Goal:** Method from prior lessons integration.

**Tasks:**
Implement `ReadIntPrompt(string message)` with loop + `TryParse` + early **`return`**.

Build mini calculator: read two ints, print sum/product via separate methods.

**Success criteria:** Invalid input re-prompts; no duplicate parse loops in Main.

---

### M4. Validation without Console
**Goal:** Return bool, caller branches.

**Tasks:**
`static bool IsValidEmail(string? email)` — minimal rules: non-null, contains `@`, length ≥ 5.

`Main` reads email, uses **`if/else`** from `1.1.9` to print valid/invalid.

**Success criteria:** Validator has no `Console.WriteLine` inside.

---

## Hard (stretch)

### H1. Mini report pipeline
**Goal:** Compose methods.

**Tasks:**
Methods chain:
1. `ReadScores()` → `int[]`
2. `ComputeAverage(int[])` → `double`
3. `ComputeMax(int[])` → `int`
4. `PrintReport(int[], double avg, int max)` → void

Wire in `Main`. Handle empty array in `ComputeAverage` with **`return 0`** or guard — document choice.

**Success criteria:** Single responsibility per method; empty array doesn't crash.

---

### H2. Named arguments menu
**Goal:** Named + optional params.

**Tasks:**
`static void ShowOrder(string item, int qty = 1, bool express = false)`

Call from `Main` using **named arguments** in different orders at least twice.

**Success criteria:** Compiles; output reflects express flag.

---

## Submission checklist

- [ ] No CS0161 (all paths return where needed)
- [ ] Overloads represent one concept (Area, Print)
