# Answers: CancellationToken

## Answers

### 1. [Recall]
**Answer:** Cooperative — code observes token; preemptive kill thread **не используется** для safe cancel.
**Why:** Avoid corrupt state.

### 2. [Recall]
**Answer:** Schedules **`Cancel()`** after timeout on timer.
**Why:** Built-in timeout without manual Task.Delay thread.

### 3. [Understanding]
**Answer:** Cancel when **any** linked source cancels (user OR timeout OR abort).
**Why:** Combine independent cancel conditions.

### 4. [Understanding]
**Answer:** Delay task → **Canceled**; await throws **`TaskCanceledException`**; state machine **exception path** in `MoveNext`.
**Why:** `1.4.7` fault/cancel propagation.

### 5. [Application]
**Answer:** `using var t = new CTS(30s); using var linked = CreateLinkedTokenSource(user, t.Token); await Op(linked.Token);`
**Why:** First fire wins.

### 6. [Application]
**Answer:** **Synchronously during `Cancel()`** invocation.
**Why:** Register contract.

### 7. [Analysis]
**Answer:** **Да** — если catch broad Exception без filter/rethrow.
**Why:** Catch `OperationCanceledException` separately or use `when`.

### 8. [Analysis]
**Answer:** **Нет** — token invalid after dispose.
**Why:** Undefined/canceled state.

### 9. [Understanding]
**Answer:** **`CanBeCanceled == false`** — never.
**Why:** `CancellationToken.None`.

### 10. [Application]
**Answer:** Model binding from **`HttpContext.RequestAborted`**.
**Why:** ASP.NET Core convention.

### 11. [Analysis]
**Answer:** **После** sync section completes (delayed cooperative).
**Why:** Must poll `ThrowIfCancellationRequested`.

### 12. [Application]
**Answer:** **Да, продолжают** — parent cancel не останавливает children без token.
**Why:** Pass token to each parallel task (`1.4.12`).
