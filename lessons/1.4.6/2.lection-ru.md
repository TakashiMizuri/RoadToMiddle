# async/await — обзор трансформации компилятора

> Roadmap: `1.4.6` · Node: `1.4` — C# async ⚡ · Depth: **глубоко**

## Learning Objectives

- Как compiler lowers **`async` methods** в **state machines**.
- Роли **`IAsyncStateMachine`**, **`MoveNext`**, **`SetResult`**, **`SetException`**.
- Почему **`async Task`** return **до** завершения work.
- **`await`** = registration continuation на **Task**.
- Связь с **`Task`** (`1.4.3`) и threads (`1.4.1`).

## Why This Matters

```csharp
var json = await httpClient.GetStringAsync(url);
```

Compiler → **state machine class**: fields для locals, awaiter, **state int**. Объясняет: no **`await` in lock**, **`async void`**, exception on **`await`**, **no extra thread** на I/O wait.

## Core Concepts

**`async Task<int> Foo()`** →

1. Nested type **`IAsyncStateMachine`**
2. **`MoveNext()`** — switch по **state**, split на каждом **await**
3. **`AsyncTaskMethodBuilder<int>`** — Task для caller
4. Stub init + **Start**

Caller получает **Task** сразу; **MoveNext** до incomplete **await** → return; continuation → **MoveNext** снова.

### await decomposition

**GetAwaiter**, **IsCompleted**, если false — save state, **AwaitUnsafeOnCompleted**, return. Thread free.

### async Task vs async void

Task — observable. void — **AsyncVoidMethodBuilder**, no Task (`1.4.13`).

## Under the Hood

Locals across awaits → **fields**. **`__state`**, **`__builder`**, awaiters.

**MoveNext** switch cases per await boundary.

**AwaitUnsafeOnCompleted** registers **MoveNext** on Task completion — pool thread may differ.

**AsyncMethodBuilder** — SetResult/Exception, Task creation.

SharpLab — view **`d__`** type.

## Examples

### Пример 1: Два await — три сегмента MoveNext

Start → await HTTP → await ReadContent → return. Два suspension point; **MoveNext** до трёх вызовов.

### Пример 2: Sync completion

**`await Task.FromResult(1)`** — **IsCompleted true**; весь метод может выполниться в одном **MoveNext** без return посередине.

### Пример 3: try/catch вокруг await

Compiler генерирует try/finally **внутри MoveNext** — semantics сохраняются across states.

## Common Mistakes & Anti-patterns

**«async создаёт thread»** — нет; continuation на pool при completion I/O.

**Огромные async methods** — giant state machine, allocation, сложный debug.

**Large captured state** — hoisted в fields → GC pressure.

**await in lock** — `lock` не async-compatible (`1.4.33`).

---

## Production & Real-World Notes

Stack traces: **MoveNext** — normal; .NET улучшает async stacks (`1.4.37`).

CS1998 — async без await — useless state machine.

**ValueTask** когда часто sync complete (`1.4.15`).

Parallel I/O: **`await Task.WhenAll`** — one suspension (`1.4.12`).

## Comparison / Trade-offs

| Пишешь | Runtime |

| await | Continuation + state |
| async Task | Task сразу caller |
| Sequential awaits | Sequential states |
| WhenAll | Single await combined |

## Key Takeaways

- async/await = state machine + builder.
- MoveNext until next incomplete await.
- I/O frees thread via continuation.
- Далее: **disassembly** (`1.4.7`).
