# Value types vs Reference types — краткая выжимка

> `1.1.6` · ~10 мин на повторение

## Главная идея

**Value type** — переменная *содержит* данные; assignment **копирует значение**. **Reference type** — переменная хранит **reference** на object на heap; assignment **копирует ссылку** (alias). Примитивы **`1.1.3`** — value types; **`string`** — reference type (immutable).

## Ключевые пункты

- Stack frame: value types **inline**; reference locals — **pointer-sized ref** → heap object.
- **«Struct = stack» — упрощение**: struct на heap как field class, element array, boxed value.
- **`int?`** — value type «optional int»; plain **`int`** не бывает `null`.
- Default: value `0`/`false`; reference `null`.
- Pass-by-value: value — копия bits; reference — копия ref, **общий object**.
- **`decimal`**, **`bool`**, **`char`** — value types; копируются по значению.

## Самые важные примеры

```csharp
int a = 10, b = a; b = 20;        // a == 10
var L1 = new List<int>(); var L2 = L1; L2.Add(1);  // L1 тоже изменился
Point p2 = p1; p2.X = 99;         // p1.X без изменений (struct)
string? s = null;                 // OK
// int x = null;                  // ошибка
```

## «Под капотом» в одном абзаце

JIT копирует `sizeof(T)` для value types; reference locals — managed pointers, GC сканирует roots (stack, statics, registers). Reference type objects на heap с method table. Boxing (`1.1.7`) помещает value type в heap object. Большие struct — CPU copy cost без GC.

## Частые ошибки

- Ожидать deep clone при `list2 = list1`.
- Думать, что value types *никогда* не на heap.
- Использовать `0` вместо «нет значения» без domain-договорённости.

## Мнемоника

> **Value — копируешь число; reference — копируешь адрес, объект один.**
