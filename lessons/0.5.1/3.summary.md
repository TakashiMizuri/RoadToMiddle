# DOCTYPE, html lang, head meta charset UTF-8 — краткая выжимка

> `0.5.1` · ~8 мин на повторение

## Главная идея

Каждый HTML-документ начинается с **каркаса**: **DOCTYPE** включает standards mode, **`lang`** на `<html>` задаёт язык для a11y/SEO, **`<meta charset="UTF-8">`** в начале `<head>` декодирует байты. Три слоя должны совпадать: кодировка файла, meta и HTTP `Content-Type`.

## Ключевые пункты

- **`<!DOCTYPE html>`** — не элемент DOM; переключатель парсера HTML5 (no-quirks mode).
- **`<html lang="ru">`** — BCP 47; вложенный `lang` для фрагментов на другом языке.
- **`<head>`** — title, meta, link, script; **`<body>`** — видимый контент.
- **UTF-8** — стандарт веба; meta charset — в первых 1024 байтах, лучше первой в head.
- Без charset — mojibake; без DOCTYPE — возможен quirks mode и «странный» CSS.
- View Source ≠ Elements panel (парсер может автоисправлять разметку).

## Самые важные примеры

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Заголовок</title>
</head>
<body>
  <p>Контент</p>
</body>
</html>
```

Фрагмент на другом языке: `<span lang="en">quote</span>` внутри `lang="ru"`.

## Частые ошибки

- Нет DOCTYPE или charset «где-нибудь внизу» head.
- `lang="en"` на полностью русской странице.
- Файл UTF-8, а сервер отдаёт другую кодировку в заголовке.

## Мнемоника

> **D-L-C**: **D**OCTYPE → **L**ang → **C**harset — до первого видимого тега в body.
