# Nullable Reference Types — краткая выжимка

> `1.3.17` · ~10 мин на повторение

## Главная идея

**NRT** — compile-time null analysis для reference types (`string?` vs `string`). **Не меняет runtime** — NRE возможен. Roslyn отслеживает null-state и выдаёт warnings (CS8602, CS8618).

## Ключевые пункты

- `#nullable enable` / `<Nullable>enable</Nullable>`.
- **`string`** — intent non-null; **`string?`** — may be null.
- Narrowing после `if (x is not null)`, `ThrowIfNull`, pattern match.
- **`!`** — null-forgiving, только suppress warning.
- Attributes: `[NotNullWhen(true)]`, `[MaybeNull]`, `[MemberNotNull]`.
- CS8618: `required`, initializer, или `string?`.
- JSON/API может нарушить intent — validate at boundary.

## Самые важные примеры

```csharp
#nullable enable
string? s = Get();
if (s is null) return;
Console.WriteLine(s.Length); // OK after check

ArgumentNullException.ThrowIfNull(user);
```

## «Под капотом» в одном абзаце

Metadata `NullableAttribute`; analysis pass в Roslyn; IL без injected checks. Cross-assembly annotations для NuGet consumers.

## Частые ошибки

- Спам `!` вместо guard/fix.
- Non-nullable properties на DTO без init.
- Ожидание runtime enforcement от NRT.

## Мнемоника

> **`?` = honest maybe-null; `!` = lie to compiler — prove it first.**
