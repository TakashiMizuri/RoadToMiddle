## Answers

### 1. [Recall]
**Answer:** Выполнение query **откладывается** до enumeration; operators возвращают lazy **`IEnumerable`**, не materialized result.
**Why:** Iterator/deferred model — construction vs execution.

### 2. [Recall]
**Answer:** Любые три из: **`ToList`**, **`ToArray`**, **`Count`**, **`First`**, **`Sum`**, **`Max`**, **`foreach`**, **`Aggregate`**, …
**Why:** Terminal operators pull/enumerate source.

### 3. [Understanding]
**Answer:** **`Where`** возвращает новый lazy view; **`list`** не мутируется до materialize, и даже тогда filter не удаляет из list — только view при enum.
**Why:** Non-destructive deferred transform.

### 4. [Understanding]
**Answer:** Pipeline **выполнится дважды** от source; side effects и I/O — дважды; результаты совпадут, если source/captures не менялись между passes.
**Why:** No default caching on **`Where`/`Select`**.

### 5. [Application]
**Answer:** Элементы **`> 100`** (threshold читается при execution).
**Why:** Closure over variable **`threshold`**.

### 6. [Analysis]
**Answer:** Первый **`WriteLine`**: **0**. Второй: **10** (после **`Count()`** один full pass).
**Why:** Lambda не вызывалась до enumeration.

### 7. [Understanding]
**Answer:** **`OrderBy`** buffer **all** elements (from upstream), sort in memory, then yield — **`O(n log n)`** memory/time vs streaming **`Where`**.
**Why:** Sorting requires full key set (typically).

### 8. [Application]
**Answer:** **`var list = q.ToList()`** then check + loop; или один **`foreach`** без prior **`Any()`**; или **`if (!q.Any())`** осознанно accepting double pass.
**Why:** Materialize or single pass.

### 9. [Analysis]
**Answer:** **`23`** (elements 2 and 3).
**Why:** Source mutated before enum; filter **`> 1`**.

### 10. [Understanding]
**Answer:** Оба defer до terminal; **`IEnumerable`** runs CLR iterators, **`IQueryable`** runs provider (SQL). Same «compose then execute once» discipline.
**Why:** Shared deferral pattern, different execution engine (`1.3.11`).

### 11. [Application]
**Answer:**
```csharp
IEnumerable<Order> query = orders;
if (from.HasValue) query = query.Where(o => o.Date >= from);
if (minTotal.HasValue) query = query.Where(o => o.Total >= minTotal);
var result = query.ToList();
```
**Why:** Wrapping preserves single execution at **`ToList`**.

### 12. [Analysis]
**Answer:** **Небезопасно** для deterministic logging: **`Select`** может не run (no enum), run once, or **N times** on re-enumeration; order tied to consumer.
**Why:** Side effects must not live in deferred projections without materialize.
