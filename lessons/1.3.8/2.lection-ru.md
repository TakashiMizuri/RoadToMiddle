# LINQ query syntax — когда читается лучше

> Roadmap: `1.3.8` · Node: `1.3` — C#: продвинутые темы · Depth: практика

## Learning Objectives

После этой лекции ты сможешь:

- Писать equivalent queries в **query syntax** (`from` / `where` / `select`) и **method syntax** (`1.3.7`).
- Объяснить, что query syntax — **syntactic sugar**, компилируемый в static **`Enumerable`** / **`Queryable`** method calls.
- Выбирать query syntax для **joins**, **group joins** и multi-source queries; method syntax — для простых pipelines.
- Использовать **`let`**, **`into`**, nested **`from`** для промежуточных variables и continuation queries.
- Mix query и method syntax там, где compiler позволяет.
- Знать limitations query syntax — не каждый LINQ operator имеет keyword form.

---

## Why This Matters

В `1.3.7` ты строил pipelines через **method syntax**. C# даёт вторую surface: **query syntax**, visually близкую к SQL. Обе компилируются в один IL. Ни одна не «более LINQ», чем другая.

Teams спорят о readability. Method syntax доминирует в simple chains. Query syntax остаётся уместной при **`join`**, **`group join`**, нескольких **`from`** — когда мыслишь relationally. Middle developer читает и пишет обе fluently.

Урок **practice-heavy**: translation между формами, keyword mapping, desugaring rules compiler.

## Core Concepts

### Два syntax, один API

Каждый query expression translate в method calls. Пример:

```csharp
// Query syntax
var result =
    from u in users
    where u.IsActive
    orderby u.LastName
    select u.Email;

// Method syntax (equivalent)
var result = users
    .Where(u => u.IsActive)
    .OrderBy(u => u.LastName)
    .Select(u => u.Email);
```

Deferred execution (`1.3.9`) одинаков. **`from`+`where`** → **`Where`**; **`orderby`** → **`OrderBy`/`ThenBy`**; **`select`** → **`Select`**; **`group`** → **`GroupBy`**.

### Basic clauses

**`from x in source`** — range variable **`x`**.  
**`where`** → **`Where`**.  
**`select`** → **`Select`**.  
**`orderby`** → **`OrderBy`/`OrderByDescending`/`ThenBy`**.

### Grouping

```csharp
var query =
    from o in orders
    group o by o.Country into g
    select new { Country = g.Key, Total = g.Sum(x => x.Amount) };
```

**`group ... by ... into g`** — новый scope; **`g`** — **`IGrouping<...>`**.

### Join — сильная сторона query syntax

```csharp
var query =
    from o in orders
    join c in customers on o.CustomerId equals c.Id
    where o.Total > 500
    select new { o.Id, c.Name, o.Total };
```

**`equals`**, не **`==`** — важно для **`IQueryable`** expression trees (`1.3.12`).

**`join ... into ...`** — group join (LEFT JOIN pattern):

```csharp
from c in customers
join o in orders on c.Id equals o.CustomerId into orderGroup
from o in orderGroup.DefaultIfEmpty()
select new { c.Name, OrderId = o != null ? o.Id : (int?)null };
```

Details **`Join`/`GroupJoin`** — `1.3.21`.

### `let`

```csharp
from u in users
let fullName = u.FirstName + " " + u.LastName
where fullName.Length > 5
orderby fullName
select fullName
```

Desugars в nested **`Select`**. **`let`** когда complex expression повторяется.

### Mix query и method

```csharp
var top =
    (from s in sales
     where s.Year == 2026
     group s by s.Region into g
     select g.Sum(x => x.Amount))
    .OrderByDescending(total => total)
    .Take(5);
```

Parentheses обязательны перед dot.

### Без keyword в query syntax

**`Take`**, **`Skip`**, **`Distinct`**, **`Aggregate`** — method only.  
Multiple **`from`** без **`join`** = **`SelectMany`** (cross / flatten).

## Under the Hood

Desugaring **deterministic** по language spec. Performance identical. На **`IQueryable`** lambdas в **`where`** становятся **expressions**, не compiled delegates (`1.3.12`).

**`let`**, **`into`** используют **transparent identifiers** в compiler-generated code.

## Examples

Multi-from flatten:

```csharp
from o in orders
from line in o.Lines
where line.Quantity > 0
select new { o.Id, line.ProductId };
```

## Common Mistakes & Anti-patterns

**`==` в join** — error; **`equals`**.  
**Query для one-line Where** — method часто яснее.  
**Выглядит как SQL ≠ SQL executes** на **`IEnumerable`**.
