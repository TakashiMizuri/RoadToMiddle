# LINQ Any, All, Contains — existence checks

> Roadmap: `1.3.24` · Node: `1.3` · Depth: практика

## Learning Objectives

- **`Any()`**, **`Any(predicate)`** с short-circuit.
- **`All(predicate)`** и vacuous truth на empty sequence.
- **`Contains(value)`** с **`IEqualityComparer`**.
- Избегать **`Count() > 0`**; **`AnyAsync`** vs **`CountAsync`** в EF.
- De Morgan и security pitfalls с empty collections.

---

## Why This Matters

«Есть ли unpaid invoice?», «Все lines valid?», «Есть role Admin?» — **`Any`**, **`All`**, **`Contains`**, не full count. **`Where().Count() > 0`** лишняя work. EF: **`Any`** → **`EXISTS`**, **`Count > 0`** → **`COUNT(*)`**. **`All`** на empty → **true** — баги validation и auth.

---

## Core Concepts

### Any

```csharp
orders.Any();
orders.Any(o => o.Total > 10_000);
```

Empty → **false**. Short-circuit на first match.

### All

```csharp
order.Lines.All(l => l.Qty > 0);
```

**Empty → true** (vacuous). Для «non-empty и all valid»:

```csharp
lines.Any() && lines.All(l => l.Qty > 0);
```

### Contains

```csharp
allowed.Contains(user.Role, StringComparer.OrdinalIgnoreCase);
```

**`HashSet`** для repeated lookups. Reference vs value equality.

### Security

```csharp
// BUG: empty roles → true → grant
roles.All(r => r != "Admin");

// OK
!roles.Contains("Admin");
```

---

## Under the Hood

Foreach с early exit. **`IQueryable`** → **`EXISTS`**, **`IN`**. Immediate execution. **`AnyAsync`** в EF.

---

## Examples

### Validation API

```csharp
public static bool IsValidOrder(Order o) =>
    o.Lines.Any() &&
    o.Lines.All(l => l.Qty > 0 && l.UnitPrice >= 0);
```

**`Any()`** на lines явно отсекает empty order до vacuous **`All`**.

### Permission check

```csharp
public bool CanModerate(IReadOnlyList<string> roles) =>
    roles.Contains("Moderator", StringComparer.OrdinalIgnoreCase)
    || roles.Contains("Admin", StringComparer.OrdinalIgnoreCase);
```

Hot path: один раз построить **`HashSet<string>`** с comparer — **`Contains`** O(1).

### EF existence before insert

```csharp
if (await db.Customers.AnyAsync(c => c.Email == dto.Email))
    return Conflict("Email taken");
```

**`AnyAsync`**, не **`CountAsync() > 0`** — EXISTS-style, меньше work на server.

### Negation batch validation

```csharp
var hasInvalid = batch.Any(item => !validator(item));
if (hasInvalid) return BadRequest();
```

Яснее, чем **`!batch.All(validator)`**, когда validator назван **`IsValid`**.

## Common Mistakes & Anti-patterns

**`Count() > 0`** вместо **`Any()`** — лишний full walk / COUNT SQL.

**`All` без non-empty guard** — vacuous true на empty lines/order.

**`roles.All(r => r != "Admin")` для security** — empty roles → true; use **`!Contains`**.

**Double enumeration** expensive **`IEnumerable`** для отдельных **`Any`** и **`All`** — one pass или materialize.

---

## Production & Real-World Notes

EF handlers: **`AnyAsync`** для uniqueness/existence. **`AllAsync`** — verify SQL; иногда client eval. FluentValidation часто читабельнее для DTO, но LINQ fine на in-memory domain collections.

Repeated membership checks → **`HashSet`**, не list **`Contains`** каждый request.

---

## Key Takeaways

- **`Any`** > count for existence.
- **`All`** empty = true — явный guard.
- **`Contains`** + comparer для strings.
- **`AnyAsync`** в handlers.
