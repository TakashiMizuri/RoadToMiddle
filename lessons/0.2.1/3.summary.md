# Branch, checkout, switch — краткая выжимка

> `0.2.1` · ~10 мин на повторение

## Главная идея

**Branch** — mutable ref (файл `.git/refs/heads/<name>`) с hash tip commit'а, а не копия папки проекта. **HEAD** (`0.1.4`) указывает на текущую ветку; при **commit** ref ветки сдвигается вперёд. **`git switch`** меняет ветку и синхронизирует working tree + index; **`git restore`** — только файлы.

## Ключевые пункты

- Создание ветки = запись ref на текущий commit — **O(1)**, objects не копируются.
- **`git switch -c name`** — создать и переключиться; **`git branch name`** — только создать.
- **Detached HEAD** — HEAD содержит hash commit'а, не `ref: refs/heads/...`; commit'ы без branch теряются при уходе (спасает reflog / `git switch -c`).
- Namespaces refs: **`refs/heads/`** (local branches), **`refs/tags/`**, **`refs/remotes/`** (после fetch).
- Switch блокируется, если local changes перезапишутся — commit, stash или discard.
- Git 2.23+: **`switch`** для веток, **`restore`** для файлов; **`checkout`** — legacy, всё ещё работает.

## Самые важные примеры

**Параллельная feature:**
```bash
git switch -c feature/login   # ref на том же commit, что main
# commits только на feature/login; main ref не двигается
```

**Спасение detached HEAD:**
```bash
git switch --detach abc1234
# work + commit...
git switch -c fix/repro        # attach branch name
```

**Цепочка resolution:**
```
HEAD → refs/heads/main → commit hash → tree → blobs → files
```

## «Под капотом» в одном абзаце

`git switch feature` читает hash из `refs/heads/feature`, сравнивает tree текущего и целевого commit, обновляет tracked files в working tree и index, пишет `ref: refs/heads/feature` в HEAD. Commit objects не меняются. `git branch new` только создаёт `.git/refs/heads/new` с hash текущего HEAD — HEAD и working tree без изменений. Ветка и commit object — разные сущности: ref mutable, object immutable.

## Частые ошибки

- Думать, что ветка = папка на диске.
- Commit в detached HEAD без `git switch -c`.
- Путать `git restore` (файлы) и `git switch` (ветки).
- Удалять ветку, ожидая «очистки history» — удаляется только имя ref.

## Мнемоника

> **Branch** = закладка на commit; **switch** = переставить закладку «где я работаю»; **restore** = откатить файл, не меняя закладку ветки.
