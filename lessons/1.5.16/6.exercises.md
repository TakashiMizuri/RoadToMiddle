# Exercises: NuGet and Central Package Management

> Complete after passing `4.test-yourself.md`. Hands-on in repo or scratch solution.

## Easy

### E1. List dependency graph
**Goal:** Run `dotnet list package --include-transitive` on `app/` or a sample csproj.
**Success criteria:** List 3 direct + 2 transitive with versions.

### E2. Outdated check
**Goal:** `dotnet list package --outdated`.
**Success criteria:** Note one outdated direct package (if any).

## Medium

### M1. dotnet nuget why
**Goal:** Pick transitive package from E1; run `dotnet nuget why`.
**Success criteria:** Paste dependency chain (one path).

### M2. CPM scratch setup
**Goal:** New solution with 2 projects; add `Directory.Packages.props` with 2 **PackageVersion**; projects reference without Version.
**Success criteria:** `dotnet restore` succeeds.

### M3. Vulnerable audit
**Goal:** `dotnet list package --vulnerable` (may be empty — document result).
**Success criteria:** Explain what you'd do if CVE found.

## Hard (stretch)

### H1. NU1605 reproduction
**Goal:** Force downgrade conflict (two packages needing different versions same dependency); fix with explicit version or upgrade.
**Success criteria:** Build clean; explain fix.

### H2. Lock file
**Goal:** Enable `RestorePackagesWithLockFile`; commit lock; explain CI benefit.
**Success criteria:** `packages.lock.json` generated.

## Submission checklist

- [ ] One `--include-transitive` output reviewed
- [ ] CPM props file created OR documented why repo already uses/alternatives
- [ ] One security audit command run
