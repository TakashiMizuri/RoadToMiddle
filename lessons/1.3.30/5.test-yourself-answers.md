# Answers: LINQ Performance — Multiple Enumeration

## Answers

### 1. [Recall]
**Answer:** Deferred: `Where`, `Select`. Immediate: `ToList`, `Count`, `First`, `Any`.
**Why:** Standard LINQ classification.

### 2. [Recall]
**Answer:** Forces full enumeration; stores in `List<T>`; returns reusable materialized collection.
**Why:** Materialization boundary.

### 3. [Understanding]
**Answer:** Два полных прохода — predicate вызывается ~2× на каждый element (если оба foreach полные).
**Why:** Cold re-execution.

### 4. [Understanding]
**Answer:** `Any()` — first pass; `ToList()` — second full pass.
**Why:** Classic double enumeration.

### 5. [Application]
**Answer:** `var list = items.ToList();` then use `list.Count` and iterate `list`.
**Why:** Single materialization.

### 6. [Application]
**Answer:** N+1 queries. Fix: one query `Where(p => ids.Contains(p.Id))` + ToDictionary.
**Why:** Batch fetch.

### 7. [Analysis]
**Answer:** O(1) — `Count()` detects ICollection.
**Why:** Extension method optimization.

### 8. [Analysis]
**Answer:** `OrderBy` buffers entire filtered sequence before Take; Where streams into OrderBy buffer.
**Why:** Sort needs all keys (for that segment).

### 9. [Understanding]
**Answer:** Значение на момент enumeration (после изменения).
**Why:** Closure over variable, not snapshot.

### 10. [Understanding]
**Answer:** Single forward streaming processing (large/infinite) where memory must stay bounded — one foreach, no Count first.
**Why:** Constant memory path.

### 11. [Application]
**Answer:** `IReadOnlyList<T>` or `List<T>` (materialized snapshot).
**Why:** Documents safe reuse.

### 12. [Analysis]
**Answer:** Два полных прохода Parse (Count + ToList).
**Why:** Both immediate on same cold chain.
