# Generics: классы, методы, constraints — краткая выжимка

> `1.3.1` · ~10 мин на повторение

## Главная идея

**Generics** параметризуют типы (`List<T>`, `Stack<T>`), перенося проверку типов с runtime на **compile time** и убирая **boxing** для value types. **`where`-constraints** сообщают компилятору, какие операции допустимы над `T`.

## Ключевые пункты

- **Open type** (`Stack<T>`) нельзя `new`; **closed type** (`Stack<int>`) — можно.
- Generic **методы** могут жить в non-generic классе; compiler **infers** `T` из аргументов.
- Constraints: `struct`, `class`, `notnull`, `unmanaged`, `new()`, base class, interface — комбинируются через AND.
- **`new()`** требует public parameterless constructor — abstract class не пройдёт при инстанцировании.
- **`default(T)`** — null для ref types, zero для value types.
- Static поля в generic class — **отдельные для каждого closed type**.
- .NET **reifies** generics: `typeof(List<int>)` ≠ `typeof(List<string>)`.

## Самые важные примеры

**Без boxing:**
```csharp
var stack = new Stack<int>();
stack.Push(42); // int в T[], не object на heap
```

**Constraints:**
```csharp
public static T Max<T>(T a, T b) where T : IComparable<T>
    => a.CompareTo(b) >= 0 ? a : b;
```

**Open generic DI:**
```csharp
services.AddScoped(typeof(IRepository<>), typeof(EfRepository<>));
```

## «Под капотом» в одном абзаце

CLR хранит generic metadata в runtime (reification). JIT для reference type arguments часто генерирует shared code; для каждого value type (`int`, `Guid`, …) — отдельная специализация с правильным layout массива. IL для `List<int>.Add` не содержит `box`. Constraints проверяются компилятором и частично runtime; они не создают runtime «фильтры» — это compile-time rules для безопасного IL.

## Частые ошибки

- Избыточные `where` — усложняют API без пользы.
- Думать, что static field generic class один на все `T`.
- Использовать `List<object>` вместо generic abstraction.
- Путать open type с closed при регистрации в DI.

## Мнемоника

**«T — placeholder до закрытия; where — контракт для компилятора.»**
