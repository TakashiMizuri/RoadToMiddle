## Core Concepts

### Arithmetic Operators

| Operator | Meaning | Notes |
|----------|---------|-------|
| `+` `-` `*` | Add, subtract, multiply | Also string concat for `+` with string |
| `/` | Division | **Integer division truncates** toward zero |
| `%` | Remainder | Sign follows dividend in C# |
| `++` `--` | Increment/decrement | Prefix vs postfix differ |

**Integer division trap:**

```csharp
int a = 5 / 2;           // 2, not 2.5
double b = 5 / 2;        // 2.0 â€” still int division first!
double c = 5.0 / 2;      // 2.5 â€” promote to floating division
double d = (double)5 / 2;
```

For money from `1.1.3`, use **`decimal`**:

```csharp
decimal total = price * quantity;
decimal tax = total * 0.075m;   // m suffix required on literal
```

**Overflow:** default integer arithmetic **wraps in unchecked** context (two's complement). **`checked`** throws **`OverflowException`**:

```csharp
unchecked
{
    int x = int.MaxValue;
    int y = x + 1;           // wraps to int.MinValue
}

checked
{
    int x = int.MaxValue;
    int y = x + 1;           // OverflowException
}
```

Project setting `<CheckForOverflowUnderflow>true</CheckForOverflowUnderflow>` makes checked default.

### Comparison Operators

| Operator | Meaning |
|----------|---------|
| `==` `!=` | Equal / not equal |
| `<` `>` `<=` `>=` | Ordering (numeric, char, DateTime, etc.) |

**Value types** compare by **value** (unless overloaded).

**Reference types** default `==` compares **reference identity** unless **`Equals` overloaded** or **`==` operator overloaded** (`string` compares content).

```csharp
string s1 = new string("hi".ToCharArray());
string s2 = new string("hi".ToCharArray());
Console.WriteLine(s1 == s2);      // True â€” string overloads ==
Console.WriteLine(ReferenceEquals(s1, s2));  // False â€” different objects
```

**Floating point:** avoid `==` for `double`/`float`; use epsilon tolerance or `decimal` for money.

**NaN:** `double.NaN == double.NaN` is **false**; use `double.IsNaN(x)`.

### Logical Operators

| Operator | Meaning | Short-circuit |
|----------|---------|---------------|
| `&&` | AND | Yes â€” skip RHS if LHS false |
| `\|\|` | OR | Yes â€” skip RHS if LHS true |
| `!` | NOT | Unary |

Short-circuit enables safe patterns:

```csharp
if (user != null && user.IsActive)
{
    // user.IsActive not evaluated if user is null
}
```

**Bitwise** `&`, `|`, `^` exist for integers and **non-short-circuit** bool evaluation â€” rare for conditions; used for flags (`1.1.17` enums flags).

**Do not confuse** `&` / `|` on bool with `&&` / `||` unless you intentionally want both sides evaluated.

### Null-Coalescing Operators

**`??`** â€” if left operand is **not null**, result is left; otherwise right:

```csharp
string name = input ?? "Anonymous";
int pages = requested ?? 10;   // if requested is int?
```

Right side type must convert to left; **`??` chains** left-associative:

```csharp
string s = a ?? b ?? "default";
```

**`??=`** (C# 8+) â€” assign only if null:

```csharp
list ??= new List<int>();   // same as list = list ?? new List<int>();
```

Works with **nullable reference types** and **nullable value types**:

```csharp
int? maybe = null;
int value = maybe ?? 0;

string? text = null;
text ??= "placeholder";
```

**Cannot** use `??` with non-nullable value type left side unless nullable:

```csharp
// int x = 0 ?? 1;   // compile error â€” int not null
int? x = 0;
int y = x ?? 1;      // OK â€” but x is not null, result 0
```

### Other Operators You Will Use Immediately

**Ternary conditional** `? :` â€” expression, not statement:

```csharp
string status = score >= 70 ? "Pass" : "Fail";
```

**Assignment** `=`, compound `+=`, `-=`, `*=`, `/=`, `%=`, `??=`.

**Operator precedence** (high to low snapshot): primary â†’ unary â†’ multiplicative â†’ additive â†’ relational â†’ equality â†’ logical AND â†’ logical OR â†’ null-coalescing â†’ conditional. **When in doubt, use parentheses** for readers.

## Under the Hood

### Numeric Promotion

Binary operations on mixed numeric types follow **numeric promotion** rules from the C# spec: smaller integral types promote to `int` for arithmetic; `float`/`double`/`decimal` have defined conversion paths. That is why `5 / 2` is integer â€” both are `int`.

IL emits **`add`**, **`sub`**, **`mul`**, **`div`**, **`rem`**, **`clt`**, **`ceq`**, etc. **`checked`** blocks emit **`add.ovf`** variants that throw on overflow.

### Short-Circuit IL

`&&` and `||` compile to **conditional branches** â€” RHS expressions may never execute. Side effects (method calls, increments) depend on this â€” a common interview and bug source.

### Null-Coalescing

`??` lowers to conditional logic with **null checks**; for reference types CLR compares against null reference; for `Nullable<T>` checks **`HasValue`**. No boxing for `int? ?? 0`.

## Common Mistakes & Anti-patterns

**Expecting real division from two ints.**

**Comparing doubles with `==`.**

**Using `&` instead of `&&`** and calling methods on null RHS.

**`??` with non-nullable left** â€” compile error; use nullable type.

**Overloaded ternary nesting** without parentheses â€” unreadable.

**Assuming `++i` and `i++` identical** in complex expressions.

## Production & Real-World Notes

Configuration: `builder.Configuration["Key"] ?? "default"` common; prefer **Options pattern** with defaults in `appsettings.json`.

API validation combines comparisons: `quantity > 0 && quantity <= MaxOrderQuantity`.

Logging: avoid `&&` chain that hides expensive call on left â€” order cheap checks first.

Nullable reference types (NRT) + `??` reduce `NullReferenceException` in modern codebases.

## Key Takeaways

- **Integer `/` truncates** â€” cast or use floating/`decimal` for real division.
- **`decimal` + `m` suffix** for money literals.
- **`==` on reference types** is identity unless overloaded.
- **`&&` / `||` short-circuit** â€” order operands deliberately.
- **`??` and `??=`** provide null-safe defaults â€” bridge to `1.1.18`.
- Use **parentheses** when precedence is unclear.
