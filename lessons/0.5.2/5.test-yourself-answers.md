# Answers: Semantic layout

## Scoring

- 90–100%: отлично · 80–89%: pass · <80%: повтори lection + summary

## Answers

### 1. [Recall]
**Answer:** Ровно один.
**Why:** Спецификация и a11y: один main landmark на document.

### 2. [Recall]
**Answer:** header, nav, main, article, section, aside, footer.
**Why:** Core HTML5 layout vocabulary.

### 3. [Recall]
**Answer:** `<nav>`.
**Why:** Элемент для navigation links.

### 4. [Understanding]
**Answer:** article — self-contained, syndicatable; section — thematic group внутри document/article, обычно с heading.
**Why:** Independence test vs chapter grouping.

### 5. [Understanding]
**Answer:** Section без heading не задаёт тематическую группу в outline; путает AT и авторов.
**Why:** Section = themed group with title.

### 6. [Understanding]
**Answer:** Logo + menu → `<header>` (nav внутри); статья → `<main>` (часто `<article>` внутри).
**Why:** Chrome outside main.

### 7. [Application]
**Answer:** `<article>` — каждая новость самостоятельный teaser/entry.
**Why:** RSS/share test passes per item.

### 8. [Application]
**Answer:** Два `<nav>` с разными `aria-label` (Primary / Table of contents).
**Why:** Distinguish landmarks for AT.

### 9. [Understanding]
**Answer:** `<aside>` — tangentially related, не основной поток описания.
**Why:** Complementary content role.

### 10. [Analysis]
**Answer:** D
**Why:** Скрытый второй main всё ещё нарушает правило; один main per page.

### 11. [Analysis]
**Answer:** Обернуть пост в `<article>`, главы в `<section>` с h2; div только для styling без semantics.
**Why:** Reflect document structure.

### 12. [Understanding]
**Answer:** Landmarks для SR/keyboard/SEO; CSS не передаёт роль region.
**Why:** Semantics ≠ presentation.

### 13. [Application]
**Answer:** Sidebar links → nav (вне main); таблица → main; Help → aside.
**Why:** Purpose-based mapping.

### 14. [Understanding]
**Answer:** Нет — body footer = site-wide contentinfo; article footer = мета поста (tags, share).
**Why:** Scoped to nearest sectioning root.

### 15. [Recall]
**Answer:** navigation (landmark).
**Why:** Implicit role for nav element.
