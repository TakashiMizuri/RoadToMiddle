# Polly — краткая выжимка

> `1.4.28` · ~10 мин на повторение

## Главная идея

**Polly** (через **IHttpClientFactory** handler pipeline) добавляет **retry**, **timeout** и **circuit breaker** для outbound HTTP — transient faults переживаем, chronic outage не усиливаем retry storm.

## Ключевые пункты

- **Retry:** transient status/exceptions, max attempts, exponential backoff + **jitter**.
- **Timeout:** per-attempt vs **HttpClient.Timeout** vs total request budget.
- **Circuit breaker:** closed → open (fail fast) → half-open probe.
- **.NET 8+:** `AddStandardResilienceHandler()` на named client.
- Не retry **400**, **401**, **404** без веской причины.
- **POST** retry только с idempotency design.
- **Parallel.ForEachAsync** × retries = multiplied load — cap оба.
- Respect **429 Retry-After** когда есть.

## Самые важные примеры

```csharp
builder.Services.AddHttpClient("catalog")
    .AddStandardResilienceHandler(o =>
    {
        o.Retry.MaxRetryAttempts = 3;
        o.TotalRequestTimeout.Timeout = TimeSpan.FromSeconds(30);
    });
```

## «Под капотом» в одном абзаце

ResiliencePipeline оборачивает SendAsync в DelegatingHandler. Каждый retry — полный проход chain (auth, logging). Open circuit бросает BrokenCircuitException без вызова downstream. Factory connection pool reuse сохраняется между attempts.

## Частые ошибки

- Retry всех 5xx включая non-transient business errors.
- HttpClient.Timeout 100s + 3 retry × 30s.
- Breaker без metrics/logs.
- Aggressive retry на dying dependency под high concurrency.

## Мнемоника

**Retry transient, break chronic, timeout everything, jitter the herd.**
