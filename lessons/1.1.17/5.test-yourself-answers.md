## Answers

### 1. [Recall]
**Answer:** `[Flags]` помечает enum как bitmask и влияет на форматирование (`ToString()` → `"A, B"`), документацию и некоторые API. Bitwise `|`, `&` работают **без** атрибута — он не включает «магию» комбинирования.
**Why:** Атрибут — metadata для tooling и человекочитаемого вывода, не отдельный runtime type.

### 2. [Recall]
**Answer:** `Parse` бросает **`ArgumentException`** (или related); `TryParse` возвращает **`false`** и default в `out`, не бросая.
**Why:** На границах недоверенного ввода исключения дороги и дают 500 вместо validation error.

### 3. [Understanding]
**Answer:** При степенях двойки каждый flag занимает уникальный bit; `A | B` не совпадает с другим named member. Sequential `1,2,3` даёт `A|B=3`, что может равняться `C=3` — коллизия имён и значений.
**Common mistake:** Думать, что `[Flags]` автоматически назначает степени двойки.

### 4. [Understanding]
**Answer:** С `[Flags]`: `"Read, Write"` (или локализованный порядок имён). Без `[Flags]`: часто `"3"` — numeric string underlying value.
**Why:** Formatter проверяет `FlagsAttribute` при декомпозиции bits в имена.

### 5. [Understanding]
**Answer:** `IsDefined` проверяет только **объявленные** именованные константы с exact integer value, не все mathematically valid bit combinations.
**Why:** `(Read|Write)` может быть 3, но если нет member `= 3`, `IsDefined` false.

### 6. [Application]
**Answer:** `(value & Permissions.Edit) == Permissions.Edit` или `(value & Permissions.Edit) != 0` для `[Flags]`.
**Why:** Эквивалент `HasFlag` без boxing.

### 7. [Application]
**Answer:** `Enum.TryParse<UserRole>(role, ignoreCase: true, out var r)`. При `"Admn"` — `false`; вернуть **400/422** с перечислением valid values, не throw.
**Why:** Fail-safe API boundary pattern.

### 8. [Application]
**Answer:** `Enum.TryParse<Permissions>("View, Edit", ignoreCase: true, out var p)` или `Parse` с теми же параметрами.
**Why:** BCL flags parser понимает comma-separated names при `[Flags]`.

### 9. [Analysis]
**Answer:** `C = 3` коллидирует с `A | B` (1|2=3). Комбинация не однозначна; нарушен power-of-two design.
**Why:** Classic flags anti-pattern.

### 10. [Analysis]
**Answer:** `OrderStatus.Pending` (член со значением 0).
**Why:** `default(enum)` — numeric zero, mapped to named member if exists.

### 11. [Understanding]
**Answer:** Явный «нет флагов» / unset state; без него 0 ambiguous. CA1008 рекомендует zero member для flags enums.
**Why:** Tests и API contracts требуют именованного empty state.

### 12. [Application]
**Answer:** Cast `(OrderStatus)99` **успешен** в runtime — enum не проверяет range. `IsDefined(typeof(OrderStatus), 99)` → **false** если 99 не объявлен.
**Why:** Enum type safety не covers arbitrary integers from DB without validation.

### 13. [Understanding]
**Answer:** `HasFlag` — readability в cold code, tests, prototypes. `&` — hot paths, high frequency, micro-performance sensitive code.
**Why:** `HasFlag` принимает `Enum` → boxing для value type enum.

### 14. [Recall]
**Answer:** **`int`**.

## Model answers for practical sections

```csharp
public static bool TryParseStatus(string? input, out OrderStatus status)
{
    if (string.IsNullOrWhiteSpace(input))
    {
        status = OrderStatus.Pending;
        return true;
    }
    return Enum.TryParse(input, ignoreCase: true, out status);
}
```

Допустимы варианты, где null/empty возвращают `false` — если в задании явно указано иное поведение; ключ — **`TryParse`** для non-empty invalid.
