# Answers: Lambda Expressions

## Answers

### 1. [Recall]
**Answer:** Expression — single expression, implicit return; statement — `{ }` block, explicit `return` for non-void.
**Why:** Syntax sugar levels for complexity.

### 2. [Recall]
**Answer:** Cannot capture locals or `this`; avoids display class.
**Why:** Static lambda restriction.

### 3. [Recall]
**Answer:** **`Expression<Func<>>`** for IQueryable/EF SQL translation; **`Func<>>`** for in-memory LINQ to Objects.
**Why:** Expression tree vs compiled delegate.

### 4. [Understanding]
**Answer:** Closure captures **variable** `threshold`, not value at creation — reads current value at invoke time.
**Why:** Display class field reference.

### 5. [Understanding]
**Answer:** Prints **3, 3, 3** — single shared `i` equals 3 after loop.
**Why:** Classic for-loop closure.

### 6. [Understanding]
**Answer:** Lambda capturing `this` → delegate target/closure holds subscriber → publisher event list prevents GC (`1.3.5`).
**Why:** Strong reference chain.

### 7. [Application]
**Answer:**
```csharp
s =>
{
    if (string.IsNullOrWhiteSpace(s)) throw new ArgumentException(nameof(s));
    return int.Parse(s);
}
```

### 8. [Application]
**Answer:** `(_, e) => { /* use e */ }` or `(sender, e) =>` ignoring sender.

### 9. [Analysis]
**Answer:** **`Action a = async () => ...`** — **async void** semantics; exceptions crash unobserved. **`Func<Task> f`** — OK.
**Why:** Action returns void.

### 10. [Analysis]
**Answer:** **Does not compile** — static lambda cannot access instance field `Count`.
**Why:** static lambda no instance capture.

### 11. [Application]
**Answer:**
```csharp
for (int i = 0; i < 3; i++)
{
    int copy = i;
    actions.Add(() => Print(copy));
}
```

### 12. [Understanding]
**Answer:** Private nested class with fields for captured variables + method implementing lambda body.
**Why:** Roslyn lowering.

### 13. [Application]
**Answer:** **`Func<User, bool>`** (Where predicate).

### 14. [Recall]
**Answer:** `(int x) => ...` — parentheses with type on parameter.

### 15. [Analysis]
**Answer:** Recursion, `ref`/`out` params, reuse multiple times in method, clearer stack traces — local function preferred.
**Why:** Local functions first-class in C# 7+.
