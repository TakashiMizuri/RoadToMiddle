# DOM-дерево — краткая выжимка

> `0.5.10` · ~10 мин на повторение

## Главная идея

Браузер парсит HTML в **live tree of nodes**: корень **`Document`**, **element nodes** (теги), **text nodes** (текст и whitespace). JS и React мутируют это дерево; CSS и a11y tree производны от element nodes.

## Ключевые пункты

- `nodeType`: Document **9**, Element **1**, Text **3**, Comment **8**, Fragment **11**.
- **`childNodes`** — все узлы; **`children`** — только elements.
- Whitespace между тегами — **text nodes** → ловушка `firstChild`.
- Parser **исправляет** invalid nesting; DevTools = canonical DOM.
- **`textContent`** — строка без HTML parse (safe); **`innerHTML`** — parse + **XSS**.
- **Accessibility tree** — фильтрованная проекция DOM, не копия 1:1.
- React **diff** → DOM ops; SSR **hydration** требует совпадения trees.

## Самые важные примеры

```javascript
el.textContent = userInput;  // one TEXT node, safe
el.innerHTML = userInput;    // creates elements — XSS if untrusted
list.children.length;        // elements only
list.childNodes.length;      // includes whitespace text
```

## «Под капотом» в одном абзаце

HTML parser (state machine) строит C++ node objects; incremental parsing + blocking scripts. Live HTMLCollection обновляется при mutation. Platform a11y API map element semantics + ARIA. Shadow DOM — отдельные subtrees.

## Частые ошибки

- `firstChild` вместо `firstElementChild`.
- innerHTML с user input.
- Думать, что DOM = исходный HTML файл.
- Ignored hydration whitespace mismatch.

## Мнемоника

> **Document down, elements act, text between** — корень document, elements с tag/attrs, text в щелях.
