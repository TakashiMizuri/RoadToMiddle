# IDisposable, using — краткая выжимка

> `1.3.16` · ~10 мин на повторение

## Главная идея

**GC** освобождает managed memory, но **не OS handles**. **`IDisposable.Dispose`** — deterministic cleanup. **`using`** / **`await using`** = try/finally + Dispose/DisposeAsync даже при exception.

## Ключевые пункты

- Unmanaged: files, sockets, DB handles — через wrappers с `IDisposable`.
- **`using var x`** — dispose в конце block; порядок — reverse declaration.
- **`IAsyncDisposable`** + **`await using`** для async I/O cleanup.
- **Finalizer** `~Class()` — non-deterministic, дорогой; **`GC.SuppressFinalize(this)`** после `Dispose`.
- **Idempotent** `Dispose` (`_disposed` guard).
- Dispose pattern: `Dispose(bool disposing)` для inheritance.
- **DbContext**, streams — always dispose; shared `HttpClient` — по lifetime policy.

## Самые важные примеры

```csharp
using var stream = File.OpenRead(path);
// finally stream.Dispose()

await using var cmd = connection.CreateCommand();
await cmd.ExecuteNonQueryAsync();
```

## «Под капотом» в одном абзаце

Finalizer → extra GC cycle, finalization queue, extended object lifetime. `using` compiles to try/finally. `SuppressFinalize` убирает object из finalization queue.

## Частые ошибки

- Забытый `using` на connection/stream.
- Dispose singleton HttpClient.
- Sync block в `Dispose` вместо `DisposeAsync`.

## Мнемоника

> **If it holds a handle — `using` it.**
