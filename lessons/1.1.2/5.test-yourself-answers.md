## Answers

### 1. [Recall]
**Answer:** Полное имя типа включая все namespace segments, например `Contoso.App.Services.EmailSender`.

**Why:** FQN однозначно идентифицирует тип в compilation.

---

### 2. [Recall]
**Answer:** File-scoped: `namespace X.Y;` — semicolon, без оборачивающих `{}` для всего файла. Block-scoped: `namespace X.Y { ... }` с телом в braces.

**Why:** File-scoped = один namespace на файл, меньше indent (C# 10+).

---

### 3. [Recall]
**Answer:** Позволяет вызывать **static members** `Console` (например `WriteLine`) **без префикса** `Console.`.

**Why:** `using static` imports members, not types only.

---

### 4. [Understanding]
**Answer:** C# **не требует** соответствия folder ↔ namespace; compiler смотрит только на **declared namespace** в source. Convention — для людей и tooling.

**Why:** Physical layout ≠ logical naming rule in language spec.

---

### 5. [Understanding]
**Answer:** **Compile error — ambiguous reference** (CS0104), пока не используешь FQN или alias.

**Why:** Compiler не выбирает «случайный» namespace.

---

### 6. [Understanding]
**Answer:** **Assembly** — физическая `.dll` (deploy/reference unit). **Namespace** — логическая группа имён типов; может span несколько assemblies или many namespaces в одной assembly.

**Why:** Different dimensions of organization.

### 10. [Analysis]
**Answer:** **Ровно один** (или zero, тогда global namespace).

**Why:** CS8954 при более одном file-scoped namespace.

---

### 11. [Analysis]
**Answer:** **B** — C# **не имеет** wildcard `using` как Java; только explicit namespace imports.

---

### 12. [Understanding]
**Answer:** Автоматически добавляет common **global usings** (`System`, `System.Linq`, …) для всех файлов project — меньше boilerplate в templates.

**Why:** SDK injects GlobalUsings.g.cs at build.

---

### 13. [Application]
**Answer:** Просто **`Foo`** или **`new Foo()`** — типы в **одном namespace** видят друг друга без `using`.

**Why:** Same namespace = same declaration scope for unqualified lookup.

---

### 14. [Analysis]
**Answer:** **Валидно.** Trade-offs: zero ambiguity, но **verbose**, хуже readability; refactoring harder. Teams обычно `using` + occasional FQN.
