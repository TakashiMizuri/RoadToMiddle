# Exercises: WebSockets vs HTTP

> Conceptual + optional browser/curl.

## Easy

### E1. Transport picker
**Goal:** Apply decision tree.

**Tasks:** For each scenario, pick HTTP poll, SSE, or WebSocket and one sentence why:
1. Public REST catalog, updates hourly.
2. Server log tail in admin UI.
3. Two-player tic-tac-toe moves.

**Success criteria:** Three choices with direction/frequency justification.

---

### E2. Handshake headers
**Goal:** Recall upgrade shape.

**Tasks:** Write minimal request headers for WebSocket upgrade (no need valid key math).

**Success criteria:** Includes Upgrade, Connection, Sec-WebSocket-Version.

---

## Medium

### M1. EventSource vs fetch
**Goal:** SSE mental model.

**Steps:**
1. Read MDN `EventSource` page summary.
2. Explain why POST from same EventSource is not the model.

**Success criteria:** One paragraph server-push vs client commands via HTTP.

---

### M2. SignalR architecture sketch
**Goal:** Scale-out awareness.

**Tasks:** Diagram: 2 Kestrel nodes, LB, Redis backplane, 3 browser clients — show which messages need backplane.

**Success criteria:** Cross-node broadcast path labeled.

---
