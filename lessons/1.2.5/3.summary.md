# Encapsulation — краткая выжимка

> `1.2.5` · ~10 мин на повторение

## Главная идея

**Encapsulation** — object сам guard invariants; снаружи **stable API** (methods/properties), не direct field access. **Public fields** — любой caller ломает rules без hook; refactor = pain.

## Ключевые пункты

- **`private` storage + public behavior** — idiomatic C#.
- Public `{ get; set; }` без validation — leak как у field.
- **Tell, Don't Ask** — logic в owner type, не снаружи через public data.
- **Mutable collection property** — частый leak; private list + read-only view/methods.
- **DTO** на boundary OK; **domain entity** — behavior + invariants inside.
- Encapsulation ≠ runtime security (reflection); = maintainability и contract stability.

## «Под капотом» в одном абзаце

Properties compile в accessor methods — seam для EF, serializers, logging. Fields — direct IL access без intercept. Public field change breaks metadata consumers; private storage behind property preserves API.

## Частые ошибки

- Public fields «для JSON/скорости».
- Return internal `List<T>` from getter.
- Anemic model + god service.

## Мнемоника

> **Не давай ключи от сейфа — давай операции «положить/снять» с правилами.**
