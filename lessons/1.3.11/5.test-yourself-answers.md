# Answers: IEnumerable vs IQueryable

## Scoring

- 90–100%: отлично · 80–89%: pass · <80%: `1.3.11` lections + `1.3.12` preview

---

## Answers

### 1. [Recall]
**Answer:** **`Expression`** (query tree) и **`IQueryProvider`** (+ **`ElementType`**).
**Why:** Core of remote query model.

### 2. [Recall]
**Answer:** Переводит дальнейшие operators на **LINQ to Objects** — execution in CLR on items from upstream (often already fetched).
**Why:** Explicit client boundary.

### 3. [Understanding]
**Answer:** Первый загружает **все** products в memory, потом filter; второй filter в **SQL**, materialize только matches.
**Why:** Materialization placement.

### 4. [Understanding]
**Answer:** **CLR** — **`EnumerableQuery`**, local provider.
**Why:** No EF/SQL provider attached.

### 5. [Application]
**Answer:** SQL filters first on **`IQueryable`**, then **`ToList()`** or **`AsEnumerable()`**, then local **`Where`** scoring on bounded set.
**Why:** Minimize rows from DB.

### 6. [Analysis]
**Answer:** Variable typed **`IEnumerable`** may still hold **`IQueryable`** instance at runtime; enumeration **may** still trigger provider if underlying object is **`IQueryable`**. Но assignment to **`IEnumerable`** can hide intent and cause accidental client operators. Prefer explicit **`IQueryable`** until materialize.
**Why:** Type erasure / overload pitfalls.

### 7. [Understanding]
**Answer:** **`Func`** — compiled delegate, local execution (**`IEnumerable`**). **`Expression`** — data structure for translation (**`IQueryable`**).
**Why:** Dual overload design (`1.3.12`).

### 8. [Application]
**Answer:** Unbounded query; **`DbContext`** lifetime; serializer may double-enumerate; no control over SQL at API edge.
**Why:** Boundary + security + lifetime.

### 9. [Understanding]
**Answer:** **`ToListAsync`** triggers provider to **execute** current **`Expression`** tree → generate SQL → materialize results.
**Why:** Terminal immediate execution on provider.

### 10. [Analysis]
**Answer:** Lazy = **deferred**, not free. Each terminal hits DB with cost; bad composition loads excess data; translation failures at runtime.
**Why:** Confuse deferral with zero cost (`1.3.9`).

### 11. [Application]
**Answer:** **`IQueryable`** — compose **`Where`** clauses, single **`ToListAsync`** at end.
**Why:** One SQL with AND filters.

### 12. [Recall]
**Answer:** **`System.Linq.Queryable`** (vs **`System.Linq.Enumerable`**).
**Why:** Parallel static classes.
