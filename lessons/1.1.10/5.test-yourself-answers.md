## Answers

### 1. [Recall]
**Answer:** `while` проверяет condition **до** body; `do-while` — **после**.
**Why:** Отсюда минимум одна итерация у `do-while`.

### 2. [Recall]
**Answer:** **`IEnumerable<T>`** (или non-generic `IEnumerable` для object).
**Why:** `foreach` вызывает `GetEnumerator()` / `MoveNext()`.

### 3. [Understanding]
**Answer:** При `i == arr.Length` index out of range — valid indices `0 … Length-1`.
**Why:** `<= Length` даёт off-by-one на последней итерации.

### 4. [Understanding]
**Answer:** `foreach` — когда index не нужен, только elements. `for` — index, parallel arrays, reverse iteration, modify by index.
**Why:** `foreach` убирает index bugs; `for` даёт контроль позиции.

### 5. [Understanding]
**Answer:** **`InvalidOperationException`** — collection modified during enumeration.
**Why:** Enumerator invalidates при structural change.

### 6. [Application]
**Answer:** **3** строки: `0`, `1`, `2`.
**Why:** `i < 3` → i = 0,1,2.

### 7. [Application]
**Answer:** Пример:

```csharp
string password;
do {
    Console.Write("Password: ");
    password = Console.ReadLine() ?? "";
} while (password.Length < 8);
```

**Why:** Prompt нужен хотя бы раз → `do-while`.

### 8. [Analysis]
**Answer:** Бесконечный вывод `0` (infinite loop).
**Why:** Condition `x < 3` всегда true без increment.

### 9. [Analysis]
**Answer:** **B** верно.
**Why:** A — не always; C — unsafe; D — `i` scoped только в `for`.

### 10. [Understanding]
**Answer:** Чтобы **Dispose** enumerator (и underlying resource) в `finally`.
**Why:** Streams, DB readers освобождаются даже при exception в body.

### 11. [Application]
**Answer:** **6** (2 × 3).
**Why:** Inner полностью на каждый outer step.

### 12. [Understanding]
**Answer:** Когда несколько exit points (sentinel input, errors) — `break` явнее, чем сложный boolean в header.
**Why:** Читаемость vs `while (!done)` — trade-off; главное — visible exit.

### 13. [Recall]
**Answer:** Ошибка на ±1 от правильного числа итераций или index (часто `<=` vs `<`).
**Why:** Самый частый loop bug.

### 14. [Application]
**Answer:** **`foreach`** с `if (value > 7)` — index не нужен.
**Why:** LINQ `Where` тоже ok позже; для practice `foreach` + `if` достаточно.
