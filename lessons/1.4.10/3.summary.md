# Thread pool starvation — краткая выжимка

> `1.4.10` · ~10 мин

## Главная идея

**Thread pool starvation** — все (или большинство) worker threads **blocked** sync операциями; **async continuations** (`MoveNext`) и новые задачи ждут в очереди. На ASP.NET Core это главный «async deadlock-like» сценарий вместо SyncContext.

## Ключевые пункты

- `.Wait()`, `.Result`, `Thread.Sleep`, sync I/O на pool thread — причины.
- Симптомы: latency↑, queue length↑, CPU не 100%.
- `SetMinThreads` маскирует, не лечит.
- Fix: async all the way; bounded `Task.Run` для CPU.
- `ConfigureAwait(false)` не создаёт threads.
- Связь `1.4.7`: completion queue'ит `MoveNext`.
