# Exception hierarchy — краткая выжимка

> `1.5.15` · ~8 мин

## Главная идея

**`throw;`** сохраняет stack trace; **`throw ex;`** сбрасывает его. На границах API — **wrap** с **InnerException**; в catch — только то, что можешь обработать.

## Ключевые пункты

- **ArgumentException** — bad input; **InvalidOperationException** — bad state.
- **throw;** после log — rethrow same exception.
- **ExceptionDispatchInfo** — rethrow позже с stack.
- **catch (Ex ex) when (...)** — filter.
- Middleware — один global handler; data layer — не глотать всё.

## Пример

```csharp
catch (Exception ex)
{
    _logger.LogError(ex, "...");
    throw;
}
```

## Частые ошибки

- throw ex;
- empty catch
- Exception для control flow

## Мнемоника

> **Bare throw preserves; throw ex erases.**
