# .gitattributes — line endings (LF/CRLF)

> Roadmap: `0.2.14` · Узел: `0.2` — Git: ветки и collaboration · Глубина: понимание

## Learning Objectives

После лекции ты сможешь:

- Объяснить **LF vs CRLF** и почему это ломает diff и merge.
- Настроить **`.gitattributes`**: `text`, `eol=lf`, `* text=auto`.
- Описать взаимодействие **`core.autocrlf`** с `.gitattributes` на Windows/Linux.
- Применить **`git add --renormalize`** после смены policy.
- Диагностировать **«весь файл изменился»** phantom diff.
- Задать defaults для **.NET + React** cross-platform команды.

---

## Why This Matters

Backend в Linux containers, frontend с Windows laptops — без policy `git diff` показывает каждую строку изменённой из‑за `\n` vs `\r\n`. Review невозможен, CI видит лишние changes, shell scripts в Docker падают на `\r`.

**`.gitattributes`** — контракт в репозитории: «в Git храним LF; checkout по правилам». Едет с clone, в отличие от только local `core.autocrlf`. Middle dev чинит phantom diffs один раз через renormalize.

## Core Concepts

### LF и CRLF

- **LF** (`\n`) — Unix/Linux/macOS, большинство server tools.
- **CRLF** (`\r\n`) — Windows default во многих редакторах.

Git сравнивает **bytes** — разные endings = другой файл.

### .gitattributes

```gitattributes
* text=auto eol=lf

*.png binary
*.dll binary
*.sh text eol=lf
```

**`text=auto`** — detect text, normalize. **`eol=lf`** — LF в repo и при checkout (team choice). **`binary`** — без конвертации.

### Commit и checkout

При **commit** text → **LF в object database**. При **checkout** — по `eol=`.

**Best practice:** `.gitattributes` + **`core.autocrlf=false`** (repo — source of truth). **EditorConfig** `end_of_line = lf` — синхрон с editor save.

### Renormalize

После смены rules:

```bash
git add --renormalize .
git commit -m "chore: normalize line endings"
```

Один большой commit — координация с командой.

---

## Under the Hood

**Clean/smudge filters:** on add — clean to LF; on checkout — smudge to `eol`. Index хранит hash **после** clean.

**`text=auto`** — heuristic; binaries помечай явно.

**`git ls-files --eol`** — inspect index vs working tree endings.

---

## Syntax / Commands / API

```gitattributes
* text=auto eol=lf
*.bat text eol=crlf
```

```bash
git add --renormalize .
git diff --check
git ls-files --eol
```

## Examples

Starter для .NET+React; phantom diff → `--ignore-space-at-eol`; `*.sh eol=lf` для CI.

---

## Common Mistakes & Anti-patterns

Только global autocrlf; нет `binary` для images; renormalize без предупреждения команды; CRLF globally + LF CI.

---

## Production & Real-World Notes

CI Linux + Windows devs; Docker shebang; `.git-blame-ignore-revs` для formatting commit.

---

## Comparison / Trade-offs

| Подход | Плюс | Минус |
|--------|------|-------|
| .gitattributes | Единые rules | Renormalize PR |
| autocrlf per user | Без repo change | Inconsistent |

Trend: **LF everywhere** с редкими исключениями (`.bat`).

## Quick Reference

## Key Takeaways

- Endings = bytes → false full-file diffs.
- **`.gitattributes`** versioned policy.
- Renormalize после fix legacy repo.
- EditorConfig + Git вместе.
