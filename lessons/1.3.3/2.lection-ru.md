# Delegates: multicast и удаление подписчиков

> Roadmap: `1.3.3` · Node: `1.3` — C# продвинутые темы · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Объяснить **delegate** в runtime (type-safe callback + target object).
- Объявлять **custom delegate types** и отличать от **`Action`/`Func`** (`1.3.4`).
- Комбинировать handlers через **`+=`** (**multicast**) и предсказывать порядок.
- Удалять через **`-=`** с пониманием **reference equality**.
- Использовать **`GetInvocationList()`** для изоляции ошибок.
- Описать **Under the Hood**: `Invoke`, immutability, allocation.

---

## Why This Matters

Delegates — основа events, LINQ, ASP.NET middleware. **Multicast** — несколько подписчиков на один сигнал. Ошибки **`-=`** (handler не снялся → leak, double fire) и **return values** в multicast — частые production bugs.

## Core Concepts

### Delegate как callback

**Reference type**, наследник **`MulticastDelegate`**. Хранит **target** + **method pointer**.

```csharp
public delegate void Notify(string message);
Notify log = Console.WriteLine;
```

### Multicast: `+=`

Delegates **immutable** — `+=` создаёт **новый** combined delegate:

```csharp
combined += handler2;
combined("event"); // handler1, handler2, handler3 по порядку
```

**`Delegate.Combine`** строит invocation list.

### `-=` по identity

Удаляет **последнее совпадение** target + method.

**Pitfall** — новая lambda ≠ старая:

```csharp
combined += n;
combined -= msg => Console.WriteLine(msg); // не удалит n
```

Сохраняй lambda в variable.

### Return values

Multicast **`Func`**: возвращается **только последний** handler.

```csharp
f += () => 1;
f += () => 2;
f(); // 2
```

Агрегация — через **`GetInvocationList()`**.

### Exceptions

Handler2 throws → later handlers могут не выполниться. Isolate через loop + try/catch.

---

## Under the Hood

**`MulticastDelegate`**: `_invocationList`, `_invocationCount`. **`Invoke`** loops list. Каждый **`+=`** — allocation нового delegate + array.

**`event`** блокирует external **`=`** — только **`+=`/`-=`** (`1.3.5`).

---

## Syntax / Commands / API

```csharp
d += handler;
d -= handler;
Delegate[] list = d.GetInvocationList();
d?.Invoke(arg);
```

---

## Examples

```csharp
void OnData(string s) => Console.WriteLine(s);
log += OnData;
log -= OnData; // OK — same method ref
```

---

## Common Mistakes & Anti-patterns

- `-=` с новой lambda.
- Ожидание sum returns от multicast.
- Public delegate field вместо event.
- Duplicate registration без guard.
- Thread safety на shared `+=`/`-=`.

---

## Production & Real-World Notes

Domain events — часто dispatcher вместо raw multicast.

UI — leak без `-=` on dispose.

Testing — count invocation list.

---

## Quick Reference

| Op | Effect |
|---|---|
| `+=` | Append (new instance) |
| `-=` | Remove last match |
| `Func` multicast | Last return only |

---

## Key Takeaways

- **`+=`** immutable chain, order preserved.
- **`-=`** needs same reference/lambda variable.
- **`GetInvocationList()`** for control.
- Use **`event`** for protection.
