# Режимы GC: Workstation vs Server — краткая выжимка

> `1.5.9` · ~8 мин на повторение

## Главная идея

**Workstation GC** оптимизирует **короткие паузы** для интерактивных приложений; **server GC** — **throughput на multi-core** через отдельные heap'ы и параллельные GC threads. Режим не меняет корректность, но влияет на latency и загрузку CPU.

## Ключевые пункты

- **Workstation** — типично один heap; default для многих console/UI.
- **Server GC** — heap per logical CPU (default); default для **ASP.NET Core**.
- Проверка: **`GCSettings.IsServerGC`**, **`runtimeconfig.json`** (`System.GC.Server`), **`DOTNET_gcServer`**.
- **Background/concurrent GC** уменьшает stop-the-world для Gen2 — часть flavor, не отдельная «галочка» в коде.
- В **контейнерах** с малым CPU server GC может проигрывать — benchmark.
- **`GCLatencyMode.LowLatency`** — кратковременно, не «навсегда».

## Самые важные примеры

```csharp
Console.WriteLine(GCSettings.IsServerGC);
Console.WriteLine(GC.GetGCMemoryInfo().HeapCount);
```

```xml
<ServerGarbageCollection>true</ServerGarbageCollection>
```

## Частые ошибки

- Сравнивать dev F5 с production без проверки runtimeconfig.
- Server GC на 1 vCPU без load test.
- Менять только GC mode, игнорируя лишние аллокации.

## Мнемоника

> **Server serves many cores; workstation works for one user.**
