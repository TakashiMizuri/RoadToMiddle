# Stack frames и stack overflow — выжимка

> `1.5.4` · ~7 мин

## Главная идея

Каждый вызов метода создаёт **stack frame** на thread stack. Locals из **`1.1.6`** живут во frame; объекты — на heap. **StackOverflowException** — исчерпание stack (recursion, huge stackalloc).

## Ключевые пункты

- Frame: return address, params, locals, GC maps для refs.
- Active frame refs = **GC roots** (`1.5.6`).
- IL evaluation stack ≠ native call stack.
- Stack overflow often fatal — fix algorithm.

- Следующая: `1.5.5`
