# Value types vs Reference types — stack и heap

> Roadmap: `1.1.6` · Node: `1.1` — C#: основы языка · Depth: **глубоко**

## Learning Objectives

После этого урока ты сможешь:

- Различать **value types** и **reference types** по **семантике**, а не только по ключевым словам.
- Объяснить **расположение данных** в модели памяти CLR: stack frames vs managed heap (с точными нюансами).
- Рисовать **диаграмму stack/heap** для locals, fields и assignment.
- Предсказывать **поведение копирования**: копия значения vs копия ссылки.
- Связать **примитивы `1.1.3`** (`int`, `bool`, `char`, `decimal`…) с value-type storage.
- Объяснить, почему **`null`** для reference types и **`int?`**, но не для plain `int`.

---

## Why This Matters

В `1.1.3` ты узнал, что `int` хранит целое, `bool` — `true`/`false`. В `1.1.5` объявлял locals через `var` или explicit types. Модель «коробка `count` содержит 42» работает локально, но ломается при передаче в методы, хранении в fields и сравнении двух `string`.

C# делит типы на две категории с **разной семантикой копирования** и **разной историей в памяти**. Ошибки здесь — реальные баги: мутация «копии», неожиданный `NullReferenceException`, performance при копировании больших struct, путаница «почему `int` не null, а `int?` — да».

Boxing (`1.1.7`), GC, generics, async state machines опираются на этот split. Middle не заучивает «struct = stack», а понимает **что копируется**, **что отслеживает GC**, **где живут references**.

## Core Concepts

### Две категории

**Value types** хранят данные **непосредственно** в памяти переменной (переменная *есть* значение). Примитивы `1.1.3` — value types. **`struct`** — тоже value type.

**Reference types** хранят **reference** (адрес) на объект на **managed heap**. Переменная указывает на объект, не содержит его целиком. **`class`**, **`string`**, **array**, **delegate** — reference types.

**Assignment value type** — копируются **биты значения**:

```csharp
int a = 10;
int b = a;
b = 20;      // a == 10
```

**Assignment reference type** — копируется **reference**; обе переменные указывают на **один объект**:

```csharp
var list1 = new List<int> { 1, 2 };
var list2 = list1;
list2.Add(3);   // list1 тоже { 1, 2, 3 }
```

**`string`** — reference type, но **immutable** (`1.1.4`).

### Stack vs Heap — точная модель

**Stack (per thread):** stack frame метода содержит locals и parameters. Value types — **значение inline** в frame. Reference types — в frame лежит **reference**, объект — на heap.

**Managed heap:** тела reference type objects; GC их отслеживает.

**Нюанс:** value types тоже на **heap**, если они **fields класса**, элементы array, boxed или захвачены closure. Правило: **object reference types на heap; value types embedded там, где их container**.

```
Thread stack                         Managed heap
┌─────────────────────┐             ┌──────────────────────┐
│ Main()              │             │ List<int>            │
│  count: 42          │             │  _items → int[]      │
│  listRef ───────────┼────────────►│                      │
│  nameRef ───────────┼────────────►│ String "Ada"         │
└─────────────────────┘             │  struct field inline │
                                    └──────────────────────┘
```

### Примитивы 1.1.3

| Type | Category | Local |
|------|----------|-------|
| `int`, `long`, `bool`, `char` | Value | Inline |
| `float`, `double`, `decimal` | Value | Inline (`decimal` = 128-bit struct) |
| `string` | Reference | Ref → heap object |

### `null` и nullable

```csharp
string? name = null;
int count = 0;        // не «пусто» — ноль валиден
int? maybe = null;    // HasValue == false
```

`int?` — **value type** (обёртка с флагом + `T`).

### Default values

- Value types: `0`, `false`, `'\0'`
- Reference types: `null`

### Передача в методы

По умолчанию **by value**: для value type копируются bits; для reference type — **reference**, объект общий. `param = new List<int>()` не меняет caller's variable — только local ref slot. `ref`/`out`/`in` — в `1.1.13`.

## Under the Hood

### CLR

Reference types наследуют **`System.Object`**; object на heap имеет **method table pointer**.

Value types без boxing не имеют object header inline; JIT копирует **`sizeof(T)`** bytes.

**`ldloc`** — load value type; **`ldloca`** — address (для `ref` и mutating struct).

### Value types на heap

1. Field reference type class  
2. Boxing (`1.1.7`)  
3. `int[]` — elements на heap  
4. Closure display class  

**GC** reclaim unreachable heap objects. Копирование reference не создаёт object. Большие struct — CPU cost без GC.

## Examples

**Shared reference:**

```csharp
var a = new StringBuilder("hi");
var b = a;
b.Append("!");
// a → "hi!"
```

**Struct copy:**

```csharp
Point p1 = new() { X = 1, Y = 2 };
Point p2 = p1;
p2.X = 99;
// p1.X == 1
```

## Common Mistakes & Anti-patterns

«Struct always stack» — неверно. Assignment reference type ≠ deep clone. `int` для «нет значения» — используй `int?`. Огромные struct by value без `in`/`ref`.

---

## Production & Real-World Notes

Малые immutable data — struct; identity/lifetime — class. Boxing и string concat — частые источники allocations в ASP.NET Core.

---

## Comparison / Trade-offs

Value types: copy data, no null (кроме `T?`), нет inheritance. Reference types: copy ref, null allowed, inheritance, shared mutation.

---

## Quick Reference

```
Local int     → value in frame
Local string  → ref in frame → object on heap
Ref assignment → alias
Value assignment → independent copy
```

---

## Key Takeaways

- **Value = copy bits; reference = copy pointer.**
- **Heap — objects; stack frames — locals.**
- **Value types могут быть на heap** как fields/array/boxed.
- **Примитивы 1.1.3 — value; string — reference.**
- **Default pass-by-value**, не deep clone.
