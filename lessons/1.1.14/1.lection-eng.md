# Arrays: Single, Jagged, Multidimensional

> Roadmap: `1.1.14` · Node: `1.1` — C# Language Basics · Depth: practice

## Learning Objectives

After this lesson you will be able to:

- Declare and initialize **single-dimensional**, **jagged**, and **multidimensional** arrays.
- Explain **memory layout** and performance differences between jagged and rectangular arrays.
- Use **`Length`**, **`GetLength`**, **`Rank`**, and **`GetUpperBound`** correctly.
- Iterate arrays with **`for`**, **`foreach`**, and **`Span<T>`** (concept).
- Avoid **off-by-one** and **rectangular vs jagged** confusion in APIs.
- Choose the appropriate array shape for matrices, rows of variable length, and buffers.

---

## Why This Matters

Arrays are the simplest **collection** in C#: fixed-length, indexed, zero-based, contiguous for single-dimensional types. Before `List<T>` and LINQ, almost all batch data lived in arrays — and under the hood, many collections still use arrays internally (`List<T>` stores items in `T[]`). Understanding array kinds prevents bugs when you declare `int[,]` but treat it like `int[][]`, or when you assume `GetLength(0)` equals `Length` on a jagged array.

In fullstack work, arrays appear in DTO mapping, image buffers, game grids, CSV rows, JSON deserialization to arrays, and interop with native APIs. Middle developers must know not only syntax but **which shape** to pick: a chess board is often `Piece[,]`; a ragged list of user permissions per role is `string[][]`; a stream buffer is `byte[]`. Wrong choice costs clarity, performance, or both.

This lesson builds on value types and references from earlier `1.1.x` topics. Arrays themselves are **reference types** (except when used as stack buffers via `stackalloc` — a later topic). The variable holds a reference; the elements live in a contiguous block on the heap (for single-dim and multidimensional rectangular layouts).

## Core Concepts

### Single-Dimensional Arrays

A **single-dimensional array** is an ordered sequence of elements of one type, accessed by index `0 .. Length - 1`.

```csharp
int[] numbers = new int[5];           // five zero-initialized ints
string[] names = { "Ann", "Bob" };    // array initializer
int third = numbers[2];               // index 2 = third element
numbers.Length;                       // 5
```

**Declaration patterns:** `type[] name` declares a variable that will hold a reference to an array; `new type[n]` allocates the array object. **Array initializer syntax** `{ ... }` lets the compiler infer length. **Default values:** numeric elements `0`, `bool` `false`, references `null`.

**Bounds checking:** Every index access is bounds-checked at runtime. Out-of-range throws **`IndexOutOfRangeException`** (or `ArgumentOutOfRangeException` in some APIs). The compiler does not prove safety for dynamic indices.

**Immutability of length:** Once created, **`Length` is fixed**. To grow, you allocate a new array and copy — which is exactly what `List<T>` automates.

### Multidimensional (Rectangular) Arrays

**Multidimensional arrays** use comma-separated ranks in the type: `int[,]` is a **2D rectangular** array — every row has the same number of columns. Memory is a **single block** laid out in row-major order (last index varies fastest in C#/.NET).

```csharp
int[,] grid = new int[3, 4];   // 3 rows, 4 columns — 12 cells
grid[1, 2] = 7;
int rows = grid.GetLength(0);  // 3
int cols = grid.GetLength(1);  // 4
int rank = grid.Rank;          // 2
```

**Cannot** use jagged syntax `grid[1][2]` on `int[,]`. The comma index `grid[row, col]` is required. **`Length`** on multidimensional arrays is the **total element count** (rows × columns for 2D), not one dimension.

Rectangular arrays suit **fixed grids**: maps, matrices, pixels in a known width × height, tabular data where every row aligns.

### Jagged Arrays (Array of Arrays)

A **jagged array** is an array whose elements are themselves arrays — `int[][]` is "array of `int[]`". Each row can have **different length**.

```csharp
int[][] jagged = new int[3][];  // three row references, initially null
jagged[0] = new int[] { 1, 2 };
jagged[1] = new int[] { 3, 4, 5 };
jagged[2] = new int[1];

int element = jagged[1][2];     // 5 — two indexing steps
int rowCount = jagged.Length;   // 3 — outer array length
int colsInRow1 = jagged[1].Length; // 3
```

**Memory layout:** The outer array holds references to row arrays scattered on the heap — **not** one contiguous rectangle. That affects cache locality: iterating a jagged array row-by-row may jump between heap regions.

Jagged arrays suit **rows of variable length**: sparse data, lines in a file, adjacency lists in graphs, JSON arrays of arrays with differing sizes.

### Comparing Shapes

```
Rectangular int[3,4]:          Jagged int[][] with 3 rows:

┌───┬───┬───┬───┐              jagged → [0] → {1,2}
│   │   │   │   │                        [1] → {3,4,5}
├───┼───┼───┼───┤                        [2] → {6}
│   │   │   │   │
├───┼───┼───┼───┤              Outer Length = 3; inner lengths differ
│   │   │   │   │
└───┴───┴───┴───┘              One block vs many row objects
One object, 12 ints contiguous
```

**Rule of thumb:** Fixed rectangle → `T[,]` (or `T[,]` / higher ranks). Variable row sizes → `T[][]`. Single sequence → `T[]`.

### Initialization and Covariance (Reference Types Only)

```csharp
string[] a = { "x", "y" };
object[] b = a;  // allowed — array covariance for reference types
// b[0] = 123;   // runtime InvalidCastException — slot was string
```

**Array covariance** allows assigning `string[]` to `object[]` but writes must still match actual element type. Value type arrays are **not** covariant. Modern APIs prefer **`IReadOnlyList<T>`** over arrays for public parameters to avoid covariance traps.

## Under the Hood

Single-dimensional arrays inherit from **`System.Array`**. The runtime stores **length** and a pointer to element data. For **`int[]`**, elements are contiguous — good cache locality for sequential scans.

**`int[,]`** is still one array object with metadata for two dimensions; indexing `[i,j]` compiles to a formula on a flat buffer (row-major). **`int[][]`** is an array of references; each `int[]` row is a separate object with its own header and length.

**Bounds checks** emit compare + branch before access; JIT can elide checks when it proves index safe (e.g., fixed loop `for (int i = 0; i < arr.Length; i++)`).

**Zero-length arrays** are valid: `new int[0]` has `Length == 0`. **`Array.Empty<T>()`** returns a singleton empty array — preferred in libraries to avoid allocating new empty arrays repeatedly.

## Syntax / Commands / API

| Type | Syntax | Index | Length |
|------|--------|-------|--------|
| 1D | `T[]` | `a[i]` | `a.Length` |
| 2D rectangular | `T[,]` | `a[i,j]` | `a.GetLength(dim)` |
| Jagged | `T[][]` | `a[i][j]` | outer `a.Length`, inner `a[i].Length` |

```csharp
// 3D rectangular
float[,,] cube = new float[2, 3, 4];

// Jagged initializer
int[][] triangle = {
    new[] { 1 },
    new[] { 2, 3 },
    new[] { 4, 5, 6 }
};
```

**Useful static members:** `Array.Sort`, `Array.BinarySearch`, `Array.Copy`, `Array.Clear`, `Array.IndexOf`.

## Examples

### Example 1: Sum a 1D array

```csharp
static int Sum(int[] values)
{
    int total = 0;
    for (int i = 0; i < values.Length; i++)
        total += values[i];
    return total;
}
```

`foreach` works on single-dimensional arrays because `int[]` implements `IEnumerable<int>`. Multidimensional **`int[,]`** does not support simple `foreach` over all cells without nested loops or flattening.

### Example 2: Traverse rectangular grid

```csharp
static void Fill(int[,] grid, int value)
{
    for (int r = 0; r < grid.GetLength(0); r++)
        for (int c = 0; c < grid.GetLength(1); c++)
            grid[r, c] = value;
}
```

Always loop with **`GetLength(dimension)`** — hard-coding dimensions breaks when array size changes.

### Example 3: Jagged rows from input

```csharp
// Three users with different role counts
string[][] rolesByUser = {
    new[] { "Admin" },
    new[] { "User", "Editor" },
    new[] { "User" }
};

foreach (string[] roles in rolesByUser)
    foreach (string role in roles)
        Console.WriteLine(role);
```

Outer `foreach` walks row references; inner walks each row's elements.

## Common Mistakes & Anti-patterns

**Using `int[][]` syntax on `int[,]`** — compile error. Comma vs double brackets matters.

**Assuming `Length` is row count on 2D arrays** — `Length` is total cells; use **`GetLength(0)`**.

**Forgetting to allocate jagged rows** — `new int[3][]` leaves `null` rows; `jagged[0][0]` throws **`NullReferenceException`**.

**Array covariance abuse** — storing wrong reference types in covariant arrays fails at runtime.

**Using arrays when `List<T>` is needed** — frequent insert/delete in the middle favors list; array wins for fixed size and interop.

## Production & Real-World Notes

Public APIs often expose **`IReadOnlyList<T>`** or **`ReadOnlySpan<T>`** instead of `T[]` to document immutability of size and avoid covariance. Internal hot buffers remain `T[]` for simplicity.

JSON serializers map JSON arrays to **`T[]`** or **`List<T>`**; jagged JSON arrays become **`T[][]`** or `List<List<T>>`. ORMs and Dapper map SQL results to arrays rarely — **`List<T>`** dominates — but byte buffers from streams are **`byte[]`**.

For large 2D scientific data, libraries sometimes use **single `T[]` + manual indexing** (`index = row * width + col`) for performance instead of `T[,]` — measure before optimizing.

## Comparison / Trade-offs

| Need | Prefer |
|------|--------|
| Fixed grid, same columns per row | `T[,]` |
| Variable row lengths | `T[][]` |
| Dynamic grow/shrink | `List<T>` |
| Pass to legacy API | match expected shape |
| Max cache locality rectangular | `T[,]` or flat `T[]` |

**Jagged vs rectangular performance:** Rectangular one-block layout often wins sequential row scans; jagged wins when rows differ greatly in size (no wasted cells).

## Quick Reference

```
T[]       — 1D, a[i], Length
T[,]      — 2D rectangular, a[r,c], GetLength(0/1)
T[][]     — jagged, a[r][c], outer Length + inner [r].Length
```

---

## Key Takeaways

- Arrays are **reference types** with **fixed length** after allocation.
- **`T[]`**: one dimension; **`T[,]`**: rectangular block; **`T[][]`**: array of arrays, variable row lengths.
- Indexing syntax differs: comma vs chained brackets.
- Use **`GetLength(dim)`** for multidimensional ranks; **`Length`** alone is not row count for `T[,]`.
- Jagged rows must be **allocated separately**.
- Choose shape based on data geometry, not habit.
