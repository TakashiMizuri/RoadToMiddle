# Exercises: LINQ Distinct, Union, Intersect, Except

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Distinct integers
**Goal:** Remove duplicate IDs from an array.
**Steps:** Given `int[] ids = { 5, 1, 5, 2, 1, 3 }`, produce ordered unique list.
**Success criteria:** Output `5, 1, 2, 3` (first-seen order).

### E2. Union two tag lists
**Goal:** Merge unique tags from two products.
**Steps:** Two `string[]` tag arrays; case-insensitive union.
**Success criteria:** Each tag once; `"API"` and `"api"` treated as same tag.

## Medium

### M1. Role permission audit
**Goal:** Compute shared and exclusive permissions.
**Steps:** `HashSet<string>` or `IEnumerable<string>` for Editor and Viewer roles. Return `(shared, editorOnly, viewerOnly)`.
**Success criteria:** Uses `Intersect` and `Except`; correct set logic.

### M2. DistinctBy last login per user
**Goal:** From a list of login events, one row per UserId — the **most recent** event.
**Steps:** Hint: ordering + `DistinctBy`, or `GroupBy` — choose and justify.
**Success criteria:** Correct winner per user; explain trade-off vs `GroupBy`.

## Hard (stretch)

### H1. EF-style mental SQL
**Goal:** Write LINQ on in-memory `List<Customer>` that mirrors `active EXCEPT blocked`.
**Steps:** Two collections by Id; translate to `Except`; then rewrite using `Where` + `Contains` and compare readability/performance.
**Success criteria:** Same result; short paragraph on when each style wins.
