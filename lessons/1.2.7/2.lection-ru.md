# Полиморфизм: virtual/override и runtime dispatch

> Roadmap: `1.2.7` · Node: `1.2` — C# OOP · Depth: **глубоко**

## Learning Objectives

После этого урока ты сможешь:

- Определить **polymorphism** как один abstraction (base reference), много реализаций at runtime.
- Объяснить, как **`virtual`**, **`override`** и **`abstract`** работают в inheritance chain.
- Проследить **virtual method dispatch** от исходного кода через IL **`callvirt`** до выбора vtable slot.
- Различать **compile-time type** и **runtime type** и предсказывать, какой method выполнится.
- Отделить polymorphic **instance methods** от **static**, **private** и **`new`-hidden**.
- Применять polymorphism в abstractions, не нарушая **Liskov Substitution Principle**.

## Why This Matters

Урок `1.2.6` ввёл **`override`** vs **`new`**: один участвует в polymorphism, другой скрывает. Здесь — **как** runtime выбирает implementation, когда ты пишешь **`animal.Speak()`**, а **`animal`** может быть **`Dog`**, **`Cat`** или **`Robot`**.

Polymorphism — главный выигрыш OOP в C#. Payment processors, notification channels, pricing strategies, ASP.NET middleware опираются на **substitutable** implementations за shared abstraction. Без **runtime dispatch** ты misread-ишь код, пишешь **`new`** вместо **`override`**, или ломаешь callers, ужесточая behavior в derived.

Middle-разработчик debug-ит production, где вызывается «не тот» **`ToString`**, **`Equals`**, **`Handle`** — wrong concrete type или method never was **virtual**. Polymorphism — не magic, а **vtable indirection** с чёткими правилами.

## Core Concepts

### Polymorphism одной фразой

**Polymorphism** — одна operation (method call на shared abstraction) выполняет **разный code** в зависимости от **actual type объекта at runtime**, пока caller держит **base class или interface reference**.

```csharp
IEnumerable<Animal> zoo = new[] { new Dog(), new Cat() };
foreach (Animal a in zoo)
    a.Speak(); // каждая итерация — свой override
```

**`a`** имеет compile-time type **`Animal`**. Каждый object — runtime type **`Dog`** или **`Cat`**. **`Speak`** резолвится без **`if (a is Dog)`** — branching внутри virtual mechanism.

Это **subtype polymorphism**. Ещё есть **ad hoc** (overloads, compile time) и **parametric** (generics, `1.3`) — не смешивай.

### virtual, override, abstract

Method polymorphic, если base объявил **`virtual`** или **`abstract`**:

- **`virtual`** — default implementation в base; derived может **`override`**.
- **`abstract`** — нет body в base; derived **must override** (или derived тоже abstract).
- **`override`** — замена slot в virtual chain; может **`base.Method()`**.

```csharp
Shape s = new Circle(5);
s.Area(); // Circle.Area — без cast
```

**`sealed override`** в intermediate class останавливает chain (`1.2.6`).

### Compile-time type vs runtime type

| Понятие | Смысл |
|---------|--------|
| Compile-time (static) type | Для overload resolution, visibility members |
| Runtime type | Actual object на heap |

**Virtual** calls — **runtime type**. **Non-virtual**, **`new` hide**, **static**, **private** — **compile-time type**.

Cast **`((Circle)s).Area()`** не меняет virtual dispatch — runtime type уже использовался. Cast — для members вне static type.

### Что не polymorphic

**Static methods** — **`call`**, не **`override`**.

**Private methods** — не в inheritance surface.

**Struct instance methods** — по умолчанию non-virtual; polymorphism через **interface + boxing** (`1.2.14`).

**`new`-hidden** — static dispatch (`1.2.6`).

**Default interface methods** — особые правила (`1.2.10`).

### Liskov Substitution Principle (LSP)

Polymorphism несёт **contract**: derived **substitutable** для base без поломки callers. **`Bird.Fly()`** throwing для **`Penguin`** ломает код по **`Bird`**. LSP — design discipline, не syntax rule.

Production document-ит, какие virtual hooks безопасны. **`OnModelCreating`**, **`Dispose(bool)`** — extension points с documented contracts.

## Under the Hood

### Vtables и callvirt

Reference types с **virtual methods** (включая от **`object`**) несут **method table pointer**. Каждый virtual method — **slot** в chain. **`Dog`** override **`Speak`** → slot указывает на **`Dog.Speak`**.

**`animal.Speak()`** → IL **`callvirt`**:

```
1. Load reference
2. callvirt → JIT читает method table объекта
3. slot Speak → jump к Dog.Speak / Cat.Speak / ...
```

**`callvirt`** делает **null check** — virtual на null → **`NullReferenceException`** до dispatch.

**`GetType()`** — runtime type для logging; не замена polymorphism в domain logic.

### Override metadata chain

Compiler строит **override chain** в metadata. **Interface** implementation — **interface maps** на object (`1.2.9`); та же идея dispatch при call через **`IEnumerable`**.

### Devirtualization и sealed

JIT **devirtualize**-ит, когда runtime type известен — **`sealed class`**, **`sealed override`**, local analysis. **`string`** sealed помогает hot paths.

### Covariant return types (C# 9+)

```csharp
public override Dog Create() => new Dog(); // OK если base возвращает Animal
```

Dispatch rules те же; caller видит **`Animal`**, может получить **`Dog`**.

## Common Mistakes & Anti-patterns

**if/else по типам** вместо polymorphism — Open/Closed violation.

**Base без `virtual`** → accidental **`new`**.

**LSP break** — **`Square : Rectangle`** с mutable width/height.

**Virtual из base constructor** — partial construction (`1.2.6`).

## Comparison / Trade-offs

| Механизм | Dispatch |
|----------|----------|
| virtual/override | Runtime callvirt |
| new hide | Compile-time |
| Interface | Runtime interface map |
| Overload | Compile-time |

**Polymorphism vs generics** — runtime per object vs compile-time per type parameter.

## Quick Reference

- Base reference → derived behavior at runtime.
- virtual/abstract + override = polymorphic instance methods.
- callvirt + vtable = mechanism.
- LSP = substitutability obligation.

---

## Key Takeaways

- Polymorphism убирает type-switch в extensible designs.
- Runtime dispatch по **actual object type** для virtual/abstract.
- **`abstract`** forces implementation; **`virtual`** — overridable default.
- Interface dispatch — через interface mapping.
- Хороший polymorphism соблюдает LSP.
