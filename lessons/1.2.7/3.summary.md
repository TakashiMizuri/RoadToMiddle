# Полиморфизм: virtual/override runtime dispatch — краткая выжимка

> `1.2.7` · ~8 мин на повторение

## Главная идея

**Polymorphism** — вызов через base/interface reference выполняет **реализацию runtime type** объекта. Механизм: **`virtual`/`abstract` + `override`**, dispatch через **vtable** и IL **`callvirt`**.

## Ключевые пункты

- **Compile-time type** — visibility и overloads; **runtime type** — target virtual call.
- **`abstract`** — нет default body; concrete derived must **`override`**.
- **`callvirt`** — null check + slot lookup.
- **Не polymorphic**: static, private, struct default instance, **`new` hide**.
- **LSP** — derived must substitute base без сюрпризов для callers.
- **Template Method** — non-virtual orchestration + virtual/abstract hooks.
- **Covariant return** (C# 9+) — override может вернуть более derived type.

## Самые важные примеры

```csharp
Shape s = new Circle(3);
s.Area(); // Circle.Area — runtime dispatch

foreach (Animal a in zoo) a.Speak(); // без is/switch по типу
```

## «Под капотом» в одном абзаце

Object header → method table → slot per virtual method. **`override`** переназначает slot для типа. JIT может **devirtualize** при **`sealed`** или proven type. Interface calls — отдельная **interface map**, но тоже runtime resolution.

## Частые ошибки

- Long **`switch` on type** вместо polymorphism.
- Забыли **`virtual`** на base → hide или CS0506.
- Override ломает контракт base (LSP).

## Мнемоника

> **Virtual call follows the object, not the variable's declared type.**
