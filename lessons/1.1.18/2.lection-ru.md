# Nullable value types: int?, DateTime?, оператор ??

> Roadmap: `1.1.18` · Node: `1.1` — C#: основы языка · Depth: понимание

## Learning Objectives

После этого урока ты сможешь:

- Объяснить **`Nullable<T>`** (`T?` для value types) как struct с флагом **`HasValue`**.
- Отличить **`null`** у nullable value type от **`default(T)`** у обычного value type.
- Использовать **`??`**, **`??=`** и **`GetValueOrDefault`** для defaults без exceptions.
- Предсказать lifting для арифметики, сравнений и связь с SQL **`NULL`**.
- Разделить **nullable value types** и **nullable reference types** (NRT, C# 8).

---

## Why This Matters

Value types — `int`, `DateTime`, `decimal`, `bool` — всегда содержат значение; они не могут быть **`null`**. Это кажется безопасным, пока не моделируешь реальные данные: optional age в форме, незаданная дата отгрузки, nullable column в БД. Раньше использовали **sentinel values**: `-1` для «возраст неизвестен», **`DateTime.MinValue`** для «нет даты». Sentinel collides с legit data и silently проходит через вычисления.

C# 2.0 принёс **`Nullable<T>`**, синтаксис **`T?`**, позже **`??`** упростил defaults. EF Core мапит **`int?`** на nullable SQL columns. API DTOs честно выражают optional numbers. Понимание nullable **value** types нужно для EF, SQL **`NULL`**, LINQ translation — и чтобы не путать с **nullable reference types** для **`string?`**.

## Core Concepts

### Что такое Nullable<T>

**`Nullable<T>`** где **`T : struct`** — **struct** с **`bool HasValue`** и **`T Value`**. **`int?`** ≡ **`Nullable<int>`**. Можно присвоить **`int`** или **`null`** → **`HasValue = false`**.

```csharp
int? age = null;
age = 25;
```

**`.Value`** при **`HasValue == false`** → **`InvalidOperationException`**. Production code: **`GetValueOrDefault()`**, pattern matching, **`??`**.

**`default(int?)`** — **`null`**. **`default(int)`** — **`0`**. Из БД: **`NULL`** → **`null`**, **`0`** → **`0`**.

### Синтаксис T?

```csharp
int? count = 42;
DateTime? shippedAt = null;
bool? isActive = null; // tri-state
```

Присвоение **`T`** → **`T?`** implicit. **`null`** нельзя присвоить plain **`int`**.

Boxing: при **`HasValue`** box underlying value; **`null`** nullable → **`null`** reference.

### Оператор ??

**`??`**: левый operand, если не **`null`** (для **`int?`**: **`HasValue`**); иначе правый. Правая сторона **не вычисляется**, если левая ok.

```csharp
int effective = input ?? 0;
DateTime deadline = due ?? DateTime.UtcNow.AddDays(7);
```

**`??=`**: присвоить только когда left **`null`**.

### GetValueOrDefault и pattern matching

**`GetValueOrDefault()`** → **`Value`** или **`default(T)`** (для **`int?`** zero может быть misleading).

```csharp
if (age is int years) { /* use years */ }
```

### Lifting операций

Nullable operands → **lifted** operators: если operand без value, результат arithmetic **`null`**; comparisons → **`bool?`** с **`null`** = unknown.

```csharp
int? sum = a + b;   // null если b null
bool? eq = a == b;  // null, не false
```

Как SQL **`NULL`** propagation. EF Core переводит многие lifted ops в SQL **`NULL`**.

### Nullable value types vs NRT

**`int?`** — runtime struct с **`HasValue`**. **`string?`** (NRT) — compile-time annotation; runtime **`string`** как всегда. Общий **`??`**, разные проблемы.

## Under the Hood

**`Nullable<T>`** — struct в **`System`**: **`hasValue` + `value`**. **`.Value`** на empty → throw path в IL.

**`??`** компилируется в conditional с проверкой **`HasValue`**.

**`Nullable<string>`** illegal — constraint **`T : struct`**.

## Syntax / Commands / API

```csharp
int? x = null;
int y = x ?? 10;
x ??= 10;
int? z = x + 5;
if (x is int n) { }
```

| Выражение | HasValue false |
|-----------|----------------|
| `.Value` | throw |
| `GetValueOrDefault()` | `default(T)` |
| `?? d` | `d` |

## Examples

### Optional DTO

```csharp
public record CreateProductRequest(string Name, decimal Price, int? Stock = null);
```

**`Stock = 0`** и omitted/`null` — разные смыслы.

### Configuration default

```csharp
int timeout = configuration.GetValue<int?>("Request:TimeoutSeconds") ?? 30;
```

### Tri-state API filter

```csharp
bool? published = query.Published;
if (published is bool p)
    q = q.Where(x => x.IsPublished == p);
```

## Common Mistakes & Anti-patterns

Sentinel вместо **`T?`**. Blind **`.Value`**. Lifted **`false`** как definite false. Путать NRT и **`T?`**. **`GetValueOrDefault()`** когда zero — valid data.

---

## Production & Real-World Notes

PostgreSQL **`NULL` ≠ 0**. EF: translatable **`!= null`**, **`HasValue`**. JSON **`null`** vs omitted property. **`Nullable<T>`** без heap unless boxed.

---

## Comparison / Trade-offs

**`T?` vs sentinel** — clarity vs legacy. **`??` vs if** — краткость vs ветвление. **`bool?` vs enum** — anonymous unknown vs named states.

## Quick Reference

```
int? ≡ Nullable<int>   null → HasValue false
??  → left if has value else right (lazy right)

```

## Key Takeaways

1. **`T?`** + **`HasValue`**; **`null`** = absent.
2. Не **`Value`** без check — **`??`**, patterns.
3. Lifting как SQL **`NULL`**.
4. **`default(int?)` ≠ default(int)`**.
5. NRT ≠ nullable value types.
