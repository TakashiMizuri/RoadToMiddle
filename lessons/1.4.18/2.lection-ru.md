# SemaphoreSlim для throttling

> Roadmap: `1.4.18` · Node: `1.4` — C# async · Depth: практика

## Why This Matters

10 000 параллельных API/DB вызовов через **`Task.WhenAll`** ломает production: **connection limits**, **429**, **thread pool**, **max_connections**. Unlimited parallelism — частая причина outage.

**`SemaphoreSlim`** — стандартный primitive **«не более N одновременно»** в async. В отличие от **`lock`**, **`WaitAsync`** не блокирует thread pool thread — continuation при освобождении permit.

Применение: **bulk import**, **parallel HTTP**, лимит concurrent EF saves, guard non-thread-safe ресурсов. **`1.4.33`** — **`lock` vs SemaphoreSlim`** глубже; здесь — **практика throttling**.

## Core Concepts

### Counting semaphore

```csharp
var gate = new SemaphoreSlim(10, 10);
await gate.WaitAsync(ct);
try { await DoWorkAsync(ct); }
finally { gate.Release(); }
```

**`finally`** обязателен — без **`Release`** concurrency падает навсегда.

### Throttling many tasks

```csharp
var sem = new SemaphoreSlim(5);
var tasks = urls.Select(async url =>
{
    await sem.WaitAsync(ct);
    try { return await _http.GetStringAsync(url, ct); }
    finally { sem.Release(); }
});
await Task.WhenAll(tasks);
```

Все tasks стартуют, но HTTP одновременно — max 5.

### vs lock

**`lock { await ... }`** — forbidden: thread держит lock во время suspend → deadlock/starvation. **`SemaphoreSlim.WaitAsync`** — правильный async gate.

## Examples

### ThrottledDbCaller

```csharp
public async Task<T> QueryAsync<T>(Func<Task<T>> query, CancellationToken ct)
{
    await _dbGate.WaitAsync(ct);
    try { return await query(); }
    finally { _dbGate.Release(); }
}
```

Singleton; **`N`** согласовать с DB pool.

---

## Common Mistakes

Missing **`Release`**, double **`Release`**, **`Wait()`** вместо **`WaitAsync`**, new semaphore per request, **`N >> pool size`**.

---

## Key Takeaways

- **`SemaphoreSlim(N,N)`** + **`try/finally`**
- Throttling ≠ rate limiting
- Singleton для global cap
