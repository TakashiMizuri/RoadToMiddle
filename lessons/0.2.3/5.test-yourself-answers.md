# Answers: Rebase

## Answers

### 1. [Recall]
**Answer:** Hash commit'а включает **parent(s)**, tree, author, committer, message. Rebase меняет **parent chain** → другой hash при том же patch content.

### 2. [Recall]
**Answer:** **`git rebase --abort`**

### 3. [Understanding]
**Answer:** **Rebase на feature:** `main` **не меняется**; двигается/переписывается `feature`. **Merge на main:** создаётся merge commit (или FF), **`main` ref** двигается; `feature` обычно без изменений.

### 4. [Understanding]
**Answer:** Оба meld в предыдущий commit. **`squash`** объединяет и даёт edit message; **`fixup`** meld **без** сохранения message melded commit (discard).

### 5. [Understanding]
**Answer:** **Не переписывай commit'ы, которые уже на shared remote и от которых зависят другие** (pull/fetch/branch). Rebase только private/local или согласованный force-push feature.

### 6. [Application]
**Answer:** `git rebase -i HEAD~3` (или ~N), первый `pick`, остальные `fixup`/`squash`, save; затем push с lease если уже на remote.

### 7. [Application]
**Answer:** Fix files → `git add` → **`git rebase --continue`**. Повторять до конца chain.

### 8. [Application]
**Answer:** **`git push --force-with-lease origin feature`**

### 9. [Analysis]
**Answer:** Pull может создать **duplicate commit'ы**, messy merge, или не fast-forward. Нужен reset/rebase на новый `develop` или re-clone — иначе history fork локально и на remote.

### 10. [Analysis]
**Answer:** **Неверно.** Objects остаются; unreachable с ref до **gc**. **Reflog** ещё хранит старые tips.

### 11. [Recall]
**Answer:** Commit **4 шага назад** по first-parent chain от HEAD — база для списка из 4 commit'ов в `-i`.

### 12. [Understanding]
**Answer:** После rebase tip `feature` — **прямой descendant** tip `main` (linear extension). Merge = FF без divergence.

### 13. [Application]
**Answer:** **`git rebase --abort`**

### 14. [Analysis]
**Answer:** Rebase **упрощает review** (linear diff на актуальном main) и уменьшает conflicts при merge; **не обязателен** если squash всё равно сожмёт на main — но без rebase PR diff может включать merge noise и stale base conflicts.

### 15. [Understanding]
**Answer:** **`--force-with-lease`** push'ит только если **remote ref** всё ещё тот, что ты ожидал (не перезаписывает чужие новые commit'ы на той же ветке).
