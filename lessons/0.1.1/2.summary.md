# VCS: local vs centralized vs distributed — краткая выжимка

> `0.1.1` · ~8 мин на повторение

## Главная идея

**Version Control System (VCS)** хранит историю изменений файлов, чтобы команда могла collaborate, откатываться и audit'ить код. Существует три поколения архитектур: **local → centralized → distributed**. **Git** — distributed VCS, и это industry standard для разработки.

## Ключевые пункты

- **Local VCS** (RCS) — история только на одном ПК. Нет командной работы.
- **Centralized VCS** (SVN, CVS) — один central server = единственный источник правды. Checkout/commit через сеть. Server down = проблемы.
- **Distributed VCS** (Git, Mercurial) — **полная копия репозитория** у каждого разработчика. Commit локально без сети. Remote (GitHub) — для sync, не единственное хранилище history.
- Git победил: скорость, дешёвые branches, экосистема GitHub, open source.
- **Anti-pattern:** использовать Git как SVN (все пушат в main без branches/PR).

## Самые важные примеры

**Centralized (SVN-style):**
```bash
svn checkout https://svn.company.com/repo/trunk
# edit files...
svn commit -m "Fix login bug"   # requires network, server is authority
```

**Distributed (Git-style):**
```bash
git clone https://github.com/company/app.git   # full history copied locally
git commit -m "Add feature"                     # local, no network
git push origin feature-branch                  # share when ready
```

**Аналогия:**
- Centralized = одна библиотека с master-каталогом. Сгорела — катастрофа.
- Distributed = у каждого полная копия каталога и книг. Sync между копиями.

## «Под капотом» в одном абзаце

В centralized модели клиент **запрашивает у server** файлы и revisions — server хранит authoritative history. В distributed модели каждый clone содержит **objects, commits, branches** в `.git/` — push/fetch просто **обмениваются objects**, которых у другой стороны ещё нет. Remote не «владеет» history — он **convention point** для команды.

## Частые ошибки

- Думать, что Git = backup (Dropbox). Git = history + branches + merges + authorship.
- Думать, что distributed = нет central server. Server есть (`origin`), но он не единственная копия.
- Использовать Git без feature branches и code review — теряешь 80% пользы.

## Мнемоника

> **Local** = один, **Central** = один server для всех, **Distributed** = все хранят всё, server для sync.
