# Answers: NuGet and CPM

### 1. **Direct** — in csproj; **transitive** — pulled by dependencies.
### 2. **`obj/project.assets.json`** (per project).
### 3. **`Directory.Packages.props`** at repo root (typical).
### 4. **Single version source**, less drift, easier upgrades/reviews.
### 5. **Package downgrade** detected — incompatible version constraints.
### 6. **`dotnet list package --include-transitive`**
### 7. **`dotnet nuget why Project.csproj Newtonsoft.Json`**
### 8. **`ManagePackageVersionsCentrally=true`** + **`PackageVersion`** entries; remove Version from csproj **PackageReference**.
### 9. **Risky** — non-reproducible builds; pin exact in CI.
### 10. **Align** versions — upgrade packages, explicit **PackageReference** to unified version, or CPM pin; resolve NU1605.
### 11. **`dotnet list package --vulnerable`**
### 12. **No** — global NuGet cache; SDK-style uses PackageReference + restore.
