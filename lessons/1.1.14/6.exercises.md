# Exercises: Arrays

> Complete after reading lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. 1D stats
**Goal:** Min, max, average of `int[]`.
**Steps:** Create array `{ 3, -1, 7, 7, 0 }`. Loop once; print min, max, average as `double`.
**Success criteria:** Single pass; handles empty array (print message, no divide by zero).

### E2. Rectangular board
**Goal:** Initialize `char[,]` board 8×8 with `'.'`.
**Steps:** Nested loops with `GetLength`; set `[0,0]` to `'R'`; print one row.
**Success criteria:** Uses comma indexer; correct dimensions.

## Medium

### M1. Jagged triangle
**Goal:** Build Pascal-style row count triangle as `int[][]`.
**Steps:** 5 outer rows; row `i` has `i+1` elements valued `1` (simple placeholder).
**Success criteria:** Each inner array allocated; no null rows; variable inner lengths.

### M2. Transpose attempt (rectangular)
**Goal:** Understand limitation of non-square transpose.
**Steps:** Given `int[2,3]` filled with `r*10+c`, write method returning **`int[3,2]`** transposed.
**Success criteria:** `result[c,r] == source[r,c]`; uses new rectangular array.

### M3. Array vs List
**Goal:** Feel fixed length.
**Steps:** Start with `int[] arr = {1,2,3}`. "Add" 4 by allocating `new int[4]`, `Array.Copy`, assign 4th — compare steps to `List<int>.Add`.
**Success criteria:** Written comment: when array vs List is appropriate.

## Hard (stretch)

### H1. Ragged to flat
**Goal:** Flatten `int[][]` to `int[]` with known total count.
**Steps:** Two passes or one with precomputed total length; copy each row sequentially.
**Success criteria:** Correct element order; no off-by-one.

### H2. Safe get
**Goal:** Helper without exceptions for bad index.
**Steps:** `static bool TryGet(int[] arr, int index, out int value)` — pattern from `1.1.13`.
**Success criteria:** Returns false for negative or `>= Length`; no throw on expected miss.
