# Exercises: LINQ Any, All, Contains

## Easy

### E1. Replace Count
**Tasks:** Refactor `items.Where(x => x.Active).Count() > 0` → **`Any`**. Same for `== 0` → **`!Any`**.

### E2. Vacuous All
**Tasks:** Demo empty list **`All(x => false)`** returns true. Fix validator requiring at least one line.

## Medium

### M1. Order validator
**Tasks:** **`IsValidOrder`**: non-empty lines, all Qty>0, all UnitPrice>=0, **`Any`** line with Total>0 optional stretch.

### M2. Role gate
**Tasks:** **`CanAccessAdmin(IReadOnlyList<string> roles)`** — safe for empty; case-insensitive Admin/Moderator.

## Hard

### H1. EF-style mock
**Tasks:** **`IQueryable`** in-memory with **`Any`** vs **`Count() > 0`** — note both execute; discuss which you'd use in real **`DbContext`** handler and why.
