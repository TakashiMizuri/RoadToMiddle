# Exercises: .gitattributes and Line Endings

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Observe EOL metadata
**Goal:** Use `git ls-files --eol`.

**Steps:**
1. Create `eol-test.txt` with a few lines; commit with default settings.
2. Run `git ls-files --eol eol-test.txt` before and after adding `* text=auto eol=lf` + renormalize.

**Success criteria:** Explain output columns (i/w attr).

---

### E2. Add .gitattributes
**Goal:** Minimal policy file.

**Steps:**
1. Add `.gitattributes` with `* text=auto eol=lf` and one binary extension.
2. Commit policy alone first (team practice).

**Success criteria:** File committed with LF endings.

---

## Medium

### M1. Simulated phantom diff
**Goal:** Connect bytes to diff noise.

**Steps:**
1. On Windows (or convert with tool), make `phantom.cs` CRLF-only change from LF version in repo.
2. `git diff` line count vs `git diff --ignore-space-at-eol`.
3. Fix via attributes + renormalize.

**Success criteria:** Short write-up: why review was noisy and how fix works.

---

### M2. Shell script CI failure
**Goal:** `*.sh text eol=lf`.

**Steps:**
1. Add `run.sh` with shebang; accidentally save CRLF.
2. Add rule; renormalize; verify `file run.sh` or hex dump shows LF only.

**Success criteria:** Document rule line and renormalize command used.

---

## Hard (stretch)

### H1. Team rollout plan
Write 6-step plan to introduce `.gitattributes` to existing 50k-line repo with active branches (announce, merge window, renormalize PR, README, autocrlf doc, blame ignore rev).

**Success criteria:** Mentions coordination and single renormalize commit.

---

### H2. Exception files
**Goal:** When CRLF is intentional.

Research why some teams use `*.sln text eol=crlf` or `*.bat text eol=crlf`. Write when you would **not** use global `eol=lf` on every extension.

**Success criteria:** At least two file types with justified exceptions.
