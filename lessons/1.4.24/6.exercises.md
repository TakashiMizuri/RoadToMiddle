# Exercises: Task.Run vs Task.Factory.StartNew

> Complete after lectures and test. Hints only.

## Easy

### E1. Task<Task> reproduction
**Goal:** Demonstrate premature completion.
**Steps:** StartNew with async lambda + Delay 3s; await outer; log timestamp before/after; observe elapsed < 3s.
**Success criteria:** Explain in one paragraph why; fix with Unwrap or Task.Run.

### E2. Safe migration
**Goal:** Replace StartNew CPU call with Task.Run.
**Steps:** Legacy snippet `Task.Factory.StartNew(() => Compute(n))` → Task.Run equivalent with same result.
**Success criteria:** Unit test or console output match.

## Medium

### M1. ASP.NET CPU burst
**Goal:** Correct Task.Run placement.
**Steps:** Minimal API: upload bytes, CopyToAsync, CPU hash (SHA256) via Task.Run with RequestAborted.
**Success criteria:** No Task.Run on CopyToAsync; cancellation passes through.

### M2. Code review
**Goal:** Find 3 bugs in provided snippet (invent or from codebase).
**Steps:** Document StartNew+async, sync HTTP in Run, missing cancellation.
**Success criteria:** Fix list with one-line rationale each.

## Hard (stretch)

### H1. LongRunning justification
**Goal:** Write ADR paragraph when StartNew LongRunning is acceptable vs BackgroundService.
**Success criteria:** Trade-offs for blocking legacy listener in Windows service context.
