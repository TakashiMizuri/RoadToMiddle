# Answers: UnobservedTaskException

## Scoring

- 90–100%: отлично · 80–89%: pass · <80%: повтори lection + summary

## Answers

### 1. [Recall]
**Answer:** Faulted task finalized/GC без await, `.Exception`, или sync wait на нём.
**Why:** Runtime observation gate.

### 2. [Recall]
**Answer:** `TaskScheduler.UnobservedTaskException`.
**Why:** ThreadPoolTaskScheduler raises it.

### 3. [Understanding]
**Answer:** Процесс обычно не завершается; event + `SetObserved()` вместо crash by default.
**Why:** .NET 4.5 policy change.

### 4. [Understanding]
**Answer:** Request exceptions идут в middleware/filters при await; unobserved — фоновые discarded tasks вне pipeline.
**Why:** Different propagation paths.

### 5. [Application]
**Answer:** После completion: `if (task.IsFaulted) _ = task.Exception;` или inspect inner.
**Why:** Accessing Exception property observes.

### 6. [Application]
**Answer:** Startup (`Program.cs` / host), один раз.
**Why:** Global scheduler event.

### 7. [Analysis]
**Answer:** Не сразу — при GC/finalize task; timing nondeterministic.
**Why:** Finalization-driven.

### 8. [Understanding]
**Answer:** Помечает exception observed по политике CLR; предотвращает escalation.
**Why:** Documented API on event args.

### 9. [Analysis]
**Answer:** Держит tasks и exceptions alive; delayed/bursty unobserved events; memory pressure.
**Why:** Holder until observed.

### 10. [Application]
**Answer:** Подписка на event, `SetObserved()`, `foreach` по `e.Exception.Flatten().InnerExceptions`, `LogError`.
**Why:** Standard production pattern.

### 11. [Understanding]
**Answer:** Нет — только логирование; scope bug нужно чинить архитектурой.
**Why:** Handler ≠ DI fix.

### 12. [Recall]
**Answer:** await; read `.Exception` when faulted; Wait/Result/GetResult (sync).
**Why:** Observation mechanisms.

### 13. [Analysis]
**Answer:** Нет — отдельный unhandled/`SynchronizationContext` path для async void.
**Why:** Not a Task observation model.

### 14. [Understanding]
**Answer:** Исключить `OperationCanceledException` из логирования в handler.
**Why:** Cancel often benign if misused.

### 15. [Application]
**Answer:** `try { await work(ct); } catch (Exception ex) { log; }` — не rethrow unhandled из loop без политики retry.
**Why:** Immediate observation in consumer.
