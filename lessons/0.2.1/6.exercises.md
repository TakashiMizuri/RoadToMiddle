## Easy

### E1. Ref or fiction?
**Goal:** Internalize branch-as-pointer model.

For each statement, write **true** or **false** and one-sentence justification:

1. Creating `git branch hotfix` copies all blob objects into a new namespace.
2. Two branches pointing at the same commit share identical history backward from that commit.
3. `git switch feature` updates `.git/HEAD` to `ref: refs/heads/feature`.
4. Deleting a branch with `git branch -d` immediately removes all commits that were only on that branch from disk.

**Success criteria:** 4/4 with correct reasoning referencing refs vs objects.

## Medium

### M1. Detached HEAD rescue drill
**Goal:** Practice safe workflow off a branch.

**Setup:** In a throwaway repo (or `--git-dir` exercise repo):

1. Note current branch and latest commit hash.
2. `git switch --detach HEAD~1` (or a specific hash).
3. Make a small change, commit with message `experiment`.
4. **Without losing the commit**, return to your original branch and preserve `experiment` on a new branch named `lab/detached-demo`.

Write the exact command sequence you used.

**Success criteria:** `git log lab/detached-demo -1` shows `experiment`; original branch unchanged; no reflog-only orphan.

---

### M2. Explain the block
**Goal:** Understand switch safety.

**Scenario:** You edited `OrderService.cs` on `main`. Before commit, you run `git switch feature/orders`. Git prints an error about local changes.

Write 4–6 sentences explaining:
- What Git compares before allowing switch.
- Three valid ways to proceed (name commands).
- Which option you would choose for a half-finished experiment vs a finished fix.

**Success criteria:** Mentions tree/checkout overwrite risk; includes stash, commit, or restore/discard paths.

## Hard (stretch)

### H1. Predict the refs
**Goal:** Graph + ref mental model.

**Starting state:**

```
main:    C1 — C2 — C3
feature: points at C2
HEAD → main
```

Actions in order:

```bash
git switch feature
git commit -m "F1"        # new commit C4
git commit -m "F2"        # new commit C5
git switch main
git commit -m "M1"        # new commit C6
```

Without running commands (or run in a test repo and verify), answer:

1. What commit does `refs/heads/main` point to?
2. What commit does `refs/heads/feature` point to?
3. Do C4 and C6 share a parent? Which?

**Success criteria:** main → C6, feature → C5, common ancestor C2; C4 parent C3? Wait let me recalculate.

Starting:
- main: C1-C2-C3, feature at C2
- switch feature, commit F1 (C4 parent C2), F2 (C5 parent C4)
- switch main, commit M1 (C6 parent C3)

So main → C6, feature → C5, C6 and C5 share ancestor C2 but not direct parent - C6 parent is C3, C5 parent is C4.

Good - the exercise is correct.
