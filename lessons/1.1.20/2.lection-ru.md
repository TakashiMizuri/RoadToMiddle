# Naming conventions и XML doc comments

> Roadmap: `1.1.20` · Node: `1.1` — C#: основы языка · Depth: практика

## Learning Objectives

После этого урока ты сможешь:

- Последовательно применять **Microsoft C# naming conventions** для types, members, parameters, private fields.
- Писать **XML documentation comments** (`///`) для IntelliSense и generated API docs.
- Корректно использовать **`param`**, **`returns`**, **`exception`**, **`see`**, **`seealso`**, **`cref`**.
- Включать **`GenerateDocumentationFile`** и управлять **CS1591** в team projects.
- Ревьюить names и docs в PR на clarity, не ceremony.

## Why This Matters

Ты закрываешь узел **C# basics** (`1.1`). Код компилируется — но команда должна его **maintain**. **Naming** — первый слой документации: **`CalculateTotal`** vs **`Calc`**, **`cancellationToken`** vs **`ct`**. Inconsistent naming заставляет декодировать каждый file; conventions освобождают внимание для business logic.

**XML docs** — второй слой: machine-readable summaries на public API. IDE показывает их в IntelliSense; **DocFX** и NuGet.org рендерят для consumers. Middle-разработчик пишет libraries, документирует API для frontend, ревьюит juniors, чей **`///`** дублирует имя метода без смысла.

Урок практический: conventions из real .NET repos (ASP.NET Core), docs, которые помогают под pressure.

## Core Concepts

### Framework Design Guidelines: naming

**Types** — **PascalCase**: **`OrderService`**, **`PaymentStatus`**. **Interfaces** — prefix **`I`**: **`IEmailSender`**. Generic params — **`T`**, **`TKey`**, **`TValue`**.

**Methods**, **properties**, **events** — **PascalCase**: **`GetByIdAsync`**, **`CreatedAt`**.

**Parameters**, **locals** — **camelCase**: **`orderId`**, **`cancellationToken`**. Без Hungarian notation.

**Private fields** — часто **`_camelCase`**: **`_logger`**, **`_dbContext`**. Match project style.

**Async methods** — suffix **`Async`**, return **`Task`/`Task<T>`**.

**Namespaces** — **PascalCase** dotted: **`Company.Product.Feature`**.

Types — **nouns**; methods — **verbs**. Boolean properties — **`Is`/`Has`/`Can`**.

Abbreviations только universal: **`Id`**, **`Url`**, **`Async`**.

### Consistency beats personal taste

**`.editorconfig`**, analyzers (**`IDE1006`**, **`CA1707`**). В проекте — **match existing style**. Public API — stricter; private — refactor freely.

### XML documentation basics

**`///`** над types и members. Compiler пишет **`.xml`** при **`GenerateDocumentationFile`**.

```csharp
/// <summary>
/// Calculates the total amount for an order including tax and shipping.
/// </summary>
/// <param name="orderId">The unique identifier of the order.</param>
/// <param name="cancellationToken">Token to cancel the database lookup.</param>
/// <returns>The total in the order's currency.</returns>
/// <exception cref="OrderNotFoundException">Thrown when no order exists for <paramref name="orderId"/>.</exception>
public Task<decimal> CalculateTotalAsync(Guid orderId, CancellationToken cancellationToken = default)
```

**`<summary>`** — complete sentences; **first sentence** часто alone в tooltip.

**`<param>`** — name must match exactly.

**`<returns>`** — смысл return value.

**`<exception cref="">`** — thrown types; **`cref`** resolves to type.

**`<paramref>`**, **`<see cref="">`**, **`<seealso>`**, **`<remarks>`**, **`<example>`**.

### cref

**`cref="CalculateTotalAsync(System.Guid, System.Threading.CancellationToken)"`**. Generics: **`List{T}`**. Wrong **`cref`** → broken doc build.

### Что документировать

**Public/protected** API. Private — обычно **`//`**. Не restate signature:

```csharp
/// Display name shown in the storefront; may differ from internal SKU label.
public string DisplayName { get; set; }
```

Document **null**, **units** (ms, UTC), **side effects**.

### GenerateDocumentationFile

```xml
<GenerateDocumentationFile>true</GenerateDocumentationFile>
```

**CS1591** — missing docs on public members. Libraries document all; prototypes may **`NoWarn`** временно.

## Under the Hood

Compiler parses **`///`** → doc XML рядом с assembly. IDE loads **`YourLib.xml`** from NuGet package. Comments **не в IL** — zero runtime cost.

Member names в XML: **`M:Namespace.Type.Method(params)`** — metadata для IntelliSense.

## Syntax / Commands / API

| Element | Convention | Example |
|---------|------------|---------|
| Class | PascalCase | `OrderRepository` |
| Interface | I + PascalCase | `IOrderRepository` |
| Method | Verb + Async | `GetByIdAsync` |
| Parameter | camelCase | `orderId` |
| Private field | _camelCase | `_logger` |

## Examples

### Minimal API handler

**`MapOrderEndpoints`**, **`GetOrderByIdAsync`**, full **`cancellationToken`** name.

### Exception type with XML

Document **`OrderNotFoundException`**, constructor **`param`**, property **`OrderId`**.

### Generic interface

**`<typeparam name="TEntity">`**, **`FindAsync`** returns **`TEntity?`** — document null.

## Common Mistakes & Anti-patterns

Cryptic abbreviations. Missing **`Async`**. XML duplicates signature. Wrong **`param name`**. Mixed **`_field`** / **`m_field`**. Public rename без **`[Obsolete]`** path.

---

## Production & Real-World Notes

Swashbuckle **`IncludeXmlComments`** → OpenAPI from same **`///`**. Review: public members need summary when policy requires. Industry XML docs — **English**.

---

## Comparison / Trade-offs

**XML vs //** — tooling vs implementation notes. **Verbose names** in shared code; short locals in tight scope OK. **StyleCop strict** vs startup velocity.

## Key Takeaways

1. **PascalCase/camelCase/_camelCase** — mainstream .NET.
2. **`I`**, **`Async`**, **`TKey`** patterns.
3. **`///`** → IntelliSense + package docs.
4. **First summary sentence** — главная line.
5. **`cref`**, **`param name`** must be accurate.
6. **`GenerateDocumentationFile`** для libraries.
