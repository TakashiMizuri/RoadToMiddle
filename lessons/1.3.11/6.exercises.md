# Exercises: IEnumerable vs IQueryable

> Complete after `1.3.11` lecture and test. EF optional for some tasks — mental model first.

## Easy

### E1. Trace execution location
**Goal:** Label CLR vs provider.
**Steps:** For 5 LINQ snippets (list vs `AsQueryable` vs pseudo-EF), mark where each operator runs.
**Success criteria:** 5/5 correct labels.

### E2. AsEnumerable placement
**Goal:** Boundary awareness.
**Steps:** Pseudocode EF query with untranslatable predicate — write wrong vs right **`AsEnumerable`** placement in comments.
**Success criteria:** Wrong version loads full table identified.

## Medium

### M1. Materialization audit
**Goal:** Fix N+0 / full table load.
**Steps:** Given buggy service method (`ToList` first), refactor to filter/project in SQL then materialize.
**Success criteria:** Single bounded query described.

### M2. Repository return type
**Goal:** API design.
**Steps:** Design interface: search orders with optional filters. Choose return types for repository vs service vs controller (3 layers).
**Success criteria:** No `IQueryable` past DbContext scope.

## Hard (stretch)

### H1. IEnumerable from IQueryable
**Goal:** Type pitfalls.
**Steps:** Minimal console demo: assign `IQueryable` to `IEnumerable`, add second `Where` — explain which overload runs (requires EF or `AsQueryable` list mock).
**Success criteria:** Written explanation of compile-time vs runtime types.

### H2. ToQueryString sketch
**Goal:** EF preview.
**Steps:** If EF project available: log `ToQueryString()` for composed query; change `AsEnumerable` position, compare SQL. If not: describe expected SQL difference in prose.
**Success criteria:** SQL changes with composition before materialize.
