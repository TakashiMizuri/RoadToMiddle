# Lambda expressions: expression vs statement body

> Roadmap: `1.3.6` · Node: `1.3` — C# продвинутые темы · Depth: практика

## Learning Objectives

После урока ты сможешь:

- Писать **expression-bodied** и **statement-bodied** lambdas.
- Объяснить **closure**, **display class**, связь с memory leaks (`1.3.5`).
- Использовать lambdas в **LINQ**, **delegates**, **events**.
- Применять **static lambda** (C# 9+) без capture.
- Избегать **loop closure**, **async void Action**, EF **`Expression<Func<>>`** pitfalls.

---

## Why This Matters

Lambdas — default syntax для LINQ, Task, events, ASP.NET Configure. Компiler генерирует **display class** для captures — innocent syntax, subtle bugs в **`for` + lambda**.

---

## Core Concepts

### Syntax

**Expression:**
```csharp
Func<int, int> square = x => x * x;
```

**Statement:**
```csharp
Func<int, int> f = n =>
{
    if (n <= 1) return 1;
    return n * 2;
};
```

Explicit types: `(int x) => x.ToString()`. Zero params: `() => ...`.

### Closures

```csharp
int threshold = 10;
Func<int, bool> above = x => x > threshold;
threshold = 20; // visible when invoked!
```

Display class holds captured fields. Delegate lifetime extends captured objects — **event leak** с `this`.

### Loop bug

```csharp
for (int i = 0; i < 3; i++)
    actions.Add(() => Console.WriteLine(i));
// prints 3,3,3 — fix: int copy = i inside loop
```

C# 5+ **foreach** — separate variable per iteration; **for** — shared.

### Static lambda

```csharp
Func<int, int, int> add = static (a, b) => a + b;
```

No capture, no display class allocation.

### async lambda

```csharp
Func<Task<int>> work = async () => { await Task.Delay(1); return 1; };
```

**Func<Task>**, not **async void Action**.

### Expression trees

```csharp
Expression<Func<User, bool>> e = u => u.Age >= 18; // EF SQL
Func<User, bool> f = u => u.Age >= 18;               // in-memory
```

## Under the Hood

Roslyn: static method OR instance on display class. **Expression** → AST for providers. **Local functions** — alternative для recursion.

## Examples

```csharp
var active = users.Where(u => u.IsActive).ToList();
builder.Configure(o => o.Timeout = TimeSpan.FromSeconds(30));
button.Click += (_, _) => DoWork();
```

## Common Mistakes

- for-loop closure.
- async void on Action.
- Lambda `-=` without stored ref.
- EF: Func instead of Expression.

---

## Quick Reference

| Form | Use |
|---|---|
| `x => expr` | single expression |
| `x => { ... return v; }` | multi-statement |
| `static x =>` | no capture |

---

## Key Takeaways

- Expression vs statement body.
- Closures extend lifetime.
- static lambda for hot paths.
- Copy loop variable in for+lambda.
