# Формы: `form`, `label`, inputs — краткая выжимка

> `0.5.7` · ~8 мин

## Главная идея

`<form>` задаёт куда и как уходят данные; каждый control нуждается в `<label>` и **`name`** для server binding. Тип `input` — hint для keyboard, validation и semantics.

## Ключевые пункты

- **`method="post"`** — mutations; **`get`** — idempotent search в URL.
- **`enctype="multipart/form-data"`** — обязателен для **`type="file"`**.
- **`label for="id"`** + **`input id name`** — a11y и hit area.
- **Radio** — один **`name`**, разные **`value`**; **`fieldset`/`legend`**.
- **Checkbox** — independent; unchecked не submit’ится.
- **`textarea`** — multi-line; **`select`/`option`** — enum; **`value`** option → server.
- **`hidden`** — metadata; не security boundary alone.
- Placeholder ≠ label.

## Примеры

```html
<label for="email">Email</label>
<input id="email" name="email" type="email" autocomplete="email" />
```

```html
<form method="post" enctype="multipart/form-data" action="/upload">
  <input name="document" type="file" accept=".pdf" />
</form>
```

## Частые ошибки

- Нет `name` → поле не в payload.
- File upload без multipart.
- Radio с разными `name`.

## Правило большого пальца

**Label + name + type** на каждом поле перед CSS и React.
