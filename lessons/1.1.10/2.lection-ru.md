# Циклы: for, while, do-while, foreach

> Roadmap: `1.1.10` · Node: `1.1` — C#: основы языка · Depth: practice

## Learning Objectives

После этого урока ты сможешь:

- Повторять работу через **`for`**, **`while`**, **`do-while`** и **`foreach`**, объясняя, когда что уместно.
- Управлять счётчиками и **условиями завершения**, избегая infinite loops и off-by-one.
- Обходить **arrays**, **`List<T>`** и **`IEnumerable<T>`** через `foreach`.
- Понимать, почему нельзя **менять collection** во время `foreach`.
- Комбинировать циклы с **`break` и `continue`** (подробно в `1.1.11`).

## Why This Matters

Ветвление из `1.1.9` решает, *какой путь* взять один раз. Циклы решают, *сколько раз* идти по пути. Обработать каждую строку CSV, retry HTTP три раза, отрисовать строки таблицы, суммировать числа до `quit` — всё это loops. Junior запоминает syntax; middle выбирает loop, где **invariants** видны читателю и компилятору.

В backend C# ты будешь loop over query results, batch updates, stream `IAsyncEnumerable<T>` (позже). С первого дня loops — в console tools, test data, алгоритмах. `for` при нужном index vs `foreach` только для values предотвращает subtle bugs. `while (true)` без явного exit — unmaintainable code. Этот урок — toolkit до LINQ; LINQ сам построен на iterators, которые **`foreach`** потребляет — модель важна даже при `.Where().Select()`.

## Core Concepts

### Идея итерации

**Iteration** — повтор block, пока **termination condition** не станет true (или вечно, если condition забыли — **infinite loop**). У каждого loop:

1. **Initialization** — начальное состояние.
2. **Condition** — ещё одна итерация?
3. **Update** — как меняется state (иногда implicit).

Разные keywords подчёркивают разные части triple. Выбирай keyword, где важное **наиболее видно** — меньше ошибок.

### Цикл `for`

**`for`** идеален, когда известно (или вычисляется) **число итераций**, особенно с **index**:

```csharp
for (int i = 0; i < 5; i++)
{
    Console.WriteLine($"Iteration {i}");
}
```

Header: **init** (`int i = 0`), **condition** (`i < 5`), **iterator** (`i++`). Все части optional; `for (;;)` — infinite loop (лучше `while` с named condition).

**Off-by-one** — классический bug: `i <= array.Length` вместо `i < array.Length` — read past end. Safe pattern: **`0` … `Length - 1`**:

```csharp
int[] ids = { 10, 20, 30 };

for (int i = 0; i < ids.Length; i++)
{
    Console.WriteLine(ids[i]);
}
```

`for`, когда нужен **index** — параллельные arrays, char по позиции в string.

### Цикл `while`

**`while`** проверяет condition **до** итерации. Если изначально false, body **может не выполниться ни разу**:

```csharp
int remaining = 3;

while (remaining > 0)
{
    Console.WriteLine($"Retries left: {remaining}");
    remaining--;
}
```

`while`, когда число итераций **неизвестно заранее** — read until EOF, poll flag, drain queue. Condition должен когда-нибудь стать false; иначе infinite loop и зависший console.

### Цикл `do-while`

**`do-while`** проверяет condition **после** body — body выполнится **минимум один раз**:

```csharp
string? input;

do
{
    Console.Write("Enter yes or no: ");
    input = Console.ReadLine();
}
while (input != "yes" && input != "no");
```

Для **menu prompts** и **validation**, где prompt нужен хотя бы раз. Если zero iterations допустимы — `while`. Trailing semicolon после `while (...)` обязателен.

### Цикл `foreach`

**`foreach`** обходит **`IEnumerable<T>`** (arrays, lists, LINQ) без manual index:

```csharp
List<string> names = new() { "Ada", "Grace", "Linus" };

foreach (string name in names)
{
    Console.WriteLine(name);
}
```

Compiler превращает `foreach` в **`try/finally`** с **enumerator** и `Dispose` для `IDisposable` — streams, DB readers позже.

**Read-only iteration:** для **`struct`** iteration variable — **copy** (кроме `ref readonly foreach` C# 12+). Мутация loop variable не меняет collection.

**Не modify collection** during `foreach` (add/remove) — **`InvalidOperationException`**. Remove — **`for` backwards**, **`RemoveAll`**, новый list.

### Выбор loop

| Need | Loop |
|------|------|
| Index 0..n-1, fixed count | `for` |
| Unknown count, maybe zero | `while` |
| At least one run | `do-while` |
| Elements без index | `foreach` |

Transform/filter — LINQ (`Select`, `Where`) declarative; под капотом всё равно iteration.

### Вложенные циклы

Loops в loops — **2D**: tables, grids:

```csharp
for (int row = 1; row <= 3; row++)
{
    for (int col = 1; col <= 3; col++)
    {
        Console.Write($"({row},{col}) ");
    }
    Console.WriteLine();
}
```

Inner полностью на каждый шаг outer. Total = product counts. Nested loops → O(n²) — ok для demo, suspect на больших data.

## Under the Hood

`foreach` desugars примерно в enumerator + `MoveNext` + `Dispose`. Modify collection ломает enumerator state. **`for` на arrays** JIT оптимизирует с bounds checks; иногда checks elided.

**Iterator variable** в `for` scoped к header — nested `int i` legal, shadows outer.

## Syntax / Commands / API

```csharp
for (init; condition; iterator) { }
while (condition) { }
do { } while (condition);
foreach (Type item in enumerable) { }
```

Helpers: `.Length`, `.Count`, `Enumerable.Range(0, n)`.

## Examples

### Пример 1: сумма до sentinel

```csharp
Console.WriteLine("Enter numbers (0 to finish):");

int sum = 0;

while (true)
{
    Console.Write("> ");
    if (!int.TryParse(Console.ReadLine(), out int n))
        continue;

    if (n == 0)
        break;

    sum += n;
}

Console.WriteLine($"Sum = {sum}");
```

`while (true)` + **`break`** при zero — visible exit. Named condition, если проще: `while (!done)`.

### Пример 2: таблица умножения

```csharp
int size = 5;

for (int i = 1; i <= size; i++)
{
    for (int j = 1; j <= size; j++)
    {
        Console.Write($"{i * j,4}");
    }
    Console.WriteLine();
}
```

### Пример 3: foreach по string

```csharp
string word = "C#";

foreach (char c in word)
{
    Console.WriteLine(c);
}
```

`string` — `IEnumerable<char>`; UTF-16 **char** units, не grapheme clusters — важно для i18n UI позже.

## Common Mistakes & Anti-patterns

**Off-by-one** — `<= Length` при `array[i]`. **`< Length`**.

**Infinite `while`** — забыли decrement/advance.

**Modify list in `foreach`** — throws.

**`for` everywhere** — без index яснее `foreach`.

**Expensive condition** в `while` каждый раз — cache в variable.

---

## Production & Real-World Notes

В ASP.NET избегай long **blocking loops** на request thread — async later. Console/batch — loops норм.

**`foreach` + LINQ** для business collections без index. **`for`** — parallel arrays, buffers.

Logging каждой iteration в production floods observability — log aggregates.

---

## Comparison / Trade-offs

| Loop | Pros | Cons |
|------|------|------|
| `for` | Explicit index | Verbose |
| `while` | Flexible | Easy infinite |
| `do-while` | Min one run | Лишний run если не нужен |
| `foreach` | Clean | No index; no modify |

```
for (int i = 0; i < n; i++) { }
while (cond) { }

foreach (var x in xs) { }
Не modify collection в foreach
Off-by-one: i < Length
```

## Key Takeaways

- Сначала проектируй termination condition.
- `for` = counted/index; `while` = precondition; `do-while` = ≥1 раз; `foreach` = sequence.
- Off-by-one — главный bug; default `< Length`.
- `foreach` + enumerator; no structural modify mid-flight.
- Nested loops умножают iterations.
- `break`/`continue` — в `1.1.11`.
