# Answers: LINQ Aggregate, Scan

### 1. **`InvalidOperationException`** (no elements).

### 2. Returns **0** (seed, func not called).

### 3. **Нет** — return sole element.

### 4. **Aggregate** → one value; **Scan** → **`IEnumerable`** of intermediates.

### 5. **`Aggregate(1, (a,n) => a * n)`**.

### 6. **`[10, 7, 12]`** if Scan yields after each combine (no initial 0 emit).

### 7. **O(n²)** string allocations; use **StringBuilder** or **Join**.

### 8. **Нет** — extension/custom/MoreLINQ.

### 9. Usually **client eval** or not translatable — not generic fold SQL.

### 10. **`MaxBy(n => n.Length)`** clearer ( .NET 6+).

### 11. **`Aggregate(new StringBuilder(), (sb,s) => sb.Append(s))`**.

### 12. **Unsafe** — order/partition breaks non-associative folds.
