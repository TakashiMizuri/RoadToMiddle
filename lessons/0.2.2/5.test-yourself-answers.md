# Answers: Merge

## Scoring
- **90–100%:** отлично → `0.2.3`
- **80–89%:** pass — повтори ошибки
- **<80%:** lection Core + Under the Hood

## Answers

### 1. [Recall]
**Answer:** **Merge base** — nearest **common ancestor** commit, достижимый от обоих tips merging branches.

**Why:** Anchor для three-way: «что изменилось с момента расхождения».

### 2. [Recall]
**Answer:** **Два** parent — tip current branch (куда merge) и tip merged-in branch.

### 3. [Understanding]
**Answer:** **Fast-forward.** `main` ref двигается на C5. **Новый commit object не создаётся.**

**Why:** C2 (main tip) — ancestor C5.

### 4. [Understanding]
**Answer:** **`feature` ref не меняется** автоматически merge в main (если merge делали на main). Оба refs могут указывать на один commit после FF; после merge commit main на X, feature всё ещё на F.

### 5. [Understanding]
**Answer:** **Merge commit:** commit'ы feature — **ancestors** main (через second parent chain). **Squash:** на main **один** commit; intermediate feature commit'ы **не** в ancestry main — только их combined diff.

### 6. [Application]
**Answer:** `git merge --no-ff feature`

### 7. [Application]
**Answer:**
```bash
git switch main
git merge --squash feature
git commit -m "..."
```

### 8. [Analysis]
**Answer:** **Неверно.** Squash → **один parent** (previous main tip).

### 9. [Analysis]
**Answer:** Явная **граница PR** в graph; revert merge commit одной операцией; видно **когда** feature integrated, даже если FF возможен.

### 10. [Application]
**Answer:** `git merge --abort`

### 11. [Recall]
**Answer:** `git merge-base main feature`

### 12. [Understanding]
**Answer:** Ветка, **на которой выполняли merge** (target, обычно `main`) — **first parent**.

### 13. [Application]
**Answer:** Changes **staged** для commit; merge не завершён до `git commit`.

### 14. [Analysis]
**Answer:** FF **не удаляет** commit'ы — они становятся частью **first-parent history** main. Ref main **переместился** на tip feature; все commit'ы feature достижимы с main.

### 15. [Understanding]
**Answer:** После **squash** bisect на main **переключает всю feature одним шагом** (один commit). После **FF** bisect может остановиться на **каждом** intermediate commit feature.
