# Exercises: Keep-Alive and pooling

> Conceptual + optional .NET observation.

## Easy

### E1. curl connection reuse
**Goal:** See persistent connection in verbose output.

**Steps:**
1. `curl.exe -v https://httpbin.org/get https://httpbin.org/get 2>&1` (two URLs same host in one curl).
2. Note if second request reuses connection (look for "Re-using existing connection" or single TCP).

**Success criteria:** Explain whether reuse happened.

---

## Medium

### M1. HttpClientFactory sketch
**Goal:** Map DI pattern without full app.

**Tasks:**
1. Write pseudo `Program.cs` snippet: `AddHttpClient("api")`, inject `IHttpClientFactory`, `CreateClient("api")`.
2. List two settings you'd tune for high-traffic outbound calls.

**Success criteria:** No `new HttpClient()` in loop; names `PooledConnectionLifetime` or `MaxConnectionsPerServer`.

---

### M2. Chain diagram
**Goal:** Full stack pooling.

**Tasks:** ASCII: Browser → CDN → nginx → Kestrel → HttpClient → external API. Mark where keep-alive/pool applies on each hop.

**Success criteria:** At least four hops labeled.
