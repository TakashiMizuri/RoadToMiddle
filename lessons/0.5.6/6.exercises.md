# Exercises: Images — `alt`, `figure`, `picture`

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Alt decision tree
**Goal:** Classify alt strategy.

For each scenario write `alt` value or rule (`descriptive text` / `alt=""`):

1. Stock photo purely ornamental in hero background slot (content in `h1` already).
2. Graph showing server uptime 99.2% last month.
3. PDF download icon next to link text «Download invoice».
4. Product photo on product detail page, no adjacent description of color.

**Success criteria:** Four justified choices referencing informative vs decorative rules.

---

### E2. Dimensions and lazy
**Goal:** CLS-aware markup.

Add `width`, `height`, and appropriate `loading` to:

```html
<img src="/blog/inline-diagram.png" alt="Sequence: client, API, database for login request" />
```

Assume: image appears mid-article below three paragraphs; intrinsic size 600×400.

**Success criteria:** 600×400 attributes; `loading="lazy"`; brief note why not eager.

---

## Medium

### M1. Figure tutorial block
**Goal:** Semantic caption pattern.

Build HTML for a docs page section «Installing Node.js» with:

- `h2` heading
- Short paragraph
- `figure` containing screenshot (`alt` describes terminal output), `figcaption` «Node version 22.x confirmed»
- Optional `loading="lazy"`

**Success criteria:** Valid nesting; informative alt; figcaption present.

---

### M2. Responsive product image
**Goal:** `picture` with format fallback.

Create markup for product image:

- WebP source `/products/item.webp`
- JPEG fallback `/products/item.jpg`
- `alt` with product name and color
- `width="400" height="400"`

Add second `source` with `media="(max-width: 480px)"` pointing to `/products/item-square.jpg` for art direction (optional stretch: explain when browser picks it).

**Success criteria:** Valid `picture`; `alt` on `img`; at least one `source`.

---

## Hard (stretch)

### H1. Audit CMS output
**Goal:** Fix real-world anti-patterns.

Fix this fragment:

```html
<div style="background-image:url(/promo.jpg)">
  <h2>Summer sale 30% off</h2>
</div>
<img src="/promo.jpg">
<a href="/sale"><img src="/thumb.jpg"></a>
```

Provide improved HTML: semantic promo in `<main>`, proper `alt`, no duplicate inaccessible content, linked thumbnail with alt, consider `figure` if caption «Limited time».

**Success criteria:** Sale info available without seeing background image; all imgs have correct alt; link+img pattern from `0.5.5`.

---

## Submission checklist

- [ ] Every informative image has descriptive `alt`
- [ ] Decorative images use `alt=""`
- [ ] `picture` ends with fallback `img`
- [ ] width/height on content images where dimensions known
