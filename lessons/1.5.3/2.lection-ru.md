# Assembly: metadata, manifest и resources

> Roadmap: `1.5.3` · Node: `1.5` · Depth: понимание

## Learning Objectives

После урока ты сможешь:

- Определить **.NET assembly** как unit deployment с IL, metadata и resources.
- Объяснить **manifest**: name, version, culture, references.
- Описать **metadata** как embedded type system CLR.
- Различить **assembly**, **namespace**, **NuGet package**.
- Использовать базовый **reflection** для inspect metadata.
- Объяснить, зачем metadata нужна **JIT**, **GC layout**, **serialization**.

---

## Why This Matters

В `1.5.1` CLR загружает **assemblies** с IL и metadata. В `1.5.2` JIT читает method bodies и tokens. Без понимания `.dll` inside versioning conflicts и reflection failures кажутся случайными.

Каждый `dotnet publish`, NuGet graph и `[Authorize]` resolved at runtime — через **metadata**. Middle-разработчик читает assembly names в logs, чинит load errors, знает почему **InternalsVisibleTo** — assembly-level.

---

## Core Concepts

### Что такое Assembly?

**Assembly** — smallest unit, которую CLR version-resolve и load. Обычно один `.dll`/`.exe` per project output. Содержит PE wrapper, CLR header, **IL streams**, **metadata tables**, **manifest**, optional **resources**.

**Namespace** организует source; **assembly** — deployment и loading. Один namespace может span несколько assemblies.

### Manifest

Отвечает: **Who am I?** (name, version, culture, public key token). **What do I depend on?** (AssemblyRef rows). **Entry point** для exe.

Version mismatch → `FileLoadException`, `MissingMethodException`.

### Metadata Tables

Normalized tables ECMA-335: **TypeDef**, **MethodDef**, **FieldDef**, **MemberRef**, **CustomAttribute**, etc.

JIT: MethodDef → IL RVA. GC: layout + GC maps. Serializers: property names unless overridden.

**Tokens** в IL (`0x06000001`) index tables — `call` references methods.

### Resources

Embedded `.resx` → `.resources`. **Satellite assemblies** для localization (`fr/MyApp.resources.dll`).

### Reflection

```csharp
var asm = typeof(Program).Assembly;
foreach (var type in asm.GetExportedTypes())
    Console.WriteLine(type.FullName);
```

Same tables as JIT — для discovery, не hot paths.

---

## Under the Hood

Managed `.dll` = **PE** + **CLI metadata root**. Loader validates PE → CLR parses `#~` heap → runtime structures (`MethodTable`).

**Strong naming** — identity для binding. Modern apps: NuGet version + **AssemblyLoadContext**.

**Single-file publish** — logical model unchanged; physical bundling differs.

---

## Common Mistakes

Namespace ≠ assembly. Ignoring version in binding. Reflection в hot loops без cache.

---

## Production Notes

**Central Package Management** — aligned versions. **Trimming/AOT** — metadata pruned; reflection breaks без annotations. **Collectible ALC** — plugin unload.

---

## Key Takeaways

- Assembly = IL + metadata + manifest (+ resources).
- Manifest — identity + dependencies.
- Metadata — JIT, GC, attributes, reflection.
- Namespace ≠ assembly.
