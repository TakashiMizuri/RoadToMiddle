# Exercises: Boxing and Unboxing

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Box detector
**Goal:** Spot boxing sites.

Mark **BOX** or **NO** for each line:

```csharp
object a = 3.14m;
List<double> list = new();
list.Add(3.14);
ArrayList legacy = new();
legacy.Add(3.14);
decimal d = 1.0m;
decimal e = d;
IFormattable f = d;
```

**Success criteria:** 6/6 with brief reason.

---

### E2. Fix the unbox
**Goal:** Safe extraction.

```csharp
object value = GetValue(); // returns boxed int 42
// Fix so it never throws InvalidCastException for wrong type
```

Write defensive code using pattern matching.

**Success criteria:** Handles non-int `object` without exception.

---

## Medium

### M1. Allocation comparison sketch
**Goal:** Quantitative intuition.

Pseudocode loop: **`ArrayList`** vs **`List<int>`**, 1_000_000 `Add(1)`. Describe expected allocation difference in 3–5 sentences (no benchmark required, but optional).

**Success criteria:** Mentions ~1M boxes vs array growth without per-element box.

---

### M2. API redesign
**Goal:** Remove object overload.

Given:

```csharp
public class Metrics
{
    public void Record(string name, object value) { /* stores */ }
}
```

Design typed overloads for `int`, `double`, `bool`, and a generic `Record<T>` where appropriate. Explain which calls still box.

**Success criteria:** `Record("hits", 1)` uses non-boxing overload.

---

## Hard (stretch)

### H1. IL hunt
**Goal:** Connect source to `box`.

Build a console app with three boxing examples (object assign, interface assign, ArrayList.Add). Paste IL snippets for each **`box`** instruction.

**Success criteria:** Three distinct `box` tokens identified.
