# Exercises: Pattern Matching

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Grade from score
**Goal:** Map integer score to letter grade using a switch expression.
**Steps:** Create `string Grade(int score)` with arms `>= 90` … `_`. Test with 95, 72, 49.
**Success criteria:** Correct letter; most specific thresholds first; no `if` chain.

### E2. Type describer
**Goal:** `string Describe(object? value)` using type patterns.
**Steps:** Handle `null`, `string`, `int`, `bool`, discard for others.
**Success criteria:** Single switch expression; no `as` casts.

### E3. Empty / single list
**Goal:** Use list patterns on `int[]`.
**Steps:** Return `"empty"`, `"one:{n}"`, or `"many:{count}"` for 0, 1, 2+ elements.
**Success criteria:** Uses `[]`, `[var x]`, and `[..]` or equivalent.

## Medium

### M1. Order discount
**Goal:** Property pattern on a record `Order(decimal Total, Customer Customer)` where `Customer` has `bool IsVip`.
**Steps:** Return discount `0.15m`, `0.10m`, `0.05m`, or `0m` per lecture example.
**Success criteria:** One switch expression; relational + nested property patterns.

### M2. CLI router
**Goal:** Parse `string[] args` into commands: `greet {name}`, `sum {a} {b}` (ints), `help`, unknown.
**Steps:** List patterns + `when` with `int.TryParse` for sum.
**Success criteria:** At least 4 arms; invalid sum args fall through to unknown.

### M3. Refactor if-chain
**Goal:** Given a 5-branch `if/else` on `Shape` hierarchy (or discriminated record union), rewrite to switch expression returning area label.
**Success criteria:** No duplicated casts; compiler happy with exhaustiveness if using closed set.

## Hard (stretch)

### H1. Nested JSON-like nodes
**Goal:** `string Format(object? node)` for tree: `null`, scalars, `object[]` with list patterns up to depth 2 (e.g. `[ "a", [ "b", "c" ] ]` as nested arrays — simplify as `object?[]` if needed).
**Success criteria:** Combines type, list, and recursive call or nested switch.

### H2. Result mapper
**Goal:** Map `Result<T,E>`-style type (success/error) to HTTP status int via switch on shape `{ IsSuccess: true, Value: var v }` vs error.
**Success criteria:** Property patterns; no `.IsSuccess` if outside switch.

## Submission checklist

- [ ] All switch expressions compile in a console project (`net8.0`)
- [ ] At least one exercise uses list patterns
- [ ] At least one uses property + relational patterns
- [ ] You can explain arm order for relational cases orally
