# Exercises: Forms ŌĆö `form`, `label`, inputs

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Label wiring
**Goal:** Fix unlabeled fields.

Add proper labels and ids:

```html
<form action="/register" method="post">
  <input type="text" name="fullName" placeholder="Full name">
  <input type="email" name="email" placeholder="Email">
  <button>Register</button>
</form>
```

**Success criteria:** Explicit `label for` each field; submit button `type="submit"`.

---

### E2. Choose the type
**Goal:** Match input type to scenario.

For each scenario write the best `input type`:

1. Age in years (integer UI)
2. Meeting date picker
3. Accept terms toggle (single yes)
4. Pick plan: Free / Pro / Team (one choice)

**Success criteria:** Four types with one-line justification each.

---

## Medium

### M1. Contact form in semantic page
**Goal:** Full form in document context.

Build HTML page fragment:

- `<main>` with `h1` ┬½Contact us┬╗
- `form` `method="post"` `action="/contact"`
- Fields: name (text), email, phone (tel), topic (`select` with 3 options), message (`textarea`)
- Submit button

Use `fieldset` only if you add radio group for ┬½Preferred contact method┬╗.

**Success criteria:** All fields labeled; sensible `name` attributes; valid nesting in `main`.

---

### M2. File upload profile
**Goal:** multipart form.

Avatar upload form: single file, accept images, POST to `/profile/avatar`. Include hidden `name="userId" value="42"` (note tamper risk in prose).

**Success criteria:** `enctype="multipart/form-data"`; file input labeled; submit typed.

## Hard (stretch)

### H1. Model binding alignment
**Goal:** Bridge HTML to ASP.NET Core.

Given C# record:

```csharp
public record CheckoutForm(string Email, string Shipping, bool GiftWrap);
```

Write HTML form fragment with field names that bind correctly to this record on POST. Shipping as radio `standard`/`express`. GiftWrap as checkbox.

Explain in 2 sentences how unchecked checkbox affects `GiftWrap` binding.

**Success criteria:** Names match binding conventions; radio group correct; checkbox behavior explained.
