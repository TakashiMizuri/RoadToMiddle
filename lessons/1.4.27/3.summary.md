# IHttpClientFactory — краткая выжимка

> `1.4.27` · ~10 мин на повторение

## Главная идея

**IHttpClientFactory** управляет **HttpMessageHandler** lifetime (rotation ~2 min), connection pool и DNS — ты регистрируешь **named/typed clients** с pipeline **DelegatingHandler**, а не static/singleton HttpClient вручную.

## Ключевые пункты

- `AddHttpClient("name", configure)` + `CreateClient("name")`.
- Typed: `AddHttpClient<TInterface, TImpl>` — inject interface.
- **DelegatingHandler** — correlation ID, auth, logging, Polly.
- Handler order: outer registered handler sees request first.
- **SetHandlerLifetime** / default 2 min — DNS refresh.
- **ConfigurePrimaryHttpMessageHandler** — SocketsHttpHandler tuning.
- **CreateClient()** — не anti-pattern; factory owns handler.
- HostedService: **IServiceScopeFactory** для scoped dependencies.

## Самые важные примеры

```csharp
builder.Services.AddHttpClient<ICatalogApi, CatalogApi>(c =>
{
    c.BaseAddress = new Uri(baseUrl);
})
.AddHttpMessageHandler<CorrelationIdHandler>();
```

## «Под капотом» в одном абзаце

Factory хранит ActiveHandlerEntry per name; по истечении lifetime handler dispose, новый pool + DNS. HttpClient object лёгкий; pooling на уровне handler. DI validation ловит singleton handler со scoped DbContext.

## Частые ошибки

- Static HttpClient + factory mixed.
- Retry handler before auth token handler.
- Singleton BackgroundService inject scoped typed client directly.

## Мнемоника

**Factory = outbound DI: name, chain, lifetime.**
