# Answers: Assembly — Metadata and Manifest

## Answers

### 1. [Recall]
**Answer:** IL bytecode, metadata tables, manifest, optional embedded resources в PE/CLI file.
**Why:** Smallest CLR load/version unit.

### 2. [Recall]
**Answer:** Assembly identity (name, version, culture, public key) и список referenced assemblies.
**Why:** Loader resolves dependency graph.

### 3. [Understanding]
**Answer:** Namespace — logical grouping в source; assembly — physical deploy/load unit. Один namespace может быть в нескольких assemblies.
**Why:** Common confusion in load errors.

### 4. [Understanding]
**Answer:** Resolve method bodies (RVA), types for virtual dispatch, field layout for GC, attributes, exception handlers.
**Why:** IL tokens index metadata.

### 5. [Application]
**Answer:** Version mismatch referenced assembly; missing dependency on disk; wrong culture/satellite; bitness/runtime incompatibility.
**Why:** Manifest AssemblyRef vs deployed files.

### 6. [Application]
**Answer:** `Version` struct assembly из manifest (major.minor.build.revision).
**Why:** Reflection over same metadata loader uses.

### 7. [Analysis]
**Answer:** **D** — NuGet package может содержать несколько assemblies.
**Why:** Package ≠ single DLL.

### 8. [Understanding]
**Answer:** Attribute stored in metadata table, attached to property; serializer reads at runtime/generator compile time.
**Why:** CustomAttribute table pattern.

### 9. [Application]
**Answer:** Friend access — CLR enforcement at **assembly** boundary; namespace не участвует в access control.
**Why:** Internals visible to whole friend assembly.

### 10. [Understanding]
**Answer:** Loader reads manifest → loads AssemblyRefs → builds runtime types from metadata → JIT uses MethodDef/IL RVA (`1.5.1`).
**Why:** Metadata connects compile output to runtime execution.
