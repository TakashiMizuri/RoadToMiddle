# ref, out, in, params — краткая выжимка

> `1.1.13` · ~8 мин на повторение

## Главная идея

C# по умолчанию передаёт аргументы **by value** (копия). Модификаторы **`ref`**, **`out`**, **`in`** передают **alias** (managed pointer / byref) с разными контрактами на чтение и запись; **`params`** собирает variadic-аргументы в массив.

## Ключевые пункты

- **`ref`** — читать и писать в переменную вызывающего; аргумент **должен быть инициализирован**.
- **`out`** — только вывод; вызывающий **не обязан** инициализировать; callee **обязан** присвоить до `return` на всех путях.
- **`in`** — read-only alias; экономит копирование **больших readonly struct**; мутация запрещена.
- **`params`** — последний параметр; компилятор упаковывает `M(1,2,3)` в `new int[] {1,2,3}` → часто **allocation**.
- В IL **`ref`** и **`out`** — оба byref; различие — **definite assignment analysis** компилятора.
- **`TryParse` / `Try*`** — каноничный паттерн для **`out`**.

## Самые важные примеры

```csharp
Swap(ref a, ref b);                    // ref — swap in place
int.TryParse(s, out int n);            // out — Try pattern
double Dot(in Vector3 a, in Vector3 b); // in — read-only, no copy
Log("a", "b");                         // params → string[]
```

## «Под капотом» в одном абзаце

Caller для `ref`/`out`/`in` генерирует **`ldloca`** (адрес локали). Callee работает через **`ldind`/`stind`**. Это **managed pointer** — GC знает о нём; не C++ pointer arithmetic. **`params`** — синтаксический sugar: variadic call → heap-массив на call site (если не передан готовый массив).

## Частые ошибки

- Читать **`out`** до присвоения в callee.
- **`ref` «для скорости»** везде — ухудшает API и не всегда быстрее return.
- Думать, что **`params`** бесплатен в hot loop.
- Путать **`in`** (readonly alias параметра) с deep immutability объекта.

## Мнемоника / правило большого пальца

**out** = **Output**, **in** = **Input read-only**, **ref** = **Read + write Reference**, **params** = **Pack into array**.
