# Answers: Exceptions

### 1. **throw;** preserves original stack; **throw ex;** resets throw site to catch line.
### 2. Preserves **cause chain** when wrapping.
### 3. Rethrow **later/other thread** with original stack.
### 4. No semantic value; use **concrete** types.
### 5. **ArgumentNull** — null param; **InvalidOperation** — wrong object state.
### 6. `catch (SqlException ex) { Log...; throw; }`
### 7. `throw new DomainException("...", ex);`
### 8. **Swallows** all errors — hides bugs; bad unless documented rare case.
### 9. Often **499/cancel** or rethrow — not generic 500 error.
### 10. **Filter evaluated** before committing to catch; false → not caught here.
### 11. **Middleware** once; repository **specific** or propagate.
### 12. **NotImplemented** — stub; **NotSupported** — valid API but operation unsupported.
