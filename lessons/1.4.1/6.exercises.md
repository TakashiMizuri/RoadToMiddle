# Exercises: Threads 101

> Complete after reading lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Observe thread pool reuse
**Goal:** See that multiple `Task.Run` calls reuse thread IDs.
**Steps:** Console app; loop 30× `Task.Run` printing `ManagedThreadId`; `await Task.WhenAll`.
**Success criteria:** Explain why unique ID count < 30.

### E2. Main vs pool thread
**Goal:** Distinguish `IsThreadPoolThread` on main and in `Task.Run`.
**Steps:** Print flags before and inside `Task.Run`.
**Success criteria:** Main = false; inside Run = true.

## Medium

### M1. Queue depth under load
**Goal:** Relate burst work to delayed completion.
**Steps:** Queue 500 `Task.Run` with `Thread.Sleep(100)`; measure total wall time vs 50 tasks.
**Success criteria:** Describe non-linear slowdown; mention injection throttling.

### M2. dotnet-counters
**Goal:** Monitor pool metrics live.
**Steps:** Run sample under `dotnet-counters monitor --counters System.Runtime` focusing on threadpool counters during M1.
**Success criteria:** Name at least two counters and what increased.

## Hard (stretch)

### H1. Blocking vs async preview
**Goal:** Feel starvation precursor (sync blocking).
**Steps:** Minimal ASP.NET app: one endpoint `Thread.Sleep(2000)`, another fast; load test 50 concurrent with `wrk` or `ab`.
**Success criteria:** Fast endpoint slows when slow endpoint loaded; explain pool contention.
