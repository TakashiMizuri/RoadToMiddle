# Exercises: Sync-over-async fixes

## Easy

### E1. Classify snippets
**Goal:** Label sync-over-async vs async-over-sync vs correct async.
**Steps:** 5 code snippets from lection; write classification + one-line fix.
**Success criteria:** 5/5 correct.

### E2. Exception compare
**Goal:** Observe `.Result` vs `GetAwaiter().GetResult()` exception types.
**Steps:** Throwing async method; catch both ways.
**Success criteria:** Note AggregateException difference.

## Medium

### M1. Dual API migration
**Goal:** Add `GetUserAsync`, obsolete sync `GetUser`.
**Steps:** Small repo interface + controller; fix analyzer warnings in new code only.
**Success criteria:** New endpoint uses async only.

### M2. Task.Run vs refactor benchmark
**Goal:** Load test 50 concurrent (local) sync SDK shim: Task.Run vs true async HTTP.
**Steps:** Simple `HttpClient` vs `Task.Run(WebClient)` mock delay.
**Success criteria:** Document thread pool/active threads difference.

## Hard (stretch)

### H1. Architecture decision record
**Goal:** ADR for legacy billing SDK in ASP.NET Core API (500 RPS).
**Steps:** Compare microservice isolation vs Task.Run vs rewrite; pick with trade-offs.
**Success criteria:** 1-page ADR with rejected options.
