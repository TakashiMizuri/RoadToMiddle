# CancellationToken — Register, linked tokens, CancelAfter

> Roadmap: `1.4.11` · Node: `1.4` — Async/await · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Объяснить **cooperative cancellation** — нет preemptive kill thread в .NET async.
- Использовать **`CancellationToken`**, **`CancellationTokenSource`**, **`Register`**, **`CreateLinkedTokenSource`**, **`CancelAfter`**.
- Прокидывать token через async chains; выходить из loops до лишних **`await`** (state machine early exit — `1.4.7`).
- Подключать **`HttpContext.RequestAborted`** в ASP.NET Core.
- Избегать **Dispose races** и **leaked CTS**.

---

## Why This Matters

User закрывает tab; LB timeout; deploy drain. Work должен **останавливаться cooperatively**, освобождая DB, HTTP, CPU. **`CancellationToken`** — standard signal; игнор → orphan tasks и лишние **`MoveNext`**.

Middle-разработчик: **`Register`** для unmanaged cleanup, link **user + timeout + abort**, **`ThrowIfCancellationRequested`** vs graceful return.

## Core Concepts

### Cooperative model

Cancellation **cooperative**: **`IsCancellationRequested`**, **`ThrowIfCancellationRequested()`**. Code и framework **poll/register**. Runtime не abort'ит threads.

При cancel — **`OperationCanceledException`**. State machine propagates to caller (`1.4.19`).

### CancellationTokenSource

**`CTS`** owns state. **`cts.Token`** — read-only downstream. **`Cancel()`** — flag + callbacks. **`CancelAfter(ms)`** — timer на pool.

**`using`** / dispose CTS — release timer.

### Register

**`token.Register(callback)`** при cancel — close socket, release semaphore. Callbacks **sync during Cancel()** — короткие; deadlock если block.

### Linked tokens

**`CreateLinkedTokenSource(t1, t2)`** — cancel когда **любой** cancel:

```csharp
using var linked = CancellationTokenSource.CreateLinkedTokenSource(
    userCts.Token, HttpContext.RequestAborted);
await _service.ProcessAsync(linked.Token);
```

### Propagation

Token последним параметром. Cancellable overloads на каждом **`await`**. В loops — **`ThrowIfCancellationRequested()`** между iterations.

## Under the Hood

**`CancellationToken`** struct → internal CTS state или **`None`**.

**`Cancel()`** — volatile flag + callbacks under lock.

**`Task.Delay(..., token)`** — cancel → task Canceled → await throws.

State machine: throw before await — sync cancel path; during await — canceled task → exception path **`MoveNext`**.

**`RequestAborted`** — client disconnect в Kestrel.

---

## Examples

```csharp
using var timeoutCts = new CancellationTokenSource(TimeSpan.FromSeconds(10));
using var linked = CancellationTokenSource.CreateLinkedTokenSource(userToken, timeoutCts.Token);
await _engine.SearchAsync(query, linked.Token);
```

## Common Mistakes

- CTS не dispose с **`CancelAfter`** — timer leak.
- Swallow **`OperationCanceledException`** без логики.
- Token не в children **`Task.WhenAll`** (`1.4.12`).
- Long sync без check token.

---

## Key Takeaways

- Cooperative — pass and check token.
- **Register** — fast cleanup.
- **Linked** — any condition cancels.
- **CancelAfter** — timeout.
- **RequestAborted** — disconnect.
