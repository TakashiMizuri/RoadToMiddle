## Why This Matters

Классический **`IEnumerable<T>`** с **`yield return`** даёт **ленивый pull**, но **`MoveNext()`** синхронный. Если следующий элемент требует **страницы БД**, **HTTP chunk** или **чтения файла**, ты блокируешь thread или городишь костыли.

**C# 8** ввёл **`IAsyncEnumerable<T>`** и **`await foreach`**: consumer запрашивает следующий элемент **асинхронно**. Producer может **`await`** I/O между yields. Это естественная модель **streaming** — экспорт миллиона строк без `List<T>`, NDJSON из API, пагинация EF через **`AsAsyncEnumerable()`**.

Без этой модели команды буферизуют entire result set (memory spikes) или делают **`Task.Run`** на страницу (растрата threads). Middle fullstack использует async streams на границе **service → controller** и с **System.Text.Json** async serialization.

## Core Concepts

### Sync vs async iteration

**`IEnumerable<T>`** — iterator pattern: **`GetEnumerator()`**, **`MoveNext()`**, **`Current`**. Caller задаёт темп; работа между элементами **синхронна** на thread caller'а.

**`IAsyncEnumerable<T>`** — **`GetAsyncEnumerator()`**, **`MoveNextAsync()`** → **`ValueTask<bool>`**. Consumer:

```csharp
await foreach (var item in GetItemsAsync())
{
    Process(item);
}
```

Каждая итерация может **suspend** без блокировки thread pool во время I/O.

### Авторство async iterator

```csharp
async IAsyncEnumerable<OrderDto> StreamOrdersAsync(
    [EnumeratorCancellation] CancellationToken ct = default)
{
    int page = 0;
    while (true)
    {
        var batch = await _repo.GetPageAsync(page++, ct);
        if (batch.Count == 0) yield break;
        foreach (var o in batch)
            yield return Map(o);
    }
}
```

Компилятор генерирует state machine, похожую на **`async Task`**, но для iteration. **`[EnumeratorCancellation]`** пробрасывает token из **`await foreach`** в iterator — pending **`await`** producer'а отменяется при disconnect consumer'а.

### Потребление

```csharp
await foreach (var line in ReadLinesAsync(path))
    Console.WriteLine(line);

await foreach (var item in StreamAsync(ct).WithCancellation(ct))
{ }
```

**System.Linq.Async** — LINQ для async streams (deferred). EF Core: **`AsAsyncEnumerable()`** — streaming query; плохой SQL streaming не исправит.

### Память и backpressure

Async streams **pull** по одному элементу (или batch) — скорость consumer'а ограничивает producer, **если** iterator **await'ит** перед следующим yield. Это **implicit backpressure**.

Для push/multi-producer с буфером — **Channels** (**`1.4.17`**).

## Under the Hood

Async iterator state machine реализует **`IAsyncEnumerator<T>`**. **`MoveNextAsync()`** возвращает **`ValueTask<bool>`** — часто sync-complete для in-memory yield.

**`await foreach`** генерирует **`try/finally`** с **`DisposeAsync()`** — освобождай DB connections и HTTP streams.

## Examples

### Paginated HTTP client

```csharp
async IAsyncEnumerable<GitHubIssue> GetIssuesAsync(
    string repo,
    [EnumeratorCancellation] CancellationToken ct = default)
{
    int page = 1;
    while (true)
    {
        var issues = await FetchPageAsync(repo, page++, ct);
        if (issues.Count == 0) yield break;
        foreach (var issue in issues) yield return issue;
    }
}
```

Peak memory ≈ одна страница.

## Common Mistakes & Anti-patterns

**`IEnumerable` + `.Result` в iterator** — blocks; используй **`IAsyncEnumerable`**.

**Без cancellation** — stream продолжается после disconnect клиента.

**Multiple enumeration** — каждый проход может re-query БД.

**Ранний `ToList()`** — убивает streaming.

**`foreach` вместо `await foreach`** — не компилируется.

---

## Production & Real-World Notes

ASP.NET Core return **`IAsyncEnumerable<T>`** — понимай поведение serializer. HTTP streaming — headers и middleware без full-body buffer. Тестируй с **slow consumer**.

## Comparison / Trade-offs

| | `IEnumerable` | `IAsyncEnumerable` |
|---|---------------|---------------------|
| I/O между items | Block | `await` |
| Consumer | `foreach` | `await foreach` |
| Backpressure | Pull sync | Pull async |

## Key Takeaways

- **`IAsyncEnumerable<T>`** = lazy async pull.
- **`await foreach`** + **`MoveNextAsync()`** без block на I/O.
- Cancellation через **`[EnumeratorCancellation]`**.
- Не грузи huge lists — stream.
- Push/multi-producer → **Channels**.
