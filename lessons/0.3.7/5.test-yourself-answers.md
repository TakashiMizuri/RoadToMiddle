## Answers

### 1. [Recall]
**Answer:** **Content-Type**.

**Why:** Declares representation format of the message body.

---

### 2. [Recall]
**Answer:** **Accept**.

**Why:** Lists preferred response media types with optional q-values.

---

### 3. [Recall]
**Answer:** **If-None-Match**.

**Why:** Conditional GET validator paired with ETag.

---

### 4. [Understanding]
**Answer:** **Authorization** — developer **explicitly** sets token each request (often from memory/storage); **Cookie** — browser **automatically** attaches stored cookies per domain/path after Set-Cookie.

**Why:** Different auth transport models; cookies need CSRF/SameSite care (`0.3.9`).

---

### 5. [Understanding]
**Answer:** **Shared caches (CDN)** may serve **one user's `/api/me` JSON to another user** — privacy breach. `public` allows shared cache storage.

**Why:** Personalized responses need `private` or `no-store`, not `public`.

---

### 6. [Understanding]
**Answer:** Model binding may **fail** or use wrong formatter → **400/415**; silent mis-parse in edge cases; `[Consumes]` rejection.

**Why:** Server chooses input parser from Content-Type.

---

### 7. [Understanding]
**Answer:** **304 Not Modified**, **no response body** (client uses cache).

**Why:** Successful conditional GET revalidation.

---

### 8. [Application]
**Answer:** **`Cache-Control: no-store`** (strongest: do not persist). Also consider `private` if only browser cache concern — but payment data → **no-store**.

**Why:** Prevents any cache layer retention.

---

### 9. [Application]
**Answer:** **`Authorization: Bearer <jwt>`**.

**Why:** Standard OAuth 2.0 bearer token pattern.

---

### 10. [Application]
**Answer:** e.g. **`Cache-Control: public, max-age=31536000, immutable`** and **ETag** (or rely on filename hash + long max-age).

**Why:** Immutable hashed assets safe for long CDN TTL.

---

### 11. [Analysis]
**Answer:** **B** неверно — Accept describes **response** preferences, not request body (that's Content-Type).

---

### 12. [Analysis]
**Answer:** CDN returns **cached JSON from user A to user B** — IDOR-like data leak through cache.

**Why:** Cache key must vary by auth or don't cache authenticated responses; need `private/no-store` or `Vary: Authorization`.

---

### 13. [Understanding]
**Answer:** Declares **character encoding** for text body so client decodes bytes correctly (avoid mojibake).

**Why:** UTF-8 not assumed for all legacy text paths.

---

### 14. [Analysis]
**Answer:** Headers contain **session tokens, JWTs, secrets** — logs become credential store; breach of logs = account takeover; compliance violation (PCI/GDPR).

**Why:** Never log secrets; redact in middleware.

---

## Model answers for practical

```bash
curl -H "Accept: application/json" ...
curl -H "Content-Type: application/json" -H "Authorization: Bearer TOKEN" ...
curl -H 'If-None-Match: "etag-value"' ...
```
