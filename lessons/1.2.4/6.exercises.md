# Exercises: Access Modifiers

> Complete after lectures and passing `4.test-yourself.md`.

## Easy

### E1. Modifier map
**Goal:** Label members.

Given class with mix of fields/methods — assign `public`/`private`/`protected`/`internal` for: public API method, storage field, subclass hook, test-only helper.

**Success criteria:** Each choice justified in one sentence.

---

### E2. Default quiz
**Goal:** Defaults.

What is visibility of `class Foo { int x; }` for `Foo` and `x`?

**Success criteria:** `internal` class, `private` x.

---

## Medium

### M1. Refactor visibility
**Goal:** Shrink API.

Refactor class with all-public members to minimal public surface + private storage.

**Success criteria:** External code uses methods/properties only.

---

### M2. internal repository
**Goal:** Assembly pattern.

Public `IUserRepository`, internal `EfUserRepository`. Explain who can `new EfUserRepository`.

**Success criteria:** Only same assembly (or DI registration inside app).

---

## Hard (stretch)

### H1. protected vs internal design
**Goal:** Choose modifier.

Base `Validator` with hook `ValidateCore` for subclasses vs shared `internal` helper used by sibling classes in assembly — which modifier when?

**Success criteria:** protected for inheritance extension; internal for non-polymorphic shared code.
