## Why This Matters

Массив — простейшая **collection** в C#: фиксированная длина, zero-based индексы. Под капотом **`List<T>`** хранит элементы в **`T[]`**. Ошибка «объявил `int[,]`, а индексирую как `[][]`» — классика на code review.

В fullstack массивы встречаются в DTO, byte buffers, десериализации JSON, interop. Middle должен выбирать форму осознанно: шахматная доска — **`Piece[,]`**; права пользователей разной длины — **`string[][]`**; stream buffer — **`byte[]`**.

Массивы — **reference types** (кроме `stackalloc` позже). Переменная хранит reference; элементы — в heap (contiguous для 1D и rectangular).

## Core Concepts

### Одномерные массивы

Упорядоченная последовательность элементов одного типа, индексы `0 .. Length - 1`.

```csharp
int[] numbers = new int[5];
string[] names = { "Ann", "Bob" };
numbers.Length; // 5
```

**Default values:** числа `0`, `bool` false, references `null`. **Длина неизменна** после создания — для роста нужен новый массив + copy (`List<T>` автоматизирует).

**Bounds check** на каждый доступ → `IndexOutOfRangeException` при выходе за границы.

### Многомерные (rectangular) массивы

`int[,]` — 2D **прямоугольник**: у каждой строки одинаковое число столбцов. Память — **один блок**, row-major порядок.

```csharp
int[,] grid = new int[3, 4];
grid[1, 2] = 7;
grid.GetLength(0); // rows = 3
grid.GetLength(1); // cols = 4
grid.Length;       // 12 — все ячейки
```

Синтаксис **`grid[row, col]`** — не `grid[row][col]`. Подходит для **фиксированных сеток**: карты, матрицы, пиксели.

### Jagged arrays (массив массивов)

`int[][]` — outer array ссылок на row arrays; **длина строк может отличаться**.

```csharp
int[][] jagged = new int[3][];
jagged[0] = new int[] { 1, 2 };
jagged[1] = new int[] { 3, 4, 5 };
int x = jagged[1][2]; // 5
```

Layout: outer + **разрозненные** row objects — хуже locality при некоторых паттернах обхода. Подходит для **строк переменной длины**: sparse data, adjacency lists, ragged JSON.

### Сравнение форм

Rectangular — один object, все ячейки в одном flat buffer. Jagged — outer length + отдельные `int[]` на каждую строку. **Fixed rectangle → `T[,]`**. **Variable rows → `T[][]`**. **One sequence → `T[]`**.

### Covariance (только reference types)

```csharp
string[] a = { "x" };
object[] b = a; // OK
// b[0] = 123;  // InvalidCastException at runtime
```

Value type arrays не covariant. Публичные API предпочитают **`IReadOnlyList<T>`**.

## Under the Hood

Массивы наследуют **`System.Array`**. Runtime хранит length + data pointer. **`int[,]`** — один object с метаданными rank; индекс `[i,j]` → формула на flat buffer. **`int[][]`** — array references на отдельные row arrays.

JIT может **elide bounds checks** в provably safe loops. **`Array.Empty<T>()`** — singleton пустого массива без repeated allocations.

---

## Syntax / Commands / API

| Тип | Индекс | Длина |
|-----|--------|-------|
| `T[]` | `a[i]` | `a.Length` |
| `T[,]` | `a[i,j]` | `GetLength(dim)` |
| `T[][]` | `a[i][j]` | `a.Length`, `a[i].Length` |

**Static helpers:** `Array.Sort`, `Copy`, `Clear`, `IndexOf`.

## Examples

### Обход rectangular

```csharp
for (int r = 0; r < grid.GetLength(0); r++)
    for (int c = 0; c < grid.GetLength(1); c++)
        grid[r, c] = 0;
```

### Jagged с nested foreach

```csharp
foreach (string[] row in rolesByUser)
    foreach (string role in row)
        Console.WriteLine(role);
```

---

## Common Mistakes & Anti-patterns

`grid[i][j]` на `int[,]`. **`Length` как row count** на 2D. **Null jagged rows**. Covariance traps. Array вместо List при частых middle insertions.

---

## Production & Real-World Notes

Public API: **`IReadOnlyList<T>`**, **`ReadOnlySpan<T>`**. JSON: arrays → `T[]` / jagged `T[][]`. Large 2D иногда — flat `T[]` + manual index для speed.

---

## Comparison / Trade-offs

| Задача | Выбор |
|--------|-------|
| Фиксированная сетка | `T[,]` |
| Ragged строки | `T[][]` |
| Динамический размер | `List<T>` |

---

## Quick Reference

```
T[]   — a[i], Length
T[,]  — a[r,c], GetLength(0/1)
T[][] — a[r][c], outer + inner Length
```

---

## Key Takeaways

- Три формы — три разных layout и синтаксис индексации.
- Jagged rows требуют отдельной аллокации.
- `GetLength(dim)` для multidimensional; не путать с `Length`.
- Выбор формы — по геометрии данных.
