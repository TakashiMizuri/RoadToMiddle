## Why This Matters

Каждый `https://api.contoso.com` начинается с: **какой IP у `api.contoso.com`?** Без DNS нет TCP (`0.4.3`), TLS (`0.3.8`), HTTP (`0.3`). DNS — **phonebook internet**, application layer (`0.4.1`), обычно **UDP :53** (`0.4.4`).

Deploy за load balancer, migration region, DR — DNS updates. High TTL → clients на old IP hours. Low TTL → больше queries/latency. CNAME на CDN упрощает ops, но adds lookup steps.

Developers config hostnames в connection strings, CORS, certs — DNS = availability. «Works on my machine» — `/etc/hosts` или corporate cache.

## Core Concepts

### Что решает DNS

Люди — **names**; сеть — **addresses**. DNS map `www.example.com` → records. Tree: root → TLD `com` → `example.com` → `www`.

Resolution **distributed** — authoritative NS per zone; resolvers cache globally.

### Actors

**Stub resolver** — OS/browser (`getaddrinfo`). → **Recursive resolver** (ISP, 8.8.8.8, 1.1.1.1).

**Recursive** — walk tree root→TLD→authoritative; cache by **TTL**; return A/AAAA.

**Authoritative** — zone для `example.com`; задаёт TTL.

**Registrar** — domain registration; NS delegation.

```
Browser → Recursive (cache?) → root → .com → shop.com NS → A 203.0.113.55 → TCP
```

### Record types

**A** — IPv4. **AAAA** — IPv6. Dual-stack — оба.

**CNAME** — alias на другой hostname, не IP напрямую. **Нельзя** classic CNAME на zone apex — ALIAS/ANAME или A.

Also: MX, TXT, NS, SRV.

### TTL

Seconds to cache. `TTL=300` → 5 min. Low TTL перед migration — fast cutover. High TTL — stable но slow fix on error. **NXDOMAIN** тоже кэшируется.

«Some users old IP» — TTL + resolver hierarchy.

### Transport (`0.4.4`)

UDP :53 обычно; **TC truncated** или large → **TCP**. Firewall: оба.

### Link to HTTP (`0.3`)

`https://api.shop.com/v1/items`:

1. Resolve A/AAAA (этот урок)
2. TCP :443 (`0.4.3`)
3. TLS SNI `api.shop.com` (`0.3.8`)
4. HTTP Host (`0.3.7`)

DNS change без cert/CORS — fail после resolve.

## Under the Hood

### Recursive resolution

1. Root → referral `.com`
2. TLD → referral `shop.com` NS
3. Authoritative → A + TTL

**CNAME chain** до A/AAAA (limit ~10).

**DNSSEC** — larger responses, чаще TCP.

### Cache layers

Browser, OS stub, recursive (8.8.8.8), authoritative (source). `ipconfig /flushdns` — local only. Production: **lower TTL before change**.

### Message format

Header ID/flags; Question QNAME/QTYPE; Answer NAME/TYPE/TTL/RDATA. Tools: `dig +trace`, Wireshark `dns`.

### Split-horizon

`db.corp.local` — private IP internal resolver; public NXDOMAIN. K8s CoreDNS `cluster.local`.

### Failures

NXDOMAIN, SERVFAIL, stale cache, CNAME+MX apex issues, circular CNAME.

## Syntax / Commands / API

```bash
nslookup api.example.com
dig api.example.com A +ttlunits
dig www.example.com CNAME +trace
ipconfig /flushdns
```
