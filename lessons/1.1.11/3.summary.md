# break, continue, return — краткая выжимка

> `1.1.11` · ~7 мин на повторение

## Главная идея

**`break`**, **`continue`**, **`return`** меняют normal flow: выход из loop/switch, пропуск итерации, выход из method. Вместе с loops (`1.1.10`) и conditionals (`1.1.9`) дают читаемые console и service methods.

## Ключевые пункты

- **`break`** — exit **innermost** `switch` / loop; nested — только inner.
- **`continue`** — next iteration; skip rest body.
- **`return`** — end method; `return value;` или `return;` для void.
- **Guard clauses** — validate early, `return` early; меньше nesting.
- **`return` в Main** — exit app; **`break`** в menu — только loop/switch.
- C# **без** labeled `break`/`continue` → extract method или flag.

## Самые важные примеры

```csharp
if (!int.TryParse(input, out int n)) { Console.WriteLine("Bad"); continue; }
if (n == 0) break;
```

```csharp
if (string.IsNullOrWhiteSpace(name)) return "Anonymous";
return $"{name} ({age})";
```

## Частые ошибки

- `break` из nested loop — outer продолжается.
- Путаница **`return`** vs **`break`** в menu app.
- `continue` минуя update в `while`.

## Мнемоника

> **break — стоп цикл; continue — следующий; return — стоп method.**
