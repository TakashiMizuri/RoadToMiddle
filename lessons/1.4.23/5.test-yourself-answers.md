# Answers: Awaitable pattern

## Scoring

- 90–100%: отлично
- 80–89%: pass
- <80%: повтори лекции

---

## Answers

### 1. [Recall]
**Answer:** `GetAwaiter()` (на expression), `IsCompleted`, `GetResult()`, `OnCompleted` (или `UnsafeOnCompleted`).

**Why:** Минимальный контракт C# awaitable expressions.

---

### 2. [Recall]
**Answer:** `UnsafeOnCompleted` не захватывает **ExecutionContext**; дешевле, но continuation не восстанавливает ambient context автоматически.

**Why:** Performance path для BCL awaiters (`Task`).

---

### 3. [Understanding]
**Answer:** Observation/marshaling exception к моменту **observation**; `GetResult()` — точка observation при await lowering.

**Why:** Unobserved task exception — отдельный путь; await observes через awaiter.

**Common mistake:** Думать, что exception «на Task» без GetResult/await.

---

### 4. [Understanding]
**Answer:** Library может использоваться из UI (WinForms/WPF) с SyncContext; `false` документирует и гарантирует отсутствие marshal; в shared libraries — best practice независимо от host.

**Why:** Host-agnostic library design.

---

### 5. [Understanding]
**Answer:** Sync fast path — сразу `GetResult()` без allocation/register continuation; избегает лишней работы если уже complete.

**Why:** Hot path optimization; также race handling если complete между check и register.

---

### 6. [Application]
**Answer:** `TaskCompletionSource<T>`; подписка на event → `SetResult`/`SetException`; вернуть `tcs.Task` — стандартный Task awaiter.

**Why:** Production-standard interop bridge.

---

### 7. [Application]
**Answer:** Всегда async path: регистрация continuation, yield из MoveNext, resume позже (например thread pool).

**Why:** `IsCompleted false` forces OnCompleted path every time.

---

### 8. [Analysis]
**Answer:** **B** — неверно; нужен явный pattern (GetAwaiter + awaiter members).

**Why:** Compiler не делает arbitrary types awaitable.

---

### 9. [Analysis]
**Answer:** Undefined/incorrect behavior, exception, corrupted state — **ValueTask** often single-consumer unless backed by reusable source.

**Why:** Pooling/single-shot semantics.

**Common mistake:** Treat ValueTask like Task for multiple awaits.

---

### 10. [Understanding]
**Answer:** Logical call context flowing across async hops: `AsyncLocal`, security/culture in older stacks; captured in `OnCompleted`, not in `UnsafeOnCompleted`.

**Why:** Explains ConfigureAwait and ambient state.

---

### 11. [Application]
**Answer:** Decompile async method → `MoveNext` → calls to `AsyncTaskMethodBuilder.AwaitOnCompleted` vs `AwaitUnsafeOnCompleted` depending on awaiter type (e.g. Task vs custom).

**Why:** Visible compiler choice in IL/state machine.
