# Interface: implicit vs explicit implementation

> Roadmap: `1.2.9` · Node: `1.2` — C# OOP · Depth: **глубоко**

## Learning Objectives

После этого урока ты сможешь:

- Implement **interface implicitly** — public members на class и через interface.
- Implement **explicitly** — members только при reference typed as interface.
- Разрешать **name collisions** двух interfaces с одинаковыми signatures.
- Объяснить **dispatch** и metadata для implicit vs explicit.
- Выбирать explicit, чтобы **скрыть** detail или разделить conflicting APIs.
- Читать BCL patterns: **`IEnumerable`**, **`IComparable`**.

---

## Why This Matters

Урок `1.2.8` — interfaces как contracts. При **`implements`** C# даёт два синтаксиса, меняющих **кто может вызвать member** и **как он виден в IntelliSense**.

**Implicit** — interface methods как **public class API**. **Explicit** **скрывает** от direct class access — когда **`Draw()`** разный для **`IDrawable`** и **`IPrinter`**, или **`CompareTo`** не должен быть в public surface **`Widget`**.

Middle-разработчик встречает explicit в **`IComparable<T>`**, **`IEnumerable<T>`**, adapters. Explicit «для encapsulation» везде — плохая discoverability; без explicit при collision — **CS0539** / ambiguous calls.

---

## Core Concepts

### Implicit implementation

**Public** member совпадает с interface signature — member **и** class API, **и** fulfillment interface.

```csharp
public class ConsoleLogger : ILogger
{
    public void Log(string message) => Console.WriteLine(message);
}

ConsoleLogger c = new();
c.Log("hi");        // OK
ILogger l = c; l.Log("hi"); // OK
```

Когда interface **есть** суть class (**`SmtpEmailSender : IEmailSender`**) — implicit уместен.

### Explicit implementation

Prefix **`InterfaceName.`**:

```csharp
void ILogger.Log(string message) => Console.WriteLine($"[quiet] {message}");
```

**`q.Log(...)`** — CS1061. **`((ILogger)q).Log(...)`** — OK.

Explicit members **не public** на class — доступ через interface reference (или reflection). IL — **private** MethodDef + **InterfaceImpl** mapping.

Explicit когда:

1. **Скрыть** auxiliary members (**`CompareTo`**).
2. **Collision** — **`IA.Save`** vs **`IB.Save`**.
3. **Semantic duplicate** — class уже имеет **`Draw()`**.
4. **Adapter** legacy API.

### Name collisions

**`IA`** и **`IB`** оба **`void Save()`** — один public implicit **`Save`** не даёт разной semantics:

```csharp
void IA.Save() => SaveForDatabase();
void IB.Save() => SaveForBackup();
```

**One implicit OR both explicit** для ясности.

### Properties и indexers

```csharp
string IReadOnlyById.this[int id] => _map[id];
```

Public indexer на class только через interface type.

## Under the Hood

**InterfaceImpl** table: **(InterfaceMethod, ClassMethod)**. Implicit — один MethodDef. Explicit — отдельный MethodDef, **private** на class, **callvirt** через interface slot.

**Default interface methods** (`1.2.10`) — class explicit/implicit override перекрывает default.

---

## Examples

**`IEnumerable`** — generic **`GetEnumerator`** public; non-generic часто **explicit**.

**`Storage : IPersistent, ICache`** — два **`Flush`** explicit.

**`StripePaymentGateway : IPaymentGateway`** — implicit **`ChargeAsync`** as primary API.

---

## Common Mistakes

Explicit everywhere. Duplicate public + explicit. Забыли cast. **Two implicit** same signature — impossible.

---

## Comparison

| | Implicit | Explicit |
|---|----------|----------|
| На class | public | hidden |
| Вызов | `obj.M()` | `((I)obj).M()` |
| Collision | one signature | per interface |

---

## Key Takeaways

- Implicit merges interface и public class API.
- Explicit — только через interface reference.
- Explicit resolves collisions.
- BCL: **`IComparable`**, non-generic **`IEnumerable`** — типично explicit.
