## Answers

### 1. [Recall]
**Answer:** `public static` (и хотя бы один параметр — declaring type, кроме conversion).
**Why:** Спецификация C# для user-defined operators.

### 2. [Recall]
**Answer:** Нет. `=` не перегружается — присваивание встроено в язык.
**Why:** Иначе ломалась бы семантика assignment и ссылок.

### 3. [Recall]
**Answer:** `==` и `!=` (всегда парой).
**Why:** Один без другого даёт противоречивый API.

### 4. [Understanding]
**Answer:** Для struct нет встроенного `+`; компилятор ищет user-defined `operator +` и не находит.
**Why:** Operators не наследуются автоматически от «математического смысла» типа.

### 5. [Understanding]
**Answer:** Без перегрузки `==` на class — reference equality; `Equals` может быть true, `==` — false.
**Why:** Пользователи ожидают согласованности (`1.2.16`).

### 6. [Understanding]
**Answer:** Практически ничем — оба static call; JIT может инлайнить одинаково.
**Why:** Выбор — читаемость, не performance.

### 7. [Application]
**Answer:** `public static Vector3 operator +(Vector3 a, Vector3 b)`
**Why:** Бинарный operator: два операнда, возврат `Vector3`.

### 8. [Application]
**Answer:** В обоих одинаково по логике; проверка в `operator +` или в `Add`, куда делегирует operator — одна точка истины.
**Why:** Дублировать проверку в двух местах нельзя; часто `operator +` вызывает `Add`.

### 9. [Analysis]
**Answer:** Плохая — `+` читается как арифметика/объединение, не как side-effect apply; нарушает least surprise.
**Why:** Именованный `Apply`/`Migrate` явнее для аудита и onboarding.

### 10. [Analysis]
**Answer:** `false` для двух разных экземпляров с одинаковыми полями (reference equality).
**Why:** `==` не переопределён.

### 11. [Understanding]
**Answer:** Да, structural `==` и `!=` генерируются.
**Why:** Поведение record по умолчанию.

### 12. [Application]
**Answer:** `public static Temperature operator -(Temperature t)`
**Why:** Unary operator — один параметр типа `Temperature`.

### 13. [Understanding]
**Answer:** Теряется `Currency`; silent promotion в decimal вводит в заблуждение и баги в финансах.
**Why:** Implicit conversion должен быть безопасным и очевидным.

### 14. [Recall]
**Answer:** Например: `+`, `-`, `*` (также `/`, `%`).
**Why:** Стандартный набор binary arithmetic.

### 15. [Application]
**Answer:** `Money.Add(a, b)` или instance `a.Add(b)` по style guide.
**Why:** Явное имя вместо запрещённого operator.
