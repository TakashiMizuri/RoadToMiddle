## Answers

### 1. [Recall]
**Answer:** `novalidate` на `<form>`.
**Why:** Skips interactive constraint validation on submit.

### 2. [Recall]
**Answer:** `minlength` и `maxlength`.
**Why:** `min`/`max` не применяются к string length на text.

### 3. [Recall]
**Answer:** `step` (например `step="2"` для чётных при min=0).
**Why:** step определяет legal increments от min.

### 4. [Understanding]
**Answer:** Client можно обойти (curl, DevTools); attributes не на JSON body; business rules на server.
**Why:** Never trust browser.

### 5. [Understanding]
**Answer:** `typeMismatch` — не соответствует type (email/url); `patternMismatch` — не матчит regex pattern.
**Why:** Different constraint sources.

### 6. [Understanding]
**Answer:** Many browsers include title in validation bubble as format hint.
**Why:** Regex alone opaque to users.

### 7. [Application]
**Answer:** `<input type="number" min="18" max="120" step="1" required />` (+ name/id in real form).
**Why:** Integer age bounds.

### 8. [Application]
**Answer:** `<input type="password" minlength="12" required />`
**Why:** Length constraint on password string.

### 9. [Application]
**Answer:** Example: `pattern="^#[0-9A-Fa-f]{6}$" title="Six-digit hex, e.g. #1a2b3c"`
**Why:** Full-string hex match.

### 10. [Analysis]
**Answer:** Нет для длины username; `min`/`max` на text не length. Use `minlength="5" maxlength="10"`.
**Common mistake:** confusing numeric min with string length.

### 11. [Analysis]
**Answer:** Publish: `<button type="submit">`; Draft: `<button type="submit" formnovalidate>`.
**Why:** formnovalidate skips validation for that submit.

### 12. [Analysis]
**Answer:** (1) syntactically valid но несуществующий mailbox; (2) disposable/temp email; также typo domains, plus sign aliases — format check ≠ deliverability/policy.
**Why:** email type — format only.
