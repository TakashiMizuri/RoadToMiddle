## Answers

### 1. [Recall]
**Answer:** Start line, headers, body (optional).
**Why:** Классическая структура RFC 9112.

### 2. [Recall]
**Answer:** Host.
**Why:** HTTP/1.1 требует Host для virtual hosting на shared IP.

### 3. [Recall]
**Answer:** Content-Length.
**Why:** Указывает N bytes body после blank line.

### 4. [Understanding]
**Answer:** Пустая строка (`CRLF` после последнего header) — delimiter между header section и body; parser переключает state.
**Why:** Без неё нельзя надёжно найти начало body.

### 5. [Understanding]
**Answer:** Request: METHOD + target + HTTP-version. Response: HTTP-version + status code + reason phrase.
**Why:** Разные роли start line.

### 6. [Understanding]
**Answer:** Когда длина body unknown заранее или нужен streaming (chunked pipeline); proxy/server flush без total size.
**Why:** Alternative to fixed Content-Length.

### 7. [Understanding]
**Answer:** Content-Type говорит receiver, как интерпретировать bytes (JSON parser vs HTML vs binary); charset для text types.
**Why:** Same bytes, different semantics without header.

### 8. [Application]
**Answer:**
```
GET /health HTTP/1.1
Host: api.local

```
**Why:** Minimal valid GET with Host and terminating blank line.

### 9. [Application]
**Answer:** No body; 0 bytes.
**Why:** 204 semantics — success without representation body.

### 10. [Analysis]
**Answer:** B неверно.
**Why:** Content-Length + chunked together invalid/conflict on HTTP/1.1.

### 11. [Analysis]
**Answer:** Ждать оставшиеся bytes до timeout; возможен incomplete request error; keep-alive broken.
**Why:** Parser trusts Content-Length.

### 12. [Understanding]
**Answer:** Connection meaningful для текущего TCP hop; proxy may strip. Authorization must reach origin server end-to-end.
**Why:** Hop-by-hop vs end-to-end classification in HTTP.

### 13. [Understanding]
**Answer:** Pseudo-headers `:method`, `:path`, `:scheme`, `:authority`; status as `:status` in response.
**Why:** HTTP/2 binary framing replaces text start line.

### 14. [Application]
**Answer:** Header (response header section).
**Why:** Custom metadata not in start line or body.

### 15. [Recall]
**Answer:** Carriage Return + Line Feed (`\r\n`); line terminator in HTTP/1.1 messages.
**Why:** Spec-defined parsing boundaries.
