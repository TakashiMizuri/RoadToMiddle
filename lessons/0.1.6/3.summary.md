# .gitignore и .gitkeep — краткая выжимка

> `0.1.6` · ~8 мин на повторение

## Главная идея

**`.gitignore`** — список паттернов, которые Git **не предлагает** к добавлению (untracked files). Это защита от build artifacts, зависимостей и secrets в history. **`.gitkeep`** — конвенция: placeholder-файл, чтобы Git отслеживал **пустую** директорию.

## Ключевые пункты

- `.gitignore` влияет только на **untracked** файлы. Уже закоммиченный файл нужно убрать: `git rm --cached <file>`.
- **Синтаксис:** `*` (любые символы), `?` (один символ), `**` (любая глубина папок), `/` в начале (только корень repo), `!` (negation — исключение из ignore).
- **Три слоя ignore:** global (`core.excludesfile`), `.git/info/exclude` (личное, не коммитится), `.gitignore` (shared, в repo).
- Типичные паттерны: `[Bb]in/`, `[Oo]bj/`, `node_modules/`, `.env`, `!.env.example`.
- **Negation** не работает для файла внутри полностью ignored **директории** — игнорируй содержимое (`logs/*`), не саму папку (`logs/`).
- **`git check-ignore -v path`** — показывает, какое правило и из какого файла сработало.

## Самые важные примеры

**Базовый .NET + Node ignore:**
```gitignore
[Bb]in/
[Oo]bj/
node_modules/
.env
!.env.example
```

**Пустая папка через .gitkeep:**
```bash
mkdir -p logs/archive
touch logs/.gitkeep
git add logs/.gitkeep
```

**Перестать track'ить .env:**
```bash
git rm --cached .env
git commit -m "Stop tracking .env"
```

## «Под капотом» в одном абзаце

При `git status` Git сканирует working tree и для каждого path проверяет цепочку ignore-правил (global → exclude → `.gitignore` вверх по дереву). Matching — fnmatch-style globs. Файлы уже в **index** ignore не затрагивает — отсюда необходимость `git rm --cached`.

## Частые ошибки

- Добавить `.env` в `.gitignore` после коммита — secret остаётся в history.
- `git add -f` для secrets — обходит ignore.
- Слишком широкий `*.json` — ломает `package.json`.
- Путать `.gitkeep` (структура в Git) с runtime-создаваемыми папками.

## Мнемоника

> **Ignore = дверь** (untracked не проходят). **Уже внутри** — нужен `rm --cached`. **Пустая папка** — нужен хотя бы один файл (`.gitkeep`).
