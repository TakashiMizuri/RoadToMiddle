# const vs readonly vs static readonly — краткая выжимка

> `1.1.19` · ~8 мин на повторение

## Главная идея

Три способа «не менять значение» различаются **когда** value фиксируется и **как** другие assemblies его читают. **`const`** встраивается в IL consumer at compile time; **`static readonly`** и instance **`readonly`** читаются как fields at runtime.

## Ключевые пункты

- **`const`**: compile-time literal, implicitly static; только primitive/`string`/null; args для attributes.
- **`readonly`**: assign в ctor/field init; **per instance**; runtime expressions OK.
- **`static readonly`**: one per type; static ctor/init; **`ldsfld`** у consumer.
- **Cross-assembly `const` change** без recompile consumer → **stale inlined value**.
- Illegal: **`const DateTime`**, **`const new Regex(...)`**.
- **`readonly struct`**, **`init` property** — не то же самое, что **`readonly` field**.
- Production tunables → **`IOptions<T>`**, не public **`const`**.

## Самые важные примеры

```csharp
public const int PageSize = 20;              // may inline into consumers
public static readonly int Max = GetMax();   // runtime init, field load
public sealed class Order
{
    private readonly Guid _id = Guid.NewGuid(); // per instance
}
```

## «Под капотом» в одном абзаце

**`const`** → metadata **Constant** table, consumer IL **`ldc.i4`**. **`static readonly`** → **`.cctor`**, consumer **`ldsfld`**. Instance **`readonly`** → **`stfld`** в constructor. Versioning footgun — **`const`** in shared libraries.

## Частые ошибки

- Public **`const`** для limits/timeouts в NuGet.
- Ожидание hot-reload constant через DLL replace без recompile.
- **`readonly` ref** на mutable object graph.

## Мнемоника

> **Forever literal in code → `const`. Compute once per process → `static readonly`. Once per object → `readonly`.**
