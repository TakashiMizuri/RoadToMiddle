# Tags и semver — краткая выжимка

> `0.2.13` · ~7 мин

## Главная идея

**Tag** — имя для **фиксированного** commit (release). **Annotated tag** (`-a`) — объект с message и metadata для prod. **Semver** `MAJOR.MINOR.PATCH` описывает тип изменений.

## Ключевые пункты

- **Lightweight:** `git tag v1.0.0` — только ref.
- **Annotated:** `git tag -a v1.0.0 -m "..."` — release record.
- **Push:** `git push origin v1.0.0` (tags не едут с обычным push по умолчанию всегда — проверяй remote).
- **Semver:** fix→PATCH, feat→MINOR, BREAKING→MAJOR.
- **Не retag'ать** опубликованный semver на другой hash.
- **`git show v1.0.0`**, **`git log v1.0.0..v1.1.0`**.

## Примеры

```bash
git tag -a v2.3.2 -m "v2.3.2: hotfix SEC-401"
git push origin v2.3.2
```

```bash
git bisect start main v2.4.0
```

## Частые ошибки

Tag до CI; lightweight для prod; `push --tags` с мусорными local tags; tag на wrong commit.

## Мнемоника

> **Branch** двигается · **Tag** застыла · **Release = annotated + semver + push**
