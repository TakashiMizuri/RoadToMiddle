# Pattern matching — краткая выжимка

> `1.3.13` · ~8 мин на повторение

## Главная идея

**Pattern matching** превращает `switch`/`is` в классификацию **по форме данных** (тип, свойства, диапазон, элементы списка) с одновременным **binding** переменных. **Switch expression** (`x switch { ... }`) возвращает значение и должен быть exhaustive или иметь `_`.

## Ключевые пункты

- **Type pattern:** `string s`, `int n` — замена `as` + null check.
- **Property pattern:** `{ Total: > 1000, Customer.IsVip: true }` — null-safe доступ к свойствам.
- **Relational:** `>= 90`, `and`, `or`, `not` — порядок arms = first match wins.
- **List pattern:** `[]`, `[a]`, `[a, b, ..]` — arity и head/tail без ручного indexing.
- **when guard** — дополнительный фильтр после match pattern.
- **Positional patterns** для records: `( > 0, > 0)`.
- Switch **statement** для side effects; **expression** — для pure mapping.

## Самые важные примеры

```csharp
// Switch expression
var band = score switch { >= 90 => "A", >= 80 => "B", _ => "C" };

// Property + relational
var d = order switch {
    { Total: > 1000, Customer.IsVip: true } => 0.15m, _ => 0m };

// List pattern CLI
args switch {
    ["create", var name] => Create(name),
    ["list"] => List(),
    _ => Unknown };
```

## Частые ошибки

- Relational arms в неверном порядке (`>= 0` до `>= 90`).
- Side effects в switch expression.
- Non-exhaustive enum switch без `_` при десериализации новых значений.
- `[a,b,c]` вместо `[a,b,..]` когда нужно «≥2 элемента».

## Мнемоника

> **Pattern = «это выглядит как…» + bind; первая подошедшая ветка побеждает.**
