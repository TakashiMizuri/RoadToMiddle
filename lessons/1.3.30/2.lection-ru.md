# LINQ Performance — Avoid Multiple Enumeration

> Roadmap: `1.3.30` · Node: `1.3` — C#: Advanced Topics · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Определить **deferred execution**, **multiple enumeration**, **materialization**.
- Находить double/triple enumeration cold **`IEnumerable`**.
- Fixes: **`ToList` once**, **`IReadOnlyList`**, single **`IQueryable`** round-trip.
- **`Count()`** vs **`ICollection.Count`** — O(n) vs O(1).
- Optimize EF pipelines; avoid N+1.
- Trade-off memory vs repeated computation.

---

## Why This Matters

Typical footgun:

```csharp
var query = LoadFromDatabase();
if (query.Any()) {
    var list = query.ToList();
    var count = query.Count();
}
```

Каждый immediate operator на **cold** sequence может **re-run** pipeline — duplicate SQL, double file read. Junior думает «LINQ как SQL — один раз»; до materialize — **нет**.

Middle dev materialize на **stable boundaries**, signature **`IReadOnlyList`** для reuse, один **`ToListAsync`** для EF. Закрываем node **1.3** перед **async**.

---

## Core Concepts

### Deferred vs Immediate

Deferred: `Where`, `Select`, `yield` — work on pull.

Immediate: `ToList`, `Count`, `First`, `Any`, `Sum` — run now.

### Multiple Enumeration

Two **`foreach`** / **`Any` + `ToList`** на cold source → 2× work. Fix: **`var list = query.ToList()`** once.

### Count pitfalls

**`list.Count`** O(1). **`seq.Count()`** on pure IEnumerable — O(n) full scan. Extension optimizes **`ICollection<T>`**.

### Any + ToList

Two passes → **`ToList` first**, then **`Count`**.

### IQueryable

`CountAsync` + `ToListAsync` — два SQL часто OK. Loop **`First`** per id — **N+1** — один query + dictionary.

### Closures

Deferred **`Where`** captures variable — value at **enumeration time**, not definition time.

### When NOT materialize

Streaming millions — single **`foreach` forward**. **`OrderBy`/`GroupBy`** buffer all — not streaming.

---

## Under the Hood

**Where·Select** — fused iterators, no intermediate list. **`OrderBy`** — full buffer.

API: **`IEnumerable`** = lazy/one-shot ambiguity; **`IReadOnlyList`** = snapshot contract.

EF: accidental client eval + re-enumerate = disaster.

---

## Examples

```csharp
var rows = await db.Items.Where(...).ToListAsync(ct);
if (rows.Count == 0) return NotFound();
var total = rows.Sum(r => r.Amount); // same list, one SQL
```

Bad: `pipeline.Count(); return pipeline.ToList();`

Good: `var list = pipeline.ToList();` once.

---

## Common Mistakes

`Any` then `foreach`. Multiple helpers each enumerate same **`IEnumerable`**. `OrderBy` on huge stream. Changing captured local before enumerate.

---

## Key Takeaways

- Cold = re-enumerate = re-execute.
- Materialize boundary: **`ToList`/`ToArray`**.
- **`Any`+`ToList`** — two passes.
- **`OrderBy`** buffers all.
- EF: one composed query; no N+1.
