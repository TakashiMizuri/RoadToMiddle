# Conflict resolution — краткая выжимка

> `0.2.5` · ~8 мин на повторение

## Главная идея

**Conflict** — Git не может auto-merge overlapping изменения; в файле **markers**, index — stages 1/2/3. Ты редактируешь финальный код, **удаляешь markers**, `git add`, затем `git commit` (merge) или `git rebase --continue` (rebase).

## Ключевые пункты

- Markers: `<<<<<<< HEAD` / `=======` / `>>>>>>> branch`.
- **Rebase:** ours/theirs **перевёрнуты** vs merge intuition.
- `git status` — Unmerged paths; `git ls-files -u` — stages.
- Abort: `git merge --abort` / `git rebase --abort`.
- `--ours` / `--theirs` — whole file; осторожно на rebase.
- Lockfiles: regenerate, не ручной merge.
- IDE / `git mergetool` / `git diff --merge`.

## Самые важные примеры

```bash
git merge feature          # conflict
# edit files, remove markers
git add .
git commit

git rebase main            # conflict
git add .
git rebase --continue
```

## Частые ошибки

- Markers в committed code.
- Blind `--ours` при rebase.
- Resolve без tests.

## Мнемоника

> **Conflict = Git спросил «как правильно?»** — не commit'ь, пока не ответишь кодом без markers.
