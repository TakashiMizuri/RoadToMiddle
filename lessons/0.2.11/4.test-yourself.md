## Practical

### P1. Hotfix sequence
Расставь шаги hotfix (a–f) в правильном порядке:

- a) `git cherry-pick -x <fix-hash>` на hotfix-ветке  
- b) Merge PR в `main`, tag `v2.1.1`  
- c) `git checkout main && git pull`  
- d) `git checkout -b hotfix/critical-fix`  
- e) Backport cherry-pick на `release/2.0`  
- f) Push hotfix branch, открыть PR
