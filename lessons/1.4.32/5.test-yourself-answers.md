# Answers: AsyncLocal

## Scoring

- 90–100%: отлично · 80–89%: pass · <80%: повтори lection + summary

## Answers

### 1. [Recall]
**Answer:** Ambient значение, привязанное к logical execution context, а не к потоку.
**Why:** Flows across await via ExecutionContext.

### 2. [Recall]
**Answer:** После await continuation может быть на другом потоке — ThreadLocal даст другой/пустой слот.
**Why:** Physical thread changes.

### 3. [Understanding]
**Answer:** Accessor читает HttpContext из AsyncLocal, установленного middleware pipeline.
**Why:** ASP.NET Core implementation pattern.

### 4. [Understanding]
**Answer:** Pool threads переиспользуются; без clear следующий work увидит чужой correlation/tenant.
**Why:** Ambient leak anti-pattern.

### 5. [Application]
**Answer:** В middleware на входе запроса — единая граница, все downstream видят.
**Why:** Edge sets cross-cutting context.

### 6. [Analysis]
**Answer:** Не обязательно — ветки с копиями контекста на момент создания; зависит от flow/ExecutionContext.
**Why:** Copy-on-write branching.

### 7. [Understanding]
**Answer:** Cross-cutting diagnostics (correlation, culture) в глубоких стеках и чужих API.
**Why:** Avoid parameter pollution.

### 8. [Understanding]
**Answer:** Обязательные business dependencies, тестируемость, явные контракты.
**Why:** Hidden deps hurt design.

### 9. [Application]
**Answer:** Явно из payload сообщения/scope factory; не HttpContext accessor.
**Why:** No HTTP ambient in worker.

### 10. [Recall]
**Answer:** Данные (в т.ч. AsyncLocal), восстанавливаемые при async continuation.
**Why:** Logical call context.

### 11. [Analysis]
**Answer:** Ветки могут мутировать один list — гонки и cross-talk.
**Why:** Shared mutable reference.

### 12. [Understanding]
**Answer:** Нет — cancellation должен быть cooperative и явно передан в I/O API.
**Why:** AsyncLocal not wired to I/O cancellation.
