# Monorepo vs polyrepo — краткая выжимка

> `0.2.15` · ~7 мин

## Главная идея

**Monorepo** — много projects в **одном** Git repo (atomic PR, общая policy). **Polyrepo** — **несколько** repos (независимые релизы, чёткие границы). Git одинаков; отличается **организация** и **CI/versioning**.

## Ключевые пункты

- **Polyrepo:** semver per repo; cross-feature = несколько PR + version matrix.
- **Monorepo:** один clone; API+React в одном commit; tags — whole repo или `api/vX.Y.Z`.
- **Monorepo CI:** path filters / affected (Nx, `paths:` в Actions).
- **Polyrepo CI:** проще per repo; integration e2e отдельно.
- Hotfix/cherry-pick/tags **умножаются** в polyrepo или **консолидируются** в monorepo.
- Одна **`.gitattributes`** в monorepo vs drift в polyrepo.

## Частые ошибки

Monorepo без affected CI; polyrepo с вечной synchronized release; dogma «только monorepo».

## Мнемоника

> **Mono** = one repo, atomic · **Poly** = many repos, autonomy · **Coupling** decides
