# data-* attributes и custom elements (concept)

> Roadmap: `0.5.15` · Node: `0.5` — HTML · Depth: понимание

## Learning Objectives

После этого урока ты сможешь:

- Использовать **`data-*` attributes** для хранения custom metadata в valid HTML.
- Читать и писать значения через **`element.dataset`** в JavaScript.
- Объяснить, почему `data-*` лучше произвольных non-standard attributes.
- Описать **Web Components** на высоком уровне: Custom Elements, Shadow DOM, HTML templates.
- Различать **semantic HTML + data-*** и **custom element tags** и выбирать подход.

---

## Why This Matters

HTML даёт semantic elements — `<article>`, `<button>`, `<form>` — но приложениям нужна **private metadata** на nodes: SKU строки, ID tab panel, JSON config, test hooks. До `data-*` злоупотребляли `class="sku-12345"` или invalid `sku="12345"`, ломая validation и CSS.

**`data-*`** — standards-track способ расширить markup без pollution global attribute namespace. В React будут `data-testid`, `data-active`, props → DOM attributes. `dataset` связывает static HTML с DOM scripting и **Custom Elements** — native reusable widgets.

Custom Elements регистрируют `<user-card>` как полноценный tag с lifecycle. В React-коде ты реже пишешь Web Components, но **consumes** их (design systems, players) и debug в DevTools. Conceptual clarity помогает не воевать с platform.

## Core Concepts

### Паттерн data-*

Любой attribute с префиксом **`data-`** — **custom data attribute**. Spec гарантирует отсутствие conflict с future standard attributes. В HTML имена **case-insensitive**; **`dataset`** exposes **camelCase**.

```html
<article id="product-42" data-sku="WDG-42" data-stock="12" data-featured="true">
  <h2>Widget Pro</h2>
</article>
```

Правила:

- Префикс **`data-`** обязателен.
- **No uppercase** в HTML name (`data-product-id`, не `data-productId`).
- Values в DOM — **strings**; числа/boolean парси в JS.
- Metadata, не warehouse для huge JSON.

### Чтение через dataset

**`HTMLElement.dataset`** — **`DOMStringMap`**: `data-x-y` → `element.dataset.xY`.

```javascript
const article = document.querySelector('#product-42');
console.log(article.dataset.sku);      // "WDG-42"
console.log(article.dataset.stock);    // "12"
const stock = Number(article.dataset.stock);
article.dataset.featured = 'false';
```

Для nested data — `JSON.stringify` sparingly + try/catch, или API fetch.

```html
<button data-action="add-to-cart" data-product-id="42">Add</button>
```

```javascript
document.body.addEventListener('click', (e) => {
  const btn = e.target.closest('[data-action]');
  if (!btn) return;
  if (btn.dataset.action === 'add-to-cart') {
    const id = btn.dataset.productId;
  }
});
```

**Event delegation** — один listener, behavior из `data-action` — pattern для vanilla JS и analogs в React.

### data-* vs id, class, ARIA

| Mechanism | Best for |
|---|---|
| `id` | Unique id, fragments, labels |
| `class` | CSS, группы элементов |
| `data-*` | JS config, tests, invisible metadata |
| ARIA | Accessibility когда native role недостаточен |

Не дублируй ARIA roles через `data-role="button"` на `<div>` — используй `<button>` (`0.5.11`).

### Validation и selectors

```css
tr[data-status="shipped"] { opacity: 0.7; }
```

Non-`data` custom attributes fail **validation** (`0.5.16`).

### Custom Elements — overview

**Web Components:**

1. **Custom Elements** — new tags + lifecycle
2. **Shadow DOM** — encapsulated subtree/styles
3. **`<template>`** — inert markup для clone

```javascript
class UserCard extends HTMLElement {
  connectedCallback() {
    const name = this.getAttribute('name') ?? 'Guest';
    this.textContent = `Hello, ${name}`;
  }
}
customElements.define('user-card', UserCard);
```

```html
<user-card name="Ada"></user-card>
```

**Autonomous** — extend `HTMLElement`. **Customized built-in** — extend `HTMLButtonElement` (реже, Safari limits).

Lifecycle:

- **`connectedCallback`** — inserted
- **`disconnectedCallback`** — removed
- **`attributeChangedCallback`** — observed attrs change
- **`adoptedCallback`** — moved document (rare)

**`observedAttributes`** — static getter; analog «props changed» в React mentally.

### Shadow DOM (кратко)

Hidden subtree, **scoped CSS**. External styles не leak in/out. Design systems: `<ds-modal>` без war с global `.button`.

React и Web Components coexist; многие teams выбирают React для app UI — platform components ценны для **cross-framework widgets**.

### Когда data-* vs Custom Elements

**data-*** — enhance semantic HTML; test hooks; React/Vue components на framework level.

**Custom Elements** — framework-agnostic widget; Shadow encapsulation; embed script на marketing site.

Комбо: `<tabs-widget data-default="settings">` + `customElements.define`.

## Under the Hood

`data-*` — **`Attr` nodes**; `dataset` live view. Parser lowercases; hyphen → camelCase по spec algorithm.

Custom Elements registry global per window. `define` throws без hyphen или на duplicate. До define parser видит **`HTMLUnknownElement`**. **`customElements.whenDefined`** для defer interaction.

Upgrade timing и FOUC — preview проблем SSR + React hydration с CE.

## Examples

### Tabs только на data-*

```html
<div class="tabs" data-active-tab="profile">
  <button type="button" data-tab-target="profile">Profile</button>
  <button type="button" data-tab-target="billing">Billing</button>
  <section data-tab-panel="profile">...</section>
  <section data-tab-panel="billing" hidden>...</section>
</div>
```

### Minimal custom element

```javascript
class FormattedDate extends HTMLElement {
  connectedCallback() {
    const iso = this.getAttribute('datetime');
    if (iso) this.textContent = new Date(iso).toLocaleDateString();
  }
}
customElements.define('formatted-date', FormattedDate);
```

## Common Mistakes & Anti-patterns

Huge JSON в data attributes. camelCase в HTML. Dataset без parsing numbers/booleans. Custom element без hyphen. Div+data-click вместо button.

---

## Production & Real-World Notes

**`data-testid`** в RTL. Embed widgets как CE + Shadow. CSP — define в external JS. React: `data-*` на native DOM inside components normal.

---

## Comparison / Trade-offs

data-* vs hidden inputs (form submit). CE vs React (portability vs ecosystem). Attributes vs properties на custom elements.

## Key Takeaways

- **`data-*`** — valid custom metadata; **`dataset`** — camelCase, string values.
- Semantic elements first; data-* configures behavior.
- **Custom Elements** — hyphen in name; lifecycle callbacks.
- **Web Components** suite — CE + Shadow + template.
- React встречает `data-*` и CE in the wild — умеешь inspect и integrate.
