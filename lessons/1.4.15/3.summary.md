# ValueTask — pooling и «не await'ить дважды» — краткая выжимка

> `1.4.15` · ~8 мин на повторение

## Главная идея

`ValueTask`/`ValueTask<T>` — **struct awaitable**, который на sync path часто **не аллоцирует** `Task` на heap. Цена: pooled реализации через `IValueTaskSource` делают **`ValueTask` одноразовым** — повторный `await` = **undefined behavior**. Безопасный reuse: **`AsTask()`** один раз.

## Ключевые пункты

- Внутри: sync result, wrapped `Task`, или pooled `IValueTaskSource`.
- **Pool pattern:** после completion source возвращается в pool и может обслужить другой вызов.
- **Правило:** await **один раз**; не хранить в поле; не передавать в `WhenAll` без `.AsTask()`.
- **`async ValueTask`** редко выгоден — при suspend всё равно state machine + часто `Task`.
- Public API по умолчанию — **`Task`**; `ValueTask` — после profiling hot path.
- BCL: `Stream.ReadAsync`, EF Core, System.Text.Json — типичные места.

## Самые важные примеры

**Fast path без аллокации:**
```csharp
if (_cache.TryGetValue(id, out User? u))
    return new ValueTask<User?>(u);
return new ValueTask<User?>(FetchFromDbAsync(id, ct));
```

**Безопасная композиция:**
```csharp
Task<int> t = valueTask.AsTask();
await Task.WhenAll(t, other);
```

**Anti-pattern:**
```csharp
var vt = GetValueTask();
await vt;
await vt; // UNDEFINED
```

## «Под капотом» в одном абзаце

`ManualResetValueTaskSourceCore<T>` реализует `IValueTaskSource` без per-call alloc при pooling. `ValueTask` несёт version token; после `Reset()` и возврата в pool второй consumer того же struct может прочитать чужой результат. Awaiter при `IsCompleted == true` вызывает `GetResult()` inline; иначе регистрирует continuation на underlying task/source.

## Частые ошибки

- Double await «работало локально» — ломается под нагрузкой.
- Кеш `ValueTask` в поле класса для «ленивого» await.
- `Task.WhenAll` напрямую на `ValueTask` без `.AsTask()`.
- Переход всего API на `ValueTask` без измерений.

## Мнемоника

> **ValueTask = одноразовый билет в кино.** После сеанса (await) билет сгорает; нужен второй заход — купи `Task` через `.AsTask()`.
