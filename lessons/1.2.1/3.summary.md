# Class blueprint, object instance, reference semantics — краткая выжимка

> `1.2.1` · ~10 мин на повторение

## Главная идея

**Class** — blueprint (тип в metadata): fields + methods. **Instance** — конкретный object на heap после **`new`**. Переменная class-типа хранит **reference**; assignment **копирует ссылку**, не объект (`1.1.6`).

## Ключевые пункты

- `new BankAccount()` — новый heap object с собственным state; class definition одна на все instances.
- `var b = a` — **alias**; мутация через `b` видна через `a`.
- **`null`** — reference не указывает на object; dereference → `NullReferenceException`.
- Instance methods: implicit **`this`** — тот object, на котором вызван метод.
- Default **`==`** для class — **reference equality** (один object или нет).
- Два `new` с одинаковыми field values — **два разных object**.

## Самые важные примеры

```csharp
var a = new BankAccount { Balance = 50m };
var b = a; b.Deposit(25m);     // a.Balance == 75
var c1 = new BankAccount(); var c2 = new BankAccount();
c1 == c2;                       // false — разные instances
BankAccount? x = null;          // OK; x.Owner — crash
```

## «Под капотом» в одном абзаце

`new` аллоцирует heap object: header + method table pointer + field layout. Metadata class в assembly shared; field values per instance. GC roots через stack/static refs; aliases держат один object alive. Reference assignment — pointer copy без allocation.

## Частые ошибки

- Ожидать deep copy при `copy = original`.
- Путать equal **values** и equal **identity**.
- Забывать `null` на границах API.

## Мнемоника

> **Class — чертёж; `new` — дом; переменная — адрес на дом, не копия дома.**
