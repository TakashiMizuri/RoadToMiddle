## Answers

### 1. [Recall]
**Answer:** **INNER JOIN** — только matching pairs.
**Why:** Unmatched outer/inner excluded.

### 2. [Recall]
**Answer:** **`IEnumerable<TInner>`** (group matches), не single element.
**Why:** GroupJoin name and signature.

### 3. [Understanding]
**Answer:** **`Join`** — **нет**; **`GroupJoin`** — **да** (empty group).
**Why:** Inner vs group semantics.

### 4. [Understanding]
**Answer:** Эмулирует **LEFT JOIN** flat row когда inner group пуст — one row с null inner.
**Why:** DefaultIfEmpty yields one null element for empty sequence.

### 5. [Application]
**Answer:** **`GroupJoin` + SelectMany + DefaultIfEmpty`** или query **`join into` + DefaultIfEmpty**.
**Why:** Left outer flat pattern.

### 6. [Application]
**Answer:** **2** строки (по одной на каждый matching order).
**Why:** Join emits Cartesian per matching pair, not collapse.

### 7. [Analysis]
**Answer:** **`GroupJoin`** — **`into og`** binds grouped sequence.
**Why:** Query syntax desugaring.

### 8. [Understanding]
**Answer:** Floating point equality unstable — matches missed or spurious.
**Why:** Use integer/string keys, not computed floats.

### 9. [Analysis]
**Answer:** **`IQueryable`** — SQL на server at materialization; **`Enumerable`** — in-memory hash join in process.
**Why:** Provider vs LINQ to Objects.

### 10. [Understanding]
**Answer:** **Нет** — property names must match for anonymous type join in query syntax.
**Why:** Different anonymous types.

### 11. [Application]
**Answer:** **Нет** — inner join drops orders without matching customer.
**Why:** INNER semantics.

### 12. [Recall]
**Answer:** Custom key equality (case-insensitive string, culture-specific).
**Why:** Default comparer wrong for domain rules.
