# yield return — краткая выжимка

> `1.3.20` · ~8 мин на повторение

## Главная идея

**`yield return`** превращает method в lazy **iterator**: элементы производятся по **`MoveNext()`**, а компилятор генерирует **state machine** с сохранением locals между паузами.

## Ключевые пункты

- Вызов **`IEnumerable`** method **не** запускает loop — только **`foreach`**/`MoveNext`.
- **`yield break`** — досрочный конец; **`finally`**/`Dispose` при завершении enumeration.
- Каждый **`foreach`** обычно создаёт **новый** enumerator.
- **`ToList()`/`Count()`** — eager, материализация.
- **`IAsyncEnumerable`** — async streaming (`await foreach`).
- Side effects и resources привязаны к enumeration lifetime.

## Самые важные примеры

```csharp
IEnumerable<int> Range(int start, int count)
{
    for (int i = 0; i < count; i++)
        yield return start + i;
}

// Lazy file read — файл открывается на first MoveNext
IEnumerable<string> Lines(string path) { using var r = new StreamReader(path); ... }
```

## «Под капотом» в одном абзаце

Compiler emit'ит class с **`_state`**, **`MoveNext()`** как switch по **`yield`** points, lifted locals в fields. Между элементами call stack method не сохраняется — только state object.

## Частые ошибки

- Double enumeration DB/file pipeline.
- Public API **`IEnumerable`** без указания single-use.
- Ожидание выполнения до **`foreach`**.

## Мнемоника

> **Call builds recipe; foreach cooks.**
