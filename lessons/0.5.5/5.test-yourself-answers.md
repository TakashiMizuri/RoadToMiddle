## Answers

### 1. [Recall]
**Answer:** `href` задаёт URL или fragment destination, куда браузер navigates при activation ссылки.
**Why:** Без `href` элемент не является полноценной навигационной ссылкой.

### 2. [Recall]
**Answer:** `rel="noopener noreferrer"` — `noopener` блокирует доступ новой страницы к `window.opener`; `noreferrer` не отправляет Referer.
**Why:** Защита от reverse tabnabbing и утечки URL источника.

### 3. [Recall]
**Answer:** Root-relative начинается с `/` и resolve от корня origin; relative — от directory текущего документа.
**Why:** `/dashboard` одинаков с любой глубины; `about.html` зависит от текущего path.

### 4. [Understanding]
**Answer:** Новая вкладка может через `window.opener` изменить `location` исходной вкладки (например, на phishing).
**Why:** Opener — живая ссылка на исходное окно; noopener обнуляет её.

### 5. [Understanding]
**Answer:** Screen reader часто озвучивает список ссылок; «Подробнее» не сообщает destination.
**Why:** Link text — primary accessible name; контекст абзаца может не читаться в том же списке.

### 6. [Understanding]
**Answer:** Когда `#pricing` на **той же** загруженной странице и браузер только scrolls к элементу с `id="pricing"`.
**Why:** Same-document navigation не требует нового HTTP fetch (если id существует).

### 7. [Application]
**Answer:** `href="/dashboard"`
**Why:** Root-relative path не зависит от `/projects/42/edit`.

### 8. [Application]
**Answer:** Пример: `<a href="/report-2025.pdf">Download 2025 annual report (PDF)</a>`
**Why:** Self-describing label + hint формата.

### 9. [Application]
**Answer:** `aria-current="page"` на `<a href="/pricing">` (или соответствующем path).
**Why:** Стандартный способ обозначить current page в nav для AT.

### 10. [Analysis]
**Answer:** (1) Semantics: это action, не navigation — лучше `<button>`. (2) Без JS ссылка ведёт на `/delete`; keyboard/ no-JS UX сломан.
**Common mistake:** `return false` маскирует, но не заменяет правильный элемент.

### 11. [Analysis]
**Answer:** Не согласен полностью. Implicit noopener в modern browsers — defense in depth, но explicit `rel` документирует intent, защищает старые UA и проходит linters/CMS review.
**Why:** Production markup должен быть correct без reliance на конкретную версию браузера.

### 12. [Analysis]
**Answer:** Не всегда. Footer often повторяет legal/utility links intentionally; проблема — если дублирование бессмысленное или ломает outline/landmarks.
**Why:** Контекст footer vs primary nav разный; дубли legal links — норма.

## Model answers for practical sections

```html
<header>
  <nav aria-label="Primary">
    <ul>
      <li><a href="/" aria-current="page">Home</a></li>
      <li><a href="/pricing">Pricing</a></li>
      <li><a href="/contact">Contact</a></li>
    </ul>
  </nav>
  <a
    href="https://example.com/docs"
    target="_blank"
    rel="noopener noreferrer"
  >
    API docs
  </a>
</header>
```
