# Func, Action, Predicate — встроенные delegates

> Roadmap: `1.3.4` · Node: `1.3` — C# продвинутые темы · Depth: практика

## Learning Objectives

После урока ты сможешь:

- Выбирать **`Action`**, **`Func`**, **`Predicate<T>`** вместо custom delegates.
- Читать signatures до **`Func<T1,...,TResult>`** (return — последний type param).
- Использовать в **LINQ**, **Task.Run**, callback parameters.
- Применять **variance** на `Func`/`Action` (`1.3.2`).
- Избегать **`async void Action`**, путаницы **`Predicate`** vs **`Func<T,bool>`**.

---

## Why This Matters

**`Action<string>`** — lingua franca callbacks. LINQ **`Where`** → **`Func<T,bool>`**, **`Task.Run`** → **`Action`**. Меньше ceremony, быстрее читаешь BCL API. **`Predicate<T>`** — legacy alias для **`Func<T,bool>`** в `List.FindAll`.

---

## Core Concepts

### Action — void

```csharp
Action greet = () => Console.WriteLine("Hi");
Action<string> log = msg => Console.WriteLine(msg);
```

Side effects, no return.

### Func — return value

```csharp
Func<int, int, int> add = (a, b) => a + b;
```

Last type parameter — **return type**. **`Func<T, TResult>`** — один input.

### Predicate&lt;T&gt;

```csharp
Predicate<string> p = s => s.Length > 0;
names.FindAll(p);
```

≡ **`Func<T, bool>`**. Новый code — prefer **`Func`** для LINQ consistency.

### Выбор типа

| Need | Type |
|---|---|
| Side effect | `Action<...>` |
| Compute | `Func<..., TResult>` |
| Filter test | `Func<T,bool>` / `Predicate<T>` |

Custom named delegate — когда domain meaning важнее в public API.

### BCL patterns

```csharp
var adults = users.Where(u => u.Age >= 18);
await Task.Run(() => Work());
ProcessItems(items, item => Log(item));
```

## Under the Hood

**Action/Func** — generic delegates в `System`, те же multicast rules (`1.3.3`). Lambdas и method groups → delegate instances.

## Examples

### DI factory

```csharp
public ReportService(Func<DateTime, IReport> factory) { }
```

### Variance

```csharp
Func<object, string> f = o => o.ToString() ?? "";
Func<string, string> g = f;
```

---

## Common Mistakes

- **`async void`** с `Action` — use **`Func<Task>`**.
- Wrong arity: `Func<string, int, bool>` = `(string,int)->bool`.
- Public API overload с `Func` без документации.
- Null invoke без `?.`.

---

## Quick Reference

| Delegate | Shape |
|---|---|
| `Action<T>` | `void(T)` |
| `Func<T,TResult>` | `TResult(T)` |
| `Predicate<T>` | `bool(T)` |

Max 16 type parameters.

---

## Key Takeaways

- **Action** void, **Func** return (last param), **Predicate** = **Func<T,bool>**.
- LINQ/Task/callbacks daily.
- Balance convenience vs named types in public API.
