# Изображения: `alt`, `figure`, `picture` — краткая выжимка

> `0.5.6` · ~8 мин

## Главная идея

`<img alt="...">` — text alternative для AT и fallback при ошибке load; `alt=""` = decorative. `figure`/`figcaption` и `picture`/`source` решают подписи и responsive без JavaScript.

## Ключевые пункты

- **Informative** → описательный `alt`; **decorative** → `alt=""`.
- **`width`/`height`** резервируют место → меньше CLS.
- **`loading="lazy"`** — ниже fold; **не** для LCP hero.
- **`<picture>`**: `source` (media, type, srcset) + fallback `<img>` с `alt`.
- **`sizes` + `srcset`** — браузер выбирает файл по render width.
- Content не только в CSS `background-image` — нет `alt`.
- Thumbnail в `<a>` (`0.5.5`) — meaningful `alt` на `img`.

## Примеры

```html
<img src="chart.png" alt="Q1 revenue up 18% vs Q4" width="640" height="360" />
```

```html
<picture>
  <source type="image/webp" srcset="hero.webp" />
  <img src="hero.jpg" alt="Dashboard overview" width="800" height="500" />
</picture>
```

## Частые ошибки

- Пропущенный `alt` или `alt="photo.jpg"`.
- Lazy на hero (LCP).
- Keyword stuffing в `alt`.

## Правило большого пальца

**Сначала alt-решение**, потом `srcset`/format optimization.
