# Exercises: HttpClient Pitfalls

> Complete after lectures and test. Lab exercises — localhost only.

## Easy

### E1. Shared vs per-call (observation)
**Goal:** Compare exception rate pattern conceptually or in small loop.
**Steps:** 100 sequential calls — shared HttpClient vs new per call to local server; note errors/time.
**Success criteria:** Short write-up which pattern degraded first and why.

### E2. netstat TIME_WAIT
**Goal:** Correlate connections with pattern.
**Steps:** During spike, run `netstat` or `ss` TIME_WAIT count (Windows/Linux).
**Success criteria:** One sentence linking TIME_WAIT to client lifetime.

## Medium

### M1. Configure SocketsHttpHandler
**Goal:** Register named client with PooledConnectionLifetime and MaxConnectionsPerServer.
**Steps:** ASP.NET Program.cs AddHttpClient + ConfigurePrimaryHttpMessageHandler.
**Success criteria:** Can explain each property to a peer.

### M2. Fix anti-pattern service
**Goal:** Refactor class that did `new HttpClient()` in method to factory injection.
**Steps:** Create interface + IHttpClientFactory ctor; update tests.
**Success criteria:** Single handler pool per logical service name.

## Hard (stretch)

### H1. Incident write-up
**Goal:** Document fictional postmortem: deploy + load spike + socket errors.
**Steps:** Timeline, root cause (per-request client + unbounded parallel), fix (factory + ForEachAsync cap + Polly).
**Success criteria:** References lessons 1.4.25–1.4.28 coherently.
