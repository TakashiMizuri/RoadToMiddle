# Merge — краткая выжимка

> `0.2.2` · ~10 мин на повторение

## Главная идея

**Merge** объединяет divergent branch histories. Если tip `main` — прямой **ancestor** tip `feature`, Git делает **fast-forward** (двигает ref, без нового commit). Если обе ветки ушли от **merge base** — **three-way merge commit** с двумя parents. **Squash merge** кладёт net diff feature одним **single-parent commit** на `main`.

## Ключевые пункты

- **Merge base** — nearest common ancestor двух tips.
- **Fast-forward:** `main` → тот же hash, что `feature`; linear history, все commit'ы feature на main.
- **Merge commit (`--no-ff` или true merge):** diamond graph, два parents, явная точка integration.
- **Squash:** `git merge --squash` + `git commit`; commit'ы feature **не** в ancestry `main`.
- Во время merge: `MERGE_HEAD`, conflicts → index stages 1/2/3 (`0.2.5`).
- GitHub: Create merge commit | Squash and merge | Rebase and merge.

## Самые важные примеры

**FF:**
```bash
git switch main && git merge feature   # main просто переезжает на tip feature
```

**Squash:**
```bash
git merge --squash feature
git commit -m "feat: login (#42)"
```

**Merge base:**
```bash
git merge-base main feature
```

## «Под капотом» в одном абзаце

Three-way merge сравнивает trees merge base B и tips M, F. Для каждого path Git решает: взять одну сторону, объединить одинаковые изменения или пометить conflict. Clean result → новый commit object с `parent M` и `parent F`, ref current branch → новый hash. FF — только перезапись `refs/heads/main` hash'ем F без нового object. Squash вычисляет итоговый tree diff vs HEAD, staged в index; один commit с одним parent — feature chain не становится ancestors.

## Частые ошибки

- Merge устаревшего `main` → giant conflicts.
- Путать FF (нет merge commit) с «ничего не произошло».
- Squash и продолжать push в ту же feature branch без rebase.
- `--no-ff` на каждый мелкий merge без policy.

## Мнемоника

> **FF** = «main догнал feature»; **merge commit** = «две линии встретились в X»; **squash** = «вся feature одной пачкой на main».
