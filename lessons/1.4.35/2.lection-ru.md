# IAsyncDisposable — async cleanup

> Roadmap: `1.4.35` · Node: `1.4` — C# async/await · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Различать **`IDisposable`**, **`IAsyncDisposable`** и когда что применять.
- Использовать **`await using`** и async dispose в scoped/background коде.
- Корректно закрывать **`DbContext`**, streams, обёртки с I/O teardown.
- Избегать **sync-over-async** в `Dispose()`.
- Связать с **`CreateAsyncScope()`** (`1.4.34`).

---

## Why This Matters

`DbContext`, соединения, streams, gRPC могут требовать **async flush/close**. Sync **`Dispose`** блокирует или **deadlock** при ожидании async work. **`IAsyncDisposable`** и **`await using`** встраивают cleanup в async path. EF и DI в ASP.NET Core 3+ поддерживают **`CreateAsyncScope()`** — Middle обязан dispose на **том же async пути**, что и использование.

---

## Core Concepts

### IDisposable vs IAsyncDisposable

**`Dispose()`** — синхронно, быстро, на текущем потоке.

**`DisposeAsync()`** — **`ValueTask`**, можно **await** I/O при закрытии.

Тип может реализовать **оба**. Был **await** на ресурсе → предпочитай **`IAsyncDisposable`**.

### await using

```csharp
await using var stream = new FileStream(..., useAsync: true);
await ProcessAsync(stream);
// DisposeAsync в finally scope
```

Для только **`IDisposable`** `await using` работает, но без реального async benefit.

### DbContext и scope

```csharp
await using var scope = _scopeFactory.CreateAsyncScope();
await db.SaveChangesAsync(ct);
// DisposeAsync scope → DbContext
```

**`CreateAsyncScope()`** предпочтительнее в async hosts.

### Не блокируй DisposeAsync в Dispose

```csharp
DisposeAsync().GetAwaiter().GetResult(); // deadlock risk
```

Предпочитай **`await using`** для callers.

---

## Under the Hood

`await using` встраивает dispose в **async state machine**. **`DisposeAsync`** → **`ValueTask`** (часто без аллокации). DI dispose services в **обратном** порядке. Finalizers — не замена явному `await using`.

## Common Mistakes & Anti-patterns

`using` вместо **`await using`** на IAsyncDisposable.

Dispose **singleton** per request.

**`_ = DisposeAsync()`** — unobserved faults.

Использование `DbContext` после end scope.

---

## Quick Reference

- **`await using`** → `DisposeAsync`.
- **`CreateAsyncScope()`** в async workers.
- Не **`.Result`** на `DisposeAsync` в sync Dispose.
- Scope boundary = dispose scoped services.
- Idempotent dispose.

---

## Key Takeaways

- Async I/O teardown → **`IAsyncDisposable`**.
- **`await using`** = `using` для async.
- **`CreateAsyncScope()`** с queue consumer.
- Sync-over-async в Dispose — deadlock.
- Await dispose, не fire-and-forget.
