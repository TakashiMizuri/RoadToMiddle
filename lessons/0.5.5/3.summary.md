# Ссылки: `a`, `href`, `target`, `rel noopener` — краткая выжимка

> `0.5.5` · ~8 мин на повторение

## Главная идея (1–2 предложения)

Элемент `<a href="...">` — стандартный способ навигации в HTML; `target="_blank"` без `rel="noopener noreferrer"` создаёт дыру в безопасности (reverse tabnabbing), а неописательный link text ломает accessibility.

## Ключевые пункты

- **`href`** — destination: absolute, relative, root-relative (`/path`), fragment (`#id`), `mailto:`, `tel:`.
- **`target="_blank"`** — новая вкладка; всегда добавляй **`rel="noopener noreferrer"`**.
- Текст ссылки должен быть понятен **в списке ссылок без контекста** — не «Click here».
- Primary nav — в `<nav>` (`0.5.2`); **`aria-current="page"`** на активном пункте.
- Fragment links (`href="#section"`) требуют **`id`** на целевой секции.
- Для действий без навигации — **`<button>`**, не `<a href="#">`.

## Самые важные примеры

```html
<a href="/docs" aria-current="page">Documentation</a>
```

Root-relative путь работает с любой глубины URL.

```html
<a href="https://mdn.example" target="_blank" rel="noopener noreferrer">
  MDN reference
</a>
```

Безопасное открытие внешнего ресурса в новой вкладке.

## Частые ошибки

- `_blank` без `noopener` — уязвимость через `window.opener`.
- Вложенный `<button>` в `<a>` — invalid HTML.
- Ссылка только с иконкой без accessible name.
- `href="#"` + `preventDefault` вместо `<button>`.

## Мнемоника / правило большого пальца

**Blank → noopener.** Если видишь `target="_blank"`, проверь `rel` в той же строке.
