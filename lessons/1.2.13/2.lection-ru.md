# Records: positional, with-expressions, equality

> Roadmap: `1.2.13` · Node: `1.2` — C# ООП · Depth: практика

## Learning Objectives

После этой лекции ты сможешь:

- Объявлять **positional records** и объяснять, что синтезирует компилятор.
- Использовать **`with`-expressions** для non-destructive updates (иммутабельные изменения).
- Предсказывать **value equality** для records vs classes.
- Выбирать **record class** vs **record struct** vs обычный **class** для DTO и domain values.

- Избегать ловушек: mutable reference types внутри record, наследование records, копирование больших struct.

## Why This Matters

ASP.NET Core API постоянно передаёт данные: request DTO, response models, MediatR commands, EF projections. Классы с boilerplate конструкторами, `Equals`, `GetHashCode` и `==` тратят время и дают баги — два экземпляра с одинаковыми значениями сравниваются как неравные из-за reference identity.

**Records** (C# 9+) — **reference types** (по умолчанию), оптимизированные под **immutable data с value semantics**. Одна строка:

```csharp
public record CreateProductCommand(string Name, decimal Price, int CategoryId);
```

даёт constructor, read-only properties, value equality, `ToString` и `with` copies. Records встречаются в minimal APIs, CQRS handlers, `System.Text.Json` deserialization. Middle-разработчик знает, когда record идеален, когда нужен mutable class, и когда подходит `record struct` (C# 10).

## Core Concepts

### Что такое record?

**Record** — reference type (если не `record struct`), чья главная роль — **перенос данных** с **structural equality**: два record равны, если **data members** равны по значению, а не по ссылке на один объект.

```csharp
public record Point(int X, int Y);

var a = new Point(1, 2);
var b = new Point(1, 2);
Console.WriteLine(a == b);           // True — value equality
Console.WriteLine(ReferenceEquals(a, b)); // False
```

У обычного **class** без overrides `a == b` было бы **false**. Records синтезируют `Equals`, `GetHashCode`, `==`/`!=`.

### Positional records

**Positional syntax** объявляет параметры primary constructor, которые становятся **init-only public properties** (то же имя, PascalCase):

```csharp
public record CustomerDto(int Id, string Name, string Email);
```

Компилятор генерирует примерно:

- Constructor `(int Id, string Name, string Email)`
- Properties `public int Id { get; init; }` и т.д.
- `Equals`/`GetHashCode` по всем positional members
- `ToString()` → `CustomerDto { Id = 1, Name = Ann, ... }`
- `Deconstruct` для `(int id, string name, string email) = customer;`

Можно добавлять члены в body:

```csharp
public record OrderLine(int ProductId, int Quantity)
{
    public decimal UnitPrice { get; init; }
    public decimal LineTotal => UnitPrice * Quantity;
}
```

`UnitPrice` участвует в equality как instance property.

### with-expressions: non-destructive copy

Records favor **immutability**: вместо `dto.Name = "x"` создаёшь **копию с изменениями**:

```csharp
var original = new CustomerDto(1, "Ann", "ann@x.com");
var updated = original with { Email = "ann@new.com" };
```

`with` клонирует record (shallow copy) и применяет overrides. `original` не меняется. Под капотом — `<Clone>$()` и object initializer.

**Shallow copy caveat:** если property — `List<T>` или array, `with` копирует **ссылку** — оба record делят один list:

```csharp
var r1 = new Container(new List<int> { 1 });
var r2 = r1 with { };
r2.Items.Add(2);       // r1 тоже видит изменение
```

Для immutable records — `IReadOnlyList<T>` или immutable collections.

### record class vs class с ручным equality

| Feature | `record` | Regular `class` |
|---------|----------|-----------------|
| Value equality | Synthesized | Manual override |
| `with` | Да | Нет (без ручной реализации) |
| Primary constructor | Positional syntax | C# 12+ primary constructors — другая семантика |
| Inheritance | `record Derived : Base` с правилами | Полный OOP |
| Mutability | Обычно init-only | Любая |

**Records** — DTOs, commands, events, value-like aggregates. **Classes** — entities с identity и mutating behavior (EF tracked entities часто classes).

### record struct (C# 10+)

```csharp
public readonly record struct Pixel(byte R, byte G, byte B);
```

**Value type** с record equality и `with`. На stack когда не boxed. **`readonly record struct`** запрещает mutating struct после construction.

Trade-off из `1.2.14`: копирование больших struct дорого — держи record struct маленькими.

### Наследование records

Records наследуют records; **sealed** часто на leaf DTOs.

```csharp
public abstract record Shape(string Name);
public sealed record Circle(string Name, double Radius) : Shape(Name);
```

Equality включает base properties. Не смешивай легкомысленно class hierarchies с records.

### Init и required members

Records хорошо сочетаются с **`init`** (`1.2.2`):

```csharp
public record Product(string Name)
{
    public DateTime CreatedAt { get; init; } = DateTime.UtcNow;
}
```

**`required`** (C# 11) гарантирует установку при создании/deserialization:

```csharp
public record ApiRequest
{
    public required string CorrelationId { get; init; }
}
```

## Under the Hood

### Compiler synthesis

Roslyn emit **synthesized private `Clone`** и overrides `Equals`, `GetHashCode`, operators. Equality сравнивает каждое поле через `EqualityComparer<T>.Default`.

Для `record struct`, typed `Equals(T)` избегает boxing.

### Equality и collections

Records — хорошие **`Dictionary` keys** и **`HashSet`** elements: стабильный `GetHashCode` — в отличие от mutable classes, где hash меняется после mutation и ломает collections.

### JSON deserialization

`System.Text.Json` десериализует positional records через constructor parameters, matching JSON property names.

```json
{ "name": "Widget", "price": 9.99, "categoryId": 3 }
```

→ `CreateProductCommand("Widget", 9.99m, 3)`.

## Examples

### Пример 1: API Command

```csharp
public record UpdateStockCommand(int ProductId, int Delta, string Reason);
// cmd immutable — без defensive copy при передаче в handler
```

### Пример 2: Immutable domain update

```csharp
public record Cart(IReadOnlyList<CartLine> Lines)
{
    public Cart AddLine(CartLine line) =>
        this with { Lines = Lines.Append(line).ToList() };
}
```

### Пример 3: with и nested mutation trap

```csharp
public record Settings(Dictionary<string, string> Values);
var s2 = s1 with { };
s2.Values["k"] = "changed"; // s1 тоже changed — shared reference
```

**Fix:** `ImmutableDictionary` или копия dictionary в constructor.

## Common Mistakes & Anti-patterns

**Mutable collections внутри immutable record** — иллюзия immutability.

**Records для EF entities** с navigation properties — EF ожидает mutable classes.

**Огромные positional records** (20+ parameters) — читаемость страдает.

**Deep equality не автоматическая** — reference-type properties сравниваются shallowly.

**Records для behavior-heavy types** — логика в services, не в 500-line record.

## Production & Real-World Notes

Стандарт: **`public record`** для commands/queries. Добавление optional init property — backward compatible; новый positional parameter в primary constructor — **breaking**.

**OpenAPI** мапит records как classes. **Validation:** FluentValidation / data annotations.

**Performance:** records — reference types, heap как classes; выгода в correctness и меньшем boilerplate; stack — `record struct`.

## Comparison / Trade-offs

| Type | Когда |
|------|-------|
| `record class` | DTOs, commands, events |
| `record struct` | Маленькие immutable bundles |
| `class` | Mutable entities, ORM |
| `struct` | Performance value types без synthesized equality |

## Key Takeaways

- **Records** — data carriers с **value equality** и **`with`**.
- **Positional syntax** → constructor + init properties + `Deconstruct`.
- **`with`** — **shallow**; осторожно с mutable references.
- Records для **commands/DTOs**; не для **mutable EF entities**.
- **`record struct`** — маленькие stack-friendly values.
- Equality по **всем instance properties** (если не кастомизировано).
