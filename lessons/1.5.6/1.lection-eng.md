# GC Roots, Reachability, and the Mark Phase

> Roadmap: `1.5.6` · Node: `1.5` — .NET Runtime and Memory · Depth: **deep**

## Learning Objectives

After this lesson you will be able to:

- Explain **reachability** — an object is live if reachable from any root via reference chains.
- Describe the **mark phase** of mark-sweep-compact GC.
- List common root categories: **stack slots**, **static fields**, **GC handles**, **finalizer queue**.

## Why This Matters

In `1.5.5` objects live on the heap until the GC reclaims them. **When** is an object dead? Not when you last used it — only when **no reference path exists from any root**. Misunderstanding causes **memory leaks** ( accidental roots via events, static caches) and false confidence that "I set my variable to null so it's gone" (other references may exist).

The **mark phase** is the graph traversal that defines liveness. Server GC pauses, LOH fragmentation (`1.5.8`), and finalizer delays all start here. Middle developers use **dotnet-gcdump**, fix event handler leaks, and know why **WeakReference** does not prevent collection.

## Core Concepts

### What Makes an Object Live?

The GC collects objects that are **unreachable** — no chain of references from a **GC root** to the object. If reachable, the object (and everything it references) survives this collection (subject to generation policy).

**Roots** are not "important objects" chosen by you — they are **starting points** the runtime scans:

| Root source | Example |
|-------------|---------|
| Thread stacks | Local `Customer c`, parameters in active frames (`1.5.4`) |
| Static fields | `static Cache _items` |
| GC handles | `GCHandle.Alloc(obj, Normal/Pinned)` |
| Finalizer queue | Objects with pending `Finalize` not yet run |
| COM interop / P/Invoke | JNI-style external refs (edge cases) |

After marking from roots, anything unmarked is **garbage** (for this collection).

### Reachability and Reference Graph

```
Roots → A → B → C
              ↘ D

If reference B→C removed and no other path to C:
  C and (if nothing else refs D) D become unreachable
```

**Cycles** alone don't prevent collection — if `{A,B}` cycle has no root path, entire cycle is collected. Classic **reference counting** fails on cycles; .NET tracing GC handles them.

Setting a local to `null` removes **one** link; static or event references keep objects alive.

### The Mark Phase

**Mark-sweep-compact** (generational variant in `1.5.7`) mark phase:

1. **Suspend** or coordinate with mutator threads (server GC concurrent phases differ).
2. **Scan roots** — for each pointer-like value, if it points to heap object, **mark** object.
3. **Trace** — follow marked object's reference fields (from GC descriptor in MethodTable `1.5.5`), mark targets, repeat (work-list or recursive).
4. Unmarked objects in collected generation → **sweep** (reclaim) or **compact** (move survivors, update refs).

Mark uses **bitmap or mark bit** in object header or side table per generation.

### Write Barriers (Preview for 1.5.7)

When **generational GC** collects Gen0 only, older objects referencing new ones must be tracked — **card table** + **write barriers** record cross-generation stores so Gen0 roots include old→young edges. Mark phase respects these remembered sets.

### Finalizers and Reachability

Object with finalizer: when unreachable, not immediately reclaimed — **finalization queue** roots it until `Finalize` runs, then another GC needed to reclaim memory. **`IDisposable` pattern** (`1.5.10`) avoids relying on this delayed path.

## Under the Hood

### Exact vs Conservative Scanning

.NET uses **exact scanning** for managed stacks and object fields — JIT maps which bits are references. **Conservative** stacks (some interop scenarios) treat ambiguous bit patterns as possible refs — rare retention.

### STW and Concurrent Mark

**Stop-the-world (STW)** — all managed threads pause for some GC phases. **Background GC** (server mode) allows mutator work during concurrent mark/sweep with **write barriers** keeping consistency.

### Weak References

`WeakReference` / `WeakReference<T>` — special handle; target **not counted as strong root**. If only weak refs remain, object can be collected; weak ref zeroed when collected.

## Examples

### Event Handler Leak

```csharp
class Publisher {
    public event EventHandler? Updated;
}
class Subscriber {
    public Subscriber(Publisher p) { p.Updated += OnUpdate; }
    void OnUpdate(object? s, EventArgs e) { }
}
// Subscriber unreachable but Publisher alive → delegate holds Subscriber → leak
// Fix: unsubscribe or WeakEvent pattern
```

Publisher holds delegate → target (`Subscriber`) — **reachable from static/singleton Publisher root**.

### Null Doesn't Help If Shared

```csharp
var a = new Customer();
var b = a;
a = null; // b still roots Customer
```

## Common Mistakes & Anti-patterns

**"GC will collect when variable out of scope"** — scope ends local root; **other roots** may exist.

**Static collections without eviction** — unbounded growth, all items strongly rooted.

**Forgetting GCHandle.Free** — pinned root until AppDomain unload.

**Assuming circular refs leak** — tracing GC collects unreachable cycles.

## Production & Real-World Notes

**Memory dumps** in production — `dotnet-dump analyze`, `dumpheap -stat`, `gcroot` SOS command shows **why** object still live.

**Caching** — size limits, LRU, `IMemoryCache` eviction policies; weak refs for optional caches only when stale OK.

**Long-lived services** — subscription leaks show as Gen2 growth over days.

## Comparison / Trade-offs

| Mechanism | Prevents GC? | Use case |
|-----------|--------------|----------|
| Strong reference | Yes | Normal object graph |
| Static field | Yes until cleared | Singletons, caches |
| GCHandle.Normal | Yes until Free | Interop callbacks |
| GCHandle.Pinned | Yes + pin | Native API buffers |
| WeakReference | No (alone) | Secondary caches |

## Key Takeaways

- **Liveness = reachable from roots**, not time since last use.
- **Stacks** (`1.5.4`), **statics**, **handles** are primary root sources.
- **Mark phase** traverses object graph from roots.
- **Cycles** without external roots are collected.
- **Leaks** = accidental strong roots — events, statics, missed unsubscribe.
