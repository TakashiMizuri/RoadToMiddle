## Answers

### 1. [Recall]
**Answer:** Выбор **реализации method** по **actual runtime type** объекта при вызове через base/interface reference; реализуется через vtable + **`callvirt`**.
**Why:** Static type не определяет target для virtual chain.

### 2. [Recall]
**Answer:** **`virtual`** или **`abstract`** (или **`override`** дальше по chain).
**Why:** Без этого только **`new` hide**, не polymorphism.

### 3. [Understanding]
**Answer:** Compile-time — тип переменной для checker и overload resolution. Runtime — фактический class object на heap (**`GetType()`**).
**Why:** Virtual calls следуют runtime; hidden/static — compile-time.

### 4. [Understanding]
**Answer:** Instance call требует объекта; **`callvirt`** проверяет **`this`** до indirect jump — стандартное поведение CLR для virtual instance methods.
**Why:** Null reference fail-fast.

### 5. [Understanding]
**Answer:** Объекты derived type должны **заменять** base без изменения корректности программы, использующей только контракт base.
**Common mistake:** Путать с syntactic **`override`**.

### 6. [Application]
**Answer:** **Runtime type** — **`Dog`**, не **`Animal`**.
**Why:** Virtual dispatch.

### 7. [Application]
**Answer:** Base: **`public virtual decimal Calculate()`** (или abstract); Derived: **`public override decimal Calculate()`**.
**Why:** Polymorphic chain.

### 8. [Analysis]
**Answer:** **`Base.M()`** → `1`. **`Der.M()`** → `2`. **`b.M()`** если **`M`** static на **`Base`** — **`Base.M()`** через **`Base`**, static не inherited как instance; вызов **`b.M()`** — compile error unless **`M`** instance. Typично: только **`Base.M()`** и **`Der.M()`** — 1 и 2. Static не polymorphic.
**Why:** Static bound to declaring type.

### 9. [Analysis]
**Answer:** **Нет** — abstract class нельзя instantiate; CS0144.
**Why:** Abstract = incomplete implementation.

### 10. [Understanding]
**Answer:** Оба — **runtime resolution** реализации при call через abstraction reference (class vtable vs interface map).
**Why:** Разная metadata, похожая semantics для caller.

### 11. [Application]
**Answer:** **Derived `Step`** — **`Run`** non-virtual, но **`Step()`** inside — virtual call on **`this`** → runtime type derived.
**Why:** **`this`** в **`Run()`** — derived object.

### 12. [Understanding]
**Answer:** Убрать indirect call → direct call/inlining → performance.
**Why:** Sealed closed hierarchy.

### 13. [Analysis]
**Answer:** **Legal** (C# 9+ covariant returns). **`a`** compile-time **`Animal`**, runtime type экземпляра — **`Dog`** (если factory derived).
**Why:** Return type covariance не меняет dispatch **`Create`** itself.

### 14. [Application]
**Answer:** **`virtual Speak()`** (или interface) на каждом типе; **`foreach`** calls **`Speak()`** без switch.
**Why:** Open/Closed — новые animals без изменения loop.
