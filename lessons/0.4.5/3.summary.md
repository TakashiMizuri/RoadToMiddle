# DNS — краткая выжимка

> `0.4.5` · ~10 мин

## Главная идея

**DNS** переводит hostname (`0.3.2`) в IP (`0.4.2`) **до** TCP/HTTP/TLS. **Recursive resolver** кэширует ответы на **TTL** seconds.

## Ключевые пункты

- **A** — IPv4; **AAAA** — IPv6; **CNAME** — alias (не на classic apex).
- Path: stub → recursive (cache?) → authoritative.
- Low TTL перед migration; stale cache → partial outage.
- UDP :53; TCP если truncated/large (`0.4.4`).
- TLS SNI + HTTP Host нуждаются в correct name (`0.3.8`).

## «Под капотом»

Resolver walk: root → TLD → zone NS → answer. CNAME chain до A/AAAA. Cache: browser, OS, ISP 8.8.8.8. Split-horizon — разные answers internal/public.

## Частые ошибки

Hard-code IP; забыть lower TTL; CNAME на apex; нет AAAA.
