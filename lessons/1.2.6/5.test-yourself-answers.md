## Answers

### 1. [Recall]
**Answer:** Ровно **один** direct base class (кроме неявного **`object`**).
**Why:** C# не поддерживает multiple inheritance для classes; interfaces — отдельно (`1.2.8`).

### 2. [Recall]
**Answer:** Только members, помеченные **`virtual`**, **`abstract`**, или уже **`override`** в chain.
**Why:** Non-virtual members override нельзя — только hide через **`new`**.

### 3. [Understanding]
**Answer:** **`override`** — **runtime dispatch**: вызывается реализация **actual object type**. **`new`** — **static dispatch**: вызывается member, видимый по **compile-time type** reference (часто base).
**Common mistake:** Ожидать polymorphism при **`new`**.

### 4. [Understanding]
**Answer:** Документирует **намеренное hiding**; без **`new`** — warning CS0108. Явность помогает на review и при refactoring.
**Why:** Implicit hide легко принять за bug.

### 5. [Understanding]
**Answer:** **`sealed class`** — нельзя наследовать тип. **`sealed override`** — нельзя дальше override этот method в derived classes.
**Why:** Разные уровни «заморозки» — type vs single virtual slot.

### 6. [Application]
**Answer:** `public Dog(...) : base(name) { ... }` — explicit call **`protected`** base constructor.
**Why:** Compiler не найдёт parameterless base ctor.

### 7. [Application]
**Answer:** **`public override string ToString()`** с **`return ... + base.ToString();`** (или наоборот по порядку).
**Why:** **`base.ToString()`** вызывает реализацию immediate base, не рекурсию derived.

### 8. [Analysis]
**Answer:** Выведет **`A`**. Reference **`x`** типа **`A`**; **`M`** hidden, не overridden — **`call`**, не **`callvirt`** по derived slot.
**Why:** Compile-time type **`A`** → **`A.M`**.

### 9. [Analysis]
**Answer:** Выведет **`B`**. **`override`** + reference на **`B`** instance → virtual dispatch.
**Why:** Runtime type **`B`** определяет slot.

### 10. [Understanding]
**Answer:** Derived **override** может выполниться до завершения инициализации derived fields — **partially constructed object**, null/default state, нарушение invariants.
**Why:** Fragile base class anti-pattern.

### 11. [Application]
**Answer:** **`public sealed override void Calculate()`** в derived (после **`virtual`** в base).
**Why:** **`sealed override`** блокирует следующий уровень override.

### 12. [Understanding]
**Answer:** **Нет** — compile error CS0509.
**Why:** **`sealed class`** закрывает inheritance.

### 13. [Analysis]
**Answer:** **Compile error** — нет доступного base constructor call.
**Why:** Compiler требует chain к accessible ctor.

### 14. [Application]
**Answer:** **`sealed`** на class: `public sealed class Money`.
**Why:** Запрет subclassing сохраняет invariants value type semantics.
