# Answers: Task vs ValueTask

## Scoring

- 90–100%: отлично · 80–89%: pass · <80%: повтори lection

---

## Answers

### 1. [Recall]
**Answer:** Value type (`struct`).
**Why:** Stack-friendly wrapper; не всегда zero heap (может держать ссылку на Task).

### 2. [Recall]
**Answer:** `.AsTask()`.
**Why:** Создаёт Task, безопасный для multiple await.

### 3. [Understanding]
**Answer:** Может хранить `T` напрямую без создания `Task<T>` на heap.
**Why:** Sync path обходит Task allocation.

### 4. [Understanding]
**Answer:** Struct consumed после первого await; повторное чтение undefined/exception; pooled source может вернуть в pool.
**Why:** ValueTask не гарантирует idempotent await как Task.

### 5. [Understanding]
**Answer:** Когда метод **всегда** делает await и suspend — state machine + Task path как у Task.
**Why:** Выгода только при частом sync completion.

### 6. [Application]
**Answer:** `ValueTask<T>` (при single immediate await, после profiling); иначе Task тоже OK для простоты.
**Why:** Classic cache-hit scenario.

### 7. [Application]
**Answer:** `.AsTask()` на каждом или изменить API на `Task` для composability.
**Why:** WhenAll expects Task.

### 8. [Analysis]
**Answer:** Breaking change для consumers; семантика await differs; test mocks сложнее.
**Why:** Public contract stability.

### 9. [Analysis]
**Answer:** Второй await — invalid operation / corrupted result / exception.
**Why:** Double-consumption ValueTask.

### 10. [Understanding]
**Answer:** Tight loop read; часто sync из buffer; микро-alloc важны.
**Why:** Framework hot path pattern.

### 11. [Application]
**Answer:** `Task` / `Task<IActionResult>` / `Task<T>` — default.
**Why:** Composability, filters, no double-await traps.

### 12. [Understanding]
**Answer:** Pooled async operation; реализуют library authors (`Pipelines`, custom IO).
**Why:** Expert-level perf with reuse.
