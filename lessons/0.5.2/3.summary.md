# Semantic layout — краткая выжимка

> `0.5.2` · ~8 мин на повторение

## Главная идея

HTML5 **landmarks** (`header`, `nav`, `main`, `article`, `section`, `aside`, `footer`) описывают **назначение** регионов, а не внешний вид. Один **`main`** на страницу; навигация в **`nav`**; самодостаточные блоки — **`article`**; тематические главы с заголовком — **`section`**.

## Ключевые пункты

- **`main`** — уникальный primary content; site chrome вне main.
- **`nav`** — блок ссылок для перемещения; несколько nav OK с `aria-label`.
- **`article`** — syndicatable (пост, карточка); **`section`** — часть целого + heading.
- **`aside`** — tangential (sidebar, related); не primary column из-за CSS.
- **`header`/`footer`** — на уровне страницы и внутри article.
- Implicit ARIA landmarks → shortcuts в screen reader.

## Самые важные примеры

```
body
├── header (logo + nav)
├── main
│   └── article
│       ├── header (h1 post)
│       ├── section (h2 chapter)
│       └── footer (tags)
├── aside (related)
└── footer (copyright)
```

## Частые ошибки

- Два `<main>` на одной view.
- `<section>` без заголовка.
- Меню только в `<div class="menu">`.
- Статья целиком в `<header>`.

## Мнемоника

> **MAIN once, NAV links, ARTicle alone, SECtion needs a heading.**
