# LINQ performance — multiple enumeration — краткая выжимка

> `1.3.30` · ~8 мин

## Главная идея

LINQ chains **lazy (cold)** до materialize или immediate operator. Повторная enumeration = **повторное выполнение** pipeline (2× SQL, 2× file I/O).

## Ключевые пункты

- Materialize once: `ToList()` / `ToArray()` / `ToListAsync()`.
- Anti-pattern: `Any()` + `foreach` / `Count()` + `ToList()` на одном cold source.
- `Count()` на IEnumerable — O(n); `List.Count` — O(1).
- `OrderBy`, `GroupBy` — buffer entire sequence.
- EF: N+1 loop vs one `Where(ids.Contains)`.
- API reuse → return `IReadOnlyList<T>`, not `IEnumerable`.

## Примеры

```csharp
var list = query.ToList();
if (list.Count == 0) return;
// reuse list
```

## «Под капотом»

Fused Where+Select — nested MoveNext без List. OrderBy ломает streaming. Closure в deferred Where — value на момент enumerate.

## Частые ошибки

- Double enumeration.
- `Count()` только для проверки empty на cold seq (если потом снова walk — materialize).
- Hidden re-query в foreach по IQueryable inside loop.
