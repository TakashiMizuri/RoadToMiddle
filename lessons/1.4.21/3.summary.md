# TaskScheduler.FromCurrentSynchronizationContext — краткая выжимка

> `1.4.21` · ~6 мин на повторение

## Главная идея

**`SynchronizationContext`** определяет, **на каком thread/context** выполнится continuation после **`await`**. **`TaskScheduler.FromCurrentSynchronizationContext()`** — legacy способ queue на текущий context. **ASP.NET Core** — **`SyncContext == null`**, resume на **thread pool**.

## Ключевые пункты

- UI (WPF/WinForms): capture UI → continuation на UI для touch controls.
- **`ConfigureAwait(false)`** — не capture; library default.
- **UI + `.Result`** на async с UI capture → **deadlock**.
- **ASP.NET Core** — нет request thread affinity; starvation, не classic UI deadlock.
- **`Task.Run`** — всегда pool, игнорирует SyncContext.
- **`async void`** — только event handlers UI с try/catch.

## Пример deadlock

```
UI thread: .Result → blocked
Continuation: needs UI thread → deadlock
```
