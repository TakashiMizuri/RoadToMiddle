# Анатомия URL: scheme, host, port, path, query, fragment

> Roadmap: `0.3.2` · Node: `0.3` — HTTP и Web · Depth: понимание

## Learning Objectives

После этой лекции ты сможешь:

- Разобрать URL на **scheme**, **authority** (host, port), **path**, **query**, **fragment**.
- Объяснить, какие части **client отправляет server** в HTTP request, а какие остаются локально.
- Отличить **absolute** и **relative** URL и где каждый встречается в HTML и API client'ах.
- Безопасно кодировать **query parameters**, включая reserved characters и пробелы.
- Назвать default ports для **http** (80) и **https** (443) и когда port пишут явно.
- Связать design path с **REST resources** (`0.3.10`), не смешивая URL с method.
- Отлаживать типичные ошибки: wrong host, missing encoding, fragment в API calls.

---

## Why This Matters

Каждый HTTP request из `0.3.1` нацелен на адрес — почти всегда **URL**. React: `fetch('/api/products?page=2')` — browser склеивает с origin. ASP.NET Core: `GET /api/products/{id}` — match по **path**. OAuth redirect: `?code=abc&state=xyz` — данные в **query**. Ошибки в понимании URL: double-encoding `%20`, потеря query при redirect, секреты в query (логи), ожидание что server прочитает `#section`.

URL выглядит как плоская строка, но это structured data (WHATWG URL Standard, RFC 3986). Middle-разработчик проектирует API paths, CORS origins, link generators, чинит «404» из-за wrong host или trailing slash. Парсинг URL в голове и инструментами — так же базов, как JSON.

Урок опирается на request-response: **request line** содержит target из **path + query** (origin-form). **Fragment** на wire к HTTP server не уходит. Освоив anatomy, избежишь класса routing/integration багов до `0.3.3`.

## Core Concepts

### Шесть блоков

```
https://shop.example.com:443/catalog/shoes?color=blue&size=42#reviews
```

| Part | Value | Role |
|------|-------|------|
| **Scheme** | `https` | Протокол (HTTP + TLS) |
| **Host** | `shop.example.com` | DNS или IP server'а |
| **Port** | `443` (часто опущен) | TCP port; default от scheme |
| **Path** | `/catalog/shoes` | Иерархический путь на server |
| **Query** | `color=blue&size=42` | Параметры после `?` |
| **Fragment** | `reviews` | Только client-side, после `#` |

**Authority** = `host:port` (port optional). После `://` до path — authority, включая редкий `user:pass@host`.

Для узла `0.3` главны **http** и **https**; также `ws`/`wss`, `mailto:`.

### Что идёт по HTTP

Target request использует **path и query**, не fragment:

```
GET /catalog/shoes?color=blue&size=42 HTTP/1.1
Host: shop.example.com
```

`#reviews` **не появляется** на wire. Browser/SPA использует fragment для якорей, HashRouter, legacy OAuth. API logs без hash — норма.

**Host header** несёт hostname. Scheme влияет на TLS и default port на client до формирования request; в типичной request line нет `https://` — только path + query.

### Defaults и явный port

| Scheme | Default port |
|--------|--------------|
| http | 80 |
| https | 443 |

`https://example.com/api` → port 443 implicit. `:8443` — non-default (Docker, corporate). Omit port для читаемости; explicit для исключений.

### Path semantics

Path иерархичен, slash-separated; case-sensitive на Linux/Kestrel — не полагайся на case для security. **Trailing slash**: `/api/users` vs `/api/users/` — разные routes или redirect; команда выбирает convention.

Path — не только «файлы»: `/orders/42/lines/3` — REST resources. Расширения в path (`/report.pdf`) vs header `Accept`.

**Relative URL** резолвятся от **base** (document origin). На `https://app.example.com/dashboard` путь `/api/me` → `https://app.example.com/api/me`, а `api/me` → `https://app.example.com/dashboard/api/me` — частый баг.

### Query string

Начинается с `?`. Пары `key=value` через `&`. Duplicate keys — framework-dependent parsing.

**Percent-encoding:** reserved (`&`, `=`, пробелы, Unicode) кодируют в values. `encodeURIComponent('hello world')` → `hello%20world`. ASP.NET Core model binding декодирует.

Query для **filters, pagination, sort** — `?page=2&sort=-createdAt`. Query попадает в logs, Referer, history — **не клади секреты**; POST body или headers.

### Fragment

`#...` — для **user agent**, не origin server в HTTP. SSR не видит `#` в standard request parsing. `window.location.hash`, History API без hash.

## Under the Hood

Browser: WHATWG **URL parser** → structured object (origin, pathname, searchParams, hash). Invalid URL — throw в `new URL(relative, base)`.

Kestrel: **PathString**, **QueryString** на `HttpRequest`. Routing match по path.

DNS по **host** до TCP. Wrong host → NXDOMAIN или wrong machine. **IDN** в UI Unicode, в DNS Punycode (`xn--...`).

## Syntax / Commands / API

**JavaScript:**

```javascript
const u = new URL('https://shop.example.com/catalog?q=1#top');
u.pathname;  // '/catalog'
u.search;    // '?q=1'
u.hash;      // '#top'
```

**C# Uri:**

```csharp
var uri = new Uri("https://shop.example.com/catalog?q=1#top");
uri.AbsolutePath; // /catalog
uri.Fragment;     // #top
```

**curl** — fragment не уходит на server:

```bash
curl "https://httpbin.org/get?color=blue"
```

## Examples

**Pagination:** `GET .../v1/orders?page=3&pageSize=20` — path = collection, query = slice.

**Wrong relative:** `fetch('api/export')` на `/admin/users` → `/admin/api/export`. Fix: `/api/export`.

**Fragment в API:** `fetch('.../users#meta')` → server видит только `GET /users`.

## Common Mistakes & Anti-patterns

JWT/API keys в query — logs, cache, Referer leak. **Authorization** header.

Double encoding — `%2520`. Encode once при сборке URL.

Query vs path params — `/users/42` vs `/users?id=42`; будь consistent.

Server не читает hash — не проектируй API на `#`.

POST + redirect + trailing slash — legacy method change; normalize в gateway.

## Production & Real-World Notes

Versioning: `/v1/products` vs header — path visible для routing.

nginx path strip — rewrite mismatch → 404.

**CORS origin** = scheme + host + port; `localhost` ≠ `127.0.0.1`.

Link generation через helpers, не concat strings.

---

## Comparison / Trade-offs

Path param — identity; query — optional filters. `/products/42?fields=name,price`.

Absolute — curl, HttpClient; relative — same-origin SPA с правильным base.

Readable URL vs opaque UUID — trade-off privacy и stability.

## Quick Reference

```
scheme://host:port/path?query#fragment
```

| В HTTP request | Не на server |
|----------------|--------------|
| Path, query | Fragment |
| Host header | Scheme (client-side TLS) |

Defaults: http 80, https 443.

---

## Key Takeaways

- URL = scheme + authority + path + query + fragment.
- **Fragment не отправляется** server в standard HTTP.
- Default ports опускают; explicit при non-standard.
- Percent-encode query; секреты в headers.
- Relative URL — leading `/` критичен.
- Path = resource; query = filters/pagination.
- Host → DNS/TLS; wrong host — fail до routing.
