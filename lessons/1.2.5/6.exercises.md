# Exercises: Encapsulation

> Complete after lectures and passing `4.test-yourself.md`.

## Easy

### E1. Field to property
**Goal:** Remove public field.

Refactor `public class Point { public int X; public int Y; }` to encapsulated shape preserving external read; mutation via constructor or methods.

**Success criteria:** No public fields; external cannot set X/Y arbitrarily if you choose immutability.

---

### E2. Find the leak
**Goal:** Analysis.

Review class with public `{ get; set; }` on `List<T>` and document three ways caller breaks invariants.

**Success criteria:** Lists replace, clear, mutate items.

---

## Medium

### M1. BankAccount rules
**Goal:** Invariants inside type.

No overdraft, no negative deposit. No public balance field.

**Success criteria:** Invalid operations throw; valid paths work.

---

### M2. Tell, Don't Ask refactor
**Goal:** Move logic inward.

Given controller code with `if (order.Total > ...)` chain, sketch `order.ApplyBestDiscount()` API and what moves into Order.

**Success criteria:** Controller thinner; rules in domain type.

---

## Hard (stretch)

### H1. DTO → domain map
**Goal:** Boundary pattern.

`CreateOrderDto` with public properties → factory builds encapsulated `Order`. List why DTO can stay anemic but Order cannot.

**Success criteria:** Two-layer explanation in 5–8 sentences.
