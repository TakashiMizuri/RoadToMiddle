## Easy

### E1. Inspect remotes
**Goal:** See how `origin` is configured.

**Steps:**
1. `cd` into any Git repo with a remote (or clone a public repo).
2. Run `git remote -v` and `git remote show origin`.
3. Write down fetch URL and which local branches track remotes (if any).

**Success criteria:** You can explain what `origin` points to and that fetch/push URLs may differ.

---

### E2. Fetch without changing files
**Goal:** Experience safe fetch.

**Steps:**
1. Note current branch and run `git status`.
2. Run `git fetch origin`.
3. Run `git status` again and `git branch -vv`.

**Success criteria:** Working tree unchanged; you can name at least one `origin/*` ref that was updated or confirmed current.

---

## Medium

### M1. Ahead / behind inspection
**Goal:** Read divergence without merging.

**Steps:**
1. On `main` (or default branch): `git fetch origin`.
2. Run `git log --oneline main..origin/main` and `git log --oneline origin/main..main`.
3. In one sentence each: what does non-empty output mean?

**Success criteria:** Correct interpretation of behind vs ahead; no accidental `git pull`.

---

### M2. Publish a branch
**Goal:** First push of a local branch.

**Setup:** Create branch `exercise/remote-demo` with one commit (edit README or add `remote-demo.txt`).

**Tasks:**
1. Push to `origin` with upstream: `git push -u origin exercise/remote-demo`.
2. Verify on hosting UI or `git ls-remote origin exercise/remote-demo`.
3. Run `git branch -vv` â€” note tracking info.

**Success criteria:** Remote branch exists; local branch shows upstream tracking (`0.2.7` preview).

---

### M3. Recover from rejected push (simulated)
**Goal:** Practice non-fast-forward workflow.

**Setup:** Two clones of the same repo (A and B), same branch `exercise/push-test`.

**Tasks:**
1. In clone A: commit and push.
2. In clone B (outdated): commit different change, attempt push â€” observe rejection.
3. In B: `fetch`, integrate (merge or rebase), push successfully.

**Success criteria:** No `--force`; both commits end up on remote; you can narrate each step.

---

## Hard (stretch)

### H1. Fetch-first daily routine
**Goal:** Build a scriptable habit.

**Tasks:**
1. Write a shell/PowerShell snippet: checkout `main`, `fetch`, show commits you lack, then ask (echo) before merge.
2. Run it twice on different days or after a teammate push.

**Success criteria:** Script never auto-merges; documents `main..origin/main` before integration.
