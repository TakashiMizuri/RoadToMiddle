# Exercises: ValueTask — Pooling and Single-Await Contract

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Spot the bug
**Goal:** Identify single-await violations.

**Steps:** Review the snippet below. List every line that breaks the `ValueTask` contract and explain why.

```csharp
public class Loader
{
    private ValueTask<string> _pending;

    public async Task RunAsync()
    {
        _pending = LoadAsync();
        await Task.Delay(10);
        string a = await _pending;
        string b = await _pending;
    }

    private ValueTask<string> LoadAsync() => _client.GetStringAsync("/api/name");
}
```

**Success criteria:** At least two distinct violations named with correct reasoning.

### E2. Safe WhenAll
**Goal:** Parallelize two `ValueTask<int>` operations correctly.

**Steps:** Given `ValueTask<int> FetchA()` and `FetchB()` from a library, write `SumBothAsync()` using `Task.WhenAll` without double-awaiting any single `ValueTask`.

**Success criteria:** Compiles; each `ValueTask` consumed exactly once.

## Medium

### M1. Cache fast path
**Goal:** Implement sync fast path with `ValueTask<T>`.

**Steps:** Implement `GetSettingAsync(string key)` on an in-memory `Dictionary<string,string>`:
- Return `new ValueTask<string>(value)` on cache hit.
- On miss, call `await _remote.FetchAsync(key)`, cache, return (wrap slow path in `ValueTask` without `async ValueTask` signature if possible).

**Success criteria:** No `async ValueTask` that always suspends; cache hit allocates no `Task`.

### M2. Convert at the boundary
**Goal:** Bridge library `ValueTask` to application composition.

**Steps:** Write a method `ProcessBatchAsync(IEnumerable<Func<ValueTask<int>>> factories)` that runs all factories (invoke each once), converts safely, sums results via `Task.WhenAll`.

**Success criteria:** Handles empty batch; no stored `ValueTask` fields.

## Hard (stretch)

### H1. Explain BCL behavior
**Goal:** Connect theory to real API.

**Steps:** Read the signature of `Stream.ReadAsync(ReadOnlyMemory<byte>, CancellationToken)` in docs. Write a short paragraph: why BCL chose `ValueTask<int>`, what the caller must not do, and when `.AsTask()` is appropriate.

**Success criteria:** Mentions pooling/single-await; mentions buffered/sync-complete path.

### H2. Micro-benchmark design (no code required)
**Goal:** Design a measurement plan.

**Steps:** Describe how you would compare `Task<T>` vs `ValueTask<T>` for a 90%-sync method: what to measure (allocations, latency), tool (`dotnet-counters`, `BenchmarkDotNet`), and what result would **not** justify public API change.

**Success criteria:** Includes false-positive scenario (always-async workload).
