# Exercises: Firewalls and Ports

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Port matrix sketch
**Goal:** Map a typical three-tier stack to port exposure rules.

Draw a table with columns: **Service**, **Port**, **Expose to Internet?**, **Source allowed**, **Justification**.

Include at minimum: React (static via CDN), ASP.NET API, PostgreSQL, SSH admin access.

**Success criteria:** PostgreSQL marked not public; 443 public; internal app port only from LB.

---

### E2. Symptom matcher
**Goal:** Connect firewall concepts to error messages.

Match each symptom to the **most likely first check**:

| Symptom | First check |
|---------|-------------|
| `curl https://api.example.com` → connection timed out | ? |
| `curl https://api.example.com` → connection refused | ? |
| API logs: PostgreSQL connection timed out from private subnet | ? |
| Browser loads SPA but API calls fail with CORS (not timeout) | ? |

**Success criteria:** Four matches with one-sentence rationale each.

## Medium

### M1. Docker Compose review
**Goal:** Spot accidental exposure in a compose file.

Given this snippet (intentionally flawed):

```yaml
services:
  api:
    ports:
      - "5000:8080"
  postgres:
    ports:
      - "5432:5432"
```

List **two** security improvements for running this on a laptop connected to a corporate Wi‑Fi. Explain what each change prevents.

**Success criteria:** At least one change addresses PostgreSQL binding; reasoning references `0.4.6` and firewall concepts.

## Hard (stretch)

### H1. Security group PR review
**Goal:** Think like an infra reviewer.

A teammate proposes Terraform:

```hcl
ingress {
  from_port   = 5432
  to_port     = 5432
  protocol    = "tcp"
  cidr_blocks = ["0.0.0.0/0"]
}
```

Write a **PR comment** (4–6 sentences): what is wrong, suggested fix, and how the app tier should reach the database instead.

**Success criteria:** Mentions private subnet/SG-to-SG rule; no public CIDR on 5432.

## Hints (read only if stuck)

<details>
<summary>Hint E2 — timeout vs refused</summary>
Timeout often means packets never got a response (dropped). Refused means the host responded that nothing accepts the connection.
</details>

<details>
<summary>Hint M1 — postgres ports</summary>
Consider binding to `127.0.0.1` only, or removing the host port mapping and using internal Docker network DNS from the `api` service.
</details>
