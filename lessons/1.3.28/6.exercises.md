# Exercises: OrderBy, ThenBy — Stable Sort

## Easy

### E1. Two-level sort
**Goal:** Sort `Person` by City then Age descending.
**Success criteria:** Correct ThenByDescending placement.

### E2. Stability demo
**Goal:** Create 5 items, same sort key 2×, `OrderBy` key — verify stable order in debugger.
**Success criteria:** Document observed order vs input.

## Medium

### M1. Paged API
**Goal:** LINQ query with filter, `OrderBy`+`ThenBy(Id)`, Skip/Take, Count.
**Success criteria:** Explain why Id in ThenBy.

### M2. Fix the bug
**Goal:** Refactor double `OrderBy` chain to proper ThenBy — behavior changes explain.
**Success criteria:** Before/after sort order differs as expected.

## Hard (stretch)

### H1. Dynamic sort whitelist
**Goal:** Design safe mapping `sortBy` string → OrderBy/ThenBy (no arbitrary lambda injection).
**Success criteria:** Whitelist columns; default sort documented.
