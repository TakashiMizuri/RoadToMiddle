# Exercises: ref, out, in, params

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Safe division with out
**Goal:** Implement the `TryDivide` pattern.
**Steps:**
1. Create a console project.
2. Write `static bool TryDivide(int a, int b, out int quotient)`.
3. Return `false` when `b == 0`; otherwise set `quotient` and return `true`.
4. Call from `Main` with valid and invalid input; print results only on success.
**Success criteria:** Code compiles; failed division does not throw; `quotient` used only when method returns `true`.

### E2. Swap with ref
**Goal:** Swap two integers in place.
**Steps:**
1. Implement `static void Swap(ref int x, ref int y)`.
2. Swap `a` and `b` in `Main`; verify with `Console.WriteLine`.
**Success criteria:** After call, values are exchanged without return value.

## Medium

### M1. TryParse wrapper
**Goal:** Chain parsing with `out` variables.
**Steps:**
1. Read a line like `"10 20"` from console.
2. Split and use `int.TryParse` twice with inline `out int` declarations.
3. If both succeed, print sum; else print which part failed.
**Success criteria:** Uses C# flow analysis correctly; no use of unassigned `out` vars.

### M2. Read-only struct distance
**Goal:** Use `in` for a small readonly struct.
**Steps:**
1. Define `readonly struct Point2D { public readonly double X, Y; ... }`.
2. Implement `static double Distance(in Point2D a, in Point2D b)` using Pythagoras.
3. Call with two points; compare call with explicit `in` vs implicit.
**Success criteria:** Method does not mutate parameters; compiles with readonly fields.

### M3. params logger
**Goal:** Observe params array behavior.
**Steps:**
1. Write `static void Log(params string[] parts)`.
2. Call `Log("a")`, `Log("a", "b")`, and `Log(existingArray)` where `existingArray` is `string[]`.
3. In debugger or print `ReferenceEquals` — note when new array is created at call site.
**Success criteria:** You can explain allocation behavior for each call form.

## Hard (stretch)

### H1. TryGet with multiple out
**Goal:** Design a mini dictionary lookup API.
**Steps:**
1. Use `Dictionary<string, int>`.
2. Write `static bool TryGetScore(Dictionary<string, int> map, string key, out int score)`.
3. Add overload `TryGetScore(..., out int score, out string normalizedKey)` that trims key before lookup.
**Success criteria:** All `out` params assigned on every return path; no exceptions for missing keys.

### H2. ref in array slot
**Goal:** Mutate array element via ref.
**Steps:**
1. Create `int[] data = { 1, 2, 3 }`.
2. Write `static void DoubleAt(ref int slot) { slot *= 2; }`.
3. Call `DoubleAt(ref data[1])`.
**Success criteria:** `data[1]` is `4` after call; explain why `ref data[1]` is legal.
