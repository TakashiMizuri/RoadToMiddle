# Events: accessors и memory leak pattern

> Roadmap: `1.3.5` · Node: `1.3` — C# продвинутые темы · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Отличить **`event`** от delegate field; объяснить **add/remove accessors**.
- Реализовать **custom event accessors** с lock/logging.
- Объяснить **publisher-subscriber leak**: long-lived publisher держит subscriber.
- Применять fixes: **`-=`**, **weak events** (concept), **CancellationToken**.
- Распознавать leaks от **lambda capture** и **static events**.
- Безопасно **raise** events (`?.Invoke`, snapshot pattern).

---

## Why This Matters

`button.Click += Form.HandleClick` — если **Form** disposed, но **Button** жив (static toolbar), **Form остаётся reachable** через delegate **Target** — **memory leak** + zombie callbacks.

Domain events, UI, legacy integrations — missing **`-=`** = «memory растёт overnight».

---

## Core Concepts

### `event` — encapsulation

Compiler: private backing field + public **add/remove** only. External code не может **`=`** wipe all subscribers.

Raise inside class: **`SomethingHappened?.Invoke(this, EventArgs.Empty);`**

### Custom accessors

```csharp
public event EventHandler Click
{
    add { lock (_lock) { _click += value; } }
    remove { lock (_lock) { _click -= value; } }
}
```

For logging, weak refs, thread safety.

### Memory leak pattern

**Publisher** (long life) → multicast delegate → **subscriber targets** strongly reachable.

```
Static EventBus ──► delegate ──► Form (should be GC'd)
```

**Subscriber короче publisher** → **must `-=`** on Dispose.

**Lambda** `(s,e) => this.DoWork()` — closure holds **`this`**.

### Static events

**`public static event`** — subscribers pinned until unsubscribe. Prefer instance events or scoped mediator.

### Fix strategies

1. **Unsubscribe in Dispose** / component teardown.
2. **Weak event pattern** — WPF `WeakEventManager`; complex.
3. **Named handler** for reliable **`-=`** (`1.3.3`).
4. **CancellationToken** for async workflows.

### Safe raise

```csharp
protected virtual void OnChanged(EventArgs e)
{
    Changed?.Invoke(this, e);
}
```

Snapshot: `var h = Changed; h?.Invoke(this, e);` — reduces race with unsubscribe.

## Under the Hood

**`event`** metadata: `.addon`/`.removeon`. Delegate **Target** — strong ref.

**EventHandlerList** — WinForms multi-event storage.

**IObservable** + **IDisposable** subscription — alternative lifetime model.

---

## Common Mistakes

- Subscribe in ctor, no unsubscribe.
- Lambda without stored reference — cannot `-=`.
- Think `event` is weak — **always strong** by default.
- Static event + short-lived subscribers.

---

## Quick Reference

| Pattern | Leak risk |
|---|---|
| Instance + Dispose unsub | Low |
| Static event | High |
| Lambda += no -= | High |

**Rule:** subscriber lifetime < publisher → **unsubscribe**.

---

## Key Takeaways

- **event** = protected invocation list.
- Delegates **strongly reference** subscribers.
- **Static events** — global leak risk.
- **`-=`** in Dispose — production standard.
