# Exercises: LINQ Where, Select, OrderBy, GroupBy

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Active product names
**Goal:** Filter and project a collection.
**Steps:** Create `Product { Id, Name, IsActive }` list (5+ items). Return alphabetically sorted names of active products using method syntax only.
**Success criteria:** Single pipeline ending in `ToList()`; no manual loops.

### E2. Index labels
**Goal:** Use `Select` with index overload.
**Steps:** Given `string[] lines`, produce `"1: first line"` format for non-empty lines only.
**Success criteria:** `Select` with index; `Where` for whitespace.

## Medium

### M1. Regional revenue report
**Goal:** Combine `Where`, `GroupBy`, `Select`, `OrderBy`.
**Steps:** `Sale { Region, Amount, Date }` — last 90 days, group by region, `{ Region, Total, Count }`, top 5 regions by total.
**Success criteria:** Correct operator order; `Take(5)` after sort on aggregated data.

### M2. Multi-level sort
**Goal:** `OrderBy` + `ThenBy`.
**Steps:** Sort `Employee { Dept, LastName, FirstName }` by dept ASC, last name ASC, first name ASC. Print first 10.
**Success criteria:** One `OrderBy`, two `ThenBy` — not three `OrderBy`.

## Hard (stretch)

### H1. Dynamic minimum price filter
**Goal:** Understand closure capture.
**Steps:** Build `IEnumerable<Product>` pipeline in a loop: each iteration changes `minPrice`. Enumerate once per iteration. Document when filter threshold applies.
**Success criteria:** Written explanation matching observed output.

### H2. Pipeline vs SQL sketch
**Goal:** Connect to EF mentally.
**Steps:** For M1 pipeline, write pseudo-SQL (`WHERE`, `GROUP BY`, `ORDER BY`, `LIMIT`) that a database would execute. Mark which part must not happen client-side in production.
**Success criteria:** SQL order mirrors LINQ semantics.

## Submission checklist

- [ ] All pipelines use method syntax (not query syntax)
- [ ] No side effects inside `Select`/`Where`
- [ ] Code compiles with `dotnet run`
