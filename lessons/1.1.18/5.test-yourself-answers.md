## Answers

### 1. [Recall]
**Answer:** **`int?`** — **`Nullable<int>`**, struct с **`HasValue`**; может быть **`null`**. **`int`** всегда holds value (default 0).
**Why:** Runtime wrapper, не «расширенный int» без флага.

### 2. [Recall]
**Answer:** **`default(int?)` → null**; **`default(int) → 0`**.
**Why:** Nullable default — отсутствие значения.

### 3. [Understanding]
**Answer:** Throw **`InvalidOperationException`**. Альтернативы: **`??`**, **`GetValueOrDefault(...)`**, pattern **`if (age is int v)`**.
**Common mistake:** Считать nullable «безопасным» при прямом **`.Value`**.

### 4. [Understanding]
**Answer:** **`c` is `null`** (lifted addition: any null operand → null result).
**Why:** Nullable lifting rules.

### 5. [Understanding]
**Answer:** **`eq` is `null`** (`bool?`), не **`false`** — «unknown», не «not equal».
**Why:** SQL-like three-valued logic.

### 6. [Understanding]
**Answer:** **`?? 10`** → 10 при null. **`GetValueOrDefault()`** без args → **`0`**, не 10 — different fallback.
**Why:** Default of **`int`** is zero.

### 7. [Application]
**Answer:** **`0`** может быть valid stock; **`-1`** collides. **`null`** = unknown/not provided distinctly.
**Why:** Honest optional semantics in API/DB.

### 8. [Application]
**Answer:** После первого **`??=`**, **`cache`** has value; повторные обращения не вызывают **`Load()`**.
**Why:** **`??=`** assigns only when null.

### 9. [Analysis]

### 10. [Analysis]
**Answer:** Если **`GetNullable()`** returns null → throw on **`.Value`**. Need null check or **`??`**.
**Why:** Classic nullable bug.

### 11. [Understanding]
**Answer:** SQL **`NULL`** = absent; EF maps nullable column to **`DateTime?`**, **`null`** CLR ↔ **`NULL`** SQL; non-nullable **`DateTime`** requires value on read.
**Why:** Alignment of three-layer model (SQL, EF, C#).

### 12. [Understanding]
**Answer:** **Нет.** **`string?`** — compile-time flow analysis; **`int?`** — real struct **`HasValue`** at runtime.
**Why:** Different language features sharing **`??`** syntax.

### 13. [Application]
**Answer:** Parameter **`decimal? minPrice`**. Filter **`if (minPrice is decimal mp) q = q.Where(p => p.Price >= mp)`** — не treat 0 as «missing» unless spec says so.
**Why:** **`null`** = omit filter; **`0`** = valid threshold.

### 14. [Recall]
**Answer:** Assign right to left **only if** left is currently **`null`** (for nullable/reference).
