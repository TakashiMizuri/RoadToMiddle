## Answers

### 1. [Recall]
**Answer:** **Method name** + **parameter types** (и ref/in/out modifiers); return type **не** part of signature для overload.
**Why:** Правила C# overload resolution.

### 2. [Recall]
**Answer:** **Нет** — CS0111 duplicate signature.
**Why:** Return type alone не различает overload.

### 3. [Understanding]
**Answer:** **Parameter** — variable в declaration; **argument** — expression в call site.
**Why:** Definition vs invocation vocabulary.

### 4. [Understanding]
**Answer:** **`Print(int)`** — `42` — int literal, exact match предпочтительнее widening без необходимости.
**Why:** Best match rules favor exact type.

### 5. [Understanding]
**Answer:** Positional omit trailing — compiler must know какие args skipped; required first задаёт fixed positions.
**Why:** Optional только в хвосте list.

### 6. [Application]
**Answer:** **5**.
**Why:** Simple addition.

### 7. [Application]
**Answer:** **`static int Area(int side)`** — one int argument.
**Why:** Exact match square overload.

### 8. [Analysis]
**Answer:** **Нет**, duplicate signature.
**Why:** Same name + same param types.

### 9. [Analysis]
**Answer:** **CS0161** — not all code paths return value.
**Why:** Need `return b;` after if.

### 10. [Application]
**Answer:** `static bool IsEven(int n) => n % 2 == 0;`
**Why:** Expression-bodied bool.

### 11. [Understanding]
**Answer:** Separation of concerns — validator reusable; caller decides UI/logging; testable without console.
**Why:** Pure-ish logic vs I/O coupling.

### 12. [Application]
**Answer:** **Named arguments** (`times: 2`).
**Why:** Skip optional positional params by name.

### 13. [Recall]
**Answer:** **PascalCase** (`CalculateTotal`).
**Why:** Microsoft C# conventions.

### 14. [Understanding]
**Answer:** Когда operations **conceptually different** — `SaveUser` vs `DeleteUser`, not overload `Process`.
**Why:** Overload = same operation, different shapes.
