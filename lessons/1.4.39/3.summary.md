# Task vs ValueTask — краткая выжимка

> `1.4.39` · ~7 мин на повторение

## Главная идея

**`Task`** — default для API и app code. **`ValueTask`** — когда hot path **часто завершается синхронно** и caller делает **ровно один await** сразу. **Нельзя await один ValueTask дважды** — используй `.AsTask()`.

## Ключевые пункты

- `Task` — heap, composable, safe double-await (completed).
- `ValueTask` — struct; может обернуть `T`, `Task`, `IValueTaskSource`.
- Sync path: `return new ValueTask<T>(result)` — без Task alloc.
- `Task.WhenAll`, storage, multiple consumers → `Task` или `.AsTask()`.
- `async ValueTask` выгоден при частых early sync return.
- Profile перед сменой сигнатур; public API — осторожно.

## Decision tree (коротко)

1. App/public API? → **Task**
2. Hot + profiling + often sync-complete? → **ValueTask**
3. Нужен WhenAll/store? → **Task**

## Частые ошибки

- Массовая замена Task на ValueTask без данных.
- Double-await ValueTask.
- ValueTask в field + отложенный await.

## Мнемоника

> **Task by default; ValueTask once** — Task по умолчанию, ValueTask только при однократном await.
