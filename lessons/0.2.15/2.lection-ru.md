## Core Concepts

### Polyrepo

**Несколько Git repositories** — часто один deployable unit на repo:

```
acme/api-backend
acme/web-frontend
acme/terraform-infra
```

Свой history, CI, tags, permissions. Cross-cutting feature = **несколько PR** + совместимые версии пакетов.

**Плюсы:** ownership, независимые релизы, меньший clone, изоляция CI.  
**Минусы:** drift зависимостей, сложные atomic changes, дублирование tooling.

Git-команды — **per repo**: cherry-pick в `api`, tag `v2.1.0` только там.

### Monorepo

**Один repo**, много projects:

```
acme-platform/apps/api
acme-platform/apps/web
acme-platform/packages/contracts
```

Один clone; **один PR** может менять API + React + contracts atomically.

**Плюсы:** single source of truth, atomic refactors, одна `.gitattributes`, unified CI cache.  
**Минусы:** большой clone/history, сложный CI (path filters), coarse permissions, нужны conventions для semver tags.

### Versioning

**Polyrepo:** semver **per repo** — естественно.  
**Monorepo:** один tag на весь repo **или** `api/v2.0.0`, `web/v1.5.0` — team convention.

Hotfix: polyrepo — два cherry-pick, два tag; monorepo — один commit может чинить оба слоя.

### CI и DX

**Polyrepo:** простой CI per repo; e2e orchestration отдельно.  
**Monorepo:** **affected detection** — build только `apps/api/**` при изменении API.

Hybrid: submodules, meta-repo — awareness, часто painful.

## Under the Hood

Git не знает «monorepo» — это **организация**. Один `.git`, один object graph. Scale: shallow clone, sparse checkout. **Bisect** ищет по всем paths — нужны path-specific tests. **Polyrepo:** `.gitattributes` может drift между repos.

---

## Examples

Startup → monorepo. Enterprise 50 teams → polyrepo. Hotfix auth library: polyrepo bump dependency в API+web; monorepo один commit.

---

## Common Mistakes & Anti-patterns

Monorepo без path CI filters; polyrepo с жёсткой coupled release; dogma; duplicated policies; неясный scope tag на monorepo root.

---

## Comparison / Trade-offs

| | Monorepo | Polyrepo |
|---|----------|----------|
| Atomic change | Легко | Сложно |
| Clone | Больше | Меньше per repo |
| CI | Path logic | Проще per repo |
| Permissions | Грубее | Тоньше |

Monorepo: один product, coupled fullstack. Polyrepo: independent services, разные SLA.

---

## Key Takeaways

- Организация, не Git feature.
- Monorepo = atomic cross-layer; polyrepo = independent release.
- CI/tags/hotfix масштабируются по-разному.
- Выбор по **coupling и команде**, не hype.
