# IAsyncEnumerable — async streams — краткая выжимка

> `1.4.16` · ~6 мин на повторение

## Главная идея

**`IAsyncEnumerable<T>`** — **ленивая async-последовательность**: consumer тянет элементы через **`await foreach`**, producer между **`yield return`** может **`await`** I/O. Альтернатива загрузке всего результата в память.

## Ключевые пункты

- **`IEnumerable`** — sync `MoveNext()`; **`IAsyncEnumerable`** — `MoveNextAsync()` → `ValueTask<bool>`.
- Авторство: **`async IAsyncEnumerable<T>`** + **`yield return`** / **`yield break`**.
- **`[EnumeratorCancellation]`** — token из consumer в async iterator.
- **`.WithCancellation(ct)`** при `await foreach`.
- **Implicit backpressure** — slow consumer задерживает следующий `MoveNextAsync`.
- EF: **`AsAsyncEnumerable()`**; LINQ: пакет **System.Linq.Async**.
- **`DisposeAsync`** — освобождение connection/stream в `finally` у `await foreach`.

## Самые важные примеры

```csharp
async IAsyncEnumerable<Order> StreamAsync([EnumeratorCancellation] CancellationToken ct)
{
    await foreach (var page in _repo.PagesAsync(ct))
        foreach (var o in page)
            yield return o;
}

await foreach (var o in StreamAsync(ct).WithCancellation(ct))
    await ProcessAsync(o);
```

## Частые ошибки

- Блокирующий «async» через sync iterator + `.Result`.
- Нет cancellation — утечки работы после disconnect.
- `ToList()` на большом stream.
- Путать с **Channels** (push, multi-producer).

## Мнемоника

> **`IAsyncEnumerable` = netflix с паузой на загрузку** — следующая серия только когда буфер и сеть готовы, без скачивания всего сезона сразу.
