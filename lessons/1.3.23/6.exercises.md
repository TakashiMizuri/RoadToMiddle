# Exercises: LINQ Aggregate and Scan

## Easy

### E1. Seed vs no-seed
**Tasks:** Predict results for empty/single/multi on both overloads; verify in LINQPad/console.

### E2. Product
**Tasks:** **`Product(IEnumerable<int>)`** with empty → 1 via **`Aggregate`**.

## Medium

### M1. Implement Scan
**Tasks:** Extension **`Scan(source, seed, func)`** with **`yield`**. Unit test running sums.

### M2. String build
**Tasks:** Build CSV line from 1000 strings — compare **`Aggregate +`** vs **`StringBuilder` seed** (time or allocation note).

## Hard

### H1. Reducer
**Tasks:** **`Aggregate`** over event list into immutable **`State`** (Apply per event). Optional **`Scan`** for audit log of states.
