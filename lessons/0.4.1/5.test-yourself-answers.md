## Answers

### 1. [Recall]
**Answer:** Application → Transport → Internet → Link (сверху вниз).
**Why:** Это четырёхуровневая Internet model / TCP/IP stack, используемая в dev docs и kernel.

### 2. [Recall]
**Answer:** Application layer (прикладной уровень).
**Why:** HTTP и DNS — протоколы для приложений; они используют transport ниже, но сами не маршрутизируют packets.

### 3. [Understanding]
**Answer:** HTTP описывает формат сообщений, но не знает global routing. **Internet layer (IP)** доставляет packets между сетями по IP addresses; transport (TCP) строит connection поверх IP.
**Why:** Separation of concerns — каждый уровень добавляет свою гарантию.

### 4. [Understanding]
**Answer:** Каждый нижний уровень оборачивает данные верхнего в свой header (frame/packet/segment). HTTP bytes → TCP segment → IP packet → link frame → bits на medium.
**Why:** Encapsulation — механизм layering; upper layers остаются opaque для lower.

### 5. [Understanding]
**Answer:** HTTP не начался — нет listening process на port 5000 или TCP отклонил connection. Диагностика с **transport** (слушает ли Kestrel?) и **binding** (`0.4.6`), не HTTP semantics.
**Common mistake:** искать controller/route при refused connection.

### 6. [Application]
**Answer:** 1) DNS lookup → 2) TCP three-way handshake → 3) TLS handshake → 4) HTTP GET.
**Why:** Нужен IP (DNS), затем TCP channel, затем TLS поверх TCP, затем HTTP.

### 7. [Application]
**Answer:** `Connected ... port 443` — **transport** (TCP socket established). TLS details — **application/presentation** (TLS перед HTTP bytes).
**Why:** TCP connect завершается до TLS ClientHello.

### 8. [Analysis]
**Answer:** **C** неверно.
**Why:** IP address — internet layer; port — transport layer. A, B, D — корректны.

### 9. [Analysis]
**Answer:** 404 — valid HTTP response; network доставила bytes успешно. Проблема **application** (route/resource), не link/internet failure.
**Why:** Status code implies HTTP exchange completed.

### 10. [Understanding]
**Answer:** Link — один local segment (MAC, same subnet). Internet — end-to-end между networks через routers по IP.
**Why:** Gateway передаёт packet с link segment на link segment глобально.

### 11. [Recall]
**Answer:** OSI **Layer 4** (Transport).
**Why:** Прямое mapping TCP/UDP.

### 12. [Application]
**Answer:** **Transport TCP** — connection reset до/во время HTTP; proxy уже на application, но upstream оборвал TCP. Проверять upstream process, firewall L4, TLS mismatch causing reset.
**Why:** «Connection reset» — TCP RST, не malformed HTTP line.

## Model answers for practical sections

Encapsulation (пример):
```
Link:    [Eth hdr | IP pkt | TCP seg | TLS+HTTP | Eth FCS]
Internet:[IP hdr | TCP seg | TLS+HTTP]
Transport:[TCP hdr | TLS+HTTP]
App:     [TLS | HTTP GET ...]
```
(На практике TLS считают частью application path перед TCP payload.)
