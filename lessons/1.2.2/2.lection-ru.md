# Fields и properties — auto, full, init-only, required

> Roadmap: `1.2.2` · Node: `1.2` — C#: ООП · Depth: **практика**

## Learning Objectives

После этого урока ты сможешь:

- Различать **fields** (storage) и **properties** (controlled accessors).
- Писать **auto-properties**, **full properties** с backing fields и выбирать, когда что уместно.
- Использовать **`init`** для immutable-after-construction state.
- Применять **`required`** members (C# 11+), чтобы callers обязаны были задать критичные значения при создании.
- Понимать **default values**, **property initializers** и compiler-generated backing fields.
- Избегать утечки invariants через плохие public setters.

---

## Why This Matters

В `1.2.1` class — blueprint со state и behavior. Следующий вопрос: **как хранить и expose state**. Новички часто ставят public fields — быстро компилируется. Профессиональный C# использует **properties**: API surface, который можно evolve (validation, logging, change notification) без поломки callers.

ASP.NET Core model binding, JSON serializers, EF Core и React DTOs работают с **property names**, не с raw fields. `init` и `required` поддерживают **immutable configuration** и безопасное construction — options classes, DTOs, domain entities.

Понимание translation `{ get; set; }` → hidden backing field помогает читать decompiled code, debug serialization и знать, когда **full property** оправдана.

## Core Concepts

### Fields: raw storage

**Field** — variable внутри type. Instance fields — **per object**; static — per type (`1.2.12`).

```csharp
public class TemperatureReading
{
    private decimal _celsius;
}
```

Fields — **`private`** для storage, **`readonly`**, **`const`**. Fields держат **bits/references** в object layout на heap. Properties обычно **front** fields.

### Properties: controlled access

**Property** выглядит как field для caller, реализована через **`get`** / **`set`**:

```csharp
public decimal Celsius
{
    get => _celsius;
    set => _celsius = value;
}
```

Call site: `reading.Celsius = 20m;` — единый синтаксис. Libraries bind к properties.

**Full properties** добавляют logic:

```csharp
public decimal Fahrenheit
{
    get => _celsius * 9m / 5m + 32m;
    set => _celsius = (value - 32m) * 5m / 9m;
}
```

Validation и side effects — здесь, не на каждом assignment site.

### Auto-properties

Compiler генерирует hidden **backing field**:

```csharp
public string Sku { get; set; }
public int Quantity { get; init; }
```

IL: `<Sku>k__BackingField`. **Property initializers**:

```csharp
public bool IsActive { get; set; } = true;
public List<string> Tags { get; set; } = new();
```

Осторожно с **mutable reference defaults**: каждый instance получает свой list, но public `set` всё ещё может заменить list целиком.

### Read-only surface: get-only и init

**Get-only:**

```csharp
public Guid LineId { get; } = Guid.NewGuid();
```

**`init`** (C# 9+): set только при **object initialization**:

```csharp
public class Product
{
    public required string Sku { get; init; }
    public required string Name { get; init; }
}
```

После construction `p.Sku = "B2"` — ошибка. `init` — immutable-after-build без giant constructors.

### Required members (C# 11+)

**`required`** — при creation **обязан** assign:

```csharp
public required string Email { get; set; }
```

Без `Email` — **compile error** CS9035. Работает с object initializer, constructor, `with` на records.

### Full property с explicit backing field

Когда auto недостаточно — validation, lazy init:

```csharp
private string _email = string.Empty;

public string Email
{
    get => _email;
    set
    {
        if (string.IsNullOrWhiteSpace(value))
            throw new ArgumentException("Email required.");
        _email = value.Trim().ToLowerInvariant();
    }
}
```

Convention: **`_camelCase`** field, **`PascalCase`** property.

## Syntax / API

```csharp
public int Count { get; set; }
public string Code { get; init; }
public required string Title { get; init; }
public DateTime CreatedUtc { get; } = DateTime.UtcNow;
public string Status { get; private set; } = "Draft";
```

## Examples

**Example 1 — validation в setter:**

```csharp
public class BankAccount
{
    private decimal _balance;

    public decimal Balance
    {
        get => _balance;
        private set
        {
            if (value < 0) throw new InvalidOperationException("Negative balance.");
            _balance = value;
        }
    }

    public void Deposit(decimal amount) => Balance = _balance + amount;
}
```

Внешний код читает `Balance`, но не assign напрямую — мутации через methods с rules.

**Example 2 — required init DTO:**

```csharp
public class CreateCustomerCommand
{
    public required string Name { get; init; }
    public required string Email { get; init; }
    public string? Phone { get; init; }
}
```

Missing `Name`/`Email` при `new` — compile error; optional `Phone` можно не задавать.

**Example 3 — computed property без backing field:**

```csharp
public class Rectangle
{
    public double Width { get; set; }
    public double Height { get; set; }
    public double Area => Width * Height;
}
```

`Area` не хранится — вычисляется при каждом get из Width/Height.

## Common Mistakes & Anti-patterns

**Public fields «для скорости»** — ломают encapsulation (`1.2.5`); serializers всё равно prefer properties.

**Mutable collections с public `set`** — caller replace/clear list минуя domain rules.

**Validation только в UI** — API clients обходят UI.

**`required` без понимания init contexts** — assign только при creation, не произвольно после.

**Auto-property когда нужна logic** — refactor на full property без смены call syntax.

---

## Production & Real-World Notes

System.Text.Json deserializes в **public properties** с set/init. EF Core maps columns к properties; backing field access настраивается для encapsulation. `IOptions<T>` options classes часто `{ get; init; }` после configuration binding — immutable snapshot settings.

---

## Comparison / Trade-offs

| Mechanism | After ctor | Enforcement |
|-----------|------------|-------------|
| `{ get; set; }` | Mutable | — |
| `{ get; init; }` | Frozen | Init only |
| `required` | Must set at creation | Compiler |
| Full property | Controlled | Custom rules |

---

## Quick Reference

```
field → storage (private)
property → get/set API
auto → compiler backing field
init → set during initialization only
required → must assign at creation
```

---

## Key Takeaways

- **Fields store; properties expose.**
- **Auto** — без logic; **full** — validation/computed.
- **`init`** + **`required`** — safe construction patterns.
- Libraries bind к **properties**.
