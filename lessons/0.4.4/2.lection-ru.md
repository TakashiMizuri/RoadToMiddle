# UDP — когда datagrams лучше byte streams

> Roadmap: `0.4.4` · Node: `0.4` — Основы сетей · Depth: понимание

## Learning Objectives

- Объяснить semantics UDP: connectionless, unreliable datagram delivery поверх IP.
- Сравнить UDP с TCP (`0.4.3`) и выбрать для сценария.
- Объяснить, почему DNS, video, VoIP, games часто на UDP.
- Message boundaries vs byte stream для design протоколов.
- Роль UDP под QUIC/HTTP/3 (`0.3.13`).
- Когда оставаться на TCP (HTTP API, DB).

---

## Why This Matters

`0.4.3` — TCP для HTTP, TLS, PostgreSQL. Но **UDP** everywhere: **DNS** (`0.4.5`), Zoom/Discord AV, multiplayer games. Выбор UDP — **trade-off**: retransmit 200 ms old video frame бессмысленен; TCP handshake на каждый DNS lookup — лишняя latency.

HTTP/3 (**QUIC**) — reliability в user space **поверх UDP** (`0.3.13`). Classic UDP cases объясняют, зачем так сделали.

## Core Concepts

### UDP одной фразой

**UDP** — header (ports, length, checksum) + payload → IP. No connection state, ACK, retransmit, order — **fire and forget** с port multiplexing (`0.4.2`).

Каждый `send()` — один **datagram** (practical limit ~1472 B payload на Ethernet). Receiver — discrete datagrams или nothing.

### UDP vs TCP

| Need | Prefer |
|------|--------|
| Correct every byte (HTTP, SQL) | TCP |
| Low latency, tolerate loss | UDP + app logic |
| Small fast query (DNS) | UDP first |
| Kernel congestion control | TCP or QUIC |

TCP — handshake (`0.4.3`); UDP — immediate send.

### DNS (preview 0.4.5)

Queries на port **53** UDP. Large response → **TC flag** → retry **TCP**. Block UDP 53 — broken DNS при working TCP 443.

### Video / VoIP

Late frame useless; TCP retransmit → stall. UDP drop and move; RTP often on UDP.

### Games

Position 20–60/s; old state irrelevant. TCP → lag/rubber-band. UDP + prediction; critical events — reliable channel optional.

### Message boundaries

TCP merges writes in one read — HTTP uses Content-Length. UDP: one datagram = one unit; framing в protocol design.

### QUIC / HTTP/3

QUIC on UDP :443 — streams + TLS inside; firewall must allow UDP 443 for HTTP/3 (`0.3.13`).

## Examples

REST API — TCP 443. FPS server — UDP movement, TCP login maybe. HTTP/3 needs UDP 443 in security group.

## Common Mistakes

«UDP always faster». Block all UDP — breaks DNS/QUIC. UDP file upload without reliability — loss.

---

## Production

DNS UDP/TCP ratio; WebRTC STUN/TURN; UDP amplification DDoS; DTLS for encryption.

---

## Quick Reference

UDP: no connection, no reliability, datagram boundary. TCP: stream, reliable. DNS 53 UDP+TCP fallback.

---

## Key Takeaways

- UDP — datagrams без setup/retransmit.
- Timeliness > completeness → UDP (video, games).
- DNS — UDP first (`0.4.5`).
- QUIC — UDP carrier, reliability inside (`0.3.13`).
- APIs/DB — TCP-first.
