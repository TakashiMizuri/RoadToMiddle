## Why This Matters

Web-приложения **produce** work (HTTP → email, webhooks, audit) быстрее, чем **consumers** обрабатывают (SMTP, external API, DB). Наивный **`List<T>` + lock`** растёт до **OutOfMemoryException**. **`BlockingCollection`** блокирует threads — плохо для async ASP.NET Core.

**`System.Threading.Channels`** (.NET Core 3.0+) — **async-friendly queues** для single/multi producer. **`WriteAsync`** на bounded channel **await'ит**, когда buffer полон — **backpressure** на producer вместо RAM.

Channels в **background task queues**, **pipeline stages**, **rate-limited ingestion**, **`1.4.34` BackgroundTaskQueue**. Отличает structured throughput от fire-and-forget **`Task.Run`**.

## Core Concepts

### Producers, consumers, buffer

```
Producers → [ bounded buffer ] → Consumers
```

**`Channel<T>`** → **`Writer`** (producers) + **`Reader`** (consumers). Стороны передаются разным tasks — чёткая граница API.

### Bounded vs unbounded

**Unbounded** — **`WriteAsync`** не ждёт места (кроме completed). Память растёт при lag consumer — только для tiny volume.

**Bounded** — при full **`WriteAsync`** ждёт slot — **explicit backpressure**. Default для request-sourced workloads.

```csharp
var channel = Channel.CreateBounded<OrderEvent>(new BoundedChannelOptions(100)
{
    FullMode = BoundedChannelFullMode.Wait
});
```

**`FullMode`:** **`Wait`**, **`DropWrite`**, **`DropOldest`** — correctness vs telemetry.

### Read / Write

```csharp
await channel.Writer.WriteAsync(item, ct);
await foreach (var item in channel.Reader.ReadAllAsync(ct))
    await ProcessAsync(item);
```

**`Writer.Complete()`** — shutdown; readers drain и завершаются.

### Completion

**`Complete(Exception)`** — readers могут получить **`ChannelClosedException`**. Координируй shutdown через **`CancellationToken`**.

## Under the Hood

Channels используют lock-free техники и **`IValueTaskSource`** для efficient **`ReadAsync`/`WriteAsync`**. **In-process only** — не distributed queue (Redis, RabbitMQ).

---

## Examples

### Background work queue

```csharp
private readonly Channel<WorkItem> _channel =
    Channel.CreateBounded<WorkItem>(500);

public async Task RunConsumerAsync(CancellationToken ct)
{
    await foreach (var work in _channel.Reader.ReadAllAsync(ct))
        await work.ExecuteAsync(ct);
}
```

Handler **`WriteAsync`**; hosted service — consumer loop.

---

## Common Mistakes & Anti-patterns

Unbounded channel под unbounded request rate. Никогда **`Complete()`** — consumer висит. **`TryWrite` false** без обработки — потеря work. Channel как cross-process queue.

## Comparison / Trade-offs

| | `IAsyncEnumerable` | `Channel` |
|---|-------------------|-----------|
| Модель | Pull iterator | Buffered queue |
| Producers | Обычно один | Много |
| Backpressure | Implicit pull | Bounded buffer |

---

## Key Takeaways

- **Bounded channels** — backpressure через await write.
- **`ReadAllAsync`** — idiomatic consumer.
- **`Complete()`** на shutdown host'а.
- Не broker — только in-process.
