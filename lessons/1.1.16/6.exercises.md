# Exercises: StringBuilder vs concatenation

> Complete after lectures, **1.1.4** recap, and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Three parts
**Goal:** Compare readability.
**Steps:** Build `"User: {name}, Id: {id}"` with `$""`, with `+`, with `StringBuilder.Append`. Same result.
**Success criteria:** Three variants compile; comment which you'd keep and why.

### E2. Join lines
**Goal:** Use Join not loop concat.
**Steps:** `string[] lines = { "a", "b", "c" }`; single `string.Join('\n', lines)`.
**Success criteria:** One statement; no `+=` in loop.

## Medium

### M1. HTML snippet builder
**Goal:** StringBuilder in loop.
**Steps:** List of 100 item names; build `<ul><li>...</li></ul>` with StringBuilder; preset capacity ~estimate.
**Success criteria:** Single `ToString()` at end; no `+=` on string in loop.

### M2. Bad vs good
**Goal:** Refactor.
**Steps:** Implement naive `BuildBad(IEnumerable<int> ids)` with `s += id + ","`; implement `BuildGood` with StringBuilder or Join.
**Success criteria:** Same output; written 2-sentence comparison of allocation behavior.

### M3. Capacity experiment
**Goal:** Feel reallocations.
**Steps:** Append 10_000 chars to StringBuilder with default vs initial capacity 10_000; read `Capacity` after (debug print).
**Success criteria:** Observe lower Capacity growth steps when pre-sized.

## Hard (stretch)

### H1. CSV export
**Goal:** Realistic export.
**Steps:** List of records `{ Id, Name, Email }`; StringBuilder with header row + data rows; escape commas in Name by wrapping quotes (simple).
**Success criteria:** Correct CSV shape; no string concat in inner loop body except via Append.

### H2. When not to build string
**Goal:** Streaming mindset.
**Steps:** Pseudocode or comment: write same CSV to `StreamWriter` line by line instead of one string; list memory trade-off.
**Success criteria:** Explains peak memory difference vs single StringBuilder.ToString().
