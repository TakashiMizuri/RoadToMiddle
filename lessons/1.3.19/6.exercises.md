# Exercises: Local Functions vs Private Methods

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Local or private?
**Goal:** Apply the decision rubric.

**Tasks:** For each scenario, choose **local function**, **private method**, or **lambda**, with one-line justification:
1. Trim and dedupe tags inside one API handler only.
2. Same email validation in three controllers.
3. `items.Where(x => x.IsActive)` filtering.
4. Recursive directory walk inside one export method.

**Success criteria:** Four choices with clear reuse/capture/delegate reasoning.

---

### E2. Add static
**Goal:** Use `static` local functions correctly.

**Tasks:** Refactor so inner helpers are `static` where capture is unnecessary:
```csharp
public int CountVowels(string text)
{
    int CountIn(string s)
    {
        return s.Count(c => "aeiou".Contains(char.ToLower(c)));
    }
    return CountIn(text);
}
```

**Success criteria:** Compiles; no accidental capture; behavior unchanged.

## Medium

### M1. Parse with local functions
**Goal:** Capture and `ref`/`out` in one workflow.

**Tasks:** Implement `TryParsePoint(string text, out int x, out int y)` for format `"10,20"` using a local function `TryReadInt` that advances a span or index.

**Success criteria:** Single outer public method; local helper; no class-level private methods.

---

### M2. Extract when reuse appears
**Goal:** Recognize when to promote local → private.

**Tasks:**
1. Start with local `ValidateSku` in one minimal API route.
2. Add second route needing same rule — refactor to shared `private static` or static class.
3. Write two sentences: why promotion was necessary.

**Success criteria:** No duplicated validation logic; tests could target static helper if added.

---

## Hard (stretch)

### H1. Stack trace observation
**Goal:** Connect generated names to source.

**Tasks:** Write a method with a capturing local function that throws `InvalidOperationException`. Run and paste stack trace. Identify the generated method name pattern.

**Success criteria:** Explain mangled name vs your chosen local name.
