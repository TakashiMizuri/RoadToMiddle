# Exercises: Enums — Flags, Parsing, TryParse

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Define a flags enum for HTTP methods
**Goal:** Practice `[Flags]` and power-of-two assignment.

**Tasks:**
1. Create `[Flags] enum HttpMethodFlags` with `None`, `Get`, `Post`, `Put`, `Delete`.
2. Combine `Get | Post` into variable `allowed`.
3. Write `bool CanPost(HttpMethodFlags m)` using bitwise `&`.

**Success criteria:** `CanPost(allowed)` is true; `ToString()` on `allowed` shows both names.

---

### E2. Safe parse from console input
**Goal:** Use `TryParse` instead of `Parse`.

**Tasks:**
Write `static OrderStatus ReadStatus()` that loops until user enters a valid `OrderStatus` name (case-insensitive) or `q` to quit.

**Success criteria:** Invalid input prints helpful message with `Enum.GetNames<OrderStatus>()`; no unhandled exceptions.

---

## Medium

### M1. Permission checker service
**Goal:** Model authorization flags in a small class.

**Tasks:**
```csharp
[Flags]
public enum Permission { /* design members */ }

public sealed class UserContext
{
    public Permission Grants { get; init; }
    public bool Has(Permission required) => /* you implement */;
    public bool HasAll(Permission mask) => /* all bits in mask set */;
}
```

Unit-test: user with `Read | Write` passes `Has(Read)`, fails `Has(Delete)`, passes `HasAll(Read | Write)`.

**Success criteria:** Uses `&`, not `HasFlag`, in `Has`/`HasAll`.

---

### M2. API query parser
**Goal:** Simulate Minimal API enum binding behavior.

**Tasks:**
Implement:
```csharp
public static Result<SortDirection, string> ParseSort(string? sort)
```
Return error message listing valid values when `sort` is invalid; treat null as `SortDirection.Asc`.

**Success criteria:** `"desc"`, `"Desc"`, `"DESC"` work with `ignoreCase: true`; `"down"` returns error, not throw.

---

### M3. Find the design bug
**Goal:** Code review practice.

**Tasks:**
Given broken flags enum (sequential values causing collision), explain bug and refactor to power-of-two. Add `None = 0`.

**Success criteria:** Written explanation + fixed enum compile and demonstrate collision before/after with `|`.

---

## Hard (stretch)

### H1. Custom strict flags parser
**Goal:** Stricter than BCL comma parsing.

**Tasks:**
Parse space-separated flags `"View Edit"` (not comma) into `Permission` by splitting and OR-ing individual `TryParse` results. Reject unknown tokens.

**Success criteria:** `"View Edit"` OK; `"View,Edit"` rejected or documented; unknown token returns clear error.
