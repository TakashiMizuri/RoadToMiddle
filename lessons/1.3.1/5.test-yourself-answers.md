## Answers

### 1. [Recall]
**Answer:** Open type содержит незаполненный type parameter (`Stack<T>`, `Dictionary<,>`). Closed constructed type — все параметры конкретны (`Stack<int>`, `Dictionary<string, int>`).
**Why:** Instantiate можно только closed type — компилятору нужен полный layout и проверки.

### 2. [Recall]
**Answer:** `T` должен иметь public parameterless constructor; внутри можно `new T()`.
**Why:** Отклонит abstract class, interface без default ctor, или type без accessible parameterless constructor.
**Common mistake:** Думать, что `new()` работает с private ctor — нужен **public** (или accessible) parameterless.

### 3. [Recall]
**Answer:** Value type → zero (0, false, default struct); reference type → `null`.
**Why:** `default(T)` — semantic zero для категории типа.

### 4. [Understanding]
**Answer:** `List<int>` хранит элементы в `int[]` — value inline, без heap object per element. `ArrayList` принимает `object` → `int` boxing'ится в отдельный heap object.
**Why:** Generic specialization + reification дают type-specific storage.

### 5. [Understanding]
**Answer:** Без constraints `T` treated as `object` для member access — нельзя вызвать `T`-specific methods, operators, constructors (кроме `new()` с constraint).
**Why:** Compiler не знает capabilities `T` до constraints.

### 6. [Understanding]
**Answer:** В generic class static members **per closed type**: `Cache<int>.Count` и `Cache<string>.Count` — разные variables в memory.
**Why:** CLR создаёт отдельный type object для каждого closed generic; statics привязаны к type, не к open definition.

### 7. [Understanding]
**Answer:** Reification — generic type info сохраняется в runtime (`typeof(List<int>)` реален). Java erasure — mostly compile-time; runtime видит raw types.
**Why:** Позволяет reflection, `typeof(T)`, different JIT code per value type.

### 8. [Application]
**Answer:**
```csharp
public static void Swap<T>(ref T a, ref T b) => (a, b) = (b, a);
```
Constraints не нужны — assignment и tuple swap работают для любого `T`.

### 9. [Application]
**Answer:**
```csharp
public interface IFactory<T> where T : class, new()
{
    T Create() => new T();
}
```
`Create()` на `IFactory<AbstractEntity>` не скомпилируется при использовании `AbstractEntity` как `T`, если abstract — нельзя `new AbstractEntity()`. Нужен concrete `T : AbstractEntity, new()`.

### 11. [Analysis]
**Answer:** С `where T : struct` (non-nullable value type constraint) `T?` означает **nullable value type** — OK в C# 8+. Потенциальные issues: `Convert.ChangeType` returns `object`, cast `(T?)` может fail runtime; для unconstrained `struct` без `Enum`/`IParsable` лучше `TryParse` pattern. Не «ошибка компиляции» per se в modern C# — вопрос на понимание nullable value types vs NRT.
**Why:** `struct` constraint excludes nullable reference types as `T`.

### 12. [Analysis]
**Answer:** Base class фиксирован at compile time; `T` unknown — CLR cannot lay out inheritance. Only interfaces/constraints allowed.

### 14. [What happens]
**Answer:** `False` (different Type objects); `True` for `List<>` as generic type definition.
**Why:** Reification gives distinct closed types.

### 15. [Understanding]
**Answer:** Когда аргументы не одного типа и нет общего inference (e.g. `Max(3, 3.14)` — ambiguous int vs double), или когда `T` не появляется в parameters (only return type).
**Why:** Inference flows from parameter types to type parameters.
