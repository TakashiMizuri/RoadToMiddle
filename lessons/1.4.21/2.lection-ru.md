# TaskScheduler.FromCurrentSynchronizationContext

> Roadmap: `1.4.21` · Node: `1.4` — C# async · Depth: понимание

## Learning Objectives

После урока ты сможешь:

- Объяснить **`SynchronizationContext`** и marshaling continuations на конкретный thread.
- Описать **`TaskScheduler.FromCurrentSynchronizationContext()`** и связь с **`Task.Factory.StartNew`**.
- Сопоставить **UI** (WinForms, WPF) и **ASP.NET Core** (нет request SyncContext).
- Связать **`ConfigureAwait(false)`** с capture context (**`1.4.8`**, **`1.4.9`**).
- Предсказать **deadlock** UI thread + **`.Result`**.
- Выбирать UI dispatch vs thread pool.

---

## Why This Matters

До **`async/await`** **`TaskScheduler.FromCurrentSynchronizationContext()`** возвращал work на **UI thread**. **`await`** автоматизирует capture **`SynchronizationContext.Current`** и **Post** continuation.

На UI без **`ConfigureAwait(false)`** continuation на UI — OK для **`button.Text`**, плохо для CPU на UI.

**ASP.NET Core:** **`SynchronizationContext.Current == null`** — continuations на **thread pool**. **`ConfigureAwait(false)`** в controllers менее критичен, в **shared libraries** — обязателен.

Middle dev понимает **«works in API, deadlocks in WPF»**.

## Core Concepts

### SynchronizationContext

Абстракция **куда выполнить delegate**: UI message loop, legacy ASP.NET HttpContext, или thread pool при **null**.

**`await`** с capture → **`Post`** continuation; **`ConfigureAwait(false)`** → pool.

### FromCurrentSynchronizationContext

```csharp
Task.Factory.StartNew(UpdateUi, CancellationToken.None,
    TaskCreationOptions.None,
    TaskScheduler.FromCurrentSynchronizationContext());
```

Legacy. Сегодня:

```csharp
await Task.Run(Compute);
label.Text = result; // UI if awaited on UI without ConfigureAwait(false)
```

### UI deadlock

UI thread **`.Result`** на async с continuation на UI → **deadlock**. Fix: **`async void`** handler + **`await`**, или library **`ConfigureAwait(false)`**.

### ASP.NET Core

**Null SyncContext** — UI-deadlock pattern редок; **starvation** от **`.Result`** (**`1.4.20`**).

## Examples

### WPF handler

```csharp
private async void Load_Click(object sender, RoutedEventArgs e)
{
    var data = await _api.FetchAsync();
    StatusLabel.Text = data;
}
```

### Library

```csharp
await _http.GetStringAsync(url).ConfigureAwait(false);
```

## Key Takeaways

- **SyncContext** = где resume после **await**
- **FromCurrentSynchronizationContext** — legacy
- **ASP.NET Core** — null, pool
- **Libraries** — **ConfigureAwait(false)**
- UI — never **`.Result`**
