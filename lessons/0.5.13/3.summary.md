# SEO: title, meta description, canonical, Open Graph — краткая выжимка

> `0.5.13` · ~8 мин на повторение

## Главная идея

Metadata в `<head>` — первый сигнал для **search crawlers** и **social preview fetchers**: что за страница, какой URL canonical, как выглядит карточка при шаринге. Без per-route title/OG React SPA часто «невидима» в выдаче и в Slack.

## Ключевые пункты

- `<title>` — один на документ; уникален per URL; ~60 символов; **Primary — Brand**.
- `meta name="description"` — hint для **snippet**; влияет на CTR, не magic ranking.
- `link rel="canonical"` — **absolute URL** preferred версии среди дублей (`?utm_`, `www`, trailing slash).
- **Open Graph**: `og:title`, `og:description`, `og:image`, `og:url`, `og:type`; атрибут **`property`**, не `name`.
- Social bots часто **не выполняют JS** — OG в server HTML / SSR.
- `robots noindex` ≠ canonical: первое исключает, второе указывает «индексируй другой URL».
- Structured data (JSON-LD) — дополнение, не замена title/description.

## Самые важные примеры

```html
<title>PostgreSQL Index Guide — DevDocs</title>
<meta name="description" content="B-tree, GIN, partial indexes with EXPLAIN examples.">
<link rel="canonical" href="https://example.com/blog/postgres-indexes">
<meta property="og:image" content="https://example.com/og/postgres.png">
```

## «Под капотом» в одном абзаце

Crawlers парсят `<head>` рано; Google может переписать title/snippet эвристиками. Canonical сверяется с redirects и sitemap — конфликты → clustering на «не тот» URL. OG fetchers кешируют head metadata; `og:image` должен быть публичным absolute URL.

## Частые ошибки

- Один title/description на все routes SPA.
- Relative canonical или `og:image`.
- Keyword stuffing; misleading description.
- OG только после client hydration.

## Мнемоника / правило большого пальца

> **Title** — «как вкладка и заголовок в Google»; **description** — «пitch в двух предложениях»; **canonical** — «главный адрес среди копий»; **OG** — «обложка ссылки в чате».
