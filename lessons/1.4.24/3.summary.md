# Task.Factory.StartNew — краткая выжимка

> `1.4.24` · ~10 мин на повторение

## Главная идея

**Task.Run** — безопасный default для queue на thread pool. **Task.Factory.StartNew** — мощный, но с footguns: wrong scheduler, **`Task<Task>`** при async lambda без **Unwrap()**.

## Ключевые пункты

- `StartNew(async () => await X())` → outer `Task` complete до конца `X`.
- Fix: **Unwrap()** или **Task.Run(async () => await X())** или просто `await X()`.
- **Task.Run** — CPU-bound offload, не замена async I/O.
- **LongRunning** — dedicated thread для blocking legacy.
- ASP.NET: `Task.Run(..., RequestAborted)` для CPU; I/O — `await`.
- Background work — **BackgroundService**, не fire-and-forget StartNew.

## Самые важные примеры

```csharp
// Broken
Task.Factory.StartNew(async () => await GetData());

// Fixed I/O
await GetData();

// Fixed CPU offload
await Task.Run(() => Parse(data), cancellationToken);
```

## «Под капотом» в одном абзаце

StartNew привязывает Task к конкретному TaskScheduler; async lambda возвращает Task как результат body → nested Task без Unwrap. Task.Run ставит work в global pool queue с предсказуемой семантикой для типичного app code.

## Частые ошибки

- StartNew «потому что так в старом коде».
- Task.Run вокруг sync HTTP.
- Ignore **Task<Task>** при code review.

## Мнемоника

**Run for routine; StartNew only with a written excuse.**
