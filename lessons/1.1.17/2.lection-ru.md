## Why This Matters

В предыдущих уроках enum уже встречался как читаемая замена magic numbers — `OrderStatus.Pending` вместо `0`. Это простейший случай: одно значение из небольшого взаимоисключающего набора. Реальные API редко ограничиваются этим. HTTP позволяет комбинировать заголовки. Права на файлы складывают read, write и execute. Политики авторизации в ASP.NET Core накладывают несколько требований одновременно.

Для таких доменов нужны **комбинируемые** значения, а не единственный выбор из списка. C# предоставляет **`[Flags]` enums**. Отдельно каждый query string, конфиг и JSON payload приходит как **текст**. Приложение должно превратить `"Completed"` или `"Read, Write"` в типизированные enum без падения на опечатке. **`Enum.TryParse`** — защитный шлюз между внешним вводом и внутренней логикой.

Middle-разработчик ежедневно использует enum в DTO, options classes и domain models. Ошибки во flags или parsing дают тонкие баги: лишние права, фильтры, которые никогда не срабатывают, или 500 на каждом malformed request. Этот урок связывает синтаксис enum с bitwise-механикой и parsing API, которые понадобятся в ASP.NET Core и EF Core mapping.

## Core Concepts

### Enum как типизированные целочисленные константы

**Enum** объявляет именованные константы на базе integral type — по умолчанию **`int`**, но можно указать `byte`, `short`, `uint`, `long` и др. В runtime значение enum **есть** это число с type-safe именем. Компилятор не даст присвоить произвольное число без явного cast — целый класс ошибок ловится на этапе компиляции.

```csharp
public enum OrderStatus
{
    Pending = 0,
    Paid = 1,
    Shipped = 2,
    Cancelled = 3
}
```

Когда ты пишешь `OrderStatus.Paid`, в IL хранится значение `1`. JSON-сериализация обычно отдаёт имя `"Paid"` (зависит от настроек) или число `1`. В БД часто хранят integer для компактности. Понимание underlying number важно при отладке raw SQL или network payload.

У non-flag enum обычно **уникальные последовательные значения**, начиная с нуля, если нет причины выровняться под внешний протокол. Default для необъявленного члена — ноль; первый именованный член часто явно **`None = 0`** или **`Unknown = 0`**, когда ноль несёт смысл beyond «не задано».

### Flag enum и атрибут [Flags]

**Flags enum** — bitmask: каждый член занимает свой bit, несколько членов активны одновременно. Помечаешь enum атрибутом **`[Flags]`** и назначаешь степени двойки, чтобы комбинации не коллидировали.

```csharp
[Flags]
public enum FileAccess
{
    None = 0,
    Read = 1,      // 0001
    Write = 2,     // 0010
    Execute = 4,   // 0100
    ReadWrite = Read | Write  // 0011
}
```

**`[Flags]`** сам по себе не меняет runtime behavior. Он сообщает **`ToString()`** рисовать `"Read, Write"` вместо непрозрачного `3`. То же используют debuggers и doc generators. Без `[Flags]` на combined value **`ToString()`** может вывести `"3"`, скрывая intent.

Комбинируешь flags через **bitwise OR** (`|`), проверяешь через **AND** (`&`). **XOR** (`^`) переключает flag; **NOT** (`~`) инвертирует bits — реже в application code.

```csharp
FileAccess access = FileAccess.Read | FileAccess.Write;

bool canRead = (access & FileAccess.Read) == FileAccess.Read;
bool canWrite = access.HasFlag(FileAccess.Write);
```

**`HasFlag`** удобен, но в некоторых путях аллоцирует и boxing; в hot loops предпочитай `(access & flag) == flag`. Для `[Flags]` enum явно определи **`None = 0`**, чтобы «нет прав» было однозначно.

### Правильный дизайн flag values

Правило для flags: **один bit на независимый flag** — `1 << n` или явные степени двойки. Составные имена вроде **`ReadWrite`** — optional sugar; bitmask работает и без них.

Типичная ошибка — sequential `1, 2, 3, 4` на flags enum: тогда `Read | Write` даёт `3`, которое может совпасть с другим членом. Степени двойки предотвращают коллизию. При расширении enum нельзя «переиспользовать» bit — это тема API versioning до v1.

В PostgreSQL flags часто хранят как **`int`** / **`bigint`** и мапят в EF Core. JSON может отдавать comma-separated string или array — знай контракт.

### Parsing строк в enum

Внешние данные — строки. **`Enum.Parse<T>(string)`** бросает **`ArgumentException`** при неизвестном имени — допустимо для internal constants, опасно для user input.

```csharp
OrderStatus status = Enum.Parse<OrderStatus>("Paid");
```

Overload с **`ignoreCase: true`** сопоставляет `"paid"` с **`Paid`**. В HTTP и config casing плавает — **`ignoreCase: true`** частый выбор.

**`Enum.TryParse<T>`** возвращает **`false`** вместо throw — паттерн на границе API.

```csharp
if (!Enum.TryParse<OrderStatus>(input, ignoreCase: true, out var status))
{
    return Results.BadRequest($"Unknown status: {input}");
}
```

Для flags **`TryParse`** принимает comma-separated names: `"Read, Write"` → **`Read | Write`**. Опечатка в одном сегменте валит весь parse, если не парсишь вручную — выбор strict vs lenient API.

**`Enum.IsDefined`** проверяет, объявлен ли член с таким именем или числом. Для flags **`IsDefined`** true только для **точного** named value, не для каждой логически valid комбинации. `(FileAccess)3` может быть **`Read | Write`**, но **`IsDefined(..., 3)`** false, если `3` не named constant.

### Enum.GetValues и итерация

**`Enum.GetValues<T>()`** возвращает все named members — dropdowns, OpenAPI enums. Для flags перебор **всех комбинаций** экспоненциален; UI обычно показывает отдельные flags, не каждое подмножество.

## Under the Hood

Enum компилируются в integral constants. **`OrderStatus.Paid`** в IL — **`ldc.i4.1`** и операции с типом **`OrderStatus`**. Heap allocation для самого enum value не происходит; это value type на stack или в поле struct.

**`[Flags]`** — metadata на типе enum. Reflection читает **`FlagsAttribute`** при formatting/validation. **`HasFlag`** реализует примерно `(value & flag) == flag`, но принимает **`Enum`**, что вызывает boxing struct enum.

**`Enum.TryParse`** обходит static field table типа enum, сравнивая имена (для flags — split по запятым). Не читает external config — только declared members. Для request parsing достаточно быстро; не парси одну строку тысячи раз без cache.

**`default(OrderStatus)`** — **`0`**: член с value zero, если есть; иначе unnamed zero — источник subtle bugs при deserialization.

## Syntax / Commands / API

```csharp
[Flags]
public enum Permissions : uint
{
    None = 0,
    View = 1 << 0,
    Edit = 1 << 1,
    Delete = 1 << 2
}

Permissions p = Permissions.View | Permissions.Edit;
bool hasEdit = (p & Permissions.Edit) != 0;

if (Enum.TryParse<Permissions>(text, true, out var perm)) { /* use perm */ }
Enum.IsDefined(typeof(Permissions), Permissions.View);
```

| API | Throws? | Когда |
|-----|---------|-------|
| `Enum.Parse<T>` | Да | Trusted input, fail-fast config |
| `Enum.TryParse<T>` | Нет | HTTP, user input, JSON |
| `HasFlag` | Нет | Читаемость; не hot paths |
| `Enum.IsDefined` | Нет | Exact member (не все combos) |

## Examples

### Пример 1: query parameter в Minimal API

```csharp
app.MapGet("/orders", (string? status) =>
{
    if (status is not null &&
        !Enum.TryParse<OrderStatus>(status, ignoreCase: true, out _))
    {
        return Results.ValidationProblem(new Dictionary<string, string[]>
        {
            ["status"] = [$"Valid values: {string.Join(", ", Enum.GetNames<OrderStatus>())}"]
        });
    }
    return Results.Ok();
});
```

Unknown string → validation error, не **500** от **`ArgumentException`**.

### Пример 2: combine и check flags

```csharp
LogLevelFilter active = LogLevelFilter.Information | LogLevelFilter.Error;
bool logsErrors = (active & LogLevelFilter.Error) == LogLevelFilter.Error;
```

**`ToString()`** → `"Information, Error"` благодаря **`[Flags]`**.

## Common Mistakes & Anti-patterns

**Sequential values на flags enum** — коллизии при `|`. Fix: **`1 << n`**.

**`Enum.Parse` на user input** — опечатка роняет pipeline. Fix: **`TryParse`**.

**`IsDefined` для combinations** — не покрывает все bitmasks. Fix: проверка bits или allow-list.

**Нет `None = 0`** — не отличить unset от первого flag. Fix: explicit **`None`**.

**`HasFlag` в tight loops** — boxing. Fix: **`&`**.

**Cast `(OrderStatus)99` из БД** — компилируется без runtime check. Fix: **`IsDefined`** или CHECK constraint.

## Production & Real-World Notes

OpenAPI часто описывает enums как strings; **`JsonStringEnumConverter`** внутри похож на **`TryParse`**. Имена enum — часть контракта; rename ломает clients.

При миграции enum **не перенумеровывай** — values в persisted schema. Новые members — новые bits/integers.

Analyzer **`CA1008`** предупреждает об отсутствии zero value на `[Flags]` enum.

## Comparison / Trade-offs

**Enum vs static readonly int** — type safety vs гибкость open set.

**`HasFlag` vs `&`** — readability vs micro-performance.

**String vs numeric в API** — self-documenting vs compact; часто string в JSON, int в БД.

```
VALIDATE:  IsDefined → только exact named member
```

## Key Takeaways

1. Enum — **integral value types** с именами; знай underlying type и zero default.
2. **`[Flags]`** + **степени двойки** + **`None = 0`**.
3. **`|`** для combine, **`&`** для test; в hot code без **`HasFlag`**.
4. **`TryParse`** на границах; **`Parse`** только для trusted input.
5. **`IsDefined`** ≠ validation всех flag combinations.
6. Numeric values enum — часть schema contract при persist.
