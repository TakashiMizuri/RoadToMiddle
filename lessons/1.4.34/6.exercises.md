# Exercises: BackgroundTaskQueue

> Complete after passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Registration
**Goal:** Write `Program.cs` DI lines for queue capacity 50 + hosted consumer.
**Success criteria:** Singleton queue, hosted service.

### E2. Closure Audit
**Goal:** Mark safe/unsafe captures in 3 enqueue lambdas (DbContext, int id, HttpRequest).
**Success criteria:** Unsafe = request-scoped/disposed.

## Medium

### M1. Full Mini-Implementation
**Goal:** Implement `BackgroundTaskQueue`, `QueuedHostedService`, one controller `Accepted()` enqueue.
**Success criteria:** Compiles; scope per item; try/catch log.

### M2. Bounded Stress
**Goal:** Describe behavior when 200 requests enqueue while capacity 10 — latency vs memory.
**Success criteria:** Mentions producer await (Wait mode).

## Hard (stretch)

### H1. N Consumers
**Goal:** Sketch N parallel hosted readers on same channel — ordering and idempotency notes.
**Success criteria:** Throughput vs ordering trade-off.
