# JIT Tier 0/1/2 — краткая выжимка

> `1.5.2` · ~8 мин

## Главная идея

**JIT** переводит IL в native при первом (или фоновом re) вызове. **Tiered compilation**: быстрый **Tier 0** для startup, **Tier 2** с full optimization для hot paths после достаточного числа вызовов.

## Ключевые пункты

- First call медленнее — compile + возможно static ctor.
- **Tier 0** → background **reJIT** → **Tier 2** (inlining, devirtualization).
- **OSR** — замена кода hot loop без exit метода.
- **R2R** — precompiled native в assembly; **Native AOT** — без JIT at runtime.
- Benchmarks и load tests требуют **warmup**.
- JIT использует **code heap** и CPU — видно при deploy/scale-out.

## Самые важные примеры

Cold ASP.NET pod: JIT pipeline + first HTTP request JIT controllers/JSON.

```bash
DOTNET_TieredCompilation=1  # default
dotnet-counters ... gen-0-gc-count  # не путать с JIT
```

## «Под капотом»

Call counters на entry; work queue на background JIT thread. clrjit pipeline с GC info для каждого native method body.

## Частые ошибки

- Benchmark без warmup = Tier 0 perf.
- `GC.Collect()` vs JIT confusion при latency spikes.

## Мнемоника

**Cold = Tier 0, Hot = Tier 2** — как прогрев двигателя.
