## Answers

### 1. [Recall]
**Answer:** Earliest commit в диапазоне good..bad, где test **bad**, а его parent (в bisect walk) был **good**.
**Why:** Это commit, чей patch впервые вводит регрессию в искомой линии.

### 2. [Recall]
**Answer:** **0**.
**Why:** Документация git bisect run: 0 = good.

### 3. [Understanding]
**Answer:** Bisect использует **binary search** — каждый шаг ~половина оставшихся commits.
**Why:** ⌈log₂(1000)⌉ ≈ 10.

### 4. [Understanding]
**Answer:** Иначе нет линейного диапазона «между» good и bad на одной ancestry-линии для поиска.
**Why:** Bisect partition'ит commits reachable от bad минус good.

### 5. [Understanding]
**Answer:** Bisect находит commit по **поведению test**; blame — **последнее изменение строки**, не обязательно причину бага.
**Why:** Разные задачи forensics.

### 6. [Application]
**Answer:** `git bisect start main v2.4.0` (или `start` затем `bad main` + `good v2.4.0`).
**Why:** bad = текущий сломанный tip, good = последний known-good tag.

### 7. [Application]
**Answer:** `git bisect reset`.
**Why:** Завершает bisect и возвращает HEAD на исходную ветку/commit.

### 8. [Application]
**Answer:** `git bisect skip`.
**Why:** Commit исключается из кандидатов; Git выбирает другой.

### 9. [Application]
**Answer:** `git bisect run npm test` (с нужными args).
**Why:** Automates good/bad classification по exit code.

### 10. [Analysis]
**Answer:** **Неверный** first bad commit или ambiguous result — классификация inconsistent.
**Why:** Bisect assumes deterministic good/bad signal.

### 11. [Analysis]
**Answer:** `HEAD detached at <commit>` (не на branch tip).
**Why:** Bisect checkout'ит конкретные commits.

### 12. [Understanding]
**Answer:** **Skip** этот commit (как `git bisect skip`).
**Why:** Специальный код для untestable revisions.

### 13. [Application]
**Answer:** Порядка **9** (⌈log₂(501)⌉).
**Why:** Binary search bound.

### 14. [Analysis]
**Answer:** **Да** — `-S` находит где текст появился/исчез; регрессия может быть в другом commit (config, dependency, interaction).
**Why:** Code presence ≠ failing test causality.

### 15. [Recall]
**Answer:** Любые три из: `git bisect bad`, `git bisect good`, `git bisect skip`, `git bisect run`, `git bisect log`.
**Why:** Core interactive/automation commands.
