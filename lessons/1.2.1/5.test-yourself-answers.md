## Answers

### 1. [Recall]
**Answer:** **Class** — определение типа (metadata): members, layout, methods. **Instance** — конкретный object на heap с собственными значениями fields, созданный в runtime (обычно `new`).
**Why:** Blueprint vs построенный объект — разные уровни абстракции.

### 2. [Recall]
**Answer:** **Reference** (managed pointer) на heap object типа `Customer`, не embedded body объекта.
**Why:** Class — reference type (`1.1.6`).

### 3. [Recall]
**Answer:** Переменная **не ссылается** ни на один instance; dereference без проверки → `NullReferenceException`.
**Why:** `null` = отсутствие target, не «пустой object».

### 4. [Understanding]
**Answer:** `b = a` копирует **reference** на **один** object; `Deposit` мутирует shared instance.
**Why:** Reference semantics — alias behavior.

### 5. [Understanding]
**Answer:** Оба копируют **reference**, не deep clone; две переменные → один mutable heap object.
**Why:** Custom class и `List<T>` — reference types с одинаковой assignment semantics.

### 6. [Understanding]
**Answer:** **Нет**, `false` (если не overridden `==`). Разные **identities** (разные `new`), хотя Id совпадает.
**Why:** Default reference equality сравнивает адреса/identity, не field values.

### 7. [Application]
**Answer:** **`mine.Balance` остаётся 100**. Reassignment параметра `acc` меняет только local slot callee.
**Why:** Pass-by-value копирует reference; `acc = new ...` не меняет caller's variable.

### 8. [Application]
**Answer:** **Два** heap objects (`a`/`b` share one, `c` — второй).
**Why:** Два вызова `new`; `b = a` не аллоцирует.

### 9. [Application]
**Answer:** **`False`**.
**Why:** Два distinct instances; default `==` — reference equality.

### 10. [Analysis]
**Answer:** **B** неверно. **`copy = customer`** aliases; новый object только при **`new`** или explicit clone.

### 11. [Analysis]
**Answer:** Class — **один тип** в metadata; **`new User()`** каждый раз создаёт **новый instance**. Много users — много objects на heap.
**Common mistake:** Путать type definition с singleton instance.

### 12. [Understanding]
**Answer:** Два `Wallet` objects: `w1` и `w2` → первый; `w3` → второй. Две группы alias.
**Why:** Два `new`, одно assignment alias.

### 13. [Understanding]
**Answer:** Чтобы **один tracked instance** на entity в context; изменения через любую reference попадают в один change tracker entry; duplicate instances с одним Id → inconsistent state.
**Why:** Reference identity drives UoW/identity map.

### 14. [Application]
**Answer:** **Да**, `order.Status` станет `"Shipped"`. Parameter получил **копию reference** на тот же object; мутация visible caller.
**Why:** Default pass-by-value for references still shares object.
