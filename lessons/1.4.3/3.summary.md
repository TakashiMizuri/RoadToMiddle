# Task class — краткая выжимка

> `1.4.3` · ~10 мин

## Главная идея

**Task** — handle операции со **Status** (Running, RanToCompletion, Faulted, Canceled). Exception хранится на faulted task до **observation** через **`await`**. **`Wait()`/`.Result`** блокируют и дают **AggregateException**.

## Ключевые пункты

- **`IsCompleted`** — true для success, fault, cancel.
- **`IsCompletedSuccessfully`** — только success.
- **`await`** unwraps inner exception + preserves stack.
- Fire-and-forget `_ = WorkAsync()` — unobserved fault risk.
- **`Task.WhenAll`**: несколько fault → aggregate; await — first only.

## Примеры

```csharp
try { await task; } catch { /* observed */ }
// task.Wait(); // ✗ block
```

## «Под капотом»

ExceptionDispatchInfo сохраняет stack. Continuations queued on transition to final state.

## Частые ошибки

- Игнор Task (CS4014).
- `.Result` в ASP.NET request.
- Путать canceled и faulted.
