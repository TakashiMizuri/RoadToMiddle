## Answers

### 1. [Recall]
**Answer:** `bool`.
**Why:** C# не допускает implicit conversion чисел или указателей в boolean context для `if` — только явное bool-выражение.

### 2. [Recall]
**Answer:** Завершает выполнение текущего `switch`; без `break` classic switch не скомпилируется (кроме редких `goto case`).
**Why:** C# намеренно запрещает accidental fall-through из C.

### 3. [Understanding]
**Answer:** `&&` short-circuit: при `name == null` правая часть `name.Length` не вычисляется.
**Why:** Short-circuit — часть семантики языка; guard слева защищает справа.

### 4. [Understanding]
**Answer:** Classic — statement с `case`/`break`, часто mutates state. Switch expression — **expression**, возвращает value; поддерживает современные patterns.
**Why:** Expression form удобен для assign/return и exhaustive tables.

### 5. [Understanding]
**Answer:** Score 95 сначала попадёт в `>= 80` и никогда не дойдёт до `>= 90`.
**Why:** Выполняется только первая true ветка — порядок = приоритет правил.

### 6. [Application]
**Answer:** `switch` на `string` или switch expression — три discrete string values, таблица решений читаема.
**Why:** Для enum/string констант switch яснее цепочки `if`. Допустим и `if/else if`.
**Common mistake:** `switch` без `default`/`_` при неизвестных строках — silent skip.

### 7. [Application]
**Answer:** `Work`.
**Why:** Wednesday не weekend и не Monday → discard `_`.

### 8. [Analysis]
**Answer:** `case 0.3` почти никогда не сработает — `0.1 + 0.2 != 0.3` в binary floating point.
**Why:** Exact equality на `double` в switch ненадёжен; используй epsilon, integer cents или ranges.

### 9. [Analysis]
**Answer:** **B** неверно.
**Why:** C# **требует** `break`/`return`/`goto case`; fall-through как в C по умолчанию запрещён.

### 10. [Understanding]
**Answer:** Ternary — простой выбор двух values в одной строке. `if/else` — несколько statements, сложная логика, читаемость при вложенности.
**Why:** Ternary — expression; не замена полноценным блокам.

### 11. [Application]
**Answer:** Пример:

```csharp
string label = statusCode switch
{
    200 => "OK",
    400 or 404 => "Error",
    >= 500 and < 600 => "Server",
    _ => "Unknown"
};
```

**Why:** Relational + `or` покрывают диапазоны и несколько codes.

### 12. [Understanding]
**Answer:** Проверяет runtime type `string` и связывает matched value с переменной `s` для использования в arm.
**Why:** Type pattern = test + bind в одном шаге.

### 13. [Recall]
**Answer:** Logical OR двух constant patterns — match если day любой из двух.
**Why:** C# 9+ property pattern combinator для краткости.

### 14. [Application]
**Answer:** Early **`return`** из `Main` (или метода) после сообщения об ошибке; опционально `else` для valid path.
**Why:** Guard clause + `TryParse` — idiomatic без exceptions на user input.
