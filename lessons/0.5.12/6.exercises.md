# Exercises: Skip Links, Focus Visible, Contrast

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Add a skip link
**Goal:** Implement bypass block.

Create `a11y-page.html` with header nav (5+ links), `<main id="main" tabindex="-1">` with paragraph content, and skip link as first body child.

Style skip link visible on `:focus` only.

Tab from load — verify skip appears and second Tab moves into nav or main as expected after activating skip.

**Success criteria:** Working hash + focus on main; skip visible when focused.

---

### E2. Fix focus ring
**Goal:** Replace outline removal.

Given CSS:

```css
button { outline: none; }
```

Add `:focus-visible` rule meeting visible 2px indicator without breaking mouse UX unnecessarily.

**Success criteria:** Keyboard Tab shows ring; document chosen colors.

---

## Medium

### M1. Contrast audit
**Goal:** Measure and fix.

Pick three text/background pairs from a real site or mock (body text, secondary text, primary button). Record contrast ratios via DevTools or WebAIM.

Fix any pair failing AA for its text size category.

**Success criteria:** Before/after ratios documented; all pass AA for category used.

---

### M2. Color + text state
**Goal:** Not color alone.

Table row error state: implement with red left border **and** visible text "Error: …" or icon with `aria-label`.

**Success criteria:** Passes 1.4.1 conceptually; error understandable with CSS color disabled/grayscale filter test.

---

## Hard (stretch)

### H1. Design token focus + contrast
**Goal:** Production-minded spec.

Define CSS custom properties: `--focus-ring-color`, `--text-primary`, `--surface` such that focus ring has **3:1** against adjacent surface **and** body text has **4.5:1**.

Write 5-line comment block explaining token choices for designers.

**Success criteria:** Math or DevTools proof for both requirements.
