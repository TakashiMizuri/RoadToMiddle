## Answers

### 1. [Recall]
**Answer:** Любые 3 из: Endpoints, Services, EF Core, HttpClient, Middleware/Filters, Background, DI, CI/Tooling.
**Why:** Eight checklist categories in lection.

### 2. [Recall]
**Answer:** `\.(Result|Wait\(|GetAwaiter\(\)\.GetResult\()` (или эквивалентный поиск `.Result` и `.Wait(`).
**Why:** Standard sync-over-async markers.

### 3. [Understanding]
**Answer:** Нет — CPU-only методы могут быть sync; требование — **не блокировать на I/O** по цепочке к DB/network.
**Why:** Async all-the-way ≠ async every method.

### 4. [Understanding]
**Answer:** Блокирует pool thread на sync/blocking work вместо true async EF; violates Endpoints + EF categories.
**Why:** Fake async offload.

### 5. [Understanding]
**Answer:** Propagate `RequestAborted` — отмена при disconnect клиента, экономия ресурсов DB/HTTP.
**Why:** Endpoints checklist item.

### 6. [Application]
**Answer:** Пример: (1) grep clean (2) trace 3 endpoints (3) no sync EF (4) HttpClient factory (5) load test thread pool / (6) background scope rules / (7) PR allowlist reviewed.
**Why:** Audit workflow from lection.

### 7. [Application]
**Answer:** Services (sync-over-async), DI/Singleton (blocking in singleton called per request), Endpoints indirect if controller awaits singleton method.
**Why:** Blocking in singleton amplifies under concurrency.

### 8. [Analysis]
**Answer:** Unobserved exception, lost errors, request completes before email; category **Background** — need queue/hosted service + scope.
**Why:** Fire-and-forget anti-pattern (`1.4.30`).

### 9. [Analysis]
**Answer:** Allowlist документирует временные legacy exceptions с ticket; без governance — regressions и вечный debt.
**Why:** Pragmatic strictness trade-off.

### 10. [Understanding]
**Answer:** Grep не ловит sync `.ToList()` без `.Result`, runtime thread pool issues, misconfigured pool sizes, hidden library blocks.
**Why:** Dynamic validation needed.

### 11. [Application]
**Answer:** **Нет** — нарушает Services/EF: DbContext not thread-safe for parallel ops.
**Why:** Scoped ≠ parallel safe.

### 12. [Understanding]
**Answer:** `InvokeAsync` returns Task; `await next(context)`; async auth if I/O; no blocking in middleware body.
**Why:** Category 5 table.
