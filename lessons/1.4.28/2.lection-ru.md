## Why This Matters

Downstream падает — transient blips, **503** при deploy. Blind retries → **retry storm**. **Circuit breaker** — fail fast и recovery time для sick dependency.

**Polly** + **.NET 8** **Microsoft.Extensions.Http.Resilience** интегрируются с factory (`1.4.27`). Middle dev выбирает policy осознанно, не copy-paste.

---

## Core Concepts

### Retry

Transient: **HttpRequestException**, **408**, **429**, **502**, **503**, **504**. **Jitter** — anti thundering herd. Retry **idempotent** ops (GET) или **Idempotency-Key** на POST.

### Timeout

| Layer | Scope |
|-------|--------|
| HttpClient.Timeout | Whole op incl. retries |
| Polly per-attempt | Single try |
| CancellationToken | Client disconnect |

Часто HttpClient.Timeout = **Infinite** если Polly control per-try.

### Circuit breaker

**Closed → Open → Half-Open → Closed/Open**. Fail fast при open; metric state.

### Standard Resilience Handler

```csharp
builder.Services.AddHttpClient("catalog")
    .AddStandardResilienceHandler(options =>
    {
        options.Retry.MaxRetryAttempts = 3;
        options.TotalRequestTimeout.Timeout = TimeSpan.FromSeconds(30);
    });
```

---

## Under the Hood

**ResiliencePipeline** в **DelegatingHandler**. Retry = новый SendAsync через chain. **BrokenCircuitException** при open.

---

## Common Mistakes & Anti-patterns

Retry 400/404. No jitter. High parallelism × many retries. Non-idempotent POST retry.

---

## Key Takeaways

- Retry transient only + jitter + cap.
- Breaker protects downstream и pool.
- Align HttpClient.Timeout с Polly.
- Parallelism × retries = load multiplier.
- **AddStandardResilienceHandler** в .NET 8+.
