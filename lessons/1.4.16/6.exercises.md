# Exercises: IAsyncEnumerable — Async Streams

> Complete after passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Hello async stream
**Goal:** Implement and consume a trivial async iterator.

**Steps:** Write `async IAsyncEnumerable<int> CountAsync(int from, int to)` that yields each integer, awaiting `Task.Delay(10)` between yields. Consume with `await foreach` and print.

**Success criteria:** Output order correct; uses `await foreach`.

### E2. Early termination
**Goal:** Use `yield break`.

**Steps:** Extend E1 to accept `maxCount`; stop early with `yield break` when yielded count reaches limit.

**Success criteria:** Fewer items than range when `maxCount` is small.

## Medium

### M1. Paginated stream
**Goal:** Model DB paging without loading all rows.

**Steps:** Given `Task<IReadOnlyList<Product>> GetPageAsync(int page, CancellationToken ct)`, implement `StreamProductsAsync` returning `IAsyncEnumerable<Product>` with `[EnumeratorCancellation]`.

**Success criteria:** Empty page ends stream; cancellation passed to `GetPageAsync`.

### M2. Cancellation demo
**Goal:** Observe `[EnumeratorCancellation]`.

**Steps:** Consumer starts `await foreach` on slow stream, cancels after 3 items via `CancellationTokenSource.CancelAfter`. Verify iterator exits (log or exception type).

**Success criteria:** Producer stops; no infinite loop after cancel.

## Hard (stretch)

### H1. NDJSON sketch
**Goal:** Connect to HTTP streaming mental model.

**Steps:** Pseudocode ASP.NET endpoint that `await foreach` orders and writes one JSON object + newline per item to response body. List headers/settings to avoid full buffering.

**Success criteria:** Mentions cancellation and content type.
