# Answers: Conventional Commits

## Scoring

- 90–100%: отлично
- 80–89%: pass
- <80%: повтори лекции

---

## Answers

### 1. [Recall]
**Answer:** `<type>[optional scope][optional !]: <description>`

**Why:** Базовая структура спецификации conventionalcommits.org.

---

### 2. [Recall]
**Answer:** `feat` → **MINOR**; `fix` → **PATCH**. Breaking → **MAJOR**.

**Why:** SemVer automation convention.

---

### 3. [Understanding]
**Answer:** **`feat`** — новая user/API capability (`feat(auth): add OAuth login`). **`chore`** — обслуживание без новой capability (`chore(deps): update Newtonsoft.Json`).

**Why:** chore не должен триггерить feature release notes.

---

### 4. [Understanding]
**Answer:** (1) **`!`** после type/scope в header. (2) Footer **`BREAKING CHANGE: описание`**.

**Why:** Оба сигнала парсятся tooling; footer для длинных migration notes.

---

### 5. [Application]
**Answer:** Например: `fix(auth): reject expired JWT access tokens`

**Why:** type fix (исправление поведения), scope, imperative, конкретика.

---

### 6. [Application]
**Answer:** `docs(readme): document local Docker setup` (или `docs: ...` без scope).

**Why:** Только документация — не feat/fix.

---

### 7. [Analysis]
**Answer:** **C** — past tense «fixed», нет valid type prefix в lowercase conventional form.

**Why:** A, B, D — валидные примеры.

---

### 8. [Analysis]
**Answer:** **Два unrelated change** в одном commit (feature + docs typo); нарушение atomic commits. Нужно split: `feat(users): add field` и отдельно `docs(users): fix typo in README`.

**Why:** Changelog и bisect страдают.

---

### 9. [Understanding]
**Answer:** Согласованность с Git-generated messages и правилом «subject дополняет: If applied, this commit will **…**».

**Why:** Единый стиль в log.

---

### 10. [Understanding]
**Answer:** Squash title часто = один conventional header для всего PR; reviewers видят тип изменения в merged commit на `main`.

**Why:** Main history остаётся scannable после squash policy.

---

### 11. [Application]
**Answer:**
```
feat(api)!: remove GET /api/v1/legacy-orders

BREAKING CHANGE: Legacy orders endpoint removed. Clients must migrate to /api/v2/orders.
```

**Why:** `!` + footer для MAJOR и migration clarity.

---

### 12. [Understanding]
**Answer:** Документирует **откат** конкретного feat; помогает automation и audit понять, что feature снята, а не случайный random commit.

**Why:** Revert — осознанный undo в history narrative.

---

### 13. [Application]
**Answer:** (1) Lowercase **type** `fix` не `Fix`. (2) Формат `fix(scope): description` — добавить scope при необходимости, imperative description лучше чем vague «Missing semicolon» без контекста.

**Why:** commitlint проверяет regex header.

---

### 14. [Understanding]
**Answer:** Пока commits **только локальные** или branch не стянут другими — amend/reword безопасен (`0.1.10`). После push на shared PR branch — осторожно; после merge — только новый commit/revert.

**Why:** Golden rule public history.
