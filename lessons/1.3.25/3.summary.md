# LINQ Distinct, Union, Intersect, Except — краткая выжимка

> `1.3.25` · ~8 мин

## Главная идея

LINQ set operators (`Distinct`, `Union`, `Intersect`, `Except`) работают с **ordered sequences**, но ведут себя как **sets**: без duplicates, с сохранением **first-seen order** (обычно от первой/левой sequence).

## Ключевые пункты

- **`Distinct()`** — default equality; reference types без `Equals` не dedupe by value → **`DistinctBy(key)`**.
- **`Union`** = merge без duplicates; для сохранения всех элементов — **`Concat`**.
- **`Intersect`** — common elements, по одному разу, order первой sequence.
- **`Except(a, b)`** — in A not in B; **не symmetric**.
- Внутри — **HashSet**, O(n) time, O(unique) memory.
- EF: `Union` → SQL `UNION`, `Except` → `EXCEPT` / anti-join — проверяй SQL.

## Самые важные примеры

```csharp
ids.Distinct();
people.DistinctBy(p => p.Email);
a.Union(b);
editorPerms.Except(adminPerms);
rolesA.Intersect(rolesB);
```

## «Под капотом» в одном абзаце

Iterator blocks с backing **HashSet**: при enumeration каждый element (или key) проверяется «видели ли уже»; `Union` проходит first, потом second, yield только новые. На `IQueryable` provider переводит в SQL set operations — semantics duplicates/order на server могут отличаться от in-memory edge cases.

## Частые ошибки

- `Distinct()` на objects без value equality.
- Путать `Union` и `Concat`.
- Перепутать порядок operands в `Except`.
- Side effects при multiple enumeration composed pipeline.

## Мнемоника

**Union** объединяет уникальные; **Intersect** — пересечение; **Except** — «вычесть второе из первого» (порядок важен).
