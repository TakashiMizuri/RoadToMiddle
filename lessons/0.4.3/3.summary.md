# TCP — краткая выжимка

> `0.4.3` · ~10 мин

## Главная идея

**TCP** — reliable ordered **byte stream** поверх best-effort IP. **Three-way handshake** (SYN → SYN-ACK → ACK) до HTTP/TLS (`0.3`). **Keep-Alive** (`0.3.14`) переиспользует один TCP, экономя RTT и slow start.

## Ключевые пункты

- Sequence + ACK + retransmit = reliability.
- **Receive window** — flow control (медленный reader → sender pause).
- **Congestion control** — slow start, decrease on loss.
- FIN graceful close; RST abort.
- TIME_WAIT — 2×MSL после close.
- Cold HTTPS ≈ 1 RTT TCP + 1 RTT TLS до HTTP.

## Примеры

```
SYN timeout → firewall/routing/no listener
RST on connect → port closed (0.4.2)
No Keep-Alive → P99 растёт от repeated handshakes
```

## «Под капотом»

State machine: LISTEN → SYN_* → ESTABLISHED → FIN_WAIT/TIME_WAIT. SYN backlog и accept queue — drops под burst до app. Fast retransmit на 3 dup ACK.

## Частые ошибки

- TCP connection ≠ один HTTP request.
- LB idle timeout vs Keep-Alive mismatch.
