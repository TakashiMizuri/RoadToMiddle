# IP, IPv4, ports, sockets — краткая выжимка

> `0.4.2` · ~8 мин

## Главная идея

**IP address** (internet layer) — какой **host**. **Port** (transport layer) — какой **process/service** на host. **Socket** — OS endpoint `(IP, port, protocol)` для приложения. URL host/port из `0.3.2` после DNS становятся числами для TCP connect.

## Ключевые пункты

- IPv4: 32 bit, dotted decimal; private: `10/8`, `172.16/12`, `192.168/16`.
- CIDR `/24` — network prefix vs host bits.
- Well-known ports: 80, 443, 5432; client — ephemeral ports.
- **5-tuple:** `(src IP, src port, dst IP, dst port, TCP/UDP)`.
- SOCK_STREAM = TCP; SOCK_DGRAM = UDP.
- `EADDRINUSE` — port занят; refused — никто не listen.

## Примеры

```
https://api.shop.com/orders  →  DNS → 198.51.100.22:443  →  TCP  →  TLS  →  HTTP
Docker -p 8080:80  →  host:8080 maps to container:80
```

## Частые ошибки

- IP vs port vs MAC.
- Bind `127.0.0.1` и ждать доступ с LAN.
- Забыть `:443` в firewall для HTTPS.

## Мнемоника

> **IP = здание, port = квартира, socket = дверной звонок API.**
