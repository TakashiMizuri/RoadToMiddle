# Таблицы: thead, tbody, th scope — краткая выжимка

> `0.5.9` · ~8 мин на повторение

## Главная идея

`<table>` — для **табличных данных**, где ячейка имеет смысл через заголовок строки **и** столбца. Структура `thead`/`tbody`/`tfoot`, `<th>` с **`scope`**, и **`<caption>`** дают browser и screen reader правильную модель сетки.

## Ключевые пункты

- Layout-таблицы — антипаттерн; для колонок — **CSS Grid/Flexbox**.
- `<th>` — заголовок; `<td>` — данные; жирный `<td>` ≠ заголовок для AT.
- **`scope="col"`** — заголовок столбца; **`scope="row"`** — заголовок строки.
- **`<caption>`** — одно краткое имя таблицы, сразу после `<table>`.
- Сложные merged headers — **`headers`** + уникальные **`id`** на `<th>`.
- `colspan`/`rowspan` не отменяют связь заголовков с данными.

## Самые важные примеры

```html
<table>
  <caption>Monthly active users by plan</caption>
  <thead>
    <tr>
      <th scope="col">Plan</th>
      <th scope="col">Users</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Free</th>
      <td>12,400</td>
    </tr>
  </tbody>
</table>
```

## Частые ошибки

- Таблица для вёрстки лендинга.
- Нет `<thead>` или все ячейки — `<td>`.
- Статус только цветом без текста.
- Div с `display:table` без семантики `<table>`.

## Мнемоника

> **Caption, scopes, real headers** — назови таблицу, свяжи оси через scope, не притворяйся `<td>` заголовком.
