# Exercises: Postman / Bruno / curl — Manual API Testing

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. httpbin echo ladder
**Goal:** Practice GET, POST JSON, and headers with curl.

**Steps:**
1. `curl https://httpbin.org/get?name=demo`
2. POST JSON to `https://httpbin.org/post` with `Content-Type: application/json` and body `{"ok":true}`.
3. Repeat POST with `-i` and note status + `Content-Type` response header.

**Success criteria:** You can read status line and locate echoed JSON in response body.

---

### E2. Verbose TLS peek
**Goal:** Read verbose output.

**Steps:**
1. Run `curl.exe -v https://httpbin.org/get` (Windows) or `curl -v` on Unix.
2. Identify: HTTP version used, one request header you did not set explicitly, status code.

**Success criteria:** Explain what `-v` added beyond silent curl.

---

## Medium

### M1. Environment-style variables in shell
**Goal:** Mimic Postman environments without GUI.

**Steps:**
1. Set `BASE=https://httpbin.org` and `NAME=manual-test` in shell.
2. `curl "$BASE/get?tag=$NAME" -H "Accept: application/json"`
3. Write one-line script (bash or PowerShell) that changes `BASE` and reuses same path.

**Success criteria:** Same curl template, two hosts/params via variables only.

---

### M2. Multipart upload to httpbin
**Goal:** Match `0.3.11` multipart with curl.

**Steps:**
1. Create small text file `note.txt`.
2. `curl -F "title=Exercise" -F "file=@note.txt" https://httpbin.org/post`
3. Confirm `files` and `form` sections in JSON response.

**Success criteria:** No manual `Content-Type` header (curl sets boundary).

---

### M3. Cookie jar flow
**Goal:** Session simulation.

**Steps:**
1. POST urlencoded to `https://httpbin.org/cookies/set?session=abc123` (follow redirects if needed with `-L`).
2. GET `https://httpbin.org/cookies` with same cookie jar (`-c` / `-b`).

**Success criteria:** Second response shows cookie `session`.

---

## Hard (stretch)

### H1. Bruno or Postman mini-collection
**Goal:** Document three requests in a shareable form.

**Tasks:**
1. Install Bruno or Postman.
2. Create collection: `GET health`, `POST json echo`, `POST multipart` (httpbin targets OK).
3. Add environment variable `baseUrl`.
4. If Bruno — commit folder to a scratch repo; if Postman — export JSON (redact secrets).

**Success criteria:** Teammate could import and run without retyping URLs.

---

### H2. Debug checklist card
**Goal:** Production-minded habit.

**Tasks:**
1. Write markdown checklist: reproduce with curl → compare headers to spec → add correlation ID → check status vs body → save working request.
2. Apply to one failing scenario (real or simulated wrong `Content-Type`).

**Success criteria:** Checklist ≤10 bullets, actionable.

---
