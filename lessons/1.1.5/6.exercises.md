# Exercises: var vs Explicit Types

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. var or explicit?
**Goal:** Apply the readability rubric.

For each declaration, choose **`var`** or an **explicit type** and write one-sentence justification:

```csharp
// 1
??? queue = new Queue<string>();

// 2
??? amount = 150.00m;

// 3
??? result = ParseConfig(path);

// 4
??? flag = true;
```

Assume `ParseConfig` returns `AppSettings` defined in another assembly.

**Success criteria:** 4/4 with reasoning tied to "obvious RHS" vs "hidden contract".

---

### E2. Fix the inference bug
**Goal:** Connect `var` to primitive literals (`1.1.3`).

The following code should store a file size of **5_000_000_000** bytes in a **`long`**. Fix using either explicit type or correct literal suffix:

```csharp
var fileSize = 5_000_000_000;
```

**Success criteria:** Compiles; variable type is `long` (verify in IDE).

## Medium

### M1. Team style guide snippet
**Goal:** Write actionable team rules, not dogma.

Draft 5 bullet rules for when your team uses `var` vs explicit types. Include at least one rule for: generics construction, method returns, numeric literals, public API, LINQ anonymous projections.

**Success criteria:** Rules are consistent with the lecture rubric; no absolute "always var" or "never var".

---

### M2. Refactor for readability
**Goal:** Improve a micro-snippet.

Refactor this block — use `var` where it helps and explicit types where they carry meaning. Do not change behavior:

```csharp
Dictionary<string, int> wordCounts = new Dictionary<string, int>();
decimal taxRate = 0.08m;
CustomerDto customer = LoadCustomer(id);
List<string> lines = new List<string>();
bool isActive = customer.Status == AccountStatus.Active;
```

**Success criteria:** Each choice defensible in code review; no numeric inference traps.

## Hard (stretch)

### H1. When inference fails
**Goal:** Understand compiler limits.

Explain why this does not compile and show **two different fixes**:

```csharp
var value = condition ? 100 : 100L;
var mixed = condition ? 1 : "one";
```

**Success criteria:** Correct error code or message for `mixed`; fixes use explicit type or cast appropriately.
