# localhost, 127.0.0.1, binding 0.0.0.0

> Roadmap: `0.4.6` · Node: `0.4` — Основы сетей · Depth: понимание

## Learning Objectives

- Объяснить **loopback** (`127.0.0.1`, `::1`) — traffic не leaves host.
- Различить **`localhost` hostname** и numeric loopback; когда что resolves.
- Что **`0.0.0.0` / `[::]`** значат в **bind/listen**, не в client destination.
- Настроить Kestrel/Docker для local-only vs LAN dev.
- Связать loopback с DNS (`0.4.5`), sockets (`0.4.2`), HTTP dev URLs (`0.3.2`).
- Избегать bind loopback behind proxy; не использовать `0.0.0.0` в browser URL.

---

## Why This Matters

Kestrel `https://localhost:7150`, React `fetch('http://localhost:5000/api')` — **`127.0.0.1` loopback** в OS stack (`0.4.1`). Packets не hit Wi‑Fi; kernel loops to local socket (`0.4.2`). Offline works; **bind** определяет **кто может connect**.

**`0.0.0.0`** в launchSettings/Docker — **bind all IPv4 interfaces**, не «address zero в internet». Misread → teammates не reach dev server; container listen loopback only.

HTTP/TCP (`0.3`–`0.4.3`) target `(IP, port)`. Loopback vs `0.0.0.0` bind vs LAN IP — Docker publish, phone on Wi‑Fi, K8s probes. `ERR_CONNECTION_REFUSED` часто bind mismatch.

## Core Concepts

### Loopback

**127.0.0.0/8** reserved; **`127.0.0.1`** conventional. IPv6 **`::1`**. TCP to `127.0.0.1:5000` — local delivery (`0.4.3`). No physical NIC.

Remote machines **не** reach твой `127.0.0.1` — «this host» locally.

### localhost — имя

Hosts file:
```
127.0.0.1 localhost
::1       localhost
```
`getaddrinfo` → loopback. Modern OS may prefer **IPv6 `::1`** — «works with 127.0.0.1 not localhost» if server IPv4-only bind.

### Bind vs connect

**Bind (listen):**

| Address | Meaning |
|---------|---------|
| `127.0.0.1:5000` | Same machine only |
| `0.0.0.0:5000` | All IPv4 interfaces |
| `[::]:5000` | All IPv6 |
| `192.168.1.50:5000` | Specific NIC |

**Connect:** specific reachable IP/name — **never `0.0.0.0` destination**.

`bind()` / `connect()` из `0.4.2`.

### Kestrel

`"applicationUrl": "https://localhost:7150"` → loopback. Teammate: `http://192.168.1.50:5150` + **`ListenAnyIP`** or specific IP, не только 127.0.0.1.

### Docker

Container `listen 0.0.0.0:80`; `-p 8080:80` maps host. Kestrel **only 127.0.0.1** inside container — publish fails.

### HTTP dev (`0.3`)

React `:5173` → API `:5000` — both loopback same machine. **Different ports = different origins** (CORS preview `0.4.10`). Phone test — LAN IP + bind + firewall. Dev cert for **`localhost`** (`0.3.8`), не IP usually.

## Examples

Phone refused, local OK — loopback bind. Log `Listening on http://0.0.0.0:8080` — use `localhost:8080` in browser.

## Common Mistakes

Browser to `http://0.0.0.0`. Loopback bind in K8s. localhost vs IPv6 bind. `0.0.0.0` on public Wi‑Fi без auth.

---

## Production

Kestrel 127.0.0.1 behind nginx same host. Pod `0.0.0.0:8080`; Service routes. Probes — pod IP not node loopback.

---

## Quick Reference

127.0.0.1/::1 loopback; localhost → hosts; 0.0.0.0 bind all IPv4 listen only.

---

## Key Takeaways

- Loopback — same machine traffic.
- localhost usually loopback; watch IPv4/IPv6 bind.
- 0.0.0.0 listen ≠ client URL.
- Remote access — non-loopback bind + firewall + correct IP in URL.
- Docker/K8s — bind broadly in namespace.
