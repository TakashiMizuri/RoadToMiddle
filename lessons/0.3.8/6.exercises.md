# Exercises: HTTPS and TLS

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Term matching
**Goal:** Vocabulary.

Match term to definition:

1. SNI
2. SAN
3. CA
4. HSTS
5. Forward secrecy

Definitions (shuffle): hostname in ClientHello; list of valid DNS names in cert; forces HTTPS; signs certificates; past sessions safe if long-term key stolen.

**Success criteria:** 5/5 correct.

---

### E2. True or false
**Goal:** Fix myths.

1. HTTPS encrypts URL path and query from client's ISP (always completely hidden).
2. TLS and SSL are the same; SSL 3.0 is fine for APIs today.
3. After TLS handshake, HTTP status codes work the same as in lesson 0.3.6.
4. A valid HTTPS connection guarantees the site is not phishing.

**Success criteria:** 4/4 with nuance on (1) metadata/SNI and (4) domain verification.

---

## Medium

### M1. Certificate failure triage
**Goal:** Map errors to fixes.

For each browser/error scenario, write likely cause + remediation:

1. `ERR_CERT_AUTHORITY_INVALID`
2. `ERR_CERT_COMMON_NAME_INVALID`
3. `ERR_CERT_DATE_INVALID`
4. Site works in Chrome, fails on Android app

**Success criteria:** Chain/trust, SAN mismatch, expiry renewal, incomplete chain mentioned.

---

### M2. Architecture sketch
**Goal:** TLS termination.

Draw ASCII: Internet Client → Azure App Gateway (HTTPS) → VM nginx (HTTP) → Kestrel. Mark where certificates live and where HTTP bytes are encrypted.

Write one sentence on required forwarded header.

**Success criteria:** Cert on gateway/nginx; cleartext hop labeled; X-Forwarded-Proto noted.

## Hard (stretch)

### H1. Dev vs prod cert policy
**Goal:** Operational maturity.

Write a short policy (bullet prose, 6–8 sentences): dev self-signed/dev-certs, staging Let's Encrypt, prod managed renewal, monitoring, minimum TLS version, HSTS rollout steps.

**Success criteria:** No dev certs in prod; expiry monitoring; TLS 1.2+ stated.

---

### H2. Threat modeling brief
**Goal:** Connect TLS to headers.

Scenario: coffee shop Wi‑Fi, login API uses HTTP with JSON body containing password and returns `Set-Cookie` session without Secure.

List what attacker can do without TLS vs with HTTPS only (no HttpOnly/SameSite yet — preview 0.3.9).

**Success criteria:** Cleartext theft on HTTP; HTTPS protects wire but mentions cookies need more attributes.
