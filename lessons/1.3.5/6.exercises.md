# Exercises: Events — Memory Leak Pattern

> Complete after passing `4.test-yourself.md`.

## Easy

### E1. Event vs Field
**Goal:** Two classes — one with `public Action? OnDone` field, one with `event Action? Done`. Try external `= null` on both — document compile result.
**Success criteria:** Explain encapsulation difference.

### E2. Dispose Unsubscribe
**Goal:** `Cache` with `event Action Invalidated`; `Viewer` subscribes in ctor, unsubscribes in `Dispose`.
**Success criteria:** After Dispose + null refs, Viewer collectable (conceptual / comment).

## Medium

### M1. Leak Reproduction (Console)
**Goal:** Static `AppEvents.Message` + short-lived `Listener` without Dispose — show handler still reachable via `GetInvocationList` or GC.Collect test (best-effort).
**Success criteria:** Fix with `-=` restores collectability.

### M2. Custom Accessor
**Goal:** Event with add/remove logging subscription count to console.
**Success criteria:** Custom accessors compile; raise still works.

## Hard (stretch)

### H1. Weak Reference Sketch
**Goal:** Pseudocode `WeakEvent` storing `List<WeakReference<Delegate>>`; on raise prune dead refs.
**Success criteria:** Explain trade-off vs strong events.
