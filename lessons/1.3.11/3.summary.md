# IEnumerable vs IQueryable — краткая выжимка

> `1.3.11` · ~8 мин на повторение

## Главная идея

**`IEnumerable<T>`** — enumeration in-process (**`Func<>`**, LINQ to Objects). **`IQueryable<T>`** — **deferred remote query**: pipeline хранит **expression tree**, execution через **`IQueryProvider`** (EF → SQL). Syntax одинаков; engine разный.

## Ключевые пункты

- **`IQueryable`**: **`Expression`**, **`Provider`**, extends **`IEnumerable`**.
- Overloads: **`IEnumerable`** → **`Func<>`**; **`IQueryable`** → **`Expression<Func<>>`**.
- **`AsEnumerable()`** — switch to client evaluation (всё после — CLR).
- **`List.AsQueryable()`** — still local (**`EnumerableQuery`**), не PostgreSQL.
- **`ToList()` до `Where`** на DB — load all rows — anti-pattern.
- Repository: compose на **`IQueryable`** в scope **`DbContext`**; return materialized DTOs наружу.
- EF Core 3+: non-translatable LINQ → often **exception** at runtime.

## «Под капотом» в одном абзаце

**`Enumerable.Where`** wraps iterator + delegate. **`Queryable.Where`** builds **`MethodCallExpression`** node linked to previous expression. Terminal **`ToListAsync`** → EF **`Execute`** → SQL generator → driver → rows. Same deferral discipline as `1.3.9`, different execute.

## Частые ошибки

- Filter in memory после **`ToList()`** на full table
- **`IQueryable`** returned from API
- Local methods in **`IQueryable.Where`** without translation

## Мнемоника

> **Enumerable = Execute in C#** · **Queryable = Describe for provider** · **AsEnumerable = I take over in RAM**

## Связи

- **Предыдущая:** `1.3.10` — immediate execution
- **Следующая:** `1.3.12` — expression trees
- **EF Core:** phases 3+ — query translation, `ToQueryString()`
