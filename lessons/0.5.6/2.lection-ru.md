# Изображения: `img` `alt`, `figure`/`figcaption`, `picture`/`source`

> Roadmap: `0.5.6` · Node: `0.5` — HTML · Depth: practice

## Learning Objectives

После этой лекции ты сможешь:

- Размечать изображения через `<img>` и писать осмысленный **`alt`** для informative и decorative случаев.
- Группировать изображения с подписью через **`figure`** и **`figcaption`**.
- Отдавать responsive и art-directed картинки через **`<picture>`** и **`<source>`**.
- Использовать **`width`**, **`height`**, **`loading="lazy"`** для стабильности layout и performance.
- Связывать semantics изображений со структурой документа из `0.5.1`–`0.5.4`.

---

## Why This Matters

Текст из `0.5.3` и ссылки из `0.5.5` несут большую часть информации, но каталоги, dashboards и docs опираются на visuals. Изображения — это байты в сети, участники layout (CLS при поздней загрузке) и часто единственная метка, которую слышит пользователь screen reader рядом с icon-only UI.

**`alt`** — не SEO-галочка, а text alternative по спецификации HTML, когда картинка несёт информацию. Пустой `alt=""` явно помечает decoration, чтобы AT пропустила элемент. Без `alt` на informative image часто озвучивается имя файла `IMG_4832.jpg` — типичный production fail.

Responsive layout добавляет слой: один JPEG на 400px phone и 2560px monitor — лишний трафик или мыло. **`picture`** и **`source`** выбирают ресурс без JS; **`figure`** связывает visual с caption для зрячих и для тех, кто слышит figcaption.

## Core Concepts

### Элемент `<img>`

Void element **`<img>`** встраивает изображение. На практике нужны **`src`** и **`alt`**. Браузер fetch → decode → paint inline, если CSS не скрывает.

```html
<img
  src="/assets/charts/q1-revenue.png"
  alt="Столбчатая диаграмма: выручка Q1 +18% к Q4, лидирует enterprise"
  width="640"
  height="360"
/>
```

**`width`** и **`height`** в HTML задают intrinsic dimensions для резервирования места до load. С CSS `max-width: 100%; height: auto;` — responsive без layout jump (Core Web Vitals).

**`loading="lazy"`** откладывает fetch offscreen images. Не lazy-load hero/LCP — ухудшишь метрику.

### Эффективный `alt`

Спроси: **если бы я не видел картинку, какой информации не хватило бы?** Ответ — в `alt`, кратко, без префикса «изображение».

| Сценарий | Подход к `alt` |
|----------|----------------|
| Chart, photo, diagram | Суть, не каждый пиксель |
| Functional icon | Действие: `alt="Search"` |
| Decoration | `alt=""` |
| Дублирует текст рядом | Часто `alt=""` |
| Сложная схема | Короткий `alt` + описание в тексте/`aria-describedby` |

```html
<img src="/team/ada.jpg" alt="Ada Lovelace, chief architect, at desk" />
<img src="/textures/noise.svg" alt="" role="presentation" />
<button type="button">
  <img src="/icons/trash.svg" alt="" width="16" height="16" />
  Delete project
</button>
```

Не stuffing keywords в `alt`.

### `figure` и `figcaption`

**`<figure>`** оборачивает self-contained unit (image, code, quote). **`<figcaption>`** — visible caption и accessible name figure.

```html
<figure>
  <img
    src="/screenshots/login-flow.png"
    alt="Login form: email, password, Sign in"
    width="800"
    height="450"
    loading="lazy"
  />
  <figcaption>Figure 1. Login flow до MFA в v2.4.</figcaption>
</figure>
```

`figcaption` первым или последним — решает CSS. Ссылайся в prose: «как на Figure 1…».

### Responsive: `picture` и `source`

**`<picture>`** + несколько **`<source>`** + fallback **`<img>`**. Браузер берёт первый подходящий source.

**Resolution switching:**

```html
<picture>
  <source srcset="/hero-800.webp 800w, /hero-1200.webp 1200w" sizes="(max-width: 600px) 100vw, 50vw" type="image/webp" />
  <img src="/hero-800.jpg" alt="Dashboard: tasks and calendar widgets" width="800" height="500" />
</picture>
```

**Art direction:**

```html
<picture>
  <source media="(max-width: 599px)" srcset="/portrait-vertical.jpg" />
  <source media="(min-width: 600px)" srcset="/portrait-wide.jpg" />
  <img src="/portrait-wide.jpg" alt="Product team of six in lounge" width="1200" height="675" />
</picture>
```

**`type`** на source — WebP/AVIF fallback. **`alt`** всегда на финальном `<img>`.

### Изображения в semantic layout

Hero — в `<header>` или top `<main>`. Tutorial screenshots — в `<article>` внутри `<main>`. Критичную информацию не прячь только в pixels без text alternative.

Linked thumbnail (`0.5.5`):

```html
<a href="/products/aurora-headphones">
  <img src="/aurora-thumb.webp" alt="Aurora wireless headphones, black" width="120" height="120" />
</a>
```

## Under the Hood

Parser находит `src` → preload scanner fetch → decode → layout slot если заданы width/height. Без dimensions — reflow и CLS.

Для `<picture>` — source selection algorithm: `type`, `media`, затем `srcset`+`sizes`+DPR. `alt` идёт в accessibility API как accessible name; пустой `alt` убирает meaningful node для decoration.

## Syntax / Commands / API

| Элемент / атрибут | Роль |
|-------------------|------|
| `<img src alt>` | Image + alternative text |
| `width`, `height` | Layout hints |
| `loading="lazy\|eager"` | Fetch timing |
| `srcset`, `sizes` | Responsive candidates |
| `<picture>`, `<source>` | Variants |
| `<figure>`, `<figcaption>` | Captioned unit |

## Common Mistakes & Anti-patterns

Missing/filename `alt`; keyword stuffing; lazy LCP hero; `picture` без `img` alt; content только в CSS background-image.

## Production & Real-World Notes

CDN width params; design system с обязательным `alt` в props; RUM CLS из CMS uploads без dimensions.

---

## Comparison / Trade-offs

Single `img`+`srcset` vs `picture`+media; CSS background только для decoration.

## Quick Reference

```html
<img src="photo.jpg" alt="Description" width="400" height="300" loading="lazy" />
<figure><img src="c.png" alt="Chart summary" /><figcaption>Caption</figcaption></figure>
<picture><source type="image/webp" srcset="i.webp" /><img src="i.jpg" alt="Desc" /></picture>
```

---

## Key Takeaways

- Informative → descriptive `alt`; decorative → `alt=""`.
- `figure`/`figcaption` для captioned units.
- `picture`/`source` — format и art direction без JS.
- width/height + responsive CSS против CLS.
- Lazy below fold, не LCP.
