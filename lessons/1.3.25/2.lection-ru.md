## Why This Matters

В реальных приложениях постоянно нужно объединять, deduplicate и сравнивать collections: merge tag lists двух products, найти users в group A, но не в group B, permissions common двум roles, убрать duplicates из import file. В SQL есть **`DISTINCT`**, **`UNION`**, **`INTERSECT`**, **`EXCEPT`**; LINQ зеркалит их на **`IEnumerable<T>`** и **`IQueryable<T>`**.

Неверный operator или equality semantics даёт subtle bugs: два **`Person`** с одинаковым email считаются разными, потому что **`Distinct()`** использует reference equality; **`Union`** silently deduplicates, когда нужен был concat; **`Except`** на двух **`IQueryable`** sources генерирует subquery, сканирующий обе tables. Middle-разработчик должен знать **semantics** (что считается «равным», что происходит с order, lazy ли результат) и **стоимость реализации** (allocations HashSet, full materialization).

## Core Concepts

### Sets vs Sequences

**Математическое множество** не имеет duplicates и order. **Sequence** (`IEnumerable<T>`) имеет order и может содержать duplicates. LINQ set operators находятся между этими моделями: они **потребляют ordered sequences** и выдают **sequences без duplicates** (где применимо), обычно **сохраняя first-seen order** от левого operand.

Когда пишешь `first.Union(second)`, результат содержит каждый distinct element один раз, ordered by first appearance в `first`, затем first appearance в `second` для элементов, которых ещё не было. Это **не** «concatenate и sort».

### Distinct — удаление дубликатов

```csharp
var ids = new[] { 1, 2, 2, 3, 1, 4 };
var unique = ids.Distinct(); // 1, 2, 3, 4
```

**`Distinct()`** без аргументов использует **`EqualityComparer<T>.Default`**. Для **`string`** — ordinal equality. Для **reference types** без overridden **`Equals`** каждый object уникален, даже если fields совпадают — классический junior bug.

**`DistinctBy(keySelector)`** (.NET 6+) deduplicates по projected key, возвращая **original element** (first occurrence per key):

```csharp
var people = new[] {
    new { Name = "Ada", Dept = "Eng" },
    new { Name = "Lin", Dept = "Eng" },
    new { Name = "Ada", Dept = "Ops" }
};
var byName = people.DistinctBy(p => p.Name); // Ada(Eng), Lin
```

**Deferred execution:** `Distinct` оборачивает source; работа при enumeration. Внутри держится **set seen keys** — O(n) time, O(n) extra space для hash set.

### Union — A ∪ B (distinct merge)

```csharp
var a = new[] { 1, 2, 3 };
var b = new[] { 3, 4, 5 };
var u = a.Union(b); // 1, 2, 3, 4, 5
```

**`Union`** эквивалентен **`Concat` + `Distinct`** с тем же comparer, но реализован одним проходом по обеим sequences через hash set. **`UnionBy`** (.NET 6+) проецирует keys перед deduplication.

Нет overload **`Union`**, который сохраняет duplicates — для multiset merge используй **`Concat`**.

### Intersect — A ∩ B

```csharp
var a = new[] { 1, 2, 3, 3 };
var b = new[] { 3, 4, 5 };
var i = a.Intersect(b); // 3 — один раз, по order первой sequence
```

**`Intersect`** возвращает elements, присутствующие в **обеих** sequences, каждый **один раз** в order **first occurrence** первой sequence. Обе sequences полностью enumerates (не полагайся на side effects).

### Except — A \ B

```csharp
var a = new[] { 1, 2, 3, 4 };
var b = new[] { 3, 5 };
var diff = a.Except(b); // 1, 2, 4
```

**`Except`** возвращает elements первой sequence, которых **нет** во второй, deduplicated, сохраняя order первой. Не возвращает elements из B, missing в A — set difference **не symmetric**.

Symmetric difference (в either, но не в both): `a.Except(b).Union(b.Except(a))`.

### Custom Equality — IEqualityComparer<T>

Когда identity — не default equality:

```csharp
var emails = users.Select(u => u.Email).Distinct(StringComparer.OrdinalIgnoreCase);
var setA = rolesA.Union(rolesB, StringComparer.OrdinalIgnoreCase);
```

Для custom types — **`IEqualityComparer<Person>`** или **`DistinctBy(p => p.Email)`**, если key простой.

Один и тот же **comparer** нужен для consistent behavior; membership test второй sequence использует его.

## Under the Hood

### Iterator Chains и HashSet

Set operators в **`System.Linq.Enumerable`** — **iterator blocks** (`yield return`), lazy output, но **нужно помнить seen elements**. Для **`Distinct`** iterator создаёт **`Set<T>`** (internal hash set wrapper) на первом **`MoveNext`** и добавляет каждый element (или key) перед yield.

Модель для **`Union(first, second)`**:

```
seen = empty HashSet
foreach x in first:
    if seen.Add(x): yield x
foreach x in second:
    if seen.Add(x): yield x
```

**`Intersect`** обычно строит set из **второй** sequence, затем walks **first** и yields found elements, tracking dedup на output.

Memory **O(unique elements)**; плохой **`GetHashCode`** деградирует к O(n²).

### IQueryable Translation (EF Core)

На **`IQueryable`** set operators переводятся в SQL:

| LINQ | Typical SQL |
|------|-------------|
| `Distinct()` | `SELECT DISTINCT ...` |
| `Union(other)` | `UNION` |
| `Intersect(other)` | `INTERSECT` |
| `Except(other)` | `EXCEPT` |

**`Union`** в LINQ — SQL **`UNION`**, не **`UNION ALL`**. Для всех rows с duplicates — **`Concat`** или raw SQL.

Providers могут expand **`Except`** как **`NOT EXISTS`**. Client evaluation при mix in-memory и **`IQueryable`** — performance и correctness trap.

### DistinctBy — sketch

**`DistinctBy`** использует **`HashSet<TKey>`**; на первом unseen key yield **source element**. Expensive key selector — profile.

## Syntax / Commands / API

```csharp
source.Distinct()
source.Distinct(comparer)
source.DistinctBy(keySelector)           // .NET 6+
first.Union(second)
first.Union(second, comparer)
first.Intersect(second)
first.Except(second)
// *By variants — .NET 6+
```

Все возвращают **`IEnumerable<T>`** (deferred), пока не вызовешь **`ToList()`** и т.п.

## Examples

### Deduplicating Import Rows

```csharp
var imported = await ReadCsvRowsAsync();
var uniqueBySku = imported.DistinctBy(r => r.Sku, StringComparer.OrdinalIgnoreCase);
```

First row wins. «Last wins» — reverse перед **`DistinctBy`** или **`GroupBy`**.

### Permission Diff Between Roles

```csharp
var onlyEditor = editorPerms.Except(adminPerms);
var shared = editorPerms.Intersect(adminPerms);
```

### EF Core

```csharp
var activeNotBlocked = db.Users
    .Where(u => u.IsActive)
    .Select(u => u.Id)
    .Except(blockedIds);
```

Inspect **`ToQueryString()`** в development.

## Common Mistakes & Anti-patterns

**`Distinct()` на objects без equality** — **`DistinctBy`** или **`Equals`/`GetHashCode`**.

**`Union` вместо `Concat`** когда нужны duplicates.

**Wrong order в `Except`.**

**Multiplicity в `Intersect`** — один элемент, не три.

**Side effects в source** — materialize или pure sequences.

**Mixing comparers** между operands.

## Production & Real-World Notes

In-memory millions of rows — explicit **`HashSet<T>`** может быть быстрее chained LINQ. **`DistinctBy`** на hot path после **`Select`** с лёгким key (`int`, `string`) экономит память.

Pagination: **`Distinct`** после **`Skip/Take`** на SQL без subquery — duplicates могут вернуться между pages.

## Quick Reference

| Operator | Meaning | Duplicates | Order |
|----------|---------|------------|-------|
| `Distinct` | unique | no | first-seen |
| `Union` | A or B | no | A then B |
| `Intersect` | A and B | no | order of A |
| `Except` | A not B | no | order of A |

---

## Key Takeaways

- Set operators убирают duplicates, сохраняют **first-seen order**.
- **`Distinct()`** на reference types без **`Equals`** не deduplicates by value.
- **`Union`** ≠ **`Concat`**; **`Except`** asymmetric.
- Hash sets → O(n) time, O(unique) memory.
- На **`IQueryable`** — SQL set ops; проверяй generated SQL.
- **`DistinctBy`/`UnionBy`** (.NET 6+) для key-based identity.
