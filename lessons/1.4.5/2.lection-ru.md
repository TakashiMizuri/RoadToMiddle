# TaskCompletionSource — ручное управление Task

> Roadmap: `1.4.5` · Node: `1.4` — C# async ⚡ · Depth: **глубоко**

## Learning Objectives

- **`TaskCompletionSource<T>`** как manual Task factory.
- **`SetResult`/`SetException`/`SetCanceled`** — exactly once.
- Bridge **event/callback** API → **`await`**.
- **`TrySet*`** против races.
- TCS в BCL: timers, HttpClient pattern.
- **`RunContinuationsAsynchronously`**.

---

## Why This Matters

Не всё async начинается с `async Task`. Legacy: events, callbacks. **TCS** — ты **produce** Task и complete когда готов — связь с `await`.

Wrong TCS: double SetResult → exception; never complete → hang. Middle читает library source и пишет wrappers.

---

## Core Concepts

```csharp
var tcs = new TaskCompletionSource<string>();
api.OnDone += (_, r) => tcs.TrySetResult(r);
var value = await tcs.Task;
```

**`TrySetResult`** — false если уже completed (race OK).

**RunContinuationsAsynchronously** — continuations не на completing thread inline — меньше deadlock risk в libraries.

---

## Under the Hood

TCS держит Task в **WaitingForActivation** до Set*. Atomic transition + queue continuations. Many awaiters on one Task — broadcast completion.

Timeout pattern: CancellationToken.Register → TrySetCanceled.

---

## Examples

### Пример 1: Timer → Task

```csharp
Task DelayFromTimer(int ms)
{
    var tcs = new TaskCompletionSource<object?>(
        TaskCreationOptions.RunContinuationsAsynchronously);
    var timer = new Timer(_ => tcs.TrySetResult(null));
    timer.Change(ms, Timeout.Infinite);
    return tcs.Task;
}
```

BCL использует **`Task.Delay`** — тот же паттерн TCS внутри.

### Пример 2: Legacy event API

Подписка один раз → **`TrySetResult`** / **`TrySetException`** в handler → **`await tcs.Task`** → unsubscribe в finally.

### Пример 3: Race — first wins

Два callback могут вызвать **`TrySetResult`** — второй получит `false`; это OK, не exception.

---

## Common Mistakes & Anti-patterns

**SetResult** вместо **TrySet** в callback — второй callback crash.

**Never complete** — await висит вечно; leak continuations.

**Sync continuations** на thread с lock, который ждёт awaiter — deadlock.

**Handlers leak** — не отписались после complete.

---

## Production & Real-World Notes

Library code: **`RunContinuationsAsynchronously`** по умолчанию.

Всегда link **`CancellationToken`** (client disconnect, timeout).

ASP.NET: не держи request scope в callback после end request — capture data раньше.

**Channels** (`1.4.17`) часто лучше hand-rolled TCS queue.

## Comparison / Trade-offs

| Подход | Когда |
|--------|-------|
| `async Task` | Полный контроль flow |
| TCS | External callback |
| `Task.FromResult` | Result уже есть |
| `IValueTaskSource` | High-perf pooling |
