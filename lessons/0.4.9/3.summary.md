# CDN — краткая выжимка

> `0.4.9` · ~8 мин на повторение

## Главная идея

**CDN** хранит копии static/cacheable content на **edge PoP** близко к пользователю. **Origin** (S3, blob) — источник правды. React build (`*.js`, `*.css` с hash) — типичный CDN-кандидат; API — обычно нет.

## Ключевые пункты

- **Cache hit** — ответ с edge; **miss** — fetch с origin, затем cache.
- **Hashed filenames** + `max-age` long — cache busting без purge.
- **`index.html`** — короткий TTL или invalidation после deploy.
- Split: `app.example.com` → CDN, `api.example.com` → ALB/Kestrel.
- CDN ≠ load balancer: cache delivery vs dynamic compute.
- DNS CNAME на CDN distribution (`0.4.5`).

## Самые важные примеры

```
Cache-Control: public, max-age=31536000, immutable  → app.[hash].js
Cache-Control: no-cache                             → index.html
```

## Частые ошибки

- Год cache на `index.html`.
- Кеширование персонализированного API JSON на CDN.

## Мнемоника

> **Hash in filename = новый URL = новый cache** — не полагайся только на purge.
