# DOM-дерево: Document, Element, Text nodes

> Roadmap: `0.5.10` · Node: `0.5` — HTML · Depth: **глубоко**

## Learning Objectives

После этого урока ты сможешь:

- Описывать **Document Object Model** как дерево узлов с корнем `document`.
- Различать **типы узлов**: `Document`, `Element`, `Text`, `Comment`, `DocumentFragment` и др.
- Объяснять, как HTML-парсер строит DOM и когда **script**/**stylesheet** блокируют parsing.
- Обходить и изменять дерево через `parentNode`, `childNodes`, `children`, `firstElementChild`.
- Связать DOM с **accessibility tree** и reconciliation React virtual DOM.
- Предсказывать эффект `textContent` vs `innerHTML` на узлы и безопасность.

## Why This Matters

Все HTML-уроки `0.5.x` — формы, таблицы — это **сериализация дерева**, которое браузер построит в памяти. `<p>Hello</p>` — не строка в памяти, а узлы со связями parent/child. JavaScript, React, Vue читают и пишут это дерево. ASP.NET Core отдаёт HTML bytes; клиентский parser создаёт ту же модель. Middle-разработчик отлаживает «handler не срабатывает», hydration mismatch, дубликаты id в a11y tree, XSS через innerHTML — через **узлы**, не через «теги как текст».

DOM — граница между **разметкой** и **runtime**. CSS selectors попадают в element nodes; screen readers берут **accessibility tree** из элементов и атрибутов. Путаница innerText/textContent и миф «textContent опасен как innerHTML» — решения на уровне узлов. Глубина здесь окупается во всём frontend и при review SSR из Razor.

## Core Concepts

### Document как корень

При загрузке страницы parser создаёт узел **`Document`** — корень дерева. В JS: `document`. Document владеет doctype, **`documentElement`** (`<html>`), `title`, `URL`, **ready state**.

Всё из markup висит под `document.documentElement`. DOM **live**: удаление узла в script меняет layout и listeners; React заменяет subtrees. Это отличает DOM от одноразово распарсенного JSON.

### Node и nodeType

Базовый интерфейс **`Node`**: `nodeType`, `nodeName`, `parentNode`, `childNodes`, `textContent`.

| Constant | Value | Пример |
|----------|-------|--------|
| `DOCUMENT_NODE` | 9 | `document` |
| `ELEMENT_NODE` | 1 | `<div>`, `<p>` |
| `TEXT_NODE` | 3 | текст между тегами |
| `COMMENT_NODE` | 8 | `<!-- -->` |
| `DOCUMENT_FRAGMENT_NODE` | 11 | fragment |

**Element nodes** — tagName, attributes, `children`, `querySelector`, `append`. **Text nodes** — только character data; `nodeValue`; без attributes.

Whitespace между тегами часто даёт **text nodes**. `<div>\n  <span>x</span>\n</div>` — три child у div: text, span, text. `firstChild` может попасть в `\n`, не в span → используй **`firstElementChild`** и **`children`**.

### Построение дерева из HTML

Tokenize → construct tree по правилам HTML spec → attach. Misnested tags **auto-correct** — DevTools DOM может отличаться от исходника; **canonical DOM** — то, что видят CSS и JS.

Void elements без children. **Script/style** — особый режим текста. **Template** — content в **DocumentFragment** до activation.

Ресурсы и parsing: classic script без async/defer **блокирует** parsing; defer — после parse, до DOMContentLoaded; async — когда готов. CSS render-blocking по умолчанию.

SSR: сервер шлёт HTML; browser строит DOM; hydration **привязывается** к существующим element nodes — mismatch, если server/client trees различаются.

### Traversal и mutation

`childNodes` — все типы; `children` — только elements. Mutation: `appendChild`, `append`, `remove`, `replaceWith`. `createElement`, `createTextNode`. **`textContent =`** — одна text node, **без HTML parse**, safe от injection. **`innerHTML =`** — parse markup, **XSS** с untrusted input.

**DocumentFragment** — batch insert вне document, меньше reflows.

### Document vs Window

`window` — global; `document` — active document. Events **bubble** по цепочке element nodes до document/window.

## Under the Hood

### Parser и реализация DOM

Движки (Blink, Gecko, WebKit) — state machine по WHATWG HTML. **Open elements stack** решает insertion point. **Incremental parsing** — DOM доступен до полной загрузки.

Nodes — native objects с layout cache после style calc. Text nodes — UTF-16. **Shadow DOM** — отдельные subtrees на shadow roots.

### Live collections

`HTMLCollection`, live NodeList обновляются при mutation DOM — осторожно в циклах. `querySelectorAll` — static. Read/write geometry + style interleaved → **layout thrashing**.

### Accessibility tree

Не каждый DOM node в **accessibility tree**. Text внутри button сливается в accessible name. `aria-hidden`, `display:none` — pruning (с нюансами). `<span onclick>` — всё ещё span element, не button role без semantics/keyboard.

### React

Virtual DOM diff → minimal `insertBefore`/`removeChild`. **Hydration** match SSR nodes — whitespace text mismatch → warnings. **Keys** сохраняют identity при reorder списков.

```javascript
node.nodeType === Node.ELEMENT_NODE

parent.children
```

## Examples

**Whitespace trap** — `list.childNodes.length` vs `list.children.length`.

**textContent wipe** — после innerHTML замена на textContent оставляет одну TEXT_NODE.

**Fragment batch** — append many `li` через fragment, один reflow.

## Common Mistakes & Anti-patterns

**firstChild вместо firstElementChild** — whitespace.

**innerHTML для user data** — XSS.

**DOM = source HTML** — parser repairs.

**Live collection в loop** — баги при mutation.

**Detached DOM + closures** — memory leaks.

**innerText vs textContent** — innerText учитывает layout/visibility.

## Production & Real-World Notes

## Comparison / Trade-offs

createElement vs innerHTML templates — verbose/safe vs fast/untrusted. childNodes vs children — text matters or not. Imperative DOM vs React — abstraction vs debug need.

## Key Takeaways

- DOM — **live tree of nodes**.
- Document root; elements + text nodes; whitespace — text nodes.
- Parser **repairs** HTML.
- **firstElementChild** / **children** для elements.
- **textContent** safe; **innerHTML** parses + XSS.
- A11y tree **derived**, not 1:1 DOM.
- React reconciles to DOM ops; hydration needs match.
