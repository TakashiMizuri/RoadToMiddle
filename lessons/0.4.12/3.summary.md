# ping / traceroute — краткая выжимка

> `0.4.12` · ~8 мин на повторение

## Главная идея

**ping** — ICMP reachability + **RTT** (latency). **traceroute/tracert** — path hop-by-hop, где растёт delay. Не заменяют **curl**/port test: ICMP часто blocked (`0.4.7`); ping OK ≠ 443 open.

## Ключевые пункты

- `ping -c N` (Unix) / `ping -n N` (Windows).
- `traceroute` / `tracert` — TTL-based hop discovery.
- **100% loss** на ping + HTTP 200 — ICMP block, не «server down».
- **`* * *`** — router не отвечает на probe, часто normal.
- RTT jump на hop — geography/congestion (`0.4.11`).
- Unknown host → **DNS** (`0.4.5`) first.
- Incident template: ping + tracert + curl :443.

## Самые важные примеры

```bash
ping -c 4 api.example.com
traceroute api.example.com
curl -I --connect-timeout 5 https://api.example.com/health
```

```powershell
tracert api.example.com
Test-NetConnection api.example.com -Port 443
```

## Частые ошибки

«ping не проходит — сервер мёртв» без curl. Диагностика CORS через ping.

## Мнемоника

> **ping** = «достучаться до IP?»; **traceroute** = «где по дороге тормоз?»; **curl** = «сервис на порту жив?»
