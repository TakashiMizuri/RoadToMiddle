## Answers

### 1. [Recall]
**Answer:** Переходит к **следующей итерации** loop, не выполняя оставшийся code текущей итерации.
**Why:** Jump к loop condition / increment.

### 2. [Recall]
**Answer:** **Нет** — compile error.
**Why:** `break` только в switch/loop context.

### 3. [Understanding]
**Answer:** **Нет**, только inner loop.
**Why:** `break` не label-based в C#.

### 4. [Understanding]
**Answer:** Flat structure; preconditions видны сверху; меньше nesting и mental stack.
**Why:** Guard checklist vs pyramid.

### 5. [Understanding]
**Answer:** Когда найден результат и дальнейшая работа method не нужна — `return` value сразу.
**Why:** `break` + code после loop vs direct exit — clarity.

### 6. [Application]
**Answer:** **`012`** (без newline между, если Write).
**Why:** i=3 → break до Write.

### 7. [Application]
**Answer:** **`13`** (odd only).
**Why:** even → continue skip Write.

### 8. [Analysis]
**Answer:** **5**.
**Why:** Second branch `return -x` → `-(-5)`.

### 9. [Analysis]
**Answer:** **B** неверно.
**Why:** C# **не** имеет labeled break как Java.

### 10. [Application]
**Answer:** Extract method с loops и **`return`** из него; или **flag** в outer condition `&& !done`.
**Why:** Idiomatic C# без labels.

### 11. [Understanding]
**Answer:** **Да** — early exit с message, status 0 (или `Environment.Exit` для non-zero позже).
**Why:** Guard на entry point.

### 12. [Application]
**Answer:** **`continue`** — skip one line, не exit loop.
**Why:** `break` остановил бы весь foreach.

### 13. [Recall]
**Answer:** В каждом **`case`** (кроме fall-through через empty case stack) перед следующим case — нужен `break`, `return`, или `goto case`.
**Why:** No accidental fall-through.

### 14. [Understanding]
**Answer:** Update в конце может **не выполниться** → infinite loop.
**Why:** `continue` jumps до condition, минуя update — ставь update в for header или перед continue path carefully.
