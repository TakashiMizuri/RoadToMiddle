# Implicit vs explicit interface implementation — краткая выжимка

> `1.2.9` · ~8 мин на повторение

## Главная идея

**Implicit** — public member одновременно API class и реализация interface; вызов **`obj.M()`** и **`((I)obj).M()`**. **Explicit** — **`void IInterface.M()`**, member **не виден** на class; только **`((IInterface)obj).M()`**.

## Ключевые пункты

- Implicit требует **public** (для classic interface members).
- Explicit — отдельный MethodDef, IL **private**, mapping через **InterfaceImpl**.
- **Collision** двух interfaces с одной signature → **explicit per interface**.
- BCL: **`IComparable<T>.CompareTo`**, non-generic **`IEnumerable.GetEnumerator`** — часто explicit.
- Primary service contract (**`IPaymentGateway`**) → обычно **implicit**.
- Auxiliary / hide from public surface → **explicit**.

## Самые важные примеры

```csharp
public void Log(string msg) { }           // implicit

void ILogger.Log(string msg) { }        // explicit — только через ILogger

void IA.Save() => ...;
void IB.Save() => ...;                  // collision fix
```

## «Под капотом» в одном абзаце

CLR **InterfaceImpl** связывает interface slot с class method. Explicit methods не в public API class, но dispatch **`callvirt`** через interface reference работает. Class override / explicit impl beats **default interface method** (`1.2.10`).

## Частые ошибки

- Explicit «для красоты» без причины.
- Забыли cast к interface.
- Два разных **`Save`** через один public implicit method.

## Мнемоника

> **Public face of the class → implicit. Hide or collide → explicit.**
