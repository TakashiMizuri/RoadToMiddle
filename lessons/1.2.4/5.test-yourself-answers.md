## Answers

### 1. [Recall]
**Answer:** **`private`**.

### 2. [Recall]
**Answer:** **`internal`**.

### 3. [Recall]
**Answer:** Только **declaring class** (не derived, не другие types).

### 4. [Understanding]
**Answer:** **Нет**, если другой project — другая assembly (без InternalsVisibleTo). `internal` — assembly boundary.

### 5. [Understanding]
**Answer:** **`protected`** — derived **может** access (даже cross-assembly). **`internal`** — derived **в другой assembly не может**, если member only internal and not protected.

### 6. [Understanding]
**Answer:** **Union (OR)** — доступ если выполняется protected **или** internal условие.

### 7. [Application]
**Answer:** **`private`** — storage не part of public contract.

### 8. [Application]
**Answer:** **`[assembly: InternalsVisibleTo("Tests.Assembly")]`** в production assembly.

### 9. [Analysis]
**Answer:** Field **public** — и visibility слишком широкая, и encapsulation broken (`1.2.5`); нужен private field + property/methods.

### 10. [Analysis]
**Answer:** Tests через **public behavior** или InternalsVisibleTo / test hooks `internal`; public everything **locks API** и ломает encapsulation.

### 11. [Understanding]
**Answer:** Hook для **subclasses**, не для external callers — narrower contract, template method pattern.

### 12. [Application]
**Answer:** Только **QueryBuilder** и nested scope; внешний code не видит `Builder`.

### 13. [Recall]
**Answer:** **Declaring class + derived classes in same assembly only**.

### 14. [Understanding]
**Answer:** Small public API — semver stability, hide refactor surface; facade adds indirection but protects consumers. Balance for libraries vs apps.
