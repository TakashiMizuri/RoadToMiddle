# Default interface methods (C# 8+)

> Roadmap: `1.2.10` · Node: `1.2` — C# OOP · Depth: понимание

## Learning Objectives

После этого урока ты сможешь:

- Определить **default interface methods (DIM)** — instance methods с body на interface.
- Объяснить мотивацию C# 8 — **evolution interface** без break existing implementers.
- Описать, как class **наследует**, **override**-ит или **explicit**-но реализует defaults.
- Назвать **ограничения**: нет instance fields, diamond conflicts, versioning.
- Сравнить DIM с **abstract class** и **extension methods**.
- Читать framework interfaces с defaults; применять DIM осторожно в app code.

---

## Why This Matters

Долго interfaces в C# были **чистыми contracts** — только signatures. Новый method в shipped **`IRepository`** ломал все implementers до добавления stub. Java добавила default methods; **C# 8 (2019)** — то же.

Ты **consume** DIM в modern .NET, даже если не пишешь их: evolution BCL interfaces, plugin APIs. Class «не implement» method, но call через interface **работает** — нужно понимать **dispatch**.

Путаница: **`((IService)obj).New()`** OK, **`obj.New()`** — нет public member. На design review предлагают «default на interface» вместо abstract base — нужны **trade-offs** из этого урока.

## Core Concepts

### Что такое DIM

```csharp
public interface IGreeter
{
    string Greet(string name) => $"Hello, {name}"; // C# 8+
}

public class SimpleGreeter : IGreeter { }

IGreeter g = new SimpleGreeter();
g.Greet("Ada"); // Hello, Ada
```

**`SimpleGreeter`** не объявляет **`Greet`** public. Call работает через **`IGreeter`** reference — interface mapping включает default body.

**Не extension methods**: DIM — **instance** methods в interface metadata; extensions — static.

### Зачем: interface evolution

**Без DIM:** новый **`Shutdown`** в **`IPlugin`** → все implementers CS0535.

**С DIM:**

```csharp
void Shutdown() { /* no-op default */ }
```

Existing classes **compile unchanged**. Plugins с cleanup **override**:

```csharp
public void Shutdown() { /* disconnect */ }
```

Additive API с меньшим break; design cost остаётся.

### Override defaults

**Public member** с той же signature — как implicit impl (`1.2.9`):

```csharp
public class HexFormatter : IFormatter
{
    public string Format(int value) => value.ToString("X");
}
```

**Explicit** тоже override slot. Без impl class — **InterfaceImpl** → default on interface.

### Что interfaces всё ещё не могут

- **No instance fields** (state — в class, abstract base, composition).
- **No instance constructors**.
- Default logic — locals, static, parameters.

Heavy shared state → **abstract class** (`1.2.8`).

### Diamond problem

**`IA`** и **`IB`** с разными default **`Log()`**:

```csharp
public class Worker : IA, IB
{
    public void Log() => Console.WriteLine("Resolved");
}
```

Class **must resolve**. Explicit per interface — как при name collision (`1.2.9`).

### DIM vs abstract vs extensions

| | DIM | Abstract | Extension |
|---|-----|----------|-----------|
| Instance via interface | Да | Да | Static syntax |
| Instance fields | Нет | Да | Нет |
| Break on add | Нет с default | Да если abstract | Нет |

## Under the Hood

Default methods в **interface metadata** с bodies. **InterfaceImpl** class → default def если нет class impl. Dispatch: **class override > default**. **callvirt** через mapping.

---

## Examples

**`IAuditable.OnAudit()`** — empty default; **`SecureOrder`** overrides.

Framework docs «added with default» — твой handler compiles; новая capability через **`IHandler`** или opt-in override.

---

## Common Mistakes

DIM для shared state. **`obj.Default()`** без public impl — need cast. Fat defaults в interface. Diamond ignore. Replace abstract bases blindly.

---

## Production & Real-World Notes

Apps **implement** DIM чаще, чем **define**. Library authors — additive NuGet APIs. Moq — verify Setup для new members при upgrade.

Defaults — **small** (no-op), не complex untested logic.

---

## Key Takeaways

- DIM — optional behavior на interface; C# 8+.
- Existing implementers compile; override optional.
- Call через **interface reference** если нет public override.
- Не замена abstract class (нет instance state).
- Diamond → resolve на class.
- Основная мотивация — **library evolution**.
