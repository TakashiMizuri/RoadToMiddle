# AsyncLocal — краткая выжимка

> `1.4.32` · ~8 мин на повторение

## Главная идея

**`AsyncLocal<T>`** хранит ambient-данные, которые **переезжают через `await`** вместе с execution context, в отличие от `ThreadLocal`, привязанного к физическому потоку.

## Ключевые пункты

- Один запрос = логический поток, много потоков pool.
- Middleware задаёт correlation/tenant; **`finally`** сбрасывает.
- **`IHttpContextAccessor`** — built-in AsyncLocal для HTTP.
- Не прятать обязательные business-зависимости.
- Parallel ветки получают **копии** контекста на момент ветвления.
- Mutable объекты в AsyncLocal — риск shared mutation.

## Самые важные примеры

```csharp
private static readonly AsyncLocal<string?> _cid = new();
// set at request start, null in finally
```

## Частые ошибки

- Static вместо AsyncLocal для per-request state.
- Забыть clear → leak на pool thread.
- Использовать вместо явного `CancellationToken`.

## Мнемоника

**«Async flows logic, not threads — AsyncLocal follows logic»**
