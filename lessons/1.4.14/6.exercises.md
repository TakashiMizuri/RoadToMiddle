# Exercises: Sync-over-Async Deadlocks

## Easy

### E1. Exception Compare
**Goal:** Faulted Task — catch from `.Wait()` vs `GetAwaiter().GetResult()` — note exception types.
**Success criteria:** Aggregate vs direct documented.

### E2. Code Search
**Goal:** Find `.Result` and `.Wait()` in sample ASP.NET repo; classify SyncContext vs starvation risk.
**Success criteria:** Per-file recommendation.

## Medium

### M1. Reproduce UI Deadlock
**Goal:** Minimal WinForms/WPF app: button `.Result` on delay async — confirm hang; fix with async handler.
**Success criteria:** Before/after explanation with MoveNext Post.

### M2. Refactor Controller
**Goal:** Sync action using `.Result` → async `Task<IActionResult>` with await.
**Success criteria:** No blocking API on request path.

## Hard

### H1. Legacy Bridge Design
**Goal:** Sync interface `IFoo` must call async SDK — propose options without `.Result` on request thread (queue, dedicated worker, IAsyncFoo).
**Success criteria:** Trade-off table; refs `1.4.41`.
