# Exercises: Operator overloading (basics)

> Complete after reading lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Vector addition
**Goal:** Implement `+` and `-` for `readonly struct Vector2`.
**Steps:**
1. Properties `X`, `Y`; constructor.
2. `operator +` and `operator -` return new instances.
3. Console: `(1,2) + (3,4)` → `(4,6)`.
**Success criteria:** Compiles; no mutation of operands.

### E2. Equality operators
**Goal:** Align `==` with `Equals` for a small `sealed class EmailAddress` (wraps `string Value`).
**Steps:**
1. Override `Equals`/`GetHashCode` (ordinal).
2. Add `==`/`!=` delegating to `Equals`.
3. Demonstrate `a.Equals(b)` true implies `a == b` true.
**Success criteria:** Consistent behavior; links to `1.2.16`.

## Medium

### M1. Money with guard
**Goal:** `operator +` only for matching currency.
**Steps:**
1. `Money(decimal amount, string currency)`.
2. `operator +` throws on mismatch; returns new `Money`.
3. Optional: implement `Add` and call it from operator (single validation).
**Success criteria:** Clear exception message; immutable result.

### M2. When NOT to overload
**Goal:** Practice naming over operators.
**Steps:**
1. Sketch `Order` and `Discount` types.
2. Write `ApplyDiscount(Order, Discount)` — no operators.
3. In 3 sentences explain why `order + discount` would confuse reviewers.
**Success criteria:** Written justification references least surprise.

## Hard (stretch)

### H1. Explicit conversion
**Goal:** Safe narrowing from `Money` to `decimal`.
**Steps:**
1. Add `explicit operator decimal(Money m)` that returns `m.Amount` (document currency loss).
2. Show call site: `decimal d = (decimal)money;`
3. Explain why implicit was not used.
**Success criteria:** Compiles; comment block explains trade-off.

## Submission checklist

- [ ] All operators are `public static`
- [ ] No business side effects hidden in operators
- [ ] Equality exercises coordinate with `GetHashCode`
