# const vs readonly vs static readonly

> Roadmap: `1.1.19` · Node: `1.1` — C#: основы языка · Depth: **глубоко**

## Learning Objectives

После этого урока ты сможешь:

- Точно различать **compile-time constant** и **runtime-initialized immutable** field.
- Выбирать **`const`**, instance **`readonly`**, **`static readonly`** для config, defaults, domain constants.
- Объяснить **const inlining** и влияние на versioning без recompile consumer assemblies.
- Читать IL/metadata: literal constants vs static init vs instance init.
- Не путать **`readonly` field**, **`readonly struct`**, **`init`**, **`readonly` methods**.

---

## Why This Matters

В каждом проекте есть неизменяемые значения: имена HTTP headers, default page size, ключи feature flags. C# даёт **`const`**, **`readonly`**, **`static readonly`** — и их часто считают взаимозаменяемыми. Это не так. Ошибка выбора → production bugs: **`const`** в library не обновится в consumer без **recompile**; instance **`readonly`** там, где хватило **`static readonly`**; попытка **`const`** с **`DateTime.UtcNow`** не компилируется, а **`static readonly`** — правильный инструмент.

Middle-разработчик поддерживает shared libraries, ASP.NET options и ревьюит PR, где computed value помечен **`const`** через hack. Понимание **когда compiler embeds value в IL** vs **CLR loads field из metadata** — разница между предсказуемым deploy и «мы поменяли constant, а prod показывает старое». Урок идёт under the hood — IL, metadata, initialization order.

## Core Concepts

### const: compile-time constants

**`const`** инициализируется **constant expression**, вычислимым at compile time. Типы: numeric primitives, **`bool`**, **`char`**, **`string`**, **`null`**. Нельзя **`DateTime`**, **`new`**, вызовы методов (кроме concat **`string`** constants).

```csharp
public static class ApiConstants
{
    public const int DefaultPageSize = 20;
    public const string VersionHeader = "X-Api-Version";
}
```

**`const`** **implicitly static**. В metadata — **literal constants**. Когда **другая assembly uses** твой **`const`**, C# compiler **inlines literal** в IL consumer — не всегда field read at runtime.

Trade-off: performance (**`ldc.i4`**) и **`switch`** case labels vs **binary coupling**. Library **`Max = 100`**, App скомпилирован с inlined **`100`**. Library **`Max = 200`**, App **не пересобран** — может still use **`100`**. **`static readonly`** не inlines так в consumer.

**`const`** нельзя assign в constructor — value fixed at compile time в каждой assembly, которая inlined.

### readonly: immutability после construction (instance)

Instance **`readonly`** assign только в declaration или **constructors** того же type. После construction — запрет.

```csharp
public sealed class Order
{
    private readonly Guid _id;
    public Order() => _id = Guid.NewGuid();
}
```

Каждый instance — свой **`_id`**. **`DateTime.UtcNow`** legal здесь, illegal для **`const`**.

**`readonly`** на field struct не делает весь struct immutable. **`readonly struct`** — отдельная feature.

Instance **`readonly`** — slot в object layout на heap; external callers не inline через metadata.

### static readonly: runtime constants на уровне type

**`static readonly`**: one storage per type (AppDomain), **runtime initialization** в static field initializer или **`.cctor`**.

```csharp
public static class AppMetadata
{
    public static readonly string EnvironmentName =
        Environment.GetEnvironmentVariable("DOTNET_ENVIRONMENT") ?? "Production";
}
```

Callers read **same field** via **`ldsfld`**. Compiler **не inlines** **`static readonly`** из referenced assemblies как **`const`**.

**`static readonly` когда:** runtime computation, update value при deploy только defining DLL, shared reference types (**`Regex`**, arrays).

**`const` когда:** true forever literal, **attributes** (args must be constant), **`switch`** cases.

**Instance `readonly` когда:** value per object at creation (id, frozen injected dependency).

### Сравнение

| | `const` | `readonly` | `static readonly` |
|---|---------|------------|-------------------|
| Storage | Metadata literal | Per instance | Per type |
| Init | Compile time | Constructor | Static ctor |
| `DateTime.Now` | Illegal | OK in ctor | OK in static init |
| Consumer rebuild для нового value | Often **required** | Field load | Field load |
| Implicitly static | Yes | No | Yes |

### Initialization order

**`static readonly`** before first static access triggers **`.cctor`**. Instance **`readonly`**: field initializers, then constructor. **Type initializer cycles** → **`TypeInitializationException`**. DI: **`readonly IRepo _repo`** in constructor — frozen dependency reference.

## Under the Hood

### Metadata и IL для const

Consumer:

```csharp
int x = ApiConstants.DefaultPageSize;
// IL: ldc.i4 20 — не ldsfld
```

Число **вшито** в consumer. Attribute arguments — тот же compile-time encoding.

### IL для static readonly

```
ldsfld int32 Settings::ReadonlyTimeout
```

CLR runs **`.cctor`**, assigns field. New **`Settings.dll`** on deploy может изменить behavior без recompile consumer EXE.

### IL для instance readonly

**`newobj`**, constructor **`stfld`** into object. **`readonly`** — compile-time enforcement; reflection может обойти.

### JIT

**`const`** cross-assembly уже inlined at compile consumer. **`static readonly`** — normal static field load. Performance rarely matters; **versioning** drives choice.

### readonly vs init properties

**`init`** (C# 9) — property immutability, **`with`** on records. **`readonly field`** — lower level. Different IL, different patterns.

## Examples

### Library versioning trap

**`public const int MaxItems = 10`** → consumer inlines 10. DLL **`20`** без recompile App → still 10. Fix: **`static readonly`** or **`IOptions`**.

### Per-instance identity

**`Guid.NewGuid()`** в constructor — runtime, not **`const`**.

### Shared Regex

**`static readonly Regex Email = new(..., RegexOptions.Compiled)`** — single instance.

## Common Mistakes & Anti-patterns

**`const` для deploy-time changes**. **`const`** с runtime computation — не компилируется. Instance **`readonly`** для process-wide value. **`readonly` reference** не deep-freeze object. **`readonly struct` ≠ readonly field**. Static ctor throws → type broken forever.

---

## Production & Real-World Notes

Public **`const`** в NuGet — behavior change без signature change; semver/docs. ASP.NET **`IOptions<T>`** для tunable values. **`CA1802`**. Secrets не в **`const string`** (inlined everywhere).

---

## Comparison / Trade-offs

**`const` → static readonly → IOptions → remote config`**. **`Lazy<T>`** vs eager **`static readonly`**. **`record` + init** vs **`readonly` fields + ctor**.

---

## Quick Reference

```
const             → inlined in consumers, compile-time only
readonly          → per instance, ctor init
static readonly   → per type, ldsfld, runtime init OK

Library const change → consumers recompile
static readonly change → often redeploy DLL enough
```

---

## Key Takeaways

1. **`const`** inlines в referencing assemblies.
2. **`readonly`** — per instance, runtime expressions OK.
3. **`static readonly`** — shared, no cross-assembly inline stale.
4. **`const`** ≠ **`DateTime.Now`**, **`new`**.
5. Shared libraries: **`static readonly`** or options для tunable «constants».
6. **`readonly struct` / `init`** — другие immutability tools.
