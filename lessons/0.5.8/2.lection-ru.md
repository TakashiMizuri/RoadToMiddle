# Атрибуты валидации форм: `required`, `pattern`, `min`, `max`

> Roadmap: `0.5.8` · Node: `0.5` — HTML · Depth: practice

## Learning Objectives

После этой лекции ты сможешь:

- Применять **HTML5 constraint validation** на controls из `0.5.7`.
- Использовать **`required`**, **`min`/`max`/`step`**, **`minlength`/`maxlength`**, **`pattern`** по типу поля.
- Объяснить, как браузер блокирует submit и показывает native errors.
- Понимать, почему **HTML validation ≠ security** — server re-validates всегда.
- Стилизовать состояния через **`:valid`/`:invalid`** и связь с accessible errors.

## Why This Matters

Forms из `0.5.7` собирают data; validation решает, уйдут ли плохие данные в сеть. HTML5 **constraint validation API** даёт feedback без JS — пустые required, кривой email, quantity вне диапазона отсекаются до round-trip к ASP.NET Core и PostgreSQL.

Junior error — думать, что attributes достаточно. curl обходит DOM; браузеры чуть по-разному трактуют `pattern` и dates. Middle dev: HTML validation = **first-line UX**, authority = server + DB.

## Core Concepts

### Как работает constraint validation

Constraints на control; при submit браузер проверяет. Invalid → submit cancelled + native message.

API: **`checkValidity()`**, **`reportValidity()`**, **`setCustomValidity(msg)`**.

**`novalidate`** на form, **`formnovalidate`** на button — для «Save draft».

### `required`

Non-empty value; для checkbox — checked; для radio group — selected option; для select — не `value=""`.

### `minlength` / `maxlength`

Для text и textarea. Не путать с **`min`/`max`** для number/date.

### `min`, `max`, `step`

Для number, range, date, time, datetime-local. Dates: `YYYY-MM-DD`. **`step="any"`** для decimals на number.

### `pattern`

Regex **без** `/delimiters/`; match на **всю** строку. С **`title`** для подсказки в error UI.

```html
<input type="text" pattern="[A-Za-z]{1,2}[0-9][A-Za-z0-9]? ?[0-9][A-Za-z]{2}" title="Format like SW1A 1AA" />
```

### Built-in по `type`

`email`, `url`, `number` несут implicit checks + ваши attributes.

### CSS `:valid` / `:invalid`

**`:user-invalid`** / **`:user-valid`** — не красить empty required до interaction. Production: custom errors + **`aria-invalid`**, **`aria-describedby`**.

## Under the Hood

Submit → unless novalidate → ValidityState per control (`valueMissing`, `patternMismatch`, …) → invalid cancels submit.

Server: ASP.NET **DataAnnotations** `[Required]`, `[Range]`, `[RegularExpression]` — parallel layer, always on.

## Common Mistakes & Anti-patterns

Validation only in HTML; `min` on text; regex hell in `pattern`; `required` on hidden; trust `type="email"` for security.

## Production & Real-World Notes

React Hook Form + Zod; native messages на языке браузера — i18n через JS; JSON APIs без HTML attributes — FluentValidation.

## Quick Reference

```html
<input required minlength="2" maxlength="50" />
<input type="number" min="0" max="100" step="5" />
<input pattern="^#[0-9A-Fa-f]{6}$" title="#rrggbb" />
```

---

## Key Takeaways

- Client HTML = UX; server = truth.
- Length vs numeric bounds — разные attributes.
- `title` помогает pattern errors.
- Draft submit — `formnovalidate`.
