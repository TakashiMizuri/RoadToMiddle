# Answers: LINQ Any, All, Contains

### 1. **false**

### 2. **true** (vacuous — no element violates)

### 3. **Short-circuit** at first match; no need count all; clearer intent; better SQL EXISTS.

### 4. Empty order lines: **`Lines.All(l => Qty>0)`** true without lines — submit invalid order.

### 5. **`lines.Any() && lines.All(l => l.Qty > 0)`**

### 6. **`AnyAsync(c => c.Email == email)`**

### 7. **Unsafe** — empty roles → All true → treats as «no Admin» but may grant wrongly depending on logic.

### 8. **`roles.Contains("Admin", StringComparer.OrdinalIgnoreCase)`**

### 9. Often similar O(n); **Contains** on List direct; custom equality → **Any** with predicate.

### 10. **`Any(x => !valid(x))`** (De Morgan)

### 11. At least one invalid item exists.

### 12. **Immediate** terminal — runs enumeration/query when invoked.
