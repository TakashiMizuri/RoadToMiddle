# NuGet, transitive deps, CPM — краткая выжимка

> `1.5.16` · ~8 мин · практика

## Главная идея

**NuGet restore** строит **граф** direct + **transitive** packages. **CPM** (`Directory.Packages.props`) централизует версии в monorepo.

## Ключевые пункты

- **PackageReference** в SDK-style csproj.
- **project.assets.json** — полный closure.
- `dotnet list package --include-transitive --vulnerable`
- **`dotnet nuget why`** — зачем package в графе.
- **NU1605** — package downgrade conflict.
- **ManagePackageVersionsCentrally** + **PackageVersion**.

## Пример CPM

```xml
<!-- Directory.Packages.props -->
<PackageVersion Include="Serilog.AspNetCore" Version="8.0.3" />

<!-- App.csproj -->
<PackageReference Include="Serilog.AspNetCore" />
```

## Частые ошибки

- Разные версии одного package в solution без CPM.
- Floating versions в production CI.
- Игнор transitive vulnerabilities.

## Мнемоника

> **Restore reveals the whole tree — CPM owns the versions.**
