# Exercises: readonly struct, ref struct

> Complete after lectures and passing `4.test-yourself.md`.

## Easy

### E1. readonly record struct
**Goal:** `readonly record struct Rgb(byte R, byte G, byte B)` with `ToHex()` returning `#RRGGBB`.
**Success criteria:** No mutable fields; compiler enforces readonly.

### E2. in vs by-value
**Goal:** Dummy struct with 8 `long` fields; method `Sum(in Dummy d)` vs `Sum(Dummy d)` — observe intent in comments (benchmark optional).
**Success criteria:** Explain when `in` preferred.

## Medium

### M1. Span parsing
**Goal:** `static bool TryParseInt(ReadOnlySpan<char> digits, out int value)` without allocating substring.
**Success criteria:** Works for `"123"` slice of larger buffer.

### M2. stackalloc formatting
**Goal:** Method building uppercase trimmed string using `stackalloc char[]` + `Span<char>` for chars under 256 length.
**Success criteria:** No `char[]` allocation on stack path.

### M3. Memory + Span pattern
**Goal:** Class `BufferService` holding `byte[]` or `Memory<byte>`; method `Process()` creates local `Span<byte>` over buffer for mutation.
**Success criteria:** Span not stored in field; valid lifetime.

## Hard (stretch)

### H1. Compiler errors catalog
**Goal:** Write three intentionally invalid snippets (Span in class field, box Span, async+Span) and record compiler error codes/messages.
**Success criteria:** Each error explained with lifetime reasoning.

### H2. Immutable value object
**Goal:** `readonly struct Money` with `Add`, `Multiply`; unit test shows original unchanged after operations.
**Success criteria:** All mutations return new instances.
