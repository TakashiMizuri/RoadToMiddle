# WebSockets vs HTTP — краткая выжимка

> `0.3.15` · ~7 мин на повторение

## Главная идея

**HTTP** — client-initiated request-response. **WebSocket** после **101 Upgrade** — persistent **full-duplex** channel. **SSE** — проще **server→client** push на том же HTTP. Выбирай transport по direction, частоте и operational cost.

## Ключевые пункты

- Polling прост, но дорог при scale; long polling — held connections.
- **WebSocket:** chat, games, collab — частые сообщения **в обе стороны**.
- **SSE:** notifications, logs, progress — только server push.
- **SignalR:** WebSockets + SSE + long poll fallback; Redis **backplane** для scale-out.
- WS connections **sticky** — unlike stateless REST.
- Production: **`wss://`**, ping/pong, reconnect backoff, validate **Origin**.

## Самые важные примеры

```javascript
const ws = new WebSocket("wss://api.example.com/hubs/chat");
const es = new EventSource("/api/notifications/stream");
```

## Частые ошибки

- WebSocket «для всего real-time».
- Нет heartbeat → NAT drop silent conn.
- POST retry logic переносят на WS messages без idempotency.

## Мнемоника

> **HTTP = спросил–ответили. SSE = радио (слушаешь). WebSocket = телефон (оба говорят).**
