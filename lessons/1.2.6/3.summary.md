# Наследование: base, override, sealed, hidden (new) — краткая выжимка

> `1.2.6` · ~8 мин на повторение

## Главная идея

C# — **single inheritance** для классов. **`override`** подменяет **virtual** member и участвует в **runtime polymorphism**; **`new`** **скрывает** non-virtual member и dispatch-ится по **compile-time type** reference. **`base`** вызывает родителя; **`sealed`** останавливает override или наследование типа.

## Ключевые пункты

- **`class Derived : Base`** — один direct base; без указания → **`object`**.
- **`: base(args)`** — обязательная цепочка constructors; порядок: base fields → base ctor → derived fields → derived ctor.
- **`override`** только на **`virtual`/`abstract`/`override`** в base.
- **`new`** — hide; warning CS0108 без явного **`new`**.
- **`Report r = new DetailedReport(); r.Print()`** → base **`Print`**, если hide, не override.
- **`sealed override`** — дальше override нельзя; **`sealed class`** — subclassing запрещён.
- Virtual call из base constructor может попасть в override на **partially constructed** объекте.

## Самые важные примеры

```csharp
public Dog(string name, string breed) : base(name) { Breed = breed; }

public override void Speak() => Console.WriteLine("Woof");  // polymorphic

public new void Print() => ...;  // hides; static dispatch on reference type
```

## «Под капотом» в одном абзаце

Объект derived содержит **все fields** base в одном heap block. Virtual methods — **vtable slots**; **`override`** переназначает slot для runtime type. **`new`** добавляет отдельный method без замены slot — **`callvirt`** vs **`call`** в IL. Metadata: **`.override`** vs **`hidebysig`**.

## Частые ошибки

- Думали **`override`**, написали **`new`** (или base не **`virtual`**).
- **`new`** там, где нужен polymorphism — переименуй или сделай **`virtual`**.
- Deep inheritance вместо composition (`1.2.11`).

## Мнемоника

> **Polymorphism → `virtual` + `override`. Same name, no polymorphism → `new` (осторожно). Freeze → `sealed`.**
