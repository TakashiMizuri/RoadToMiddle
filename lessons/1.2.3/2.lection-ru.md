# Constructors, `: this()`, primary constructors

> Roadmap: `1.2.3` · Node: `1.2` — C#: ООП · Depth: **практика**

## Learning Objectives

После этого урока ты сможешь:

- Объяснить роль **constructors** в инициализации новых instances.
- Писать **instance constructors** и отличать их от ordinary methods.
- Связывать constructors через **`: this(...)`** и **`: base(...)`** (preview для `1.2.6`).
- Совмещать **constructor initializers** и **object initializers** без дублирования.
- Применять **primary constructors** (C# 12+) и понимать, что генерирует compiler.
- Выбирать между constructors, static factories и object initializers.

## Why This Matters

`new Customer()` аллоцирует память (`1.2.1`), но пока fields и invariants не заданы — object не trustworthy. **Constructors** выполняются сразу после allocation и переводят instance в **valid state**. Правила вроде «у каждого order есть customer id» — в construction paths, вместе с **`required`** из `1.2.2`.

Команды накапливают overloads или chained constructors, ведущие в **один canonical constructor**. Primary constructors сокращают boilerplate, но **capture semantics** parameters может удивить.

## Core Concepts

### Что такое constructor

**Constructor** — special method при **object creation**. **Имя = имя class**, нет return type, вызывается **`new`**:

```csharp
public Customer(int id, string name)
{
    Id = id;
    Name = name ?? throw new ArgumentNullException(nameof(name));
}
```

Если constructors **не объявлены** — compiler emit implicit **parameterless** constructor. После **любого** явного constructor parameterless **не генерируется**, пока не объявишь его сам.

Constructors **не наследуются** (`1.2.6`). Каждый class обеспечивает usable instances.

### Object initializer vs constructor

**Object initializer** выполняется **после** constructor body:

```csharp
var c = new Customer(1, "Ada") { /* init-only props */ };
```

Constructor — **invariants**; initializer — optional/supplemental. Не дублируй validation без причины.

### Chaining `: this()`

**`: this(...)`** — первый statement, делегирует sibling constructor:

```csharp
public Rectangle(double side) : this(side, side) { }

public Rectangle(double width, double height)
{
    if (width <= 0 || height <= 0) throw new ArgumentOutOfRangeException();
    Width = width; Height = height;
}
```

Избегай copy-paste validation. Нельзя одновременно `: this()` и `: base()` на одном constructor.

### `: base()` — preview

```csharp
public PremiumCustomer(int id, string name, int tier) : base(id, name)
{
    Tier = tier;
}
```

**`: base(...)`** до derived body. Каждый derived constructor ultimately вызывает base constructor.

### Primary constructors (C# 12+)

```csharp
public class Point(int x, int y)
{
    public int X { get; } = x;
    public int Y { get; } = y;
    public Point() : this(0, 0) { }
}
```

DI-style:

```csharp
public class OrderService(IOrderRepository repo)
{
    public void Ship(Guid id) => repo.MarkShipped(id);
}
```

**Caution:** reference-type parameters могут **capture** — осторожно с mutability.

### Private constructor + factory

```csharp
private Money(decimal amount) => Amount = amount;
public static Money FromMajorUnits(decimal amount) => ...
```

Factory именует intent и контролирует creation.

### Static constructor — preview

**Static constructor** (`static ClassName()`) инициализирует type-level state один раз — подробнее `1.2.12`.

### `required` и constructor вместе

```csharp
public class Invoice
{
    public required Guid Id { get; init; }
    public required decimal Total { get; init; }

    public Invoice(Guid id, decimal total)
    {
        Id = id;
        Total = total >= 0 ? total : throw new ArgumentOutOfRangeException();
    }
}
```

Constructor может assign **required** members; compiler проверяет completeness при object creation.

## Examples

**Example 1 — canonical constructor с chaining:**

```csharp
public class EmailAddress
{
    public string Value { get; }

    public EmailAddress(string value)
    {
        if (!value.Contains('@'))
            throw new ArgumentException("Invalid email");
        Value = value.Trim().ToLowerInvariant();
    }
}
```

Validation в одном месте; overloads могут делегировать через `: this(...)`.

**Example 2 — primary constructor service:**

```csharp
public class Greeter(string prefix)
{
    public string Greet(string name) => $"{prefix}, {name}!";
}
```

**Example 3 — explicit parameterless для serializers:**

```csharp
public class Config
{
    public string Environment { get; set; } = "Development";
    public Config() { }
}
```

Некоторые serializers требуют parameterless ctor; знай контракт своего stack.

## Common Mistakes & Anti-patterns

**God constructor** с десятками parameters — builder, record или grouped value objects.

**Circular `: this()` chain** — compile error; проектируй DAG делегирования.

**Throw в constructor** после partial base init — unusable object; иногда factory с pre-validation лучше.

**Primary ctor + mutable injected singleton** — shared mutation через captured reference.

**Parameterless + mutable setters** обходят invariants — prefer `required` + ctor/factory.

## Production & Real-World Notes

DI containers вызывают **public constructor** с resolvable dependencies; primary constructors хорошо ложатся на ASP.NET Core injection. EF Core часто нужен parameterless ctor (может быть private) для materialization entities. Unit tests строят valid objects через ctor или object initializer.

## Comparison / Trade-offs

| Подход | Плюсы | Минусы |
|--------|-------|--------|
| Single ctor + validation | Один invariant gate | Verbose при many optional |
| `: this()` chaining | DRY | Indirection |
| Object initializer | Readable optional fields | Может обойти rules без enforcement |
| Primary constructor | Меньше boilerplate | Capture semantics |
| Static factory | Named intent | Extra discovery step |

## Quick Reference

```
Constructor → имя class, no return
: this()    → sibling ctor
: base()    → base ctor
Primary     → params на class (C# 12)
```

---

## Key Takeaways

- Constructors **initialize** после `new`.
- **`: this()`** — DRY validation.
- **Primary constructors** — меньше boilerplate, знай capture.
- **Factory + private ctor** — controlled creation.
