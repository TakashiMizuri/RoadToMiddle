# .NET SDK и структура проекта — краткая выжимка

> `1.1.1` · ~8 мин на повторение

## Главная идея

**.NET SDK** — toolchain для разработки (Roslyn, MSBuild, шаблоны, NuGet, CLI **`dotnet`**). **`dotnet new console`** создаёт SDK-style project (`.csproj` + `Program.cs`); **`dotnet build`** компилирует в IL assembly в `bin/`; **`dotnet run`** собирает и запускает entry point.

## Ключевые пункты

- **SDK** — для build/publish; **runtime** — для запуска; **TFM** (`net8.0`) — целевой API/runtime в `.csproj`.
- Файлы: **`*.csproj`** (манифест), **`Program.cs`** (entry), **`bin/`** (output), **`obj/`** (intermediate) — последние два не коммитят.
- **`OutputType`**: `Exe` / `Library`; **`ImplicitUsings`** и **`Nullable`** — стандарт modern template.
- Зависимости — **`PackageReference`** + `dotnet add package`; restore в global NuGet cache.
- Top-level statements → compiler генерирует **`Main`**.
- **Solution** (`.sln`) группирует несколько projects — позже в roadmap.

## Самые важные примеры

```bash
dotnet new console -n MyApp -o MyApp
cd MyApp
dotnet run
```

```csharp
string name = args.Length > 0 ? args[0] : "World";
Console.WriteLine($"Hello, {name}!");
```

## «Под капотом» в одном абзаце

`<Project Sdk="Microsoft.NET.Sdk">` подтягивает MSBuild targets: какие `.cs` компилировать, куда писать output. Roslyn: source → syntax/semantic → IL → `.dll`. CLR JIT-ит IL при запуске. Shared framework assemblies (`System.Console`) идут с runtime, не копируются в FDD deploy.

## Частые ошибки

- Ставить только runtime на dev-машину.
- Коммитить `bin/` / `obj/`.
- Путать версию SDK и TFM.
- `dotnet run` не из каталога project без `--project`.

## Мнемоника

> **new** — создать, **build** — собрать, **run** — запустить; **csproj** — контракт project, **TFM** — «на каком .NET живёт».
