### 4. [Understanding]
**Answer:** **`double` — IEEE 754 binary**; 0.1 и 0.2 не имеют exact binary representation → сумма округляется с погрешностью.

**Why:** Fundamental binary float limit, not random bug.

### 5. [Understanding]
**Answer:** **Value types** — assignment **копирует значение** (независимые copies). **Reference types** — assignment **копирует ссылку** на один heap object.

### 9. [Application]
**Answer:** **`a * b` вычисляется в `int`** и может overflow **до** cast. Безопасно: **`(long)a * b`** или **`long.Multiply`** / checked logic.

**Common mistake:** Cast после multiply.

### 12. [Understanding]
**Answer:** Упаковка `int` в **`object`** на heap с копией value; unbox извлекает copy.

**Why:** Value types не наследуют `object` directly without boxing.

---

### 13. [Application]
**Answer:** **`long`** — ms since epoch уже > `int.MaxValue` (~2038 problem для seconds; ms for years exceeds int quickly).

---

### 14. [Analysis]
**Answer:** **`float`** — 4 bytes vs 8 → ~40 MB vs ~80 MB for 10M points; **`double`** — better precision for coordinates. Trade-off memory vs accuracy.

### 15. [Understanding]
**Answer:** **`TryParse`** не бросает exception на invalid input — возвращает `false`, безопаснее на user input и hot paths.
