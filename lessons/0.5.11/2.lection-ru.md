# Доступность: ARIA roles, keyboard navigation, focus

> Roadmap: `0.5.11` · Node: `0.5` — HTML · Depth: **глубоко**

## Learning Objectives

После этого урока ты сможешь:

- Сформулировать **first rule of ARIA**: native HTML раньше roles и attributes.
- Отличать **implicit roles** от explicit `role` и знать, когда explicit помогает или вредит.
- Делать custom widgets **keyboard-operable**: tabindex, roving tabindex, key handlers по ARIA APG.
- Управлять **focus order**, **focus trap** в modal, **focus restoration** при закрытии.
- Объяснить движение **focus** и разницу `:focus-visible` vs `:focus`.
- Связать ARIA (`aria-label`, `aria-expanded`, `aria-live`) с accessibility tree из DOM (`0.5.10`).

## Why This Matters

Accessibility — не слой «после того как мышь работает». Тот же UI через **keyboard**, **screen reader**, voice control, switch devices. Таблицы с `scope` (`0.5.9`) и DOM (`0.5.10`) уже показали, как browser строит a11y tree. Здесь — когда native elements не хватает и почему **focus** — «курсор» keyboard и AT — должен быть осознанным.

React-библиотеки превращают `<button>` в `<div role="button">` без keyboard. Razor dropdowns без `aria-expanded`. Middle dev ловит это в review, чинит modal traps, выбирает **semantic HTML first**. Keyboard — ещё и power-user UX.

## Core Concepts

### Semantic HTML first, ARIA second

**WAI-ARIA** дополняет HTML, когда native element не выражает role/state/properties. **First rule**: если есть native с semantics и behavior — **не** меняй ARIA. `<button>` — role, Tab, Enter/Space, disabled бесплатно. `<div role="button" tabindex="0">` — всё implement сам.

ARIA для **composite widgets** (tabs, combobox) или **augmentation** (`aria-describedby` для ошибки). Не для починки там, где хватило бы `<button>`.

### Roles: implicit и explicit

У узла a11y tree есть **role**. HTML tags дают **implicit roles**. Explicit **`role`** — осторожно; конфликты (heading на div без `aria-level`).

Landmarks — prefer `<nav>`, `<main>`, `<aside>`. Widget roles — button, tab, dialog. Screen reader: "Edit, Email" для input type=email. Wrong role на div — misleading.

### Keyboard navigation

**Tab** / **Shift+Tab** — между **tabbable**: links, buttons, fields, `tabindex="0"`. **`tabindex="-1"`** — вне tab order, но `.focus()` programmatically.

Tab order ≈ **DOM order**; CSS `order` не меняет Tab без reorder DOM. **`tabindex > 0`** — антипаттерн.

Patterns из **ARIA APG**: button — Enter/Space; tabs — arrows; dialog — trap, Escape, restore focus.

### Focus management

**Focus** — active element для keyboard events. Modal open → focus inside; **trap** Tab; close → **restore** к trigger. **Roving tabindex** в tablist/toolbar — один `tabindex="0"`, остальные `-1`, arrows swap.

### Essential ARIA attributes

`aria-label`, `aria-labelledby`, `aria-describedby`, `aria-expanded`, `aria-selected`, `aria-hidden`, `aria-live`, `aria-disabled`, `aria-modal`.

**Accessible name** — visible text, label, aria-label. **`aria-live`** — toasts, errors без обязательного focus steal.

## Under the Hood

### DOM → accessibility tree

AX tree: role, name, state, relations. Decorative `alt=""` pruned; text merged в names. OS APIs → NVDA, VoiceOver. **`aria-hidden="true"`** на ancestor скрывает subtree — **never** на focusable focused content.

### Focus ring и :focus-visible

`outline: none` без замены — WCAG 2.4.7 fail. **`:focus-visible`** — ring для keyboard, не всегда для mouse click.

### tabindex algorithm

Positive tabindex first (numeric order), then 0/undefined in DOM order, -1 skipped. Shadow DOM — focus across boundaries; Web Components — delegatesFocus.

### ARIA в React/SSR

`aria-expanded` must match hydration. eslint jsx-a11y — partial guard.

## Syntax / Commands / API

```html
<button aria-expanded="false" aria-controls="menu">Menu</button>
<div role="dialog" aria-modal="true" aria-labelledby="dlg-title">...</div>
<div role="status" aria-live="polite"></div>
```

Custom button keydown: Enter/Space → click.

## Key Takeaways

- **First rule of ARIA** — native HTML.
- **Roles** для AT; wrong role хуже на native tags.
- **Keyboard** обязателен; APG для custom widgets.
- **Focus** management on open/close/route.
- **tabindex="-1"** programmatic; avoid tabindex > 0.
- **aria-*** — name, state, live updates.
- A11y tree from DOM — align both.
