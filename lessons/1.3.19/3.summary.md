# Local functions vs private methods — краткая выжимка

> `1.3.19` · ~7 мин на повторение

## Главная идея

**Local function** объявляется внутри другого member и видна только там; **private method** — часть класса и переиспользуется между members. Выбор — про локальность workflow vs reuse и тестируемость.

## Ключевые пункты

- Local function **не** member класса; вызов только из enclosing method/accessor.
- **Private method** — когда helper нужен в 2+ местах или требует изолированных тестов.
- **`static` local function** — без capture; данные через parameters.
- Non-static local может **capture** outer locals (как lambda).
- Local function поддерживает **`ref`/`out`**, **`yield`**, **`async`**; lambda — когда нужен **delegate** для LINQ.
- Компилятор генерирует nested methods с mangled names в stack trace.

## Самые важные примеры

```csharp
// Local — один workflow
decimal Calc(Order o)
{
    decimal Rate() => o.Weight * 0.5m;
    return Rate();
}

// static local — pure helper
static bool IsValidSku(string sku) => sku.Length >= 3;
```

## Частые ошибки

- Local function вместо private при reuse в нескольких methods.
- 50+ строк nested logic — extract class/private method.
- Over-capture вместо explicit parameters.

## Мнемоника

> **Один раз здесь → local. Несколько раз в классе → private. Без outer state → static local.**
