# .gitattributes и line endings — краткая выжимка

> `0.2.14` · ~7 мин

## Главная идея

**LF** vs **CRLF** — разные bytes → шумные diff и merge. **`.gitattributes`** задаёт normalization в repo (обычно **LF in Git**). После смены policy — **`git add --renormalize .`**.

## Ключевые пункты

- **LF** `\n` — Unix/macOS/CI; **CRLF** `\r\n` — Windows.
- **`* text=auto eol=lf`** — типичный cross-platform default.
- **`*.png binary`** — не трогать EOL.
- **`core.autocrlf`** — дополнение; **repo rules важнее**.
- **EditorConfig** `end_of_line = lf` — редактор, не Git.
- **`git ls-files --eol`**, **`git diff --check`**.

## Примеры

```gitattributes
* text=auto eol=lf
*.sh text eol=lf
*.png binary
```

```bash
git add --renormalize .
git commit -m "chore: normalize line endings"
```

## Частые ошибки

Только autocrlf без `.gitattributes`; renormalize без координации; binary не помечен.

## Мнемоника

> **Repo LF** in objects · **attributes** = law · **renormalize** = one-time cleanup
