## Answers

### 1. [Recall]
**Answer:** `action`, `method`, `enctype` (для post).
**Why:** URL destination, GET vs POST, body encoding.

### 2. [Recall]
**Answer:** `id` — document identity для label/JS/CSS; `name` — key в submitted form data / model binding.
**Why:** Разные роли в DOM vs HTTP payload.

### 3. [Recall]
**Answer:** `multipart/form-data`
**Why:** Binary file parts требуют MIME multipart, не urlencoded.

### 4. [Understanding]
**Answer:** Placeholder исчезает при вводе; часто низкий contrast; AT не заменяет persistent label.
**Why:** Accessible name должна быть stable.

### 5. [Understanding]
**Answer:** Browser treats same `name` radios as mutually exclusive group; один selected value submit’ится.
**Why:** Radio semantics — one of set.

### 6. [Understanding]
**Answer:** Нет, не попадает.
**Why:** Server должен трактовать absence как false/off, не полагаться на presence only.

### 7. [Application]
**Answer:** `type="tel"`, например `autocomplete="tel"` или `tel-national` по контексту.
**Why:** tel keyboard; autocomplete helps autofill.

### 8. [Application]
**Answer:** `method="get"`, input `name="q"` (или осмысленное имя), `action` на search endpoint.
**Why:** GET кодирует params в URL для bookmark/share.

### 9. [Application]
```html
<fieldset>
  <legend>Address type</legend>
  <label><input type="radio" name="addressType" value="billing" /> Billing</label>
  <label><input type="radio" name="addressType" value="shipping" /> Shipping</label>
</fieldset>
```

### 10. [Analysis]
**Answer:** Нет accessible name когда placeholder ушёл; smaller hit target без label click; SR may read placeholder unreliably as name.
**Why:** WCAG requires labels.

### 11. [Analysis]
**Answer:** File content не передаётся корректно — server получит broken/missing binary или только filename string.
**Why:** Default enctype не для files.

### 12. [Analysis]
**Answer:** Нет; `type="button"` (или link вне form) — иначе submit form.
**Why:** Default button type in form is submit in HTML.
