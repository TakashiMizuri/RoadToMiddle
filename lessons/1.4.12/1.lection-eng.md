# Task.WhenAll and Task.WhenAny ‚ÄĒ Parallel I/O

> Roadmap: `1.4.12` ¬∑ Node: `1.4` ‚ÄĒ Async/await ¬∑ Depth: **–Ņ—Ä–į–ļ—ā–ł–ļ–į**

## Learning Objectives

After this lesson you will be able to:

- Run **multiple independent I/O operations** concurrently with **`Task.WhenAll`** and await a single composite task.
- Use **`Task.WhenAny`** for **first-completed** patterns (timeout race, redundant providers).
- Pass **`CancellationToken`** to all child operations and handle **`OperationCanceledException`** (`1.4.11`).
- Avoid **sequential await** anti-pattern when calls are independent.
- Handle **`WhenAll` exceptions**: one fault fails aggregate; inspect **`task.Exception`** or await individually.
- Understand that parallel async I/O does not mean extra threads for waiting ‚ÄĒ continuations still use the pool (`1.4.7`).

---

## Why This Matters

A dashboard loads user profile, permissions, and notifications with three HTTP calls. Sequential awaits add latencies; **`Task.WhenAll`** overlaps I/O so total time approaches **max**, not **sum**. Wrong exception handling loses which call failed; missing **cancellation** leaves orphan requests when the user navigates away.

This is daily Middle work: API aggregation, batch fetches, graceful degradation with **`WhenAny`**.

## Core Concepts

### Sequential vs Parallel I/O

**Sequential** (independent calls ‚ÄĒ wasteful):

```csharp
var a = await GetAAsync();
var b = await GetBAsync();
var c = await GetCAsync();
// Time ‚Čą Ta + Tb + Tc
```

**Parallel**:

```csharp
var aTask = GetAAsync(ct);
var bTask = GetBAsync(ct);
var cTask = GetCAsync(ct);
await Task.WhenAll(aTask, bTask, cTask);
var a = aTask.Result; // safe AFTER WhenAll completes
// Time ‚Čą max(Ta, Tb, Tc)
```

Prefer reading results via **`await`** each task after WhenAll or use tuple deconstruction pattern:

```csharp
await Task.WhenAll(aTask, bTask);
var a = await aTask; // completed ‚ÄĒ runs synchronously
```

Each `GetXAsync` returns a **`Task`** immediately; work starts before awaits. State machines for three methods interleave on thread pool as I/O completes.

### Task.WhenAll

**`Task.WhenAll(IEnumerable<Task>)`** returns **`Task`** or **`Task<TResult[]>`** when tasks are homogeneous generic.

- All succeed ‚Üí completes when **all** complete.
- One faults ‚Üí returned task **faults**; other tasks may still run to completion unless canceled.
- **`await WhenAll`** throws **first** exception (not `AggregateException` on single await in async method ‚ÄĒ unwrap behavior).

To collect all errors:

```csharp
var tasks = new[] { A(), B(), C() };
try { await Task.WhenAll(tasks); }
catch
{
    foreach (var t in tasks)
        if (t.IsFaulted) Log(t.Exception);
    throw;
}
```

Pass **`CancellationToken`** into each child so **`WhenAll`** cancellation propagates cooperatively.

### Task.WhenAny

**`Task.WhenAny`** completes when **first** task completes (success, fault, or cancel). Use cases:

- **Timeout race:** `await Task.WhenAny(work, Task.Delay(timeout, ct))`
- **Fallback:** first responsive microservice wins
- **Progress:** first chunk ready

After `WhenAny`, **cancel or ignore** losing tasks to avoid wasted work ‚ÄĒ often link CTS and cancel others when winner found.

```csharp
var winner = await Task.WhenAny(providerA.Get(), providerB.Get());
var result = await winner; // may throw if winner faulted
```

### Parallel I/O vs Parallel CPU

**WhenAll** for I/O ‚ÄĒ good. For **CPU-bound** work on many items, unbounded **`Task.Run` + WhenAll** can **starve thread pool** (`1.4.10`). Use **`Parallel.ForEachAsync`**, **`SemaphoreSlim`**, or **`Channel`** with limits (`1.4.25`, `1.4.18`).

## Syntax / API

```csharp
await Task.WhenAll(task1, task2, task3);

Task<TResult[]> all = Task.WhenAll(listOfTasks);

Task<Task> any = Task.WhenAny(tasks); // .NET: Task<Task> ‚ÄĒ unwrap
Task<TTask> anyT = Task.WhenAny(taskList);

// ValueTask ‚ÄĒ often convert to Task first or await individually
```

## Examples

### Aggregating API Data

```csharp
public async Task<DashboardDto> LoadDashboardAsync(int userId, CancellationToken ct)
{
    var profileTask = _users.GetProfileAsync(userId, ct);
    var permsTask = _auth.GetPermissionsAsync(userId, ct);
    var notifTask = _notify.GetUnreadAsync(userId, ct);

    await Task.WhenAll(profileTask, permsTask, notifTask);

    return new DashboardDto
    {
        Profile = await profileTask,
        Permissions = await permsTask,
        Notifications = await notifTask
    };
}
```

Three HTTP calls overlap; each async method‚Äôs state machine resumes independently on completions.

### Timeout with WhenAny

```csharp
using var cts = CancellationTokenSource.CreateLinkedTokenSource(userToken);
cts.CancelAfter(TimeSpan.FromSeconds(5));

var work = _slow.QueryAsync(cts.Token);
var delay = Task.Delay(Timeout.Infinite, cts.Token); // or race with Delay(5000)

var completed = await Task.WhenAny(work, Task.Delay(5000, cts.Token));
if (completed != work)
    throw new TimeoutException();
return await work;
```

(Cleaner: **`Polly` timeout** or **`WaitAsync`** on .NET 6+.)

## Common Mistakes & Anti-patterns

**`WhenAll` with `async` lambda that sequentially awaits inside** ‚ÄĒ still serial if you built one task.

**Using `.Result` on incomplete tasks** before WhenAll ‚ÄĒ blocks and risks deadlock (`1.4.14`).

**Ignoring losing tasks after `WhenAny`** ‚ÄĒ background work continues.

**WhenAll on 10_000 CPU `Task.Run`s** ‚ÄĒ pool starvation.

**Not handling partial failure** in batch imports ‚ÄĒ one fault fails entire WhenAll.

---

## Production & Real-World Notes

Add **circuit breaker** per dependency when one slow service blocks dashboard **`WhenAll`** ‚ÄĒ consider **`WhenAny`** + fallback or cached defaults.

Log **per-task timing** to find straggler in WhenAll bundle.

---

## Key Takeaways

- Independent I/O ‚Üí start tasks, then **`Task.WhenAll`**.
- Latency ‚Čą **max**, not sum.
- **`WhenAny`** ‚ÄĒ first wins; cancel losers.
- Pass **`CancellationToken`** to every child (`1.4.11`).
- Exception: one fault fails WhenAll await ‚ÄĒ plan logging.
- CPU parallel needs **bounds**, not raw WhenAll on Task.Run.
