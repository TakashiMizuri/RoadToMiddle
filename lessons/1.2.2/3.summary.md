# Fields и properties — краткая выжимка

> `1.2.2` · ~8 мин на повторение

## Главная идея

**Field** — storage внутри object (обычно **private**). **Property** — controlled API (`get`/`set`) для доступа к state. В production почти всегда expose **properties**, не fields.

## Ключевые пункты

- **Auto-property** `{ get; set; }` — compiler создаёт hidden backing field.
- **Full property** — свой `_field` + validation/computed logic в accessors.
- **`init`** — set только при object initialization; после ctor frozen.
- **`required`** — compiler требует assign при создании object.
- **`{ get; private set; }`** — mutate только внутри type.
- Property initializers: `= true`, `= new()` — per-instance defaults.

## Самые важные примеры

```csharp
public string Sku { get; set; }                    // auto
public required string Name { get; init; }         // init + required
private decimal _bal;
public decimal Balance { get => _bal; private set => _bal = value; }
public double Area => Width * Height;              // computed, no storage
```

## Частые ошибки

- Public fields вместо properties.
- Public `set` на `List<T>` — replace/clear снаружи.
- Validation только в UI, не в property/domain.

## Мнемоника

> **Field — сейф; property — окошко с правилами доступа.**
