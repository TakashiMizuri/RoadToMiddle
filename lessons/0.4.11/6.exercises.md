## Medium

### M1. Back-of-envelope
**Goal:** Bandwidth math.

100 Mbps link. How many seconds for 12 MB file (ignore overhead)? Show calculation (bits vs bytes).

**Success criteria:** ~1 second order (12 MB = 96 Mbit; 96/100 ≈ 0.96 s).

---

### M2. Fix recommendation
**Goal:** Apply concepts.

Problem A: 3 KB API, p99 500 ms, DB in same AZ. Problem B: Users download 80 MB export, takes 2 min on 100 Mbps. One fix idea each — different metrics.

**Success criteria:** A = query/index/CPU latency; B = bandwidth/compression/chunked CDN.

---

## Hard (stretch)

### H1. Waterfall analysis
**Goal:** Connect to frontend perf.

A page makes 8 serial API calls each 60 ms server+RTT before render. Bundle download 200 ms parallel on CDN. What dominates? One optimization strategy.

**Success criteria:** Serial API ~480 ms dominates; batch/combine endpoints or parallel fetch.

## Hints

<summary>Hint M1 bytes vs bits</summary>
Mbps = megabits per second. 1 byte = 8 bits. 12 MB = 12 × 8 = 96 megabits.
</details>
