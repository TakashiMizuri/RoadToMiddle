## Answers

### 1. [Recall]
**Answer:** Public method **PascalCase**; parameter **camelCase**; private field **`_camelCase`** (common convention).
**Why:** FDG + modern .NET style.

### 2. [Recall]
**Answer:** Interface **`I` + PascalCase** (`IOrderService`); async method **`OperationAsync`** returning Task.
**Why:** Framework Design Guidelines.

### 3. [Understanding]
**Answer:** Suffix signals awaitable API at call site; tooling/analyzers; consistency with BCL (`File.ReadAllTextAsync`).
**Why:** Name communicates usage pattern beyond type alone.

### 4. [Understanding]
**Answer:** Adds no info beyond signature; noise in IntelliSense; will drift.
**Why:** Document behavior, constraints, units — not obvious getters.

### 5. [Understanding]
**Answer:** Documents thrown exception; **`cref`** must resolve to existing type for doc compiler/IDE link.
**Why:** Consumer knows failure modes.

### 6. [Application]
**Answer:** e.g. **`ProcessOrder`**, **`IRepository`**, **`GetDataAsync`** (if async) or **`GetOrderById`**, **`_logger`**.
**Why:** Verbs, no redundant Interface suffix, async suffix, underscore field.

### 7. [Application]
**Answer:** `<summary>`, `<param name="id">`, `<param name="cancellationToken">` (match names!), `<returns>` describing null when not found.
**Why:** Complete public API doc set.

### 8. [Application]
**Answer:** `<GenerateDocumentationFile>true</GenerateDocumentationFile>` in csproj PropertyGroup.
**Why:** MSBuild emits XML alongside assembly.

### 9. [Analysis]
**Answer:** **`param name="orderID"`** ≠ **`orderId`** — doc warning / broken param doc.
**Why:** Must match identifier exactly (case-sensitive).

### 10. [Analysis]
**Answer:** Private implementation details, algorithms, TODOs — not part of public contract tooling.
**Why:** **`///`** for exported API surface.

### 11. [Understanding]
**Answer:** IDE shows first sentence alone in tooltip — gist must fit there.
**Why:** UX of IntelliSense.

### 12. [Application]
**Answer:** **`CanEdit`**, **`CanEditPost`**, or **`IsEditable`** depending on domain clarity — prefer **`CanEdit`** for capability.
**Why:** Boolean as question.

### 13. [Understanding]
**Answer:** Single source for OpenAPI descriptions; Swagger UI shows same text as IntelliSense.
**Why:** DRY for API documentation.

### 14. [Recall]
**Answer:** **`TKey`** (with **`TValue`** for dictionary).
**Why:** Descriptive generic parameter naming.

## Model answer for practical

```csharp
/// <summary>
/// Applies a percentage discount to a price and returns the discounted amount.
/// </summary>
/// <param name="price">The original price; must be non-negative.</param>
/// <param name="discountPercent">Discount from 0 to 100 inclusive.</param>
/// <returns>The price after discount; never less than zero.</returns>
public static decimal ApplyDiscount(decimal price, decimal discountPercent)
```

Optional **`<exception>`** if method throws on invalid range.
