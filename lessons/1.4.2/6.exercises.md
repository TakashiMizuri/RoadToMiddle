# Exercises: I/O-bound vs CPU-bound

## Easy

### E1. Classify endpoints
**Goal:** Label I/O vs CPU for 5 fictional API routes.
**Steps:** Write table: route, dominant bound, recommended pattern.
**Success criteria:** At least 2 I/O with `await`; 2 CPU with offload or sync honesty.

### E2. Spot fake async
**Goal:** Find anti-patterns in snippet list (provided by you or AI).
**Success criteria:** Explain why each blocks pool.

## Medium

### M1. Load test sketch
**Goal:** Design experiment sync vs async HTTP proxy.
**Steps:** Two minimal endpoints; describe expected p99 under 100 concurrent.
**Success criteria:** Predict pool exhaustion for sync.

### M2. Refactor proposal
**Goal:** Given sync repository, propose true async migration checklist.
**Success criteria:** Mention driver, EF async, no Task.Run on I/O.

## Hard

### H1. Mixed workload design
**Goal:** API: fetch order (DB) + calculate tax (CPU) + send email (I/O).
**Steps:** Sequence diagram with thread usage per phase.
**Success criteria:** Correct await/offload per phase.
