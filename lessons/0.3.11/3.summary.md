# Content types: JSON, urlencoded, multipart — краткая выжимка

> `0.3.11` · ~7 мин на повторение

## Главная идея

**`Content-Type`** говорит, как интерпретировать bytes в HTTP body. **`application/json`** — structured data для REST; **`application/x-www-form-urlencoded`** — плоские формы и OAuth tokens; **`multipart/form-data`** — поля + файлы через boundary.

## Ключевые пункты

- Без согласованного **Content-Type** server не знает, какой parser применить → 415 или пустой model.
- **JSON** + `[FromBody]` — стандарт SPA ↔ ASP.NET Core API.
- **Urlencoded** + `[FromForm]` — HTML forms, **`/connect/token`** (OAuth 2.0).
- **Multipart** + `IFormFile` — upload без Base64 overhead.
- **`Accept`** (client) зеркалит идею для response format.

## Самые важные примеры

**JSON POST:**
```bash
curl -H "Content-Type: application/json" -d '{"title":"Task"}' /api/tasks
```

**OAuth-style form:**
```bash
curl -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials&client_id=..." /token
```

**File upload:**
```bash
curl -F "file=@photo.jpg" -F "title=Vacation" /api/upload
```

## Частые ошибки

- JSON на endpoint с `[FromForm]` (все поля null).
- OAuth token POST как JSON (spec violation).
- Доверие `Content-Type` файла от client без server-side validation.

## Мнемоника

> **JSON — данные. Urlencoded — форма. Multipart — форма + файл.**
