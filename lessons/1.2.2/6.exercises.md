# Exercises: Fields and Properties

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Auto to full
**Goal:** Refactor without breaking callers.

Start with `public string Name { get; set; }`. Refactor to trim whitespace and reject empty strings in setter. Call sites stay `obj.Name = " Ada "`.

**Success criteria:** `"Ada"` stored; empty throws.

---

### E2. Required init DTO
**Goal:** Compiler-enforced creation.

Define `CreateOrderCommand` with `required Guid CustomerId`, `required int ItemCount`, optional `string? Note`. Show one valid and one invalid construction at compile time (comment invalid).

**Success criteria:** CS9035 on invalid example.

---

## Medium

### M1. Temperature conversion
**Goal:** Computed + backing field.

Class `Temperature` with private `_celsius`, property `Celsius`, computed `Fahrenheit` get/set converting both ways.

**Success criteria:** Round-trip within reasonable precision.

---

### M2. Read-only exposure
**Goal:** `private set` pattern.

`Order` with `Status { get; private set; } = "Draft"`. Methods `Submit()` and `Cancel()` change status with rules. External code cannot assign Status directly.

**Success criteria:** Compiler blocks external set.

---

## Hard (stretch)

### H1. Collection encapsulation
**Goal:** Fix leaky API.

Given public `List<string> Lines { get; set; }`, redesign so callers cannot replace list without your method but can read lines. Discuss trade-offs (expose `IReadOnlyList`, copy on get, etc.).

**Success criteria:** Identifies mutation/replace risks and one mitigation.

---
