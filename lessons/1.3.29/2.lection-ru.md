## Why This Matters

Сила LINQ — **lazy pipelines**: каждый stage работает только когда consumer pulls next item. Тот же стиль для domain sources: paginated API, file chunks, tree traversal.

**`yield return`** — синтаксис **cooperative** producer/consumer: method **pauses** после каждого value и resumes на **`MoveNext()`**. Понимание **`yield`** → readable ETL, streaming parsers, test fakes без **`List<T>`**. Та же модель у **`IAsyncEnumerable`** в ASP.NET streaming.

## Core Concepts

### IEnumerable vs IEnumerator

**`IEnumerable<T>`** — **`GetEnumerator()`** factory.

**`IEnumerator<T>`** — **`MoveNext()`**, **`Current`**, **`Dispose`**.

**Foreach** → **`using`** + loop; **`Dispose`** closes files/network.

### yield return

```csharp
static IEnumerable<int> NaturalsFrom(int start)
{
    var n = start;
    while (true)
        yield return n++;
}
```

**`Take(5)`** stops pull — infinite loop не уходит дальше пяти yields (iterator disposed).

### yield break

Early termination в iterator.

### Compose с LINQ

Custom **`IEnumerable`** — full LINQ. **`Take`** stops после N **outputs**, но filtered rejects всё ещё **consume** source до N matches.

### Mutable state

Shared mutable fields + iterator — bugs при reuse. Each **`GetEnumerator()`** — new state machine, но shared fields остаются.

### try/finally

**`using`** в iterator — compiler emits cleanup on early **`foreach`** break.

## Under the Hood

### State Machine

Roslyn → nested class с **`_state`**, fields вместо locals across yields. Каждый **`yield return`** — state label.

Debug: **`<Method>d__N`** в Debug build.

### Multiple enumerators

Two **`GetEnumerator()`** — independent cursors. Two **`foreach`** на cold source — **two passes** I/O.

### Cold vs Hot

LINQ operators — **cold** unless **`ToList`/`ToArray`**.

### IAsyncEnumerable

**`async IAsyncEnumerable`** + **`await foreach`** — async pull model для EF **`AsAsyncEnumerable`**, streaming APIs.

## Examples

### Paginated API

```csharp
IEnumerable<Product> AllProducts(IProductApi api) {
    string? cursor = null;
    do {
        var page = api.GetPage(cursor);
        foreach (var p in page.Items) yield return p;
        cursor = page.NextCursor;
    } while (cursor is not null);
}
var first20 = AllProducts(api).Take(20).ToList();
```

### Tree Walk

Recursive **`yield`** preorder — deep trees → iterative stack in production.

## Common Mistakes

Double enumeration expensive source. Mutate during iterate. Infinite without **`Take`**. Hide **`IQueryable`** behind **`IEnumerable`** — lose SQL translation.

---

## Key Takeaways

- **`yield`** — lazy pull; compiler state machine.
- **`foreach`** disposes — cleanup in **`finally`**.
- Compose с LINQ; materialize когда нужны multiple passes.
- Cold = re-enumerate re-executes.
