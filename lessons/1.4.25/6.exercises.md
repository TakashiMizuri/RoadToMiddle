# Exercises: Parallel.ForEachAsync

> Complete after lectures and test.

## Easy

### E1. Bounded downloads
**Goal:** Fetch 20 HTTP URLs with max 4 concurrent.
**Steps:** Hardcode URL list; ForEachAsync; log start/end per URL with timestamp.
**Success criteria:** Never more than 4 overlapping requests (log evidence).

### E2. Cancel mid-batch
**Goal:** Cancel after 2 seconds.
**Steps:** CTS CancelAfter(2000); slow endpoints (local delay server); observe partial completion.
**Success criteria:** OperationCanceledException caught and explained.

## Medium

### M1. ASP.NET batch endpoint
**Goal:** POST `/sync` with id[] calls downstream with parallelism 8.
**Steps:** Minimal API + IHttpClientFactory named client; ConcurrentBag results.
**Success criteria:** Passes cancellation from client disconnect (optional manual cancel test).

### M2. Compare WhenAll vs ForEachAsync
**Goal:** Same 50 URLs, measure peak open connections or elapsed time.
**Steps:** Two console modes; document difference in 5 sentences.
**Success criteria:** ForEachAsync shows lower peak concurrency or safer behavior.

## Hard (stretch)

### H1. Failure report DTO
**Goal:** Return `{ succeeded: [], failed: [{ id, error }] }` from parallel fetch.
**Steps:** Per-item try/catch; thread-safe aggregation; no single failure aborts all.
**Success criteria:** Mixed success/failure test case passes.
