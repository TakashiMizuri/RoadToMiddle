# Exercises: .gitignore Patterns and .gitkeep

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Write a minimal .gitignore
**Goal:** Apply standard patterns for a .NET + React monorepo.

**Steps:**
1. Create `.gitignore` at repo root.
2. Include patterns for: `bin/`, `obj/`, `node_modules/`, `.env`, IDE folders (`.vs/` or `.idea/`), OS junk (`.DS_Store` or `Thumbs.db`).
3. Add `!.env.example` so a template can be committed.
4. Run `git check-ignore -v` on at least two paths that should match.

**Success criteria:** All listed artifact types covered; `git check-ignore -v` shows correct rule lines.

---

### E2. Debug an ignore mismatch
**Goal:** Use `git check-ignore` as a forensic tool.

**Setup:** Create files `tmp/notes.log`, `build/output.dll`, `src/build/cache.tmp`.

**Steps:**
1. Add to `.gitignore`:
   ```gitignore
   /build/
   *.log
   ```
2. Predict which files are ignored before running commands.
3. Run `git check-ignore -v` on each file.
4. Fix `.gitignore` if `src/build/cache.tmp` should also be ignored (hint: `build/` vs `/build/`).

**Success criteria:** You can explain every match/non-match with the exact pattern.

## Medium

### M1. Stop tracking a committed secret
**Goal:** Understand untrack workflow (no real secrets required).

**Steps:**
1. Create `fake-secrets.env` with dummy content `API_KEY=dummy`.
2. `git add fake-secrets.env` and commit.
3. Add `fake-secrets.env` to `.gitignore`.
4. Run `git status` â€” file still shows as tracked? Explain why.
5. Run `git rm --cached fake-secrets.env`, commit again.
6. Verify: file exists on disk, `git status` clean, `git check-ignore -v fake-secrets.env` works.

**Success criteria:** Two commits; verbal explanation of cached vs working tree.

### M2. Empty directory with .gitkeep
**Goal:** Track folder structure without tracking user uploads.

**Steps:**
1. Create `storage/incoming/` and `storage/processed/`.
2. Add `.gitkeep` only in `incoming/`.
3. Add `.gitignore` rules so future files in `storage/processed/` are ignored but folder exists via `.gitkeep` or `README.md` in `processed/`.
4. Commit structure.

**Success criteria:** `git ls-files storage/` shows only placeholders; `storage/processed/large.bin` (create locally) stays untracked/ignored.

## Hard (stretch)

### H1. Negation puzzle
**Goal:** Master `!` rules and directory ignore limits.

**Given:**
```gitignore
logs/
!logs/keep/
logs/keep/*
!logs/keep/audit.log
```

**Tasks:**
1. For each path, write **tracked on add** or **ignored**: `logs/app.log`, `logs/keep/audit.log`, `logs/keep/other.log`, `logs/keep/sub/deep.log`.
2. If any path fails your expectation, rewrite the four lines to achieve: only `audit.log` under `logs/keep/` is not ignored; everything else under `logs/` ignored.
3. Test with `git add -n` (dry run) or `git check-ignore -v`.

**Success criteria:** Working four-line `.gitignore` + written explanation of parent-directory rule.

---

### H2. Three-layer ignore map
**Goal:** Map global, exclude, and repo ignores.

**Steps:**
1. Set global excludesfile to a new file with `*.bak`.
2. Add `local-scratch.txt` to `.git/info/exclude`.
3. Add `*.tmp` to repo `.gitignore`.
4. Create `test.bak`, `local-scratch.txt`, `draft.tmp` in repo.
5. Document which file each rule comes from using `git check-ignore -v`.

**Success criteria:** Table with columns: file, ignored?, source file path.

Use `[Bb]in/` and `[Oo]bj/` for case-insensitive .NET folders. `node_modules/` without a leading slash matches at any depth.
</details>

<details>
<summary>Hint M1</summary>

After step 3, `git status` may still show modifications to the tracked file until you `git rm --cached`. The local file never disappears unless you delete it manually.
</details>

<details>
<summary>Hint H1</summary>

Git cannot re-include a file if an ancestor directory is ignored. Ignore `logs/*` instead of `logs/`, then un-ignore specific paths.
</details>
