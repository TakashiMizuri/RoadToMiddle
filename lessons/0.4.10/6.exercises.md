## Easy

### E1. Origin same or different?
**Goal:** Recognize origins.

| A | B | Same? |
|---|---|-------|
| `https://api.example.com` | `https://app.example.com` | |
| `https://example.com:443` | `https://example.com` | |
| `http://127.0.0.1:5000` | `http://localhost:5000` | |

**Success criteria:** Three answers with scheme/host/port reasoning.

## Medium

### M1. Preflight prediction
**Goal:** Classify requests.

Will browser send OPTIONS preflight? (cross-origin to `api.example.com`)

1. `GET /health` no custom headers
2. `POST /login` `Content-Type: application/json`
3. `DELETE /items/1` with `Authorization` header

**Success criteria:** Yes/no each with rule reference.

---

### M2. curl CORS probe
**Goal:** Server-side header check.

Write a `curl` command that sends `Origin: http://localhost:5173` to `http://localhost:5000/weatherforecast` and shows response headers.

**Success criteria:** Uses `-H Origin` and `-I` or `-v`.

## Hard (stretch)

### H1. Architecture choice
**Goal:** Trade-off analysis.

Compare for a new project:
- A) `app.example.com` + `api.example.com` (CDN + API)
- B) `example.com` SPA + `example.com/api` via reverse proxy

Which requires browser CORS? Pros/cons in 4–6 sentences.

**Success criteria:** B same-origin for `/api`; A needs CORS on API.
