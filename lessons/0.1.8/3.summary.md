# Undo: restore, revert, reset — краткая выжимка

> `0.1.8` · ~10 мин на повторение

## Главная идея

Git хранит состояние проекта в **трёх деревьях**: **HEAD** (последний commit), **index/staging** (следующий snapshot), **working tree** (файлы на диске). Команды undo двигают **разные** деревья. На **shared pushed** ветках безопасен **`git revert`** (новый commit); **`git reset`** переписывает history — только для локальной работы.

## Ключевые пункты

- **HEAD** → tip текущей ветки (последний commit).
- **Index** → что попадёт в `git commit` (`git add` кладёт сюда).
- **Working tree** → реальные файлы в каталоге проекта.
- **`git restore file`** — сбросить unstaged изменения (WT ← index).
- **`git restore --staged file`** — unstage (index ← HEAD), правки на диске остаются.
- **`git reset --soft HEAD~1`** — HEAD назад, index и WT **без изменений** (commit остаётся staged).
- **`git reset HEAD~1`** (`--mixed`) — HEAD и index назад, WT без изменений (изменения unstaged).
- **`git reset --hard HEAD~1`** — всё к предыдущему commit; **незакоммиченное теряется**.
- **`git revert <hash>`** — новый commit с обратным patch; оригинал в history.
- **`git reflog`** — журнал перемещений HEAD локально; спасение после hard reset.

## Самые важные примеры

**Unstage без потери правок:**
```bash
git restore --staged secrets.env
```

**Отменить последний локальный commit, сохранить файлы unstaged:**
```bash
git reset HEAD~1
```

**Отменить pushed commit на main:**
```bash
git revert badcommit_hash
git push origin main
```

**Восстановление после ошибочного hard reset:**
```bash
git reflog
git reset --hard HEAD@{1}
```

## «Под капотом» в одном абзаце

Branch ref (`refs/heads/main`) — указатель на commit. `git reset` двигает этот указатель и (в зависимости от режима) перезаписывает `.git/index` и working tree из tree object целевого commit. `git revert` **не** двигает указатель назад — создаёт новый commit через patch/merge machinery. Reflog пишет в `.git/logs/` каждое изменение HEAD — поэтому «исчезнувшие» commits достижимы по hash, пока не собран GC.

## Частые ошибки

- `reset --hard` на shared + force push.
- Путать `restore` (файлы) и `reset` (HEAD).
- Думать, что `revert` удаляет commit.
- `reset --hard` без понимания потери unstaged work.
- Забыть `git reflog` при панике.

## Мнемоника

> **Restore** = починить **файлы**. **Reset** = сдвинуть **HEAD** (и maybe index/WT). **Revert** = **новый** commit-отмена. **Shared? Revert.**

Таблица reset:

| Mode | HEAD | Index | WT |
|------|------|-------|-----|
| soft | ✓ | — | — |
| mixed | ✓ | ✓ | — |
| hard | ✓ | ✓ | ✓ |
