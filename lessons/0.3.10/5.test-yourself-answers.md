## Answers

### 1. [Recall]
**Answer:** **Resource** — identifiable abstraction (user, order, catalog), addressed by **URI**, manipulated via **representations**.

---

### 2. [Recall]
**Answer:** Each request contains **all information** needed to process it; server **does not rely on prior request context** stored only for that client connection (no conversational server memory without identifiers in request).

---

### 3. [Recall]
**Answer:** **0** — single endpoint POST; **1** — multiple resource URIs; **2** — HTTP verbs + status codes; **3** — hypermedia HATEOAS.

---

### 4. [Understanding]
**Answer:** `GET /users/42` uses **HTTP method semantics**, **cacheable** safe GET, clear resource identification. POST tunnel hides action in body — **level 0 RPC**, loses uniform interface benefits.

---

### 5. [Understanding]
**Answer:** **Нет** — **resource state** in DB is expected. Statelessness forbids **client session/conversation state** unknown to next request, not persistent domain data.

---

### 6. [Understanding]
**Answer:** Standard **GET** safe/idempotent → intermediaries can **cache** with ETag/Cache-Control; **status codes** communicate outcome without parsing body — **self-descriptive messages**.

---

### 7. [Application]
**Answer:** **B) `POST /products`** — noun collection, create via POST method.

---

### 8. [Application]
**Answer:** Violates **statelessness** — fix: expose `checkoutId` in URI (`/checkouts/{id}`) or include in each request body/header so any server can continue.

---

### 9. [Application]
**Answer:** **GET** `/catalog` with **`Cache-Control`** (e.g. public/max-age or revalidation) and **ETag** for **304**.

---

### 10. [Analysis]
**Answer:** **B** неверно — REST not tied to JSON; representations can be JSON, XML, HTML, etc.

---

### 11. [Analysis]
**Answer:** Breaks **self-descriptive messages** (status should be **404**) and **uniform interface**/intermediary semantics; breaks link to HTTP status model (`0.3.6`).

---

### 12. [Understanding]
**Answer:** **Resource** — conceptual entity; **representation** — byte/format snapshot (JSON DTO) transferred over wire at a time.

---

### 13. [Analysis]
**Answer:** **Complex state transition** awkward as pure PUT on representation — pragmatic **RPC sub-resource** accepted when documented; trade clarity vs purity.
