# Exercises: Composition over Inheritance

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Bird refactor
**Goal:** Replace an inheritance-based bird hierarchy with composition.
**Steps:**
1. Create `IFlyBehavior` with `Flying` and `Flightless` implementations.
2. Create sealed `Bird` with constructor-injected `IFlyBehavior`.
3. Delete any `Bird` subclasses that only differed by flight capability.
**Success criteria:** No `throw` in flight path; callers choose behavior at construction/DI.

### E2. Extract interface from base
**Goal:** Take one `protected virtual` method from a small base class and move it to an interface.
**Steps:**
1. Pick or sketch a `ReportExporterBase` with `protected virtual FormatHeader()`.
2. Introduce `IReportHeaderFormatter` and inject it into a sealed `ReportExporter`.
3. Keep public `Export()` algorithm unchanged.
**Success criteria:** No inheritance required to customize header formatting.

## Medium

### M1. Validator decorator chain
**Goal:** Build a composable validation pipeline for `Order`.
**Steps:**
1. Define `IOrderValidator` with `ValidationResult Validate(Order order)`.
2. Implement `BasicOrderValidator` (non-empty lines, positive total).
3. Implement `FraudCheckingValidator` and `VipDiscountValidator` as decorators wrapping `_inner`.
4. Wire chain manually or via a small factory: fraud → vip → basic.
**Success criteria:** Add/remove a validator without editing others; unit-test each in isolation.

### M2. Logger composition
**Goal:** Refactor `JsonLogger : Logger` inheritance into composition.
**Steps:**
1. Define `ILogFormatter` and `ILogSink`.
2. Implement `PlainTextFormatter`, `JsonFormatter`, `ConsoleSink`.
3. Sealed `Logger` composes formatter + sink.
**Success criteria:** Swap JSON vs plain at runtime/DI without subclassing `Logger`.

## Hard (stretch)

### H1. BaseController strangler
**Goal:** Plan (pseudo-code acceptable) migration from `BaseApiController` with CRUD helpers to composition.
**Steps:**
1. List responsibilities in a hypothetical fat base (auth, mapping, validation, repository access).
2. Map each to an interface, filter, or injected service.
3. Show one action method in a thin controller using only composition.
**Success criteria:** No `protected` helpers; cross-cutting via `IActionFilter` or middleware.

### H2. LSP violation fix
**Goal:** Fix the Rectangle/Square inheritance problem.
**Steps:**
1. Document why `Resize(Rectangle)` breaks with `Square`.
2. Remove inheritance; introduce `IShape` with `Area` or separate immutable types `Rectangle` and `Square`.
3. Ensure `Resize` cannot accept a square disguised as rectangle.
**Success criteria:** No subtype weakens base invariants; tests pass for both shapes independently.

## Submission checklist

- [ ] No inheritance used only for reuse (document any remaining `:` with LSP justification)
- [ ] Dependencies injected via constructor, fields `private readonly`
- [ ] At least one decorator or strategy demonstrated
- [ ] Unit tests use mocks/fakes for composed interfaces
