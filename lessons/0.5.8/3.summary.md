# Валидация форм: `required`, `pattern`, `min`, `max` — выжимка

> `0.5.8` · ~8 мин

## Главная идея

HTML5 constraints дают мгновенный client-side UX; **server validation обязательна** — attributes можно обойти.

## Ключевые пункты

- **`required`** — значение обязательно (checkbox = checked).
- **`minlength`/`maxlength`** — длина строки (text, textarea).
- **`min`/`max`/`step`** — number, date, time, range.
- **`pattern`** — regex full string; **`title`** — hint в error.
- **`novalidate`** / **`formnovalidate`** — отключить проверку (draft).
- **`ValidityState`**: `valueMissing`, `patternMismatch`, `rangeUnderflow`, …
- **`:user-invalid`** — стиль после взаимодействия пользователя.
- Дублируй правила в ASP.NET DataAnnotations + DB.

## Примеры

```html
<input type="email" name="email" required />
<input type="number" name="qty" min="1" max="99" required />
<input type="password" minlength="12" required />
```

## Частые ошибки

- `min` на `type="text"` вместо `minlength`.
- Только HTML validation на API.
- Слишком strict `pattern` для international data.

## Правило большого пальца

**Три слоя:** HTML UX → server rules → DB constraints.
