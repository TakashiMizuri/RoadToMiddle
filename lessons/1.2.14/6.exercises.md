# Exercises: struct vs class

> Complete after lectures and passing `4.test-yourself.md`.

## Easy

### E1. Copy behavior lab
**Goal:** Console app with `struct Point` and `class PointClass`; demonstrate assignment copy difference.
**Success criteria:** Mutating copy affects only struct copy, not class original.

### E2. foreach trap
**Goal:** Reproduce foreach mutation bug on `List<Point>`; fix with indexed for or immutable pattern.
**Success criteria:** Explain why foreach failed in comments.

## Medium

### M1. ref vs value
**Goal:** Implement `Move(ref Point p)` vs `Move(Point p)` and show caller state after each.
**Success criteria:** Only `ref` version mutates caller's struct.

### M2. Boxing observation
**Goal:** Box struct to `object` and use `ReferenceEquals` / `GetType` to show heap object; compare `List<int>` vs legacy `ArrayList` conceptually.
**Success criteria:** Articulate GC cost of boxing in loop.

### M3. Size decision
**Goal:** Given domain types `Money(decimal amount, string currency)` and `Customer(int id, string name, List<Order> orders)` — struct or class each? Write justification.
**Success criteria:** Two criteria per choice aligned with lecture.

## Hard (stretch)

### H1. in parameter
**Goal:** `LargeStruct` (~64+ bytes dummy fields); compare disassembly or stopwatch: by value vs `in` parameter called 10M times.
**Success criteria:** Measurable or reasoned performance difference.

### H2. Array locality
**Goal:** Allocate `Point[1_000_000]` vs `class Node[1_000_000]`; discuss memory layout (diagram acceptable).
**Success criteria:** Explain cache / GC trade-offs without running if no time.
