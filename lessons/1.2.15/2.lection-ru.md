# readonly struct, ref struct limitations

> Roadmap: `1.2.15` · Node: `1.2` — C# ООП · Depth: понимание

## Learning Objectives

После этой лекции ты сможешь:

- Объяснить гарантии **`readonly struct`** и как компилятор enforce immutability.
- Описать **stack-only** ограничения **`ref struct`** и зачем они существуют.
- Безопасно использовать **`Span<T>`** и **`ReadOnlySpan<T>`** как buffer views.
- Применять **`ref`**, **`in`**, **`out`**, **`scoped`** для избежания копий (`1.2.14`).
- Перечислить, что **ref struct нельзя** (boxing, async, поля обычного class).
- Распознать, когда уместен **`readonly record struct`** (`1.2.13`).

---

## Why This Matters

`1.2.14` показал: struct копируются и путают при mutation через copies. **`readonly struct`** отвечает: «значение не меняется после construction» — компилятор блокирует instance mutations.

**`ref struct`** идёт дальше: типы вроде **`Span<T>`** живут только на **stack** — views в memory без heap allocation для самого view. Современный .NET (`System.Text.Json`, Kestrel, slicing `Memory<T>`) опирается на `Span<T>`. Нарушение правил ref struct — compile error, защита от use-after-free, если span ушёл в heap.

Custom `ref struct` редки, но **`ReadOnlySpan<char>`**, `stackalloc` buffers — ежедневность; нужно знать, почему нельзя хранить `Span<T>` в поле class или `await` с span (в старых правилах).

## Core Concepts

### readonly struct

**`readonly`** на struct — instance state не меняется после construction:

```csharp
public readonly struct Temperature
{
    public double Celsius { get; }
    public Temperature(double celsius) => Celsius = celsius;

    public Temperature ToFahrenheit()
        => new Temperature(Celsius * 9 / 5 + 32);
}
```

Правила:

- Все **instance fields** — `readonly`.
- Instance methods не присваивают полям (кроме ctor).
- Меньше **defensive copies** при вызове методов на readonly instance.

**`readonly record struct`** (C# 10):

```csharp
public readonly record struct Point(int X, int Y);
```

Mutation API возвращает **новый** instance (`with`).

### Defensive copies

Для non-readonly struct вызов метода на **readonly field** может копировать struct. **`readonly struct`** уменьшает скрытые копии.

### ref struct: stack-only types

**`ref struct`** нельзя:

- Box в `object` / interface
- Хранить как instance fields обычных **class** / **struct** (узкие исключения `ref` fields C# 11+)
- Захватывать в **async** / **iterator** state machines (правила эволюционируют в C# 13+)
- Использовать как type argument обычных generics без `allows ref struct` (C# 12)

```csharp
public ref struct StackOnlyBuffer
{
    public Span<byte> Data;
}
```

**Цель:** by-ref views с lifetime stack frame.

### Span<T> и ReadOnlySpan<T>

Флагманские **ref struct** BCL:

```csharp
Span<int> slice = array.AsSpan(2, 5);
ReadOnlySpan<char> text = "hello".AsSpan();
```

`Span<T>` — **окно** над contiguous memory. Slicing — без heap allocation view.

```csharp
Span<byte> buffer = stackalloc byte[256];
```

**Нельзя:**

```csharp
public class Holder
{
    public Span<byte> Buffer;  // CS8345
}
```

Для long-lived buffers — **`Memory<T>`** или **`byte[]`**; `Span` — local scope.

### ref, in, out parameters

| Modifier | Смысл |
|----------|-------|
| `ref` | Alias; definitely assigned |
| `out` | Output alias |
| `in` | Readonly alias; без copy большого struct |
| `scoped` | ref не escape scope метода |

```csharp
static int Parse(ReadOnlySpan<char> digits) { ... }
```

### readonly vs const vs init

- **`const`** — compile-time static
- **`readonly` field** — в ctor
- **`readonly struct`** — весь instance immutable
- **`init`** — при construction (`1.2.2`, `1.2.13`)

## Under the Hood

### Почему ref struct нельзя box

Boxing — копия в heap с object header; lifetime переживёт stack buffer → **dangling view**. CLR запрещает boxing ref struct.

### Span internals (concept)

`Span<T>`: **ref + length**. Array-backed — в data массива; `stackalloc` — в stack frame; нельзя escape frame.

### readonly struct JIT

JIT знает: fields не меняются — enregistration, elision copies.

```csharp
Span<char> buffer = stackalloc char[input.Length];
```

### async error

```csharp
async Task BadAsync()
{
    Span<byte> s = stackalloc byte[10];
    await Task.Delay(1); // ошибка: ref struct across await
```

## Common Mistakes & Anti-patterns

**`Span<T>` в instance field** class.

**Mutable struct с readonly label** — public setters.

**Большой `stackalloc` в recursion** — stack overflow.

**Огромный `readonly struct`** — copy cost (`1.2.14`).

**Игнор `scoped`** — ref escape failures.

## Production & Real-World Notes

ASP.NET Core внутри использует Span для parsing. Application layer часто `string`/`ReadOnlyMemory<char>` до profiling.

BCL: `int.TryParse(ReadOnlySpan<char>, ...)`.

CA1815 — сделай struct readonly где возможно.

## Comparison / Trade-offs

| Type | Immutability | Restrictions |
|------|--------------|--------------|
| `readonly struct` | Да | Copy cost |
| `ref struct` | Часто readonly usage | No heap escape |
| `class` | Опционально | Heap, OOP |

## Quick Reference

| Нужно | Используй |
|-------|-----------|
| Immutable small value | `readonly struct` / `readonly record struct` |
| Slice без heap | `Span<T>` / `ReadOnlySpan<T>` |
| Long-lived buffer | `byte[]`, `Memory<T>` |
| Без copy большого struct | `in` |
| Mutation caller struct | `ref` |

---

## Key Takeaways

- **`readonly struct`** — immutability + меньше defensive copies.
- **`ref struct`** — stack-only; **no boxing**; основа **`Span<T>`**.
- **Не храни `Span<T>` в полях class** — `Memory<T>` для heap.
- **`ref` / `in` / `out` / `scoped`** — aliasing и copies.
- **`readonly record struct`** — value equality + immutability.
