# Answers: Merge vs rebase

## Scoring: 80%+ pass → `0.2.5`

### 1. [Recall]
**Answer:** Не rebase commit'ы на shared remote branches, которые другие уже fetch'нули/ built upon.

### 2. [Understanding]
**Answer:** **Blame** указывает на **разные commit hash'и** (merge commit / replayed commits vs squashed single commit), хотя line content совпадает.

### 3. [Understanding]
**Answer:** Lease **откажет**, если remote ref изменился с last fetch — **не перезапишет** чужие новые commit'ы.

### 4. [Application]
**Answer:** **Нет**, если `develop` shared и others track it — rewrite ломает clones. Rebase только **local/private** или **feature** before merge to develop.

### 5. [Application]
**Answer:** `git fetch origin && git switch feature/x && git reset --hard origin/feature/x` (или rebase onto new tip); **не push** старых hash'ей.

### 6. [Analysis]
**Answer:** (1) **Unsafe on shared tips** — duplicates/force-push pain. (2) **Merge preserves audit topology** — compliance/incident forensics. (3) **Rebase conflict chains** on long branches.

### 7. [Analysis]
**Answer:** Merge commit имеет **2 parents**; revert нужно указать **mainline parent** (`-m 1`), иначе revert не знает, какую линию «отменить».

### 8. [Understanding]
**Answer:** Меньше divergence → **меньше conflicts** и меньше разницы между merge/rebase; debate becomes less painful.

### 9. [Application]
**Answer:** **Merge commit** сохраняет **5 contributor commits** в history. **Squash** — один commit на main (authorship often in message/PR metadata, not 5 hashes on main).

### 10. [Recall]
**Answer:** (1) Don't rewrite shared published history. (2) Force-with-lease. (3) Integrate often/small branches.

### 11. [Application]
**Answer:** Rebase **local/private feature** before merging to `develop`; **not** on shared `develop`/`main`.

### 12. [Analysis]
**Answer:** **Local-only rebase** before first push; rebase **remote feature** if only you use it (lease push); **never** on protected shared tips.

### 13. [Understanding]
**Answer:** **First-parent** `git log main` reads **linear** — one commit per feature after squash/rebase+FF, без diamond merge bubbles.

### 14. [Application]
**Answer:** **`git reflog`** (local or teammate) найти **pre-bad-push hash**; coordinate restore push — не panic commit.

### 15. [Understanding]
**Answer:** **Rebase and merge:** PR commits replayed on base, then **FF** — linear, no merge commit. **Create merge commit:** **two-parent** merge commit на base.
