# Async all the way — без sync-over-async

> Roadmap: `1.4.20` · Node: `1.4` — C# async · Depth: **глубоко**

## Why This Matters

ASP.NET Core **без `SynchronizationContext`**, но request processing на **thread pool**. **`.Result`** на incomplete task **блокирует** thread, continuation нужен **другой** thread → **pool exhaustion**: queue, latency, failed health checks.

**Async all the way** — от первого **`await`** в request delegate до **`DbContext`**, **`HttpClient`**, **`Stream`**: **`async Task`** + **`await`**, без blocking waits. Sync-over-async — **scalability bug**.

**`Task.Run`** как «fix» — маскирует debt и **удваивает** thread usage. Middle dev рефакторит call chain и выносит blocking legacy на **edges**.

## Core Concepts

### Sync-over-async

```csharp
public IActionResult Get(int id)
{
    var order = _service.GetOrderAsync(id).Result; // BAD
    return Ok(order);
}
```

Blocked thread + continuation на pool → **starvation**; с UI **SyncContext** — **deadlock**.

### Async-over-sync

```csharp
return await Task.Run(() => File.ReadAllBytes(path)); // fake async for I/O
```

Используй **`ReadAllBytesAsync`**.

### Pipeline

```
Kestrel → Middleware InvokeAsync → async Controller → async Service → EF/Http async
```

Любой sync block снижает throughput.

### Refactoring

```csharp
public async Task<IActionResult> Get(int id) => Ok(await _svc.GetAsync(id));
```

**Interface cascade** — один sync signature заставляет **`.Result`** где-то выше.

### Task.Run ≠ fix

**`Task.Run(() => api.GetAsync().Result)`** — blocking внутри pool thread. CPU-bound offload — отдельная осознанная история (**`1.4.18`**).

---

## Under the Hood

**`Wait()`** на incomplete task: thread blocked → continuation in pool → все threads в **`.Wait()`** → **self-deadlock/starvation**. ASP.NET: null SyncContext — меньше UI-deadlock, **starvation** остаётся.

## Common Mistakes

Partial async (async controller + sync repo). **`.GetAwaiter().GetResult()`**. **`lock` + await**. **`Task.Run`** для I/O.

## Key Takeaways

- **`.Result`/`.Wait()`** — pool starvation в ASP.NET
- Async cascade через interfaces
- Legacy sync — off request path
- Native `*Async` для I/O
