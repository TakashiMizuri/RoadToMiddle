# Exercises: Default interface methods (C# 8+)

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Default greeter
**Goal:** Minimal DIM consume.

**Tasks:**
Define **`IGreeter`** with default **`Greet`**. **`SimpleGreeter : IGreeter`** empty. Call **`Greet`** via **`IGreeter`** reference.

**Success criteria:** Works without public **`Greet`** on class; note why **`simple.Greet()`** fails.

---

### E2. Override default
**Goal:** Replace default behavior.

**Tasks:**
**`IFormatter`** default **`Format(int)`** → decimal string. **`HexFormatter`** overrides to hex. Compare output **`255`**.

**Success criteria:** Override used through **`IFormatter`** reference.

---

## Medium

### M1. Evolve interface
**Goal:** Simulate non-breaking add.

**Tasks:**
Start **`IWorker { void Work(); }`** with **`Worker`**. Add **`void Rest()`** with empty default to interface — verify **`Worker`** still compiles without change. Call **`Rest`** via **`IWorker`**.

**Success criteria:** Written note: old implementers unchanged; new capability optional.

---

### M2. Diamond resolve
**Goal:** Conflict resolution.

**Tasks:**
**`IA`** / **`IB`** with different default **`Describe()`**. **`Item : IA, IB`** — implement single public **`Describe()`** or two explicit. Document call results **`((IA)item).Describe()`** vs **`((IB)item).Describe()`** if using explicit.

**Success criteria:** Compiles; predictable output per interface cast.

---

## Hard (stretch)

### H1. DIM vs extension vs abstract
**Goal:** Choose tool.

**Tasks:**
For scenarios (shared **`DbContext`**, **`ToUpper` helper on **`string`**, add **`Validate`** to 50 implementers), pick **DIM**, **extension**, or **abstract base** with paragraph each.

**Success criteria:** Choices align with state/evolution/contract rules from lecture.
