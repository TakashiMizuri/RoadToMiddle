# Generations Gen0/1/2 — выжимка

> `1.5.7` · ~7 мин

## Главная идея

**Generational GC**: young objects (Gen0) collect often cheaply; survivors **promote** to Gen1/Gen2. Long-lived static data sits Gen2 — full GC дороже.

## Ключевые пункты

- Hypothesis: most objects die young.
- ASP.NET Core → **Server GC** default.
- Don't call `GC.Collect()` routinely.
- Card table + write barriers for old→young refs.
