# Таблицы: thead, tbody, th scope — доступность

> Roadmap: `0.5.9` · Node: `0.5` — HTML · Depth: практика

## Learning Objectives

После этого урока ты сможешь:

- Строить табличные данные через `<table>`, `<thead>`, `<tbody>`, `<tfoot>`, `<tr>`, `<th>`, `<td>`.
- Отличать, когда нужна таблица, а когда — CSS Grid или Flexbox.
- Применять `scope="col"` и `scope="row"` для связи заголовков с ячейками данных.
- Писать `<caption>`, описывающий назначение таблицы, без дублирования всей страницы.
- Размечать сложные таблицы через `headers` и `id` при пересечении заголовков.
- Проверять разметку в DevTools и accessibility checker.

---

## Why This Matters

Таблицы существуют, чтобы показать **связь между двумя и более измерениями данных**: SKU по складам, строки счёта с количеством и ценой, метрики спринта по членам команды. Когда данные действительно табличные, `<table>` даёт браузеру и assistive technology предсказуемую модель сетки. Когда таблицы используют для вёрстки страницы — пережиток 1990-х — получается разметка, которую сложно поддерживать, адаптировать под мобильные экраны и которую screen reader объявляет как «таблица, 47 строк, 12 столбцов» ещё до формы входа.

В `0.5.8` ты разбирал валидацию форм на стороне браузера. Таблицы часто стоят рядом с формами: результаты поиска, админ-панели, матрицы сравнения. Middle-разработчик на ASP.NET Core + React рендерит таблицы и на сервере (Razor), и в React — но **контракт доступности начинается в HTML-семантике**, а не в UI-библиотеке. TanStack Table, AG Grid и JSX `<table>` в итоге создают DOM; если нет `<thead>` или заголовки размечены как `<td>`, CSS не спасёт keyboard и screen reader users.

Доступные таблицы — часто и юридическое требование (WCAG 2.x, ADA, EAA). Исправление обычно простое: структура, scope, caption. Цена пропуска — тикеты поддержки, аудит, потеря доверия.

## Core Concepts

### Когда таблица — правильный инструмент

Используй `<table>`, когда смысл ячейки зависит **и** от заголовка строки, **и** от заголовка столбца — или когда модель данных изначально матричная. Сравнение тарифов, календарь занятости, финансовая отчётность — типичные случаи. Если нужен просто ряд карточек или toolbar — Flexbox или Grid. «Выглядит как таблица» в code review недостаточно: спроси, потеряется ли смысл без связи строка/столбец.

### Анатомия: thead, tbody, tfoot

- **`<thead>`** — строки заголовков, обычно один `<tr>` с `<th>`.
- **`<tbody>`** — строки данных, в основном `<td>`.
- **`<tfoot>`** — опционально итоги; в разметке может идти до `<tbody>`, но отображается после тела.

Строка — `<tr>`. Заголовок — `<th>`, данные — `<td>`. Браузер стилизует `<th>` жирным, но выбор `<th>` — про **семантику**, не про CSS.

```html
<table>
  <caption>Q1 2026 sales by region (USD thousands)</caption>
  <thead>
    <tr>
      <th scope="col">Region</th>
      <th scope="col">January</th>
      <th scope="col">February</th>
      <th scope="col">March</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">North</th>
      <td>120</td>
      <td>135</td>
      <td>142</td>
    </tr>
    <tr>
      <th scope="row">South</th>
      <td>98</td>
      <td>101</td>
      <td>115</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th scope="row">Total</th>
      <td colspan="3">711</td>
    </tr>
  </tfoot>
</table>
```

`<caption>` — accessible name таблицы, объявляется при входе фокуса. Один caption сразу после `<table>`, краткая подпись, не маркетинговый абзац.

### scope: col и row

Атрибут `scope` на `<th>` говорит assistive technology, что заголовок относится ко **столбцу** или **строке**:

- **`scope="col"`** — для всех ячеек столбца ниже.
- **`scope="row"`** — для ячеек строки справа (в LTR).

Для простых таблиц — одна строка заголовков и метки строк в первом столбце — `scope` достаточно и его легко аудировать. Screen readers строят связку: «January, 135», а не сиротский «135».

Не ставь `scope` на `<td>`. `scope="colgroup"` / `scope="rowgroup"` — для группировки с `<colgroup>` и вложенными заголовками; продвинутый случай.

### Сложные таблицы: headers и id

При объединённых ячейках и вложенных категориях `scope` может не хватить. Атрибут **`headers`** на `<td>` перечисляет `id` всех релевантных заголовков:

```html
<th id="y2026" colspan="4">2026</th>
...
<th id="q1" headers="y2026">Q1</th>
<td headers="q1 y2026">42</td>
```

Поддержка дороже: уникальные `id` на странице, рефакторинг ломает связи молча. В продуктовом UI упрощай дизайн; `headers`/`id` — для статических отчётов.

### colspan, rowspan и доступность

Spanning объединяет ячейки визуально, но не отменяет заголовки. У total с `colspan="3"` должен быть row header с `scope="row"`. Пустые `<td>` «для угла» в сложных сетках — явные `headers`. Colspan ради выравнивания несвязанного контента — антипаттерн.

## Under the Hood

Браузер строит **внутреннюю table model** из DOM: столбцы, строки, слоты заголовков, связь data cells с header cells. CSS `display: table` повторяет layout, но **не** даёт тот же accessibility tree, что нативная `<table>` с `<th>`. Поэтому «div-таблицы» падают на аудите.

Accessibility tree экспортирует `role="table"`, `row`, `columnheader`, `rowheader`, `cell` — связи из scope, headers/id и секций. ARIA `role="grid"` — для интерактивных spreadsheet с roving tabindex, не замена статической data table.

При React re-render tbody из JSON браузер пересчитывает связи на каждом commit. Условные `<th>` и нестабильные keys ломают ту же модель, на которую опираются пользователи AT.

## Syntax / Commands / API

| Элемент / атрибут | Назначение |
|-------------------|------------|
| `<table>` | Корень табличных данных |
| `<caption>` | Accessible name |
| `<thead>`, `<tbody>`, `<tfoot>` | Секции строк |
| `<th scope="col">` | Заголовок столбца |
| `<th scope="row">` | Заголовок строки |
| `<td>` | Ячейка данных |
| `headers="id1 id2"` | Явная связь с заголовками |

Проверка: DevTools → Accessibility → columnheader/rowheader и Name у caption.

## Examples

**Пример 1 — строки счёта.** Колонки Item, Qty, Price, Total. `<thead>` с `scope="col"`. Если первый столбец — имя товара как идентификатор, лучше `<th scope="row">`, не `<td>`.

**Пример 2 — расписание.** Строки — дни, столбцы — слоты; пересечение — booked/free. Нужны оба scope. Caption «Room B — weekly availability» ориентирует до обхода десятков ячеек.

**Пример 3 — антипаттерн.** Одна `<tr>` из `<td>` для трёхколоночного лендинга. Заменить на Grid; axe/Lighthouse часто ругаются на layout tables.

## Common Mistakes & Anti-patterns

**Layout tables** — таблица вместо колонок; бьёт по AT и responsive. **Grid/Flexbox.**

**Жирный `<td>` вместо `<th>`** — вид как заголовок, семантики нет. **`<th scope>`**

**Нет caption** — «таблица» без контекста. **Один осмысленный caption.**

**Пустой `<th>` для отступа** — отступы через CSS.

**Только цвет** для статуса (red/green) без текста — провал для дальтоников и WCAG.

**Огромная таблица** без caption и thead — pagination и sticky header помогают sighted; caption и scope — всем.

## Production & Real-World Notes

Design system оборачивает `<table>` в компоненты — lint/Storybook a11y, чтобы нельзя было забыть scope. В React не вкладывай `<div>` в `<table>` мимо tbody без понимания auto-insert.

Razor: header в `<thead>` вне `@foreach`, строки в `<tbody>`. Экспорт в Excel иногда сохраняет caption как имя листа.

Широкие таблицы на mobile: `overflow-x: auto` на wrapper с видимым focus, не tabindex на каждой ячейке без grid-паттерна.

WCAG **1.3.1 Info and Relationships** — частая цель аудита для таблиц.

## Comparison / Trade-offs

**Native table vs ARIA grid** — read-only data → `<table>`. Редактируемая сетка → `role="grid"`.

**Table vs CSS Grid** — Grid визуально похож, но связи заголовков вручную через ARIA — больше работы.

**scope vs headers/id** — scope для ~90% UI; headers/id при colspan/rowspan.

**Client sort** — после сортировки сохраняй thead и scopes; порядок AT = визуальный.

## Key Takeaways

- `<table>` только для **табличных данных**, не для layout.
- **thead / tbody / tfoot** и `<th>` vs `<td>` по смыслу.
- **`scope="col"`** и **`scope="row"`** связывают заголовки с ячейками.
- **`<caption>`** именует таблицу.
- Сложные merged headers — **`headers`** и уникальные **`id`**.
- Div-«таблицы» не дают native accessibility mapping.
- Проверяй keyboard, screen reader и автоматические a11y tools.
