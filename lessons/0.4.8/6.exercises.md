# Exercises: Proxy, Reverse Proxy, Load Balancer

> Complete after reading lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Label the diagram
**Goal:** Distinguish forward vs reverse.

For each scenario, write **forward proxy**, **reverse proxy**, **load balancer**, or **combination**:

1. Corporate firewall proxy for employee web browsing.
2. nginx in front of three Kestrel pods terminating HTTPS.
3. AWS ALB distributing traffic to EC2 Auto Scaling group.
4. Browser extension routing traffic through a VPN exit node (conceptually similar to which type?).

**Success criteria:** Four labels with one-sentence justification each.

---

### E2. Header purpose
**Goal:** Understand forwarded headers.

Explain in 2–3 sentences what breaks if `X-Forwarded-Proto` is missing but users access the site via HTTPS through a reverse proxy.

**Success criteria:** Mentions redirect URLs, cookies (`Secure`), or HSTS-related issues.

---

## Medium

### M1. nginx upstream sketch
**Goal:** Connect config to architecture.

Write a minimal nginx config fragment (not full file) with:
- `upstream` block with two servers
- `listen 443 ssl` server block
- `proxy_pass` to upstream
- `X-Forwarded-For` and `X-Forwarded-Proto` set

**Success criteria:** Valid directive names; two backends; both headers present.

---

### M2. L4 vs L7 choice
**Goal:** Apply trade-offs.

Your architecture needs:
- Route `api.example.com` → API cluster
- Route `www.example.com` → static S3/ CDN origin
- Same public IP / ALB

Which layer balancer do you need (L4 or L7)? Why won't pure L4 suffice?

**Success criteria:** Chooses L7 with Host/path routing explanation.

---

## Hard (stretch)

### H1. Forwarded headers security
**Goal:** Trust boundary design.

Kestrel is accidentally exposed on a public IP **and** behind nginx on a private IP. Attacker sends request directly to Kestrel with `X-Forwarded-Proto: https`.

Describe the attack impact and two configuration/network fixes.

**Success criteria:** Mentions SG/firewall so only nginx reaches Kestrel; `KnownProxies`; not trusting headers from world.
