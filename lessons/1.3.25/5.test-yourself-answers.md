## Answers

### 1. [Recall]
**Answer:** `1, 2` (по одному разу, order сохранён).
**Why:** `Distinct` убирает consecutive/non-consecutive duplicates по default equality для `int`.

### 2. [Recall]
**Answer:** `Concat` сохраняет все элементы включая duplicates; `Union` — merge **без** duplicates (set union).
**Why:** `Union` ≈ `Concat` + `Distinct` с тем же comparer.

### 3. [Understanding]
**Answer:** `Distinct` использует `EqualityComparer<Person>.Default` → reference equality (если не overridden). Два разных instance с одним Email — разные объекты.
**Why:** Нужен `DistinctBy(p => p.Email)` или корректные `Equals`/`GetHashCode`.
**Common mistake:** Думать, что compiler сравнивает поля автоматически.

### 4. [Understanding]
**Answer:** `a.Except(b)` — элементы из **a**, которых нет в **b**. `b.Except(a)` — наоборот. Set difference не symmetric.
**Why:** Semantics «вычитания» привязана к первому operand.

### 5. [Understanding]
**Answer:** Один элемент `1`, не три.
**Why:** `Intersect` deduplicates output; multiplicity первой sequence не сохраняется.

### 6. [Application]
**Answer:** `editorPerms.Except(adminPerms)`.
**Why:** Нужны permissions in Editor **not in** Admin.

### 7. [Application]
**Answer:** `listA.Union(listB, StringComparer.OrdinalIgnoreCase)` или `UnionBy(s => s, StringComparer.OrdinalIgnoreCase)`.
**Why:** Custom comparer для case-insensitive identity.

### 8. [Analysis]
**Answer:** Нет (если `Person` — class без value equality).
**Why:** `Select(u => u)` не меняет identity; `Distinct` сравнивает references.

### 9. [Analysis]
**Answer:** В order **first appearance в `first`**; из `second` общие элементы не добавятся повторно.
**Why:** `Union` сначала полностью обрабатывает first с dedup, потом second только novel elements.

### 10. [Understanding]
**Answer:** SQL **`UNION`** (deduplicating), не `UNION ALL`.
**Why:** LINQ `Union` semantics — unique rows; для all rows нужен другой подход.

### 11. [Application]
**Answer:** `a.Except(b).Union(b.Except(a))`.
**Why:** Classic symmetric difference через два directed except.

### 12. [Recall]
**Answer:** `DistinctBy` yields **first element** per distinct key; `GroupBy`+`First` может materialize groups и дать другой semantics/cost для сложных правил выбора.
**Why:** `DistinctBy` — streaming single pass с HashSet keys; `GroupBy` — полная группировка.

## Model answers for practical sections

`a.Intersect(b)` → `{ 2, 3 }` (order of first: 2 before 3 in `a`).
`a.Except(b)` → `{ 1 }` (3 and 2 are in b).
Result: `{ 2, 3, 1 }`.
