# struct vs class — краткая выжимка

> `1.2.14` · ~10 мин на повторение

## Главная идея

**struct** — value type: присваивание и передача by value **копируют поля**; **class** — reference type: копируется **ссылка**, объект общий. Выбор влияет на memory, GC, identity и mutability bugs.

## Ключевые пункты

- Assignment `b = a`: struct — независимая копия; class — alias.
- **Identity:** только у reference types (`ReferenceEquals`).
- **struct** не наследует; polymorphism — interfaces (риск **boxing**).
- **foreach** по `List<Point>` — `p.X++` меняет **копию**, не элемент списка.
- Struct с **reference fields** — копия дублирует указатели, не deep copy.
- **Boxing** struct → `object` — heap + GC (`1.1.7`).
- **~16 bytes** — эвристика размера struct; большие — частые копии, consider `in`/`ref` или class.
- **class:** identity, EF entities, inheritance, большие mutable graphs.
- **struct:** малые immutable values, `record struct`, hot local math.
- Stack allocation **не гарантирована** — escape, async, closures → heap.

## Самые важные примеры

```csharp
var b = a; b.X = 99;        // struct: a.X unchanged
var d = c; d.Name = "X";    // class: c.Name is "X"

foreach (var p in points) p.X++;  // does NOT update list
```

## «Под капотом» в одном абзаце

Reference type object: header + method table + fields на heap. Struct local часто в stack slot или регистре JIT; при boxing появляется heap object-обёртка. `Point[]` хранит struct inline contiguously — лучше cache locality, чем `Customer[]` из references на разрозненные объекты. Interface-вызов на struct через `IComparable` обычно boxes. `ref struct` (`1.2.15`) запрещает boxing и heap escape принудительно.

## Частые ошибки

- Mutable struct + foreach modification.
- Огромный struct в API — silent copy на каждый вызов.
- «DTO всегда struct» без учёта размера.
- Путать value copy struct с deep copy вложенных ссылок.

## Мнемоника

> **Маленький immutable value без id — struct; identity, null, наследование, ORM — class.**
