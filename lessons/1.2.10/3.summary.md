# Default interface methods (C# 8+) — краткая выжимка

> `1.2.10` · ~6 мин на повторение

## Главная идея

**Default interface methods** — method **body на interface** (C# 8+). Existing implementers **не обязаны** добавлять method, если устраивает default; **override** — public или explicit member на class. Вызов default — через **interface reference**, если нет public member на class.

## Ключевые пункты

- Мотивация: **non-breaking evolution** shipped interfaces.
- Override = обычный implicit/explicit impl (`1.2.9`).
- **Нет instance fields** на interface — не замена abstract class.
- **Diamond**: два default с одной signature → class must implement/resolve.
- DIM ≠ **extension methods** (static vs instance on interface).
- App code чаще **consumes** DIM; authors — library/framework.

## Самые важные примеры

```csharp
public interface IPlugin
{
    void Init();
    void Shutdown() { } // default no-op
}

IGreeter g = new SimpleGreeter();
g.Greet("Ada"); // OK — через IGreeter, не g as SimpleGreeter public API
```

## Частые ошибки

- **`obj.NewMethod()`** когда только default — нужен **`((I)obj).NewMethod()`**.
- Shared state в DIM вместо abstract/composition.
- Сложная logic в interface body — hard to test.

## Мнемоника

> **Evolve interfaces with small defaults; override when behavior differs; state stays in classes.**
