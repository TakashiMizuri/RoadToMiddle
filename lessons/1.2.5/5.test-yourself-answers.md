## Answers

### 1. [Recall]
**Answer:** Объединение data и behavior в type с **скрытием internal representation** и доступом через controlled API.
**Why:** Core OOP principle.

### 2. [Recall]
**Answer:** Visible instance fields **expose implementation**, bypass validation hooks, increase coupling — против encapsulation guidelines.
**Why:** CA1051 rationale.

### 3. [Understanding]
**Answer:** Auto-property **можно** позже превратить в full property с validation **без changing call syntax**; public field → property часто **breaking** для field access syntax и metadata.
**Why:** Evolution seam.

### 4. [Understanding]
**Answer:** Вместо снаружи читать Total/IsVip и решать discount — **`order.ApplyBestDiscount()`** переносит rules inside Order; меньше duplicated logic.
**Why:** Tell, Don't Ask.

### 5. [Understanding]
**Answer:** Caller может **Clear**, **replace list**, mutate items **минуя** pricing/invariant logic в `AddLine`.
**Why:** Mutable reference escape.

### 6. [Application]
**Answer:** `private decimal _balance; public decimal Balance => _balance; public void Deposit(...) / Withdraw(...)`.
**Why:** Read preserved; write controlled.

### 7. [Application]
**Answer:** **Constructor** (immutable) или **private/full setter** — single gate; public field assign **не** centralize validation.
**Why:** Invariant at boundary.

### 8. [Analysis]
**Answer:** JIT часто inlines simple properties; разница negligible vs correctness. Profile hot paths; default **encapsulate**. Public fields **не** дают evolution/validation.
**Why:** Premature micro-optimization myth.

### 9. [Analysis]
**Answer:** **DTO** — transport shape, anemic OK at boundary. **Entity** — owns invariants and transitions; public mutable bags scatter rules.
**Why:** Layer responsibilities.

### 10. [Understanding]
**Answer:** **Нет** — reflection может читать private members. Encapsulation — против **accidental** cross-module coupling, не security boundary.
**Why:** Process-local trust model.

### 11. [Understanding]
**Answer:** Public field = part of **binary/source contract**; changing layout/name breaks consumers. Private storage + public API allows internal refactor.
**Why:** Library evolution.

### 12. [Application]
**Answer:** Public field — caller sets `Status = Shipped` **минуя** Pending→Running rules; private set + `Start()` **enforce** state machine.
**Why:** Invariant bypass.

### 13. [Analysis]
**Answer:** **Да** — subclasses couple to base layout, fragile base class; prefer protected methods/controlled hooks.
**Why:** Inheritance breaks encapsulation of base.

### 14. [Understanding]
**Answer:** Change tracking hooks **property setters**; public field writes may **не** trigger same paths; less consistent tracking/intercept.
**Why:** EF convention on properties.

### 15. [Application]
**Answer:** Pattern:
```csharp
private readonly List<Item> _items = new();
public void Add(Item i) { ... _items.Add(i); }
public IReadOnlyList<Item> Items => _items;
```

### 16. [Understanding]
**Answer:** **Anemic wrapper** — no behavior, indirection без value; balance **rich domain methods** vs trivial pass-through everywhere.
**Why:** Over-encapsulation noise.
