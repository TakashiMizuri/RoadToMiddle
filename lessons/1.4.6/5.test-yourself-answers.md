# Answers: async/await Compiler

### 1. IAsyncStateMachine (struct/class) + MoveNext + AsyncTaskMethodBuilder; method stub starts machine.
### 2. Returns Task to caller immediately; MoveNext returns at first incomplete await while work continues via continuation.
### 3. At least 2 — start + resume after completion (may run sync if completed before register).
### 4. No — IsCompleted true, continues same MoveNext without AwaitOnCompleted return.
### 5. False for I/O — registers continuation; thread not dedicated for wait. CPU between awaits runs on current thread until suspend.
### 6. Creates Task returned to caller; SetResult/SetException; coordinates await continuations.
### 7. async Task — caller can await/observe; async void — no Task, fire-and-forget, dangerous in apps.
### 8. Max 2 suspensions (one per incomplete await); 3 MoveNext segments if both suspend.
### 9. Method stack frame destroyed at return; resume needs persisted locals in heap/struct fields.
### 10. Yes — compiler emits try/catch in MoveNext across state transitions preserving semantics.
