# Exercises: Cookies and SameSite

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Attribute purpose
**Goal:** Map attributes to threats.

| Attribute | Primary protects against / enables |
|-----------|-------------------------------------|
| HttpOnly | ? |
| Secure | ? |
| SameSite=Lax | ? |
| SameSite=None | ? |
| Path=/api | ? |

**Success criteria:** XSS read, network HTTP leak, CSRF cross-site subresource, cross-site embed, scope limit.

---

### E2. Send or withhold?
**Goal:** SameSite behavior.

Session cookie `SameSite=Lax`. Will browser send it?

1. Same-site GET `/dashboard`
2. Cross-site `<img src="https://yoursite.com/track">`
3. Cross-site top-level link click to `https://yoursite.com/home`
4. Cross-site `fetch` POST with credentials from evil.com

**Success criteria:** yes, no, yes, no (Lax rules).

## Medium

### M1. Fix the Set-Cookie
**Goal:** Spot bad configs.

For each Set-Cookie, list problems and improved line:

1. `session=abc; SameSite=None`
2. `session=abc; HttpOnly; SameSite=Lax` on HTTP production site
3. `token=jwt...` (no HttpOnly, no Secure) for SPA auth

**Success criteria:** Missing Secure on None; Secure missing on prod HTTP; HttpOnly+Secure needed.

---

### M2. Architecture decision
**Goal:** Cookie vs Bearer.

Scenario: React SPA `app.example.com`, API `api.example.com`, mobile app same API.

Write 6–8 sentences comparing cookie session vs Bearer token for each client, mentioning SameSite and CORS.

**Success criteria:** Cross-site cookie pain; mobile prefers Bearer; BFF mentioned as option.

## Hard (stretch)

### H1. CSRF + SameSite lab (paper)
**Goal:** Connect to future security topics.

Describe attack: evil.com auto-submit POST form to `bank.com/transfer` with victim's cookie session.

Explain how **SameSite=Lax**, **SameSite=Strict**, and **SameSite=None + antiforgery token** each change outcome.

**Success criteria:** Cookie send rules + need CSRF when None.

---

### H2. ASP.NET Core checklist
**Goal:** Production config.

Write checklist (8 items) for cookie authentication middleware: HttpOnly, SecurePolicy, SameSite, logout server invalidation, Data Protection keys rotation, cookie size, HTTPS redirection, antiforgery for form endpoints.

**Success criteria:** Covers security attributes and ops, not only code snippet.
