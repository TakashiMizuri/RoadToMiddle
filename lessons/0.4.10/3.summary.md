# CORS (preview) — краткая выжимка

> `0.4.10` · ~8 мин на повторение

## Главная идея

**Origin** = scheme + host + port. Браузер блокирует JS от reading cross-origin responses без **CORS headers** от сервера. React на CDN + API на другом host → CORS **обязателен** в production. CORS — не auth; curl не affected.

## Ключевые пункты

- **Same-origin policy** — защита пользователя; CORS — opt-in exception.
- **Simple request** — без preflight; JSON + `Authorization` → **OPTIONS preflight**.
- **`Access-Control-Allow-Origin`** — must on actual response (не только OPTIONS).
- **`Allow-Credentials`** + specific origin — нельзя `*` с cookies.
- Vite **dev proxy** — workaround, не production fix.
- **Full config** — later: ASP.NET `AddCors`/`UseCors`, React `fetch` credentials.

## Самые важные примеры

```
OPTIONS /orders → 204 + Allow-Origin, Methods, Headers
GET /orders     → 200 + Allow-Origin
```

## Частые ошибки

- Чинить CORS только во frontend.
- Думать, что Postman = browser security.

## Мнемоника

> **CORS headers live on the API**, not on the static HTML host.
