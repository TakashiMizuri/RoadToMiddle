# Exercises: Links — `a`, `href`, `target`, `rel noopener`

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. URL resolution
**Goal:** Predict resolved URLs.

Current page: `https://shop.io/catalog/winter/boots.html`

Write the full resolved URL for each `href`:

1. `href="gloves.html"`
2. `href="../accessories/hats.html"`
3. `href="/cart"`
4. `href="https://pay.shop.io/checkout"`

**Success criteria:** Four correct absolute URLs with brief one-line rule each.

---

### E2. Fix the link text
**Goal:** Improve accessibility.

Rewrite this block using self-describing links (keep same destinations):

```html
<p>Our product has many features. <a href="/features">Click here</a> to see them.
Need help? <a href="/support">Read more</a>.</p>
```

**Success criteria:** No generic "click here" / "read more"; meaning clear in a links-only list.

---

## Medium

### M1. Safe external links component
**Goal:** Enforce security in reusable markup.

Write a static HTML snippet with three external links (MDN, GitHub, npm). All open in a new tab with correct `rel`. Add visually hidden "(opens in a new tab)" for one link using a `<span class="visually-hidden">`.

**Success criteria:** Every `target="_blank"` has `rel="noopener noreferrer"`; at least one new-tab hint for AT.

---

### M2. In-page TOC
**Goal:** Connect semantic sections and fragments.

Build a mini documentation page:

- `<main>` with two `<section>` elements: `id="install"` and `id="env"`, each with `h2` from `0.5.3`.
- Top `<nav aria-label="On this page">` with ordered list linking to both sections.
- One paragraph link from install section to env section via fragment.

Validate: clicking TOC entries scrolls without leaving the page.

**Success criteria:** Valid HTML; fragment targets exist; headings hierarchy not skipped.

---

## Hard (stretch)

### H1. Audit a bad nav
**Goal:** Analysis + refactor.

Given:

```html
<header>
  <div class="nav">
    <a href="#" onclick="goHome()">Home</a>
    <a href="/pricing" target="_blank">Pricing</a>
    <a href="docs.html">Docs</a>
  </div>
</header>
```

List **at least five** issues (semantics, security, URL resolution assuming page at `/app/dashboard.html`, a11y). Provide corrected HTML using `<nav>`, list from `0.5.4`, and fixes.

**Success criteria:** ≥5 distinct issues with justified fixes; corrected nav uses semantic patterns from `0.5.2`–`0.5.4`.

---

## Submission checklist

- [ ] All external `_blank` links include `noopener noreferrer`
- [ ] Link text descriptive out of context
- [ ] Fragment links match existing `id` attributes
- [ ] No interactive nesting violations
