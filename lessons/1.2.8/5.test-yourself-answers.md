## Answers

### 1. [Recall]
**Answer:** **Нет** — CS0144 / abstract cannot be instantiated.
**Why:** Incomplete type until concrete derived.

### 2. [Recall]
**Answer:** **Неограниченно** (практически — сколько нужно; все на одном class).
**Why:** Multiple interface implementation.

### 3. [Understanding]
**Answer:** Abstract class может иметь **instance fields** и **constructors**, shared всеми derived. Classic interface **не задаёт** instance state — state в implementer.
**Why:** Structural difference driving design.

### 4. [Understanding]
**Answer:** Unrelated **`SmtpSender`**, **`FakeSender`** без общего ctor/state; DI подменяет implementation; не нужен single inheritance slot.
**Why:** Capability contract, not DNA sharing.

### 5. [Understanding]
**Answer:** Base defines **non-virtual algorithm** calling **virtual/abstract hooks**; shared orchestration + varied steps — natural in **abstract class** with protected helpers.
**Why:** Needs concrete base method bodies + inheritance.

### 6. [Application]
**Answer:** **`interface IExportFormat`** (или аналог).
**Why:** No shared state; multiple unrelated plugins.

### 7. [Application]
**Answer:** **`abstract class RepositoryBase<T>`** с **protected DbContext** и ctor.
**Why:** Shared wiring every repo needs.

### 8. [Analysis]
**Answer:** **Один** — **`Base`**. **`IFoo`**, **`IBar`** — interfaces, не class bases.
**Why:** Single inheritance rule.

### 9. [Analysis]
**Answer:** **Compile break** у всех concrete derived, пока не добавят **override** — breaking change.
**Why:** Must implement new abstract member.

### 10. [Understanding]
**Answer:** Клиенты не должны зависеть от fat interfaces — many small role-specific contracts лучше одного giant.
**Why:** ISP from SOLID.

### 11. [Application]
**Answer:** **Interface** — Moq легко mock **`IRepository`**; abstract base может тащить ctor dependencies и non-virtual behavior.
**Why:** Test isolation and DI norms.

### 12. [Understanding]
**Answer:** **`ControllerBase`** даёт **shared ASP.NET implementation** (HTTP, ModelState, etc.); **interfaces** — optional cross-cutting filters/disposal без occupying sole base slot.
**Why:** One class base + many capabilities.

### 13. [Analysis]
**Answer:** **`static class`** (или static members); не abstract/interface для pure helpers.
**Why:** No instantiation, no polymorphism needed.

### 14. [Application]
**Answer:** Library exposes **`IHandler`** for DI/polymorphism. Team adds **`abstract HandlerBase : IHandler`** with logging in **`Handle`**, apps inherit **`HandlerBase`**. Interface — contract; abstract — team DRY.
**Why:** Layered pattern from lection.
