## Why This Matters

В `1.1.6` ты узнал: reference types хранят **reference** на heap data, assignment **не клонирует** object, а создаёт alias. До сих пор примеры были в основном на `List<int>` и `string`. Реальные приложения моделируют **domain concepts** — `Customer`, `Order`, `InvoiceLine` — через **`class`**.

Class — не то, что «запускаешь». Это **blueprint**: именованный контракт с **state** (fields) и **behavior** (methods). **Instance** — живой object, созданный из blueprint в runtime. Путаница между ними — типичные junior-ошибки: business logic в static helpers без instance state, сравнение references через `==` вместо value equality, уверенность что `customerCopy = customer` скопировал customer, хотя скопирован только **адрес**.

Middle-разработчик держит в голове три слоя: **source code** (class definition), **runtime object** (heap instance с identity), **variables** (reference slots). ASP.NET Core controllers, EF Core entities, DTO из JSON — всё опирается на этот треугольник.

## Core Concepts

### Class как blueprint

**Class** объявляет новый reference type. Он группирует data и operations:

```csharp
public class BankAccount
{
    public string Owner { get; set; }
    public decimal Balance { get; set; }

    public void Deposit(decimal amount)
    {
        Balance += amount;
    }
}
```

Этот код **ещё не создал** account в памяти. Он сообщает компилятору и CLR: «Objects типа `BankAccount` имеют такие members». Class живёт в **metadata** (assembly). Instances появляются только при **construction**.

Class — архитектурный план этажа. План — не дом; из одного плана можно построить много домов с разной мебелью внутри.

### Object instance и identity

**Object instance** — конкретный блок памяти на **managed heap** для reference type. Обычно создаётся через **`new`**:

```csharp
var account = new BankAccount { Owner = "Ada", Balance = 100m };
```

После этой строки существуют три идеи:

1. **Type** — `BankAccount` (общий для всех accounts).
2. **Object** — один heap instance со своими `Owner` и `Balance`.
3. **Variable** — `account`, local reference slot, указывающий на object.

Каждый `new BankAccount()` даёт **новый object** с **новой identity**, даже если field values совпадают. Два account с `"Ada"` и balance `100` — **два object**, если код не делит одну reference намеренно.

### Reference semantics на практике

Variables class-типа хранят **references**, не embedded body:

```csharp
var a = new BankAccount { Owner = "Ada", Balance = 50m };
var b = a;
b.Deposit(25m);
Console.WriteLine(a.Balance); // 75 — shared mutation
```

Та же история, что `List<int>` в `1.1.6`, но для своих типов. **`a` и `b` — aliases** одного object. На heap по-прежнему **один** `BankAccount`.

Независимая копия требует explicit copy (factory, clone, record `with`). Язык **не deep-clone** class по умолчанию.

### `null`: отсутствие instance

Reference variable может быть **`null`** — нет target object:

```csharp
BankAccount? savings = null;
// savings.Owner;  // NullReferenceException
```

`null` — не «пустой object», а **отсутствие target**. У value types variable всегда holds bit pattern (default `0` для `int`). Nullable reference types (`BankAccount?`) и `#nullable` помогают compiler предупреждать до dereference.

Production проверяет `null` на границах: API input, DB lookup, deserialization. **`?.` и `??`** — синтаксис безопасной навигации; модель та же.

### Methods работают на instances

Instance methods получают implicit **`this`** — object, на котором вызван call:

```csharp
account.Deposit(10m);   // "this" внутри Deposit — этот instance
```

Разные instances — разный state:

```csharp
var x = new BankAccount { Balance = 10m };
var y = new BankAccount { Balance = 20m };
x.Deposit(5m);   // x.Balance == 15, y.Balance == 20
```

**Method table** shared (определена class); **field values** — per instance. Одно определение, много независимых states — сердце OOP.

### Diagram: blueprint vs instances

```
        CLASS (metadata)                 HEAP (runtime)
   ┌─────────────────────┐         ┌──────────────────┐
   │ BankAccount         │         │ Instance #1      │
   │  Owner : string     │  new    │  Owner → "Ada"   │◄── ref account
   │  Balance : decimal  │ ──────► │  Balance = 100   │
   │  Deposit(...)       │         └──────────────────┘
   └─────────────────────┘         ┌──────────────────┐
                                   │ Instance #2      │
                                   │  Owner → "Bob"   │◄── ref backup
                                   │  Balance = 50    │
                                   └──────────────────┘
```

## Under the Hood

### От source к heap object

При JIT компиляции `new BankAccount()` CLR:

1. Разрешает **type handle** и **method table** для `BankAccount`.
2. Аллоцирует память на **managed heap** под header + instance fields.
3. Инициализирует object header (sync block, method table pointer).
4. Запускает **constructor chain** для начальных field values.
5. Возвращает **managed reference** в local variable slot.

**Class definition** не дублируется per instance. Metadata (field offsets, signatures) в **assembly**, shared. Каждый instance хранит **свои field values** по layout.

### Object header и method table

Каждый reference-type object на heap начинается с **header** → **method table** (virtual dispatch, casting). Instance methods используют `this` как pointer на object. Virtual calls смотрят method table в runtime (`1.2.7` углубит).

Reference fields — pointer-sized slots; value-type fields (`decimal`) **embedded** в body object на heap (decimal bits копируются by value внутри object payload).

### GC roots и aliasing

GC reclaim heap objects, unreachable от **GC roots** (stack refs, statics, handles). Когда `a` и `b` alias один account, обе references держат **один** object alive. Когда **все** references исчезают — object eligible for collection.

Копирование references **не аллоцирует** новый object. **`var b = a`** — cheap pointer copy. **`new BankAccount()`** — allocation. Путать costs важно в hot paths.

### Reference equality vs value equality

По умолчанию **`==` для classes** (без override) сравнивает **references**:

```csharp
var c1 = new BankAccount { Owner = "Ada", Balance = 0m };
var c2 = new BankAccount { Owner = "Ada", Balance = 0m };
Console.WriteLine(c1 == c2);   // false
```

Value-based equality проектируют явно (`1.2.16`, records). Reference semantics объясняют почему.

## Examples

**Example 1 — два instance, независимый state:**

```csharp
var first = new BankAccount { Owner = "A", Balance = 10m };
var second = new BankAccount { Owner = "B", Balance = 10m };
second.Deposit(5m);
// first.Balance == 10, second.Balance == 15
```

**Example 2 — alias через parameter:**

```csharp
void AddBonus(BankAccount acc) { acc.Deposit(100m); }
var mine = new BankAccount { Balance = 0m };
AddBonus(mine);
// mine.Balance == 100
```

**Example 3 — reassignment parameter не меняет caller:**

```csharp
void Replace(BankAccount acc) { acc = new BankAccount { Balance = 999m }; }
var original = new BankAccount { Balance = 1m };
Replace(original);
// original.Balance still 1
```

## Common Mistakes & Anti-patterns

**Class variables как value copies.** `var copy = customer` — alias, не clone.

**Equal field values ⇒ equal objects.** Reference и value equality расходятся без override `Equals`.

**Забытый `null` на API edges.** → `NullReferenceException`.

**Лишние `new` в loops.** Каждый `new` — allocator pressure.

## Production & Real-World Notes

EF Core **tracks entities by reference** в identity map. Две variables на один tracked `Order` — одна row в change tracker.

DI регистрирует **types**, resolve **instances** per lifetime. Путать type registration с object identity — bugs в custom caches.

JSON deserialization создаёт **новые objects**; два deserialize — unrelated references.

## Comparison / Trade-offs

Class vs **struct** (`1.2.14`): class — reference type, identity, heap, inheritance. Struct — value semantics, copying.

## Quick Reference

```
class     → blueprint (metadata)
new T()   → heap instance + reference
var a = b → alias
null      → no instance
==        → reference equality (default)
this      → current instance
```

---

## Key Takeaways

- **Class** — definition; **instance** — heap object со своим state.
- Assignment **aliases** objects (`1.1.6`).
- **`new`** аллоцирует; **`null`** — нет instance.
- **`this`** в instance methods; state per object.
- Default **`==`** — reference equality.
- GC — по **objects**, не по class definitions.
