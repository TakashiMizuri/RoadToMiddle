# Exercises: Debugging async — Tasks window

> Complete after passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Observe task states
**Goal:** See `Running` → `RanToCompletion` in Tasks window.
**Steps:** Console app with `await Task.Delay(2000)`. Break inside delay; inspect Tasks. Continue to completion.
**Success criteria:** Screenshot or notes: Id, Status transitions.

### E2. Enhanced exception stack
**Goal:** Compare stack with/without async enhancement.
**Steps:** Throw inside nested `async` call chain (3 levels). Run on .NET 8 with VS 2022.
**Success criteria:** Identify originating caller method in exception dialog.

## Medium

### M1. Reproduce sync-over-async deadlock
**Goal:** Diagnose with Tasks + Threads.
**Steps:** `async Task<string> GetDataAsync()` + sync `string s = GetDataAsync().Result` on UI or second thread.
**Success criteria:** Document blocked threads and fix with `await` all-the-way.

### M2. Fire-and-forget fault
**Goal:** Catch unobserved exception.
**Steps:** `_ = FaultyBackgroundAsync();` without await. Enable break on `UnobservedTaskException`.
**Success criteria:** Debugger stops; locate faulted task in Tasks.

## Hard (stretch)

### H1. ASP.NET hang
**Goal:** Debug minimal API that awaits slow EF query + hidden `.Result` in injected legacy service.
**Steps:** WebApplicationFactory or F5; use Tasks when request times out.
**Success criteria:** Written post-mortem: task states, root cause line, fix.

## Submission checklist

- [ ] Used Tasks window at least once
- [ ] Used Parallel Stacks or Threads for deadlock
- [ ] No `.Result`/`.Wait()` in fixed code
