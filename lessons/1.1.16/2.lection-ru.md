# StringBuilder vs string concatenation

> Roadmap: `1.1.16` · Node: `1.1` — C#: основы языка · Depth: **глубоко**

## Learning Objectives

После лекции ты сможешь:

- Объяснить **immutability `string`** (`1.1.4`) и почему repeated concat аллоцирует.
- Использовать **`StringBuilder`** для multi-part construction.
- Сравнить **`+`**, **`Concat`**, **`Join`**, interpolation и **`StringBuilder`** по allocations.
- Описать **internal buffer** StringBuilder и initial capacity.
- Знать, когда **простая concat достаточна**.
- Упомянуть production patterns: **`String.Create`**, streaming.

---

## Why This Matters

В **`1.1.4`** — **`string` immutable**: любая «мутация» создаёт **новый объект** в heap. Одна concat дёшева; десять тысяч в loop — GC pressure, latency в ASP.NET при сборке HTML/SQL/logs.

**`StringBuilder`** — mutable buffer: many **`Append`**, один **`ToString()`** в конце. Не использовать везде cargo-cult: **`Join`**, interpolation, **`String.Create`** часто лучше для малых случаев. Skill = **match tool to pattern** + понимание allocations.

## Core Concepts

### Immutability recap (связь с 1.1.4)

```csharp
string s = "Hello";
s += " World"; // новый string; старый eligible for GC
```

**`+`** → **`string.Concat`**. Chain в **loop** — intermediate strings каждый шаг → **O(n²)** total copying.

### Concat в loop — classic problem

```csharp
string csv = "";
foreach (var row in rows)
    csv += row.ToCsvLine() + "\n"; // плохо at scale
```

Fix: **`StringBuilder`**, **`string.Join`**, **`StreamWriter`**.

### StringBuilder basics

```csharp
var sb = new StringBuilder(capacity: 256);
sb.Append("Hello").AppendLine(" World");
string result = sb.ToString();
```

**Not thread-safe**. Methods: `Append`, `AppendLine`, `Insert`, `Clear`, `EnsureCapacity`.

### Join и Concat

```csharp
string joined = string.Join(", ", names);
string all = string.Concat(a, b, c);
```

**Join** часто one allocation для всего результата.

### Interpolation

```csharp
string msg = $"User {userId} logged in";
```

Efficient для few parts; **не замена** StringBuilder в loop (каждая итерация — новая string).

## Under the Hood

**Concat:** total length → one `string` alloc → copy parts.

**StringBuilder:** chunked **char** buffers (modern .NET — linked chunks для very large); grow on append; **`ToString()`** — one final copy to immutable string.

**LOH:** huge strings (>~85KB) — дороже Gen2. **Intern pool** (`1.1.4`) не помогает dynamic concat без explicit **`Intern`**.

**.NET 6+:** **`String.Create`**, **`DefaultInterpolatedStringHandler`**, UTF-8 writers — alternatives для hot paths.

---

## Examples

CSV loop → StringBuilder с preset capacity. **`string.Join`** для готового **`string[]`**. **`base + "/" + file`** — OK для 2–3 parts once.

---

## Common Mistakes

StringBuilder для 2 parts. Loop **`+=`**. **`ToString()`** внутри loop. Static shared builder без lock. Megastring вместо stream response.

---

## Production

Structured logging без manual concat. SQL — parameterize, не concat user input. Profile (**BenchmarkDotNet**) before micro-opts.

---

## Comparison

| Сценарий | Prefer |
|----------|--------|
| 2–4 parts once | +, Concat, $"..." |
| Many appends in loop | StringBuilder / stream |
| Join collection | string.Join |
| Huge text | Stream, not one string |

---

## Key Takeaways

- Immutability (`1.1.4`) → loop concat expensive.
- StringBuilder amortizes; set capacity.
- Join/Concat не obsolete.
- Stream для very large output.
