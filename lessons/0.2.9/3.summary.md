# Conventional Commits — краткая выжимка

> `0.2.9` · ~7 мин на повторение

## Главная идея

**Conventional Commits** — формат сообщений `type(scope)!: imperative description`, делающий history читаемой для людей и инструментов (changelog, SemVer).

## Ключевые пункты

- **`feat`** — новая capability (MINOR); **`fix`** — баг (PATCH).
- **`chore`** — deps, tooling; **`docs`**, **`test`**, **`refactor`**, **`ci`** — по смыслу.
- **Breaking:** `!` в header и/или `BREAKING CHANGE:` в footer (MAJOR).
- Subject: imperative, конкретный, ~50 символов; без `update`/`wip`.
- Один logical change per commit (`0.1.2`); squash PR title тоже conventional.

## Самые важные примеры

```
feat(auth): add refresh token rotation
fix(checkout): disable submit while loading
chore(deps): bump EF Core to 8.0.4
feat(api)!: remove v1 orders endpoint

BREAKING CHANGE: use /api/v2/orders instead.
```

## Частые ошибки

- Всё помечать как `fix`; past tense; breaking спрятан в body без `!`.
- Один commit = feat + unrelated chore.

## Мнемоника

> **feat = новое, fix = починили, chore = обслуживание, ! = ломаем контракт.**
