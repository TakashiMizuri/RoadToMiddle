## Answers

### 1. [Recall]
**Answer:** Document — **9** (`DOCUMENT_NODE`); element — **1** (`ELEMENT_NODE`).
**Why:** Numeric nodeType identifies node kind in DOM API.

### 2. [Recall]
**Answer:** `childNodes` — все дочерние узлы (text, comment, elements); `children` — только element nodes (`HTMLCollection`).
**Why:** Text whitespace nodes excluded from children.

### 3. [Recall]
**Answer:** **3** (`TEXT_NODE`).
**Why:** Character data nodes between/among elements.

### 4. [Understanding]
**Answer:** Перед `<span>` часто text node с `\n` и пробелами; `firstChild` указывает на него.
**Why:** HTML whitespace becomes text nodes.
**Common mistake:** assume firstChild is element.

### 5. [Understanding]
**Answer:** innerHTML **parses** string as HTML — attacker injects `<script>`, event handlers, `<img onerror>`; создаются element nodes. textContent создаёт только text node, markup не интерпретируется.
**Why:** XSS via parsed markup.

### 6. [Understanding]
**Answer:** Изменения через JS (или React) **сразу** отражаются в дереве, collections, layout — не immutable snapshot.
**Why:** DOM is mutable in-memory tree.

### 7. [Application]
**Answer:** Один child — **text node** (`nodeType` 3) со значением `"Hi"`.
**Why:** textContent replaces all children with single text node.

### 8. [Application]
**Answer:** Batch insert off-document → один append в DOM → меньше **reflows**/layout work.
**Why:** DocumentFragment avoids repeated layout thrashing.

### 9. [Analysis]
**Answer:** **Неверно** — a11y tree filtered (hidden, decorative, text merged into names).
**Why:** Not 1:1 mapping.

### 10. [Analysis]
**Answer:** **Hydration mismatch** warning/error — client vdom не совпадает с existing text/element nodes.
**Why:** React expects identical server/client structure.

### 11. [Understanding]
**Answer:** Blocks parsing while fetch+execute (unless module/type defer/async) — DOM ниже script не строится до выполнения.
**Why:** Classic script blocking behavior.

### 12. [Understanding]
**Answer:** HTML parsing rules **fix misnesting** (e.g. p cannot contain div) — canonical tree differs from author source.
**Why:** Spec insertion algorithm repairs structure.

### 13. [Recall]
**Answer:** `document.documentElement` (элемент `<html>`).
**Why:** Document node's single document element child.

### 14. [Application]
**Answer:** **`textContent`** (или `createTextNode` + append).
**Why:** No HTML parsing of user string.

### 15. [Understanding]
**Answer:** Element node type + tag name → browser assigns **implicit role** and default keyboard behavior for native controls.
**Why:** A11y tree built from element semantics.

### 16. [Analysis]
**Answer:** Collection **live** — length/index shift при remove → skipped elements или infinite loop.
**Why:** Mutating DOM during live collection iteration is unsafe without copy.
