# Span, ReadOnlySpan и Memory — stack-safe slicing

> Roadmap: `1.5.12` · Node: `1.5` — C#: .NET Runtime и память · Depth: **глубоко**

## Learning Objectives

- Объяснить **`Span<T>`** / **`ReadOnlySpan<T>`** как **ref struct** view без копирования.
- Slicing массивов и string с bounds checks.
- Различать **`Memory<T>`** для async/fields vs **`Span`** (stack-only).
- Использовать Span API для parse без аллокаций.
- Понимать ограничения **ref struct** (no async, no boxing).
- Связь с **`1.5.13` stackalloc** и **`1.5.14` ArrayPool**.

---

## Why This Matters

`Substring`, LINQ на hot path → Gen0 pressure. **`Span`** — окно в память (array, stack, native) **zero-copy**. BCL: **`Utf8Parser`**, **`SequenceReader`**, Kestrel pipes. **`Memory`** — когда есть **`await`**.

---

## Core Concepts

### Span как view

```csharp
int[] data = { 10, 20, 30 };
Span<int> slice = data.AsSpan(1, 2);
slice[0] = 99; // data[1] = 99
```

**`ReadOnlySpan<char>`** — parse string без новых string до **`ToString()`**.

### ref struct rules

Stack-only (с оговорками C# 12 ref fields): нельзя в **`object`**, **`async`**, большинство generics, escaping lambda. **Byref** не должен пережить backing memory.

### Memory<T>

Heap-storable, **`WriteAsync(ReadOnlyMemory<byte>)`**. Sync: **`memory.Span`**.

### stackalloc

```csharp
Span<byte> buf = stackalloc byte[128];
```

См. **`1.5.13`** — limits, не return Span из method.

---

## Under the Hood

Span = ref + length; JIT bounds check elimination. Pin для native interop. **ReadOnlySpan** over string — inline UTF-16.

---

## Common Mistakes

Return Span на stackalloc. Huge stackalloc. Span в async. **`ToArray()`** когда думал zero-copy.

---

## Key Takeaways

- **Span** — sync hot path; **Memory** — async/storage.
- **AsSpan()** — idiom для arrays/strings.
- ref struct = lifetime discipline.
