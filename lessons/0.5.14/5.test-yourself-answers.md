## Answers

### 1. [Recall]
**Answer:** Вложенный (nested) **browsing context** — отдельные window, document, origin.
**Why:** iframe — не просто «окошко», а полноценный nested document в tree of frames.

### 2. [Recall]
**Answer:** `X-Frame-Options: DENY` / `SAMEORIGIN` и CSP **`frame-ancestors`** (например `'none'` или `'self'`).
**Why:** Запрещают embedding вашей страницы в чужой frame — классическая clickjacking defense.

### 3. [Recall]
**Answer:** Maximum restrictions: opaque origin, no scripts, no forms, no top navigation, no popups (unless tokens added), и т.д.
**Why:** Пустой sandbox включает full set restrictive flags по spec.

### 4. [Understanding]
**Answer:** **Same-origin policy** — разные origins не могут access друг друга DOM/storage напрямую.
**Why:** Security boundary между browsing contexts; иначе любой embed читал бы cookies parent.

### 5. [Understanding]
**Answer:** Scripts получают normal same-origin privileges на вашем домене — могут access cookies, storage, DOM parent (если same site) — sandbox теряет смысл.
**Why:** Комбинация снимает изоляцию для same-origin scripted content — documented anti-pattern.

### 6. [Understanding]
**Answer:** Screen readers объявляют iframe как отдельный region; `title` даёт human-readable label (`0.5.11`).
**Why:** WCAG — embedded content must have accessible name; inner video title не заменяет iframe title для всех AT.

### 7. [Application]
**Answer:** `sandbox=""` (без tokens) или явно без `allow-scripts`; контент в iframe, не в parent — чтобы XSS в preview не выполнялся в origin приложения.
**Why:** Parent document XSS = полный compromise app; sandboxed iframe blocks script execution.

### 8. [Application]
**Answer:** `Content-Security-Policy: frame-ancestors 'none';` (или `'self'` если нужен только same-site framing).
**Why:** Modern preferred control vs legacy XFO; `'none'` = nobody may frame.

### 9. [Analysis]
**Answer:** **Mixed content** — browser блокирует или не загружает active mixed iframe на HTTPS page.
**Why:** Downgrade attack prevention; embed URL must be HTTPS.

### 10. [Understanding]
**Answer:** **`window.postMessage`**; parent validates **`event.origin`** against allowlist before trusting data.
**Why:** Единственный стандартный cross-origin channel; без origin check — spoofing.

### 11. [Analysis]
**Answer:** Static map не нуждается в camera/mic/geo — over-delegation; compromised embed could request sensitive permissions.
**Why:** Permissions Policy `allow` should follow least privilege.

### 12. [Recall]
**Answer:** iframe даёт consistent document model, **`title`**, **`sandbox`**, easier styling of container; embed — legacy replaced element.
**Why:** iframe — standard for navigable embedded content and security controls.
