# Exercises: Nullable Value Types

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Nullable calculator
**Goal:** Observe lifted arithmetic.

**Tasks:**
Write method `int? SafeAdd(int? a, int? b)` returning lifted sum. Test with `(5,null)`, `(null,null)`, `(3,4)`.

**Success criteria:** Predict outputs before running; no `.Value` usage.

---

### E2. Configuration reader
**Goal:** Practice `??` defaults.

**Tasks:**
```csharp
public sealed class AppOptions
{
    public int? MaxRetries { get; init; }
    public int EffectiveMaxRetries => /* implement: default 3 */;
}
```

**Success criteria:** `null` → 3; explicit `0` → 0.

---

## Medium

### M1. Optional search filters
**Goal:** Tri-state API parameters.

**Tasks:**
Model `ProductSearchQuery` with `decimal? MinPrice`, `decimal? MaxPrice`, `bool? InStock`. Method `IQueryable<Product> Apply(IQueryable<Product> q)` applies only specified filters.

**Success criteria:** All-null query returns unfiltered; `InStock = false` filters correctly.

---

### M2. Replace sentinel refactor
**Goal:** Migrate legacy code.

**Tasks:**
Refactor class using `DateTime.MinValue` as "no date" to `DateTime? ShippedAt`. Update methods comparing to `MinValue`.

**Success criteria:** No sentinel; compile; explain one bug old code allowed.

---

### M3. Null-coalescing chain
**Goal:** `??` evaluation order.

**Tasks:**
Write demo where `a ?? b ?? c` short-circuits expensive `c`. Use logging/counter in `c()` to prove call count.

**Success criteria:** When `a` has value, `c()` never invoked.

---

## Hard (stretch)

### H1. Lifted comparison quiz
**Goal:** Three-valued logic.

**Tasks:**
Fill table for `int? a`, `int? b` pairs: result of `a == b` as `bool?`. Cases: `(5,5)`, `(5,null)`, `(null,null)`, `(5,3)`.

**Success criteria:** Explain why `(null,null)` yields `true` for `==` but `(5,null)` yields `null`.
