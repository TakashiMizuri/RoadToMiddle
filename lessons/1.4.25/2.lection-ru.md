# Parallel.ForEachAsync — bounded parallel I/O

> Roadmap: `1.4.25` · Node: `1.4` — Async + ASP.NET preview · Depth: практика

## Learning Objectives

После урока ты сможешь:

- Использовать **`Parallel.ForEachAsync`** (.NET 6+) с **bounded concurrency** и **`CancellationToken`**.
- Настраивать **`MaxDegreeOfParallelism`** для защиты downstream и thread pool.
- Сравнивать **ForEachAsync** vs **Task.WhenAll** vs sequential **foreach** для I/O.
- Реализовать batch HTTP fan-out в **ASP.NET Core** (shared HttpClient, limits, cancel).

## Why This Matters

Задача: вызвать API для **500 ID**. Naive **`Task.WhenAll(ids.Select(...))`** — **500 connections**, socket exhaustion, **429**, memory pressure. Sequential **foreach** — safe but slow.

**Parallel.ForEachAsync** — structured concurrency с **ceiling** на parallel ops — middle ground для production. Поддерживает async lambdas, **CancellationToken**, **ParallelLoopResult**. В ASP.NET fan-out к microservices должен bound parallelism и пробрасывать **RequestAborted**.

## Core Concepts

### API

```csharp
await Parallel.ForEachAsync(source, new ParallelOptions
{
    MaxDegreeOfParallelism = 8,
    CancellationToken = cancellationToken
}, async (item, ct) => await ProcessItemAsync(item, ct));
```

**ForEachAsync** — для **I/O**; **Parallel.ForEach** — **CPU sync**, не для await.

**MaxDegreeOfParallelism** — max concurrent delegate invocations. Default **ProcessorCount** — часто не то для I/O; для HTTP выбирай по **downstream capacity** (4–16).

### vs Task.WhenAll

| Pattern | Concurrency |
|---------|-------------|
| Sequential | 1 |
| Task.WhenAll | Unbounded N |
| ForEachAsync | Bounded |

**WhenAll** создаёт все tasks сразу — при большом N опасно. ForEachAsync — bounded scheduling.

### ASP.NET

```csharp
await Parallel.ForEachAsync(ids, new ParallelOptions
{
    MaxDegreeOfParallelism = 10,
    CancellationToken = cancellationToken
}, async (id, ct) =>
{
    var dto = await client.GetFromJsonAsync<ItemDto>($"/items/{id}", ct);
    results.Add(dto);
});
```

**IHttpClientFactory**, не new HttpClient per item. **ConcurrentBag** для results.

## Under the Hood

Internal scheduling с respect к **MaxDegreeOfParallelism** (semaphore-like). I/O wait освобождает threads — не как sync **Parallel.ForEach**.

---

## Examples

URLs download limit 6. Chunked WhenAll как alternative. **Anti-pattern:** Parallel.ForEach + `.Result`.

---

## Common Mistakes & Anti-patterns

Unbounded WhenAll. Sync ForEach для I/O. Ignore cancel. Non-thread-safe list. New HttpClient per iteration.

---

## Production & Real-World Notes

Tune по p95, 429, socket errors. Polly + high parallelism = retry storm. **RateLimiter** для API limits.

---

## Quick Reference

I/O → **ForEachAsync**. Override **MaxDegreeOfParallelism**. Factory HttpClient. Forward **CancellationToken**.

---

## Key Takeaways

- **ForEachAsync** bounds concurrent async work.
- Не путать с **Parallel.ForEach**.
- **Task.WhenAll** только для small N.
- ASP.NET: factory + **RequestAborted**.
