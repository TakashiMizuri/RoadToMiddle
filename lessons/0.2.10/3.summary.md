# Code review etiquette — краткая выжимка

> `0.2.10` · ~6 мин на повторение

## Главная идея

Code review — **collaboration gate** в PR workflow (`0.2.8`): качество и знания до merge в `main`. Etiquette — **ясная engineering communication**, не вежливость ради галочки.

## Ключевые пункты

- **Зачем:** баги + knowledge sharing + standards + design + ownership.
- **Дача:** review **code not person**; specific, actionable; tiers: **blocking | suggestion | nit | question | praise**.
- **Приём:** assume good intent; read all first; **fix or explain** с evidence.
- **Blocking:** correctness, security, critical paths без tests.
- **Non-blocking:** style nits, optional refactors.
- Async: полные предложения, resolve threads, no sarcasm.

## Самые важные примеры

Плохо: «This is wrong.»  
Хорошо: **[blocking]** 200 при validation error ломает API contract — use `ValidationProblem`.

Author: `fix(auth): address review — validate audience` + push → resolve thread.

## Частые ошибки

Rubber-stamping, bikeshedding, defensiveness, drive-by refactor, merge при red CI.

## Мнемоника

> **Block risk, suggest craft, nit kindly. Fix or explain — never defend ego.**
