## Answers

### 1. [Recall]
**Answer:** **2 minutes** (default `HandlerLifetime`).

**Why:** Documented factory default for DNS/pool rotation.

---

### 2. [Recall]
**Answer:** Type-safe DI, no magic strings, easier unit testing with mocked handler, encapsulates routes.

**Why:** Typed client pattern benefits.

---

### 3. [Understanding]
**Answer:** CreateClient returns thin facade over **shared rotated handler pool**; new HttpClient creates **new handler/pool** each time → socket exhaustion.

**Why:** Core factory vs new() distinction from 1.4.26.

---

### 4. [Understanding]
**Answer:** Cross-cutting outbound concerns (auth, logging, resilience) composable like middleware without duplicating per call site.

**Why:** DelegatingHandler purpose.

---

### 5. [Application]
**Answer:**
```csharp
services.AddTransient<CorrelationIdHandler>();
services.AddHttpClient("payments", (sp, c) =>
{
    c.BaseAddress = new Uri(config["Payments:BaseUrl"]!);
}).AddHttpMessageHandler<CorrelationIdHandler>();
```
Inject `IHttpClientFactory`, `CreateClient("payments")`.

**Why:** Standard registration pattern.

---

### 6. [Application]
**Answer:** Inject **`IServiceScopeFactory`**; in `ExecuteAsync`, `using var scope = _scopeFactory.CreateScope(); var api = scope.ServiceProvider.GetRequiredService<ICatalogApi>();`

**Why:** Resolve scoped/transient typed client per operation from singleton host.

---

### 7. [Analysis]
**Answer:** Retries run **without** auth token on first attempts → 401 loops, wasted retries, possible lockout; auth should be **outer** (early) or token refreshed before retry policy.

**Why:** Handler order matters.

---

### 8. [Analysis]
**Answer:** **Yes** — cannot tune timeout, resilience, base URL per service; coupling; wrong Polly thresholds; violates one-client-per-downstream practice.

**Why:** Production naming/config separation.

---

### 9. [Understanding]
**Answer:** **Primary** = inner **SocketsHttpHandler** (TCP/TLS/pool); **AddHttpMessageHandler** = outer **DelegatingHandler** wrappers for app logic/resilience.

**Why:** Two layers of handler stack.

---

### 10. [Application]
**Answer:** **`AddStandardResilienceHandler()`** on `IHttpClientBuilder` (.NET 8+ Microsoft.Extensions.Http.Resilience).

**Why:** Bridges to Polly pipelines (`1.4.28`).
