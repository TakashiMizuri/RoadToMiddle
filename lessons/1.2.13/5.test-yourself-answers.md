## Answers

### 1. [Recall]
**Answer:** Primary constructor; init-only properties `X`, `Y`; `Equals`/`GetHashCode`/`==`; `ToString`; (бонус) `Deconstruct`, `<Clone>$` for `with`.
**Why:** Compiler synthesizes data-class boilerplate.

### 2. [Recall]
**Answer:** Создаёт **копию** record `a` с указанными изменениями; исходный `a` **не меняется**.
**Why:** Non-destructive mutation via clone.

### 3. [Recall]
**Answer:** **Reference type** (heap, как class). `record struct` — отдельный keyword для value type.
**Why:** Default `record` = `record class`.

### 4. [Understanding]
**Answer:** Record **overrides equality** по значениям полей; class по умолчанию сравнивает **reference identity**.
**Why:** Synthesized structural equality.

### 5. [Understanding]
**Answer:** `with` копирует **ссылку** на тот же `List`; добавление элементов видно в обеих «копиях».
**Why:** Shallow copy семантика.

### 6. [Understanding]
**Answer:** EF **change tracker** ожидает mutable instance с обновляемыми properties/navigations; immutable record ломает update patterns и identity semantics.
**Why:** ORM designed for mutable entities.

### 7. [Application]
**Answer:** `var u = customer with { Email = "new@mail.com" };`
**Why:** Idiomatic `with` update.

### 8. [Application]
**Answer:** `a.Qty = 2`, `b.Qty = 3`, `b.Price = 5m` (скопировано из `a`).
**Why:** `with` меняет только `Qty`; остальные поля копируются.

### 9. [Application]
**Answer:** `public readonly record struct Rgb(byte R, byte G, byte B);`
**Why:** Positional readonly record struct.

### 10. [Analysis]
**Answer:** **`2`** — shared `List` reference.
**Common mistake:** Думать, что `with` глубоко клонирует коллекции.

### 11. [Analysis]
**Answer:** **Да, breaking** — все call sites / JSON clients должны передать новый parameter; deserialization contract меняется.
**Why:** Primary constructor — часть public contract.

### 12. [Understanding]
**Answer:** `record struct` — **value type**, обычно stack (если не boxed); `record class` — **heap** object.
**Why:** CLR value vs reference semantics.

### 13. [Application]
**Answer:** **`record`** — immutable command, value equality для тестов/logging, мало boilerplate.
**Why:** Commands — данные без identity.

### 14. [Analysis]
**Answer:** **Нет**, если mutable property — **ссылочный тип**, не заменённый в `with` (shared inner object). **Да**, если все вложенные данные immutable или property полностью заменена новым объектом.
**Why:** Shallow copy rule.

### 15. [Application]
**Answer:** Удобная **deconstruction** в tuple/pattern matching: `(int x, int y) = point;`
**Why:** Synthesized `Deconstruct` из positional parameters.
