# Answers: yield return — iterator pattern

- 90–100%: отлично

## Answers

### 1. [Recall]
**Answer:** **`IEnumerable`**, **`IEnumerable<T>`**, **`IEnumerator`**, **`IEnumerator<T>`**, **`IAsyncEnumerable<T>`** (async iterators).
**Why:** Language rules for iterator blocks.

### 2. [Recall]
**Answer:** Завершает iteration (end of sequence).
**Why:** Аналог **`return`** без value в iterator.

### 3. [Understanding]
**Answer:** Lazy evaluation — код до first **`yield`**/`MoveNext`** не выполнен; **`StreamReader`** открывается при start enumeration.
**Why:** Deferred execution.

### 4. [Understanding]
**Answer:** Сохраняются в **fields generated state machine** между **`MoveNext()`** calls.
**Why:** Compiler lifts locals into iterator object.

### 5. [Understanding]
**Answer:** Каждый **`foreach`** создаёт **новый enumerator** → method body / source work runs again.
**Why:** No shared memoization unless implemented.

### 6. [Application]
**Answer:** **`ToList()`** (или **`ToArray()`**) — фиксированный snapshot для multiple passes.
**Why:** **`IEnumerable`** lazy и не кэширует по умолчанию.

### 7. [Application]
**Answer:** **`IAsyncEnumerable`** + **`await foreach`** (или sync **`yield`** over sync stream if API sync) — не materialize 100k.
**Why:** Streaming vs O(n) memory.

### 8. [Analysis]
**Answer:** **Да** — **`foreach`** вызывает **`Dispose()`** на enumerator → **`finally`** в iterator.
**Why:** Resource cleanup contract.

### 9. [Analysis]
**Answer:** **0** — **`start`** печатается на first **`MoveNext()`**, не при **`M()`**.
**Why:** Только создание iterator object при **`M()`**.

### 10. [Understanding]
**Answer:** Async uses **`MoveNextAsync`**, **`await`** между yields; не блокирует thread pool на I/O; другой compiler state machine.
**Why:** Async streams C# 8+.

### 11. [Analysis]
**Answer:** **Нет** — **`yield return`** в **`catch`** forbidden; в **`finally`** тоже restrictions.
**Why:** Iterator safety rules.

### 12. [Application]
**Answer:** При **enumeration consumer'а**, на каждом element при **`MoveNext()`**, не при вызове **`Where(...)`**.
**Why:** Deferred LINQ-like semantics.
