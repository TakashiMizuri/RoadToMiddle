# Exercises: Class Blueprint, Object Instance, Reference Semantics

> Complete after reading `1.lection-eng.md`, `2.lection-ru.md` and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Instance count
**Goal:** Connect `new` to heap allocation.

Write a small program with class `Counter { public int Value; }`. Create variables `a`, `b`, `c` as in lecture Example 2 alias pattern. Answer: how many `Counter` objects exist after each line?

**Success criteria:** Correct count (2) with explanation referencing `new` vs assignment.

---

### E2. Predict output
**Goal:** Reference semantics.

```csharp
class Scoreboard { public int Points; }
var s1 = new Scoreboard { Points = 10 };
var s2 = s1;
s2.Points = 99;
Console.WriteLine(s1.Points);
```

Predict output without running, then verify.

**Success criteria:** Output `99` with alias explanation.

---

## Medium

### M1. Parameter mutation vs reassignment
**Goal:** Distinguish shared mutation from local reassignment.

Implement `Bump(Scoreboard s)` (+1 to Points) and `Replace(Scoreboard s)` (assigns `s = new Scoreboard()`). Call both from `Main` and document which affects caller's variable.

**Success criteria:** Written observations matching lecture Example 3.

---

### M2. Reference vs value equality
**Goal:** Identity awareness.

Create two `Customer` instances with same `Name`. Compare with `==`, `ReferenceEquals`, and (optional) override discussion. Document results.

**Success criteria:** Shows `==` false by default; `ReferenceEquals` same as `==` unless overridden.

---

## Hard (stretch)

### H1. Diagram a mini domain
**Goal:** Blueprint vs instances visually.

Pick a domain type (`Invoice`, `ShoppingCart`). Draw ASCII: one class box, three instances, two alias arrows. Label heap vs stack references.

**Success criteria:** Diagram shows shared alias group and separate instance.
