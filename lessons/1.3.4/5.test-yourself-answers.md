## Answers

### 1. [Recall]
**Answer:** **`TResult`** — last type parameter.
**Why:** BCL convention for all Func arities.

### 2. [Recall]
**Answer:** **`Func<T, bool>`** — `bool(T obj)`.
**Why:** Predicate is specialized delegate for boolean tests.

### 3. [Recall]
**Answer:** **16** type parameters (for `Action<T1,...,T16>`).
**Why:** BCL definition limit.

### 4. [Understanding]
**Answer:** When signature expresses domain contract reused across assembly — named delegate or interface improves readability and versioning.
**Why:** `Func<Order, bool>` hides business meaning (IsValid? IsShipped?).

### 5. [Understanding]
**Answer:** `Task.Run` accepts `Func<Task>` and awaits; `Action` is void — async lambda becomes **async void**, exceptions unobservable, no await by caller.
**Why:** async void only for event handlers by convention.

### 6. [Application]
**Answer:** `Func<User, bool> filter = u => u.Age >= 18;` or lambda inline in Where.

### 7. [Application]
**Answer:** `Func<int, string> f = int.ToString;`

### 8. [Application]
**Answer:** **`Predicate<int>`** (equivalently lambda → `Func<int,bool>`).

### 9. [Analysis]
**Answer:** **Does not compile** — cannot convert different Func delegate types (different arity/signature).
**Why:** Not variance case — arity mismatch.

### 10. [Analysis]
**Answer:** **`Action<Item>`** — no return needed; `Func<Item,bool>` implies boolean result misuse unless filtering.
**Why:** Match signature to intent.

### 11. [Application]
**Answer:**
```csharp
int Process(IEnumerable<int> nums, Func<int, int> transform)
    => nums.Sum(transform);
```

### 12. [Understanding]
**Answer:** **Yes** — contravariance on input `T`, covariance on `string` return.
**Why:** `Func<in T, out TResult>`.

### 13. [Application]
**Answer:** `nums.Where(p);` — implicit conversion Predicate to Func or overload accepts.

### 14. [Recall]
**Answer:** **No `Func<void>`** — use **`Action`** for void.
**Why:** void not a type argument.

### 15. [Analysis]
**Answer:** New lambda/delegate allocation each iteration if recreated; capture overhead — cache delegate or use for-loop body inline.
**Why:** Micro-optimization / allocation awareness.
