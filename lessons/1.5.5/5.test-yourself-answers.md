# Answers: Heap Allocation and Object Headers

## Answers

### 1. [Recall]
**Answer:** `new` class/array/string/delegate, boxing, many runtime objects (exceptions, async machines, etc.).
**Why:** Reference payloads and boxed values on heap.

### 2. [Recall]
**Answer:** Object header (~8 bytes) and MethodTable pointer (~8 bytes) before fields.
**Why:** Sync/type/GC metadata.

### 3. [Recall]
**Answer:** ~**24 bytes** (header + MT + alignment) typical 64-bit CLR.
**Why:** Fixed overhead per object.

### 4. [Understanding]
**Answer:** Type identity, virtual dispatch (vtable), GC descriptor for scanning reference fields.
**Why:** Core runtime type representation.

### 5. [Understanding]
**Answer:** **Embedded inside** instance object on heap — not separate stack object.
**Why:** 1.1.6 container rule.

### 6. [Understanding]
**Answer:** Gen0 bump pointer fast until collection triggered; GC mark/sweep/compact costs CPU/pauses.
**Why:** Generational model 1.5.7.

### 7. [Application]
**Answer:** **Boxing** — heap object wrapping `int` 42.
**Why:** Value stored in heap box.

### 8. [Application]
**Answer:** 1000 bytes SOH Gen0; 100_000 ≥ LOH threshold → LOH Gen2 immediately.
**Why:** 1.5.8 threshold.

### 9. [Analysis]
**Answer:** Minimum ~24 bytes overhead even with zero fields.
**Why:** Header + MethodTable.

### 10. [Analysis]
**Answer:** CPU alignment — `long` at 8-byte boundary; padding between fields.
**Why:** Performance correctness.

### 11. [Application]
**Answer:** Approx bytes allocated on current thread since tracking start (API semantics per docs).
**Why:** Allocation profiling aid.

### 12. [Understanding]
**Answer:** Static — per-type storage (one slot per load context); instance — per object on heap.
**Why:** Different lifetimes.

### 13. [Application]
**Answer:** Enumerators, delegates, closure objects may allocate per iteration.
**Why:** Hidden GC pressure.

### 14. [Analysis]
**Answer:** Struct — copy 256 bytes on stack/frame each call; class — copy reference, object on heap once.
**Why:** Trade copy vs indirection.

### 15. [Understanding]
**Answer:** Pin prevents move during compact — can fragment heap (SOH/LOH).
**Why:** GC cannot relocate pinned objects.
