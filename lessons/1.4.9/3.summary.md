# SynchronizationContext — краткая выжимка

> `1.4.9` · ~10 мин

## Главная идея

**SynchronizationContext** — абстракция «выполни continuation на нужном scheduler'е». UI имеет context; **ASP.NET Core — `Current == null`**, continuations на thread pool. `HttpContext` flow'ит через **AsyncLocal**, не через SyncContext.

## Ключевые пункты

- `Post` / `Send` — queue delegate на UI или special thread.
- Default `await` захватывает SyncContext если non-null.
- Framework ASP.NET — был request SyncContext → deadlock с `.Result`.
- Core — нет request SyncContext → `ConfigureAwait(false)` в app optional.
- WPF/WinForms — custom context; updates только на UI thread.
- State machine `MoveNext` — target queue зависит от capture (`1.4.7`).

## Частые ошибки

- Перенос Framework deadlock recipes на Core без проверки.
- Думать, что `ConfigureAwait(true)` «сохраняет HttpContext».
