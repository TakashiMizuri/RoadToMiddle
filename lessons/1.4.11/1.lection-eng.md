## Why This Matters

Users close browser tabs; load balancers timeout; deployments drain connections. Work must **stop cooperatively** to free DB connections, HTTP sockets, and CPU. **`CancellationToken`** is the standard signal; ignoring it wastes resources and causes **orphan tasks** whose continuations still run **`MoveNext`**.

Middle developers implement **`Register`** callbacks for unmanaged handles, link **user cancel + timeout + request abort**, and know **`ThrowIfCancellationRequested`** vs returning gracefully.

## Core Concepts

### Cooperative Model

Cancellation in .NET is **cooperative**: `CancellationToken` exposes **`IsCancellationRequested`** and **`ThrowIfCancellationRequested()`**. Your code and framework methods (`Task.Delay`, `HttpClient`, EF) must **poll** or register callbacks. The runtime does not abort threads.

When cancelled, async methods typically throw **`OperationCanceledException`** (often with **`task.IsCanceled`** true). The state machine propagates exception to the awaiting caller — same channel as other failures (`1.4.19`).

### CancellationTokenSource

**`CancellationTokenSource`** owns the mutable cancellation state. **`cts.Token`** is the read-only **`CancellationToken`** passed downstream. **`cts.Cancel()`** sets flag and invokes registered callbacks. **`CancelAfter(ms)`** schedules cancellation on timer thread pool work.

Always **`using`** or dispose CTS when done to release timer registrations.

```csharp
using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(30));
// or
using var cts = new CancellationTokenSource();
cts.CancelAfter(30_000);
var token = cts.Token;
await FetchAsync(token);
```

### Register

**`token.Register(callback)`** runs when cancellation fires — useful for closing **`Socket`**, releasing **`Semaphore`**, or logging. Callbacks run synchronously during **`Cancel()`** — keep them short; deadlocks possible if callback blocks.

**`Register(callback, state, useSynchronizationContext, executeSynchronously)`** — control marshaling; default avoids SyncContext unless needed.

Multiple registrations are allowed; order is undefined.

### Linked Tokens

**`CancellationTokenSource.CreateLinkedTokenSource(token1, token2)`** cancels when **any** source cancels — pattern for **user cancel OR timeout OR RequestAborted**:

```csharp
using var linked = CancellationTokenSource.CreateLinkedTokenSource(
    userCts.Token,
    HttpContext.RequestAborted);
await _service.ProcessAsync(linked.Token);
```

Linking does not extend lifetime of source tokens — dispose linked CTS when operation completes.

### Propagation in Async Chains

Pass **`CancellationToken`** as last parameter convention. At each **`await`**, pass token to cancellable overloads. In loops, check **`token.ThrowIfCancellationRequested()`** between iterations so state machine exits without starting next I/O.

Non-cancellable work: respect token before/after CPU sections; use **`WaitHandle.WaitOne`** with token via **`token.WaitHandle`** carefully (deprecated patterns — prefer async APIs).

## Under the Hood

**`CancellationToken`** is a struct wrapping reference to **`CancellationTokenSource`** internal state (or **`None`** singleton). **`None`** never cancels; **`CanBeCanceled`** false.

**`Cancel()`** sets volatile flag, then invokes callback list under lock. **`ThrowIfCancellationRequested`** checks flag and throws **`OperationCanceledException`** with token stored on exception in modern patterns.

**`Task.Delay(1000, token)`** registers with CTS timer; on cancel completes task as **Canceled** state — await throws **`TaskCanceledException`**.

State machine: if `ThrowIfCancellationRequested` runs before await, method completes synchronously as canceled without scheduling continuation. If cancel during await, completing task faults/cancels → **`MoveNext`** throws path in state machine.

**`HttpContext.RequestAborted`** links to connection reset — Kestrel triggers CTS when client disconnects.

## Syntax / API

```csharp
// None — default for optional APIs
public Task DoWorkAsync(CancellationToken cancellationToken = default);

// Throw pattern
cancellationToken.ThrowIfCancellationRequested();

// Graceful return
if (cancellationToken.IsCancellationRequested)
    return;

// Register cleanup
await using var reg = cancellationToken.Register(() => socket.Close());
// C# 8+: registration disposes with await using pattern via Dispose()

CancellationTokenSource.CreateLinkedTokenSource(a, b, c);
cts.CancelAfter(TimeSpan.FromMinutes(5));
```

## Examples

### Timeout + User Cancel

```csharp
public async Task SearchAsync(string query, CancellationToken userToken)
{
    using var timeoutCts = new CancellationTokenSource(TimeSpan.FromSeconds(10));
    using var linked = CancellationTokenSource.CreateLinkedTokenSource(
        userToken, timeoutCts.Token);

    await _engine.SearchAsync(query, linked.Token);
}
```

Whichever fires first cancels the search.

### ASP.NET Core Action

```csharp
[HttpGet]
public async Task<IActionResult> Stream(CancellationToken cancellationToken)
{
    // Same as RequestAborted when from model binding
    await foreach (var chunk in _reader.ReadAsync(cancellationToken))
        await Response.WriteAsync(chunk, cancellationToken);
    return new EmptyResult();
}
```

## Common Mistakes & Anti-patterns

**Creating new CTS per call but never disposing** — timer leak with **`CancelAfter`**.

**Swallowing `OperationCanceledException`** without rethrow or logging — hides real cancel vs fault.

**Not passing token to `Task.WhenAll` children** — parallel work continues after parent canceled (`1.4.12`).

**Long sync work ignoring token** — cancellation delayed until sync section ends.

**Using `cts.Token` after dispose** — undefined behavior; capture token before dispose scope.

## Production & Real-World Notes

Log cancellation at **Debug** level with correlation ID — not Error unless unexpected. **`Polly`** combines timeout + retry with tokens (`1.4.28`).

Background **`IHostedService`** uses **`stoppingToken`** from host shutdown — link with internal work tokens.

## Key Takeaways

- Cancellation is **cooperative**; pass and check **`CancellationToken`**.
- **`Register`** for cleanup; keep callbacks fast.
- **`CreateLinkedTokenSource`** — cancel on any condition.
- **`CancelAfter`** — built-in timeout CTS.
- State machine exits via throw or early return — plan **`OperationCanceledException`** handling.
- **`RequestAborted`** — client disconnect in ASP.NET Core.
