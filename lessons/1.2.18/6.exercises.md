# Exercises: Partial classes

> Complete after reading lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Split a class by hand
**Goal:** Experience manual partial merge.
**Steps:**
1. Create `Widget.cs`: `public partial class Widget { public int Id { get; set; } }`
2. Create `Widget.Behavior.cs`: `public partial class Widget { public string Describe() => $"Widget {Id}"; }`
3. Build; instantiate and call `Describe()`.
**Success criteria:** Single type with members from both files; both files use `partial`.

### E2. Spot the wrong file
**Goal:** Practice generated-file discipline.
**Steps:**
1. Create `Service.g.cs` with header `// <auto-generated>` and `public partial class Service { public void Generated() {} }`
2. Add `Service.cs` partial with `public void Custom() {}`
3. Document in a one-line comment where you would add `Validate()` and why not in `.g.cs`.
**Success criteria:** Written rule in comment matches lecture.

## Medium

### M1. Partial method hook
**Goal:** Optional partial method behavior.
**Steps:**
1. `Logger.generated.cs`: declare `partial void OnWrite(string msg);` and call it from `Write`.
2. `Logger.cs`: implement `OnWrite` to print to console.
3. Build; call `Write("test")`.
4. Remove implementation from `Logger.cs`; rebuild — confirm project still compiles (calls stripped).
**Success criteria:** Observed compile with and without implementation.

### M2. DbContext-style split
**Goal:** Mimic EF customization boundary.
**Steps:**
1. `AppDbContext.cs`: constructor + empty `OnModelCreating` stub (no EF required — plain methods).
2. `AppDbContext.Data.cs`: same partial with `DbSet`-like properties as auto-properties only.
3. Explain which file a teammate should edit for new entity property if `.Data.cs` were regenerated.
**Success criteria:** Clear ownership explanation in README comment or submission note.

## Hard (stretch)

### H1. Find real generated code
**Goal:** Map theory to repository or template.
**Steps:**
1. Search a .NET solution for `partial class` + `Designer` or `.g.cs`.
2. List: type name, generated file, hand-written partial file.
3. Identify one customization point (event handler, `OnModelCreating`, partial method).
**Success criteria:** Three-item inventory with paths (no need to paste secrets).

## Submission checklist

- [ ] Every fragment of the same type has `partial`
- [ ] No business logic placed in mock `.g.cs` / `.Designer.cs`
- [ ] You can explain partial method zero-cost when unimplemented
