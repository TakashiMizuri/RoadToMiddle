# Postman, Bruno и curl — ручное тестирование API

> Roadmap: `0.3.12` · Node: `0.3` — HTTP и Web · Depth: практика

## Learning Objectives

После этого урока ты сможешь:

- Объяснить, **зачем manual HTTP clients** нужны рядом с automated tests и Swagger UI.
- Отправлять запросы через **curl** — methods, headers, body types, чтение status и response headers.
- Использовать **Postman** для collections, environments и reusable auth.
- Использовать **Bruno** как **Git-friendly** file-based альтернативу cloud collections.
- Воспроизводить баги точными requests (content types из `0.3.11`, auth, query strings).
- Экспортировать и делиться requests форматами для команды и CI.

## Why This Matters

Swagger UI и integration tests покрывают happy path, но production debugging начинается с вопроса: **что именно мы отправили и что вернулось?** Баг frontend может быть missing header; 403 — stale token; 415 — wrong `Content-Type`. Manual API tools **изолируют HTTP layer** — без React state, EF Core и browser cache — итерация за секунды.

**curl** есть на каждом server и в CI image — must-have для Middle. **Postman** доминирует там, где нужен GUI, docs и mocks. **Bruno** растёт там, где **collections живут в репозитории**, а не в vendor cloud. Одна mental model: собрать request → raw response → менять одну переменную. Fluency ускоряет OAuth debugging и webhook replay.

## Core Concepts

### Manual testing vs другие слои

**Unit tests** mock dependencies — real HTTP serialization может не тестироваться. **E2E** гоняет full stack, но прячет request details в traces. **OpenAPI/Swagger** документирует contract, но генерирует наивные requests. Manual clients: контроль **каждого byte и header**, reproducible examples, share в Slack/ticket.

Workflow научный: **гипотеза** → **одно изменение** → **наблюдение** → **save в collection**. Так debug'ят seniors до правки application code.

### curl — универсальный CLI client

**curl** (Windows: **`curl.exe`**, не путать с PowerShell alias) говорит HTTP напрямую. Scripts, README, SSH на jump box без GUI.

Флаги:

- **`-X METHOD`** — verb (POST часто inferred с `-d`).
- **`-H "Name: value"`** — headers.
- **`-d 'body'`** — body; default POST.
- **`-G`** — data в query string на GET.
- **`-i`** — response headers в output.
- **`-v`** — verbose (TLS, headers) — must для debug.
- **`-F`** — multipart.
- **`--data-urlencode`** — безопасный urlencoded.

```bash
curl -sS -w "\nHTTP %{http_code}\n" \
  -X POST "https://httpbin.org/post" \
  -H "Content-Type: application/json" \
  -d '{"probe":true}' -i
```

Output: status line, headers, blank line, body. `jq` для JSON в shell.

### Postman — GUI collections и environments

**Postman:** **collections**, **environments** (`baseUrl`, `accessToken`), **pre-request scripts** / **tests** (JS). Tabs: Params, Headers, Body (raw JSON, form-data, urlencoded) → **Send**.

**Environments:** `{{baseUrl}}/api/users` — localhost vs staging. Login test script: `pm.environment.set("accessToken", ...)`.

Share: workspace или export JSON. Minus: drift от code; secrets в export; compliance vs cloud sync.

### Bruno — collections как files в Git

**Bruno:** requests как `.bru` в repo, `environments/local.bru`. **Git diff** при смене health-check URL; PR review включает API changes.

Trade-off: меньше ecosystem; нет heavy cloud — плюс для regulated/offline teams.

**Postman** — individual productivity + platform sharing; **Bruno** — **version control** и review рядом с source.

### Reproducible request

Любой tool должен фиксировать:

1. Method + full URL (query included).
2. Headers — `Authorization`, `Content-Type`, `Accept`, correlation ID.
3. Body — encoding из `0.3.11`.
4. Expected status + key response headers.
5. Environment — server, user role.

### Auth patterns

- **Bearer JWT:** `Authorization: Bearer eyJ...`
- **Basic:** `-u user:pass`
- **API key:** header `X-Api-Key`
- **Cookie:** `-b` / cookie jar после login

Secrets — **environment variables**, не commit.

## Under the Hood

curl — **libcurl** (DNS, TCP, TLS). **`-v`** показывает TLS fail, redirects (`-L`), HTTP/2. Postman/Bruno embed HTTP clients; multipart boundaries как у browser. **Code gen** Postman → curl/fetch после успешного manual test.

**localhost vs Docker:** `host.docker.internal` vs `localhost` — manual tools сразу показывают wrong-host.

## Syntax / Commands / API

```bash
curl "https://api.example.com/items?status=open"
curl -X PUT "$BASE/items/1" -H "Content-Type: application/json" -d @body.json
curl -X DELETE "$BASE/items/1" -H "Authorization: Bearer $TOKEN"
curl -c cookies.jar -X POST .../login -d ...
curl -b cookies.jar .../me
curl -L -w "%{http_code}" https://example.com/redirect
```

| Task | Postman | Bruno |
|------|---------|-------|
| Variables | Environment | Environment folder |
| Share | Export / workspace | Git commit |

## Common Mistakes & Anti-patterns

Swagger Try it без Content-Type check. Secrets в shared collections. PowerShell **`curl`** ≠ curl.exe. Игнор response headers. Только GUI — на production SSH есть curl. TLS — смотри `-v`.

---

## Production & Real-World Notes

Golden requests в repo (Bruno) или runbook. **Newman** / **bru CLI** в CI smoke. Rate limits при team manual testing. **X-Correlation-ID** в manual tests для log trace.

## Comparison / Trade-offs

| Tool | Strength | Weakness |
|------|----------|----------|
| curl | Universal, CI | No collection UI |
| Postman | GUI, mocks | Cloud, drift |
| Bruno | Git-native | Smaller community |

## Key Takeaways

1. Manual tools **изолируют wire protocol**.
2. **curl** — servers, scripts, repro.
3. **Postman** — GUI + team sync.
4. **Bruno** — **files in Git**.
5. Explicit **Content-Type** и **Authorization**.
6. Читай **status + headers**.
7. Save working requests как living docs.
