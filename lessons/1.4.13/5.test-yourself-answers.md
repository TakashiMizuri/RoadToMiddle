# Answers: async void

## Answers

### 1. [Recall]
**Answer:** **Event handlers** (UI) where signature must be void.
**Why:** Language/event model constraint.

### 2. [Recall]
**Answer:** **Нет** — compile error.
**Why:** void not awaitable.

### 3. [Understanding]
**Answer:** **Task** — on Task object; **void** — unhandled / SyncContext / AppDomain, not to caller.
**Why:** No task surface.

### 4. [Understanding]
**Answer:** **`AsyncVoidMethodBuilder`**.
**Why:** vs AsyncTaskMethodBuilder.

### 5. [Application]
**Answer:** **Untestable, unobservable exceptions**, cannot await.
**Why:** Service anti-pattern.

### 6. [Application]
**Answer:** **Unhandled exception** — dialog or crash via dispatcher pipeline.
**Why:** No task catch.

### 7. [Analysis]
**Answer:** **Task return ignored** by event wire-up — still fire-and-forget; void + try/catch or Task helper common.
**Why:** Events expect void.

### 8. [Analysis]
**Answer:** **Ненадёжен** — failures may not fail test.
**Why:** xUnit expects async Task.

### 9. [Understanding]
**Answer:** **Да** — IAsyncStateMachine MoveNext (`1.4.7`); different builder/completion.
**Why:** Same codegen core.

### 10. [Application]
**Answer:** **High** — unobserved faults, pipeline break.
**Why:** Use Task<IActionResult>.

### 11. [Application]
**Answer:** `_ = RunSafe(SaveAsync);` where RunSafe logs Task exceptions.
**Why:** `1.4.30` pattern.

### 12. [Analysis]
**Answer:** **Нет** — void returns immediately after first suspend; no join.
**Why:** No Task handle.
