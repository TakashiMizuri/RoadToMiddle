# Exercises: LINQ Performance — Multiple Enumeration

## Easy

### E1. Count passes
**Goal:** Write snippet with `Any` + `ToList`; refactor to single pass; comment before/after pass count.
**Success criteria:** One enumeration after refactor.

### E2. ICollection check
**Goal:** Given `IEnumerable<T>`, get count O(1) when possible without full scan.
**Success criteria:** Pattern match `ICollection` / `IReadOnlyCollection`.

## Medium

### M1. Fix the handler
**Goal:** Refactor method that calls `query.Any()`, then `query.Count()`, then `query.ToList()`.
**Success criteria:** One materialization; same behavior.

### M2. N+1 to one query
**Goal:** Pseudocode: load order lines for 50 order IDs — from loop+First to single Contains query.
**Success criteria:** One DB round-trip sketch.

## Hard (stretch)

### H1. Streaming vs buffer
**Goal:** 10GB log file — pipeline filter+map+write. List which operators are forbidden and why.
**Success criteria:** No OrderBy/GroupBy/ToList on full file; single pass explained.
