# Exercises: SemaphoreSlim for Throttling

> Complete after passing `4.test-yourself.md`.

## Easy

### E1. Limit to 2
**Goal:** Observe throttling.

**Steps:** Create 10 tasks each `await Task.Delay(1000)`. Without semaphore — measure wall time (~1s). With `SemaphoreSlim(2)` — measure (~5s). Use `Stopwatch`.

**Success criteria:** Explain ratio.

### E2. Finally discipline
**Goal:** Prove leak.

**Steps:** Intentionally throw in `try` without `finally Release` once; observe hung second batch. Fix with `finally`.

---

## Medium

### M1. ThrottledHttpFetcher
**Goal:** Production-style helper.

**Steps:** Class with `SemaphoreSlim(6)`, method `GetAllAsync(IEnumerable<Uri> urls, CancellationToken ct)` using `WhenAll` + wait/release pattern.

**Success criteria:** Never exceeds 6 concurrent `HttpClient` calls.

### M2. Align with pool
**Goal:** Configuration.

**Steps:** Read connection string `Max Pool Size=50`. Justify semaphore **`N=40`** vs **`N=200`** in comment block.

---

## Hard (stretch)

### H1. Semaphore + Channel
**Goal:** Compose patterns.

**Steps:** Producers write to bounded channel; **3** consumer tasks each use shared semaphore before DB call. Sketch architecture.
