# Answers: Task.Factory.StartNew vs Task.Run

## Scoring

- 90–100%: отлично
- 80–89%: pass
- <80%: повтори лекции

---

## Answers

### 1. [Recall]
**Answer:** `Task<Task>` (outer wraps async lambda's returned Task).

**Why:** StartNew treats async lambda as `Func<Task>` body result.

---

### 2. [Recall]
**Answer:** **`Task.Run`**.

**Why:** Safer defaults, correct async lambda handling.

---

### 3. [Understanding]
**Answer:** Outer task completes when async **state machine starts** and returns inner Task reference — not when inner Delay finishes.

**Why:** Classic Task<Task> pitfall without Unwrap.

**Common mistake:** Assuming await outer awaits inner work.

---

### 4. [Understanding]
**Answer:** `Task.Run` always queues to **thread pool** (`TaskScheduler.Default`); `StartNew` overload may use **current scheduler** or ambiguous defaults.

**Why:** Stephen Toub guidance; predictable vs flexible.

---

### 5. [Understanding]
**Answer:** Blocks pool thread during sync HTTP — thread pool starvation under load; не освобождает thread during I/O.

**Why:** Task.Run doesn't make I/O async — only moves blocking to another thread.

---

### 6. [Application]
**Answer:** Prefer **`await Fetch()`** directly; if legacy wrapper needed: **`await Task.Run(async () => await Fetch())`** or StartNew with **`Unwrap()`** — first option usually unnecessary for pure I/O.

**Why:** I/O should be async all the way.

---

### 7. [Application]
**Answer:** **`await Task.Run(() => ParseCsv(bytes), HttpContext.RequestAborted)`** after **`await CopyToAsync`** — CPU parse only, not on upload I/O.

**Why:** Separate I/O async from CPU-bound parse.

---

### 8. [Analysis]
**Answer:** **B**.

**Why:** LongRunning = dedicated thread for blocking work without async alternative.

---

### 9. [Analysis]
**Answer:** Returns **`Task`** that completes when **inner** nested task completes — flattens one level.

**Why:** Fixes Task<Task> observation.

---

### 10. [Understanding]
**Answer:** Prevents child tasks attaching to parent — avoids legacy **AttachedToParent** completion coupling bugs.

**Why:** Safer StartNew options in migrations.
