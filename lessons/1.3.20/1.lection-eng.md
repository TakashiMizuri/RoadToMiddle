# yield return — The Iterator Pattern

> Roadmap: `1.3.20` · Node: `1.3` — C#: Advanced Topics · Depth: **deep**

## Learning Objectives

After this lesson you will be able to:

- Explain how **`yield return`** implements **`IEnumerable<T>`** / **`IEnumerator<T>`** without hand-written iterator classes.
- Describe **lazy evaluation**, **deferred execution**, and when iteration actually runs.
- Read the **compiler-generated state machine** conceptually (`MoveNext`, `_state`, captured locals).
- Combine **`yield return`** with **`yield break`**, **`try/finally`**, and **`using`** for resource cleanup.
- Avoid pitfalls: **multiple enumeration**, **modified collection during iteration**, and **`IEnumerable` as return type leaking mutation**.
- Contrast iterator blocks with **LINQ**, **`ToList()`**, and **`IAsyncEnumerable<T>`** for async streams.

---

## Why This Matters

Not every sequence should be materialized into a **`List<T>`** upfront. Log files with millions of lines, infinite sequences, paginated API results, and pipeline stages over large datasets need **pull-based** iteration: produce the next element only when the consumer asks. C#'s **`yield return`** syntax is syntactic sugar over the **iterator pattern** — you write what looks like a simple loop, and the compiler emits a state machine that pauses and resumes between elements.

LINQ itself is built on this idea: **`Select`**, **`Where`**, and your custom **`yield`** methods compose lazy pipelines. Misunderstanding laziness causes classic bugs — enumerating the same **`IEnumerable`** twice hits the database twice, or a **`yield`** method that opens a file never disposes it because nobody finished iterating. Middle developers implement custom serializers, CSV readers, tree traversals, and chunked EF reads; all of it sits on **`IEnumerator.MoveNext()`** whether you see it or not.

This lesson goes under the hood: state fields, **`Dispose`**, exception behavior, and when **`yield`** is the wrong tool.

## Core Concepts

### The Iterator Pattern in .NET

The pattern separates **sequence abstraction** from **iteration mechanics**:

- **`IEnumerable<T>`** — "you can get an enumerator."
- **`IEnumerator<T>`** — **`MoveNext()`**, **`Current`**, **`Reset`** (avoid), **`Dispose`**.

Hand-written iterators are verbose — a class holding index state, implementing both interfaces. **`yield return`** lets the compiler generate that class from a method body.

```csharp
public static IEnumerable<int> Range(int start, int count)
{
    for (int i = 0; i < count; i++)
        yield return start + i;
}
```

Calling **`Range(1, 3)`** does **not** run the loop yet. It returns an **`IEnumerable<int>`** wrapper. The loop runs when the consumer calls **`GetEnumerator()`** and then **`MoveNext()`** — for example via **`foreach`**.

```csharp
foreach (var n in Range(1, 3))
    Console.WriteLine(n); // 1, 2, 3 — one iteration step at a time
```

Each **`yield return`** **pauses** the method, returns a value to the consumer, and **resumes** on the next **`MoveNext()`**. Local variables and loop position survive between pauses because the compiler stores them in the generated iterator object's fields.

### Lazy vs Eager

**Lazy (deferred):** work happens per **`MoveNext()`**. **Eager:** work happens when you **`ToList()`**, **`Count()`**, **`First()`**, or otherwise force full evaluation.

```csharp
IEnumerable<string> ReadLinesLazy(string path)
{
    using var reader = new StreamReader(path);
    string? line;
    while ((line = reader.ReadLine()) != null)
        yield return line;
}
```

Until **`foreach`** runs, no file opens. First **`MoveNext()`** opens the file. If the consumer stops after two lines, the rest of the file is never read — efficient. If nobody enumerates at all, the method body never executes past setup — also means **`using`** cleanup runs only when enumeration completes or is disposed.

**`ToList()`** on the same sequence forces **all** lines into memory — opposite trade-off.

### yield break and Early Exit

**`yield break`** ends iteration — like **`return`** inside an iterator block (you cannot **`return`** a value from **`IEnumerable`** iterator method except via **`yield return`**).

```csharp
public static IEnumerable<int> TakeWhilePositive(IEnumerable<int> source)
{
    foreach (var x in source)
    {
        if (x <= 0)
            yield break;
        yield return x;
    }
}
```

When consumer stops **`foreach`** early, the iterator's **`Dispose()`** runs **`finally`** blocks in the **`yield`** method — important for releasing files, connections, or rented arrays.

### Restrictions on Iterator Methods

An iterator method must return **`IEnumerable`**, **`IEnumerable<T>`**, **`IAsyncEnumerable<T>`**, **`IEnumerator`**, or **`IEnumerator<T>`**. It cannot have **`ref`**/`out` parameters (except **`async`** iterators have separate rules). **`unsafe`** contexts and some **`lock`** patterns are restricted. You cannot **`yield`** inside an anonymous function unless it is a **local function** in some cases — **`yield` in lambda** is not allowed directly.

Iterator methods cannot use **`await`** unless they return **`IAsyncEnumerable<T>`** (**`await foreach`** consumer) — sync **`yield return`** and **`await`** in the same block follow different compiler paths.

## Under the Hood

Roslyn transforms a **`yield return`** method into a **nested struct or class** implementing **`IEnumerator<T>`** (and often **`IEnumerable<T>`** on the same object for single-method iterators).

Conceptual shape:

```csharp
// Conceptual — not exact emitted code
class RangeIterator : IEnumerator<int>
{
    private int _state;      // which yield point
    private int _current;
    private int start, count, i; // lifted locals

    public bool MoveNext()
    {
        switch (_state)
        {
            case 0: i = 0; goto case 1;
            case 1:
                if (i >= count) return false;
                _current = start + i++;
                _state = 1;
                return true;
            default: return false;
        }
    }
    public int Current => _current;
    public void Dispose() { /* finally from yield method */ }
}
```

**`_state`** records which **`yield return`** to resume from. **`MoveNext()`** is a **`switch`** on state — the method is **not** kept on the real call stack between elements (unlike coroutines in some languages that stack frames). Deep recursion in **`yield`** still uses the thread stack for the initial call until the first yield; after that, state is in the iterator object.

**Multiple enumeration** creates **new iterator instances** — each **`foreach`** over **`Range(1,3)`** gets fresh state. If the **`yield`** method reads a **shared mutable field** or a **single-use stream**, second enumeration may fail or return wrong data.

**`finally`** in iterator methods runs when:
- iteration completes normally;
- consumer calls **`Dispose()`** on enumerator ( **`foreach`** does this);
- exception propagates out of **`MoveNext()`**.

If consumer never starts enumeration, **`finally`** may not run — another reason side effects in iterator setup before first **`yield`** are dangerous.

### IEnumerable as Return Type: Leaky Abstraction

Returning **`IEnumerable<T>`** from a public API exposes **lazy** semantics. Callers may enumerate zero, one, or many times. If your method opens a DB connection and **`yield`**s rows, document **single enumeration** or return **`IReadOnlyList<T>`** / **`List<T>`** instead.

```csharp
// Dangerous if called twice — two queries or disposed context
public IEnumerable<OrderDto> GetOrders(AppDbContext db) =>
    db.Orders.Select(o => new OrderDto(o.Id, o.Total)); // IQueryable deferred — different beast
```

For **in-memory yield**, side effects run per enumeration. Materialize when the cost is high or semantics must be snapshot.

## Syntax / API

```csharp
// Basic
IEnumerable<T> Method() { yield return item; }

// Early stop
yield break;

// Local function iterator
IEnumerable<int> Outer()
{
    IEnumerable<int> Inner() { yield return 1; yield return 2; }
    return Inner();
}

// Async stream (C# 8+)
async IAsyncEnumerable<int> StreamAsync()
{
    await Task.Delay(10);
    yield return 42;
}
```

**`foreach`** expands to enumerator **`Dispose`**. **`using var e = collection.GetEnumerator()`** — manual pattern.

## Examples

### Tree traversal (DFS)

Lazy traversal avoids allocating a full node list:

```csharp
public static IEnumerable<TreeNode> Descendants(TreeNode root)
{
    foreach (var child in root.Children)
    {
        yield return child;
        foreach (var d in Descendants(child))
            yield return d;
    }
}
```

Each **`foreach`** over **`Descendants(root)`** walks the tree again. For repeated queries, cache **`ToList()`** once.

### Chunked processing

```csharp
public static IEnumerable<IReadOnlyList<T>> Chunk<T>(IEnumerable<T> source, int size)
{
    if (size <= 0) throw new ArgumentOutOfRangeException(nameof(size));

    List<T>? bucket = null;
    foreach (var item in source)
    {
        bucket ??= new List<T>(size);
        bucket.Add(item);
        if (bucket.Count == size)
        {
            yield return bucket;
            bucket = null;
        }
    }
    if (bucket is { Count: > 0 })
        yield return bucket;
}
```

Consumer processes batches without loading entire source into memory — if source itself is lazy (file lines), memory stays bounded by chunk size.

### Custom LINQ-like operator

```csharp
public static IEnumerable<T> Where<T>(this IEnumerable<T> source, Func<T, bool> predicate)
{
    foreach (var item in source)
        if (predicate(item))
            yield return item;
}
```

This mirrors **`Enumerable.Where`** — deferred until consumer enumerates. **`predicate`** runs at iteration time, not at call time.

## Common Mistakes & Anti-patterns

**Assuming the iterator method body ran because you called it.** Calling **`GetLines(path)`** only builds the iterator object; **`StreamReader`** may not open until **`MoveNext()`**.

**Multiple enumeration of expensive sequences.** Database **`IQueryable`**, file **`yield`**, or network pagination — enumerate once or materialize explicitly.

**Mutating collection during `foreach`.** Undefined behavior for most collections; **`InvalidOperationException`** on **`List<T>`**.

**`yield return` inside `try` with `catch` that yields.** C# disallows **`yield return`** in **`catch`** blocks; **`finally`** can run on dispose — know language restrictions.

**Returning `IEnumerable` from public API without documenting single-use.** Callers will **`ToList()`** or enumerate twice.

**Using `yield` for trivial one-shot lists.** **`return new[] { 1, 2, 3 }`** or **`[1, 2, 3]`** collection expression is clearer when there is no laziness benefit.

## Production & Real-World Notes

**ASP.NET Core** streaming responses sometimes use **`IAsyncEnumerable<T>`** with **`yield return`** per record — reduces memory vs loading all rows. EF Core **`AsAsyncEnumerable()`** streams entities; still mind **`DbContext`** lifetime — context must live until enumeration completes.

**Logging and metrics:** long **`yield`** pipelines should propagate **`CancellationToken`** in async variants; sync iterators have no built-in cancellation — check token in loop body if passed manually.

Performance: iterator state machines add **allocations** (enumerator object) vs **`for`** over **`Span<T>`**. Hot paths use **`for`**, **`ReadOnlySpan`**, or **`ArrayPool`** — not **`yield`**.

**Unit tests:** assert on **`ToList()`** or materialized arrays when testing **`yield`** methods; do not depend on side effect order unless you enumerate explicitly.

## Comparison / Trade-offs

| Approach | Memory | When runs | Best for |
|---|---|---|---|
| **`yield return`** | O(1) extra if consumer stops early | On **`MoveNext`** | Streams, pipelines, large inputs |
| **`ToList()`** | O(n) | Immediately | Multiple passes, indexing |
| **`IAsyncEnumerable`** | Streaming async | **`MoveNextAsync`** | DB/API async streams |
| **`for` over array** | Minimal | Immediate | Hot loops, known size |

**LINQ** operators return deferred **`IEnumerable`**; **`yield`** lets you write custom operators without **`Expression`** trees (in-memory only).

## Quick Reference

| Construct | Effect |
|---|---|
| **`yield return x`** | Pause; emit **`x`**; resume on next **`MoveNext`** |
| **`yield break`** | End sequence |
| **`foreach`** | **`GetEnumerator` + `Dispose`** |
| **`ToList()`** | Force full evaluation |
| Second **`foreach`** | New enumerator (usually) |

## Key Takeaways

- **`yield return`** builds a **state machine** implementing **`IEnumerator<T>`**.
- Execution is **lazy** until enumeration forces **`MoveNext()`**.
- **`Dispose`**/**`finally`** tie resource lifetime to enumeration.
- **Multiple enumeration** may repeat work or break single-use sources.
- Use **`IAsyncEnumerable`** for async streaming; **`List`** when you need snapshot or random access.
- **`yield`** composes with LINQ into lazy pipelines — understand both layers.
