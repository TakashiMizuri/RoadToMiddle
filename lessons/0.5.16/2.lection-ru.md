# HTML Validation — W3C Validator

> Roadmap: `0.5.16` · Node: `0.5` — HTML · Depth: практика

## Learning Objectives

После этого урока ты сможешь:

- Объяснить, **зачем нужна HTML validation** для accessibility, SEO и cross-browser consistency.
- Проверять документы в **W3C Nu Html Checker** (validator.w3.org) по URL, file upload и direct input.
- Читать сообщения **error vs warning** и находить строку в source.
- Исправлять типичные ошибки: unclosed tags, invalid nesting, missing `<title>`, duplicate IDs, obsolete attributes.
- Встроить validation в **local workflow** перед публикацией markup.

---

## Why This Matters

Browsers **прощают** почти всё — **HTML parsing algorithm** построит DOM даже из сильно broken markup. Визуально «работает», но DOM может не совпадать с ожиданиями CSS и `querySelector`. Assistive tech, crawlers и email clients по-разному recover from errors; valid markup снижает **surprise**.

Validation — быстрый feedback loop при изучении HTML (`0.5.x`) и при review **output** Razor, React SSR, CMS. Ловит `<ul>` без `<li>`, nested `<button>`, duplicate `id`, invalid `data-productId` (`0.5.15`). Произвольный `sku="..."` fail validation — подтверждение роли `data-*`.

Это **практический capstone** узла **0.5 — HTML**. Дальше — **CSS cascade** (`0.6.1`). Valid semantic HTML даёт CSS и React predictable structure. Validator — **lint для markup**, рядом с ESLint и axe.

## Core Concepts

### Что проверяет validation

**W3C Nu Html Checker** парсит документ по правилам **HTML living standard**:

- **Errors** — нарушение conformance (invalid child, forbidden attribute).
- **Warnings** — сомнительно (obsolete, redundant ARIA).
- **Info** — мелочи.

Проверяются **syntax и content models** — кто в кого может вкладываться, required attributes, unique `id`. **Не** полная a11y (axe/WAVE), не performance, не качество текста. Validate ≠ good UX.

### Интерфейс validator

**[https://validator.w3.org/](https://validator.w3.org/)**

Режимы:

1. **Validate by URI** — public URL.
2. **Validate by File Upload** — `.html` с диска; лучше для local WIP.
3. **Validate by Direct Input** — paste snippet.

Рекомендуемые опции:

- **HTML5**, **UTF-8**
- **Show source** — numbered lines
- **Show outline** — структура headings

Результат: **line:column**, текст, excerpt source.

### Как читать results

```
Error: Element div not allowed as child of element ul in this context.
From line 24, column 5
```

Workflow: открыть line 24 → fix structure → **re-validate** до zero errors.

Сообщения про **phrasing/flow content** — см. MDN permitted contents.

**False positives** редки (bleeding-edge features). Не игнорируй real errors.

### Частые категории ошибок

**Unclosed / mis-nested tags** — parser recovery ≠ intent (`0.5.10`).

**Missing** `<!DOCTYPE html>`, `lang`, `<title>`, charset.

**Duplicate `id`**.

**Invalid attribute names** — нужен `data-` prefix (`0.5.15`).

**Interactive nesting** — `<a>` in `<a>`, `<button>` in `<button>`.

**Obsolete** — `align`, `border` (→ CSS в `0.6`).

**Unescaped `&`** — `AT&amp;T`.

### Valid skeleton

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Validation Practice Page</title>
</head>
<body>
  <main>
    <h1>Hello</h1>
    <p>Valid minimal body.</p>
  </main>
</body>
</html>
```

Upload — **no errors**. Добавь bug — validate — fix — muscle memory.

### Local и CI

- **Nu Html Checker CLI** (`vnu.jar`, `html5validator`).
- Editor extensions.

Razor: validate **rendered HTML** (View Source), не raw `.cshtml`. React: SSR snapshot; client-only `<div id="root">` trivially valid — не representative.

## Under the Hood

Checker использует **parser rules** WHATWG + **constraints** content model. Optional tags (`<tbody>`) parser may insert — explicit markup яснее.

`<!DOCTYPE html>` → **no-quirks mode** — важно для CSS (`0.6`).

File/direct input: markup уходит на W3C service — для sensitive HTML используй **local CLI**.

## Syntax / Commands / API

### W3C Web UI

1. `https://validator.w3.org/`
2. **File Upload** или **Direct Input**
3. **Check**
4. Fix by line numbers
5. Repeat

### Broken snippet для теста

```html
<ul><div>List item?</div></ul>
```

→ fix `<ul><li>...</li></ul>`.

### CLI (concept)

```bash
html5validator --format text practice.html
```

Non-zero exit при errors — CI gate.

## Common Mistakes & Anti-patterns

«Browser OK — skip validation». Validate raw `.tsx`/`.cshtml`. Ignore all warnings forever. Chase zero on vendor embed only with documentation.

## Production & Real-World Notes

CI для static pages и email templates. SSR fixtures в component libraries. Boilerplate layout в Razor/React — меньше repeated head errors.

## Comparison / Trade-offs

Validator vs DevTools DOM (source vs parsed). Errors vs warnings triage. URI vs file. W3C vs ESLint html plugin — complementary.

## Key Takeaways

- Valid HTML → predictable DOM для CSS, JS, SEO, a11y.
- **validator.w3.org** — URI / file / paste.
- Fix errors по line numbers; re-validate.
- Validate **rendered** template output.
- Закрываем HTML `0.5` → **`0.6.1` CSS Cascade**.
