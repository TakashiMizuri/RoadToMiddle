# stackalloc — краткая выжимка

> `1.5.13` · ~7 мин

## Главная идея

**`stackalloc`** выделяет buffer на **stack** текущего метода как **`Span<T>`** — без GC, но **короткий lifetime** и **жёсткий лимит размера**.

## Ключевые пункты

- Память invalid после return — **не возвращать Span**.
- Типично **≤256–512 bytes** (зависит от контекста).
- **User input size** — опасно (overflow/DoS).
- **CA2014**: не stackalloc в loop.
- Больше buffer → **`ArrayPool`** (`1.5.14`).

## Пример

```csharp
Span<char> tmp = stackalloc char[64];
```

## Частые ошибки

- Огромный stackalloc.
- stackalloc в цикле.
- Return Span наружу.

## Мнемоника

> **Stack is for seconds and small bytes.**
