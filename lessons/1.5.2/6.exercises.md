# Exercises: JIT Tier 0/1/2

## Easy

### E1. Observe tiering (conceptual)
**Goal:** Connect warmup to measurement.
**Steps:** Write pseudo-benchmark plan: 1 cold call, 100_000 warm calls, what to measure each phase.
**Success criteria:** Mention JIT + Tier 0 vs Tier 2 explicitly.

### E2. Environment variables
**Goal:** Know tuning knobs exist.
**Steps:** List `DOTNET_TieredCompilation`, `DOTNET_ReadyToRun`, `PublishReadyToRun` — one sentence each from docs.
**Success criteria:** Do not recommend disabling tiering in prod without reason.

## Medium

### M1. ASP.NET cold start checklist
**Goal:** Production warm-up design.
**Steps:** Design readiness probe flow: 3 endpoints to hit before marking pod ready; explain JIT benefit.
**Success criteria:** Health + one business endpoint; mention Tier 0→2.

### M2. R2R vs Native AOT decision
**Goal:** Trade-off evaluation.
**Steps:** Scenario: edge function cold start vs long-running API — pick R2R, AOT, or default JIT with 3 bullet justification each scenario.
**Success criteria:** Startup vs reflection/dynamic mentioned.

## Hard

### H1. dotnet-trace JIT events
**Goal:** Diagnostic literacy.
**Steps:** Read docs for `Microsoft-Windows-DotNETRuntime` Jit keywords; outline how you'd capture 30s trace during app start.
**Success criteria:** Tool name + provider keyword mentioned.
