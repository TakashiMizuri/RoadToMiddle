# Static members — краткая выжимка

> `1.2.12` · ~7 мин на повторение

## Главная идея

**`static`** привязывает поля и методы к **типу**, а не к экземпляру: одна общая копия на загруженный тип в app domain. Уместен для stateless utilities; **mutable static** в ASP.NET — почти всегда баг под concurrency.

## Ключевые пункты

- **Instance member** — per object; **static member** — per type, вызов без `new`.
- Static methods **без `this`**, не участвуют в virtual polymorphism как instance methods.
- **`static class`** — `abstract sealed`, только static members, нельзя наследовать/инстанцировать.
- **`static constructor`** (`.cctor`) — один раз, thread-safe, до первого использования типа; нельзя вызвать явно.
- **`static readonly` / `const`** — предпочтительны для «констант»; `const` — compile-time.
- **`TypeCache<T>`** — отдельный static storage для каждого closed generic (`int` vs `string`).
- **Singleton в DI** — один **instance**, но mockable; **static class** — нет.
- **Anti-pattern:** `public static int CurrentUserId` в web — race между запросами.

## Самые важные примеры

```csharp
public static class Guard
{
    public static void AgainstNull(object? v, string name) { ... }
}

static DatabaseBootstrap()  // runs once, before first access
{
    ConnectionString = Environment.GetEnvironmentVariable("DB_CONN")!;
}
```

## Частые ошибки

- Mutable static как глобальное состояние в multi-threaded server.
- Static service locator вместо constructor injection.
- Ожидание детерминированного порядка static ctor между несвязанными типами.

## Мнемоника

> **Static — один на весь тип; mutable static в API — красный флаг; side effects — в DI instance.**
