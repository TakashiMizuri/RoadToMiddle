# Parallel.ForEachAsync — Bounded Parallel I/O

> Roadmap: `1.4.25` · Node: `1.4` — Async + ASP.NET preview · Depth: практика

## Learning Objectives

After this lesson you will be able to:

- Use **`Parallel.ForEachAsync`** (.NET 6+) to process collections with **bounded concurrency** and **`CancellationToken`**.
- Configure **`ParallelOptions.MaxDegreeOfParallelism`** to protect downstream services and thread pool.
- Compare **`Parallel.ForEachAsync`** vs **`Task.WhenAll`** vs sequential **`foreach`** for I/O workloads.
- Implement a batch HTTP fan-out pattern safe for **ASP.NET Core** (shared `HttpClient`, limits, cancellation).
- Handle per-item errors without failing the entire batch when appropriate.
- Avoid unbounded **`Task.WhenAll`** that overwhelms sockets or remote API rate limits.

---

## Why This Matters

A common requirement: call an external API for **each of 500 IDs** in a request or background job. The naive approach **`Task.WhenAll(ids.Select(id => client.GetAsync(...)))`** launches **500 simultaneous connections** — socket exhaustion (`1.4.26`), **429 Too Many Requests**, memory pressure, and tail latency for other users. Sequential **`foreach` + await** is safe but slow.

**`Parallel.ForEachAsync`** provides **structured concurrency** with a **ceiling** on parallel operations — the middle ground production needs. It integrates **`async` lambdas**, honors **`CancellationToken`**, and reports progress via **`ParallelLoopResult`**. In ASP.NET Core, a controller that fan-outs to microservices must bound parallelism and forward **`HttpContext.RequestAborted`** so client disconnect cancels outstanding calls.

This lesson is the practical complement to **`1.4.22`** (why async scales) and **`1.4.28`** (Polly retry under bounded load).

## Core Concepts

### API Shape

```csharp
await Parallel.ForEachAsync(
    source,
    new ParallelOptions
    {
        MaxDegreeOfParallelism = 8,
        CancellationToken = cancellationToken
    },
    async (item, ct) =>
    {
        await ProcessItemAsync(item, ct);
    });
```

**`Parallel.ForEachAsync`** is **async-aware** — unlike **`Parallel.ForEach`**, which is for **CPU-bound** synchronous bodies and uses thread pool workers for the duration of each body (wrong tool for I/O).

**`MaxDegreeOfParallelism`**: maximum concurrent invocations of the delegate. Default is **`Environment.ProcessorCount`** — often **too low for I/O** (waiting) or **too high** if mistaken for CPU work. For HTTP fan-out, choose based on **downstream capacity** (e.g. 4–16), not CPU count alone.

**`CancellationToken`** in **`ParallelOptions`** stops scheduling new iterations; in-flight work should observe **`ct`** passed to the delegate.

### vs Task.WhenAll

| Pattern | Concurrency | Best for |
|---------|-------------|----------|
| Sequential foreach | 1 | Small N, strict ordering |
| Task.WhenAll(all tasks) | Unbounded (N) | Small N, trusted limit |
| Parallel.ForEachAsync | Bounded | Large N, I/O batch |
| SemaphoreSlim + foreach | Bounded | Manual control, same idea |

**Task.WhenAll** with **`Select`** creates **all tasks immediately** — even if only 8 run at once internally, each **`HttpClient` request** may allocate state; **Parallel.ForEachAsync** only maintains up to **`MaxDegreeOfParallelism`** active delegates at scheduling level (still create tasks per iteration internally — understand it's **bounded scheduling**, not magic throttling of task creation for all patterns).

For very large N, consider **batching** (chunks of 50) + ForEachAsync per chunk to limit memory.

### Error Handling

Default: **first exception** may fault the loop; **`ParallelLoopResult.IsCompleted`** false. **`OperationCanceledException`** on cancel.

Per-item resilience:

```csharp
await Parallel.ForEachAsync(items, options, async (item, ct) =>
{
    try
    {
        await CallApi(item, ct);
    }
    catch (HttpRequestException ex) when (ex.StatusCode == HttpStatusCode.NotFound)
    {
        // log and continue
    }
});
```

For aggregate reporting, collect failures in **`ConcurrentBag`** (thread-safe) and return summary DTO.

### ASP.NET Core Integration

```csharp
public async Task<IActionResult> SyncMany(
    [FromBody] int[] ids,
    CancellationToken cancellationToken)
{
    var client = _httpClientFactory.CreateClient("downstream");
    var results = new ConcurrentBag<ItemDto>();

    await Parallel.ForEachAsync(ids, new ParallelOptions
    {
        MaxDegreeOfParallelism = 10,
        CancellationToken = cancellationToken
    }, async (id, ct) =>
    {
        var dto = await client.GetFromJsonAsync<ItemDto>($"/items/{id}", ct);
        if (dto is not null) results.Add(dto);
    });

    return Ok(results.ToList());
}
```

Use **`IHttpClientFactory`** (`1.4.27`) — not **`new HttpClient()`** per item. Forward **`CancellationToken`** from minimal APIs/controllers — links to **`RequestAborted`**.

## Under the Hood

**Parallel.ForEachAsync** uses **`async` state machines** per iteration scheduled through **`Task`** coordination — internal implementation queues work while respecting **`MaxDegreeOfParallelism`** via semaphoric scheduling (conceptually similar to **`SemaphoreSlim`**).

It does **not** use **`Parallel.ForEach`** thread pinning — I/O waits release threads.

**ProcessorCount default** reflects CPU parallel heritage — override explicitly for I/O.

## Syntax / Commands / API

```csharp
// Read-only ParallelOptions
public class ParallelOptions
{
    public TaskScheduler? TaskScheduler { get; set; }
    public int MaxDegreeOfParallelism { get; set; } = -1; // -1 => ProcessorCount
    public CancellationToken CancellationToken { get; set; }
}
```

Overload with **`IAsyncEnumerable<T>`** (.NET 8+) for streaming sources.

Return value: **`ParallelLoopResult`** — **`IsCompleted`**, **`LowestBreakIteration`** (break API rare in async).

## Examples

### Example 1: Download many URLs with limit

```csharp
var urls = File.ReadAllLines("urls.txt");
await Parallel.ForEachAsync(urls, new ParallelOptions { MaxDegreeOfParallelism = 6 },
    async (url, ct) =>
    {
        var html = await http.GetStringAsync(url, ct);
        await File.WriteAllTextAsync($"out/{Hash(url)}.html", html, ct);
    });
```

### Example 2: Chunked WhenAll alternative

```csharp
foreach (var chunk in ids.Chunk(20))
{
    await Task.WhenAll(chunk.Select(id => FetchOne(id, ct)));
}
```

Chunking + WhenAll = manual bounding; ForEachAsync cleaner for uniform limit.

### Example 3: Wrong tool — Parallel.ForEach for I/O

```csharp
// DON'T — blocks thread per wait
Parallel.ForEach(urls, url => { var s = http.GetStringAsync(url).Result; });
```

Use **ForEachAsync** or **await** in loop.

## Common Mistakes & Anti-patterns

**Unbounded Task.WhenAll** on large collections.

**Parallel.ForEach** (sync) with **`.Result`** on async HTTP.

**MaxDegreeOfParallelism = ProcessorCount** for I/O without thinking — may under-utilize or still overload network.

**Ignoring cancellation** — orphaned calls after client disconnect.

**Shared mutable non-thread-safe collections** without **`ConcurrentBag`** / locking.

**New HttpClient per iteration** — socket exhaustion.

## Production & Real-World Notes

Tune **`MaxDegreeOfParallelism`** with metrics: downstream **p95**, **429 rate**, **socket errors**. Coordinate with **Polly** circuit breaker (`1.4.28`) — parallelism + retry storms multiply load.

For **rate-limited APIs**, combine bounded parallelism with **token bucket** or **`RateLimiter`** (.NET 7+).

Background jobs (Hangfire, **`BackgroundService`**) use same pattern with **`IHostApplicationLifetime.ApplicationStopping`**.

## Comparison / Trade-offs

| Approach | Pros | Cons |
|----------|------|------|
| ForEachAsync | Built-in bound, cancel | Less flexible than custom semaphore |
| SemaphoreSlim | Fine-grained control | More boilerplate |
| Task.WhenAll | Simple for small N | Unbounded at scale |
| Sequential | Safest, ordered | Slow |

## Quick Reference

```csharp
await Parallel.ForEachAsync(source, new ParallelOptions
{
    MaxDegreeOfParallelism = 8,
    CancellationToken = ct
}, async (item, ct) => { await Work(item, ct); });
```

- I/O → **ForEachAsync**, not **ForEach**.
- Factory **HttpClient**, one client many requests.
- Forward **CancellationToken** from ASP.NET.
- Catch per-item or fail fast — explicit choice.

## Key Takeaways

- **Parallel.ForEachAsync** bounds concurrent async work — essential for large I/O batches.
- Default parallelism = **ProcessorCount** — override for HTTP fan-out.
- **Not** the same as **Parallel.ForEach** (CPU sync).
- Prefer over unbounded **Task.WhenAll** for large N.
- Use **ConcurrentBag** or immutable aggregation for thread-safe results.
- ASP.NET: **IHttpClientFactory** + **RequestAborted** token.
- Tune limits with downstream health and Polly policies.
