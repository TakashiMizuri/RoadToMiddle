## Why This Matters

`0.4.9`: React на CDN `app.example.com`, API `api.example.com`. Locally Vite proxy — works. After deploy: server может log **200**, browser **blocks JS** from reading response — CORS error в DevTools.

**CORS** — controlled relaxation **same-origin policy**. Server говорит browser, какие cross-origin reads OK. **Не** блокирует curl/Postman/server-to-server.

**Preview** на network layer — enough для debug deploy. **Full setup:** `AddCors`, credentials, pipeline order — later ASP.NET/React nodes.

## Core Concepts

### Origin

**(scheme, host, port)** — всё три должны match для same origin. Path/query не влияют.

Different origins: `app` vs `api` host; `:5173` vs `:5000`; `http` vs `https`.

**Same-origin policy:** script с origin A не читает response B без CORS headers от B.

### Browser vs Server

Request **доходит** до Kestrel; JSON строится. Browser смотрит response headers — без `Access-Control-Allow-Origin` JS не видит body. Network tab: 200 + red CORS.

Postman **ignore** CORS.

### Simple vs Preflight

**Simple:** GET/HEAD/POST с limited content-types, safelisted headers — **без** preflight.

**Non-simple:** PUT/DELETE, `Authorization`, `Content-Type: application/json` → **OPTIONS** first.

Typical JWT JSON API → preflight almost always.

### Key Headers

- **`Access-Control-Allow-Origin`** — кто может read (`*` или specific origin).
- **`Allow-Methods`**, **`Allow-Headers`** — для preflight.
- **`Allow-Credentials: true`** — cookies; **не** с `*` origin.
- **`Max-Age`** — cache preflight.

Fix на **API** (or gateway), не React alone (except dev proxy).

## Under the Hood

CORS + SOP limit what attacker JS reads cross-origin. Credentialed fetch требует specific origin + Allow-Credentials.

## Syntax / Commands / API

```javascript
fetch('https://api.example.com/orders', {
  headers: { Authorization: `Bearer ${token}` },
});
```

```csharp
// Preview — full lesson later
builder.Services.AddCors(/* WithOrigins("https://app.example.com") */);
app.UseCors("Spa");
```

```bash
curl -H "Origin: https://app.example.com" -I https://api.example.com/health
```

- `AllowAnyOrigin` + credentials.
- CORS как замена auth.
- OPTIONS не доходит до pipeline.
