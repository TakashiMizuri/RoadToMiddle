## Answers

### 1. [Recall]
**Answer:** `<!DOCTYPE html>`
**Why:** Единственное каноническое объявление HTML5; case-insensitive.

### 2. [Recall]
**Answer:** Атрибут `lang`; значения по BCP 47 (например `ru`, `en-US`).
**Why:** Стандартизированные языковые теги понимают браузеры, SR и SEO.

### 3. [Recall]
**Answer:** В `<body>`.
**Why:** `<head>` — только метаданные; видимый контент не в head.

### 4. [Understanding]
**Answer:** Quirks mode эмулирует старые баги layout (другая box model и т.д.); CSS из standards-ориентированных материалов ведёт себя иначе.
**Why:** DOCTYPE переключает режим парсера до построения DOM.

### 5. [Understanding]
**Answer:** Браузер иначе угадывает кодировку для первых байт; возможен неверный декод и перепарсинг.
**Why:** Charset meta должна быть в первых ~1024 байтах.

### 6. [Understanding]
**Answer:** Screen reader неверно произносит русский; SEO/перевод получают ложный сигнал языка.
**Why:** `lang` на `<html>` — primary language document.

### 7. [Application]
**Answer:** Полный каркас: `<!DOCTYPE html>`, `<html lang="ru">`, `<head>` с `<meta charset="UTF-8">` и `<title>О нас</title>`, `<body>` с `<p>...</p>`.
**Why:** Все обязательные части подтемы на месте.

### 8. [Application]
**Answer:** Обернуть цитату в элемент с `lang="ru"` (например `<blockquote lang="ru">` или `<span lang="ru">`).
**Why:** Локальный язык фрагмента без смены корня.

### 9. [Understanding]
**Answer:** HTTP charset часто имеет приоритет или создаёт конфликт; декодирование может идти не как UTF-8.
**Why:** Нужно согласование file + meta + HTTP.

### 10. [Analysis]
**Answer:** A
**Why:** DOCTYPE не создаёт узел в DOM.

### 11. [Analysis]
**Answer:** Файл на диске не UTF-8 (другая кодировка при сохранении); BOM/редактор; реже — повреждённая копия. Также: meta отсутствует/не в head при `file://` без правильной локальной кодировки OS.
**Common mistake:** Винить только браузер, не проверяя bytes файла.

### 12. [Understanding]
**Answer:** `head` — данные о документе (title, charset, links); `body` — контент для пользователя.
**Why:** Семантическое разделение HTML.

### 13. [Application]
**Answer:** Кодировка файла на диске; `<meta charset="UTF-8">`; HTTP `Content-Type` с `charset=utf-8`.
**Why:** Три слоя декодирования должны совпадать.

### 14. [Understanding]
**Answer:** Парсер прощает ошибки и автоисправляет; quirks/encoding могут проявиться только в другом браузере, с другими данными или в Lighthouse.
**Why:** Forgiving parser скрывает проблемы.
