# Upstream tracking, push -u, pull --rebase — краткая выжимка

> `0.2.7` · ~7 мин на повторение

## Главная идея

**Upstream** — default remote branch для локальной ветки (хранится в `.git/config`). **`git push -u`** публикует ветку и настраивает tracking; **`git pull --rebase`** интегрирует remote work без лишних merge commits.

## Ключевые пункты

- **`git branch -vv`** — tracking + ahead/behind.
- **Ahead** → нужен push; **behind** → pull / `pull --rebase`.
- **`-u`** при first push — стандарт; без него `git push` без args не работает.
- **`git pull --rebase`** на feature — linear history (`0.2.3`, `0.2.4`).
- Перед PR: **`git fetch && git rebase origin/main`**.
- После rebase уже запушенной ветки: **`git push --force-with-lease`**.

## Самые важные примеры

```bash
git push -u origin feature/health
git pull --rebase
git fetch origin && git rebase origin/main
git push --force-with-lease
```

## «Под капотом» в одном абзаце

Upstream в `[branch "name"]` → `remote` + `merge`. `pull --rebase` = fetch + rebase на `@{upstream}`; replay создаёт новые SHA. Ahead/behind — чистое сравнение графа commits после fetch.

## Частые ошибки

- Путать upstream текущей ветки с `main`.
- `push --force` вместо `--force-with-lease` на feature.
- Rebase ветки, которую уже стянули другие, без координации.

## Мнемоника

> **-u один раз — push/pull без имени ветки всегда.**
