## Answers

### 1. [Recall]
**Answer:** Manual tools дают **полный контроль** над method, headers, body и позволяют **быстро изолировать** HTTP-слой при debug, воспроизвести edge cases (auth chains, wrong content type), которых нет в Swagger/automated happy path.
**Why:** Exploration speed + reproducible repro steps.

### 2. [Recall]
**Answer:** **`-i`** включает **response headers** в output вместе с body.
**Why:** Status line и headers (Set-Cookie, Location) часто ключ к проблеме.

### 3. [Recall]
**Answer:** Requests хранятся как **файлы в репозитории** — Git diff, PR review, версионирование рядом с кодом, без обязательного vendor cloud.
**Why:** Bruno — file-first модель.

### 4. [Understanding]
**Answer:** Verbose показывает **TLS handshake**, certificate chain, SNI, точную ошибку verify — до разбора application body.
**Why:** Many failures are SSL/TLS, not JSON.

### 5. [Understanding]
**Answer:** Один request template работает на **dev/staging/prod** — меняется только environment (`localhost` vs staging URL).
**Why:** DRY для manual и shared collections.

### 6. [Application]
**Answer:** (1) Copy exact URL/headers/body из DevTools (minus browser-only). (2) Obtain fresh token via login request in tool. (3) Compare manual 401 vs 200 — if manual works, bug in client storage/header; if not, server/auth config.
**Why:** Bisect client vs server.

### 7. [Application]
**Answer:** `curl -sS -H "Accept: application/json" -w "\nHTTP %{http_code}\n" https://api.example.com/health`
**Why:** `-w` writes custom trailer with status.

### 8. [Application]
**Answer:** **`-F`** (e.g. `-F "title=Report" -F "file=@report.pdf"`)
**Why:** `-F` sets multipart/form-data with boundary.

### 9. [Analysis]
**Answer:** PowerShell **`curl`** — alias для **`Invoke-WebRequest`**, не libcurl. Use **`curl.exe`**.
**Common mistake:** Different syntax and behavior.

### 10. [Understanding]
**Answer:** Token в Git history = **credential leak**; rotation, compliance violation, anyone with repo access impersonates user.
**Why:** Use `{{token}}` placeholders and local secrets.

### 11. [Analysis]
**Answer:** Rate limit — подождать **60 секунд** (`Retry-After`) перед повтором; не спамить manual retries on shared staging.
**Why:** Respect server policy.

### 12. [Application]
**Answer:** **Tests tab** (or post-response script): parse JSON, `pm.environment.set("accessToken", json.access_token)`; следующие requests — Authorization Bearer `{{accessToken}}`.
**Why:** Chained auth flow automation.

### 13. [Recall]
**Answer:** Echo service — возвращает отправленные headers/body/json для **practice без своего API**.
**Why:** Safe sandbox.

### 14. [Understanding]
**Answer:** **`-d`** sends body (usually POST); **`-G`** appends `-d` data as **query string** on GET.
**Why:** GET must not have body in typical REST style.

### 15. [Analysis]
**Answer:** Reviewers **видят API contract change** в том же PR, что backend routing — меньше drift «docs vs reality».
**Why:** Git-native collections as living docs.
