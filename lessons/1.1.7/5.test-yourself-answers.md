## Answers

### 1. [Recall]
**Answer:** CLR **allocate** object на heap, **копирует** `int` 42 в payload; `o` хранит **reference** на box.

### 2. [Recall]
**Answer:** IL instruction **`box`** (с metadata token типа, например `System.Int32`).

### 3. [Recall]
**Answer:** `List<int>` — generic storage **`int[]`** inline, без conversion к `object`. `ArrayList` хранит **`object[]`** — каждый `int` **implicitly boxed** при `Add`.

### 4. [Understanding]
**Answer:** `int a = b` копирует bits между stack slots — **без heap object**. Boxing **создаёт heap object** и копирует value **внутрь** reference type payload.

### 5. [Understanding]
**Answer:** **`InvalidCastException`**. Unbox требует **exact** `(int)`, не widening cast на unbox step.

### 6. [Understanding]
**Answer:** **Boxing не происходит** для значения; **`o == null`**. Nullable без value → null reference.

### 7. [Understanding]
**Answer:** Variable типа **`IComparable`** — reference/interface slot; `int` must be **boxed** to implement interface through object dispatch.

### 8. [Application]
**Answer:** **3 boxing** — каждый `int` literal в `object[]` converts/boxes.

### 9. [Application]
**Answer:** Overload **`Log(int message)`**; или **`Log<T>(T message)`** generic; или pass **`ReadOnlySpan<char>`** / string без object API.

### 10. [Application]
**Answer:** **Да, boxing.** Enum — value type; conversion к `object` boxes underlying integral value.

### 11. [Analysis]
**Answer:** **B** неверно. Unboxing **не удаляет** box; object становится unreachable позже GC.

### 12. [Analysis]
**Answer:** Плохой совет: boxing — **allocation каждый раз** + GC; unboxing — check + copy. Pattern box/unbox в loop **убивает performance**.

### 13. [Understanding]
**Answer:** Значение живёт **в payload boxed object на managed heap**; `o` — reference в stack frame (`1.1.6`).

### 14. [Application]
**Answer:** **`False`** (reference equality) для двух distinct box objects, даже с same numeric value — unless optimized (don't rely). **Value equality** needs `Equals` or unbox.

### 15. [Understanding]
**Answer:** `Dictionary<int,string>` stores keys as **`int`** in buckets — no boxing. `Hashtable` uses **`object` keys** — boxing int keys.

### 16. [Analysis]
**Answer:** **`is int i` on boxed int** performs unbox/pattern — **`i`** получает unboxed value (C# 7+ pattern). Safe vs blind cast.
