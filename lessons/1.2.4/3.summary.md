# Access modifiers — краткая выжимка

> `1.2.4` · ~7 мин на повторение

## Главная идея

Access modifiers задают **кто может обращаться** к type/member на этапе компиляции. Минимальная visibility → меньше coupling, проще refactor.

## Ключевые пункты

- **`private`** — только declaring class (default для members).
- **`protected`** — class + derived (inheritance hooks).
- **`internal`** — same assembly (default для top-level types).
- **`public`** — везде; это **API contract**.
- **`protected internal`** = protected **OR** internal.
- **`private protected`** = derived только в same assembly.

## Частые ошибки

- Public everything.
- `protected` fields вместо controlled API.
- `internal` как «безопасность» — это не runtime barrier.
