# Exercises: async void

## Easy

### E1. Find async void
**Goal:** Search codebase (or sample) for `async void` outside event handlers.
**Success criteria:** List refactor targets to async Task.

### E2. Handler Wrapper
**Goal:** Implement `RunSafe(Func<Task>)` for WPF click handler.
**Success criteria:** Logged exceptions, no crash.

## Medium

### M1. Test Comparison
**Goal:** xUnit: one `async Task` test that throws vs `async void` — observe runner behavior.
**Success criteria:** Document which fails reliably.

### M2. Extract Pattern
**Goal:** Refactor `async void` handler with 50 lines to void shell + `async Task` core.
**Success criteria:** Unit test covers Task method.

## Hard

### H1. ASP.NET Audit
**Goal:** Explain why `async void` filter/action breaks exception middleware; propose fix.
**Success criteria:** References Task return and pipeline observation.
