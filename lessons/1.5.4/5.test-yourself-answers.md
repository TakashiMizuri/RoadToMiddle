# Answers: Stack Frames and Stack Overflow

## Answers

### 1. [Recall]
**Answer:** Activation record одного вызова метода на thread stack.
**Why:** Unit of call/return.

### 2. [Recall]
**Answer:** Return address, parameters, locals, saved registers; optional frame pointer chain.
**Why:** Execution + debugging + GC scan.

### 3. [Understanding]
**Answer:** `int x` inline во frame; `c` — reference slot во frame, **Customer object body** на heap.
**Why:** Reference type semantics 1.1.6.

### 4. [Understanding]
**Answer:** GC must treat reachable refs as live; stack slots hold managed pointers while method active.
**Why:** Root set includes thread stacks.

### 5. [Understanding]
**Answer:** IL stack — virtual для JIT/verification; native stack — real frames per call (`1.5.1`).
**Common mistake:** Conflating the two.

### 6. [Application]
**Answer:** **StackOverflowException**; stack exhausted; catch often fails — insufficient stack to dispatch handler.
**Why:** Documented fragile exception.

### 7. [Application]
**Answer:** **Stack overflow** — allocation on thread stack, limited size (~1MB default).
**Why:** stackalloc not heap.

### 8. [Application]
**Answer:** Frame popped; slots abandoned. Value types gone; heap objects remain if still reachable elsewhere.
**Why:** Stack doesn't GC objects; removes root.

### 9. [Analysis]
**Answer:** Frames hold locals and **references**; object **payloads** on managed heap.
**Why:** Core 1.1.6 model.

### 10. [Analysis]
**Answer:** Each recursion adds 512 bytes+ frame → faster overflow; copying struct cost on entry.
**Why:** Stack pressure + perf.

### 11. [Application]
**Answer:** **PDB** debug symbols mapped to IL offsets via metadata method.
**Why:** Not in IL itself.

### 12. [Understanding]
**Answer:** May reuse caller frame for callee — reduce depth; not guaranteed for all C# patterns.
**Why:** Tier 2 optimization.

### 13. [Application]
**Answer:** Async splits logical call graph; physical frames include state machine MoveNext methods.
**Why:** Compiler desugaring.

### 14. [Analysis]
**Answer:** Stack overflow — thread stack limit; OOM — heap cannot satisfy allocation/GC.
**Why:** Different memory arenas.

### 15. [Understanding]
**Answer:** Tell GC which slots contain managed pointers during stack walk.
**Why:** Exact scanning.
