# Naming conventions, XML doc comments — краткая выжимка

> `1.1.20` · ~7 мин на повторение

## Главная идея

**Naming** — первая документация кода; **`///` XML** — вторая для public API (IntelliSense, NuGet, OpenAPI). Consistency в repo важнее личных предпочтений.

## Ключевые пункты

- Types/members/properties/events → **PascalCase**; parameters/locals → **camelCase**.
- Private fields → **`_camelCase`** (de facto .NET convention).
- Interfaces → **`IName`**; async public methods → **`NameAsync`**.
- Generics → **`T`** or **`TKey`**, **`TValue`**.
- Boolean → **`Is`/`Has`/`Can`** prefix.
- **`///`**: `<summary>` (first sentence = tooltip), `<param>`, `<returns>`, `<exception cref="">`.
- **`cref`** must resolve; **`param name`** = exact identifier.
- **`.csproj`**: `<GenerateDocumentationFile>true</GenerateDocumentationFile>`; **CS1591** policy.
- Document **behavior**, null rules, units — not «Gets the Id».

## Самые важные примеры

```csharp
/// <summary>Finds an order by id or returns null.</summary>
/// <param name="orderId">Primary key.</param>
/// <returns>Order or null.</returns>
Task<Order?> FindAsync(Guid orderId, CancellationToken cancellationToken = default);
```

```xml
<GenerateDocumentationFile>true</GenerateDocumentationFile>
```

## Частые ошибки

- **`ProcOrd`**, **`GetData`** без контекста.
- **`Async`** suffix missing.
- XML = copy of signature.
- Mixed field naming styles in one project.

## Мнемоника

> **Public surface: PascalCase + Async + meaningful ///. Locals: camelCase. Private fields: _leadingUnderscore.**
