# LINQ Join, GroupJoin — краткая выжимка

> `1.3.21` · ~8 мин на повторение

## Главная идея

**`Join`** = **INNER JOIN** (пары outer-inner по ключу). **`GroupJoin`** = outer row + **`IEnumerable`** всех inner matches; с **`DefaultIfEmpty`** — flat **LEFT JOIN**.

## Ключевые пункты

- **`Join`**: `(outer, inner) => result` — без match строки нет.
- **`GroupJoin`**: `(outer, innerGroup) => result` — outer всегда есть.
- Query: **`join ... into g`** then **`from x in g.DefaultIfEmpty()`** для left flat.
- Composite key: `new { A, B } equals new { A, B }` — имена свойств важны.
- **`IEnumerable`**: hash lookup O(n+m); **`IQueryable`**: SQL JOIN.
- Duplicate inner key → несколько строк в **`Join`**.

## Самые важные примеры

```csharp
// Inner
orders.Join(customers, o => o.CustomerId, c => c.Id, (o, c) => ...);

// Left flat
from c in customers
join o in orders on c.Id equals o.CustomerId into og
from o in og.DefaultIfEmpty()
select ...;
```

## «Под капотом» в одном абзаце

Join строит **Lookup** по inner keys, проходит outer, emit matches. GroupJoin не flatten — отдаёт группы. EF translate в SQL JOIN когда keys translatable.

## Частые ошибки

- Inner вместо left — потеря rows.
- Swap keys / mismatch anonymous properties.
- **`ToList()`** внутри GroupJoin selector на IQueryable без проверки SQL.
