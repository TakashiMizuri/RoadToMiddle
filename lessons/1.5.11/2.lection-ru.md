# Утечки памяти в managed code — events и static roots

> Roadmap: `1.5.11` · Node: `1.5` — C#: .NET Runtime и память · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Объяснить **managed memory leaks** — объект **reachable** из GC root, хотя логически «не нужен».
- Прослеживать roots: **static**, **events**, caches, timers, **DI singletons**.
- Диагностировать **event handler leaks** (`+=` без `-=`).
- Находить **static collections** и unbounded caches.
- Использовать **dotnet-gcdump** / heap snapshots для retention paths.
- Применять fixes: unsubscribe, scoped lifetime, bounded cache, **`WeakReference`**.

---

## Why This Matters

«GC есть — leak невозможен» — миф. **Leak** = strong reference chain от root. RAM растёт часами, Gen2 не сжимается, handlers на «уничтоженных» UI, static cache держит миллион DTO.

**Singleton** с **scoped DbContext**, **static event** с instance handlers, **Timer** с closure на `this` — типичные production bugs. Связь с **`1.5.6`** (roots) и **`1.5.10`** (disposal).

---

## Core Concepts

### Reachability

GC собирает при отсутствии strong chain от **roots** (stack, static, handles…). **Logical leak** — chain есть, ownership забыли.

### Event leaks

Delegate хранит **Target** + method. **`publisher.Event += this.Handler`** — `publisher` (особенно **static**/singleton) держит **`this`**.

**Lambda без сохранённой ссылки** — **нельзя `-=`** → leak.

Fix: **`-=`** в `Dispose`/end of lifetime; stored delegate; weak events в UI.

### Static roots

```csharp
static readonly List<byte[]> _cache = new();
public static event Action? GlobalNotify;
```

Живут **process lifetime**. Unbounded cache = leak by design.

### Timers и ASP.NET

**Timer**/`Task.Run` capture `this` — **Dispose** timer.

| Anti-pattern | Проблема |
|--------------|----------|
| Singleton → DbContext | Connection leak |
| Singleton List per-request | Рост forever |
| Static event | Все subscribers pinned |

Fix captive dependency: **`IServiceScopeFactory.CreateScope()`**.

### Caches

**MemoryCache** с limits/expiration. **`WeakReference<T>`** — optional cache; **`TryGetTarget`**.

## Under the Hood

**MulticastDelegate** invocation list — strong refs. **Static event field** = GC root. **dotnet-gcdump** diff — кто держит `UserDto`?

Finalizers **не** лечат reference leaks — object reachable.

## Examples

```csharp
public class SessionCleanup
{
    public SessionCleanup() => AppEvents.UserLoggedOut += OnLogout;
    public void Detach() => AppEvents.UserLoggedOut -= OnLogout;
}
```

---

## Common Mistakes

Static events без unsubscribe contract. Lambda `+=`. Singleton с per-user `List`. `= null` на field при живом event.

---

## Key Takeaways

- Leak = **retention**, не «GC сломан».
- **Events** держат target; **static** = root навсегда.
- Каждый **`+=`** — plan для **`-=`**.
- Heap diff в diagnostics.
