# var vs explicit types — краткая выжимка

> `1.1.5` · ~8 мин на повторение

## Главная идея

**`var`** — compile-time type inference: компилятор выводит **статический** тип из initializer. Это **не** dynamic typing. Выбор между `var` и явным типом — вопрос **читаемости**: что reviewer должен увидеть, не глядя на правую часть `=`?

## Ключевые пункты

- **`var` требует initializer**; без него — ошибка компиляции.
- **`var x = null`** не компилируется без target type (`string? s = null`).
- **Явный тип**, когда return method неочевиден или тип несёт domain-смысл (`decimal` для денег).
- **`var`**, когда тип дублируется: `var list = new List<T>()`; для **anonymous types** — `var` обязателен.
- **Литералы из `1.1.3`:** `var x = 19.99` → `double`; `var x = 19.99m` → `decimal`; большие целые — явный `long` + `L`.
- **Public API** — всегда explicit types; `var` — для locals.
- **Runtime/IL** — разницы с explicit нет.

## Самые важные примеры

```csharp
decimal total = CalculateOrderTotal(id);  // explicit — тип не obvious
var orders = new Dictionary<Guid, Order>(); // var — тип на RHS
var summary = orders.Select(o => new { o.Id, o.Total }); // anonymous
long size = 2_147_483_648L;               // не var — иначе int/trap
```

## Частые ошибки

- Думать, что `var` = «любой тип в runtime».
- `var data = GetData();` — скрывает контракт.
- Забыть суффикс `m`/`L`/`f` при `var`.

## Мнемоника

> **`var`, когда тип уже сказан справа; explicit, когда тип — новость для читателя.**
