# Answers: LINQ Take, Skip, TakeWhile, SkipWhile

## Scoring

- 90–100%: отлично · 80–89%: pass · <80%: повтори лекции

## Answers

### 1. [Recall]
**Answer:** `3, 4`
**Why:** Skip 2 → start at 3; Take 2 → 3 and 4.

### 2. [Recall]
**Answer:** `TakeWhile` — contiguous prefix, stops at first false; `Where` scans entire sequence and filters all matches anywhere.
**Why:** Early termination vs full scan.

### 3. [Understanding]
**Answer:** `3, 1`
**Why:** Skip while `<3` drops 1,2; from 3 onward all elements pass through unfiltered.

### 4. [Understanding]
**Answer:** Without deterministic order, page contents change between queries/executions.
**Why:** SQL and in-memory order undefined without ORDER BY.

### 5. [Application]
**Answer:** `Skip(20)` — (3-1)*10 = 20.
**Why:** Standard offset formula.

### 6. [Application]
**Answer:** `TakeWhile(line => !string.IsNullOrWhiteSpace(line))` — stops at blank without reading rest.
**Why:** Efficiency and prefix semantics.

### 7. [Analysis]
**Answer:** Empty sequence, no exception.
**Why:** Take(0) is valid edge case.

### 8. [Analysis]
**Answer:** Engine must scan/skip 1M rows before returning 20 — O(offset) work.
**Why:** OFFSET pagination cost; prefer keyset.

### 9. [Understanding]
**Answer:** Empty — zero elements yielded.
**Why:** Prefix never started.

### 10. [Application]
**Answer:** `.Skip(5).Take(5)` — indices 5,6,7,8,9.
**Why:** Skip first 5, take next 5.
