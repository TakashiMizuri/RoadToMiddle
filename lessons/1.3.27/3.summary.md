# First vs Single vs FirstOrDefault — краткая выжимка

> `1.3.27` · ~7 мин

## Главная идея

Operators кодируют **cardinality contract**: `First` — ≥1 (берём первый); `Single` — ровно 1; `*OrDefault` — 0 OK → default.

## Ключевые пункты

| | 0 | 1 | 2+ |
|--|---|---|-----|
| First | throw | ✓ | first |
| FirstOrDefault | default | ✓ | first |
| Single | throw | ✓ | throw |
| SingleOrDefault | default | ✓ | throw |

- `Single` сканирует до второго match — проверка uniqueness.
- `First` на «unique» поле — anti-pattern; нужен `Single` + DB UNIQUE.
- Immediate execution, не deferred.
- EF: `First` → LIMIT 1 без проверки дубликатов на server.

## Примеры

```csharp
await db.Users.SingleAsync(u => u.Email == email);
await db.Flags.SingleOrDefaultAsync(f => f.Key == key);
orders.OrderByDescending(o => o.CreatedAt).FirstOrDefault();
```

## «Под капотом»

`Single` iterator держит flag `seen`; второй matching element → throw. `First` return на первом же.

## Частые ошибки

- `First` вместо `Single` для unique invariant.
- `FirstOrDefault` без null-check.
- `Single` на необ indexed full table scan.
