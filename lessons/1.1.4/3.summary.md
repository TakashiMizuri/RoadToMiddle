# string — краткая выжимка

> `1.1.4` · ~10 мин на повторение

## Главная идея

**`string`** — **immutable reference type** (UTF-16 на heap). Любое «изменение» создаёт **новый** object. **`$""`** — interpolation; **`@""`** — verbatim; **`"""`** — raw multiline. **Intern pool** может dedupe **literals**; runtime strings обычно отдельные instances.

## Ключевые пункты

- `s[0]` read-only; `s += "x"` — новая allocation + новая reference.
- **`Length`** — UTF-16 code units, не всегда «символы» (emoji = 2).
- **`string.Equals(..., StringComparison.Ordinal)`** для IDs/logs; culture-aware для UI sort.
- **`+` в цикле** — O(n²) allocations → **`StringBuilder`** / **`Join`** (`1.1.16`).
- **`ReferenceEquals`** для literals может быть true; для `Concat` — false.
- **`string.Intern`** — не для unbounded user input (pool не shrinking).
- Logs/API: **InvariantCulture** в formatting.

## Самые важные примеры

```csharp
string path = @"C:\data\file.txt";
string msg = $"Order {id} total {total:C}";
string built = string.Concat("hel", "lo"); // typically not interned
```

## «Под капотом» в одном абзаце

String object хранит fixed-length char buffer; hash cache после первого GetHashCode. `+=` копирует в новый buffer. Roslyn lowers `$"a {x}"` в Format или DefaultInterpolatedStringHandler (.NET 6+). Intern pool — hash table canonical strings; compile-time literals часто shared; dynamic intern держит memory до process lifetime.

## Частые ошибки

- Думать, что string mutable
- SQL через `$"SELECT ... {user}"`
- `==` vs `ReferenceEquals` путаница
- Concat в hot loop

## Мнемоника

> **S**tring **S**tays — меняешь переменную, не содержимое object; для сборки в цикле — **B**uilder, не **+**.
