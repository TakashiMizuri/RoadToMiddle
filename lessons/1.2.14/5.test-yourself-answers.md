## Answers

### 1. [Recall]
**Answer:** Struct — **копия всех полей**; class — **копия ссылки** (тот же объект).
**Why:** Value vs reference semantics.

### 2. [Recall]
**Answer:** **Нет** наследования struct от struct/class (кроме implicit `ValueType`); class может наследовать class.
**Why:** CLR value type layout rules.

### 3. [Recall]
**Answer:** Оборачивание value type в **heap object** при присваивании к reference type (`object`, non-generic API).
**Why:** CLR needs object header for reference semantics.

### 4. [Understanding]
**Answer:** `foreach` итерирует **копию** каждого struct; `p.X++` меняет копию, не элемент в list.
**Why:** `List<T>` indexer returns copy for struct.

### 5. [Understanding]
**Answer:** Нужны **identity**, **mutability**, change tracking, references/navigations на heap graphs — struct copy semantics ломают ORM model.
**Why:** DbContext tracks object identity by reference.

### 6. [Understanding]
**Answer:** **Копирование ~128 bytes** (плюс alignment) на каждый call — CPU/memory bandwidth.
**Why:** By-value pass copies entire struct.

### 7. [Application]
**Answer:** **`2`** — `a` и `b` один объект.
**Why:** Reference copy.

### 8. [Application]
**Answer:** **`1`** — `M` получает copy.
**Why:** By-value parameter.

### 9. [Application]
**Answer:** Struct: small immutable value, no identity, high-frequency local use. Class: identity, inheritance, large/mutable state, null sentinel.
**Why:** Standard decision criteria.

### 10. [Analysis]
**Answer:** **`"b"`** — struct copy duplicated **reference** to same string; reassignment `b.Text` меняет поле в обеих копиях struct (один string object).
**Why:** Shallow copy of reference field.

### 11. [Analysis]
**Answer:** **C** неверно — struct **может** оказаться в heap (boxed, field of class, escape analysis).
**Common mistake:** «struct = stack» dogma.

### 12. [Understanding]
**Answer:** **`in`** передаёт **readonly reference** — избегает копирования большого struct.
**Why:** Performance + readonly safety.

### 13. [Application]
**Answer:** Когда struct приведён к **interface type** и вызов идёт через interface reference — типичный boxing (если не generic specialization JIT).
**Why:** Interface dispatch needs object or boxed value.

### 14. [Analysis]
**Answer:** Struct array — **contiguous data** в одном объекте массива; class array — массив **pointers**, объекты разбросаны в heap → cache misses.
**Why:** Memory locality.

### 15. [Application]
**Answer:** **Struct** — immutable logical instant, small fixed size, копируется by value без allocation, нет identity между двумя `DateTime` values.
**Why:** BCL value semantics for dates.
