# Access modifiers — public, private, protected, internal

> Roadmap: `1.2.4` · Node: `1.2` — C#: ООП · Depth: **понимание**

## Learning Objectives

После этого урока ты сможешь:

- Применять **`public`**, **`private`**, **`protected`**, **`internal`** к classes и members.
- Понимать **`protected internal`** и **`private protected`** на базовом уровне.
- Выбирать **minimum necessary visibility**.
- Связать modifiers с **assembly boundaries** и **`InternalsVisibleTo`** (concept).
- Знать defaults: members **`private`**, top-level types **`internal`**.
- Preview **`protected`** для inheritance (`1.2.6`).

---

## Why This Matters

Encapsulation (`1.2.5`) — не только properties, а **кто может трогать state**. Access modifiers — compiler-enforced fence. Public surface — **contract**; остальное — implementation detail для свободного refactor.

Libraries держат helpers **`internal`**. Domain classes — fields **`private`**, behavior через methods. Middle читает **`internal`** в framework code: «same assembly (+ friends)», не «secret from runtime».

## Core Concepts

### Спектр visibility

| Modifier | Same class | Derived | Same assembly | Other assembly |
|----------|------------|---------|---------------|----------------|
| `private` | Yes | No | No | No |
| `protected` | Yes | Yes | * | * |
| `internal` | Yes | ** | Yes | No |
| `public` | Yes | Yes | Yes | Yes |

\*Derived across assemblies если subclass exists.  
\*\*Unless derived.  
**`protected internal`** = protected **OR** internal (union).  
**`private protected`** = derived **only in same assembly**.

Members default **`private`**. Top-level types default **`internal`**.

### `public` — universal contract

Callable из любого code с reference на assembly (если type public). **API surface** — breaking changes больны consumers.

### `private` — implementation detail

Только **declaring class**. Default для fields — правильный выбор (`1.2.2`). **`private`** methods — helpers без расширения API.

### `protected` — inheritance hook

Class + **derived classes**:

```csharp
protected void MarkUpdated() => LastModifiedUtc = DateTime.UtcNow;
```

Для **extension points**, не general API — overuse → fragile base class (`1.2.11`).

### `internal` — assembly-local

Same **assembly** (.dll/.exe):

```csharp
internal class SqlMigrationRunner { ... }
```

Implementation types не для NuGet consumers. Tests: **`InternalsVisibleTo`**.

### Nested types

**`private nested`** helper внутри public class — builder pattern details.

## Examples

**Invoice** — public `Total`, private `_lines`, public `AddLine`.  
**EfOrderRepository** — `internal`, public `IOrderRepository` interface.  
**Parser** abstract — `protected` Tokenize hook, public Parse.

---

## Common Mistakes

Everything public. **`protected` fields**. Думать что **`internal` = security** (reflection exists). Over-wide **`protected internal`**.

---

## Production

NuGet: public API = semver. Friend assemblies для tests. ASP.NET — minimize public **mutable** state.

---

## Quick Reference

```
private   → this class (default member)
protected → + derived
internal  → same assembly (default type)
public    → everywhere
```

---

## Key Takeaways

- Modifiers — **compile-time who can call**.
- **`public`** small; **`private`** default storage.
- **`internal`** — assembly implementation, not runtime security.
- **`protected`** — deliberate inheritance hooks.
