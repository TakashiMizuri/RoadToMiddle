# Exercises: Upstream tracking, push -u, pull --rebase

> Complete after reading lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Set upstream on first push
**Goal:** Link local branch to remote.

**Steps:**
1. Create branch `exercise/upstream` with one commit.
2. `git push -u origin exercise/upstream`.
3. Run `git branch -vv` and `git rev-parse --abbrev-ref @{upstream}`.

**Success criteria:** Upstream shows `origin/exercise/upstream`; bare `git push` works.

---

### E2. Read ahead/behind
**Goal:** Interpret tracking status.

**Steps:**
1. On tracked branch, make a local commit without pushing.
2. `git status` — note "ahead" count.
3. Push, then status again.

**Success criteria:** Explain ahead before push and "up to date" after.

---

## Medium

### M1. pull --rebase vs pull
**Goal:** Feel history difference.

**Setup:** Two clones, branch `exercise/rebase-pull`, both commit in sequence (A pushes, B commits locally, A pushes again).

**Tasks:**
1. In B: try `git pull` (merge) in one experiment OR `git pull --rebase` in another (use fresh branches for each).
2. `git log --oneline --graph -10` — compare shapes.

**Success criteria:** Describe merge commit vs linear replay in your own words.

---

### M2. Rebase onto main before "PR"
**Goal:** Pre-PR hygiene from `0.2.7`.

**Tasks:**
1. On `exercise/pre-pr`, add a commit.
2. Ensure `origin/main` has moved ahead (fetch; merge main into remote if needed via another clone).
3. `git fetch origin && git rebase origin/main`.
4. Push with `--force-with-lease` if branch was already on remote.

**Success criteria:** Feature commits sit on top of latest `origin/main` in log.

---

### M3. Fix missing upstream
**Goal:** Recover from push without `-u`.

**Tasks:**
1. Push new branch **without** `-u`.
2. Verify `git push` fails.
3. `git branch -u origin/<branch>` and push again.

**Success criteria:** Document error message and fix.

---

## Hard (stretch)

### H1. Configure pull.rebase locally
**Goal:** Understand global default.

**Tasks:**
1. `git config pull.rebase true` (local repo only).
2. Create divergence scenario and run bare `git pull`.
3. Confirm rebase behavior via `git reflog`.

**Success criteria:** Explain when you would **not** set this globally.

---
