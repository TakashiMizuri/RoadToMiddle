# LINQ First vs Single vs FirstOrDefault — Semantics

> Roadmap: `1.3.27` · Node: `1.3` — C#: Advanced Topics · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Сформулировать **cardinality contracts** для **`First`**, **`Single`**, **`Last`** и **`*OrDefault`** variants.
- Предсказать throw vs **`default`**, **`InvalidOperationException`** vs другие exceptions.
- Объяснить **short-circuit** для **`First`** vs possible full scan для **`Single`**.
- Выбирать operator под **business invariants** (unique email, optional row).
- Mapping к **SQL** на **`IQueryable`**.
- Не подменять **`Single`** на **`First`**, когда uniqueness — invariant.

---

## Why This Matters

На первый взгляд все методы делают «дай один element», но они кодируют **разные обещания**:

- **`First`** — «нужен **какой-то** match; order решает какой».
- **`Single`** — «должен быть **ровно один**; иначе bug или corrupted data».
- **`*OrDefault`** — «zero matches **OK** → **`default`**».

**`First`** на query, который must be unique, скрывает duplicates до financial/security bugs. **`Single`** на legitimately empty filter — лишние exceptions. Middle dev читает **`Single`** как **assertion** о data integrity — как unique index в БД.

## Core Concepts

### Cardinality Matrix

| Method | 0 | 1 | 2+ |
|--------|---|---|-----|
| `First` | throw | returns | first only |
| `FirstOrDefault` | default | returns | first only |
| `Single` | throw | returns | throw |
| `SingleOrDefault` | default | returns | throw |

**`Last`/`LastOrDefault`** — mirror **`First`**, но often full scan (или O(1) на **`List<T>`**).

Throws: **`InvalidOperationException`**, не **`IndexOutOfRangeException`**.

### First — at least one

```csharp
var admin = users.First(u => u.Role == "Admin");
```

Stop на **first** match — **не** проверяет uniqueness. Zero → throw. Three admins → один silently — **wrong**, если business rule «exactly one».

**`FirstOrDefault`**: **`null`** / **`default struct`**. Для value types **`0`** может быть valid — нужен **`int?`** или explicit check.

### Single — exactly one

```csharp
var settings = db.AppSettings.Single(s => s.TenantId == tenantId);
```

После first match **продолжает** scan для **second** — O(n) worst case **намеренно**.

Use когда **unique constraint** expected. **`SingleOrDefault`**: zero OK, many throw.

### Predicate overloads

**`First(predicate)`** ≈ **`Where` + `First`**, но early stop. **`Single(predicate)`** must prove no second — full scan possible.

### Nullability

```csharp
User? u = users.FirstOrDefault(x => x.Id == id);
if (u is null) { /* not found */ }
```

**.NET 6+** **`FirstOrDefault(defaultValue)`** для explicit default.

## Under the Hood

### Iterator sketch

**`First`**: foreach, return first (match); end without match → throw.

**`Single`**: track `seen`; second match → throw «more than one»; end without match → throw «no elements».

Two **different** reference objects, оба match predicate → throw.

### IQueryable

| LINQ | SQL |
|------|-----|
| `First` | `TOP 1` / `LIMIT 1` — **без** server-side uniqueness check |
| `Single` | provider-dependent; may fetch 2 rows |

EF **`SingleAsync`** — same semantics. SQL **не** enforces **`Single`**, unless unique key in query.

## Examples

### Single by unique email

```csharp
var user = await db.Users.SingleAsync(u => u.Email == email, ct);
```

Duplicate emails → exception surfaces bug.

### SingleOrDefault optional config

```csharp
var banner = await db.FeatureFlags.SingleOrDefaultAsync(f => f.Key == "HomeBanner", ct);
```

### Anti-pattern: First when unique required

```csharp
// BAD if Email unique:
var user = users.First(u => u.Email == email);
```

### Latest via OrderBy + First

```csharp
var latest = orders.OrderByDescending(o => o.CreatedAt).FirstOrDefault();
```

---

## Common Mistakes

**`First` для unique lookup.** **`Single` без index** на huge table. **`FirstOrDefault` + `!`**. **`Single()`** на empty group.

---

## Production & Real-World Notes

**`Single`** + **`UNIQUE`** index в БД. HTTP: no rows → 404, duplicates → 409. **`Any` + `First`** — double enumeration.

## Key Takeaways

- **`First`** — first match, не uniqueness.
- **`Single`** — exactly one, integrity signal.
- **`*OrDefault`** — empty → default, handle explicitly.
- **`First`** short-circuit; **`Single`** may scan all.
