# Exercises: IDisposable and using

> Complete after passing `4.test-yourself.md`.

## Easy

### E1. File with using
**Goal:** Read all text from file with `using var stream` + `StreamReader`.
**Success criteria:** File handle closed even if you throw after open (test with intentional exception).

### E2. Temp file disposable
**Goal:** `TempFile` class creates temp file on ctor, deletes on `Dispose`.
**Success criteria:** Idempotent double-Dispose safe.

### E3. using declaration order
**Goal:** Two `using var` in one block; log dispose order in `Dispose`.
**Success criteria:** Confirm reverse declaration order.

## Medium

### M1. Dispose pattern
**Goal:** Base `ResourceBase : IDisposable` with `protected virtual Dispose(bool)`; derived adds managed field dispose.
**Success criteria:** `SuppressFinalize` in public `Dispose`.

### M2. Async dispose
**Goal:** Type with `IAsyncDisposable` that `await Task.Delay(1)` then closes fake resource.
**Success criteria:** Consumer uses `await using`.

### M3. DbContext scope (conceptual)
**Goal:** Document in comments when ASP.NET disposes scoped `DbContext` vs manual `using` in console app.
**Success criteria:** Mention request end vs block end.

## Hard (stretch)

### H1. Leak demo
**Goal:** Console app opens files in loop **without** dispose until `Too many open files` (or stop at N); repeat with `using` — no leak.
**Success criteria:** Explain OS handle limit in README.

## Submission checklist

- [ ] At least one `using` and one idempotent `Dispose`
- [ ] One exercise with `await using` (optional if no async project)
- [ ] Comment on finalizer avoidance
