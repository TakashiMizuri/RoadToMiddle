# .Result / .Wait() — краткая выжимка

> `1.4.14` · ~10 мин

## Главная идея

**Sync-over-async** — `.Wait()`, `.Result`, `GetAwaiter().GetResult()` block thread, пока continuation (`MoveNext`, `1.4.7`) не может выполниться → **deadlock** (SyncContext, `1.4.9`) или **starvation** (Core, `1.4.10`).

## Ключевые пункты

- Все три API block equally.
- `.Wait()` → AggregateException; GetResult → direct.
- UI/Framework: Post на blocked thread.
- Core: blocked pool threads.
- Library ConfigureAwait(false) — mitigation, не fix `.Result`.
- Fix: async all the way.
