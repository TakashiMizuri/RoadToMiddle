# Exercises: ConfigureAwait(false)

> Complete after passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Trace Continuation
**Goal:** Sketch sequence: UI thread awaits library method (no `ConfigureAwait(false)`) while button handler uses `.GetAwaiter().GetResult()`. Label SyncContext Post and blocked thread.
**Success criteria:** Diagram shows deadlock loop.

### E2. ASP.NET Core Audit
**Goal:** Review a controller with 5 awaits — list which (if any) would benefit from `ConfigureAwait(false)` on ASP.NET Core and why.
**Success criteria:** Justified answer referencing absent SyncContext.

## Medium

### M1. Refactor Library Method
**Goal:** Take `async Task<T>` data access method with 3 awaits; add `ConfigureAwait(false)` consistently; explain each line to a junior.
**Success criteria:** Every await configured; prose explains state machine resume.

### M2. UI vs Library Split
**Goal:** Design: `CoreLib.FetchAsync` (library) + WPF `ViewModel.LoadAsync` (app). Where `ConfigureAwait(false)`? Where default?
**Success criteria:** Library all false; ViewModel default before binding update.

## Hard (stretch)

### H1. Analyzer Policy
**Goal:** Write team guideline: when enforces Roslyn analyzer `ConfigureAwait(false)` vs exceptions for UI projects.
**Success criteria:** Tables for library/app/UI; links to `1.4.9`.
