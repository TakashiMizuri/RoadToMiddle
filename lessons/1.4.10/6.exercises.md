# Exercises: Thread Pool Starvation

## Easy

### E1. Block Detection
**Goal:** Grep a sample ASP.NET project for `.Result`, `.Wait()`, `Thread.Sleep`, `.GetAwaiter().GetResult()` in request path.
**Success criteria:** List files and suggest async replacement.

### E2. Counter Watch
**Goal:** Run `dotnet-counters monitor System.Runtime` during load test; note queue length when latency spikes.
**Success criteria:** Correlate queue growth with blocking code.

## Medium

### M1. Reproduce Mini-Starvation
**Goal:** Console app: start 100 `Task.Run` each `Thread.Sleep(10000)` while firing async chain that needs pool continuations — observe delay.
**Success criteria:** Explain MoveNext queue wait.

### M2. Fix Middleware
**Goal:** Refactor sync `.Wait()` middleware to fully async; document thread impact before/after.
**Success criteria:** No blocking call before `await next()`.

## Hard

### H1. Capacity Plan
**Goal:** Given 200 RPS and 50ms sync block hidden in service, estimate pool threads tied up and propose architecture fix (channel, dedicated executor).
**Success criteria:** Numbers + async strategy.
