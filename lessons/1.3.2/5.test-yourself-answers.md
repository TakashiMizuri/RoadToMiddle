## Answers

### 1. [Recall]
**Answer:** `out T` ‚Äî covariant, T —Ç–æ–ª—å–∫–æ output; `in T` ‚Äî contravariant, T —Ç–æ–ª—å–∫–æ input.
**Why:** Compiler enforces position of T in members.

### 2. [Recall]
**Answer:** Classes with mutators cannot declare variant parameters ‚Äî only interfaces/delegates. `List` has `Add(T)`.
**Why:** Language rule + type safety.

### 3. [Recall]
**Answer:** Interfaces and delegates.

### 4. [Understanding]
**Answer:** `IEnumerable<out T>` read-only regarding T flow out; no `Add(T)`. `List<T>` has write ‚Äî invariant.
**Why:** Covariance requires producer-only API.

### 5. [Understanding]
**Answer:** `Action<string> a = actionObject` where `Action<object> actionObject` ‚Äî legal. Reverse illegal.
**Why:** Handler expecting `object` can accept any string argument.

### 6. [Understanding]
**Answer:** Variance conversions apply to reference types; `int` is not substitutable for `object` without boxing each element ‚Äî different types.
**Why:** CLR variance rules for reference types only.

### 7. [Application]
**Answer:** **Compiles** ‚Äî contravariance on `Func` first type param, covariance on return (both `string`).
**Why:** `string` ‚Üí `object` for input parameter position (contravariant).

### 8. [Application]
**Answer:** **Does not compile** ‚Äî `List<T>` invariant.
**Why:** Would allow unsound writes.

### 9. [Application]
**Answer:** `IEnumerable<object> o = listOfStrings;` ‚Äî implicit conversion via `IEnumerable<out T>` covariance.
**Why:** No copy required; view as sequence of objects at compile time (enumeration boxes value types if any ‚Äî here strings are references).

### 10. [Analysis]
**Answer:** Line 3 throws **`ArrayTypeMismatchException`** at runtime ‚Äî array is `string[]` stored as `object[]`.
**Why:** Reference array covariance is shallow; store must match actual element type.

### 11. [Analysis]
**Answer:** **`Store(T item)`** takes T as input ‚Äî violates `out T`.
**Why:** Covariant interface cannot have input methods using T.

### 12. [Understanding]
**Answer:** **Yes** ‚Äî contravariance: `IComparer<object>` assignable to `IComparer<string>`.
**Why:** Comparer for `object` can compare any strings.

### 13. [Analysis]
**Answer:** **No** ‚Äî outer `List<>` invariant even if inner `IEnumerable<>` covariant.
**Why:** `List<T>` type argument must match exactly.

### 14. [Recall]
**Answer:** Neither covariant nor contravariant ‚Äî `IGeneric<A>` and `IGeneric<B>` not related by subtyping unless `A` equals `B` (for invariant parameter).
**Why:** Default for unmarked parameters and all generic classes.

### 15. [Understanding]
**Answer:** Callers can treat result as wider type (`IEnumerable<Base>`) when pipeline produces `Derived`; composable without copying to `List<Base>`.
**Why:** Covariant return enables substitutability in APIs.
