## Core Concepts

### IAsyncStateMachine Interface

```csharp
public interface IAsyncStateMachine
{
    void MoveNext();
    void SetStateMachine(IAsyncStateMachine stateMachine);
}
```

**`MoveNext`** — entire async method body rewritten as **stateful procedure**.

**`SetStateMachine`** — boxes struct state machine to heap when needed (when address taken for continuation registration).

Compiler generates **`struct`** state machine when possible (allocation-free start); may **box** on first suspend.

### Typical Generated Shape (C# Decompiled)

```csharp
[CompilerGenerated]

{
    public int <>1__state;
    public AsyncTaskMethodBuilder<string> <>t__builder;
    public HttpClient http;

    private TaskAwaiter<string> <>u__2;

                        return;
                    }

                    <>s__1 = <>u__1.GetResult();
                    <>u__2 = <>s__1.Content.ReadAsStringAsync().GetAwaiter();

                        <>1__state = 1;
                        <>t__builder.AwaitUnsafeOnCompleted(ref <>u__2, ref this);

                    }
                    break;

                    break;
            }
            result = <>u__2.GetResult();
        }
        catch (Exception ex)
        {
            <>1__state = -2;
            <>t__builder.SetException(ex);
            return;
        }
        <>1__state = -2;
        <>t__builder.SetResult(result);
    }

    void IAsyncStateMachine.SetStateMachine(IAsyncStateMachine _) { }
}
```

Names vary (`d__`, numbered states). **`default`** case = initial run (**`state == -1`** before start in some versions).

### State Integer Lifecycle

- **`-1`** — before start / sometimes after sync complete start.
- **`0`, `1`, ...** — resume after Nth await.
- **`-2`** — finished (terminal marker in many generators).

Each **`await`** that suspends increments assigned **case** label.

### AwaitUnsafeOnCompleted

Registers **`MoveNext`** (via runner) on **`INotifyCompletion`** / **`ICriticalNotifyCompletion`**. When task completes, **`MoveNext`** re-enters **`switch`** at saved **`__state`**.

**`Unsafe`** variant — avoids capturing **ExecutionContext** in some paths (performance; BCL uses carefully).

### SetResult and SetException

Successful path: assign result local → **`__state = -2`** → **`builder.SetResult(result)`** — completes **`Task`** returned to original caller.

Catch block: **`SetException(ex)`** — faulted task.

If **`MoveNext`** returns without Set* after suspend — still in progress. If never Set* — ** hung Task**.

## Under the Hood

### IL Highlights

In IL, **`MoveNext`** is one method with **branches** instead of C# switch. Look for:

- **`ldfld <>1__state`**
- **`switch`** or chain of **`beq`** to labels
- **`call`** `AsyncTaskMethodBuilder.AwaitOnCompleted`
- **`call`** `TaskAwaiter.GetResult`

**`dotnet run`** + **`ildasm`** or SharpLab IL mode.

### Struct vs Class State Machine

**Struct** on stack initially; first **`AwaitOnCompleted`** may copy to heap (**`SetStateMachine`**) so continuation holds stable reference.

Large async methods → **class** state machine always heap — allocation per call.

### AsyncMethodBuilder.Start

Stub method:

```csharp
public Task<string> FetchAsync()
{
    var sm = new <FetchAsync>d__2 { <>t__builder = AsyncTaskMethodBuilder<string>.Create(), http = this.http };
    sm.<>t__builder.Start(ref sm);
    return sm.<>t__builder.Task;
}
```

**`Start`** calls first **`MoveNext`**.

### Debugging Stack Traces

You see:

```
Program+<FetchAsync>d__2.MoveNext()
AsyncTaskMethodBuilder.Start(...)
Program.FetchAsync()
```

Line mapping points to **`await`** lines via PDB sequence points — not arbitrary.

**.NET 9+** improvements continue for async stack trace clarity (`1.4.37`).

### Connection to 1.4.1–1.4.6

- **Thread**: between **`MoveNext`** segments, no thread owned by this logical method during I/O.
- **Task**: **`builder.Task`** is what caller awaits; same object **`SetResult`** completes.
- **TCS**: different path — manual completion; no **`MoveNext`** unless you **`await`** TCS inside async method (then nested machine).

## Examples

### Example 1: Single await — Minimal Machine

States: **default** → maybe **0** → SetResult. One **`TaskAwaiter`** field.

### Example 2: await in loop

Compiler may generate **loop inside MoveNext** with state preserved — **`foreach await`** (`IAsyncEnumerable`) more complex (`1.4.16`).

### Example 3: Exception before await

Throw in **`default`** case before any **`AwaitOnCompleted`** → catch → **`SetException`** — never suspended.

## Common Mistakes & Anti-patterns

**Editing generated files** — never; always fix source.

**Assuming state numbers stable** — compiler version may renumber; don't depend in code.

**Profiling MoveNext cost** — usually negligible vs I/O; optimize algorithm first.

**Confusing builder.Task with awaited task** — inner vs outer Task in nested async.

---

## Production & Real-World Notes

When **`Task` never completes**, dump: hung **`MoveNext`?** missing callback? TCS never set? deadlock on sync context?

Performance-sensitive paths: reduce **`await`** count, use **`ValueTask`**, avoid unnecessary state machine (**sync fast path**).

Security: state machine fields hold **captured locals** — sensitive data lives until GC — same as closures.

## Quick Reference

- **`<>1__state`** — current segment index
- **`<>t__builder`** — completes returned Task
- **`<>u__N`** — TaskAwaiter fields
- **`MoveNext`** — full method + continuations
- **`-2`** — finished state marker

## Key Takeaways

- **`IAsyncStateMachine.MoveNext`** IS your async method — split at awaits.
- **`AwaitUnsafeOnCompleted`** + saved state = thread-free I/O wait.
- **`SetResult`/`SetException`** complete caller's Task.
