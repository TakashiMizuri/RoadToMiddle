# Answers: Events — Memory Leak Pattern

## Scoring

- 90–100%: отлично · 80–89%: pass · <80%: повтори lection

---

## Answers

### 1. [Recall]
**Answer:** **event**: only `+=`/`-=`; **field**: also `=` assign/replace entire delegate.
**Why:** Encapsulation of invocation list.

### 2. [Recall]
**Answer:** **`void(object? sender, EventArgs e)`**

### 3. [Recall]
**Answer:** **Only publisher** (declaring class) should invoke; consumers subscribe only.
**Why:** Convention + `event` may restrict invoke if custom accessors hide field.

### 4. [Understanding]
**Answer:** Static EventBus reachable → holds delegate → delegate.Target = Form instance → Form reachable despite no other refs → leak.
**Why:** Strong reference chain through multicast delegate.

### 5. [Understanding]
**Answer:** Lambda closure captures `this`; delegate target is display class or instance method holding Form/Button owner alive from publisher's delegate list.
**Why:** Strong refs both ways via subscription.

### 6. [Understanding]
**Answer:** Synchronize `+=`/`-=`/`Invoke` when multi-threaded; prevent torn reads of delegate field.
**Why:** Delegate assignment not atomic for combined operations without lock.

### 7. [Application]
**Answer:**
```csharp
public void Dispose()
{
    _dataService.OnUpdated -= HandleUpdated;
}
```

### 8. [Application]
**Answer:**
```csharp
var handlers = OnUpdated;
handlers?.Invoke(this, EventArgs.Empty);
```

### 9. [Analysis]
**Answer:** **High** — any subscriber pinned until explicit `-=`; global lifetime.
**Why:** Static holder lives app lifetime.

### 10. [Analysis]
**Answer:** Handler still called on disposed object → **`ObjectDisposedException`**, invalid state, or silent bug.
**Why:** Subscription still in invocation list.

### 11. [Understanding]
**Answer:** **WeakReference** to subscriber; dead refs pruned on raise — subscriber GC when no other refs. Rare in BCL due complexity, edge cases, performance.
**Why:** WPF WeakEventManager specialized.

### 12. [Application]
**Answer:**
```csharp
_cache.Invalidated += OnInvalidated;
// ...
_cache.Invalidated -= OnInvalidated;

private void OnInvalidated(object? s, EventArgs e) { }
```

### 13. [Analysis]
**Answer:** **Field** `public EventHandler MyEvent` — compiles. **`event`** — compile error for external `=`.
**Why:** event add/remove only public.

### 14. [Understanding]
**Answer:** **Subscribe returns IDisposable** — `using` disposes subscription explicitly; clearer lifetime than `-=` for streams.
**Why:** Rx pattern.

### 15. [Recall]
**Answer:** **`TEventArgs : EventArgs`**
