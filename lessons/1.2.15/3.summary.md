# readonly struct, ref struct — краткая выжимка

> `1.2.15` · ~7 мин на повторение

## Главная идея

**`readonly struct`** — компилятор запрещает менять instance после construction; **`ref struct`** (включая **`Span<T>`**) — только stack lifetime, **нельзя box** и нельзя хранить в полях обычного class.

## Ключевые пункты

- **`readonly struct`** — все instance fields readonly; mutation → новый instance.
- **`readonly record struct`** — immutability + record equality (`1.2.13`).
- **Defensive copies** — меньше с readonly struct при вызове методов.
- **`ref struct`** — no boxing, no class fields (обычно), ограничения async/iterator.
- **`Span<T>` / `ReadOnlySpan<T>`** — zero-allocation slice array/stackalloc.
- Long-lived buffer → **`Memory<T>`** / `byte[]`, не `Span` в поле class.
- **`ref`** — mutate caller; **`in`** — readonly без copy; **`scoped`** — ref не escape.
- **`stackalloc`** — stack buffer; осторожно с размером и recursion.

## Самые важные примеры

```csharp
public readonly record struct Point(int X, int Y);

Span<byte> buf = stackalloc byte[256];
ReadOnlySpan<char> slice = "hello".AsSpan();

// CS8345: Span<byte> cannot be field in class
```

## Частые ошибки

- Хранить `Span<T>` в instance field.
- Огромный stackalloc.
- Путать `readonly struct` с «никогда не копируется» — copy by value остаётся (`1.2.14`).

## Мнемоника

> **Span — локальный взгляд на память; в class поле — Memory; readonly struct — не меняй, создай новый.**
