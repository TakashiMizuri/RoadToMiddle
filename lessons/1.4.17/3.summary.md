# Channels — producer/consumer — краткая выжимка

> `1.4.17` · ~6 мин на повторение

## Главная идея

**`System.Threading.Channels`** — **async in-process очередь** между producer и consumer. **Bounded** channel при переполнении **await'ит `WriteAsync`** → backpressure. Альтернатива unbounded `List` и thread-blocking **`BlockingCollection`**.

## Ключевые пункты

- **`ChannelWriter<T>`** / **`ChannelReader<T>`** — разделение сторон.
- **`CreateBounded(capacity)`** — production default; **`CreateUnbounded()`** — риск OOM.
- **`FullMode`:** `Wait`, `DropWrite`, `DropOldest`.
- Consumer: **`await foreach (var x in reader.ReadAllAsync(ct))`**.
- Shutdown: **`writer.Complete()`** + drain consumer.
- **In-process** — не Redis/RabbitMQ.

## Самые важные примеры

```csharp
var ch = Channel.CreateBounded<Job>(100);
await ch.Writer.WriteAsync(job, ct);
await foreach (var j in ch.Reader.ReadAllAsync(ct))
    await ProcessAsync(j);
ch.Writer.Complete();
```

## Частые ошибки

- Unbounded под HTTP flood.
- Забытый **`Complete()`**.
- Игнор **`TryWrite == false`**.

## Мнемоника

> **Channel = конвейерная лента фиксированной длины** — если упаковщик быстрее грузчиков, лента останавливается (`Wait`), а не заваливает склад.
