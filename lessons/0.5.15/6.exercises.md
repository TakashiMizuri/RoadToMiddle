# Exercises: data-* and Custom Elements

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Dataset Drill
**Goal:** Practice HTML ↔ JS mapping.
**Steps:** Markup three `<li>` with `data-price`, `data-currency`, `data-taxable`. Script sums prices where `data-taxable="true"` (parse numbers).
**Success criteria:** Correct sum; explicit `Number()` parsing documented in comment.

### E2. Delegation Handler
**Goal:** One listener, many buttons.
**Steps:** Table with `[data-action="edit|delete"]` buttons. Single `click` on `<tbody>` uses `closest`.
**Success criteria:** Works for dynamically added row; no per-row listeners.

## Medium

### M1. Register formatted-date
**Goal:** Minimal Custom Element.
**Steps:** Implement `formatted-date` reading `datetime` attribute; `define` in external JS; use in static HTML.
**Success criteria:** Hyphenated name; `connectedCallback` only; graceful missing attribute.

### M2. data-* vs JSON script
**Goal:** Choose storage for config.
**Steps:** Given 2KB nested config for chart widget, argue against `data-config` and propose `<script type="application/json" id="chart-config">` or fetch.
**Success criteria:** Mentions HTML size, escaping, maintainability.

## Hard (stretch)

### H1. Shadow DOM Sketch
**Goal:** Conceptual encapsulation.
**Steps:** Pseudocode `attachShadow({ mode: 'open' })` inside custom element; explain one styling benefit vs global CSS.
**Success criteria:** External `.title` does not style internal `.title` without ::part or slots discussion (brief).
