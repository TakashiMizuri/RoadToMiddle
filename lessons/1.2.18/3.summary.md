# Partial classes — краткая выжимка

> `1.2.18` · ~7 мин на повторение

## Главная идея

**`partial`** позволяет объявить один тип в нескольких файлах; компилятор **сливает** фрагменты в **один** class/struct/record/interface. Generated tools (designer, EF, source generators) пишут один файл — ты расширяешь тип в другом, не боясь overwrite.

## Ключевые пункты

- **`partial` на каждом** фрагменте; same name, namespace, compatible modifiers.
- Merge — **compile time**, не runtime.
- **Не редактировать** `*.Designer.cs`, `*.g.cs`, `<auto-generated>`.
- **`partial method`**: без реализации — вызовы **удаляются** компилятором (zero cost hook).
- **private** в одной части виден в другой части того же типа.
- Кастомизация: свой partial file, partial methods, `OnModelCreating`, wrapper.
- Поддерживаются: class, struct, record, interface (не enum).

## Самые важные примеры

```csharp
// You
public partial class AppDbContext : DbContext
{
    protected override void OnModelCreating(ModelBuilder b) { /* configs */ }
}

// Generated
public partial class AppDbContext
{
    public DbSet<Order> Orders => Set<Order>();
}
```

## Частые ошибки

- Правки в designer/generated → потеря при regen.
- Забытый `partial` на одном файле.
- Бизнес-логика в Designer.cs.
- Разные namespace между частями.

## Мнемоника

**Tools own .Designer.cs — you own .cs. partial on both. Never fight the generator.**
