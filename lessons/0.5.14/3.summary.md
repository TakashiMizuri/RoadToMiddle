# iframe, embed, sandbox — краткая выжимка

> `0.5.14` · ~8 мин на повторение

## Главная идея

`<iframe>` — вложенный **browsing context** с отдельным origin и DOM. **`sandbox`** и CSP **`frame-ancestors`** — основные HTML/HTTP рычаги least privilege и защиты от clickjacking при embeds.

## Ключевые пункты

- Parent **не access** cross-origin iframe DOM — мост **`postMessage`** + проверка `event.origin`.
- **`sandbox`** (пустой) — max lockdown; tokens `allow-scripts`, `allow-forms` и т.д. снимают ограничения.
- **`allow-scripts` + `allow-same-origin`** на untrusted HTML — опасная комбинация.
- **`title`** на iframe — accessibility requirement.
- **`allow`** — Permissions Policy (camera, mic, payment…).
- **Clickjacking:** CSP `frame-ancestors 'none'|'self'` на admin/sensitive UI.
- HTTPS page → только **HTTPS** embed (mixed content).
- `<embed>` — legacy/plugin-style; для documents предпочитай **iframe**.

## Самые важные примеры

```html
<iframe src="https://player.example.com/v/1"
        title="Demo video" sandbox="allow-scripts"
        allow="fullscreen"></iframe>
```

```
Content-Security-Policy: frame-ancestors 'self';
```

## Частые ошибки

Embed без title. Ослабление sandbox для user HTML на main origin. postMessage без origin check.

## Мнемоника

> **iframe** = «мини-браузер в странице»; **sandbox** = «whitelist capabilities»; **frame-ancestors** = «кто может обрамить нас».
