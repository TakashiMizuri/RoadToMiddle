# Exercises: HTTP → service → DB async diagram

## Easy

### E1. Trace your API
**Goal:** Map one real endpoint to layers.
**Steps:** Pick GET endpoint; list middleware, controller, service, EF call.
**Success criteria:** Written diagram matching lection.

### E2. Cancellation
**Goal:** Observe `RequestAborted`.
**Steps:** Start long `await Task.Delay` in action with `ct`; cancel client (curl Ctrl+C).
**Success criteria:** OperationCanceledException or graceful stop.

## Medium

### M1. OpenTelemetry spans
**Goal:** Three spans: middleware, service, EF.
**Steps:** Add basic OTel; one request; view parent-child in UI or console.
**Success criteria:** Spans align with sequence diagram.

### M2. Anti-pattern fix
**Goal:** Replace `Task.Run`+sync EF with `*Async`.
**Steps:** Repro thread pool pressure (conceptual or load test); fix.
**Success criteria:** Async all-the-way to Npgsql.

## Hard (stretch)

### H1. POST flow diagram
**Goal:** `POST /orders` with transaction `SaveChangesAsync`, auth middleware, outbox placeholder.
**Success criteria:** Mermaid with commit/rollback paths.
