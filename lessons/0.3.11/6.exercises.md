# Exercises: Content types

> Complete after reading lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Three curls, one public API
**Goal:** Send the same logical data in three encodings (where applicable).

**Steps:**
1. Pick a public API that accepts POST (e.g. httpbin.org/post) or your local test API.
2. POST JSON object `{ "name": "test" }` with correct `Content-Type`.
3. POST urlencoded `name=test`.
4. POST multipart with field `name=test`.

**Success criteria:** Each response echoes a different `Content-Type` / parsed body shape; you can explain which parser ran.

---

### E2. Wrong binding simulation
**Goal:** See empty model from mismatch.

**Steps:**
1. If you have ASP.NET Core API: call login endpoint with JSON while action uses `[FromForm]` (or vice versa) — observe behavior.
2. Or use httpbin to note raw body vs what a strict server would bind.

**Success criteria:** You explain why fields were null without changing server code blindly.

---

## Medium

### M1. OAuth form body
**Goal:** Build spec-shaped token request body (no real secrets needed).

**Tasks:**
1. Write urlencoded string: `grant_type=client_credentials&client_id=demo&client_secret=demo`.
2. POST to httpbin.org/post with correct header.
3. Verify form keys appear in response JSON.

**Success criteria:** No JSON body; `Content-Type` is urlencoded.
