# Answers: IAsyncDisposable

## Scoring

- 90–100%: отлично · 80–89%: pass · <80%: повтори lection + summary

## Answers

### 1. [Recall]
**Answer:** `ValueTask`.
**Why:** Async dispose pattern BCL.

### 2. [Recall]
**Answer:** `await using`.
**Why:** Compiler-generated DisposeAsync call.

### 3. [Understanding]
**Answer:** Когда cleanup требует await I/O (flush, graceful protocol close, async provider).
**Why:** Sync Dispose cannot await.

### 4. [Understanding]
**Answer:** Sync-over-async deadlock/block on pool thread, especially ASP.NET (`1.4.14`).
**Why:** Blocking wait on async dispose.

### 5. [Application]
**Answer:** `await using var scope = factory.CreateAsyncScope();` resolve DbContext inside; await work; end scope disposes.
**Why:** Per-job scope async disposal.

### 6. [Application]
**Answer:** `await using` — async close path aligned with async reads.
**Why:** useAsync + IAsyncDisposable on stream.

### 7. [Analysis]
**Answer:** Unobserved exception if fault; dispose may not complete before scope exit logic wrong.
**Why:** Same as fire-and-forget tasks.

### 8. [Understanding]
**Answer:** Properly awaits IAsyncDisposable services during scope teardown in async method.
**Why:** Avoid sync block in async consumer.

### 9. [Recall]
**Answer:** Да — повторный вызов safe no-op.
**Why:** Standard dispose pattern.

### 10. [Application]
**Answer:** `await using var r = new AsyncConnection(); await r.UseAsync();`
**Why:** DisposeAsync at scope end.

### 11. [Understanding]
**Answer:** При Dispose/DisposeAsync scope (конец request/job scope).
**Why:** DI owns scoped lifetime.

### 12. [Analysis]
**Answer:** Нет — singleton/factory-managed; dispose at app shutdown, not per request.
**Why:** Lifetime mismatch if per-request dispose.

### 13. [Understanding]
**Answer:** `DisposeAsync` runs в finally — как `using` при exception.
**Why:** Guaranteed cleanup.

### 14. [Application]
**Answer:** Inner disposed first, then outer (reverse construction order).
**Why:** Nested using semantics.

### 15. [Understanding]
**Answer:** Часто dispose синхронный — ValueTask без Task allocation.
**Why:** Performance API design.
