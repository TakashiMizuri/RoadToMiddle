# Exercises: Static Members

> Complete after lectures and passing `4.test-yourself.md`.

## Easy

### E1. Guard utility
**Goal:** Create a stateless `static class` `Guard` with `AgainstNull` and `AgainstNullOrEmpty`.
**Success criteria:** No instance members; throws correct exception types.

### E2. Lazy feature flags
**Goal:** Load feature names from a text file once using `static readonly Lazy<HashSet<string>>`.
**Success criteria:** File read happens at most once; `IsEnabled(name)` is O(1).

## Medium

### M1. Replace static clock
**Goal:** Refactor a service using `DateTime.UtcNow` to use injected `IClock`.
**Steps:** Define `IClock`, `SystemClock`, fake `FixedClock` for tests.
**Success criteria:** Unit test asserts behavior at fixed time without changing system clock.

### M2. Generic counter
**Goal:** Implement `CallCounter<T>` with `static int Count` incremented in `RecordCall()`.
**Success criteria:** Demonstrate `CallCounter<int>.Count` ≠ `CallCounter<string>.Count`.

## Hard (stretch)

### H1. Fix web anti-pattern
**Goal:** Replace `public static int CurrentUserId` with ASP.NET-appropriate design.
**Options:** scoped service, `IHttpContextAccessor`, or `AsyncLocal<int>` with explanation.
**Success criteria:** Two parallel logical "requests" do not overwrite each other's user id.
