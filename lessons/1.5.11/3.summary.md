# Memory leaks — events, static — краткая выжимка

> `1.5.11` · ~10 мин · **глубоко**

## Главная идея

Managed **leak** — объект остаётся **reachable** из GC root (static, event, singleton cache), хотя бизнес-логика считает его мёртвым. GC работает **правильно** — проблема в **strong references**.

## Ключевые пункты

- **Event `+=`**: delegate → **target**; long-lived publisher держит short-lived subscriber.
- **Static** fields/events — roots на весь process.
- **Lambda** без сохранённого delegate — нельзя `-=`.
- **Singleton + scoped** — captive dependency; `IServiceScopeFactory`.
- **Timer**/background callback captures `this` — Dispose timer.
- Fix: **`-=`**, bounded cache, scoped state, heap snapshot diff.

## Самые важные примеры

```csharp
AppEvents.UserLoggedOut += OnLogout;
// must: AppEvents.UserLoggedOut -= OnLogout;
```

## «Под капотом»

MulticastDelegate invocation list — strong refs к каждому target. Static event field регистрируется как permanent root.

## Частые ошибки

- Static «global» events без lifecycle.
- Singleton список всех request objects.
- Надежда на GC вместо unsubscribe.

## Мнемоника

> **+= is a contract — who will -=?**
