# Expression Trees — краткая выжимка

> `1.3.12` · ~8 мин на повторение

## Главная идея

**Expression tree** — дерево объектов (**`Expression`**), описывающее код (predicate, projection), **без** его выполнения в CLR. **`IQueryable`** pipeline накапливает **`MethodCallExpression`** nodes; **`IQueryProvider.Execute`** (EF) переводит tree в **SQL**.

## Ключевые пункты

- **`Func<T,bool>`** — compiled delegate, local run.
- **`Expression<Func<T,bool>>`** — data for provider; same lambda syntax, different compile target.
- Nodes: **`ParameterExpression`**, **`MemberExpression`**, **`BinaryExpression`**, **`MethodCallExpression`**, …
- Each **`Queryable.Where/Select/OrderBy`** wraps previous **`Expression`**.
- **Translatable** subset → SQL; local methods / arbitrary C# → fail or client eval (avoid).
- **`expr.Compile()`** → delegate — только после **`AsEnumerable()`**, не для EF.
- Debug: **`ToQueryString()`**, read which node failed in exception.

## «Под капотом» в одном абзаце

Roslyn builds **`Expression.Lambda`** when target is **`Expression<TDelegate>`**. EF **`QueryCompiler`** visits tree, maps **`Queryable.*`** calls to relational operators, generates parameterized SQL. **`EnumerableQuery`** on **`List.AsQueryable()`** compiles same API to in-memory **`Func`** — test harness without PostgreSQL.

## Частые ошибки

- Думать, что lambda в EF = «обычная функция»
- **`Compile()`** before provider
- Dynamic SQL strings вместо **`Expression`** composition (injection + unmaintainable)

## Мнемоника

> **Func = run now in C#** · **Expression = describe for translator**
