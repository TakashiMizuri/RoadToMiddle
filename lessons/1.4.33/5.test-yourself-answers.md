# Answers: SemaphoreSlim vs lock

## Scoring

- 90–100%: отлично · 80–89%: pass · <80%: повтори lection + summary

## Answers

### 1. [Recall]
**Answer:** `lock` thread-bound; `await` освобождает поток — нарушается инвариант Monitor.
**Why:** CS1996 language rule.

### 2. [Recall]
**Answer:** `WaitAsync` (с опциональным `CancellationToken`).
**Why:** Async-compatible wait.

### 3. [Understanding]
**Answer:** `lock` re-entrant на том же потоке; `(1,1)` не re-entrant — повторный wait без release deadlock.
**Why:** Different mutex semantics.

### 4. [Understanding]
**Answer:** Блокирует pool thread на I/O → starvation других async операций (`1.4.10`, `1.4.14`).
**Why:** Sync-over-async under lock.

### 5. [Application]
**Answer:** `await sem.WaitAsync(ct); try { ... } finally { sem.Release(); }`
**Why:** Standard async mutex.

### 6. [Application]
**Answer:** Короткое обновление in-memory структуры без await/I/O.
**Why:** lock efficient for sync.

### 7. [Analysis]
**Answer:** Self-deadlock на втором WaitAsync.
**Why:** No re-entrancy.

### 8. [Understanding]
**Answer:** Ограничить параллельные исходящие вызовы до 8 — backpressure (`1.4.18`).
**Why:** Throttle not exclusive.

### 9. [Analysis]
**Answer:** Нет — CS1996.
**Why:** await in lock prohibited.

### 10. [Application]
**Answer:** Пока ждали lock, другой поток мог заполнить cache — избегаем лишний DB hit.
**Why:** Classic double-checked locking.

### 11. [Understanding]
**Answer:** Нет — только in-process; нужен distributed lock.
**Why:** Semaphore is local.

### 12. [Recall]
**Answer:** В `finally` после успешного `Wait`/`WaitAsync`.
**Why:** Prevent semaphore leak.

### 13. [Analysis]
**Answer:** Нужно `Dispose` неиспользуемых semaphores; иначе handle/memory leak.
**Why:** Per-key lifecycle.

### 14. [Understanding]
**Answer:** Immutable snapshot / `Interlocked.Exchange` reference на новую dictionary.
**Why:** Lock-free reads.

### 15. [Application]
**Answer:** Убрать lock; `await WaitAsync`; await HTTP внутри try; Release в finally; убрать GetResult.
**Why:** Async all the way.
