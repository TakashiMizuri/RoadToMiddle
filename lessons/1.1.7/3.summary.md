# Boxing/unboxing — краткая выжимка

> `1.1.7` · ~10 мин на повторение

## Главная идея

**Boxing** — value type копируется в новый **object на heap**; variable типа `object`/interface получает **reference** на box. **Unboxing** — cast к **точному** value type и копия обратно. Связь с `1.1.6`: boxing — главный способ «положить value type на heap».

## Ключевые пункты

- **`object o = 42;`** — implicit boxing.
- **`ArrayList.Add(42)`** — boxes; **`List<int>.Add(42)`** — нет storage boxing.
- **Interface variable от struct/primitive** (`IComparable c = 10`) — boxing.
- **Enum → object** — boxing underlying value.
- **`int? null` → object** — **`null` reference**, box не создаётся.
- Unbox: **`(int)o`**, не **`(long)o`** для int box.
- IL: инструкции **`box`**, **`unbox.any`** + load.

## Самые важные примеры

```csharp
object o = 42;              // box
int n = (int)o;             // unbox
IComparable c = 100;        // box
var list = new List<int>(); // no per-element box
int? x = null;
object p = x;               // p == null
```

## «Под капотом» в одном абзаце

Roslyn emit **`box`** с type token: allocate object, copy value в payload, push ref. Unbox — **`unbox.any`** проверяет type handle, **`ldind`** копирует bits. Каждый box — Gen0 allocation. Generics JIT специализирует storage для `T` struct без conversion к `object`.

## Частые ошибки

- Non-generic collections для value types в hot loops.
- API с единственным `object` parameter.
- Неверный тип при unbox.

## Мнемоника

> **Box — value залезает в object на heap; unbox — достаёшь тем же типом, что положил.**
