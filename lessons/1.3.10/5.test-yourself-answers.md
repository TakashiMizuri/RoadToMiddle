## Answers

### 1. [Recall]
**Answer:** **Immediate** (terminal / materializing).
**Why:** Forces full enumeration into **`List<T>`**.

### 2. [Recall]
**Answer:** **`Any()`** can **short-circuit** at first element; **`Count()`** must visit all (unless special **`ICollection`** fast path without **`Where`** wrapper).
**Why:** Performance on large sequences.

### 3. [Understanding]
**Answer:** Copies current query results into new **`List`** at execution time; decoupled from later mutations of source **`users`**.
**Why:** Materialization = snapshot (`1.3.9`).

### 4. [Application]
**Answer:** **`.ToList()`** at end: `users.Where(u => u.IsActive).ToList()` ( **`OrderBy`** before if needed).
**Why:** Single terminal materialization.

### 5. [Analysis]
**Answer:** **Два полных прохода** (два вызова **`Count()`** на deferred **`q`**).
**Why:** Each terminal re-enumerates.

### 6. [Application]
**Answer:** **`FirstOrDefault`** if missing user is normal (404); **`First`** if must exist or exceptional.
**Why:** Throw semantics vs nullable/default.

### 7. [Understanding]
**Answer:** Serializer/context may enumerate after **`DbContext`** disposed; execution undefined; security (unbounded query). Materialize inside scope with **`ToListAsync`**.
**Why:** Boundary + lifetime.

### 8. [Recall]
**Answer:** Both materialize fully; **`List`** growable, common API type; **`Array`** fixed size, slightly less overhead in some cases.
**Why:** Different collection types, same enumeration cost.

### 9. [Application]
**Answer:** After **`Take`**: `.Skip(...).Take(...).ToList()` — one enum produces page.
**Why:** Terminal at boundary of page.

### 10. [Analysis]
**Answer:** **`list.Count`** — **O(1)** property. **`list.Where(...).Count()`** — **O(n)** scan with filter.
**Why:** **`Where`** wrapper prevents **`ICollection.Count`** fast path on filtered view.
