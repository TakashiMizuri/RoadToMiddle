# Exercises: LOH and Fragmentation

## Easy
### E1. Threshold experiment
**Goal:** Verify LOH boundary.
**Steps:** Allocate `byte[84999]` and `byte[85000]`; print `GC.GetGeneration` for each (single-object test).
**Success criteria:** Gen0 vs Gen2 difference observed.

## Medium
### M1. ArrayPool refactor
**Goal:** Reduce LOH churn.
**Steps:** Refactor method that `new byte[200_000]` each call to `ArrayPool<byte>.Shared.Rent/Return` with try/finally.
**Success criteria:** Return in finally; document clear before sensitive data if needed.

## Medium
### M2. Upload endpoint design
**Goal:** Production LOH awareness.
**Steps:** Design file upload API handling 50MB files without holding full buffer in LOH — streaming outline.
**Success criteria:** Stream to disk/cloud; no single 50MB byte[].

## Hard
### H1. LOH monitoring
**Goal:** Ops literacy.
**Steps:** List dotnet-counters or PerfView metrics to watch LOH size trend over 7-day service.
**Success criteria:** `loh-size` or equivalent named.
