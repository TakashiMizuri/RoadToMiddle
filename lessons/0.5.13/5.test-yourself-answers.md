## Answers

### 1. [Recall]
**Answer:** В `<head>`.
**Why:** Metadata документа объявляется до body; crawlers и браузер читают head первым для title, linked resources и machine-readable hints.

### 2. [Recall]
**Answer:** `property` (например `property="og:title"`).
**Why:** Open Graph следует RDFa-конвенции; `name` на `og:` tags ломает совместимость с частью parsers.
**Common mistake:** Копировать синтаксис `meta name="description"` на OG-теги.

### 3. [Recall]
**Answer:** `og:title`, `og:description`, `og:image`, `og:url` (часто также `og:type`).
**Why:** Этого набора достаточно для базовой link card с заголовком, текстом, картинкой и URL.

### 4. [Understanding]
**Answer:** Google и другие engines могут сгенерировать snippet из body, если meta irrelevant, duplicate или пустой.
**Why:** Snippet оптимизируется под запрос пользователя; meta description — лишь один из источников.

### 5. [Understanding]
**Answer:** Canonical на «чистый» URL (`https://example.com/post`) консолидирует ranking signals и избегает duplicate content на UTM-варианте.
**Why:** Crawlers иначе могут индексировать оба URL отдельно и размывать link equity.

### 6. [Understanding]
**Answer:** Social fetchers часто не выполняют JavaScript; видят только initial HTML без OG или с дефолтными тегами layout.
**Why:** Preview bots делают быстрый GET и парсят `<head>`, не дожидаясь React bundle.

### 7. [Application]
**Answer:** Пример:
```html
<title>Pricing — SaaSApp</title>
<meta name="description" content="Compare Starter, Pro, and Enterprise plans with monthly and annual billing.">
<link rel="canonical" href="https://saasapp.io/pricing">
```
**Why:** Уникальный title, human description, absolute canonical — минимальный SEO-каркас страницы.

### 8. [Application]
**Answer:** Выбрать один authoritative host (например `https://shop.com/p/42` если это production standard) и указать его в canonical на **обеих** версиях. Absolute URL нужен, чтобы crawlers однозначно резолвили preferred URL без относительной неоднозначности.
**Why:** Relative canonical (`/p/42`) на разных host даёт разные интерпретации; absolute фиксирует выбор.

### 9. [Analysis]
**Answer:** Crawlers не различают страницы по title; snippets дублируются; низкий CTR; плохой UX во вкладках и закладках; упущенная релевантность по long-tail запросам.
**Why:** Title — сильный сигнал «о чём страница»; идентичность на 200 URL = потеря granularity.

### 10. [Understanding]
**Answer:** Canonical — hint при доступных дублях (200 OK на обоих); 301 — permanent move, пользователь и bot попадают на новый URL. Redirect сильнее для users; canonical когда нужны tracking URLs.
**Why:** Разные HTTP semantics и strength сигнала для consolidatioн.

### 11. [Analysis]
**Answer:** Платформа резолвит relative URL относительно своего origin или ломает preview; картинка 404 или wrong host → пустая/битая card.
**Why:** OG spec и platform docs требуют absolute URL для `og:image`.

### 12. [Understanding]
**Answer:** Прямой ranking weight минимален или отсутствует; косвенно — через CTR и engagement (bounce), если description misleading или привлекательный.
**Why:** Google публично de-emphasizes description as ranking factor; ценность в snippet quality.
