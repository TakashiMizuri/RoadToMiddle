# ArrayPool — rent/return pattern

> Roadmap: `1.5.14` · Node: `1.5` — C#: .NET Runtime и память · Depth: понимание

## Learning Objectives

- **`ArrayPool<T>.Shared`**: Rent → use → Return.
- **`clearArray`**, exact length vs **`.Length >= minimumLength`**.
- Anti-patterns: use-after-return, double return, field storage.
- Выбор: ArrayPool vs stackalloc vs `new`.

---

## Why This Matters

Hot path `new byte[4096]` → Gen0 pressure. **ArrayPool** reuse. Wrong Return → data leaks между requests или pool starvation.

---

## Core Concepts

```csharp
byte[] buffer = ArrayPool<byte>.Shared.Rent(512);
try
{
    Process(buffer.AsSpan(0, logicalLength));
}
finally
{
    ArrayPool<byte>.Shared.Return(buffer, clearArray: true);
}
```

**Rent** может вернуть **больший** array. **Return** только rented reference, один раз, в **finally**.

**clearArray: true** для sensitive data.

**Shared** vs **Create** — обычно Shared.

---

## Key Takeaways

- Rent/Return пара в finally.
- Не хранить rented array в fields.
- Span over rented: **`1.5.12`**.
