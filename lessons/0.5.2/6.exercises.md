# Exercises: Semantic Page Layout

> Complete after reading lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Blog shell
**Goal:** Build a static blog post page structure.

Create `post.html` (reuse charset/lang skeleton from `0.5.1`) with:
- Site header: site name link + primary nav (Home, Blog, About)
- One `<main>` containing one `<article>`
- Article header with `<h1>`, article footer with "Author: You"
- Page footer with copyright year

No CSS required.

**Success criteria:** Exactly one `main`; nav inside header; article inside main.

---

### E2. Landmark audit
**Goal:** Identify missing semantics.

Given div-only markup:

```html
<div id="top">...</div>
<div id="menu">...</div>
<div id="content">...</div>
<div id="sidebar">...</div>
<div id="bottom">...</div>
```

Rewrite using semantic tags (header, nav, main, aside, footer). Explain each mapping in one sentence.

**Success criteria:** Five regions mapped; one main.

---

## Medium

### M1. News index
**Goal:** Multiple articles in one main.

Create `news.html` with `<main>` containing **three** `<article>` elements. Each article: `<h2>`, teaser `<p>`, "Read more" link.

**Success criteria:** Three articles; no nested main; page-level header/footer outside main.

---

### M2. Docs page with sections
**Goal:** article + section hierarchy.

One `<main>`, one `<article>` titled "Deployment Guide". Inside article, three `<section>` elements each with `<h2>` and paragraph. Add `<aside>` with "See also" links.

**Success criteria:** Every section has h2; aside outside article OK (sibling under body or next to main).

---

## Hard (stretch)

### H1. SPA layout critique
**Goal:** Production thinking.

Pseudocode layout:

```jsx
<Header />
<SidebarNav />
<div className="page">
  <main>...</main>
  <main>...</main>
</div>
```

List accessibility problems and sketch a corrected component tree (HTML tags, not full React).

**Success criteria:** Names duplicate main issue; proposes single main wrapper for route content.
