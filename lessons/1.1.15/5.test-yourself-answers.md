## Answers

### 1. [Recall]

**Why:** List — wrapper над array с grow semantics.

### 2. [Recall]

**Why:** Indexer не различает miss vs null value without Try pattern.

### 3. [Recall]

### 4. [Understanding]
**Answer:** **Один** hash lookup vs **два** (Contains + get).
**Why:** Performance + atomicity of check.

### 5. [Understanding]
**Answer:** Изменение полей key меняет **hash code** → entry «теряется» в bucket.
**Why:** Hash table assumes immutable key identity.

### 6. [Understanding]
**Answer:** List.Contains **O(n)**; Dictionary.ContainsKey **O(1)** average.
**Why:** Linear scan vs hash lookup.

### 7. [Application]
**Answer:** `if (d.TryGetValue(999, out string? name)) ... else ...` или проверка bool.
**Why:** No exception on miss.

### 8. [Application]
**Answer:** `new HashSet<string>(list).Count` или add in loop to set then Count.
**Why:** HashSet enforces uniqueness.

### 9. [Analysis]
**Answer:** **`InvalidOperationException`** — collection modified during enumeration.
**Why:** foreach invalidates on structural change; use for backwards or RemoveAll.

### 10. [Analysis]
**Answer:** **`Dictionary<int, User>`** — frequent lookup by id needs O(1) average not O(n) scan.
**Why:** Scale + access pattern.

### 11. [Application]

### 12. [Understanding]
**Answer:** **Pre-allocate capacity** — меньше reallocations/copy при последующих Add.
**Why:** Same as array list growth optimization.

**Why:** "A" ≠ "a" unless OrdinalIgnoreCase comparer passed.
