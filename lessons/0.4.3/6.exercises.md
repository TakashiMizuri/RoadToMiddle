## Easy

**Goal:** Memorize SYN/SYN-ACK/ACK.

Draw three-way handshake with seq/ack labels (use x, y). Mark ESTABLISHED point.

**Success criteria:** Three arrows; both ISNs acknowledged.

## Medium

### M1. curl timing experiment
**Goal:** Separate TCP from TLS from TTFB.

Run twice:
```bash
curl -w "connect:%{time_connect} appconnect:%{time_appconnect} ttfb:%{time_starttransfer}\n" -o NUL -s https://example.com
curl -w "...\n" -o NUL -s http://example.com
```

Compare `time_connect` vs `time_appconnect` for HTTPS. What adds the delta?

**Success criteria:** Identifies TLS between TCP connect and appconnect.

---

### M2. Keep-Alive observation
**Goal:** Link `0.3.14` to wire behavior.

Use browser DevTools Network → single origin, two GETs. Check **Connection** column / protocol. Same TCP connection ID (Chrome) or `Connection: keep-alive`?

Write one paragraph: what would double if Keep-Alive disabled?

**Success criteria:** Mentions extra handshake (and TLS for HTTPS).

## Hard (stretch)

### H1. Zero window scenario
**Goal:** Flow control reasoning.

ASP.NET endpoint reads upload stream 1 KB/s; client uploads 10 MB JSON at 50 MB/s. Describe TCP behavior at sender when receiver window hits 0. What should app developer do?

**Success criteria:** Sender pauses; mentions async read, streaming, or increasing buffer — not "increase TCP window" manually.
