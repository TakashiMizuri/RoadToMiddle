# Exercises: Rebase

> Complete after lectures and passing `4.test-yourself.md`.

## Easy

### E1. Rebase vocabulary
Match term to definition: (a) replay onto new base, (b) edit commit list in editor, (c) cancel in-progress operation, (d) safe force push.

**Success criteria:** 4/4 matched.

---

## Medium

### M1. Interactive squash
In a test repo, create 3 commits on `feature`. Use `git rebase -i HEAD~3` to squash into one message `feat: combined work`. Verify with `git log -1`.

**Success criteria:** Single commit on feature; message correct.

---

### M2. Golden rule scenario
**Scenario:** `main` pushed to origin. Two devs branched. Dev A rebases local `main` and force-pushes. Dev B has old `main`.

Write what Dev B should **not** do blindly, and two safe recovery options.

**Success criteria:** Mentions force-push rewrite; suggests fetch + reset/rebase to origin/main or new branch.

---

## Hard

### H1. Rebase with conflict
Create divergent `main` and `feature` touching same line. Rebase feature onto main; resolve conflict; continue. Document commands.

**Success criteria:** Clean rebase; feature linear on main.
