# Skip links, focus visible, color contrast

> Roadmap: `0.5.12` · Node: `0.5` — HTML · Depth: понимание

## Learning Objectives

После этого урока ты сможешь:

- Реализовать **skip link** с переходом focus на main content.
- Стилизовать **focus indicators** через `:focus-visible`.
- Объяснить **WCAG contrast** для текста и UI components на практическом уровне.
- Избегать типичных провалов: outline none, низкий contrast placeholder, смысл только цветом.
- Связать skip links с landmarks (`<main id="main">`).
- Проверять contrast и focus в DevTools и axe.

---

## Why This Matters

Tab через dashboard: logo, nav, search, notifications — и только потом таблица. **Skip links** дают keyboard и screen reader users прыжок к **`#main`**. Дёшево добавить, высокий impact. В `0.5.11` — focus в modals; skip link — document-level аналог, часто первый Tab stop на gov/SaaS sites.

**Focus visible** — keyboard users видят **focus ring**. `outline: none` без замены — провал **WCAG 2.4.7**. **Color contrast** — измеримая читаемость для low vision и sunlight. Вместе с skip links — практический a11y baseline перед SEO в `0.5.13`.

## Core Concepts

### Skip links

Первый focusable элемент — anchor на id main content:

```html
<a class="skip-link" href="#main">Skip to main content</a>
<main id="main" tabindex="-1">...</main>
```

CSS: visually hidden до `:focus`, then visible high-contrast bar. **`tabindex="-1"`** на main — focus **переезжает** на landmark после активации, не только scroll.

Несколько skip targets допустимы на complex apps; первый Tab — usually skip to main.

### Focus visible

`:focus` — любой focus. `:focus-visible` — когда UA считает indicator нужным (обычно keyboard Tab).

```css
:focus-visible {
  outline: 2px solid #005fcc;
  outline-offset: 2px;
}
```

Не оставляй controls без visible focus. `:focus-within` для composite fields. Modal trap из `0.5.11` всё равно needs visible ring inside.

### Color contrast

**Contrast ratio** foreground/background. **Level AA**:

| Контент | Минимум |
|---------|---------|
| Normal text | **4.5:1** |
| Large text | **3:1** |
| UI components / graphics | **3:1** |

Tools: DevTools Accessibility pane, WebAIM checker, axe. **Не только цвет** для state — text/icons (`0.5.9`). Placeholder ≠ label, часто fails contrast.

### Связка

Landmarks + skip + focus-visible + contrast — complementary, не взаимозаменяемые.

## Under the Hood

`prefers-reduced-motion`, forced colors (Windows High Contrast) — focus must remain visible. Chromium `:focus-visible` heuristics — mouse vs keyboard. Contrast — **relative luminance** sRGB; alpha on images ловушка. Skip — fragment navigation + focus API.

---

## Syntax / Commands / API

Skip link + main tabindex=-1 + :focus-visible styles. DevTools contrast ratio.

## Examples

Gov-style skip — high contrast on focus. Failed global outline none. Gray-on-gray text fail 4.5:1.

---

## Common Mistakes & Anti-patterns

No skip on nav-heavy site. Skip target без tabindex. outline none без replacement. Color-only errors. Low-contrast disabled без alternative cue.

---

## Production & Real-World Notes

Design tokens: focus ring color 3:1 adjacent. Storybook first Tab test. SPA — stable main id. Audits: home + form page.

---

## Comparison / Trade-offs

Hidden-until-focus vs always visible skip. AA vs AAA. Automated contrast vs keyboard skip test.

## Key Takeaways

- Skip link — первый Tab к **main**.
- **tabindex="-1"** на main для focus after skip.
- **:focus-visible** вместо naked outline removal.
- **4.5:1** normal text AA; **3:1** large/UI.
- State не только цветом.
- DevTools + keyboard test.
