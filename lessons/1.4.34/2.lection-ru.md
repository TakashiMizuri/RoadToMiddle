# BackgroundTaskQueue в ASP.NET — channel pattern

> Roadmap: `1.4.34` · Node: `1.4` — C# async/await · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Реализовать **`IBackgroundTaskQueue`** на **`System.Threading.Channels`**.
- Хостить **`BackgroundService`**, который **await'ит** каждый item с **scoped DI**.
- Заменить небезопасный fire-and-forget (`1.4.30`) на **bounded backpressure**.
- Пробрасывать **correlation** и **idempotency** в делегаты.
- Корректно останавливаться через **`CancellationToken`** и complete channel.

---

## Why This Matters

Controller должен отвечать быстро; email и webhooks — **вне** request thread, но **внутри** процесса — с **observed exceptions**, **свежими scopes** и **ограниченной памятью**. Паттерн **очередь + BackgroundService** — стандартный ответ на ловушки fire-and-forget.

**`Channel<T>`** (`1.4.17`) — producer/consumer с **bounded capacity**, чтобы пики не создавали бесконечные closures.

## Core Concepts

### Роли

- **`IBackgroundTaskQueue`** — enqueue work delegate.
- **`Channel`** — буфер HTTP ↔ worker.
- **`QueuedHostedService`** — read → scope → await.
- **`IServiceScopeFactory`** — новый scope на item.

### Channel (bounded)

```csharp
var options = new BoundedChannelOptions(capacity)
{
    FullMode = BoundedChannelFullMode.Wait
};
```

**`Wait`** — backpressure: producer `await` при полной очереди.

### Consumer

```csharp
await using var scope = _scopeFactory.CreateAsyncScope();
await work(scope.ServiceProvider, stoppingToken);
```

**`CreateAsyncScope()`** для **`IAsyncDisposable`** (`1.4.35`). Полный **await** — exceptions observed (`1.4.31`).

### Controller

```csharp
await _queue.QueueBackgroundWorkItemAsync(async (sp, ct) =>
{
    await sp.GetRequiredService<IEmailSender>().SendAsync(orderId, ct);
});
return Accepted();
```

В closure — **`orderId`**, не **`DbContext`**.

## Under the Hood

**Singleton** queue; **scoped** resolve внутри work. Один consumer — serial; N readers — parallelism + ordering trade-offs. Shutdown: **`stoppingToken`**, опционально **`Writer.Complete()`**. In-process ≠ durable broker.

---

## Common Mistakes & Anti-patterns

Scoped service в closure снаружи. Unbounded channel под атакой. Один долгий job блокирует single consumer.

---

## Quick Reference

- Singleton queue + HostedService.
- **Scope per item** через factory.
- **Bounded** + **Wait**.
- Capture IDs only.
- Await + try/catch в consumer.

---

## Key Takeaways

- Channel queue — structured замена `_ =`.
- Scoped DI на каждый job.
- Bounded = backpressure.
- Await предотвращает unobserved.
- Must-deliver → outbox/broker.
