# Answers: StringBuilder vs concatenation

## Scoring

- 90–100%: отлично · 80–89%: pass · <80%: lection + summary + **1.1.4**

## Answers

### 1. [Recall]
**Answer:** **`string` immutable** (`1.1.4`) — нельзя изменить существующий объект; `+=` создаёт **новую** string с combined content.
**Why:** Old string остаётся для GC.

### 2. [Recall]
**Answer:** **`ToString()`**.
**Why:** Copies buffer to new immutable string instance.

### 3. [Recall]
**Answer:** **Нет** — нужен separate instance per thread или lock.
**Why:** Internal buffer mutated on Append.

### 4. [Understanding]
**Answer:** Каждая итерация копирует **весь накопленный** s плюс новый символ в новый string; суммарно ~1+2+…+n символов → **O(n²)**.
**Why:** Immutable intermediate on every step.

### 5. [Understanding]
**Answer:** Когда есть **`IEnumerable<string>`** / array — Join вычисляет длину (often) и делает **one** result allocation + single copy pass.
**Why:** BCL optimized path; clearer code.

### 6. [Understanding]
**Answer:** **Pre-size internal buffer** — меньше resize/realloc при Append до финального размера.
**Why:** Same strategy as List capacity.

### 7. [Application]
**Answer:**
```csharp
var sb = new StringBuilder();
foreach (var line in lines)
    sb.AppendLine(line);
string result = sb.ToString();
```
**Why:** Append in place; one ToString at end.

### 8. [Application]
**Answer:** **`string.Join(",", names)`** or **`string.Concat`** with separators manually / `$"{a},{b},{c}"` for fixed three.
**Why:** No loop += .

### 9. [Analysis]
**Answer:** **StringBuilder не нужен** — две-три части, одна concat operation (readable).
**Why:** Overhead builder > single Concat.

### 10. [Analysis]
**Answer:** **`ToString()` внутри loop** — каждая итерация full copy buffer → string + println; builder benefit lost.
**Why:** Should Append only; ToString once after loop.

### 11. [Understanding]
**Answer:** Intern pool хранит **literal/Intern** strings; **`a + b` dynamic result не intern'ится** автоматически — новый object в heap (unless compiler constant folding for literals only).
**Why:** Pool for reuse of known constants, not all concat.

### 12. [Application]
**Answer:** Один megastring — **high memory / LOH**; stream — **lower peak memory**, better for response pipeline; string OK only if size bounded and needed in memory.
**Why:** Production HTTP often streams.

### 13. [Analysis]
**Answer:** **`Concat(a,b,c)`** typically **one** allocation; chained **`+=`** may create **intermediate** strings (compiler may optimize simple cases but not guaranteed for all).
**Why:** Prefer Concat/Join for clarity and known alloc profile.

### 14. [Understanding]
**Answer:** **Grow buffer** — new chunk or larger array, copy existing content (strategy implementation-dependent).
**Why:** Mutable expandable storage.

### 15. [Application]
**Answer:** **Мало подстановок**, один раз, readable template — interpolation идеален; не loop, не hundreds parts.
**Why:** Compiler generates efficient handler for small interpolation.
