# git bisect — краткая выжимка

> `0.2.12` · ~7 мин на повторение

## Главная идея

**`git bisect`** — двоичный поиск **first bad commit** между known-good и known-bad. Git checkout'ит середины; ты или script помечаете good/bad. ~`log₂(N)` шагов вместо N.

## Ключевые пункты

- **`git bisect start [bad] [good]`** — начало.
- **`git bisect bad` / `good`** — сузить диапазон.
- **`git bisect reset`** — выход из bisect, вернуться на ветку.
- **`git bisect run <cmd>`** — 0=good, 1–124=bad, 125=skip.
- **`git bisect skip`** — commit не тестируется (broken build).
- Good должен быть **ancestor** bad.
- Результат: **first bad commit** → `git show` → fix/revert.

## Самые важные примеры

```bash
git bisect start main v2.4.0
# test → git bisect good|bad
git bisect reset
```

```bash
git bisect start HEAD v2.4.0
git bisect run npm test -- --grep "Login"
git bisect reset
```

## Частые ошибки

- Uncommitted changes / flaky tests.
- Good не ancestor bad.
- Забытый `bisect reset` (detached HEAD).
- Слишком медленный test в `run`.

## Мнемоника

> **Good** = было OK · **Bad** = баг есть · **Reset** = закончил игру · Steps ≈ **log₂** commits.
