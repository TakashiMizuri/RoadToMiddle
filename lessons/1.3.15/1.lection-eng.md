# Attributes and Reflection (GetCustomAttribute)

> Roadmap: `1.3.15` · Node: `1.3` — C# Advanced Topics · Depth: understanding

## Learning Objectives

After this lesson you will be able to:

- Explain what **attributes** are: metadata attached to assemblies, types, members, parameters.
- Apply common BCL attributes (`[Obsolete]`, `[Serializable]`, validation, ASP.NET routing).
- Read attributes at runtime with **`GetCustomAttribute`** / **`GetCustomAttributes`**.
- Distinguish **compile-time** consumers (analyzers, source generators) from **runtime** reflection.
- Understand **performance cost** of reflection and when to cache `MemberInfo`.
- Use reflection basics: `typeof`, `GetMethod`, `Invoke` — with safety caveats.

---

## Why This Matters

Much of the .NET ecosystem is **declarative**: you annotate code, frameworks interpret annotations. ASP.NET Core maps `[HttpGet]` to routes; EF configures columns with `[Column]`; System.Text.Json honors `[JsonPropertyName]`; validation uses `[Required]`. None of this works by magic — something **reads metadata** at compile time or runtime.

**Attributes** are the language feature for that metadata. **Reflection** is the API to inspect types and read attributes when the program runs. Middle developers configure libraries via attributes daily; debugging «why is this endpoint 404» often means tracing attribute discovery.

You rarely write heavy reflection in business code, but you must understand it to use frameworks correctly and to avoid ** brittle stringly-typed** APIs.

## Core Concepts

### What Is an Attribute?

An attribute is an instance of a class inheriting **`System.Attribute`**, attached in square brackets **before** a target:

```csharp
[Obsolete("Use NewApi instead")]
public void OldMethod() { }
```

Targets include **assembly**, **class**, **method**, **property**, **parameter**, **return value**, etc. The compiler stores attribute data in **assembly metadata** (IL tables), not as executable statements in the method body.

Attribute classes typically use **`[AttributeUsage(...)]`** to restrict where they apply and whether multiple instances are allowed.

### Defining a Custom Attribute

```csharp
[AttributeUsage(AttributeTargets.Property, AllowMultiple = false)]
public sealed class MaxLengthAttribute : Attribute
{
    public int Length { get; }
    public MaxLengthAttribute(int length) => Length = length;
}
```

Usage:

```csharp
public class UserDto
{
    [MaxLength(100)]
    public string Name { get; set; } = "";
}
```

Properties on the attribute class must be **simple** (constants, typeof, array of constants) — the compiler embeds constructor arguments and named properties into metadata.

### Reading Attributes: GetCustomAttribute

At runtime, obtain **`Type`**, **`MethodInfo`**, **`PropertyInfo`**, etc., then query attributes:

```csharp
var prop = typeof(UserDto).GetProperty(nameof(UserDto.Name))!;
var attr = prop.GetCustomAttribute<MaxLengthAttribute>();
int max = attr?.Length ?? int.MaxValue;
```

APIs:

- `GetCustomAttribute<T>()` — single or null.
- `GetCustomAttributes<T>()` — collection.
- `inherit: true/false` — whether to walk inheritance chain (default varies by overload).

Modern overloads use **`CustomAttributeData`** for lightweight inspection without instantiating attribute objects (used by serializers and DI scanners).

### Reflection Basics

**Reflection** inspects types and members at runtime:

```csharp
Type t = typeof(UserDto);
object obj = Activator.CreateInstance(t)!;
PropertyInfo? nameProp = t.GetProperty("Name");
nameProp?.SetValue(obj, "Ada");
```

`MethodInfo.Invoke` calls methods; `GetMethods`, `GetProperties` enumerate members. Generics add `MakeGenericType`, `GetGenericArguments`.

Reflection is **powerful and slow** relative to direct calls — JIT cannot inline `Invoke` easily. Frameworks cache `MemberInfo` and compiled delegates (`CreateDelegate`) for hot paths.

### Compile-Time vs Runtime Consumers

| Consumer | When | Example |
|----------|------|---------|
| Compiler | Build | `[Conditional("DEBUG")]`, nullable analysis |
| Source generator | Build | `[JsonSerializable]` → generated code |
| Analyzer/Roslyn | IDE/build | `[Required]` hints |
| Runtime reflection | Execute | ASP.NET endpoint discovery, validation |

Many modern APIs **prefer source generators** over runtime reflection for AOT and performance (.NET Native AOT trims reflection heavily).

### Common BCL / Framework Attributes

- **`[Obsolete]`** — warnings/errors for deprecated API.
- **`[Flags]`** on enum — bitwise combo semantics.
- **`[Serializable]`** — legacy binary serialization (largely deprecated).
- **ASP.NET:** `[ApiController]`, `[Route]`, `[Authorize]`, `[FromBody]`.
- **Validation:** `[Required]`, `[Range]`, `[EmailAddress]`.
- **EF Core:** `[Key]`, `[Table]`, Fluent preferred in new code but attributes exist.

Understanding **where** each is consumed prevents «attribute does nothing» surprises (wrong target, wrong namespace, missing framework registration).

## Under the Hood

Attributes compile to **`.custom`** directives in IL attached to metadata tokens. Constructor args are stored; properties set via named args are encoded separately.

`GetCustomAttribute<T>()` loads the attribute type, creates an instance via **constructor + property setters** (unless using `CustomAttributeData` only). **Inheritance:** `[AttributeUsage(Inherited = true)]` on attribute class controls whether derived classes inherit the annotation on overridden members (subtle rules for properties vs methods).

Reflection uses **`RuntimeType`** objects cached per `AppDomain`. **`BindingFlags`** control public/nonpublic, static/instance, declared-only vs flattened hierarchy.

Trimming/AOT: unused reflection targets may be **removed** — annotate with `[DynamicallyAccessedMembers]` or use `RequiresUnreferencedCode` patterns when publishing trimmed apps.

## Examples

### Mini validation scanner

```csharp
public static List<string> Validate(object dto)
{
    var errors = new List<string>();
    foreach (var prop in dto.GetType().GetProperties())
    {
        var max = prop.GetCustomAttribute<MaxLengthAttribute>();
        if (max is null) continue;
        var val = prop.GetValue(dto) as string;
        if (val?.Length > max.Length)
            errors.Add($"{prop.Name} too long");
    }
    return errors;
}
```

This mirrors what DataAnnotations validators do internally (simplified).

### Route name from attribute (conceptual)

```csharp
[HttpGet("/users/{id}")]
public User GetUser(int id) { ... }

// Framework at startup reflects controller methods, finds HttpGetAttribute, builds route table.
```

## Common Mistakes & Anti-patterns

**Assuming attributes run code by themselves** — they are data unless something reads them.

**Wrong `AttributeTargets`** — attribute silently ignored if placed on invalid target (compile error if `[AttributeUsage]` enforced).

**Reflection on hot path without cache** — measure; cache `PropertyInfo` / delegates.

**Magic strings** — `GetProperty("name")` typo fails at runtime; prefer `nameof`.

**Relying on reflection in trimmed/AOT** without preservation attributes — `MissingMethodException` at runtime.

## Production & Real-World Notes

Prefer **explicit registration** (Minimal APIs `MapGet`) when reflection cost/complexity matters. Attributes shine for **large controller surfaces** and **cross-cutting** conventions.

Logging/debug: `dotnet run` startup logs often list «discovered endpoints» from reflection — useful when route missing.

Testing: reflection can set private fields in tests — acceptable in test assemblies; avoid in production code.

## Comparison / Trade-offs

| Approach | Pros | Cons |
|----------|------|------|
| Attributes | Declarative, co-located | Hidden behavior, reflection cost |
| Fluent/config code | Explicit, refactor-safe | Verbose |
| Source generators | Fast, AOT-friendly | Build complexity |

## Key Takeaways

- Attributes = **metadata** in IL, not executable logic.
- **`GetCustomAttribute<T>()`** reads metadata at runtime on `MemberInfo`.
- Frameworks (ASP.NET, JSON, EF) **consume** attributes at startup or first use.
- Reflection is flexible but **slower** — cache when repeated.
- Source generators and AOT reduce reliance on runtime reflection.
- `[AttributeUsage]` defines valid targets and inheritance rules.
