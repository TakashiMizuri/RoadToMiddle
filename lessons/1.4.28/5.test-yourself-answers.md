## Answers

### 1. [Recall]
**Answer:** **Retry**, **timeout**, **circuit breaker** (also mention fallback as optional fourth).

**Why:** Core resilience triad for HTTP.

---

### 2. [Recall]
**Answer:** **Fail fast** without calling downstream — immediate exception (e.g. BrokenCircuitException) / fallback.

**Why:** Protects sick dependency and caller resources.

---

### 3. [Understanding]
**Answer:** Randomizes delay so clients don't retry **simultaneously** after shared outage — reduces thundering herd on recovery.

**Why:** Standard distributed systems practice.

---

### 4. [Understanding]
**Answer:** Duplicate creates — **double orders**, data corruption; POST not assumed idempotent.

**Why:** Retry safety rule.

**Common mistake:** Treat all HTTP calls as retryable.

---

### 5. [Understanding]
**Answer:** Single HttpClient.Timeout applies to **entire** send including all retries — may cancel before Polly completes planned attempts OR hides per-try limits; align: often Infinite HttpClient timeout + Polly per-attempt/total timeouts.

**Why:** Layer interaction from lecture.

---

### 6. [Application]
**Answer:**
```csharp
builder.Services.AddHttpClient("catalog")
    .AddStandardResilienceHandler(options => { ... });
```
Package: `Microsoft.Extensions.Http.Resilience`.

**Why:** .NET 8 integration path.

---

### 7. [Application]
**Answer:** Retry on 503 (predicate), max 3 attempts, exponential backoff + jitter, per-attempt timeout 5s, total timeout 30s, optional circuit breaker if sustained failures, pass CancellationToken.

**Why:** Typical transient handling package.

---

### 8. [Analysis]
**Answer:** **Retry storm** — up to 400+ concurrent attempts, downstream collapse accelerates, thread pool/socket pressure, breaker may open late; fix: lower parallelism, fewer retries, breaker threshold, fix root cause.

**Why:** 1.4.25 × 1.4.28 interaction.

---

### 9. [Analysis]
**Answer:** **No** — 404 permanent (resource missing); wastes resources, masks bugs.

**Why:** ShouldHandle must exclude permanent client errors.

---

### 10. [Understanding]
**Answer:** **Probe** recovery — limited trial request(s); if success → closed; if fail → open again.

**Why:** Breaker state machine half-open role.

---

### 11. [Application]
**Answer:** **408, 429, 502, 503, 504**, network **`HttpRequestException`**; typically **not** 404, 400, 401, 403 for blind retry.

**Why:** Transient vs permanent distinction.
