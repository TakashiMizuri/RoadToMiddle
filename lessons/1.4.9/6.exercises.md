# Exercises: SynchronizationContext

## Easy

### E1. Current Context Probe
**Goal:** Small console vs minimal ASP.NET Core app: log `SynchronizationContext.Current?.GetType().FullName ?? "null"` before and after `await Task.Delay(1)`.
**Success criteria:** Console/Core null; optional WPF shows Dispatcher type.

### E2. Diagram
**Goal:** Draw UI vs Core continuation paths after await (SyncContext Post vs pool).
**Success criteria:** References `MoveNext` from `1.4.7`.

## Medium

### M1. Framework Deadlock Story
**Goal:** Write narrative (no code) explaining Framework sync-over-async deadlock using SyncContext vocabulary.
**Success criteria:** Blocked thread + Post queue never drains.

### M2. HttpContext After Await
**Goal:** Explain why `_httpContextAccessor.HttpContext` works in Core service after `await` without SyncContext.
**Success criteria:** Mentions AsyncLocal and request scope.

## Hard

### H1. Blazor Server Policy
**Goal:** Document when to use `ConfigureAwait` in Blazor Server component vs in injected library.
**Success criteria:** UI-like host rules from `1.4.8`.
