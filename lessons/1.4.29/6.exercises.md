# Exercises: CancellationToken — RequestAborted

> Complete after passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Token Wiring Audit
**Goal:** In a sample ASP.NET Core project (or sketch), trace one GET endpoint from minimal API → service → `HttpClient` call. Mark every `await` that lacks `CancellationToken`.
**Success criteria:** List gaps and show one fixed signature chain.

### E2. Linked Timeout
**Goal:** Write middleware or endpoint code that links `RequestAborted` with a 15-second `CancelAfter`.
**Success criteria:** Single `CancellationToken` passed to downstream call; `using` disposes CTS.

## Medium

### M1. Exception Middleware Filter
**Goal:** Sketch exception handler that returns no 500 for `OperationCanceledException` when `RequestAborted` is set, but still logs real failures.
**Success criteria:** `when` filter documented; other `OperationCanceledException` still handled.

### M2. EF + External API
**Goal:** Endpoint loads entity from PostgreSQL then calls external API; both honour same request token.
**Success criteria:** Two I/O boundaries with same `ct`; explain disconnect mid-second-call behavior.

## Hard (stretch)

### H1. Graceful vs Hard Cancel
**Goal:** Design endpoint that streams 1000 items via `IAsyncEnumerable`; cancellation stops enumeration without leaking DB connection.
**Success criteria:** `WithCancellation(ct)` or manual check; describe connection pool impact if done wrong.
