# HTML Validation (W3C) — краткая выжимка

> `0.5.16` · ~10 мин на повторение

## Главная идея

**W3C Nu Html Checker** (validator.w3.org) сравнивает source с HTML spec **content models**. Browser «чинит» broken markup молча — validator показывает расхождение до production bugs в DOM/CSS/a11y.

## Ключевые пункты

- Режимы: **URI**, **file upload**, **direct input**; для local — upload/paste.
- **Errors** — fix before ship; **warnings** — triage (obsolete attrs → CSS в `0.6`).
- Read **line:column** → fix → **re-validate** до zero errors.
- Skeleton: `<!DOCTYPE html>`, `lang`, charset, **`<title>`**, valid nesting.
- Duplicate **`id`** — error; произвольные attrs — только **`data-*`** (`0.5.15`).
- Validate **rendered HTML** (SSR, View Source), не `.cshtml`/`.tsx` source.
- Optional: **html5validator** CLI для CI.

## Самые важные примеры

```html
<!-- ERROR -->
<ul><div>item</div></ul>
<!-- FIX -->
<ul><li>item</li></ul>
```

Workflow: validator.w3.org → Upload → Check → fix lines → repeat.

## Частые ошибки

Пропуск validation «потому что Chrome OK». Nested interactive elements. Unescaped `&`.

## Мнемоника

> **Validate early** — spec lint дешевле, чем debug «почему querySelector не тот element».
