# Answers: Span, ReadOnlySpan, Memory

---

### 1. [Recall]
**Answer:** **`Memory<T>`** can be stored and used with **`await`** APIs; **`Span<T>`** is ref struct — **not** across async suspension.
**Why:** ref struct stack lifetime rules.

### 2. [Recall]
**Answer:** Struct that must live on **stack** (mostly) — cannot box/async/escape unsafely.
**Why:** Protects interior pointers.

### 3. [Recall]
**Answer:** **`array.AsSpan()`** or **`AsSpan(start, length)`**.
**Why:** Extension on arrays.

### 4. [Understanding]
**Answer:** View references **string's internal UTF-16** buffer directly.
**Why:** No new allocation for view.

### 5. [Understanding]
**Answer:** Async method may **resume on different stack**; Span would be invalid.
**Why:** ref struct cannot be field in state machine across await.

### 6. [Understanding]
**Answer:** **Yes** if backed by same array — Span is window; writes visible.
**Why:** Shared backing store.

### 7. [Application]
**Answer:** **`int.TryParse(ReadOnlySpan<char>, out int)`**.
**Why:** BCL overload since .NET Core.

### 8. [Application]
**Answer:** **`ReadOnlyMemory<byte>`** / **`Memory<byte>`** with **`WriteAsync`**.
**Why:** Async needs Memory.

### 9. [Analysis]
**Answer:** **Invalid** — returns Span to **freed stack** after return.
**Why:** Classic use-after-free pattern.

### 10. [Analysis]
**Answer:** **No** — allocates **new array** and copies.
**Why:** Explicit copy API.

### 11. [Application]
**Answer:** Check if **Memory** wraps **array segment** without copy when calling array-based APIs.
**Why:** Memory may be array-backed or owned.

### 12. [Recall]
**Answer:** **ReadOnly** when must not mutate (string chars, read-only parse); **Span** when need write (fill buffer).
**Why:** Type-enforced mutability.

### 13. [Understanding]
**Answer:** Could **outlive** stack-backed or interior pointer **unsafely** on heap object.
**Why:** ref struct heap restriction (C# 12 ref fields excepted).

### 14. [Application]
**Answer:** **`ReadOnlySpan<char>`** trim slice `[start..end]`, **`ToString()`** once at end.
**Why:** Defer allocation.

### 15. [Analysis]
**Answer:** **`Span`** / **`AsSpan`** — unified, modern BCL.
**Why:** ArraySegment legacy.
