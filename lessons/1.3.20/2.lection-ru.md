# yield return — паттерн итератора

> Roadmap: `1.3.20` · Node: `1.3` — C#: продвинутые темы · Depth: **глубоко**

## Learning Objectives

После урока ты сможешь:

- Объяснить, как **`yield return`** реализует **`IEnumerable<T>`** / **`IEnumerator<T>`** без ручных iterator classes.
- Описать **lazy evaluation**, **deferred execution** и момент реального выполнения.
- Понимать **compiler-generated state machine** (`MoveNext`, `_state`, lifted locals).
- Использовать **`yield break`**, **`try/finally`**, **`using`** для cleanup ресурсов.
- Избегать **multiple enumeration**, mutation коллекции при iteration, утечки семантики через **`IEnumerable`** API.
- Сравнить iterator blocks с LINQ, **`ToList()`**, **`IAsyncEnumerable<T>`**.

## Why This Matters

Не каждую последовательность нужно материализовать в **`List<T>`** сразу. Логи на миллионы строк, бесконечные ряды, paginated API, pipeline над большими данными — **pull-based** iteration: следующий элемент только по запросу consumer. **`yield return`** — syntactic sugar над **iterator pattern**: пишешь loop, компилятор emit'ит state machine с pause/resume между элементами.

LINQ построен на этом: **`Select`**, **`Where`**, твои **`yield`**-методы — lazy pipelines. Непонимание laziness → double DB hit при втором **`foreach`**, файл не закрыт, потому что enumeration не дошла до конца. Middle dev пишет CSV readers, tree traversal, chunked EF reads — везде **`IEnumerator.MoveNext()`**, видишь **`yield`** или нет.

## Core Concepts

### Iterator pattern в .NET

- **`IEnumerable<T>`** — «можно получить enumerator».
- **`IEnumerator<T>`** — **`MoveNext()`**, **`Current`**, **`Dispose`**.

Ручной iterator — verbose class с index state. **`yield return`** генерирует его из method body.

```csharp
public static IEnumerable<int> Range(int start, int count)
{
    for (int i = 0; i < count; i++)
        yield return start + i;
}
```

**`Range(1, 3)`** не запускает loop. Loop идёт при **`GetEnumerator()`** + **`MoveNext()`** ( **`foreach`** ).

Каждый **`yield return`** **паузит** method, отдаёт value, **продолжает** на следующем **`MoveNext()`**. Locals и позиция loop сохраняются в fields generated iterator object.

### Lazy vs Eager

**Lazy:** work на каждом **`MoveNext()`**. **Eager:** **`ToList()`**, **`Count()`**, **`First()`** — полная evaluation.

```csharp
IEnumerable<string> ReadLinesLazy(string path)
{
    using var reader = new StreamReader(path);
    string? line;
    while ((line = reader.ReadLine()) != null)
        yield return line;
}
```

До **`foreach`** файл не открыт. Остановка после двух строк — остаток не читается. Никто не enumerate — body может не выполниться; **`using`** cleanup — при complete/dispose enumeration.

### yield break

**`yield break`** — конец sequence (не value **`return`** из iterator).

```csharp
foreach (var x in source)
{
    if (x <= 0) yield break;
    yield return x;
}
```

Early exit **`foreach`** → **`Dispose()`** enumerator → **`finally`** в **`yield`** method.

### Ограничения iterator methods

Return type: **`IEnumerable`/`IEnumerable<T>`/`IAsyncEnumerable<T>`/`IEnumerator`/`IEnumerator<T>`**. Нет **`ref`/`out`** parameters (sync). **`yield` в lambda** напрямую нельзя; local function — отдельные правила. Sync **`yield`** + **`await`** — только через **`IAsyncEnumerable`**.

## Under the Hood

Roslyn → nested class/struct с **`IEnumerator<T>`**, поля **`_state`**, **`_current`**, lifted locals. **`MoveNext()`** — **`switch`** по state: method **не** остаётся на real call stack между элементами (в отличие от некоторых coroutine models).

**Multiple enumeration** → **новые** iterator instances. Shared mutable state / single-use stream → второй pass ломается.

**`finally`** выполняется при: normal complete; **`Dispose()`** ( **`foreach`** ); exception из **`MoveNext()`**. Enumeration не начата → **`finally`** может не run — side effects до first **`yield`** опасны.

**`IEnumerable<T>`** в public API = lazy contract. Document single enumeration или return **`IReadOnlyList<T>`**.

## Examples

### DFS по дереву

```csharp
public static IEnumerable<TreeNode> Descendants(TreeNode root)
{
    foreach (var child in root.Children)
    {
        yield return child;
        foreach (var d in Descendants(child))
            yield return d;
    }
}
```

Повторный **`foreach`** — новый обход. Для cache — **`ToList()`** once.

### Chunk

```csharp
public static IEnumerable<IReadOnlyList<T>> Chunk<T>(IEnumerable<T> source, int size)
{
    List<T>? bucket = null;
    foreach (var item in source)
    {
        bucket ??= new List<T>(size);
        bucket.Add(item);
        if (bucket.Count == size) { yield return bucket; bucket = null; }
    }
    if (bucket is { Count: > 0 }) yield return bucket;
}
```

## Common Mistakes & Anti-patterns

- Вызвал **`GetLines()`** — думал, body уже выполнился.
- Double enumeration expensive source.
- **`foreach`** + mutation collection.
- **`yield` в catch`** — запрещено языком.
- Public **`IEnumerable`** без docs для single-use.
- **`yield`** для `[1,2,3]` — overkill.

## Production & Real-World Notes

**`IAsyncEnumerable`** в ASP.NET для streaming rows; **`DbContext`** lifetime до конца enumeration. Hot paths — **`for`/`Span`**, не **`yield`**. Tests — materialize **`ToList()`**.

## Comparison / Trade-offs

| Подход | Память | Когда |
|---|---|---|
| **`yield`** | O(1) если early stop | Streams, pipelines |
| **`ToList()`** | O(n) | Multiple passes |
| **`IAsyncEnumerable`** | Async stream | DB/API |
| **`for` on array** | Minimal | Hot loops |

## Key Takeaways

- **`yield`** = state machine **`IEnumerator<T>`**.
- Lazy до **`MoveNext()`**.
- Resources → **`Dispose`/`finally`**.
- Double enum = double work или break.
- Snapshot → **`List`**, async stream → **`IAsyncEnumerable`**.
