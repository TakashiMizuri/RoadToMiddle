# Exercises: Git Clean

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Dry-run discipline
**Goal:** Build the preview-first habit.

**Steps:**
1. Init repo, commit `readme.md`.
2. Create untracked `junk.txt` and `tmp/note.txt` inside untracked dir `tmp/`.
3. Run `git clean -n` — what is shown?
4. Run `git clean -nd` — what changed in output?
5. Run `git clean -fd` only after verifying preview.
6. Confirm `git status` clean except tracked files.

**Success criteria:** You never ran `-f` without `-n` first; documented difference between `-n` and `-nd`.

---

### E2. clean vs restore
**Goal:** Pick the right tool.

**Steps:**
1. Modify tracked `readme.md` (don't commit).
2. Create untracked `scratch.txt`.
3. Use correct command to discard `readme.md` changes only.
4. Use correct command to delete `scratch.txt`.
5. Verify final state.

**Success criteria:** `readme.md` matches last commit; `scratch.txt` gone.

---

## Medium

### M1. .gitignore interaction
**Goal:** See default vs `-x`.

**Steps:**
1. Add to `.gitignore`: `artifacts/`
2. Create `artifacts/build.log` and untracked `visible.tmp`.
3. `git clean -nd` — list what would be removed.
4. `git clean -ndx` — list with ignored included.
5. Execute appropriate clean for `visible.tmp` only (no `-x` unless intentional).
6. Confirm `artifacts/` still exists until you optionally `-fdx`.

**Success criteria:** Written explanation why `artifacts/` survived step 5.

---

### M2. Simulated .NET cleanup
**Goal:** Realistic artifact clean.

**Steps:**
1. Create fake structure: `bin/Debug/app.dll`, `obj/project.assets.json` (untracked).
2. Add `bin/` and `obj/` to `.gitignore`, commit `.gitignore`.
3. Preview: `git clean -ndx`
4. Execute: `git clean -fdx`
5. Verify directories gone.

**Success criteria:** Tree has no `bin/` or `obj/`; you noted rebuild would be needed in real project.

---

## Hard (stretch)

### H1. Near-miss postmortem
**Goal:** Production mindset.

**Scenario:** A developer ran `git clean -fdx` in monorepo root without `-n`. Deleted `.env.local`, `node_modules/`, and 2 hours of untracked SQL migration scripts.

Write a short postmortem (8–10 bullets): what went wrong, prevention rules, possible recovery options (IDE history, stash alternative, backups).

**Success criteria:** Mentions dry-run, stash -u for uncertain work, `.env` risk.

---

### H2. Interactive clean
**Goal:** Selective deletion.

**Steps:**
1. Create five untracked files: `keep-me.txt`, `a.tmp`, `b.tmp`, `c.tmp`, `d.tmp`.
2. Run `git clean -i`
3. Use interactive menu to remove only `*.tmp` files, keep `keep-me.txt`.
4. Verify result.

**Success criteria:** Only intended files removed; you describe which menu options you used.

## Hints (read only if stuck)

<details>
<summary>Hint E1 step 3 vs 4</summary>
`-n` alone may not list files inside untracked directories. `-d` tells Git to treat untracked dirs as removable units.
</details>

<details>
<summary>Hint M1 step 5</summary>
`git clean -fd` removes `visible.tmp` but skips `artifacts/` because ignored paths are protected unless `-x` is present.
</details>

<details>
<summary>Hint H2</summary>
In interactive mode, look for options like "clean files" / filter by pattern depending on Git version. Alternative: delete explicitly with `git clean -f -- a.tmp b.tmp` if interactive UI is confusing.
</details>
