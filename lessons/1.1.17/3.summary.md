# Enums: flags, parsing, TryParse — краткая выжимка

> `1.1.17` · ~8 мин на повторение

## Главная идея

**Enum** — типизированные целые константы. **`[Flags]`** превращает enum в **bitmask** для комбинируемых значений; **`Enum.TryParse`** безопасно переводит строки из HTTP/JSON/config во внутренние типы.

## Ключевые пункты

- Underlying type по умолчанию **`int`**; значение enum **есть** это число.
- **`[Flags]`**: степени двойки (`1 << n`), явный **`None = 0`**.
- Combine: **`|`**; test: **`(v & flag) == flag`** (предпочтительнее **`HasFlag`** в hot code).
- **`Enum.Parse`** — throws; **`Enum.TryParse`** — **`bool` + out** на границах API.
- **`ignoreCase: true`** для query/config.
- Flags parse: `"Read, Write"` → combined value.
- **`Enum.IsDefined`** — только **exact** named member, не каждая bit-комбинация.
- **`default(TEnum)`** = 0 — часто первый член или unnamed zero.

## Самые важные примеры

```csharp
[Flags]
public enum FileAccess { None = 0, Read = 1, Write = 2, Execute = 4 }

FileAccess a = FileAccess.Read | FileAccess.Write;
bool ok = (a & FileAccess.Read) == FileAccess.Read;

if (!Enum.TryParse<OrderStatus>(input, true, out var status))
    return BadRequest();
```

## Частые ошибки

- Sequential `1,2,3` на flags → коллизии при `|`.
- **`Parse`** на user input → 500.
- **`IsDefined`** для проверки произвольных flag combos.
- Renumber enum values в БД после релиза.

## Мнемоника

> **Flags — степени двойки; границы API — TryParse, не Parse.**
