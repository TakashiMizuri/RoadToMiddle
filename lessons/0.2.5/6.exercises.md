# Exercises: Conflict Resolution

> Complete after lectures and passing `4.test-yourself.md`.

## Easy

### E1. Marker anatomy
Given sample conflict block, label which section is HEAD, which is incoming, what you must do before `git add`.

**Success criteria:** Identifies three regions; states remove all markers.

---

## Medium

### M1. Manufactured conflict
In test repo: two branches change same line in `README.md`. Merge; resolve to combined text; complete merge. Record commands.

**Success criteria:** No markers in final file; merge commit exists.

---

### M2. Rebase conflict drill
Same setup but `git rebase` instead of merge. Note which side HEAD represents in markers. Complete with `--continue`.

**Success criteria:** Can explain inversion vs merge.

---

## Hard

### H1. Lockfile policy
Simulate lock conflict (or describe steps): regenerate via package manager, add, continue. Write 4-step CONTRIBUTING rule.

**Success criteria:** No hand-edited JSON markers.

---
