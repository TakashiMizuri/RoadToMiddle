# Exercises: Covariance and Contravariance

> Complete after passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Assignability Checklist
**Goal:** For each pair, write **OK** or **Error** and one-line reason: (a) `IEnumerable<Dog>` → `IEnumerable<Animal>`, (b) `Action<Animal>` → `Action<Dog>`, (c) `List<Dog>` → `IEnumerable<Animal>`, (d) `Func<Dog, string>` → `Func<Animal, string>`.
**Success criteria:** 4/4 correct with variance terminology.

### E2. Read-Only Interface
**Goal:** Declare `IReadOnlyStack<out T>` with `TryPop(out T item)` — does it compile? Fix if not.
**Success criteria:** Explain why `out T` conflicts with `out` parameter modifier and find valid API (`T Peek()`, `bool TryPeek(out T value)`).

## Medium

### M1. Contravariant Handler
**Goal:** Implement `void RegisterHandler(Action<object> handler)` and pass it where `Action<string>` is required via contravariance.
**Success criteria:** Demonstrates assignment without cast.

### M2. Pipeline Return Type
**Goal:** Method `IEnumerable<Animal> GetAllAnimals()` returns dogs from `List<Dog>` without copying to `List<Animal>`.
**Success criteria:** Uses covariant return type or assign `List<Dog>` to `IEnumerable<Animal>`.

## Hard (stretch)

### H1. Design API Surface
**Goal:** Sketch repository interface: mutable `Add(T)` internal, public `GetAll()` returns covariant type. Split into two interfaces if needed (`IWriteRepo` invariant + `IReadRepo<out T>`).
**Success criteria:** No unsound `out T` with input methods.
