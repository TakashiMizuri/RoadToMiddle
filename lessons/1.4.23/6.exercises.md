# Exercises: Awaitable Pattern

> Complete after lectures and passing `4.test-yourself.md`. Hints only — not full solutions.

## Easy

### E1. SharpLab inspection
**Goal:** See compiler lowering of `await`.
**Steps:** Paste async method with one `await Task.Delay(1)` into SharpLab (C# → IL or decompiled C#). Locate state machine and awaiter calls.
**Success criteria:** Annotate 3 lines/comments explaining `IsCompleted` check and continuation registration.

### E2. Manual GetResult
**Goal:** Await without `await` keyword once.
**Steps:** Complete a Task with `GetAwaiter()` / `IsCompleted` / `GetResult()` in console app (no `async`/`await` in that method).
**Success criteria:** Same output as `await Task.FromResult(42)`.

## Medium

### M1. TaskCompletionSource bridge
**Goal:** Wrap timer callback in awaitable Task.
**Steps:** Simulate legacy API firing `Action` after delay; expose `Task<int>` via TCS; consumer uses `await`.
**Success criteria:** Exception path uses `SetException`; await propagates it.

### M2. Mini YieldAwaitable
**Goal:** Implement struct awaitable that always resumes on thread pool.
**Steps:** Implement pattern from lecture; `await` your type mid-method; log `Thread.CurrentThread.ManagedThreadId` before/after.
**Success criteria:** Thread id may change after yield.

## Hard (stretch)

### H1. ConfigureAwait experiment
**Goal:** Document behavior difference in UI vs console (if UI available) or explain expected diff from lecture.
**Steps:** Library method with `ConfigureAwait(true)` vs `false` called from simulated SyncContext (optional: single-thread context test).
**Success criteria:** Paragraph explaining when continuation runs relative to caller context.

## Submission checklist

- [ ] No `async void` except event handler demo
- [ ] Documented whether custom awaiter uses Safe vs Unsafe OnCompleted
- [ ] ValueTask exercise (if attempted): single await only
