## Answers

### 1. [Recall]
**Answer:** **2.4.1 Bypass Blocks** (Level A).
**Why:** Skip links are primary technique to bypass repeated blocks.

### 2. [Recall]
**Answer:** **4.5:1**.
**Why:** WCAG 1.4.3 Contrast (Minimum) Level AA for normal text.

### 3. [Recall]
**Answer:** Чтобы после активации skip **focus переместился** на main landmark, а не только scroll; иначе AT может остаться на skip link.
**Why:** Programmatic focus on target landmark.

### 4. [Understanding]
**Answer:** **Keyboard** users и часто **screen reader** users, обходящие repetitive header/nav без десятков Tab.
**Why:** Bypass blocks benefit sequential keyboard navigation.

### 5. [Understanding]
**Answer:** `:focus` — любой focused element; `:focus-visible` — когда нужен **visible indicator** (typically keyboard, not always mouse click).
**Why:** UA heuristics reduce mouse focus ring noise while keeping keyboard ring.

### 6. [Understanding]
**Answer:** Color-blind/low vision users не различают; fails **1.4.1 Use of Color**; нужен text/icon/programmatic state.
**Why:** Color cannot be sole carrier of meaning.

### 7. [Application]
**Answer:**
```html
<a href="#main" class="skip-link">Skip to main content</a>
<main id="main" tabindex="-1">...</main>
```
**Why:** href matches id; tabindex enables focus target.

### 8. [Application]
**Answer:** **3:1** (large text threshold).
**Why:** WCAG large text definition — 18pt+ or 14pt+ bold.

### 9. [Analysis]
**Answer:** **WCAG 2.4.7 Focus Visible** failure — keyboard users не видят, где focus.
**Why:** Must replace removed outline with visible focus style.

### 10. [Analysis]
**Answer:** Fragment не найдёт target — no scroll/focus jump to main; broken skip.
**Why:** id must exist on destination.

### 11. [Understanding]
**Answer:** Low contrast; placeholder disappears on input; не persistent accessible name; **labels** required for forms.
**Why:** 1.4.3 + 3.3.2 Labels or Instructions.

### 12. [Understanding]
**Answer:** **2.4.7 Focus Visible**.
**Why:** Keyboard focus indicator criterion.

### 13. [Application]
**Answer:** Inspect text element → **Accessibility** pane в DevTools → Contrast ratio section (or color picker AA/AAA indicators).
**Why:** Built-in contrast audit on selected node.

### 14. [Understanding]
**Answer:** Не clutter UI для mouse users; link **available on first Tab** with visible high-contrast style when focused.
**Why:** Common skip link UX pattern.
