# Exercises: Immediate Execution

> Complete after `1.3.9`–`1.3.10` lectures and test.

## Easy

### E1. Snapshot isolation
**Goal:** See materialization effect.
**Steps:** `List<int>`, build `Where` query, `ToList`, mutate original list, compare foreach on query vs snapshot.
**Success criteria:** Different outputs explained.

### E2. Any vs Count
**Goal:** Short-circuit.
**Steps:** Large sequence (100k+), predicate expensive; compare `Any()` vs `Count()>0` timing (rough).
**Success criteria:** Observation note.

## Medium

### M1. Single materialization
**Goal:** Fix double scan.
**Steps:** Code with `Count()` then `ToList()` on same query — refactor to one materialization.
**Success criteria:** Same behavior, one enumeration.

### M2. API boundary sketch
**Goal:** ASP.NET pattern (pseudo or minimal Web API).
**Steps:** Repository returns `IQueryable` internally; service calls `ToListAsync` before return to controller.
**Success criteria:** No `IQueryable` crosses controller return type.

## Hard (stretch)

### H1. Pagination terminal
**Goal:** Correct `ToList` placement.
**Steps:** Implement in-memory pagination with `OrderBy`, `Skip`, `Take`, `ToList`. Wrong: `ToList` before `Skip`.
**Success criteria:** Page size correct on 100+ items.
