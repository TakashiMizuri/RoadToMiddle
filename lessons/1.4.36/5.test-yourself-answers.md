## Answers

### 1. [Recall]
**Answer:** Singleton-реализация `TimeProvider`, делегирующая реальному системному времени (OS clock, реальные `Task.Delay` и таймеры).
**Why:** Это production default; в host ASP.NET Core регистрируется автоматически.

### 2. [Recall]
**Answer:** `Microsoft.Extensions.TimeProvider.Testing` (namespace `Microsoft.Extensions.Time.Testing`).
**Why:** `FakeTimeProvider` не в BCL, а в testing extensions.

### 3. [Recall]
**Answer:** `Advance(TimeSpan)` (также есть `AdvanceTo` / `SetUtcNow` для абсолютного момента).
**Why:** Advance двигает fake timeline и срабатывает due callbacks.

### 4. [Understanding]
**Answer:** Тест вынужден ждать wall-clock время → медленный CI; или разработчик подменяет delays в DEBUG → тест не проверяет prod-логику.
**Why:** Статический `Task.Delay` не контролируется тестом.

### 5. [Understanding]
**Answer:** Pending `Task` от `timeProvider.Delay` завершается (continuation/state machine продвигается), await в тестируемом коде может быть дождан без реального sleep.
**Why:** Fake provider имитирует истечение таймера на своей шкале.

### 6. [Understanding]
**Answer:** `CancelAfter` использует внутренний system timer, не привязанный к injected `TimeProvider`.
**Why:** Для детерминизма таймаутов используй `await timeProvider.Delay(timeout, ct)` в своей обёртке.

### 7. [Application]
**Answer:** Production: `services.AddSingleton(TimeProvider.System);` Тест: `services.AddSingleton<TimeProvider>(new FakeTimeProvider());`
**Why:** Один интерфейс — разные реализации per environment.

### 8. [Application]
**Answer:** `fake.SetUtcNow(expiresAt.AddTicks(1));` затем `Assert.True(validator.IsExpired(expiresAt));`
**Why:** Прыжок fake clock на момент после expiry без ожидания.

### 9. [Analysis]
**Answer:** Ветка с `DateTime.UtcNow` не подчиняется fake time → тест с `Advance` даёт ложные pass/fail; flaky около границ времени.
**Common mistake:** Частичная миграция на `TimeProvider`.

### 10. [Analysis]
**Answer:** Cross-test interference: один тест `Advance` ломает timing другого; race на shared state.
**Why:** Fake provider должен быть per-test instance.

### 11. [Understanding]
**Answer:** `Advance` — относительный сдвиг; `SetUtcNow` — установка абсолютного момента на timeline.
**Why:** Advance удобен для backoff; SetUtcNow — для «наступило expiry».

### 12. [Application]
**Answer:** Запустить операцию, последовательно `Advance(1s)`, дождаться/проверить 1-ю retry, `Advance(2s)`, `Advance(4s)`, считать вызовы mock API.
**Why:** Каждый Advance триггерит следующий due delay.
