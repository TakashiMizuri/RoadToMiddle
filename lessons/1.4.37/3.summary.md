# Debugging async — краткая выжимка

> `1.4.37` · ~7 мин на повторение

## Главная идея

Async ломает линейный call stack. **Tasks window** и **Parallel Stacks** в Visual Studio показывают состояние `Task` и блокировки; **.NET 6+** улучшает async stack traces в исключениях.

## Ключевые пункты

- После `await` код может быть на другом thread pool thread — физический stack неполный.
- **Debug → Windows → Tasks** — Status, Method, Faulted tasks.
- **Parallel Stacks** — связь threads и waiting tasks.
- Hang на `await` → Break All → ищи `.Result`/`.Wait()` на другом thread.
- **async void** — shallow stack, нет нормального Task в Tasks.
- VS Code: ограниченно; используй logging, `dotnet-dump`, OpenTelemetry.
- State machine frame: `<Method>d__N.MoveNext()` = граница await.

## Самые важные примеры

Workflow при зависании API:
1. Breakpoint на `await`
2. Tasks — Id операции
3. Break All если не вернулось
4. Parallel Stacks — SQL/network/sync block

## Частые ошибки

- Отладка только main request thread.
- Игнор Faulted tasks в списке.
- Step into `TaskAwaiter` вместо user code.

## Мнемоника

> **Break All, Tasks, Trace** — при hang сначала смотри все tasks, потом stacks.
