# Git clean — краткая выжимка

> `0.1.12` · ~7 мин на повторение

## Главная идея

**`git clean`** **безвозвратно удаляет untracked files** (и с `-d` — directories) с диска. **Tracked files не трогает.** Всегда начинай с **`git clean -n`** (dry-run).

## Ключевые пункты

- **Untracked** → `git clean`; **tracked changes** → `git restore`; **tracked removal** → `git rm`; **save untracked** → `git stash -u`.
- **`-f`** — force, без него удаления не будет (кроме `-n`).
- **`-d`** — включить untracked **directories**.
- **`-x`** — удалить и **ignored** files (`bin/`, `obj/`, `node_modules`) — осторожно!
- По умолчанию clean **уважает `.gitignore`** — ignored **не** удаляются без `-x`.
- **Golden workflow:** `git status` → `git clean -nd` → прочитать → `git clean -fd`.

## Самые важные примеры

**Безопасная очистка:**
```bash
git clean -nd    # preview
git clean -fd    # execute
```

**После build (.NET):**
```bash
git clean -ndx   # preview bin/, obj/
git clean -fdx   # удалить ignored artifacts
```

## «Под капотом» в одном абзаце

Git сканирует working tree: если path **не** в index и **не** ignored (или `-x`), помечает на удаление. `-f` — явное подтверждение irreversible операции. Ignored paths читаются из `.gitignore` / global exclude — default clean их пропускает, чтобы не снести `bin/` случайно.

## Частые ошибки

- `git clean -fdx` без preview — потеря `.env`, долгий `npm install`.
- Clean для discard **modified tracked** — нужен `git restore`.
- Clean вместо commit важной untracked работы.

## Мнемоника

> **Clean = смета для мусора Git не знает.** **-n** = «покажи, не трогай» · **-x** = «игнорируй .gitignore» (ядерная кнопка).
