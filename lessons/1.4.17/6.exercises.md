# Exercises: Channels — Producer/Consumer

> Complete after passing `4.test-yourself.md`.

## Easy

### E1. Ping-pong channel
**Goal:** Basic write/read.

**Steps:** Create bounded channel `Channel<int>(capacity 3). Writer writes 1..5; single consumer `ReadAllAsync` prints. Observe when writer awaits (log before/after each write).

**Success criteria:** All numbers received; writer pauses when buffer full.

### E2. Complete semantics
**Goal:** Understand shutdown.

**Steps:** After E1, call `Writer.Complete()` and verify consumer loop exits.

---

## Medium

### M1. Mini work queue
**Goal:** ASP.NET-style pattern.

**Steps:** `WorkQueue` with bounded channel (capacity 100), `EnqueueAsync`, `BackgroundService` consumer processing `WorkItem` with simulated delay. Register in DI sketch (pseudocode OK).

**Success criteria:** `Complete` on host shutdown described.

### M2. FullMode experiment
**Goal:** Compare drop vs wait.

**Steps:** Same producer rate, slow consumer; run with `Wait` vs `DropOldest`. Count processed vs written.

**Success criteria:** Explain trade-off in 3 sentences.

---

## Hard (stretch)

### H1. Two-stage pipeline
**Goal:** Chain channels.

**Steps:** Stage A writes raw strings; Stage B parses to int; Stage C sums. Bounded channels between stages (cap 10 each). What happens if C is slow?
