## Answers

### 1. [Recall]
**Answer:** **https://validator.w3.org/**
**Why:** Official Nu Html Checker (W3C).

### 2. [Recall]
**Answer:** **Validate by URI**, **Validate by File Upload**, **Validate by Direct Input** (paste).
**Why:** Three documented input modes on main page.

### 3. [Recall]
**Answer:** **`<title>`** (also acceptable: `<html lang>`, `<!DOCTYPE html>` as document-level requirement).
**Why:** Title element required in HTML5 document; skeleton lists it explicitly.

### 4. [Understanding]
**Answer:** Browser **repairs** markup via parsing algorithm; resulting DOM may differ from author intent — breaks selectors, a11y tree, SSR hydration.
**Why:** Validation catches source/spec mismatch before silent recovery.

### 5. [Understanding]
**Answer:** Template files contain server/JS syntax (`@Model`, JSX) — not HTML until rendered; validator parses wrong language.
**Why:** Only **output HTML** is what browsers and users receive.

### 6. [Understanding]
**Answer:** Fix all **errors** first; warnings triage second (obsolete attrs, etc.).
**Why:** Errors = non-conforming markup; warnings often non-blocking.

### 7. [Application]
**Answer:** Error: **`span` not allowed as direct child of `ul`** (content model violation). Fix: `<ul><li>Item</li></ul>`.
**Why:** `ul` children must be `li` (or script/template elements in some cases), not phrasing-only wrapper skipping `li`.

### 8. [Application]
**Answer:** (1) **Missing `<title>`** in head. (2) **Duplicate `id="x"`** on two elements.
**Why:** Both are conformance errors independent of each other.

### 9. [Analysis]
**Answer:** Error: attribute **`product-id` not allowed** on `p` (non-standard without `data-`). Fix: **`data-product-id="99"`**.
**Why:** Custom metadata must use `data-*` prefix per HTML5.

### 10. [Understanding]
**Answer:** `<!DOCTYPE html>` triggers **no-quirks (standards) mode** — consistent box model and CSS behavior vs quirks mode.
**Why:** Link to upcoming cascade/layout lessons in `0.6`.

### 11. [Application]
**Answer:** Example workflow: (1) Open validator.w3.org (2) Choose **File Upload** (3) Select `about.html` (4) Click **Check**, fix errors by line, re-upload until clean.
**Why:** No public URL needed for file mode.

### 12. [Analysis]
**Answer:** Validator may pass; **heading hierarchy** is a11y/SEO concern — run **axe/WAVE**, manual outline check (`Show outline` in validator helps skipped levels if headings present wrong order).
**Why:** Validation ≠ comprehensive accessibility audit.
