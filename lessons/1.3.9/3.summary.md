# Deferred execution — краткая выжимка

> `1.3.9` · ~8 мин на повторение

## Главная идея

LINQ на **`IEnumerable<T>`** по умолчанию **deferred**: pipeline — план; реальная работа при **enumeration** (`foreach`, **`ToList`**, **`Count`**, …). Повторный **`foreach`** = повторный проход pipeline с source.

## Ключевые пункты

- **Construction** ≠ **execution**.
- **Terminal/eager:** **`ToList`**, **`Count`**, **`First`**, **`Sum`**, **`Max`**, **`Aggregate`** — force run (`1.3.10`).
- **Multiple enumeration** — дорого; fix: **`.ToList()`** / **`.ToArray()`** once.
- **Mutable source** и **captured variables** влияют на результат at execution time.
- **Streaming:** `Where`, `Select`, `Take` · **Buffering:** `OrderBy`, `GroupBy`, `Distinct`, `Join`.
- Side effects в lambdas — непредсказуемое число вызовов.
- **`IQueryable`** — тот же deferral habit; execution = provider/SQL (`1.3.11`).

## Самые важные примеры

```csharp
var q = items.Where(x => x.Active); // ещё не scan
var list = q.ToList();              // один проход
```

```csharp
int min = 5;
var q = data.Where(x => x > min);
min = 10;
q.Count(); // порог 10, не 5
```

## «Под капотом» в одном абзаце

Operators — **`yield return`** iterators; nested enumerators pull from source on **`MoveNext`**. Results не кешируются (default). Buffering operators сначала накапливают elements (sort/hash), потом yield.

## Частые ошибки

- **`Any()`** then **`foreach`** — два scan
- Modify collection during enumeration
- EF: enumerate after **`DbContext`** disposed

## Мнемоника

> **Build the query, then pull once** — или **`.ToList()`** если pull > 1.
