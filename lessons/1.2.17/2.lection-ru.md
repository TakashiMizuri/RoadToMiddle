# Перегрузка операторов (основы)

> Roadmap: `1.2.17` · Node: `1.2` — C# OOP · Depth: понимание

## Learning Objectives

После этой лекции ты сможешь:

- Перечислить, какие операторы C# **можно** перегружать, а какие **запрещены**.
- Написать перегрузку **унарного** и **бинарного** оператора с корректной сигнатурой.
- Объяснить, почему перегрузка операторов — **опциональный sugar**, а не замена именованным методам.

- Отличать удачные доменные примеры (деньги, векторы) от вводящих в заблуждение перегрузок.
- Читать чужой код с операторами без сюрпризов на call site.

## Why This Matters

Ты пишешь `a + b` для целых чисел с первых программ. Символ `+` не встроен в язык для всех типов универсально — для `int` компилятор генерирует IL `add`. Для твоих `Money` или `Vector2` без твоего определения `left + right` — ошибка компиляции. С продуманным определением доменный код читается как постановка задачи: `total = price + tax` вместо `total = price.Add(tax)`.

Перегрузку операторов часто путают с «умным синтаксисом». В профессиональном C# это **узкий инструмент**: value objects естественно вписываются в выражения, а всё неоднозначное остаётся в **именованных методах**. BCL использует это постоянно — `string` с `+`, `TimeSpan` с `+`/`-`, полная арифметика у `decimal`. Плохой пример — `+` для «слияния баз данных»: нарушение principle of least surprise.

В `1.2.16` ты разобрал equality. Перегрузка `==` втягивает тебя в тот же мир контрактов, что `Equals`. Операторы — часть публичного API типа и должны быть **согласованы** с остальным типом.

## Core Concepts

### Что такое перегрузка операторов

**Operator overload** — **static method** со специальным именем (`operator +`, `operator ==`, …), которую компилятор подставляет вместо выражения. `c = a + b` превращается в вызов `operator +(A a, B b)` при наличии подходящей сигнатуры.

Операторы — **сyntactic sugar**. Новых runtime-возможностей они не дают. Всё, что выражено оператором, можно выразить `static` методом `Add`, `Equals`, `CompareTo` — операторы нужны для **читаемости**, когда смысл в домене очевиден.

C# ограничивает набор операторов. Нельзя изобрести `operator**`, перегрузить присваивание `=`, тернарный `?:`, доступ к члену `.`, `as`/`is` (отдельно существуют **conversion operators**).

### Какие операторы перегружаем (basics)

**Унарные:** `+`, `-`, `!`, `~`, `++`, `--`, `true`, `false` (для кастомных сценариев с `&&`/`||` — редко).

**Бинарные:** `+`, `-`, `*`, `/`, `%`, `&`, `|`, `^`, `<<`, `>>`, `==`, `!=`, `<`, `>`, `<=`, `>=`.

**Нельзя:** `=`, `.`, `?:`, `->`, `=>`, обычный вызов `f(x)` как operator (кроме implicit/explicit conversion).

В этой лекции фокус на **арифметике** (`+`, `-`) и **равенстве** (`==`, `!=`).

### Правила синтаксиса

Перегрузки — **`public static`**. Хотя бы один параметр — тип, в котором объявлен оператор (для conversion — свой тип как source или target).

Бинарный пример:

```csharp
public readonly struct Vector2
{
    public double X { get; }
    public double Y { get; }

    public Vector2(double x, double y) => (X, Y) = (x, y);

    public static Vector2 operator +(Vector2 a, Vector2 b) =>
        new(a.X + b.X, a.Y + b.Y);

    public static Vector2 operator -(Vector2 a, Vector2 b) =>
        new(a.X - b.X, a.Y - b.Y);
}
```

`var v = a + b` компилируется через `operator +`. Immutable struct — каждая операция возвращает **новый** экземпляр (`1.2.14`).

Унарный минус:

```csharp
public static Vector2 operator -(Vector2 v) => new(-v.X, -v.Y);
```

### Операторы равенства и контракт 1.2.16

Для **reference type** с value semantics перегружай `==`/`!=` **вместе** с `Equals`/`GetHashCode`:

```csharp
public static bool operator ==(Money? left, Money? right) =>
    left is null ? right is null : left.Equals(right);

public static bool operator !=(Money? left, Money? right) => !(left == right);
```

Если override `Equals` на class — дай `==`/`!=`, делегирующие в `Equals`. Пользователи ожидают `a == b` ↔ `a.Equals(b)`.

У **struct** `==` не определён, пока ты его не объявишь.

**Records** (`1.2.13`) генерируют `==`/`!=` автоматически.

### Conversion operators (awareness)

**Implicit** и **explicit** позволяют `(Target)source` или неявное преобразование:

```csharp
public static implicit operator decimal(Money m) => m.Amount; // осторожно
```

Неявная потеря контекста (валюта) опасна. Для рискованных преобразований — **explicit**. Глубокий разбор опционален; знай, что встретишь в BCL.

### Когда перегрузка уместна

Хорошие кандидаты: **маленькие immutable value types**, значение операции **закреплено конвенцией**, частое использование в **выражениях**.

- **Money:** `+`, `-`, унарный `-`.
- **Vector / Matrix:** `+`, `-`, `*` (для matrix multiply часто яснее static method).
- **Strongly typed ID:** обычно без арифметики; максимум `==`.

### Когда избегать

Неоднозначность, side effects, несимметричный смысл → именованный метод.

**Плохо:** `database + migration` = apply. **Хорошо:** `database.Apply(migration)`.

**Плохо:** `<` при настраиваемой сортировке. **Хорошо:** `IComparer<T>`.

Многие .NET-команды: **сомневаешься — `Add`, `Subtract`, `Combine`.** Операторы для типов, ведущих себя как числа/векторы, не как сервисы.

### Операторы vs static methods

Runtime-разницы нет; JIT инлайнит одинаково. Выбор — **читаемость API**.

## Examples

### Money — только одна валюта

```csharp
public static Money operator +(Money left, Money right)
{
    if (left.Currency != right.Currency)
        throw new InvalidOperationException("Currency mismatch");
    return new Money(left.Amount + right.Amount, left.Currency);
}
```

`+` фиксирует инвариант на месте выражения. `static Add` был бы равноценен; `+` идиоматичен для numeric-like типов.

### Не перегружай всё подряд

```csharp
var merged = invoice.Merge(adjustment); // предпочтительнее invoice + adjustment
```

## Common Mistakes & Anti-patterns

**`==` без `Equals`/`GetHashCode`** — тройная несогласованность.

**Мутация в operator** — для value objects возвращай **новый** экземпляр.

**Неожиданные исключения** — currency mismatch OK; null из-за аннотаций — нет.

**DSL-«умничание»** — сложно сопровождать.

**Порядок операндов** — документируй, если `A + B` ≠ `B + A` по смыслу.

## Production & Real-World Notes

В enterprise часто **запрещают** custom operators вне approved math/money библиотек — смотри style guide.

Тестируй операторы как публичные методы.

Аудит и логи не используют `+` — именованные методы лучше grep.

Source generators и records могут генерировать операторы — не редактируй вручную (`1.2.18`).

## Comparison / Trade-offs

| Подход | Плюсы | Минусы |
|--------|-------|--------|
| `operator +` | Краткие формулы | Сюрприз при неочевидном смысле |
| `static Add` | Явность | Многословность |
| Record `==` | Бесплатно | Только generated shape |

**Правило:** арифметика на numeric-like value types → operators OK; бизнес-workflow → methods.

## Key Takeaways

- Операторы — **static methods** с sugar компилятора; набор ограничен.
- Для **ясных** операций над value types (деньги, векторы).
- `==`/`!=` согласуй с **`Equals`/`GetHashCode`**.
- Неоднозначное — **именованные методы**.
- Immutable: операторы возвращают **новые** значения.
- Records генерируют equality operators.
- Следуй conventions команды.
