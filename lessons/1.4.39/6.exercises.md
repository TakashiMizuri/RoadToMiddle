# Exercises: Task vs ValueTask

## Easy

### E1. Sync path benchmark
**Goal:** Compare allocations Task vs ValueTask on cache hit.
**Steps:** 1M calls sync-complete path; dotMemory or `GC.GetAllocatedBytesForCurrentThread()`.
**Success criteria:** Document delta; explain when it matters.

### E2. AsTask fix
**Goal:** Fix double-await bug.
**Steps:** Repro failing code; apply `.AsTask()` or switch to `Task`.
**Success criteria:** Stable on repeated await.

## Medium

### M1. Cache service API
**Goal:** Design `GetUserAsync` with 80% cache hits.
**Steps:** Implement `ValueTask<User>`; caller single await; load test optional.
**Success criteria:** Code review passes decision tree justification.

### M2. WhenAll composition
**Goal:** Three `ValueTask` sources → one await all.
**Steps:** Materialize with `AsTask()`; measure readability vs all-Task API.
**Success criteria:** Correct results, no double-await.

## Hard (stretch)

### H1. Read loop
**Goal:** Implement `Stream`-like type returning `ValueTask<int>` from buffer sync path.
**Success criteria:** 4KB read loop with minimal Task allocs (conceptual or BenchmarkDotNet).
