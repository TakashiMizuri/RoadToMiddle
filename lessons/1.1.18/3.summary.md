# Nullable value types — краткая выжимка

> `1.1.18` · ~6 мин на повторение

## Главная идея

**`T?`** (`Nullable<T>`) добавляет value type третье состояние — «значение отсутствует» — без sentinel вроде `-1`. **`??`** задаёт default, когда **`HasValue`** false.

## Ключевые пункты

- **`int?`** ≡ **`Nullable<int>`** — struct: **`HasValue` + `Value`**.
- **`null`** → **`HasValue false`**; **`.Value`** бросает exception.
- **`??`** / **`??=`** — lazy default; правая часть не eval если left ok.
- **`GetValueOrDefault(d)`** — fallback без throw.
- **Lifted operators**: arithmetic → **`null`** если operand missing; **`==`** может дать **`bool?` null**.
- **`default(int?)`** = null; **`default(int)`** = 0.
- **NRT `string?`** — compile-time warnings; **не** **`Nullable<string>`**.

## Самые важные примеры

```csharp
int? stock = null;
int effective = stock ?? 0;

DateTime deadline = due ?? DateTime.UtcNow.AddDays(7);

if (published is bool p)
    q = q.Where(x => x.IsPublished == p);
```

## «Под капотом» в одном абзаце

**`Nullable<T>`** — value struct; **`??`** → IL с проверкой **`HasValue`**. Boxing только при **`HasValue`**. EF map **`NULL`** ↔ **`null`** **`T?`**.

## Частые ошибки

- Sentinel **`MinValue` / -1** вместо **`T?`**.
- Blind **`.Value`**.
- Считать lifted **`a == b`** всегда **`false`**, когда operand null.
- Путать **`string?`** (NRT) и **`int?`** (struct wrapper).

## Мнемоника

> **Нет значения — `null`, не магическое число. Default — `??`, не `.Value`.**
