# LOH (Large Object Heap) и fragmentation

> Roadmap: `1.5.8` · Depth: **глубоко**

## Learning Objectives

- Определить **LOH** и threshold **~85 000 bytes**.
- Объяснить immediate **Gen2** для large objects.
- Описать **fragmentation** и optional **LOH compaction**.
- Mitigation: **ArrayPool**, recyclable streams, streaming APIs.

---

## Why This Matters

Large `byte[]`, strings, MemoryStream buffers → **LOH**, collected only with **Gen2 GC**. **Fragmentation**: free bytes exist, но нет contiguous block → **OOM**.

Long-running services и upload endpoints — classic LOH pain.

---

## Core Concepts

### LOH Threshold

Single allocation ≥ ~85000 bytes → LOH (verify on your runtime). `new byte[85000]` LOH; `84999` SOH Gen0.

### Gen2 + Fragmentation

LOH freed on Gen2 GC but holes remain — **non-contiguous free list**. SOH compacts more readily; LOH historically rarely compact (optional compaction expensive).

### Mitigation

```csharp
var pool = ArrayPool<byte>.Shared;
var buf = pool.Rent(100_000);
try { /* use */ } finally { pool.Return(buf); }
```

**RecyclableMemoryStream**, chunking, streaming JSON.

---

## Key Takeaways

- Large alloc → LOH → Gen2 immediately.
- Fragmentation ≠ low total memory.
- Pool/stream beats giant single arrays.
- Completes SOH/LOH split from `1.1.6`.
