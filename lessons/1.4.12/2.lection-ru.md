# Task.WhenAll и WhenAny — parallel I/O

> Roadmap: `1.4.12` · Node: `1.4` — Async/await · Depth: **практика**

## Learning Objectives

После урока ты сможешь:

- Запускать **независимые I/O** параллельно через **`Task.WhenAll`**.
- Использовать **`Task.WhenAny`** для **first-completed** (timeout race, fallback).
- Прокидывать **`CancellationToken`** во все child operations (`1.4.11`).
- Избегать **sequential await** когда calls independent.
- Обрабатывать exceptions **`WhenAll`**: один fault — fail aggregate.
- Понимать: parallel async I/O ≠ extra threads на wait; continuations на pool (`1.4.7`).

---

## Why This Matters

Dashboard: profile + permissions + notifications. Sequential — sum latencies; **`WhenAll`** — ~**max**. Wrong exception handling; missing cancel — orphan HTTP.

Daily Middle work: aggregation, batch fetch, **`WhenAny`** degradation.

---

## Core Concepts

### Sequential vs parallel

```csharp
// ~ Ta + Tb + Tc
var a = await GetAAsync();
var b = await GetBAsync();

// ~ max(Ta, Tb, Tc)
var aTask = GetAAsync(ct);
var bTask = GetBAsync(ct);
await Task.WhenAll(aTask, bTask);
var a = await aTask;
```

Tasks **start** до await WhenAll. Три state machines interleave on completions.

### Task.WhenAll

- All success → complete когда **все** done.
- One fault → returned task faults.
- **`await WhenAll`** — first exception (не AggregateException в async await).

Pass token в каждый child.

### Task.WhenAny

First complete (success/fault/cancel). Timeout race, redundant providers.

После winner — **cancel losers** linked CTS.

### I/O vs CPU

**WhenAll** для I/O — OK. Unbounded **`Task.Run` + WhenAll`** — **starvation** (`1.4.10`). Limits: **`SemaphoreSlim`**, **`Parallel.ForEachAsync`**.

---

## Examples

```csharp
var profileTask = _users.GetProfileAsync(userId, ct);
var permsTask = _auth.GetPermissionsAsync(userId, ct);
await Task.WhenAll(profileTask, permsTask);
return new Dto { Profile = await profileTask, Permissions = await permsTask };
```

---

## Common Mistakes

- `.Result` до WhenAll complete (`1.4.14`).
- Ignore losing tasks после WhenAny.
- 10k CPU Task.Run + WhenAll.

---

## Key Takeaways

- Start tasks → **WhenAll** → await results.
- **WhenAny** — first wins; cancel rest.
- Token на каждый child.
- CPU parallel — bounded.
