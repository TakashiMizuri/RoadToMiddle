## Answers

### 1. [Recall]
**Answer:** **`Where(x => x.Age > 18)`**
**Why:** **`where`** always desugar to **`Where`**.

### 2. [Recall]
**Answer:** **`equals`** — часть join clause grammar; для **`IQueryable`** left/right становятся expression tree nodes с well-defined key comparison. **`==`** в join context не allowed.
**Why:** Spec + expression tree compatibility.

### 3. [Understanding]
**Answer:** **Нет.** На **`IEnumerable`**/`List` — in-memory LINQ to Objects, deferred iterators (`1.3.9`). SQL-like вид не означает SQL engine.
**Why:** Syntax ≠ execution provider.

### 4. [Application]
**Answer:**
```csharp
products.Where(p => p.Price > 0).OrderBy(p => p.Name).Select(p => p.Id)
```

### 5. [Application]
**Answer:**
```csharp
from o in orders
where o.Total > 100
select o.CustomerId
```

### 6. [Understanding]
**Answer:** **`g`** — **`IGrouping<TCountry, Order>`** (или соответствующие types). После **`into`** новый query scope; **`g.Key`** — country, **`g`** iterable orders in group.
**Why:** **`into`** introduces continuation for per-group projection.

### 7. [Application]
**Answer:** **`SelectMany`** — flatten **`users`** with **`Roles`**.
**Why:** Multiple **`from`** = nested iteration / flatten pattern.

### 8. [Analysis]
**Answer:** **Нет** — dot после **`select`** invalid. Правильно:
```csharp
(from x in items select x).Take(10)
// or
items.Take(10) in method syntax
```

### 9. [Understanding]
**Answer:** (examples) **join / group join**; **multiple from** flattening; long relational queries readable SQL-style.
**Why:** Query syntax optimized for relational patterns in spec.

### 10. [Application]
**Answer:**
```csharp
from o in orders
join c in customers on o.CustomerId equals c.Id
where o.Total > 500
select new { o.Id, CustomerName = c.Name }
```

### 11. [Recall]
**Answer:** **Нет** — use **`.Distinct()`** method.

### 12. [Analysis]
**Answer:** **A** desugars to **`Where` then `Select`**. **B** has extra identity **`Select`** before **`Where`** — usually equivalent for simple cases, but not identical desugaring; **A** is **`src.Where(...).Select(...)`**.
**Why:** Query **`where` before `select`** maps to filter before final projection.
