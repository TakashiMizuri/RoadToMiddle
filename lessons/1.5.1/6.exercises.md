# Exercises: CLR, Managed Code, and IL

> Complete after reading lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Inspect your first assembly
**Goal:** See IL produced from C#.
**Steps:** Create console app with `Add(int,int)` and `Main`. `dotnet build`. Open DLL in ILSpy or `ildasm`.
**Success criteria:** Find `Add` method IL; identify `ldarg`, `add`, `ret`; name one `call` to BCL method.

### E2. Pipeline diagram
**Goal:** Solidify compile vs runtime phases.
**Steps:** Draw diagram C# → Roslyn → assembly → CLR load → JIT → native. Label where metadata is read.
**Success criteria:** At least 6 labeled boxes; mention GC heap separately from IL.

## Medium

### M1. Managed vs unmanaged boundary
**Goal:** Articulate GC boundary.
**Steps:** Write 5-line explanation: P/Invoke allocates `IntPtr` with `Marshal.AllocHGlobal`. Who frees? What if never freed?
**Success criteria:** Mention GC does not track native block; need `FreeHGlobal`/`SafeHandle`.

### M2. Connect to 1.1.6
**Goal:** Link semantics to runtime.
**Steps:** For code with `int n`, `string s`, `new Customer()` — write which parts CLR places in stack frame vs heap (reference to lecture terms).
**Success criteria:** Object body on heap; reference slot in frame; `int` inline in frame.

## Hard (stretch)

### H1. BadImageFormat investigation
**Goal:** Practice deployment/runtime thinking.
**Steps:** Research 3 real causes of `BadImageFormatException` in Microsoft docs or GitHub issues. Summarize one paragraph each.
**Success criteria:** At least one bitness and one runtime mismatch example.
