# Exercises: Primitive Types

> Complete after lectures and passing `4.test-yourself.md`. Ask AI for hints, not full solutions.

## Easy

### E1. Literal zoo
**Goal:** Correct suffixes and bases.

In a console app, declare one variable of each type with a literal:
`int` (binary `0b1010`), `long` (`1_000_000L`), `float`, `double`, `decimal`, `bool`, `char` (`'\u263A'`).

Print `sizeof` for `int`, `long`, `decimal` (requires `unsafe` block **or** document values from lecture if unsafe disabled).

**Success criteria:** Compiles; all suffixes correct; output documented.

---

### E2. Parse boundary
**Goal:** Safe parsing.

Read no input — hard-code strings `"123"`, `"abc"`, `"99.5"`. Use `TryParse` for `int` and `decimal` (InvariantCulture for decimal). Print success/failure and parsed values.

**Success criteria:** `"abc"` fails gracefully; no unhandled exceptions.

---

## Medium

### M1. Money calculator
**Goal:** `decimal` discipline.

Given:
```csharp
decimal price = 10.00m;
int quantity = 3;
decimal taxRate = 0.08875m;
```

Compute line subtotal, tax, grand total. Print with two decimal places (`:F2`).

Duplicate logic with `double` and print both — observe difference (if any) for this case.

**Success criteria:** Decimal path correct to cents; short comment on double vs decimal.

---

### M2. Overflow lab
**Goal:** checked vs unchecked.

```csharp
int near = int.MaxValue - 1;
```

In unchecked context, print `near + 1` and `near + 2`. Wrap same additions in `checked` and catch `OverflowException`.

**Success criteria:** Observed wrap vs exception; explained in one sentence.

---

## Hard (stretch)

### H1. Box benchmark (conceptual)
**Goal:** Understand boxing cost.

Write a loop 1_000_000 iterations:
- A: sum `List<int>` with foreach
- B: sum `ArrayList` of boxed ints (if you add legacy collection or box manually)

No need for precise benchmark — note **whether** B allocates more (watch GC or reason from theory).

**Success criteria:** Paragraph: why generics avoid boxing for primitives.
