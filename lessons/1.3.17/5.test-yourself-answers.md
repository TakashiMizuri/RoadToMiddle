## Answers

### 1. [Recall]
**Answer:** **Compile-time** (Roslyn analysis); runtime без extra checks.
**Why:** Metadata annotations only.

### 2. [Recall]
**Answer:** `string` — non-nullable **intent** (warning if null assigned); `string?` — may be null.
**Why:** Dual type system for references.

### 3. [Recall]
**Answer:** **Null-forgiving** — suppresses nullable warning at that use; no runtime effect.
**Why:** Escape hatch for unprovable invariants.

### 4. [Understanding]
**Answer:** Method annotated **`DoesNotReturnIf(null)`** / `[NotNull]` — flow analysis knows execution stops if null, else `x` non-null.
**Why:** Interprocedural null-state.

### 5. [Understanding]
**Answer:** CS8618 — non-nullable property not initialized. Fix: **`required`**, **`= ""`**, or **`string?`** if optional.
**Why:** Constructor exit analysis.

### 6. [Application]
**Answer:** **Warning CS8600** — possible null assigned to non-nullable `string n`.
**Why:** GetName returns null.

### 7. [Application]
**Answer:** **Not null** (definitely not null) inside block for `s`.
**Why:** Pattern narrowing.

### 8. [Application]
**Answer:** When method returns **`true`** — `value` treated non-null per `[NotNullWhen(true)]` on `out`.
**Why:** Attribute-driven flow.

### 9. [Analysis]
**Answer:** **Anti-pattern** — double `!` without guards hides possible NRE; fix API return types or add checks.
**Why:** Defeats NRT purpose.

### 10. [Analysis]
**Answer:** Compile CS8618 + **runtime null** on `Email` after deserialize — NRE downstream. Use `required`, validation, or `string?`.
**Why:** Deserializer bypasses constructor guarantees.

### 11. [Understanding]
**Answer:** **`annotations`** — only `#` metadata, **no warnings**; **`enable`** — annotations + **warnings**.
**Why:** Migration path.

### 12. [Recall]
**Answer:** Dereference of a **maybe null** reference.
**Why:** Common CS8602.

### 13. [Application]
**Answer:** `string id` (required) vs `string? tag = null` optional — nullable annotation on optional; ASP.NET binds missing as null for `string?`.
**Why:** Signature documents contract.

### 14. [Understanding]
**Answer:** **Nullable reference type parameter** — `T` may be null instance for reference types.
**Why:** Generic nullability.

### 15. [Application]
**Answer:**
```csharp
if (text is null) return;
Console.WriteLine(text.Length);
// or
Console.WriteLine(text?.Length ?? 0);
// or
ArgumentNullException.ThrowIfNull(text);
Console.WriteLine(text.Length);
```
**Why:** Narrow or guard before dereference.
