# Exercises: UDP

> Complete after lectures and passing `4.test-yourself.md`.

## Easy

### E1. TCP vs UDP picker
**Goal:** Apply decision matrix.

For each workload choose **TCP**, **UDP**, or **Either with justification**:

1. Stripe webhook delivery
2. DNS A record lookup
3. 4K live stream CDN edge
4. EF Core to PostgreSQL

**Success criteria:** Four choices with one-line why.

---

### E2. Firewall checklist
**Goal:** Production UDP awareness.

List minimum UDP ports/rules for: public web app with HTTP/3, internal K8s cluster DNS (CoreDNS), developer laptop general browsing.

**Success criteria:** At least UDP 443 and 53 mentioned where relevant.

---

## Medium

### M1. DNS transport observation
**Goal:** Bridge to 0.4.5.

Run `nslookup -debug example.com` or `dig +tcp example.com` / `dig example.com`. Note UDP vs TCP. When would TC bit force TCP?

**Success criteria:** Describes size/truncation trigger.

---

## Hard (stretch)

### H1. Design hybrid game protocol
**Goal:** Synthesis UDP + TCP.

Sketch channels: login/inventory (reliable), movement (unreliable). Why not single TCP connection for all?

**Success criteria:** Mentions HOL blocking and stale state on TCP for movement.
